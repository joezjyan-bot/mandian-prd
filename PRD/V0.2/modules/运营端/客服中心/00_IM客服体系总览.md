# IM 客服体系总览

> **V0.2.1 开发冻结同步(2026-05-27)**: IM 客服基础能力进入 V1。本文定义“怎么接入、怎么使用、怎么和订单/工单/权限打通”。IM 不是独立聊天插件,而是统一客服中心;高危业务动作必须回到原业务模块执行权限校验、二次确认和日志。

---

## 1. 定位

IM 客服服务四类对象:

1. C 端客户:安心用长租、体验租短租、支付、合同、取还车、账单、保证金、售后咨询。
2. 商家 PC 端:订单、结算、提现、员工权限、商品、体验租车辆问题。
3. 门店手机端:办单、发货、体验租接单、取车核销、还车验收、控车异常。
4. 运营后台:客服工作台、会话分配、工单处理、归档检索、质检和日志。

V1 的目标不是先做复杂机器人,而是先做到:

- 入口统一。
- 会话带上下文。
- 客服能快速跳业务页面。
- 无法即时解决可转工单。
- 所有消息和处理过程可归档。
- 高危动作不在 IM 内绕过权限。

---

## 2. 接入策略

V1 推荐采用“自有客服中心抽象 + 渠道适配层”:

| 端 | V1 接入方式 | 要求 |
|---|---|---|
| 支付宝小程序 | 可使用官方客服能力或第三方 IM SDK 适配 | 必须回传 `customer_id`、`order_id`、`business_line` |
| 微信小程序 | 可使用微信客服能力或第三方 IM SDK 适配 | 不允许形成运营后台不可见的孤立会话 |
| H5 | 平台自建 IM 或第三方 SaaS | 必须接入统一客服中心 |
| App | 后期预留 | 使用同一 `customer_external_accounts` 体系 |
| 商家/门店端 | 平台自建客服入口 | 默认联系平台客服 |

无论底层使用官方 SDK、第三方 SaaS 还是自建 IM,运营端统一落到:

- `customer_im_sessions`
- `customer_im_messages`
- `customer_tickets`
- `ticket_events`

---

## 3. C 端入口

| 页面 | 入口 | 上下文 |
|---|---|---|
| 底部 Tab | 客服 | 最近客户、最近订单、最近业务线 |
| 安心用订单详情 | 联系客服 | 长租订单号、状态、商品、商家/门店、当前待办 |
| 体验租订单详情 | 联系客服 | 短租订单号、车辆、门店、取还时间、剩余时间 |
| 支付失败页 | 联系客服 | 支付单、失败原因、通道、订单号 |
| 合同/公证异常页 | 联系客服 | 合同状态、公证状态、订单号 |
| 取车/还车页 | 联系商家 / 联系客服 | 门店、核销码、车辆码、当前状态 |
| 保证金扣款争议页 | 联系客服 | 扣款项、证据、处理状态 |

客户从任一入口进入会话时,系统自动生成上下文卡片,客服不应再让客户手工重复提供订单号。

---

## 4. 商家端和门店端入口

商家端:

- 订单详情:联系平台客服。
- 我的钱包/提现:联系财务客服。
- 商品/套餐配置:联系运营客服。
- 体验租车辆/订单异常:联系体验租客服。

门店手机端:

- 办单助手异常:联系平台客服。
- 发货/交付异常:联系平台客服。
- 体验租接单、取车、还车、控车异常:联系平台客服。

V1 不默认开放“客户与商家自由 IM 私聊”。体验租可以保留“联系商家”电话/导航入口;如果后续开放客户-门店聊天,必须先补充会话归档、话术合规和投诉留证规则。

---

## 5. 运营端客服工作台

### 5.1 会话列表

会话列表字段:

- 客户脱敏姓名/手机号。
- 来源端:支付宝、微信、H5、App 预留。
- 业务线:安心用 / 体验租 / 商家咨询。
- 关联订单号。
- 当前状态:排队中、处理中、待客户回复、待内部处理、已转工单、已结束。
- 优先级。
- 最近消息。
- 等待时长。
- 当前客服。

### 5.2 会话详情

会话详情分四区:

1. 消息区:文字、图片、视频、订单卡片、系统事件。
2. 客户区:脱敏资料、实名状态、历史订单、黑名单/异常提醒。
3. 订单上下文区:安心用订单或体验租订单快照。
4. 操作区:快捷回复、转接、转工单、跳转订单详情、跳转退款、跳转扣款、跳转控车日志。

操作区只提供跳转,不直接执行高危动作。

## 6. 分流规则

| 队列 | 进入条件 | SLA |
|---|---|---|
| 售前咨询 | 未绑定订单、商品/门店咨询 | 5 分钟内首次响应 |
| 长租订单 | 安心用订单相关 | 3 分钟内首次响应 |
| 体验租订单 | 待接单、待取车、用车中、还车、控车异常 | 1 分钟内首次响应 |
| 财务问题 | 保证金、退款、账单、提现、结算 | 10 分钟内首次响应 |
| 客诉争议 | 投诉、扣款争议、商家争议 | 5 分钟内建工单 |
| 主管队列 | 黑名单、高风险、重大客诉、舆情 | 立即提醒主管 |

SLA 可后台配置,但体验租用车中和控车异常优先级必须高于普通售前咨询。

---

## 7. IM 与工单

IM 用于实时沟通,工单用于可追踪处理。

必须转工单的场景:

- 客服不能一次性解决。
- 涉及退款、扣款、关单、改价、结算账户、锁车/解锁等高危动作。
- 涉及合同、公证、签收、保证金扣款争议。
- 体验租无法取车、无法还车、车辆故障、控车失败。
- 客户投诉商家、门店或平台人员。

工单必须关联:

- 客户。
- 订单。
- 会话。
- 证据附件。
- 责任方。
- SLA。
- 处理人和处理结果。

---

## 8. 高危动作边界

IM 内允许:

- 发送说明。
- 发送订单卡片。
- 发送协议/帮助链接。
- 收集图片、视频、描述。
- 转接客服。
- 转工单。
- 跳转业务页面。

IM 内禁止直接执行:

- 退款。
- 关单。
- 改价。
- 保证金扣款。
- 锁车、解锁、关机、开机。
- 修改合同模板。
- 修改结算账户。
- 修改员工权限。
- 客户加黑名单。

禁止动作必须在原业务页面完成权限校验、二次确认、操作日志和结果回写。

---

## 9. 合规与归档

1. 会话内容默认按客户和订单归档。
2. 敏感信息默认脱敏展示,查看完整信息需要权限和日志。
3. 客服快捷回复必须使用全局措辞规范。
4. 系统应提醒或拦截资方、资金来源、合作模式、风控结论等 C 端禁用词外露。
5. 客服不得在 IM 中作出口头减免、包过审核、保证放款、绕过合同等承诺。
6. 会话、附件、转接、工单事件至少保留到订单结束后 3 年;合同、公证相关按合同存档规则执行。

---

## 10. V1 验收标准

1. C 端安心用和体验租订单详情均可进入客服。
2. 商家端、门店手机端可联系平台客服。
3. 客服工作台能看到客户、订单、业务线、门店、当前页面、触发原因。
4. 客服能按队列分配、转接、转工单。
5. 会话能关联长租订单和体验租订单。
6. 高危动作只能跳转业务页面,不能在 IM 内直接执行。
7. 会话和工单可归档检索。
8. C 端和客服话术不暴露资方、资金来源、合作模式、内部审核结论。
