# V0.2.2 第三方供应商适配字段映射

> **契约声明**:本文定义供应商接入的统一适配层。具体供应商只做字段映射和能力差异配置,不得改变 V0.2.2 订单状态机、资金四账、C 端敏感字段隔离和证据链规则。

## 一、统一适配原则

1. 供应商能力通过适配器接入,业务系统只依赖统一字段。
2. 所有供应商请求必须记录请求参数摘要、响应摘要、供应商流水号、幂等键、回调号、耗时、失败原因。
3. 所有供应商回调必须幂等处理。
4. 供应商失败不得直接改为业务成功;必须进入补偿任务或人工复核。
5. 供应商字段不得直接透传给 C 端,必须经 DTO 白名单转换。
6. 供应商更换只允许改配置和适配层,不得要求业务页面重写流程。

## 三、支付 / 退款字段映射

| 统一字段 | 支付供应商字段 | 退款供应商字段 | 说明 |
|---|---|---|---|
| `business_no` | `out_trade_no` | `out_refund_no` | 平台业务单号 |
| `channel_trade_no` | `trade_no` | `refund_trade_no` | 供应商交易号 |
| `amount` | `total_amount` | `refund_amount` | 金额必须与业务单一致 |
| `payer_id` | `buyer_id/openid` | 原支付付款人 | 脱敏存储 |
| `pay_status` | `trade_status` | `refund_status` | 映射为统一状态 |
| `callback_no` | `notify_id` | `notify_id` | 回调幂等 |

验收:
- 回调金额不一致进入人工复核。
- 重复回调只能更新同一条流水。
- 退款必须关联原支付流水。

| 统一字段 | 说明 |
|---|---|

| `contract_version` | 模板版本 |
| `signer_role` | 甲方 / 乙方 / 丙方 |

## 五、监管锁 / 控车字段映射

| 统一字段 | 说明 |
|---|---|
| `device_identifier` | IMEI / SN / VIN / 车架号 |
| `lock_vendor_device_id` | 供应商设备 ID |
| `lock_status` | 未安装 / 已上锁 / 解锁 / 失败 |
| `activation_status` | 未激活 / 已激活 / 失败 |
| `command_type` | 上锁 / 解锁 / 支付模式 / 查看定位 / 鸣笛 |
| `command_status` | 已发送 / 成功 / 失败 / 超时 |
| `last_callback_at` | 最后回调时间 |
| `failure_reason` | 失败原因 |

验收:
- 长租结算必须校验已上锁 + 已激活。
- 监管锁失败进入人工复核。
- 控车指令必须有权限、二次确认和审计日志。

## 八、OCR / 人脸字段映射

| 统一字段 | 说明 |
|---|---|
| `scene` | 场景:`identity_verify` / `rider_verify` / `delivery_face_match` |
| `order_id` | 订单 ID,交付合照核验时必填 |
| `attachment_id` | 人机合照 / 人车合照附件 ID |
| `reference_identity_trace_no` | 实名认证环节流水号,仅用于本次比对追溯 |
| `provider` | 第三方供应商标识 |
| `provider_trace_no` | 第三方调用流水号 |
| `match_score` | 第三方返回的相似度分值 |
| `match_result` | `high_match` / `suspect_mismatch` / `undetermined` |
| `checked_at` | 比对时间 |
| `manual_verify_result` | `pending` / `passed` / `rejected` / `need_reupload` |

验收:
- 交付合照核验只保存比对结果、评分、供应商标识和流水号,不建立可检索人脸库。
- AI 结果只作为人工核验辅助,不得自动通过或自动拒绝交付。
- C 端 DTO 不暴露人脸比对、风控、内部审核结论等字眼。
- 供应商超时或失败必须降级人工核验并写入调用日志。

## 九、供应商落地验收

1. 每个供应商接入前必须完成字段映射表。
2. 每个供应商回调必须有幂等键和补偿任务。
3. 每个供应商失败必须能在运营端查询到失败原因。
4. 每个供应商产生的关键材料必须进入附件中心或证据包。
5. 供应商替换不得影响 C 端展示字段和主状态机。
