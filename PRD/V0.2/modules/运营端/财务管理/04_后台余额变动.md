# 后台余额变动

> **来源**: P0-3
> **场景**: 对公转账充值、资金手工修正、系统异常修复

## 1. 页面定位

运营端 · 财务管理 · 后台余额变动。手工调整某个钱包余额,必须上传凭证 + 完整日志。

## 2. 使用场景

- 对公转账充值后调账
- 资金对账异常修正
- 系统 bug 后修正

## 3. 操作权限

| 角色 | 可使用 |
|---|---|
| 财务专员 | ✅ |
| 财务主管 | ✅ |
| 运营管理员 | ✅ |
| 普通客服 | ❌ |
| 门店/资方 | ❌ |

V1 不做大额审批门槛,靠日志追责。V2 可增加审批。

## 4. 操作流程

```
1. 点击"新增调账"
2. 填写:
   - 钱包类型(必选)
   - 关联主体(必选,下拉/搜索)
   - 变动类型(充值/扣减)
   - 变动金额(正数)
   - 事由类型下拉:
     · 对公转账充值 (offline_topup)
     · 资金手工修正 (manual_correction)
     · 系统异常补偿 (system_compensation)
     · 其他
   - 事由详情(自由文本≥6字)
   - 凭证上传(必传,多图/PDF ≤5MB)
3. 二次确认(弹窗展示关键信息)
4. 提交后实时执行:
   - 钱包余额变动
   - 生成流水
   - 备注:`[后台调账] 操作人 XX,事由 XX,凭证 XX`
   - 留操作日志
5. 同时清除相关状态(门店欠款补上后自动移除)
```

## 5. 调账记录列表

### 5.1 筛选
钱包类型/关联主体/变动类型/事由类型/操作人/时间

### 5.2 字段
调账 ID / 钱包类型+主体 / 变动类型 / 金额 / 事由类型 / 凭证 / 操作人 / 时间 / 备注

### 5.3 详情页
展完整信息 + 凭证预览 + 变动前后余额
仅查看,不可改(需反向调账)

## 6. 反向调账

- 不能删原记录
- 反向新增一条(原+100,反-100)
- 事由"调账错误反向修正 关联 XX"
- 都留日志

见 `00_核心决策表.md` `balance_adjustment`。
