# 核心接口清单

> [历史参考 / V0.2.2 口径覆盖] 本文为早期接口边界清单,不是最终 OpenAPI 契约。研发开工以 `17_V0.2.2_OpenAPI开发契约补丁.md`、`20_V0.2.2_API最终契约验收清单.md` 和 `openapi/core-api.v0.2.yaml` 为准。旧“资方 / 分配内部资金来源 / 内部资金来源台账”统一按“内部资金来源 / 内部资金来源分配 / 内部资金来源台账”理解。

> 所属：开发设计
> 目标：按端和业务链路列出核心 API 边界。这里不是最终接口协议，但要明确谁调用、写什么对象、触发什么事件。

---

## 1. 接口设计原则

1. API 按业务动作命名，不按页面按钮随意命名。
2. 写接口必须返回业务对象最新状态。
3. 关键写动作必须带幂等键。
4. 第三方回调接口只接收和入库，不直接信任外部状态。
5. 导出、分账、对账、回调重放等长任务走异步任务。
