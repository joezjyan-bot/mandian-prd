# 账号身份与跨端统一

> **V0.2.1 开发冻结同步(2026-05-27)**: 本文档是 C 端多入口、IM 客服、后期 App 和跨端订单连续性的身份底座。展示术语按 `10_全局措辞规范.md` 执行;数据库字段、API 路径和英文枚举不因展示口径调整而改名。

---

## 1. 目标

系统需要同时支持支付宝小程序、微信小程序、H5,并为后期 App 预留。客户可能从任一入口进入安心用或体验租,也可能在不同端咨询客服、补资料、签合同、支付、取车或查看订单。

V1 必须做到:

1. 同一客户尽量归并到同一个 `customer_id`。
2. 同一客户跨端订单、会话、工单、授权、支付、签约记录可以汇总查看。
3. 客服中心按客户和订单维度看完整上下文,不按单个小程序账号割裂。
4. 未实名前可以按手机号和外部账号做弱关联;实名后按证件哈希做强归并。

---

## 2. 身份层级

| 层级 | 说明 | 示例 |
|---|---|---|
| 业务客户 | 平台内部客户主体 | `customers.id` |
| 外部账号 | 某个渠道给出的账号身份 | 支付宝 `user_id`、微信 `openid/unionid`、H5 手机号、App user id |
| 实名身份 | 通过实名和人脸后的强身份 | 姓名、证件号哈希、人脸核验记录 |
| 联系方式 | 可变更的联系触点 | 手机号、紧急联系人 |

`customers.id` 是订单、客服、工单、账单、钱包相关客户视图的核心主键。支付宝、微信、H5、App 都只能作为外部账号映射到 `customers.id`。

## 3. 关键表

### 3.1 `customers`

| 字段 | 说明 |
|---|---|
| `id` | 内部客户 ID |
| `mobile_hash` | 手机号哈希 |
| `mobile_masked` | 脱敏手机号 |
| `real_name_masked` | 脱敏姓名 |
| `cert_no_hash` | 证件号哈希 |
| `cert_no_masked` | 脱敏证件号 |
| `realname_status` | 未实名 / 已实名 / 失败 / 人工复核 |
| `merged_from_customer_ids` | 合并来源,用于审计 |

### 3.2 `customer_external_accounts`

| 字段 | 说明 |
|---|---|
| `customer_id` | 内部客户 ID |
| `channel` | `alipay_mini` / `wechat_mini` / `h5` / `ios_app` / `android_app` / 其他 |
| `external_user_id` | 外部账号 ID |
| `union_id` | 可选,微信 unionid 或其他联合 ID |
| `mobile_hash` | 该渠道绑定的手机号哈希 |
| `bind_status` | 已绑定 / 待确认 / 已解绑 |
| `last_active_at` | 最近活跃时间 |

## 4. 归并规则

### 4.1 未实名前

按以下优先级做弱关联:

1. 同一外部账号 ID 直接命中已有 `customer_external_accounts`。
2. 同一手机号验证码登录或授权手机号一致,可提示归并。
3. 同一设备、同一浏览器、同一订单二维码只能作为辅助信号,不能自动强归并。

未实名前归并只能影响客服和断点续传视图,不得自动改变实名资料、合同主体和支付责任。

### 4.2 实名后

客户完成实名和人脸后:

1. 用证件号哈希匹配历史客户。
2. 如发现多个 `customer_id` 属于同一实名主体,进入自动合并或人工复核队列。
3. 合并后订单、会话、工单、授权记录统一归入主 `customer_id`。
4. 合并过程必须写操作日志,保留原外部账号映射。

---

## 5. 跨端会话和订单

IM 客服、订单中心和待办中心必须按 `customer_id` 汇总:

- 客户从支付宝咨询,又从 H5 补资料,客服侧应显示同一个客户视图。
- 客户从微信进入体验租订单,又从支付宝查看安心用订单,订单中心应按业务线分组展示。
- 客户未实名时,客服可看到“疑似同手机号历史会话”,但页面需标记为弱关联。

---

## 6. 推送通道

| 通道 | 用途 |
|---|---|
| 支付宝小程序订阅/模板消息 | 支付宝端订单待办、客服提醒 |
| 微信订阅消息 | 微信端订单待办、客服提醒 |
| 短信 | 关键超时、合同、支付、取还车提醒兜底 |
| H5 站内消息 | H5 登录后的待办提醒 |
| App Push | 后期 App 预留 |

推送内容不得暴露资方、资金来源、合作模式、内部审核结论等内部概念。

---

## 7. 验收标准

1. 同一客户在支付宝、微信、H5 完成实名后,能归并为同一个 `customer_id`。
2. 客服中心能按客户汇总跨端会话和订单。
3. 未实名弱关联必须有提示,不能直接覆盖合同主体。
4. 外部账号解绑、重新绑定、客户合并都必须有日志。
5. 后期新增 App 时不需要重构客户主表,只需新增 `channel` 类型和推送通道。
