# 设备状态机

> **来源**: P1-1
> **核心**: V1 共 9 个状态;已下架不显示在库存列表;V1 不支持跨门店调拨

## 1. 状态清单(9 个)

| # | 状态 | 含义 |
|---|---|---|
| 1 | 待入库 | 已采购/已配货,未完成入库登记 |
| 2 | 在库可租 | 入库完成,可对外出租 |
| 3 | 预约保留 | (新)客户已扫码或办单中,设备暂时保留 |
| 4 | 已锁定 | 已支付,等待发货/客户取走 |
| 5 | 出租中 | 已发货/已取走,在客户手中 |
| 6 | 归还待验 | 客户已归还,等待门店验收 |
| 7 | 维修中 | 验收发现问题,送修 |
| 8 | 争议中 | (新)门店与客户对设备状态有分歧 |
| 9 | 已下架 | 报废/出售二手/退还供应商/丢失等终态 |

## 2. 状态流转图

```
待入库 → 在库可租
在库可租 → 预约保留 → 在库可租 (取消保留)
                  → 已锁定 (客户付款)
已锁定 → 出租中
       → 在库可租 (取消订单/未发货)
出租中 → 归还待验
归还待验 → 在库可租 (验收通过)
        → 维修中 (验收发现问题需修)
        → 争议中 (双方有分歧)
维修中 → 在库可租 (修好)
       → 已下架 (修不好/不值得修)
争议中 → 归还待验 (协商解决继续验收)
       → 已下架 (协商以下架结算)
任何状态 → 已下架 (报废/出售/丢失等终态)
```

## 3. 关键规则

### 3.1 已下架设备
- 用 `device.downgrade_reason` 字段记录细节(报废 / 售二手 / 退还供应商 / 丢失 / 其他)
- 不细分状态
- **已下架设备不显示在库存列表**(默认筛选器排除)
- 想看已下架设备:专门的"已下架设备"查询页(运营端)

### 3.2 V1 不支持跨门店调拨
- 设备一旦入库到某门店,只能在该门店流转
- 不需要"运输中"状态
- V2 视需要加调拨能力

### 3.3 状态流转日志
所有状态变更:
- 留 `device_status_log` 记录
- 字段:设备 ID / 旧状态 / 新状态 / 变更人 / 变更时间 / 关联订单(如有)/ 备注

## 4. 操作权限

| 操作 | 角色 |
|---|---|
| 入库登记 | 门店主账号 / 门店员工(配置) |
| 取消保留 / 锁定 | 客服 / 门店主账号 |
| 标记发货 | 门店主账号 / 门店员工 |
| 验收归还 | 门店主账号 / 门店员工 |
| 标记维修 | 门店主账号 |
| 标记争议 | 门店主账号 / 客服 |
| 已下架 | 门店主账号 / 客服(填原因)|

## 5. 异常状态处理

### 5.1 设备状态与订单状态不一致
- 系统定期校验(每日)
- 不一致时告警 + 工单
- 客服介入核实

### 5.2 长期"已锁定"未发货
- > 7 天未发货 → 提醒门店
- > 30 天未发货 → 推送告警

### 5.3 长期"维修中"
- > 30 天未修好 → 提醒客服
