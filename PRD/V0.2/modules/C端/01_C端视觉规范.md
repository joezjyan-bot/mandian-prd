# C 端视觉规范(方案 1)

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 本文档定义满点 C 端(小程序 / H5 / App)的视觉设计规范。
> **方案 1 决策(2026-05-25)**:延续企业蓝主色,搭配橙色 CTA 转化色,与运营端 / 商家端中台风格协调,但具备 C 端必要的活力感。
>
> 所有 C 端页面级 PRD(安心用、体验租)的 UI 标注,必须引用本文档的色板、圆角、字号、组件规范。

---

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| 品牌一致 | C 端深蓝主色与后台 `04_UI风格与页面原型规范.md` 的企业中台蓝呼应,客户感知到"同一个平台" |
| 转化优先 | CTA 关键按钮使用橙色 `#F97316`,与浏览态深蓝形成对比,引导客户下一步操作 |
| 信息密度适中 | 比九号租车的大圆角(24px)收一档,比中台后台(4-8px)放大;采用 12-14px |
| 内容大于装饰 | 不使用大面积渐变、阴影、3D 元素;卡片用浅灰底+白卡片即可分层 |
| 无障碍 | 文字与背景对比度 ≥ 4.5:1;CTA 按钮 ≥ 48px 高;字号最小 12px |
| 合规第一 | 任何位置不暴露"联营/资方/分账/平台抽佣"等内部概念;详见 §10 |
| 图标不锁品类 | 体验租覆盖品类未来会扩展(电动车、手机、相机、户外...),Tab 图标采用**语义中性**符号,不绑定单一品类(如不用单车、摩托、手机等具象图标) |

## 4. 字号规范

| 用途 | 字号 | 字重 |
|---|---|---|
| 一级标题(页面标题、门店名) | `20px` | Semibold 600 |
| 二级标题(模块标题) | `17px` | Semibold 600 |
| 价格大字(主推价格) | `28px` | Bold 700 |
| 价格中字(列表内价格) | `20px` | Bold 700 |
| 正文 - 大 | `16px` | Regular 400 |
| 正文 - 标准 | `14px` | Regular 400 |
| 副文字 - 说明 | `13px` | Regular 400 |
| 标签 / 时间戳 | `12px` | Regular 400 |
| 按钮文字 | `16px` | Medium 500 |

行高统一为字号的 1.5 倍。中英文混排使用系统默认字体:
- iOS: `-apple-system`, `PingFang SC`
- Android: `Roboto`, `Noto Sans SC`
- 小程序: 默认字体

8 倍数体系:

| 标记 | 数值 | 使用场景 |
|---|---|---|
| xs | `4px` | 标签内 padding / icon 间距 |
| sm | `8px` | 元素紧密相关 |
| md | `12px` | 同模块元素间距 |
| **default** | `16px` | 页面左右边距 / 模块间距 / 卡片内边距 |
| lg | `24px` | 模块分组间距 |
| xl | `32px` | 大区块分隔 |
| xxl | `48px` | 顶部 / 底部留白 |

## 6. 关键组件设计

### 6.1 底部 Tab(全局)

```
┌──────────────────────────────────────────┐
│                                          │
│   shield      sparkles    headphones  user │
│   安心用       体验租       客服       我的  │
│   灰           深蓝         灰         灰   │
└──────────────────────────────────────────┘
```

**Tab 顺序固定**:安心用 / 体验租 / 客服 / 我的(共 4 个)

**图标规范(Tabler Outline 线性图标系列)**:

| Tab | 图标名 | 语义 |
|---|---|---|
| 安心用 | `ti-shield-check` 盾(含勾) | "保障/稳定/长期",对应长租以租代购的安心感 |
| 体验租 | `ti-sparkles` 星(三角星簇) | "尝鲜/体验/新意",**语义中性,不绑定任何品类**(电动车/手机/相机/户外都适用) |
| 客服 | `ti-headphones` 耳机 | 客服国际通用符号 |
| 我的 | `ti-user` 人像 | 用户中心通用符号 |

**视觉规范**:

- 高度:`56px`(图标+文字)或 `60px`(含安全区)
- 背景:白色 `#FFFFFF`
- 上分割线:`0.5px solid #E2E8F0`
- 图标尺寸:`20-22px`
- 文字字号:`11-12px`
- 图标 + 文字垂直排列(间距 2-3px)
- 选中态:图标 + 文字均为 `#1E3A8A` 深蓝,文字 `font-weight: 500`
- 未选中态:图标 + 文字均为 `#94A3B8` 灰
- 默认进入 Tab:**安心用**(冷启动落地)

**禁止**:

- ❌ 不用 emoji(🛡🛵💬👤) — 各端渲染不一致,且显得花
- ❌ 不用 `ti-motorbike` / `ti-bike` / `ti-car` / `ti-phone` 等品类绑定图标 — 体验租未来会扩品类,品类图标会锁死
- ❌ 不用填充图标(`-filled` 后缀) — 与整体克制线性风格不符
- ❌ 不在 Tab 上加角标数字 / 红点 — V1 不做(V2 客服 Tab 可加未读数)

### 6.2 顶部导航

| 类型 | 设计 |
|---|---|
| 普通页面 | 白底 + 居中黑色标题 + 左侧返回箭头 + 右侧"…⊙"小程序原生胶囊 |
| 业务首页(安心用 / 体验租) | 白底 + 左侧城市切换"杭州 ▼" + 右侧搜索图标 |
| 主 CTA 页面(订单确认) | 同普通页面 + 右侧不放胶囊外其他元素 |

### 6.3 按钮

| 类型 | 样式 |
|---|---|
| 主 CTA 全宽 | 橙色底 `#F97316` / 白字 / 胶囊圆角 28px / 高度 52px / 字号 16px |
| 次 CTA 全宽 | 深蓝底 `#1E3A8A` / 白字 / 同样圆角和高度 |
| 描边按钮 | 白底 / 深蓝边 `#1E3A8A` / 深蓝字 / 圆角 24px / 高度 44px |
| 并列双按钮 | 例如「联系商家」「导航到店」:均描边按钮,各占 50% 宽 |
| 文字按钮 | 深蓝字 `#1E3A8A` / 无底无边 / 14px |

## 10. C 端合规红线(必读)

任何 C 端页面、文案、字段、状态文案**绝对不出现**以下词:

| 禁止词 | 原因 |
|---|---|
| 联营 / 联营订单 | 平台内部订单类型,客户无需感知 |
| 资方 / 出资方 / 资金方 | 涉及金融牌照,客户暴露 = 监管风险 |
| 分账 / 抽佣 / 平台抽成 | 内部财务概念,客户无需感知 |
| 平台订单 / 商家订单 | 内部分类,客户看到统一是"满点订单" |
| 罚息 | 必须改为「逾期逾期费用」(P0 决策) |
| 留购 | 必须改为「留购」(P0 决策) |
| 融资 / 信贷 / 借款 / 贷款 | 绝对禁止(P1-4 合规死线) |

任何审核 / 资方分配 / 分账逻辑,客户侧只展示「平台处理中」「平台已确认」等中性文案。
