# 商家端体验租订单与车辆

> 商家端体验租负责门店现场动作：接单、拒单、取车核销、绑定车辆、控车、还车验收、押金扣款和退款。它不是安心用长租办单助手的扩展页。

---

## 1. 入口

### 1.1 商家 PC 端

一级模块：体验租。

页面：

- 体验租订单
- 体验租车辆
- 体验租价格方案
- 押金/扣款
- 收益流水
- 操作日志

### 1.2 门店手机端

入口：体验租待办。

待办分组：

- 待接单
- 待取车
- 已取车
- 待还车
- 押金待处理

## 3. 订单动作

| 动作 | 可操作状态 | 结果 |
|---|---|---|
| 确认接单 | 待确认 | 订单变为待取车，生成核销码，记录接单时间 |
| 拒绝接单 | 待确认 | 走取消订单逻辑，租金和押金原路退 |
| 取消订单 | 待支付、押金待支付、待确认 | 订单变为已取消，按规则退款 |
| 取车核销 | 待取车 | 输入核销码和车辆 VIN/SN，绑定车辆，进入已取车 |
| 确认还车 | 已取车 | 记录还车时间，进入押金结算或已完成 |
| 押金扣款 | 已取车/待还车确认 | 选择原因、填写金额，生成扣款记录 |
| 立即退还押金 | 已完成或押金待处理 | 扣除累计扣款后退还剩余押金 |
| 控车 | 已取车 | 开机、关机、定位、闪灯鸣笛，写控车日志 |

所有动作必须写操作日志，涉及退款、扣款、控车的动作必须二次确认。

---

## 4. 取车核销

核销字段：

- 核销码
- 车辆 VIN/SN/IMEI
- 设备号，若控车需要
- 车牌号，若有
- 交付照片，后续可选
- 操作备注

校验：

1. 订单必须为待取车。
2. 车辆必须属于该门店或该订单指定履约门店。
3. 车辆必须为在库可租状态。
4. 车辆不能已绑定其他未完成订单。
5. 核销成功后记录实际开始时间和应还时间。

---

## 5. 车辆管理

车辆字段：

- 车辆 ID
- 所属商家
- 所属门店
- 商品/车型
- VIN/SN/IMEI
- 设备号
- 车牌号
- 经营模式：门店自营、平台自营、平台货源门店履约
- 设备状态：待入库、在库可租、预约保留、已锁定、出租中、归还待验、维修中、争议中、已下架
- 售卖状态，若设备可转长租或留购
- 备注

规则：

1. 车辆唯一编码全局唯一。
2. 只有在库可租车辆才可被客户看到或被订单锁定。
3. 预约保留、已锁定、出租中、归还待验、维修中、争议中车辆不可重复下单。
4. 已下架车辆不可新增订单，但历史订单仍可处理。
5. 平台自营车辆可配置履约门店，门店只看到被分配给自己的车辆。

---

## 6. 价格方案

商家可在自己有权限的门店下配置体验租价格方案。

字段：

- 门店
- 车型
- 启用周期：1 小时、2 小时、4 小时、当日还、日租、周租、月租
- 对应价格
- 押金
- 预授权金额
- 是否启用

规则：

1. 平台可配置默认方案。
2. 商家/门店可在授权范围内覆盖价格。
3. 预授权金额创建后原则上不允许普通编辑，修改需高权限和日志。
4. 押金不能大于预授权金额。
5. 关闭某周期后 C 端不展示该周期。

---

## 7. 押金扣款和退款

扣款原因可配置：

- 逾期
- 损坏
- 丢失配件
- 清洁/维修
- 其他

扣款规则：

1. 扣款金额不能超过可用押金余额。
2. 押金扣款先记录，不立即结算完成。
3. 点击立即退还押金时，按押金金额减累计扣款金额退还。
4. 免押订单调用免押扣款接口。
5. 实付押金订单走原路退款。
6. 押金不足时第一版可记录线下补收，后续再接线上补扣。
