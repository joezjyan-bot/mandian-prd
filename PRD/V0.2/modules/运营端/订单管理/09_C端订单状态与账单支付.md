# C 端订单状态与账单支付

> P0 业务文档(2026-05-26)。
> 长租客户在 C 端能看到什么、怎么操作。定义 C 端订单状态精简到 8 个、订单详情账单瀑布流、任意期支付逻辑、留购触发条件、客户主动撤单入口。
> **核心原则**:C 端展示信息**最少**,不暴露任何内部分类、商家主体、资方、客服等敏感字段。

## 1. C 端订单状态(精简到 8 个 + 已完成子标签)

### 1.1 8 个 C 端状态 Tab

| Tab | 含义 |
|---|---|
| **全部订单** | 包含所有状态(含已取消)|
| **审核中** | 客户已提交,系统/客服在审核 |
| **待付款** | 客服已确认订单,客户需支付首期 |
| **待签约** | 首期已付,需签电子合同 / 代扣 |
| **已发货** | 订单已发出,等待物流送达 |
| **待收货** | 客户在 App 看到物流单号(收到设备后点确认收货)|
| **租用中** | 客户已收货确认,在租期内 |
| **已完成** | 归还 / 留购 / 已取消(含子标签)|

### 1.2 后台状态 → C 端状态映射

| 后台状态(全局 02 §3.1)| C 端展示 |
|---|---|
| DRAFT / PENDING_SUBMIT | (不展示,客户还没提交)|
| PENDING_REVIEW | **审核中** |
| PENDING_FUNDER | **审核中** |
| **PENDING_STORE_ASSIGN** | **审核中** |
| PENDING_PAYMENT(首期未付)| **待付款** |
| PENDING_PAYMENT(首期已付,合同未签)| **待签约** |
| PENDING_PAYMENT(合同已签,代扣未签)| **待签约** |
| PENDING_SIGN | **待签约** |
| PENDING_SHIP | **审核中** |
| PENDING_RECEIVE(已发货)| **已发货** |
| PENDING_RECEIVE(客户在 App 看到物流单号后)| **待收货** |
| PENDING_RETURN / RENTING | **租用中** |
| DUE_REMINDER / OVERDUE | **租用中**(详情页内部显示逾期标签)|
| RETURNING / RETURN_INSPECTION | **租用中**(归还中)|
| PURCHASE_PENDING | **租用中**(留购待支付)|
| COMPLETED | **已完成** 子标签"已归还" |
| RETAINED | **已完成** 子标签"已留购" |
| CLOSED | **已完成** 子标签"已取消" |
| DISPUTE | **租用中**(详情显示争议标签)|

**核心原则**:
- 审核全过程(包括分配资方/分配门店/调整套餐/等客户确认)客户都看到"审核中"
- 不显示预估时间("1-3 工作日"全部去掉)
- "已完成" Tab 通过**子标签**区分归还/留购/取消

### 1.3 Tab 角标数字规则(Q17)

| Tab | 数字逻辑 |
|---|---|
| 全部订单 | 不显示数字 |
| 审核中 | 显示数字(若有)|
| 待付款 | 显示数字(若有)|

### 3.2 任意期可支付的逻辑(Q16)

**核心规则**:**任何一期都可以点 [立即支付] 按钮,显示对应期数的金额**。

| 状态 | 表现 |
|---|---|
| 已付清 | 显示 ✅,无支付按钮 |
| 未付 | 显示 [立即支付] + 当期金额 + [申请留购] + 留购价 |
| 已逾期 | 显示 [立即支付] + 红色违约金提示 + [申请留购](留购价 + 违约金)|

**预计算**:办单助手生成订单时,**全部期数的金额 + 对应留购价**都已写入数据库。客户在 C 端看到的就是这张表。

### 3.4 申请留购的流程(Q16)

```
[第 N 期 - 立即支付]    [申请留购]
     ↓
客户点 [申请留购]
     ↓
检查前置条件:
  ✓ 当前期已付清?(必须先付完当期才能留购)
  ✓ 无未结违约金?
     ↓
  └─ 不满足 → 提示"请先支付第 N 期(¥XXX)"
  └─ 满足 → 弹出留购支付明细
     ↓
┌─────────────────────────────────┐
│ 留购确认                          │
├─────────────────────────────────┤
│ 已付期数:    第 1-N 期           │
│ 留购价:      ¥X,XXX(对应第 N 期)│
│ 押金抵扣:    -¥1,490             │
│ ─────────────                    │
│ 实付:        ¥X,XXX              │
├─────────────────────────────────┤
│  [取消]      [确认支付]            │
└─────────────────────────────────┘
     ↓
客户确认 → 调起支付通道 → 支付完成
     ↓
订单状态:租用中 → 已完成(已留购)
设备所有权归客户
```

### 3.5 提前还款 / 顺序加速

| 场景 | 处理 |
|---|---|
| 第 3 期未付,客户跳过先付第 5 期 | **不允许 — 系统强制按顺序扣账**(我之前理解错了)|
| 客户第 3 期 + 第 4 期一起付(批量支付)| V1 不支持,客户分别点 |
| 客户提前付了第 3 期(还没到期日)| 允许,系统记录;下一期顺序变为第 4 期 |

**待你确认**:**任何一期都可点,具体是不是"必须按顺序付完之前所有期数才能点当前期"?**

我对你"任何一期都可以点"的理解:
- 即使第 3 期未付,第 4 期到了客户也可以点 [立即支付] 付第 4 期
- 但第 3 期照样欠着,有违约金累计

如果是这样,**我认为这个机制对客户有风险**(容易漏付前期产生违约金)。

**正确的"顺序支付"行为应该是**:
- 客户可以**点任意一期看金额 + 留购价**
- 但**实际支付时,系统强制结清之前的欠款**
- 例如:第 3 期未付,客户点第 4 期 → 系统提示"请先支付第 3 期 ¥568.75"

**回我:你要哪种?**
- **A**:严格顺序,必须先付前面的(我推荐)
- **B**:任意期独立支付,前面的欠就欠着(违约金累计)

### 3.6 留购价的计算规则

参照你给的截图:

| 期数 | 本期金额 | 当期留购价 | 计算逻辑 |
|---|---|---|---|
| 第 1 期 | ¥10.00 | ¥6,040.00 | 留购价 = 总价 - 首期(押金抵扣前)|
| 第 2 期 | ¥568.75 | ¥5,471.25 | 留购价 = 上期留购价 - 第 1 期月供 |
| 第 N 期 | ¥568.75 | 递减 | 同上 |
| 第 9 期(最末)| ¥568.75 | ¥1,490.00 | 押金可全额抵扣留购 |

**关键**:留购价的逻辑由办单助手在生成订单时预计算并写入,**客户在 C 端只看不算**。

## 4. 客户主动撤单(Q3)

### 4.1 撤单按钮可点状态

| 订单状态 | [取消订单] 按钮 |
|---|---|
| 审核中 | ✅ 直接可取消 |
| 待付款(未付款)| ✅ 直接可取消 |
| 待付款(已付款)| ⚠️ 需要客服同意 |
| 待签约(已付款)| ⚠️ 需要客服同意 |
| 已发货 | ❌ 不可单方取消(走售后)|
| 待收货 | ❌ 不可单方取消(走售后)|
| 租用中 | ❌ 不可取消(走归还)|

### 4.2 直接可取消的撤单流程

```
[取消订单] → 客户填取消原因(可选)→ 确认
   ↓
订单 → 已完成(已取消)
押金/付款若有,按 §4.3 退款
```

### 4.3 退款规则

| 场景 | 退款 |
|---|---|
| 审核中撤单(未付款)| 无需退款 |
| 待付款撤单(未付款)| 无需退款 |
| 待付款撤单(已付首期)| 原路退回(1-3 工作日)|
| 待签约撤单(已付首期未签)| 客服同意后,原路退回 |
| **已签约撤单** | **客服同意后,扣订单总价 1.5% 违约金**(平台收),余额原路退 |

### 4.4 需客服同意的撤单流程

```
[取消订单] → 提示"已付款订单需联系客服处理"
   ↓
客户填取消原因 → 提交申请
   ↓
进入运营端"取消申请"队列
   ↓
客服审核:
  ├─ 同意 → 计算扣款(若已签约) → 退款 → 订单关闭
  └─ 拒绝 → 通知客户(给出理由)→ 订单继续
```

## 5. 订单逾期与违约金(Q3 衍生)

### 5.1 期内未付(欠当期)

如果客户当期 [立即支付] 没及时点,系统:

| 时机 | 动作 |
|---|---|
| 到期日 +1 天 | C 端推送"账单逾期,请尽快支付" |
| 到期日 +3 天 | 客服触达 |
| 到期日 +7 天 | 进入运营异常队列 |
| 到期日 +15 天 | 主管复核,考虑通知联系人 |
| 到期日 +30 天 | 启动强制处置(代扣兜底/法律)|

**逾期违约金**:**当期金额 × 0.05%/天**(待你确认),最少 ¥10/天。
