# 商家工作台

> 页面级 PRD 草案。
> 目标：让商家老板、管理员、店长、店员进入商家端后直接看到自己要处理的订单、商品、待办、交付、财务和异常。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家工作台 |
| 所属端 | 商家 PC 端，门店手机端可复用待办摘要 |
| 入口路径 | 商家中心 > 首页 |
| 使用角色 | 商家老板、商家管理员、门店店长、店员 |
| 核心目标 | 聚合商家自己的订单、商品、待办、门店发货、钱包、员工和平台订单进度 |

商家工作台不是运营端工作台的缩小版。商家只看自己的业务，不看其他商家、平台资方池和平台总财务。

---

## 2. 核心口径

1. 商家工作台按角色展示：老板看经营和钱包，店员只看办单和待办。
2. 三类订单分开展示：门店订单、分红订单、平台订单。
3. 门店订单由商家自审；联营/平台订单只展示履约视图和本商家待办，不开放审核、改价、资方、资金主控入口。
4. 默认门店发货，因此待发货、待交付材料、待归还验收是商家工作台重点。
5. 员工账号不能看到钱包、提现、分账和完整经营数据。
6. 所有卡片必须可点击进入对应列表或详情。
7. 平台复制商品、商品审核驳回、授权未完成等也要形成商家待办。

---

## 3. 页面结构

```text
商家工作台
├─ 顶部：商家名称、门店切换、当前账号、快捷搜索
├─ 我的待办：待审核、待补资料、待发货、待交付材料、待归还验收
├─ 订单概览：门店订单 / 分红订单 / 平台订单
├─ 履约进度：联营/平台订单交付、签收、归还、售后协助
├─ 商品待办：商品待审核、驳回、平台复制待确认、同步失败
├─ 财务摘要：可提现、待结算、冻结、提现中
├─ 员工与办单：今日办单、员工排行、异常二维码
└─ 通知公告：平台通知、客诉、租后提醒、授权状态
```

---

## 4. 顶部快捷区

| 模块 | 说明 |
|---|---|
| 门店切换 | 多门店商家选择数据范围 |
| 快捷搜索 | 订单号、客户姓名、手机号后四位、商品、设备码 |
| 快捷操作 | 新增商品、进入办单助手、员工管理、提现 |
| 授权状态 | 店铺资料、收款账户、e 签宝授权、合同签章授权 |

员工账号不显示提现、员工管理、商家配置等入口。

---

## 5. 我的待办

| 待办 | 来源 | 点击后 |
|---|---|---|
| 门店订单待审核 | 客户扫码下门店订单后等待商家自审 | 门店订单审核列表 |
| 待补资料 | 平台或商家要求补资料 | 补资料待办 |
| 待交付材料 | 订单要求上传人机/人车合照、设备照片 | 门店待办 |
| 待发货 | 默认门店发货的订单 | 发货列表 |
| 待客户签收 | 已交付但客户未确认 | 签收列表 |
| 待归还验收 | 客户申请归还或已寄回 | 归还验收 |
| 商品待处理 | 商品被驳回、复制商品待确认、同步失败 | 商品管理 |
| 平台订单待跟进 | 平台订单需补资料、客服沟通、交付或归还协助 | 平台订单履约视图 |
| 客诉待协助 | 客诉需要商家补充发货、交付、沟通材料 | 客诉协助 |

## 6. 订单概览

### 6.1 门店订单

| 指标 | 说明 |
|---|---|
| 今日新增 | 本商家门店订单新增 |
| 待审核 | 商家自审 |
| 待发货 | 商家/门店发货 |
| 在租 | 正常履约 |
| 已逾期 | 门店订单逾期 |
| 今日到期 | 到期提醒 |
| 平台抽佣 | 当前周期平台抽佣摘要，老板可见 |

### 6.2 分红订单

| 指标 | 说明 |
|---|---|
| 待客户签约/支付 | 客户未完成链路 |
| 待门店发货 | 平台审核通过后门店待发货 |
| 待客户签收 | 门店已交付但客户未确认 |
| 待归还验收 | 客户发起归还或到店归还 |
| 待结算收益 | 门店份额待结算 |
| 已入账收益 | 门店钱包入账 |

### 6.3 平台订单

| 指标 | 说明 |
|---|---|
| 待补资料 | 需要商家或客户补资料 |
| 客服沟通中 | 已发起 IM 或需要联系平台 |
| 待发货 | 如配置门店发货 |
| 待签收/归还 | 如配置门店协助签收或归还验收 |
| 协作收益 | 如果配置门店服务费 |
| 渠道来源 | 如该商家由渠道推广 |

## 7. 联营/平台履约进度卡片

商家端需要看到联营/平台订单与本商家相关的履约进度，但不开放内部控制权。

| 字段 | 展示 |
|---|---|
| 订单状态 | 待补资料、待签约、待支付、待交付、待签收、履约中、待归还、已完成 |
| 合同状态 | 未发起、待签、已签、失败；不展示模板策略和内部审核规则 |
| 支付状态 | 未支付、部分支付、已支付、异常；不展示资金来源和资金通道 |
| 发货主体 | 门店、平台或快递 |
| 下一步 | 商家需要做什么或等待平台处理 |
| 联系客服 | 调起 IM，自动带订单摘要 |

合作模式、资金来源、资方信息、平台抽佣明细、内部风控报告明文不展示给商家角色；C 端也不出现这些字眼。

---

## 8. 商品待办

| 待办 | 说明 |
|---|---|
| 商品待审核 | 商家提交商品等待平台审核 |
| 商品被驳回 | 展示驳回原因和修改入口 |
| 复制商品待确认 | 平台复制商品给商家，商家确认经营 |
| 商品同步失败 | 价格、规格、增值服务、短租设备缺失 |
| 增值服务异常 | 服务停用、金额冲突、绑定失效 |
| 短租库存不足 | 可租设备不足 |

---

## 9. 财务摘要

仅老板和被授权管理员可见。

| 指标 | 说明 |
|---|---|
| 待结算余额 | 未满足结算条件 |
| 可提现余额 | 可申请提现 |
| 冻结余额 | 客诉、退款、租后异常冻结 |
| 提现中 | 已申请未到账 |
| 今日入账 | 今日已入钱包 |
| 平台抽佣 | 门店订单和分红/平台相关抽佣摘要 |
| 接口扣费 | 大数据、风控、合同、公证等商家计费 |

店员、只读员工不展示财务摘要。

---

## 10. 员工与办单

| 模块 | 说明 |
|---|---|
| 今日办单数 | 按员工和入口统计 |
| 待处理二维码 | 已生成但客户未下单或已过期 |
| 办单员工排行 | 老板/管理员可见 |
| 员工异常 | 禁用账号、异常登录、跨门店访问 |
| 快捷管理员工 | 跳转员工管理 |

员工本人只看自己的办单和待办。

---

## 11. 通知与异常

| 类型 | 说明 |
|---|---|
| 平台公告 | 系统通知、规则调整 |
| 授权提醒 | 收款账户、合同签章、e 签宝授权未完成 |
| 客诉协助 | 支付宝投诉或订单售后需要商家补材料 |
| 租后提醒 | 到期、逾期、归还验收 |
| 财务异常 | 提现失败、退款冲正、接口扣费异常 |
| 商品异常 | 审核驳回、同步失败、库存不足 |

---

## 12. 权限展示

| 角色 | 工作台展示 |
|---|---|
| 商家老板 | 全部模块，含钱包、提现、员工、商品配置 |
| 商家管理员 | 按老板授权展示，可含商品、订单、部分财务 |
| 门店店长 | 本门店订单、待办、员工、交付，不默认展示钱包 |
| 店员 | 办单助手、本人待办、本人订单，不展示经营和财务 |
| 只读员工 | 只读订单进度和待办摘要 |

---

## 13. 数据互通

| 模块 | 联动内容 |
|---|---|
| 商家订单 | 三类订单概览、待办和详情 |
| 门店待办 | 交付材料、发货、归还验收 |
| 商品管理 | 商品审核、复制商品、同步办单助手 |
| 员工账号 | 办单归属、权限、异常登录 |
| 财务钱包 | 入账、提现、冻结、接口扣费 |
| 客诉管理 | 需要商家协助的投诉 |
| 租后管理 | 到期、逾期、回款、归还 |
| 授权认证 | 店铺资料、收款账户、签章授权状态 |
