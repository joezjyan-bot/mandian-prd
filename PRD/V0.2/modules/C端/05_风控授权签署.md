# 风控授权签署

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> **来源**: P1-5

## 1. 业务背景

办单流程中需向第三方风控机构查询客户信用、负债等信息。需客户先签署"风控查询授权书"。

## 2. 授权书内容

- 平台名称 + 客户名称
- 授权范围:芝麻信用、央行征信(白名单)、多头借贷查询、失信记录等
- 授权期限:本次订单办理至业务结束
- 客户签名(电子签)

## 3. 流程

```
1. 客户在小程序进入"风控授权"页
2. 阅读授权书全文(可滚动 + 必读勾选)
3. 点击"同意并签署"
4. 调电子签通道走简易签署(指纹/人脸/验证码)
5. 签署成功 → 系统调用风控接口拉取报告
6. 风控报告生成 → 客服审核(后台)
```

## 4. 评分与决策

风控报告评分:
- 通过:订单进入下一步
- 不通过:订单进入"待审核",客服决定:
  - 人工放行(需主管审批)
  - 调整方案(加保证金 / 降价)
  - 驳回订单

## 5. 失败处理

| 场景 | 处理 |
|---|---|
| 风控授权签署失败 | 自动重发起,失败 2 次进"待客服跟进" |
| 风控报告生成失败 | 重试 1 次,失败后客服决定是否人工放行 |
| 评分不通过 | 进"待审核",客服决定 |

## 6. 中控台对接

风控查询通过中控台对接,异步计费(P0-4)。

## 7. 数据模型

- `customer.risk_authorization_id`
- `order_risk_report`(订单风控报告快照)
