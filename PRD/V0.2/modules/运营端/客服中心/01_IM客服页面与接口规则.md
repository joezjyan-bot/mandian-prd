# IM 客服页面与接口规则

> 本文补齐 IM 客服基础能力的页面级和数据级设计。V1 不追求复杂机器人,但必须可用、可归档、可带订单上下文、可转工单,并能进入法诉证据包。

## 1. V1 页面结构

### 1.1 C 端入口

| 页面 | 入口名 | 自动带入上下文 |
|---|---|---|
| 首页 / 我的 | 客服 | `customer_id`、最近业务线、最近订单 |
| 安心用订单详情 | 联系客服 | 长租订单号、订单状态、商品、商家、当前待办 |
| 体验租订单详情 | 联系客服 | 短租订单号、门店、车辆、租期、剩余时间、控车状态 |
| 支付失败页 | 联系客服 | 支付单号、金额、通道、失败码 |
| 合同/公证异常页 | 联系客服 | 合同 ID、公证 ID、异常原因 |
| 保证金扣款争议页 | 联系客服 | 扣款单、证据、申诉状态 |

### 1.2 商家端入口

| 页面 | 入口名 | 队列 |
|---|---|---|
| 订单详情 | 联系平台客服 | 长租订单 |
| 我的钱包 / 提现 | 联系财务客服 | 财务 |
| 体验租订单 / 车辆 | 联系体验租客服 | 体验租订单 |
| 商品和价格方案 | 联系运营客服 | 运营 |

### 1.3 运营端客服工作台

客服工作台固定四栏:

1. 会话列表:队列、等待时长、客户/商家、业务线、订单号、优先级。
2. 消息区:文本、图片、视频、订单卡片、系统事件。
3. 上下文卡片:客户脱敏资料、订单状态、账单/支付摘要、门店、短租车辆、异常工单。
4. 操作区:快捷回复、转接、转工单、跳订单、跳退款、跳保证金、跳控车日志。

操作区只能跳转,不得在 IM 内直接退款、扣款、锁车、关单、改价、修改账户。

## 2. 会话状态

| 状态 | 含义 | 进入条件 | 退出条件 |
|---|---|---|---|
| `queued` | 排队中 | 客户发起会话但未分配 | 分配客服 |
| `active` | 处理中 | 客服接入 | 客户离开/转工单/结束 |
| `waiting_customer` | 待客户回复 | 客服发出问题 | 客户回复 |
| `waiting_internal` | 待内部处理 | 已转业务模块或工单 | 内部处理完成 |
| `ticketed` | 已转工单 | 需跨部门处理 | 工单关闭 |
| `closed` | 已结束 | 客服关闭或超时关闭 | 重新咨询新建/重开 |

## 3. 队列和 SLA

| 队列 | 首响 SLA | 升级规则 |
|---|---|---|
| 体验租用车中 / 控车异常 | 1 分钟 | 超时提醒主管 |
| 支付 / 保证金 / 退款 | 3 分钟 | 超时转财务主管 |
| 长租订单 | 3 分钟 | 超时转客服主管 |
| 客诉争议 | 5 分钟内建工单 | 自动标高优先级 |
| 售前咨询 | 5 分钟 | 超时轮转 |
| 商家财务 | 10 分钟 | 超时提醒财务 |

## 4. 接口规则

| 动作 | 接口 | 幂等键 | 关键校验 |
|---|---|---|---|
| 打开会话 | `POST /im/sessions` | `customer_id + business_line + order_id + trigger_page + time_bucket` | 订单归属、端类型、敏感字段过滤 |
| 发送消息 | `POST /im/sessions/{session_id}/messages` | `session_id + client_message_id` | 合规词、附件权限、会话状态 |
| 转接 | 待补接口 | `session_id + from + to + reason` | 目标客服在线/技能匹配 |
| 转工单 | 待补接口 | `session_id + category + order_id` | 必填责任方/证据/优先级 |
| 归档 | 后台任务 | `session_id + archive_version` | 消息、附件、上下文卡片完整 |

## 5. 合规拦截

客服输入命中以下类型时必须提醒或拦截:

| 类型 | 示例 | 动作 |
|---|---|---|
| C 端禁用字段 | 资方、资金来源、合作模式、风控结论 | 阻断发送或主管审批 |
| 绝对承诺 | 包过、一定退款、一定免赔、保证放款 | 阻断发送 |
| 金融误导 | 借款、贷款、融资租赁、账单支付压力 | 提醒改写为订单/账单/支付口径 |
| 私下收款 | 加微信转账、线下打款给个人 | 阻断并上报 |
| 高危操作承诺 | 我直接帮你锁车/扣款/关单 | 提醒跳转业务模块 |

## 6. 与法诉证据包的关系

以下内容必须可按订单导出:

- 会话基础信息
- 消息全文或脱敏摘要
- 图片/视频/附件 hash
- 订单上下文卡片快照
- 转接记录
- 转工单记录
- 合规命中记录
- 客户确认/申诉记录

法诉导出必须写审计日志,客服不能私自下载完整证据包。

## 7. V1 不做但要预留

| 能力 | 预留方式 |
|---|---|
| 机器人自动问答 | 预留 `im_bot_intents`,不阻塞 V1 |
| 智能质检 | 保留合规命中和消息归档,后续接质检 |
| 客户-门店私聊 | 先不开;后续必须先补归档、投诉、监管规则 |
| 电话录音接入 | 工单可上传录音附件 |
| App Push | 通过 `customer_external_accounts` 扩展 |

## 8. 验收标准

1. C 端安心用和体验租订单详情均能打开会话。
2. 会话自动带订单上下文,客服无需让客户重复报订单号。
3. 高危动作在 IM 内只能跳转,不能直接执行。
4. 会话可转工单,工单可追踪责任方、证据、SLA。
5. 禁用词有拦截或提醒。
6. IM 消息、附件和上下文卡片能归入法诉证据包。
