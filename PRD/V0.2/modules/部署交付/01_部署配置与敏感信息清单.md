# 部署配置与敏感信息清单

> 所属：部署交付 / 安全配置
> 目标：提前约束后续开发和部署，所有敏感信息不进入仓库，所有通道配置可切换、可回调、可审计。

---

## 1. 基本原则

1. 仓库只提交代码、配置模板和文档。
2. 真实密钥、账号、数据库访问凭证、回调签名凭证只放服务器环境变量或本地私有配置。
3. `.env`、`config.local.*`、证书文件、私钥文件必须进入 `.gitignore`。
4. README 只说明配置项名称和用途，不写真实值。
5. 生产、测试、演示环境必须分开配置。

---

## 2. 需要隔离的敏感信息

| 类型 | 示例配置名 | 存放位置 |
|---|---|---|
| 数据库 | `DB_HOST`、`DB_USER`、`DB_PASSWORD` | `.env` |
| Redis | `REDIS_HOST`、`REDIS_PASSWORD` | `.env` |
| 支付通道 | `PAY_APP_ID`、`PAY_PRIVATE_KEY` | `.env` 或密钥管理 |
| 打款通道 | `TRANSFER_APP_ID`、`TRANSFER_KEY` | `.env` 或密钥管理 |
| 合同公证 | `ESIGN_APP_ID`、`ESIGN_PRIVATE_KEY` | `.env` 或密钥管理 |
| 风控接口 | `RISK_APP_ID`、`RISK_TOKEN` | `.env` |
| 短信 | `SMS_APP_ID`、`SMS_KEY` | `.env` |
| 物流 | `SF_CLIENT_CODE`、`SF_CHECK_WORD` | `.env` |
| 监管锁 | `LOCK_APP_ID`、`LOCK_APP_SECRET` | `.env` |
| 中控台 | `ZHONGKONG_BASE_URL`、`ZHONGKONG_CLIENT_TOKEN` | `.env` |

---

## 3. 配置文件建议

仓库允许提交：

- `.env.example`
- `config.example.php`
- `config.example.py`
- `docker-compose.example.yml`
- `nginx.example.conf`
- `supervisor.example.conf`

仓库禁止提交：

- `.env`
- `.env.production`
- `.env.local`
- `config.local.php`
- `config.local.py`
- `*.pem`
- `*.key`
- `*.p12`
- `*.crt`
- 数据库导出的真实生产数据

---

## 4. 链路配置中心和部署配置关系

链路配置中心负责业务层切换：

- 支付通道启停
- 合同通道启停
- 公证通道启停
- 风控通道启停
- 物流通道启停
- 打款通道启停
- 回调重试策略
- 灰度商家范围

部署配置负责底层密钥：

- 通道真实应用标识
- 私钥
- 回调签名密钥
- 白名单 IP
- 服务地址

要求：

1. 链路配置中心不展示完整密钥。
2. 运营人员只切换已配置好的通道。
3. 更换密钥需要技术或管理员权限。
4. 每次通道切换生成配置版本和操作日志。

---

## 5. 回调地址规范

每个通道需要配置：

- 支付回调地址
- 退款回调地址
- 打款回调地址
- 合同签署回调地址
- 公证回调地址
- 风控报告回调地址
- 物流轨迹回调地址
- 监管锁回调地址

回调验收：

- 回调必须验签。
- 回调必须幂等。
- 回调原文入库。
- 失败可重试。
- 可按订单、通道、时间、状态查询。

---

## 6. 部署方式需要记录

正式项目 README 需要说明：

- 服务器
- 域名
- 前端部署目录
- 后端部署目录
- 进程管理方式
- 定时任务
- 队列 worker
- 日志目录
- 数据库连接方式
- 备份方式
- 回滚方式

不得写：

- 真实数据库访问凭证
- 真实密钥
- 真实私钥内容
- 生产白名单明文列表

---

## 7. 数据库和附件

数据库：

- 只提交 schema 和 migration。
- 不提交生产订单、客户、身份证、手机号、银行卡、合同等真实数据。
- 测试数据必须脱敏。

附件：

- 身份证、营业执照、合同、照片证据不进入 git。
- 只提交附件路径规则和存储配置模板。
- 文件访问必须经过权限校验。

---

## 8. 上线前检查

上线前必须检查：

- `.gitignore` 已覆盖敏感文件。
- 仓库没有真实手机号、身份证、订单号、密钥。
- `.env.example` 字段齐全。
- 回调地址已配置并能验签。
- 队列 worker 正常。
- 定时任务正常。
- 日志目录可写。
- 数据库 migration 可重复执行。
- 备份和回滚流程可用。

---

## 9. 验收标准

1. 新环境只凭 README、`.env.example` 和部署脚本能搭起来。
2. 不需要改代码即可切换支付、合同、公证、风控等通道。
3. 通道切换有操作日志和配置版本。
4. 敏感信息扫描无真实值。
5. 回调日志能定位所有第三方状态变化。
