# 财务流水模型与对账规则

> 所属端:运营端 / 财务管理
> 目标:把支付、账单、分账、钱包、提现、退款、冲正、对账拆成可开发的数据对象,确保钱流可追溯、可校验、可回滚。

> **⚠️ V0.2 重大修订(2026-05-27)v1.2**:
> - **整体清理"procurement_*"采购账户流水模型**(经合规复核,违反合同口径)
> - **整体重写为 `settlement_*`** 结算账户流水模型,对齐合同口径
> - 流水类型按合作模式(自营 / 联营 / 受让)重新定义
> - 三边对账升级为"四边对账":通道边 + 业务边 + 钱包边 + **平台对公账户 ↔ 通联备付金账户穿透边**
> - 详细规则见 `财务管理/07_门店结算账户与资金穿透架构.md` + `财务管理/08_订单合作模式与收益分配规则.md`

> **⚠️ V0.2 修订(2026-05-26)v1.1**:(已被 v1.2 推翻,仅保留历史记录)

## 1. 财务对象(v1.2 重写)

| 对象 | 作用 |
|---|---|
| 账单 | 客户应付计划和实付状态 |
| 支付流水 | 客户通过通道支付或代扣的结果 |
| **分账流水(allocation)** | 客户支付后,按合作模式拆分给门店/平台/渠道/平台收入 |
| **结算账户流水(settlement_entry)** ⭐ | 门店结算账户的统一入账/出账流水(7 类 entry_type) |
| **平台对公账户穿透日志(platform_passthrough_log)** ⭐ | 平台对公账户 ↔ 通联备付金账户的资金穿透对账 |
| **结算账户提现单(settlement_withdrawal)** ⭐ | 门店从结算账户自助提现 |
| 钱包流水(wallet_entry,渠道账户用)| 渠道账户余额变化 |
| 提现单(withdraw,渠道账户用)| 渠道提现 |
| 打款流水 | 通联代付通道执行结果 |
| 退款单 | 客户退款申请和财务处理 |
| 冲正单 | 已入账金额的反向调整 |
| 对账单 | 四边对账核对结果 |

## 2. 账单模型

字段:

- `bill_id`
- `order_id`
- `period_no`
- `bill_type`:首期 / 租金 / 服务费 / 公证费 / 留购 / 补差价
- `amount_due`
- `amount_paid`
- `amount_refunded`
- `due_time`
- `paid_time`
- `status`
- `price_snapshot_id`
- `config_version_id`

规则:

- 首期实付和后续账单分开。
- 部分支付时账单状态为部分支付。
- 账单结清后才进入可分账流程。
- 退款和冲正不直接删除账单,必须写反向流水。

---

## 3. 支付流水模型

字段:

- `payment_id`
- `bill_id`
- `order_id`
- `channel`
- `channel_trade_no`
- `amount`
- `fee`
- `status`
- `callback_event_id`
- `paid_time`
- `raw_status`

规则:

- 支付成功必须关联账单。
- 一个账单可有多笔支付流水。
- 支付通道回调重复时不重复累加实收。
- 支付异常进入对账中心。

## 4. 分账流水模型(v1.2 按合作模式重新定义)

字段:

- `allocation_id`
- `payment_id`
- `bill_id`
- `order_id`
- `cooperation_mode`:self_operate / joint_venture / receivables_assignment
- `total_amount`:客户支付总额
- `platform_service_fee_amount`:平台综合服务费(留对公户)
- `device_use_fee_amount`:设备使用费部分(穿透通联备付金)
- `merchant_share_amount`:门店分账金额
- `platform_share_amount`:平台分账金额(进平台台账)
- `channel_commission_amount`:渠道佣金(若有)
- `rule_version_id`:分账规则版本
- `status`:pending / completed / failed

规则(按合作模式):

| 合作模式 | 平台综合服务费 | 设备使用费部分(分账)|
|---|---|---|
| `self_operate` | 留对公户 | 100% → 门店 |
| `joint_venture` | 留对公户 | 门店出资比例 × 设备使用费 → 门店;平台出资比例 × 设备使用费 → 平台台账 |
| `receivables_assignment` | 留对公户 | 100% → 平台台账(门店应收账款已转让) |

- 分账失败不改门店结算账户余额。
- 分账成功后才触发资金穿透到通联备付金。
- 详细算法见 `财务管理/08_订单合作模式与收益分配规则.md §6.4`。

## 5. 结算账户流水模型(v1.2 新增,核心)⭐

替代 v1.1 的 procurement_inflow / procurement_withdrawal。完整模型见 `财务管理/07_门店结算账户与资金穿透架构.md §3`。

### 5.1 `settlement_entry` 字段

- `entry_id`
- `account_id`(关联 merchant_settlement_account)
- `direction`:in / out
- `entry_type`:7 类入账/出账类型(详见 §5.2)
- `amount`
- `order_id` / `order_no`(若关联订单)
- `cooperation_mode`:订单合作模式快照
- `source_funder_type`:assignment_payer / customer / joint_venturer / platform
- `platform_account_ref`:平台对公账户过路凭证(银行流水号)
- `tonglian_topup_id`:通联充值流水号
- `tonglian_payout_id`:通联代付流水号
- `balance_before` / `balance_after`
- `description`:摘要(门店端 UI 展示)
- `status`:pending / completed / failed / reversed

### 5.2 `entry_type` 7 类枚举

| 枚举值 | 中文 | 触发场景 | 方向 |
|---|---|---|---|
| `order_settlement` | 订单结算款 | 联营/受让订单监管锁回调 | in |
| `monthly_split` | 月度分账 | 客户每期租金支付后(三模式都触发) | in |
| `joint_venture_profit` | 联营分润 | (并入 monthly_split,实际无此独立流水) | - |
| `platform_reward` | 平台奖励 | 平台主动给的奖励 | in |
| `platform_rebate` | 平台返利 | 渠道返利 / 推广提成 | in |
| `platform_deduction` | 平台扣款 | 违约金扣减 / 罚款 | out |
| `withdrawal` | 提现 | 门店主动提现 | out |
| `financial_adjustment` | 财务调整 | 财务人工调账(经审批) | in/out |

### 5.3 规则

- 门店结算账户余额只由 settlement_entry 计算,不允许直接手改。
- 余额公式:`balance_total = SUM(in) - SUM(out) - balance_withdrawing - balance_pending + balance_pending = balance_available + balance_withdrawing + balance_pending`
- 流水状态 `pending` 时计入 balance_pending;`completed` 时进入 balance_available
- 异常 / 冲正必须写独立反向流水(`status=reversed`),不删原始流水

## 6. 平台对公账户穿透日志(v1.2 新增)⭐

`platform_passthrough_log` 字段:

- `log_id`
- `direction`:in(收入金)/ out(穿透出去)
- `amount`
- `source_account`:来源(客户/受让方/联营方/通联备付金)
- `target_account`:目标(平台对公/通联备付金/门店银行卡)
- `business_ref`:业务关联(订单 ID / 客户支付流水 ID 等)
- `order_id`
- `entry_type`:关联的 settlement_entry.entry_type
- `platform_commission`:本次扣的平台综合服务费(若适用)
- `bank_serial_no`:银行流水号
- `tonglian_serial_no`:通联流水号
- `created_at`

规则:
- 平台对公账户每收/出一笔钱,必须写 platform_passthrough_log
- 用于日终对账"平台对公账户 ↔ 通联备付金账户"的资金穿透是否完全闭环
- 严格的 1:1 等额穿透原则,差异即异常

## 7. 提现与打款(门店结算账户提现)

### 7.1 `settlement_withdrawal` 字段

- `withdrawal_id`
- `account_id`(关联 merchant_settlement_account)
- `merchant_id`
- `amount_requested`:申请金额
- `fee_amount`:手续费(千二,可配)
- `amount_arrival`:实际到账 = amount_requested - fee_amount
- `fee_rate`:费率快照
- `bank_account_id` / `bank_account_no_masked`
- `audit_status`:pending / approved / rejected / no_audit_needed
- `audited_by` / `audited_at` / `audit_reason`
- `tonglian_payout_id`:通联代付流水号
- `payout_status` / `payout_at` / `failure_reason`
- `status`:pending / auditing / paid_processing / paid_success / paid_failed / cancelled

### 7.2 规则

- 门店自助提现,小额(≤¥5万)系统直通,大额(>¥5万)财务主管审核
- 发起提现:冻结 balance_available → balance_withdrawing
- 通联代付成功:balance_withdrawing -= amount,total_withdrawn += amount_arrival
- 通联代付失败:balance_withdrawing 退回到 balance_available
- 财务驳回:同失败处理

## 8. 退款和冲正

退款适用:

- 客户取消订单
- 审核拒绝后退款
- 售后退款
- 重复支付

冲正适用:

- 分账错误
- 结算账户入账错误
- 通道对账差异
- 人工复核后反向调整
- **订单结算款冲正**(订单撤单后追回联营出资款 / 应收账款转让对价)

要求:

- 退款影响支付流水、账单、订单状态
- 冲正影响分账流水和 settlement_entry
- 已提现金额不能直接扣回,需生成负余额或后续抵扣规则
- 退款和冲正必须二次确认和权限控制
- 订单结算款冲正:优先在 settlement_entry 写反向记录(status=reversed),如已提现需财务主管处理

## 9. 四边对账(v1.2 升级)⭐

每日日终对账:

| 对账边 | 内容 |
|---|---|
| 通道边 | 支付、退款、通联代付流水 |
| 业务边 | 订单、账单、分账流水、提现单 |
| 钱包边 | 门店结算账户余额、settlement_entry 累计 |
| **穿透边** ⭐ | **平台对公账户 ↔ 通联备付金账户(穿透是否完全闭环)** |

### 9.1 平账条件

#### 通道边 ↔ 业务边

- 支付通道实收 = 支付流水成功金额
- 支付流水成功金额 = 账单已付金额
- 通联代付成功金额 = 提现单 amount_arrival 累计

#### 业务边 ↔ 钱包边

- 分账流水门店份额累计 = settlement_entry(monthly_split) 累计入账金额
- 订单结算款累计(联营/受让)= settlement_entry(order_settlement) 累计

#### 钱包边

- merchant_settlement_account.total_received = SUM(settlement_entry direction=in status=completed)
- merchant_settlement_account.total_withdrawn = SUM(settlement_withdrawal status=paid_success amount_arrival)
- merchant_settlement_account.balance_total = balance_available + balance_pending + balance_withdrawing

#### 穿透边

- 平台对公账户每笔"out"穿透 = 通联备付金账户对应"in"充值,严格 1:1
- platform_passthrough_log(direction=out) 累计 = 通联备付金账户充值流水累计
- 平台对公账户余额变化 = 收入金 - 平台综合服务费留存 - 穿透出去金额

#### 通联备付金账户内部

- 通联备付金主账户余额 = 所有门店子台账之和 + 平台子台账(联营/受让的平台份额)
- 每个门店子台账余额 = merchant_settlement_account 对应账户的 balance_total

## 10. 异常处理(v1.2 更新)

| 异常 | 处理 |
|---|---|
| 通道有款,系统无支付流水 | 人工关联或补单 |
| 系统支付成功,通道无款 | 冻结订单并禁止分账 |
| 分账不平 | 进入分账异常队列 |
| 门店结算账户余额不平(settlement_entry 累计 ≠ 账户余额)| 禁止提现,财务复核 |
| 通联代付失败 | 退回 balance_available,记录失败 |
| 退款成功但账单未更新 | 重放回调或人工补偿 |
| **监管锁回调到达但订单状态不符** ⭐ | 异常队列,主管核实 |
| **平台对公账户已收钱但通联充值失败** ⭐ | 系统重试 + 财务异常队列 + 人工补穿 |
| **通联充值成功但门店子台账未更新** ⭐ | 系统对账兜底 |
| **平台对公账户 ↔ 通联备付金账户穿透差额** ⭐ | 立即告警 + 财务核实 |
| **同一订单收到重复监管锁回调** ⭐ | 系统幂等校验,只处理首次 |
| **客户租金支付金额与应收金额不符** ⭐ | 进入对账异常 |
| **联营订单平台出资款打款金额错误**(device_price × platform_funding_ratio ≠ amount)| 系统阻止打款,人工核对 |

## 11. 验收标准(v1.2 更新)

1. 任一金额变动都有流水
2. 任一流水能追到来源业务对象
3. 退款和冲正不会删除原始流水
4. 日终对账能定位差异来源
5. 门店结算账户余额可由 settlement_entry 重算
6. 通联备付金账户余额可由 platform_passthrough_log 重算
7. **每笔订单结算款入账可追溯到**:订单 + 合作模式 + 监管锁回调事件 + 出资比例 + 平台对公收款流水 + 通联充值流水
8. **每笔月度分账入账可追溯到**:客户支付流水 + 平台综合服务费扣减 + 合作模式分账规则 + 通联充值流水
9. **每笔提现可追溯到**:提现申请 + 审核记录(若有)+ 通联代付流水 + 银行到账时间
