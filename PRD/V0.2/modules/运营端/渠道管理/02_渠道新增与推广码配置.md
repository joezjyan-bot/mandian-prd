# 渠道新增与推广码配置

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：运营端可创建渠道、生成推广码、配置一二级关系和佣金规则，渠道扫码后进入门店入驻，后续联营订单和平台订单可归因结算。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 渠道新增与推广码配置 |
| 所属端 | 运营端 |
| 入口路径 | 渠道管理 > 渠道列表 > 新增渠道 / 推广码 |
| 使用角色 | 渠道运营、财务、平台管理员 |
| 核心目标 | 管理渠道主体、推广码、层级关系、归属变更、佣金规则和渠道账号 |

---

## 2. 核心口径

1. 渠道推广码扫码后进入门店入驻页面。
2. 门店扫码进入入驻页后，入驻申请立即记录渠道归属和被扫码的推广人员；审核页需要展示该来源。
3. 门店入驻通过后继续绑定渠道归属。
4. 渠道佣金只统计联营订单和平台订单，商家订单默认不统计。
5. 支持一级、二级推广关系，不做无限层级。
6. 佣金可配置固定金额每单或按订单流水/账单实收比例。
7. 佣金进入渠道账户或对应门店钱包，提现由运营端财务审核。

## 5. 推广码配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 推广码名称 | 文本 | 用于区分活动或人员 |
| 推广链接 | 系统生成 | 跳转门店入驻 |
| 二维码 | 系统生成 | 可下载 |
| 有效期 | 日期 | 可长期有效 |
| 绑定渠道 | 只读/选择 | 推广归属 |
| 默认邀请码 | 可选 | 自动填入入驻页 |
| 推广人员 | 选择 | 被扫码人员或推广归属人员 |
| 状态 | 开关 | 生效、停用、过期 |
| 扫码统计 | 只读 | 扫码、注册、提交、通过 |

规则：

1. 推广码停用后不能新增归属，但历史归属保留。
2. 同一商家多次扫码，以首次有效提交或运营确认规则为准。
3. 推广码归属和推广人员写入店铺审核页、店铺详情和渠道统计。

## 6. 佣金规则配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 适用订单类型 | 多选 | 联营订单、平台订单 |
| 计算方式 | 下拉 | 固定金额/单、订单流水比例、账单实收比例 |
| 一级佣金 | 金额/比例 | 直接渠道 |
| 二级佣金 | 金额/比例 | 上级渠道，可为 0 |
| 触发节点 | 下拉 | 首期支付、当期账单结清、订单完成 |
| 逾期冻结 | 开关 | 逾期时冻结佣金 |
| 退款冲正 | 开关 | 退款或关闭时扣回 |
| 结算账期 | 数字 | 满足条件后多久可提现 |

佣金规则发布后生成版本，历史订单使用订单归属时的规则快照。

---

## 7. 渠道归属变更

| 场景 | 规则 |
|---|---|
| 入驻前扫码错误 | 可重新扫码或运营调整 |
| 入驻审核中 | 运营可调整渠道来源 |
| 入驻已通过 | 调整需主管权限 |
| 已产生佣金 | 不自动追溯，需财务冲正 |
| 渠道停用 | 历史归属保留，新订单按规则处理 |

变更必须记录原因、操作人、变更前后和影响范围。

## 8. 渠道账号与权限

| 权限 | 说明 |
|---|---|
| 查看推广码 | 自己的推广码 |
| 查看入驻统计 | 自己推广范围 |
| 查看订单统计 | 联营订单、平台订单，脱敏 |
| 查看佣金 | 自己佣金和提现 |
| 发起提现 | 按配置 |
| 查看下级 | 仅一级渠道或团队负责人 |

渠道端不展示商家订单，不展示客户敏感资料，不展示资方完整账户。
