# 订单合作模式与收益分配规则

> P0 业务文档(2026-05-27)。
> 配套 `财务管理/07_门店结算账户与资金穿透架构.md` 的核心算法补充。
>
> 定义三种合作模式的法律性质、出资模型、收益分配算法、会计科目映射、税务处理建议。
>
> **本文档是合规架构的核心**,必须与《门店入驻及合作框架协议》合同条款保持一致。

> **⚠️ v1.1 修订(2026-05-27)**:
> - 三种合作模式商家端命名统一为 **商家订单 / 联营订单 / 平台订单**(数据库 `cooperation_mode` 保留 `self_operate` / `joint_venture` / `receivables_assignment`)
> - 新增"资金来源"维度(funding_source / funder_id):平台订单可由平台自营资金或外部资方(如蓝海银行)承接
> - 新增资方台账数据模型与闭环对账要求

## 1. 三种合作模式总览

| 模式 | `cooperation_mode` | 商家端 UI 用语 | 业务含义 | 法律性质 |
|---|---|---|---|---|
| **商家订单** | `self_operate` | 商家订单 | 门店全额出资 + 自主经营 | 平台综合服务合同(服务费收入) |
| **联营订单** | `joint_venture` | 联营订单 | 双方按比例共同出资 | 联营合作(投资收益)+ 应收账款部分受让 |
| **平台订单** | `receivables_assignment` | 平台订单 | 平台支付应收账款受让对价 | 应收账款受让(受让溢价收益)|

**核心特征**:
- 三种模式法律性质独立,会计科目独立,税务处理独立
- 系统按 `cooperation_mode` 字段自动适用对应规则
- **联营订单 / 平台订单**还需要关联 `funding_source`(资金来源:平台自营 / 蓝海银行 / 其他资方),决定客户合同模板 + C 端话术模板

## 2. 模式 1:商家订单(self_operate)

### 2.1 业务实质

```
门店:全额出资采购设备 + 自主出租 + 履约服务
平台:提供综合服务(技术 / 风控 / 订单管理 / 代收代付 / 平台支持)
资金来源:不涉及(门店自有资金)
```

### 2.2 出资模型

| 出资方 | 出资比例 | 出资形式 |
|---|---|---|
| 门店 | 100% | 现金(采购设备) |
| 平台 | 0% | 仅提供服务(不出现金)|

### 2.3 收益分配

```
客户支付 ¥500 月供
   ↓
平台综合服务费 ¥10(留对公户,平台收入)
   ↓
设备使用费 ¥490 → 全部进门店结算账户
```

| 月度合计 | 平台 | 门店 |
|---|---|---|
| 收入 | ¥10 | ¥490 |
| 法律性质 | 平台综合服务费(服务收入)| 应收租金 |

### 2.4 风险承担

| 风险类型 | 承担方 |
|---|---|
| 客户违约 / 坏账 | 门店 100% |
| 设备损坏 | 门店 100% |
| 履约风险 | 门店 100% |
| 系统 / 通道风险 | 平台 |

### 2.5 会计科目

(详细科目沿用 v1.0,平台收综合服务费 6% 增值税,门店收租赁收入 9% 增值税或小规模 3%)

### 2.6 税务处理

| 项目 | 平台 | 门店 |
|---|---|---|
| 收入类型 | 服务费收入 | 租赁收入 |
| 增值税税率 | 6%(现代服务 / 商务辅助服务)| 9%(有形动产租赁服务)或小规模 3% |
| 所得税 | 25% | 25%(或小微优惠) |
| 发票开具 | 平台向门店开 6% 增值税专票 | 门店向客户开 9% 增值税普票 |

## 3. 模式 2:联营订单(joint_venture)

### 3.1 业务实质

```
门店:出资 X%(现金,采购部分设备)+ 履约服务
平台:出资 (100-X)%(现金,补足设备采购)+ 综合服务
共同对该订单经营成果有决策权,共担风险,按出资比例分润

资金来源(funding_source):
   - 默认平台自营资金(MVP 阶段联营订单全部走平台自营)
   - 未来可扩展(联营投资合作的资方)
```

### 3.2 出资模型

| 出资方 | 出资比例 | 出资形式 |
|---|---|---|
| 门店 | X%(20-80%,门店选择) | 现金(采购对应比例的设备价值) |
| 平台 | (100-X)% | 现金(补足设备采购) |

### 3.3 出资比例可选档位

| 档位 | 门店:平台 |
|---|---|
| 高门店配资 | 80:20 |
| 偏门店 | 60:40 |
| 平均 | 50:50 |
| 偏平台 | 40:60 |
| 高平台配资 | 20:80 |

由门店在办单系统下单时选择,经平台审核确认后锁定。

### 3.4 收益分配(以 50:50 为例)

#### 阶段 1:订单生效时(监管锁回调)

```
平台支付应收账款转让对价 ¥2,500(50% × ¥5,000)→ 门店结算账户
门店自己也实际投入 ¥2,500(采购设备)
合计:门店与平台共同支撑了设备采购 ¥5,000

同步动作:
- 资方台账(平台自营):funder_receivable_ledger.transferred_amount = ¥2,500
- 资方授信额度:credit_limit_used += ¥2,500
```

#### 阶段 2:客户每月支付 ¥500

```
客户支付 ¥500 → 平台对公账户
   ↓
拆分:
   ├─ 平台综合服务费 ¥10(留对公户)
   └─ 设备使用费 ¥490(穿透通联备付金)
   ↓
按出资比例分账(50:50):
   ├─ 门店 50% × ¥490 = ¥245 → 门店结算账户
   └─ 平台 50% × ¥490 = ¥245 → 通联子台账(平台台账)
                              + 资方台账 collected_amount += ¥245
                              + 资方台账 outstanding_balance -= ¥245
```

#### 阶段 3:订单完成总收益核算(以 12 期 / 总收入 ¥6,000 为例)

| 总账 | 门店 | 平台 |
|---|---|---|
| 出资 | ¥2,500 | ¥2,500 |
| 月度分账累计 | ¥245 × 12 = ¥2,940 | ¥245 × 12 = ¥2,940 |
| **本金回收** | ¥2,500 | ¥2,500 |
| **利润分成** | ¥440 | ¥440 |
| 平台综合服务费 | - | ¥120 |
| **总收益** | ¥440 | ¥560 |

### 3.5 风险承担

| 风险类型 | 承担方 |
|---|---|
| 客户违约 / 坏账 | 按出资比例分担(50:50)|
| 设备损坏 | 门店 100%(设备由门店保管) |
| 履约风险 | 门店 100% |
| 平台综合服务质量 | 平台 100% |

### 3.6 会计科目

#### 方案 A:按"联营投资"处理(争取免增值税,需税务师确认)

```
T0 平台支付应收账款转让对价 ¥2,500:
借:长期股权投资 - 联营投资 - 订单 OD001    ¥2,500
贷:银行存款                                ¥2,500

T1~Tn 客户每月付 ¥500:
借:银行存款                                ¥500
贷:其他业务收入 - 平台综合服务费(¥10)      ¥9.43
贷:应交税费 - 增值税(销项)              ¥0.57
贷:其他应付款 - 门店结算款(¥245)          ¥245
贷:投资收益 - 联营投资收益(¥245)          ¥245
```

#### 方案 B:按"应收账款部分受让"处理(税务保守方案)

(同 v1.0)

**实际选用方案 A 还是 B,由税务师最终确认**。系统设计上两种方案的数据流相同,只是会计科目不同。

### 3.7 税务处理

| 项目 | 平台 | 门店 |
|---|---|---|
| 出资款收支 | 不计收入(投资行为) | 不计收入(投资款) |
| 月度分账本金部分 | 不计税(本金回收) | 不计税(本金回收) |
| 月度分账分润部分 | 联营投资收益,**不交增值税**,交 25% 所得税 | 联营投资收益,**不交增值税**,交 25% 所得税 |
| 平台综合服务费 | 6% 增值税 + 25% 所得税 | - |

### 3.8 必要的合规设计

| 要求 | 实施 |
|---|---|
| 共同出资真实 | 双方各自有出资记录 + 银行流水 |
| 共同经营真实 | 平台真实参与:风控审核、订单授权、客户管理 |
| 共担风险真实 | 坏账损失双方按比例真实承担 |
| 不收固定回报 | 按实际经营成果分,不保底 |
| 合同形式合规 | 联营合作协议 + 单笔订单出资确认 |

详细合同条款见《门店入驻及合作框架协议》第三章(后期 Hudson 配置 + 法务定稿)。

## 4. 模式 3:平台订单(receivables_assignment)

### 4.1 业务实质

```
平台:支付应收账款转让对价(全额设备价)+ 综合服务
门店:全额接受平台支付的对价,但仍是出租方,负责履约 + 设备采购 + 售后
   ↓
平台 100% 受让门店的应收账款权益
客户租金全部归平台
门店赚"采购差价"(进货成本与平台付款的差额,门店自己消化)

资金来源(funding_source):
   - platform_self → 平台自营资金 → 用标准三方租赁服务协议
   - funder_blue_ocean → 蓝海银行 → 用蓝海银行赊销合同
   - 系统按风控规则自动推荐,运营审核确认
```

### 4.2 出资模型

| 出资方 | 出资比例 | 出资形式 |
|---|---|---|
| 门店 | 0% | 不出资(只履约) |
| 平台 / 资方 | 100% | 应收账款转让对价 |

### 4.3 收益分配

#### 阶段 1:订单生效时(监管锁回调)

```
funding_source = platform_self 时:
   平台从自有资金账户支付 ¥5,000 → 门店结算账户
   资方台账(平台自营):transferred_amount = ¥5,000

funding_source = funder_blue_ocean 时:
   蓝海银行根据赊销合同放款 ¥5,000 → 平台对公账户(再代付门店)
   或银行直接打款门店 → 门店结算账户
   资方台账(蓝海银行):transferred_amount = ¥5,000
   蓝海银行授信额度 credit_limit_used += ¥5,000
```

#### 阶段 2:客户每月支付 ¥500

```
客户支付 ¥500 → 平台对公账户
   ↓
拆分:
   ├─ 平台综合服务费 ¥10(留对公户)
   └─ 设备使用费 ¥490(穿透通联备付金,全部归平台子台账)
   ↓
门店 0%,平台子台账 +¥490
   ↓
资方台账:collected_amount += ¥490 (或按资方收益规则部分分给资方)
```

#### 阶段 3:订单完成总收益核算(以 12 期 / 总收入 ¥6,000 为例)

(同 v1.0 表格,门店 + 平台收益不变)

### 4.4 风险承担

| 风险类型 | 承担方 |
|---|---|
| 客户违约 / 坏账 | funding_source 决定:平台自营 → 平台担;蓝海银行 → 双方按合同 |
| 设备损坏 | 门店 100% |
| 履约风险 | 门店 100% |
| 客户合同纠纷 | 门店 + 平台 + 资方按合同协商 |

### 4.5 会计科目

(同 v1.0)

### 4.6 税务处理

(同 v1.0,但需注意:蓝海银行赊销情况下,平台收的"通道服务费"而非"受让溢价",税目按服务费 6% 增值税即可)

### 4.7 必要的合规设计

| 要求 | 实施 |
|---|---|
| 应收账款真实存在 | 真实的客户租赁合同 + 客户身份验证 |
| 转让对价合理 | 不显著偏离应收账款账面价值 |
| 转让真实有效 | 客户签署《应收账款转让确认书》 |
| 转让流程合规 | 合同 + 通知 + 系统留痕 |
| 蓝海银行赊销 | 客户签蓝海银行《分期付款销售合同》,设备所有权直接归客户 |

## 5. 三种模式的对比总览

(沿用 v1.0 第 5 节内容,把"自营/联营/受让"对应到"商家/联营/平台"即可)

| 维度 | 商家订单 | 联营订单(50/50)| 平台订单 |

### 6.4 分账算法(系统核心)

```python
def calculate_monthly_split(payment_amount, order):
    """客户每期租金支付后,按合作模式分账"""
    
    # 步骤 1:计算平台综合服务费(留对公户)
    platform_service_fee = payment_amount * order.platform_service_fee_rate
    
    # 步骤 2:计算设备使用费部分(穿透通联备付金)
    device_use_fee = payment_amount - platform_service_fee
    
    # 步骤 3:按合作模式分账
    if order.cooperation_mode == 'self_operate':  # 商家订单
        merchant_share = device_use_fee
        platform_share = 0
    elif order.cooperation_mode == 'joint_venture':  # 联营订单
        merchant_share = device_use_fee * order.merchant_funding_ratio
        platform_share = device_use_fee * order.platform_funding_ratio
    else:  # receivables_assignment 平台订单
        merchant_share = 0
        platform_share = device_use_fee
    
    # 步骤 4:同步更新资方台账
    if order.funder_id and platform_share > 0:
        update_funder_ledger(order.funder_id, order.id, collected_amount=platform_share)
    
    return {
        'platform_service_fee': platform_service_fee,
        'merchant_share': merchant_share,
        'platform_share': platform_share,
    }
```
