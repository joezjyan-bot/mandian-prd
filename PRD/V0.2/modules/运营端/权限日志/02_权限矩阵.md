# 权限矩阵

> **来源**: P0-2 / P1-3 / P2-2

## 1. 概述

V1 角色 + 关键操作的权限对照表。本表是 V1 上线时配置 RBAC 的依据。

## 2. 角色清单

### 2.1 平台侧(运营端)
- 平台超级管理员(super_admin)

- 客服主管(cs_lead)

- 资方管理客服(funder_manager)

## 4. 二次确认要求

以下操作即使有权限,仍需二次确认 + 完整日志:
- 退款执行 / 提现打款
- 后台余额变动
- 关闭订单 / 取消重下
- 修改抽佣比例
- 角色变更 / 权限变更
- 客户加黑名单
- 删除操作(若有)
- 修改还款日

## 5. 高频审计点

- 客服查看完整客户身份证号(展开次数)
- 客服查看完整银行卡号
- 操作日志的导出
- 大额退款发起(> X 元,X 可配)

## 6. 数据模型

权限矩阵存数据库 `permission_matrix` 表:
- role_key
- permission_key
- is_allowed
- requires_2fa
- requires_approval(谁审批)
