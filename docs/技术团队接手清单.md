# 技术团队接手清单

> 配合 `system/` 目录的 Laravel 脚手架使用。这份是“接什么、怎么接”的说明。

## 一、这套脚手架是什么 / 不是什么

**是：** 一个能跑起来、结构正确、能演示主干流程的 Laravel 11 起点。所有外部对接（签约/支付/监管锁/实名）用模拟模式，`EXTERNAL_MODE=mock` 一键跑通演示。

**不是：** 上线成品。支付/监管锁/电子签的真实对接、资金对账、安全/压测、灰度，都需要团队接手后完成。

## 二、动手前确认（已定）

- 框架：**Laravel 11**（PHP 8.2+），代码前缀 `yzz`
- 中控台（Java）：HTTP API + Webhook，**无 SDK**，PHP 用 Guzzle 调
- 旧 `mandian-java-rebuild` Java 仓已废弃
- 金额一律整数“分”（bigint），禁浮点

## 三、模拟模式怎么工作（最重要）

1. 所有外部依赖抽象成 `app/Contracts/` 下的接口
2. `app/Services/External/Mock/` 是演示实现（返回成功）
3. `app/Services/External/Real/` 是空壳 + `TODO[团队]`，等你们填真实对接
4. `app/Providers/ExternalServiceProvider.php` 根据 `EXTERNAL_MODE` 绑定 mock/real
5. **业务代码只 type-hint 接口，不感知实现**——这是“换真对接不动业务代码”的关键

你们上线要做的：填完 4 个 `Real*Service`、把 `.env` 改 `EXTERNAL_MODE=real`、在 `config/app.php` 注册 `ExternalServiceProvider`。

## 四、该接着建哪些模块（按优先级）

**第一优先（主干，脚手架已起头）：** 用户认证·商品·办单助手·订单·签约[模]·支付[模]·交付签收·账单履约·到期三选一

**第二优先（资金+后台）：** 四账分离·分账·提现·退款·对账·运营后台·三类合作模式差异

**第三优先（支撑）：** 权限与审计·风控·监管锁[模]·IM客服·设备资产·短租取还车·法诉证据包

## 五、每个模块开工前问产品的事

- 状态机怎么流转？（文档 `全局/02 状态字典`）
- 哪些字段绝不能给 C 端？（C 端红线）
- 哪些是高危操作、要审批留痕？（文档 `权限矩阵`）
- 钱怎么算、怎么分？（文档 `财务流水模型`）

## 六、关键提醒（踩坑用）

- **申请购买价 A 口径**（剩余租金+保证金）是合规敏感项，已做成可切换（`config/business.php` 的 `buyout_formula`），二期要换折旧余值，别写死。
- 支付/分账/提现回调**必须幂等**（脚手架用 `channel_trade_no + callback_event_id` 唯一约束示范）。
- 模拟与真实对接在**同一接口层**切换，业务代码不感知。
- 碰钱碰合规的部分，上线前必须经联调、测试、资金对账、灰度。

## 七、演示怎么跑

```bash
cd system
composer install
cp .env.example .env && php artisan key:generate
# 配数据库后：
php artisan migrate
php artisan serve
curl http://127.0.0.1:8000/api/health   # 看到 external_mode=mock 即正常
```
