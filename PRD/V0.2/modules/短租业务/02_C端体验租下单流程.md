# C 端体验租下单流程

> 参考九号租车客户端路径：先选城市/门店/车辆，再选短租方案，补骑行人和实名信息，支付后进入待确认、待取车、已取车、还车/续租。

---

## 1. 入口

| 入口 | 说明 |
|---|---|
| 首页体验租 | 主入口，展示城市、品类、附近门店、租赁周期 |
| 扫码下单 | 扫门店体验租二维码，自动带入门店、品类和可租方案 |
| 门店详情 | 从附近门店列表进入，展示该门店可租车型 |
| 活动 banner | 跳转体验租品类或指定门店 |

体验租入口不得跳入安心用长租办单助手。

---

## 2. 首页信息

页面字段：

- 取车城市
- 取车地点或附近门店
- 距最近门店距离
- 品类导航：电动车、手机、数码设备等
- 租车周期：时租、日租、周租、月租，季租后续可配置
- 立即选车
- 扫码下单
- 活动 banner
- 门店营业状态
- 官方反诈和用车安全提示

筛选逻辑：

1. 客户授权定位后，按 GCJ-02 经纬度查询附近门店。
2. 默认半径可配置，建议 20 km。
3. 只展示已开通体验租、当前品类有可租库存或可预约方案的门店。
4. 按距离、库存、营业状态、价格可排序。

---

## 3. 选择门店

门店卡片展示：

- 门店名称
- 距离
- 地址
- 营业时间
- 联系商家
- 导航到店
- 可租车型摘要

筛选项：

- 城市
- 门店名称搜索
- 租赁周期
- 品类
- 是否营业中
- 是否有可租库存

---

## 4. 选择车辆

车辆列表展示的是车型，不直接暴露具体 VIN。

字段：

- 车型图片
- 车型名称
- 简短卖点
- 当前周期最低价
- 可租库存状态
- 立即租
- 保障标签
- 可选短租周期标签

选中车型后进入订单确认页，系统锁定门店、车型、价格方案快照。

---

## 5. 订单确认

订单确认页包含：

- 车型卡片
- 选择租车方案
- 取还信息
- 门店信息
- 联系商家
- 用户协议勾选
- 合计金额
- 确认订单
- 保障包/服务包
- 押金/免押说明
- 到期和逾期说明
- 取车后开始计时提示

### 5.1 租车方案

第一版支持：

| 周期 | 示例 |
|---|---|
| 1 小时 | 固定价格 |
| 2 小时 | 固定价格 |
| 4 小时 | 固定价格 |
| 当日还 | 指定当日最晚还车时间 |
| 日租 | 按天 |
| 周租 | 后续可开关 |
| 月租 | 后续可开关 |

超过 8 小时是否按日租封顶由后台配置；提前还车默认不退租金。

### 5.2 协议校验

客户未勾选协议时，点击确认订单提示“请先阅读并同意相关协议”。

协议包括：

- 体验租服务协议
- 人脸识别服务协议
- 押金/免押授权协议
- 控车和定位授权说明
- 隐私政策
- 用车安全和损坏赔付说明

### 5.3 费用明细

费用展示必须拆开：

| 费用 | 说明 |
|---|---|
| 租金 | 按所选短租方案计算 |
| 保障包 | 可选，未选择时不得默认收费 |
| 押金/预授权 | 按门店、车型、客户信用和规则判断 |
| 优惠 | 优惠券、活动减免 |
| 合计应付 | 当前需支付金额 |

短租客户侧不展示门店分账、平台抽佣、渠道佣金。

---

## 6. 骑行人资料与认证

如果当前客户未维护骑行人信息，进入骑行人信息页。

字段：

- 证件类型
- 证件号码
- 姓名
- 手机号
- 紧急联系人，短租第一版可选

提交后进入人脸认证：

1. 勾选人脸识别服务协议。
2. 调用实名/人脸服务。
3. 认证成功后回到订单支付。
4. 认证失败时保留订单草稿，不生成待接单订单。

---

## 7. 支付和押金

支付顺序：

1. 支付租金和可选保障包。
2. 判断是否支持信用免押。
3. 免押成功则生成押金授权记录。
4. 免押失败或未开启免押时支付押金。
5. 支付完成后生成体验租订单和核销码。

支付方式展示：

- 全额支付
- 押金支付/预授权
- 信用免押

---

## 8. 订单状态和客户动作

| 状态 | 客户可见文案 | 客户动作 |
|---|---|---|
| 待支付 | 待支付 | 继续支付、取消 |
| 押金待支付 | 押金待支付 | 支付押金/发起免押 |
| 待确认 | 商家接单中 | 联系商家、取消 |
| 待取车 | 待取车 | 联系商家、导航到店、展示核销码/扫码绑车 |
| 已取车 | 使用中 | 开机、关机、定位、续租、去还车 |
| 待还车确认 | 还车确认中 | 查看还车进度、联系客服 |
| 已完成 | 已完成 | 查看押金退款、再次租用 |
| 已取消 | 已取消 | 查看退款状态 |

门店 1 小时内未确认接单，第一版可自动取消或转人工处理，规则由后台配置。

订单详情顶部必须展示：

- 门店名称和首租/续租标签。
- 车型名称和图片。
- 租赁方案。
- 取车时间、还车时间、租期倒计时。
- 核销码(待取车展示)。
- 联系商家、导航到店。
- 当前可操作按钮。

---

## 9. 取车

取车方式：

1. 到店出示核销码，由门店核销并绑定车辆。
2. 扫码绑车，客户扫码车辆码后门店或系统确认。

取车成功后：

- 写入实际取车时间。
- 计算租赁结束时间。
- 绑定车辆 VIN/SN/设备号。
- 调用中控开机或允许客户开机。
- 订单进入已取车。

---

## 10. 用车和续租

已取车状态展示：

- 租期剩余倒计时
- 到期时间
- 开机
- 关机
- 闪灯鸣笛
- 查看定位
- 更多控车能力
- 去还车
- 去续租

逾期后：

- 客户端提示已逾期或即将逾期。
- 可隐藏部分控车按钮，要求联系商家或运营端处理。
- 逾期费第一版可只做后台手动扣押金，不在 C 端展示复杂自动金额。

续租规则：

1. 只允许已取车订单续租。
2. 当前订单存在未结费用时不能续租。
3. 续租生成新短租订单。
4. 首租押金可复用到续租订单。
5. 续租开始时间等于上一单结束时间。

---

## 11. 还车

第一版只支持到店还车。

客户点击“去还车”：

1. 展示导航到店。
2. 到店后门店确认还车。
3. 门店检查车辆状态。
4. 系统计算是否有扣费。
5. 押金全部退还或部分退还。
6. 订单完成。

后续再扩展异地还车、调度费、自动还车点选择。

---

## 12. C 端体验租底部导航关系

体验租作为 C 端底部导航独立入口，与安心用并列。

```text
首页
  -> 体验租入口
体验租
  -> 城市/门店/车型/方案
订单
  -> 体验租订单待办
我的
  -> 骑行人、实名、人脸、客服、协议
```

体验租不得出现在门店长租办单助手三入口中，避免店员把短租订单误办成长租订单。

---

## 13. 门店端对接点

C 端体验租订单会给门店端生成待办：

| C 端状态 | 门店端动作 |
|---|---|
| 待确认 | 接单/拒单 |
| 待取车 | 核销码核销、选择并绑定唯一设备码 |
| 已取车 | 查看用车中订单、必要时协助控车 |
| 待还车确认 | 车辆验收、损坏扣款、押金处理 |
| 已完成 | 查看历史和再次出租 |

门店员工权限需细分：

- 可接单。
- 可核销取车。
- 可确认还车。
- 可发起押金扣款申请。
- 不可修改收款账户。

---

## 14. 后台配置项

体验租后台至少配置：

- 开通城市。
- 开通门店。
- 开通品类。
- 门店营业时间。
- 车型和图片。
- 车辆/设备库存。
- 时租、日租、周租、月租价格方案。
- 当日还最晚还车时间。
- 押金/免押规则。
- 保障包。
- 接单超时规则。
- 续租规则。
- 控车能力开关。
- 押金扣款审批规则。

---

## 15. 告知内容

体验租 C 端必须明确告知：

1. 取车后开始计算订单时长。
2. 到期前需要还车或续租。
3. 逾期可能影响控车能力和押金处理。
4. 车辆损坏、丢失、违规使用可能产生扣款。
5. 控车、定位、人脸识别均需客户授权。
6. 门店和平台不会要求客户向私人账户转账。
