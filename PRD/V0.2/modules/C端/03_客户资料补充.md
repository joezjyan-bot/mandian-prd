# 客户资料补充

> **来源**: P1-5

## 1. 页面定位

C 端 · 实名 / 已有资料的客户跳过 · 新客户首次办单收集基础资料

## 2. 收集字段

### 2.1 必填
- 姓名(若实名通过则锁定)
- 身份证号(若实名通过则锁定)
- 联系手机号(已登录时锁定)
- 收货地址(详细到门牌号)
- 紧急联系人(姓名 + 关系 + 手机号)
- 工作单位 / 学校(选填,部分订单走信用评估时可能要)

### 2.2 可选 / 业务规则要求时必填
- 紧急联系人 2(部分业务规则要求)
- 工作单位地址
- 月收入区间
- 学历

### 2.3 部分订单可能跳过的字段
- 短租场景:工作 / 学校 / 收入 可能不收集

## 3. 字段动态展示

根据订单的链路配置 + 业务规则要求:
- 必填字段不同
- 可选字段不同

订单创建时锁定所需字段集,客户按提示填

## 4. 门店代填(P1-5 规则)

| 字段 | 可门店代填 |
|---|---|
| 姓名 | ❌(由身份证实名后锁定) |
| 身份证号 | ❌(实名后锁定) |
| 手机号 | ❌(客户登录账号) |
| 收货地址 | ✅ |
| 紧急联系人 | ✅ |
| 工作 / 学校 | ✅ |
| 月收入 / 学历 | ✅ |

门店代填记录:operator_id + is_proxy_by_store = true

## 5. 校验

- 身份证号:格式校验(15/18 位 + 校验位)
- 手机号:格式 + 国内有效号段
- 紧急联系人手机号:不能与本人相同
- 紧急联系人手机号:格式校验

## 6. 提交

提交后:
- 创建 customer 记录或更新现有
- 关联订单
- 订单状态:pending_kyc(若未实名) / pending_review
- 跳转下一步(实名或合同等)

## 7. 隐私

资料展示:默认脱敏
完整查看权限:门店主账号(本店客户)/ 客服(主管以上)

## 8. 数据模型

- `customer` 表
- `customer_kyc_info` 表(KYC 详细字段)
- `order.customer_info_snapshot`(订单创建时客户信息快照)
