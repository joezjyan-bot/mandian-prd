## V0.2 修订原则

1. **V0.1 不直接覆盖**：V0.1 作为历史初稿保留，V0.2 记录修订后的真实口径。
2. **先定核心链路**：商品 → 办单助手 → 下单 → 审核 → 合同/支付/授权 → 发货 → 账单 → 分账 → 提现。
3. **再补页面级 PRD**：每个模块后续单独拆页面、按钮、弹窗、失败态、数据状态、权限边界。
4. **UI 必须可视化**：后续页面级 PRD 不只写文字，还要输出完整界面图或低保真原型图，包含箭头、字段说明和点击路径。
5. **钱相关动作只写流程**：涉及打款、扣款、提现、确认支付等动作，文档必须明确审核/确认边界，实操采集时不能点击最终确认。

## 下一步拆分建议

建议按以下顺序继续：

1. 全局登录、权限、左侧导航、工作台框架
2. 订单管理：订单列表、待审核、订单详情、客服 IM、分配资方
3. 商品管理与商品同步
4. 办单助手三入口
5. 三类订单创建与审核
6. 租赁模式配置：长租/短租、小时/天/周/月、起租和归还规则
7. 库存设备管理：设备码、仓库、短租锁定、交付、归还验收
8. 合同、支付、授权、发货
9. 财务分账、钱包、提现、对账
10. 渠道推广与佣金
11. 店铺入驻与员工账号
12. 黑名单、监管锁、租后管理、客诉管理
