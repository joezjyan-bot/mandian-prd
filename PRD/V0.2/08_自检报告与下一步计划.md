# V0.2 自检报告与下一步计划

> 检查时间：2026-05-24
> 检查范围：V0.2 总纲、页面级 PRD、参考资料、README 和模块缺口表。
> 检查目标：确认当前文档是否符合已沟通口径，是否只是页面描述，还是已经具备数据互通、可开发、可验收的闭环。

---

## 1. 自检结论

当前 V0.2 已经从“模块罗列”推进到“核心业务闭环”：

1. 订单类型已统一为 `门店订单 / 分红订单 / 平台订单`。
2. 运营端、商家 PC 端、门店手机端、C 端、渠道端的主链路均已有页面级 PRD。
3. 商品、办单助手、订单、审核、资方、合同、公证、支付、发货、设备、账单、分账、提现、租后、客诉、权限、日志已经有明确联动关系。
4. 中控台边界已明确：它是租赁系统外部能力系统，不并入租赁系统。
5. 普通门店端不保留资金充值入口，只保留收益、提现、流水和对账。
6. UI 已明确后置，后续默认参考 `finance-zhongtai` 企业中台风格。

本次自检发现并修复了正式 PRD 中少量历史旧称说明，避免后续开发误读为正式枚举。

## 2. 正常业务流程走查

### 2.1 商家入驻

```text
门店登录页
  -> 立即入驻
  -> 注册账号
  -> 上传营业执照、法人身份证、门头照片、地址定位、收款账户
  -> e 签宝授权签章
  -> 运营端店铺审核
  -> 通过后开通商家 PC 端、门店手机端、商家钱包、默认链路配置
```

已覆盖文档：

- `modules/商家端/入驻认证/01_门店入驻.md`
- `modules/运营端/店铺管理/01_店铺审核.md`
- `modules/运营端/店铺管理/02_店铺详情与商户配置.md`
- `modules/商家端/门店资料/01_资料变更与授权状态.md`

自检结果：闭环成立。资料、授权、渠道来源、收款账户、审核结果和商家账号都有承接位置。

  -> 平台订单：运营审核 + 匹配资方/平台主控

## 3. 数据互通检查

| 数据对象 | 当前承接 | 自检结论 |
|---|---|---|
| 商品 | 商品库、规格、办单助手同步、订单快照 | 可互通 |
| 订单 | 三类订单、订单详情、状态机、待办 | 可互通 |
| 客户 | 客户详情、实名资料、风控、客诉、租后 | 可互通 |
| 商家 | 入驻、店铺详情、资料变更、员工、钱包 | 可互通 |
| 资方 | 资方配置、订单、账单、分账、账户 | 可互通 |
| 渠道 | 推广码、归属、订单统计、佣金、提现 | 可互通 |
| 设备 | 设备码、仓库、出库、归还验收、监管锁 | 可互通 |
| 财务 | 支付、账单、分账、退款、冲正、提现、对账 | 可互通 |
| 合同公证 | 配置、发起、客户待办、回调、附件 | 可互通 |
| 日志 | 操作日志、回调日志、配置版本、敏感查看 | 可互通 |

当前风险不在“有没有页面”，而在下一阶段必须把这些对象落成统一数据模型和事件模型。否则开发时仍可能做成多个孤岛表。
