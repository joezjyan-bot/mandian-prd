# 改价补资料与客服 IM

> **⚠️ V0.2 Stage 6 同步修订(2026-05-27)v1.1**:
> - 同步展示用语:商家订单 / 联营订单 / 平台订单 / 履约中 / 逾期费用。
> - 底层字段、接口、枚举不变;状态定义以 `02_状态字典与订单状态机.md` 索引和全局状态字典为准。

> 页面级 PRD 草案。
> 目标：细化审核客服在待审核订单里改套餐、改价、补资料、联系客服的操作，确保客户看到的是最新方案，后台保留价格快照和沟通记录。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 改价补资料与客服 IM |
| 所属端 | 运营端、商家端、C 端 |
| 运营入口 | 订单详情 / 待审核队列 > 改套餐 / 补资料 / 联系客户 |
| 商家入口 | 商家端订单详情 > 联系客服 |
| 核心目标 | 审核中调整价格方案、发起补资料、同步订单上下文到 IM |

---

## 2. 核心口径

1. 客服可以改套餐、改价，但不能直接覆盖原始办单助手价格，需要生成新价格快照。
2. 改价后客户下单页、订单详情、账单、合同变量都要使用最新生效快照。
3. 补资料必须生成客户或商家待办，不能只写备注。
4. 商家 PC 端查看平台订单进度时，可以点联系客服并带订单上下文。
5. IM 是客服沟通入口，不替代审核结论和操作日志。

---

## 3. 改套餐/改价弹窗

| 字段 | 类型 | 说明 |
|---|---|---|
| 当前方案 | 只读 | 原始办单助手快照 |
| 商品/规格 | 只读/可选 | 是否允许更换按权限 |
| 租赁模式 | 只读/可选 | 长租、短租 |
| 租期 | 输入/选择 | 小时、天、周、月按配置 |
| 首期实付 | 金额 | 可调整 |
| 后期账单 | 金额/期数 | 可调整 |
| 押金 | 金额 | 可调整 |
| 服务费 | 金额 | 可调整或选择增值服务 |
| 公证费 | 金额 | 来自增值服务或公证配置 |
| 留购价 | 金额 | 可调整 |
| 配资比例 | 下拉 | 联营订单使用 |
| 改价原因 | 下拉/文本 | 必填 |
| 是否需要客户确认 | 开关 | 默认需要 |
| 是否发起补充合同 | 开关 | 金额变化建议需要 |

规则：

1. 修改后生成价格快照版本。
2. 原始价格快照不可删除。
3. 金额变化超过配置阈值时需要主管复核。
4. 合同已签署后改价必须走补充合同或订单重审流程。
5. 客户未确认新方案时，不进入后续发货。

---

## 4. 补资料弹窗

| 字段 | 类型 | 说明 |
|---|---|---|
| 补资料对象 | 下拉 | 客户、商家、门店 |
| 资料类型 | 多选 | 身份证、联系人、地址、门头照片、授权、银行卡等 |
| 补充原因 | 文本 | 必填，客户可见摘要 |
| 截止时间 | 时间 | 可配置 |
| 是否阻断审核 | 开关 | 默认阻断 |
| 通知方式 | 多选 | 小程序、短信、站内信、IM |

补资料结果：

1. 生成待办并推送到对应端。
2. 订单进入待补资料队列。
3. 客户或商家提交后重新进入审核。
4. 所有资料进入附件中心并保留旧版本。

---

## 5. 客服 IM 入口

### 5.1 商家端联系客服

| 场景 | 说明 |
|---|---|
| 平台订单待审核 | 商家想了解审核进度 |
| 需要补资料 | 商家咨询补充要求 |
| 客服改价 | 商家确认新方案 |
| 发货或签约异常 | 商家咨询下一步 |

点击后：

1. 调起聊天框。
2. 自动推送订单上下文给客服。
3. 聊天记录与订单关联。
4. 客服可以从 IM 跳回订单详情处理。

### 5.2 运营端联系客户

运营端可从订单详情联系客户，默认携带订单摘要、当前待办和需要客户完成的动作。客户隐私按权限脱敏。

## 6. IM 上下文字段

| 字段 | 说明 |
|---|---|
| 订单号 | 关联订单 |
| 订单类型 | 商家订单、联营订单、平台订单 |
| 商家/门店 | 订单来源 |
| 客户摘要 | 脱敏姓名和手机号 |
| 商品方案 | 商品、规格、租期、价格摘要 |
| 当前状态 | 待审核、待补资料、待签约等 |
| 当前待办 | 客户或商家需要完成什么 |
| 最近备注 | 审核备注摘要 |

IM 不推送完整身份证、银行卡、详细风控报告。

## 7. 操作日志

| 动作 | 日志 |
|---|---|
| 改价 | 原方案、新方案、原因、操作人、是否客户确认 |
| 改套餐 | 原商品/规格、新商品/规格、原因 |
| 发起补资料 | 资料类型、对象、截止时间 |
| 客户提交补资料 | 提交时间、附件版本 |
| 发起 IM | 发起人、接收方、订单上下文 |
| 客服处理 | 处理结论、跳转订单动作 |

---

## 8. 风险控制

1. 改价和补资料不能绕过订单审核状态机。
2. 改价后账单、合同、公证、分账必须重新读取价格快照。
3. 已支付订单改价需要走退款、补款或冲正流程。
4. IM 沟通内容不作为最终财务凭证，财务动作必须回到系统按钮和流水。
