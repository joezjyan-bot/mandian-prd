# 资方配置与分配规则

> 页面级 PRD 草案。
> 目标：给分红订单、平台订单的审核分配、财务分账、资方账户和后续资方端只读看板提供统一数据来源。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 资方配置与分配规则 |
| 所属端 | 运营端 |
| 入口路径 | 资方管理 > 资方列表 / 资方详情 / 分配规则 / 资方账单 |
| 使用角色 | 平台管理员、运营主管、财务、审核客服只读或分配 |
| 核心目标 | 维护资方主体、账户、额度、支持订单类型、支持类目、分配规则、资金流水和状态 |

V1 不建议先做复杂资方独立端，先在运营端把资方配置、订单分配、账单和账户跑通。后续资方端可以做只读看板。

---

## 2. 核心口径

1. 分红订单和平台订单分配资方前，资方必须先在运营端配置并启用。
2. 门店订单不需要资方分配。
3. 资方配置影响审核工作台可选资方、订单详情资方信息、财务分账、资方账户和对账。
4. 资方可按订单类型、商品类目、商家、区域、额度、状态限制可分配范围。
5. 资方变更必须受订单节点限制，合同签署后和已分账后不能普通变更。
6. 资方资金账户、回款、提现、冲正必须进入财务总账。

## 6. 分配规则

### 6.1 分红订单

| 规则 | 说明 |
|---|---|
| 出资比例范围 | 默认 20% 到 80%，可按资方配置支持范围 |
| 出资金额 | 设备价乘资方出资比例 |
| 门店占比 | 100% 减资方占比 |
| 抽佣 | 从门店和资方份额分别扣除平台抽佣 |
| 资方额度占用 | 分配成功后占用对应出资金额 |
| 回款释放 | 客户还款、分账、订单完结后按规则释放或滚动计算 |

### 6.2 平台订单

| 规则 | 说明 |
|---|---|
| 出资主体 | 资方或平台资金 |
| 出资比例 | 默认 100% |
| 门店收益 | 默认无，可配置协作服务费 |
| 渠道佣金 | 有渠道来源时按渠道规则生成 |
| 资方额度占用 | 按订单设备价或资金需求占用 |

### 6.3 自动推荐资方

系统可根据规则给审核客服推荐资方，但 V1 不强制自动分配。

| 推荐因子 | 说明 |
|---|---|
| 资方状态 | 启用优先 |
| 可用额度 | 额度充足优先 |
| 支持类目 | 商品类目匹配 |
| 支持区域 | 客户或门店区域匹配 |
| 历史逾期率 | 低逾期优先 |
| 当前待回款 | 风险较低优先 |
| 分配均衡 | 避免过度集中 |

## 7. 审核分配弹窗

审核台点击「分配资方」时，弹窗读取本模块配置。

| 字段 | 说明 |
|---|---|
| 当前订单 | 订单号、订单类型、商家、客户脱敏、商品 |
| 设备价/资金需求 | 办单助手或审核台传入 |
| 资方出资比例 | 分红订单可选，平台订单默认 100% |
| 可选资方 | 按状态、类目、额度、规则过滤 |
| 分账预览 | 门店、资方、平台抽佣、渠道佣金 |
| 额度占用 | 分配后占用多少额度 |
| 备注 | 分配原因或说明 |

分配成功后写入订单 `funder_id`、分账规则版本、出资比例、出资金额、分配人、分配时间。

## 9. 资方账单与账户

| 字段 | 说明 |
|---|---|
| 资方账户 | 关联财务账户 |
| 出资记录 | 分配订单产生的资金占用 |
| 回款记录 | 客户还款后资方份额入账 |
| 抽佣记录 | 平台从资方份额扣除的抽佣 |
| 待提现 | 满足结算周期的可提现金额 |
| 已提现 | 已打款金额 |
| 冻结金额 | 争议、退款、逾期风险冻结 |
| 对账状态 | 未对账、已对账、异常 |

资方账单进入 `财务管理 > 钱包、分账、提现与对账` 的统一总账，不单独形成孤立财务逻辑。

## 11. 异常规则

| 异常 | 处理 |
|---|---|
| 资方额度不足 | 审核台不可选择或提示额度不足 |
| 资方被冻结 | 未完成订单进入异常提醒，不允许新分配 |
| 已签合同后更换资方 | 禁止普通操作，需主管审批和补充合同 |
| 已分账后发现资方错误 | 走冲正或财务调整，不直接改历史记录 |
| 资方账户未实名 | 可配置为不可分配或只读观察 |
| 资方逾期率过高 | 自动降权推荐或冻结 |

---

## 12. 待确认

1. 资方额度是按设备价占用，还是按订单剩余应收或实际出资额占用。
2. 资方是否需要对分配订单进行确认；V1 建议运营分配即生效。
3. 资方端 V1 是否只读开放，还是完全由运营端代管。
