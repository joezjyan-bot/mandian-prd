# 签收确认配置

> V0.2.2 同步声明：本文件按 V0.2.2 最新口径补入正式仓库。订单展示统一为“商家订单 / 联营订单 / 平台订单”；客户侧不展示内部资金、合作模式、审核结论等敏感信息；涉及资金、账单、结算和证据链的动作必须留痕。

> 页面级 PRD 草案。  
> 目标：把客户签收确认方式统一配置为 `电子确认单` 或 `AI 问答确认`，并让订单、客户侧入口、附件中心、操作日志和起租规则联动。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 签收确认配置 |
| 所属端 | 运营端 |
| 入口路径 | 配置管理 > 发货履约 > 签收确认配置 |

| 核心目标 | 配置不同订单类型、商家、商品类目使用电子确认单还是 AI 问答确认 |

---

## 2. 核心口径

1. 客户签收确认只保留两种正式方式：`电子确认单`、`AI 问答确认`。
2. 后台配置选择哪种方式，客户侧订单详情就只展示对应入口。
3. 物流签收回调、商家上传交付材料、客服人工复核都不能替代客户本人签收确认。
4. 签收完成后才能按配置触发起租、账单计划生效、结算前置判断和监管锁状态校验。
5. 所有签收确认结果必须进入订单详情、附件中心、操作日志和异常待办。

---

## 2.1 无监管锁品类结算验收规则

适用范围:商品类目或商品配置 `has_regulatory_lock = false` 的长租订单。无监管锁是正常业务形态,不得因为没有监管锁而阻断业务。只有配置为 `requires_regulatory_lock_callback = true` 的品类/商品才要求监管锁回调后结算;当前默认苹果手机等需要监管锁回调的品类走有锁结算三条件,其它品类走本节无锁验收条件。

### 2.1.1 判定字段

| 字段 | 类型 | 来源 | 判定口径 |
|---|---|---|---|
| `has_regulatory_lock` | boolean | 商品/类目配置 | `true` 走结算三条件;`false` 走无锁替代验收 |
| `requires_regulatory_lock_callback` | boolean | 商品/类目配置 | 仅苹果手机等配置为 `true` 的品类要求锁状态回调后结算 |
| `receipt_status` | enum | 签收确认记录 | 必须为 `confirmed` |
| `receipt_confirm_method` | enum | 签收配置 | `electronic_confirmation` 或 `ai_video_confirmation` |
| `device_identifier_archived` | boolean | 发货/交付设备档案 | IMEI/SN/VIN/车辆识别码等已归档 |
| `delivery_evidence_complete` | boolean | 交付附件校验 | 交付照片、交付时间、交付人、客户确认记录齐全 |
| `delivery_evidence_attachment_ids` | array | 附件中心 | 至少包含平台配置要求的交付照片/凭证 |
| `pre_delivery_check_passed` | boolean | 审核/交付前置检查 | 审核人员已完成该品类要求的前置检测、资料核验或人工监管动作 |
| `manual_review_passed` | boolean | 人工复核记录 | 仅在规则配置要求人工复核时作为补强条件 |
| `alternative_acceptance_rule_id` | string | 替代验收规则配置 | 记录本次使用的规则版本 |
| `alternative_acceptance_result` | enum | 结算校验 | `passed / missing_rule / evidence_incomplete / manual_review_required` |

### 2.1.2 等价替代条件

有监管锁品类结算条件:

```text
receipt_status = confirmed
AND regulatory_lock_status = locked
AND activation_lock_status = activated
```

无监管锁品类等价替代条件:

```text
has_regulatory_lock = false
AND requires_regulatory_lock_callback = false
AND receipt_status = confirmed
AND device_identifier_archived = true
AND delivery_evidence_complete = true
AND pre_delivery_check_passed = true
```

说明:

1. 无监管锁品类使用平台默认验收规则,不得因“没有监管锁”本身返回 blocker。
2. 设备识别码缺失、交付证据缺失或前置检测未完成时,结算校验必须返回对应 blocker,不得进入自动结算。
3. 人工复核只允许补强无锁品类的替代验收,不能替代客户本人签收确认。
4. 替代验收通过后,订单可进入 `PENDING_PLATFORM_SETTLEMENT`,但仍需财务/系统按权限和幂等规则发起结算。
5. 若某品类被配置为 `requires_regulatory_lock_callback = true`,则必须等待监管锁回调和激活锁状态,不得套用无锁验收规则。

## 3. 配置层级

| 层级 | 说明 |
|---|---|
| 平台默认 | 全平台兜底签收方式 |
| 订单类型 | 商家订单、联营订单、平台订单 |
| 商家 | 指定商家覆盖 |
| 商品类目 | 电动车、手机、电脑等类目覆盖 |
| 商品 | 单商品特殊覆盖 |

继承优先级：商品 > 商品类目 > 商家 > 订单类型 > 平台默认。

---

## 4. 配置字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 配置名称 | 文本 | 例如“电动车长租 AI 问答确认” |
| 启用范围 | 多选 | 订单类型、商家、类目、商品 |
| 签收确认方式 | 单选 | 电子确认单 / AI 问答确认 |
| 是否启用 | 开关 | 关闭后走上级配置 |
| 起租触发 | 下拉 | 签收完成后 / 发货完成后 / 门店交付后 |
| 异常处理方式 | 下拉 | 转人工复核 / 允许重试 / 终止签收 |
| 客户提示文案 | 文本 | 客户进入签收前看到的提示 |
| 生效时间 | 时间 | 支持配置生效 |
| 版本号 | 只读 | 配置变更留痕 |

---

## 5. 电子确认单配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 确认单模板 | 下拉 | 电子签署模板 |
| 签署服务 | 下拉 | e签宝/其他电子签署通道 |
| 是否需要本人实名校验 | 开关 | 默认开启 |
| 是否允许重发 | 开关 | 签署失败可重新发起 |
| 最大重发次数 | 数字 | 防止重复骚扰 |
| PDF 归档 | 开关 | 默认进入附件中心 |

确认单模板至少包含：

1. 订单号、客户实名、商品和规格。
2. 设备识别码：IMEI/SN/VIN 或车辆识别码。
3. 租期、期数、每期应付、保证金、服务费。
4. 发货方式或商家交付信息。
5. 客户确认已收到设备或已完成交付。
6. 客户确认设备外观、配件和基础功能已验收。

## 6. AI 问答确认配置

### 6.1 模板字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 模板名称 | 文本 | 例如“电动车长租签收确认模板” |
| 总时长目标 | 数字 | 建议 20-30 秒 |
| 是否做人脸校验 | 开关 | 默认开启 |
| 是否录制视频 | 开关 | 默认开启 |
| 是否生成文本 | 开关 | 默认开启 |
| 允许重试次数 | 数字 | 默认 1-2 次 |
| 异常后动作 | 下拉 | 转人工/终止/允许重试 |

### 6.2 问题字段

| 字段 | 类型 | 说明 |
|---|---|---|
| question_id | ID | 问题 ID |
| question_text | 文本 | AI 播报问题 |
| answer_type | 枚举 | 是/否、复述、金额确认、自由回答 |
| expected_answer | 文本/规则 | 标准答案或匹配规则 |
| abnormal_answer | 文本/规则 | 异常答案或拒绝答案 |
| required | 开关 | 是否必答 |
| sort_order | 数字 | 问题顺序 |
| max_answer_seconds | 数字 | 单题回答时长 |
| fail_action | 枚举 | 转人工、允许重试、终止签收 |

### 6.3 默认问题建议

| 顺序 | 问题 | 通过口径 |
|---|---|---|
| 1 | 请确认是否您本人办理本订单 | 回答是/本人 |
| 2 | 请说出您本次办理的设备名称 | 与订单商品或设备快照匹配 |
| 3 | 请确认本订单租期期数或使用周期 | 与订单快照匹配 |
| 4 | 请确认每期应付金额或本次应付金额 | 与账单快照匹配 |
| 5 | 请确认您已清楚账单支付、归还和逾期费用规则 | 回答清楚/已知晓 |

## 7. 客户侧展示

| 配置方式 | 客户侧按钮 | 客户完成动作 |
|---|---|---|
| 电子确认单 | 签署电子确认单 | 跳转电子签署，签署完成回到订单详情 |
| AI 问答确认 | 开始 AI 问答确认 | 阅读授权，开启视频问答，完成后等待结果 |

客户进入签收前必须展示订单、设备、金额、租期和异议入口。

---

## 8. 异常与复核

| 异常 | 处理 |
|---|---|
| 电子确认单签署失败 | 可重发或转人工 |
| 电子签署回调超时 | 进入回调异常队列 |
| AI 回答异常 | 进入人工复核 |
| AI 识别失败 | 按配置允许重试或转人工 |
| 客户提出异议 | 进入交付争议 |
| 长期未完成签收 | 提醒客户、商家和客服 |

人工复核只能处理异常结果，不能绕过客户本人确认直接完成签收。

## 11. 后续制度 / 供应商事项

1. 电子确认单默认模板由法务确认。

3. AI 视频服务商和人脸识别服务商是否同源，需要技术选型。
