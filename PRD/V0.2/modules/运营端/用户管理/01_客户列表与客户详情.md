# 客户列表与客户详情

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：运营端形成客户资料总账，支持按客户、订单、门店、资方、渠道、风控、投诉、租后等维度快速检索，并把客户历史订单和风险信息聚合到审核工作台。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 客户列表与客户详情 |
| 所属端 | 运营端 |
| 入口路径 | 用户管理 > 客户列表 > 客户详情 |
| 使用角色 | 审核客服、运营客服、租后客服、客诉人员、风控、平台管理员 |
| 核心目标 | 查询客户资料、历史订单、授权状态、投诉、租后、黑名单和风控摘要 |
| 关联模块 | 订单管理、黑名单库、客诉管理、租后管理、附件中心、操作日志 |

---

## 2. 核心口径

1. 客户列表不是营销用户池，主要服务租赁订单审核、售后、租后和客诉。
2. 客户资料必须脱敏展示，完整敏感信息需要权限和查看日志。
3. 客户详情聚合所有端产生的数据，但不能让客服随意改实名资料。
4. 客户历史订单要按三类订单分开展示：商家订单、联营订单、平台订单。
5. 黑名单命中、历史逾期、投诉、重复下单、资料异常需要在详情顶部做风险标签。
6. 客户资料、风控报告、合同、公证、交付照片等统一走附件中心权限。

---

## 3. 客户列表筛选

搜索条件需要比普通列表更全，方便客服和风控快速定位。

| 字段 | 类型 | 说明 |
|---|---|---|
| 客户姓名 | 文本 | 模糊搜索，结果脱敏 |
| 手机号 | 文本 | 支持完整输入查询，列表脱敏 |
| 身份证号 | 文本 | 精确查询，结果脱敏 |
| 订单号 | 文本 | 反查客户 |
| 商品名称 | 文本 | 按历史订单商品搜索 |
| 商家/门店 | 搜索 | 订单归属 |
| 资方 | 搜索 | 联营订单、平台订单 |
| 渠道来源 | 搜索 | 推广渠道 |
| 订单类型 | 下拉 | 商家订单、联营订单、平台订单 |
| 订单状态 | 下拉 | 待审核、在租、逾期、完成、关闭等 |
| 下单日期 | 日期范围 | 按订单创建时间 |
| 支付日期 | 日期范围 | 按账单实收时间 |
| 认证状态 | 下拉 | 未认证、已实名、人脸通过、授权完成 |
| 风控标签 | 下拉 | 黑名单疑似、历史逾期、投诉、资料异常 |
| 所在地区 | 省市区 | 按收货/定位/认证地区 |

## 4. 客户列表字段

| 字段 | 说明 |
|---|---|
| 客户摘要 | 脱敏姓名、脱敏手机号、年龄、认证状态 |
| 最近订单 | 最近订单号、订单类型、商品、状态 |
| 历史订单 | 总订单数、在租数、逾期数、完成数、关闭数 |
| 风险标签 | 黑名单、逾期、投诉、资料异常、重复下单 |
| 商家/门店 | 最近订单归属 |
| 资方 | 最近联营/平台订单资方 |
| 渠道来源 | 如有则展示 |
| 最近操作 | 最近支付、授权、投诉、租后跟进等摘要 |
| 操作 | 查看详情、查看订单、加黑名单、备注、日志 |

规则：

1. 列表不展示完整身份证、银行卡、详细地址。
2. 黑名单疑似命中要突出展示，但不直接自动拒绝订单。
3. 客户信息只能从订单、授权和资料页同步，不在列表直接编辑。

## 5. 客户详情结构

```text
客户详情
├─ 概览
├─ 实名与授权
├─ 历史订单
├─ 风控资料
├─ 账单与支付
├─ 客诉记录
├─ 租后记录
├─ 地址与设备
└─ 操作日志
```

---

## 6. 概览

| 模块 | 内容 |
|---|---|
| 基础摘要 | 脱敏姓名、脱敏手机号、年龄、性别、认证状态 |
| 风险标签 | 历史逾期、投诉、黑名单疑似、资料异常 |
| 订单概览 | 商家订单、联营订单、平台订单分别统计 |
| 当前待办 | 补资料、签合同、公证、还款、归还、客诉 |
| 最近动态 | 最近一次下单、支付、签约、租后跟进、投诉 |
| 快捷入口 | 查看最新订单、查看风控、加黑名单、发起备注 |

---

## 7. 实名与授权

| 字段 | 说明 |
|---|---|
| 实名状态 | 未实名、已实名、实名失败 |
| 身份证信息 | 脱敏展示，完整查看需权限 |
| 人脸状态 | 未发起、已通过、失败、过期 |
| 风控/征信授权 | 未发起、授权中、已完成、失败 |
| 代扣绑卡 | 未绑定、签约中、已签约、失败 |
| 合同签署 | 最近合同状态摘要 |
| 公证状态 | 最近公证状态摘要 |

资料规则：

1. 客户本人提交的资料只能作为事实记录，运营修改必须走补资料或备注。
2. 授权失败、过期、取消都要记录原因和回调时间。
3. 审核资料附件进入附件中心。

## 8. 历史订单

历史订单按类型分 tab：

| Tab | 内容 |
|---|---|
| 全部订单 | 所有订单 |
| 商家订单 | 门店自营订单 |
| 联营订单 | 门店出设备、资方补足资金的订单 |
| 平台订单 | 平台/资方全额承接订单 |

字段：

| 字段 | 说明 |
|---|---|
| 订单号 | 跳转订单详情 |
| 订单类型 | 商家订单、联营订单、平台订单 |
| 商品 | 商品名称、规格 |
| 商家/门店 | 订单归属 |
| 资方 | 如有 |
| 下单时间 | 创建时间 |
| 当前状态 | 主状态 |
| 已付/未付 | 账单摘要 |
| 逾期情况 | 逾期期数、逾期金额 |
| 投诉/售后 | 是否存在 |

---

## 9. 风控资料

| 模块 | 内容 |
|---|---|
| 黑名单匹配 | 内部黑名单、外部黑名单预留、命中字段摘要 |
| 风控报告 | 报告类型、发起时间、结果摘要、附件 |
| 设备归还记录 | 是否有损坏、缺件、争议 |
| 历史逾期 | 逾期次数、最长逾期天数、是否结清 |
| 重复下单 | 近期多单、同设备、多门店下单 |
| 地理信息 | 收货地址、定位信息、地址差异摘要 |

风控报告内容按权限查看，普通客服只看结论标签和建议。

---

## 10. 客诉与租后

| 页签 | 内容 |
|---|---|
| 客诉记录 | 支付宝投诉、平台投诉、处理状态、关联订单 |
| 租后记录 | 逾期订单、跟进状态、承诺还款、法务动作 |
| 退款售后 | 退款申请、冲正、归还售后、留购售后 |

客户详情只做聚合展示，实际处理仍回到客诉、租后、订单详情等模块。

---

## 11. 操作与日志

| 操作 | 权限 |
|---|---|
| 查看完整敏感信息 | 高权限，记录日志 |
| 加黑名单 | 风控/主管权限，必须填原因 |
| 添加客户备注 | 客服可用 |
| 发起补资料 | 审核客服可用 |
| 查看附件 | 按附件权限 |
| 导出客户订单 | 管理权限 |

所有查看敏感资料、加黑名单、补资料、备注修改都必须进入操作日志。
