# 审核策略与风控规则

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：把订单审核、补资料、拒绝原因、黑名单提示、风控授权、人工复核、超时规则配置化，供商家订单、联营订单、平台订单分别使用。

---

## 2. 核心口径

1. 商家订单默认商家自审，但平台可配置监管和异常介入策略。
2. 联营订单、平台订单默认运营端审核。
3. 审核策略按平台默认、订单类型、商家、商品/规格覆盖。
4. 风控报告、黑名单、征信授权、人脸识别、补资料要求都由策略触发。
5. 审核结果必须结构化，不能只写自由备注。
6. 策略变更只影响新订单或新审核任务，历史订单按快照执行。
7. 高风险策略发布需要二次确认和日志。

## 4. 策略字段

### 4.1 基础

| 字段 | 类型 | 说明 |
|---|---|---|
| 策略名称 | 文本 | 必填 |
| 适用订单类型 | 多选 | 三类订单 |
| 适用商家 | 多选 | 可为空表示全平台 |
| 适用类目/商品 | 多选 | 可精细覆盖 |
| 审核主体 | 下拉 | 商家/门店、运营平台 |
| 是否自动分配审核人 | 开关 | 进入待审核后自动派单 |
| 是否允许商家补资料 | 开关 | 商家可协助补资料 |
| 是否允许客服改套餐 | 开关 | 审核中改价改套餐 |

### 4.2 资料要求

| 资料项 | 是否可配置 |
|---|---|
| 实名信息 | 是 |
| 身份证照片 | 是 |
| 人脸识别 | 是 |
| 收货/使用地址 | 是 |
| 定位授权 | 是 |
| 紧急联系人 | 是 |
| 工作/学生证明 | 是 |
| 银行卡/代扣签约 | 是 |
| 补充资料 | 是 |

每个资料项配置：是否必填、触发节点、是否允许后补、驳回后是否重新审核。

### 4.3 风控动作

| 动作 | 说明 |
|---|---|
| 黑名单匹配 | 内部/外部黑名单 |
| 实名校验 | 二要素、三要素 |
| 人脸识别 | 客户扫脸 |
| 风控报告 | 中控台或外部报告 |
| 流水报告 | 客户授权后生成 |
| 征信授权 | 客户签署授权 |
| 定位校验 | 下单地、收货地、使用地 |
| 历史订单校验 | 逾期、关闭、拒绝记录 |

## 5. 审核队列规则

| 条件 | 进入队列 |
|---|---|
| 资料齐全 | 待资料审核 |
| 资料缺失 | 待补资料 |
| 黑名单命中 | 异常队列/主管复核 |
| 风控报告未完成 | 待风控 |
| 联营/平台未分配内部资金来源 | 待分配内部资金来源 |
| 授权未完成 | 待客户授权 |
| 合同或支付前置缺失 | 待签约支付 |

队列规则决定运营工作台和订单审核列表的数量。

| 结果 | 必填字段 |
|---|---|
| 审核通过 | 审核意见 |
| 审核拒绝 | 拒绝原因、通知对象 |

| 原因 | 说明 |
|---|---|
| 资料不完整 | 必填资料缺失 |
| 资料不一致 | 实名、身份证、地址等冲突 |
| 风控不通过 | 报告或授权结果不满足 |
| 黑名单命中 | 内部或外部库命中 |
| 资方拒绝 | 资方不承接 |
| 重复下单 | 重复订单 |
| 客户取消 | 客户主动取消 |
| 商家撤回 | 商家不继续 |

## 7. 自动化与人工边界

| 场景 | V1 建议 |
|---|---|
| 黑名单命中 | 弹窗提示 + 主管复核，不自动拒绝 |
| 实名失败 | 允许重试，超过次数人工 |
| 人脸失败 | 允许重试，超过次数人工 |
| 风控报告异常 | 进入异常，不自动通过 |
| 资料齐全低风险 | 可自动进入待分配/待签约，但仍可人工审核 |
| 支付回调失败 | 不改变审核结果，进入异常队列 |

## 8. 超时规则

| 场景 | 规则 |
|---|---|
| 待审核超时 | 提醒审核人和主管 |
| 待补资料超时 | 提醒客户/商家，可关闭或暂挂 |
| 待客户授权超时 | 允许重发或关闭 |
| 待内部资金来源分配超时 | 提醒内部资金来源分配人员 |
| 待合同签署超时 | 重发合同或关闭 |

超时阈值按订单类型、商家、资方、商品配置。
