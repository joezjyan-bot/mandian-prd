# 商品选择器与办单助手同步字段

> 页面级 PRD 草案。
> 目标：明确运营端、商家端、门店手机端办单助手选择商品时需要同步哪些字段，避免商品库和办单助手价格方案各算各的。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商品选择器与办单助手同步字段 |
| 所属端 | 运营端、商家端、门店手机端 |
| 入口路径 | 商品管理、办单助手、审核改价 |
| 使用角色 | 商品运营、商家老板、审核客服、门店员工 |
| 核心目标 | 统一商品选择、规格、长短租、增值服务、设备库存和二维码下单字段 |

---

## 2. 核心口径

1. 同一商品可以同时有全新、二手等规格，不需要重复建两个商品。
2. 商品可配置是否同步办单助手。
3. 门店订单使用商家自己的商品和配置；分红订单、平台订单使用运营端配置或平台同步商品。
4. 商品同步到办单助手时，只同步可用规格、租赁模式、价格参数和增值服务范围。
5. 客户扫码下单后，价格快照必须带回后台订单。

---

## 3. 商品选择器字段

| 字段 | 说明 |
|---|---|
| 商品名称 | 搜索展示 |
| 类目 | 手机、电动车、电脑等 |
| 品牌/型号 | 可选 |
| 规格 | 全新、二手、容量、颜色、电池等 |
| 租赁模式 | 长租、短租 |
| 计费单位 | 小时、天、周、月 |
| 是否需要设备码 | 短租和车辆类建议必填 |
| 是否同步办单助手 | 开关 |
| 可用订单类型 | 门店订单、分红订单、平台订单 |
| 增值服务 | 可选服务范围 |

---

## 4. 同步字段

| 字段组 | 字段 |
|---|---|
| 商品基础 | 商品 ID、名称、类目、图片、规格 |
| 租赁配置 | 长租/短租、计费单位、起租时长、可选租期 |
| 价格配置 | 设备价、首期、后期账单、押金、服务费、公证费、留购价 |
| 分红配置 | 可选配资比例、默认比例、资方范围 |
| 增值服务 | 服务项、金额、是否必选、是否可改 |
| 库存设备 | 是否绑定设备码、库存状态、可用门店 |
| 链路配置 | 合同、公证、支付、发货、审核策略版本 |

---

## 5. 办单助手回传字段

客户扫码下单后回传：

| 字段 | 说明 |
|---|---|
| 商品和规格快照 | 下单时展示内容 |
| 订单类型 | 门店订单、分红订单、平台订单 |
| 租赁方案 | 租期、计费单位、起止时间 |
| 费用明细 | 首期、后期账单、押金、服务费、公证费、留购价 |
| 增值服务 | 客户选择或默认项 |
| 配资比例 | 分红订单 |
| 设备码 | 短租或指定设备 |
| 办单员工 | 员工账号 |
| 二维码来源 | 办单助手、商品二维码、设备二维码 |

---

## 6. 审核改价联动

审核客服改价时必须基于办单助手快照生成新版本。新版本要同步账单、合同变量、分账和客户确认待办。

---

## 7. 异常

| 场景 | 处理 |
|---|---|
| 商品下架 | 历史订单保留快照，新办单不可选 |
| 规格停用 | 已生成二维码可按配置失效 |
| 增值服务变更 | 历史订单不变，新订单用新版本 |
| 库存不足 | 短租不可生成可下单二维码 |
| 配置缺失 | 办单助手不展示该商品或规格 |
