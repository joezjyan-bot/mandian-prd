# 线下支付标记规则

> **来源**: P0-1 / P0-3 决策
> **跨端通用规则**: 运营端、商家 PC 端、门店手机端均涉及

---

## 1. 总规则

| 维度 | 口径 |
|---|---|
| 是否支持 | V1 支持 |
| 适用订单 | **门店订单 + 平台订单**(分红订单**禁止**线下支付) |
| 操作主体 | 门店主账号 + 后台客服 |
| 凭证要求 | 不强制上传凭证 |
| 资金流向 | 客户钱直接给门店(不经过平台代收) |
| 平台抽佣 | 平台事后从相关钱包扣 2% |

## 2. 为什么分红订单禁止线下支付

分红订单涉及资方对接平台资金,资金链路必须可追溯:
- 资方对账要求 100% 资金流可追踪
- 线下支付无线上凭证,事后追溯难
- 一旦资方质疑"客户没付",平台无法举证

**强制规则**:分红订单的所有支付必须线上。门店或客服尝试标记线下支付时,系统拒绝并提示。

## 3. 操作权限

| 角色 | 是否可标记线下支付 |
|---|---|
| 门店主账号 | ✅ 可(仅门店/平台订单) |
| 门店员工账号 | ❌ 不可(员工权限只到办单) |
| 后台普通客服 | ✅ 可(视权限矩阵分配) |
| 后台客服主管 | ✅ 可 |
| 运营管理员 | ✅ 可 |

## 4. 抽佣承担方

| 订单类型 | 线下 1000 后 2% 抽佣 |
|---|---|
| 门店订单 | 全部 20 元从**门店钱包**扣 |
| 分红订单 | ❌ 禁止线下支付 |
| 平台订单 | 全部 20 元从**资方钱包**扣(余额不足:进入待扣费队列) |

## 5. 门店标记流程

```
1. 门店在订单详情找到对应账单期
2. 点击"标记线下支付"
3. 弹窗确认:
   - 显示订单号、账单期、应收金额
   - 输入实收金额(默认=应收金额,允许部分支付)
   - 可选:简单备注
4. 二次确认("此操作不可撤销,确认已收到客户款项?")
5. 提交后:
   - 系统记录线下支付标记
   - 系统自动从门店钱包扣 2% 手续费(门店订单)
   - 余额不足时挂账,业务继续
   - 留操作日志
```

## 6. 客服代标记流程

```
1. 客服在订单详情找到对应账单期
2. 点击"标记线下支付"
3. 客服必填:
   - 实收金额
   - 资金核实情况(下拉必选):门店已确认 / 客户已确认 / 凭证已查看 / 其他(必填备注)
   - 备注(可选)
4. 二次确认
5. 提交后:
   - 系统记录线下支付标记 + 客服代标记字段
   - 自动给门店发推送通知:"客服 XX 已标记该订单第 X 期为线下支付"
   - 门店异议期 24 小时
   - 按订单类型扣 2%
   - 留操作日志
```

### 6.1 异议处理

门店发起异议后:
- 标记进入"异议中"状态
- 抽佣扣款暂停(已扣不主动退)
- 运营客服介入核实
- 处理结果:
  - 异议成立:撤销标记,已扣手续费退回
  - 异议不成立:标记正常生效,通知门店

## 7. 平台订单的资方扣费处理

| 场景 | 处理 |
|---|---|
| 资方钱包余额充足 | 实时扣款,流水"资方扣款" |
| 资方钱包余额不足 | 进入**待扣费队列**,等资方下次回款时优先扣除 |
| 长期未结清 | 月度对账时与资方协商;资方退出时挂账追溯 |

资方钱包**不阻断**业务,标记可继续。

## 8. 流水标注模板

| 场景 | 流水备注 |
|---|---|
| 门店订单 - 门店标记 | `订单 XX 第 XX 期 线下支付手续费` |
| 门店订单 - 客服代标记 | `订单 XX 第 XX 期 线下支付手续费(客服 XX 代标记)` |
| 平台订单 - 门店标记 | `订单 XX 第 XX 期 线下支付手续费(资方扣款)` |
| 平台订单 - 客服代标记 | `订单 XX 第 XX 期 线下支付手续费(资方扣款,客服 XX 代标记)` |
| 资方待扣 | `订单 XX 应扣未扣 - 待回款扣除` |

## 9. UI 入口

### 9.1 门店手机端
- 订单详情 → 账单期列表
- 每期账单显示"标记线下支付"按钮(未支付状态时)
- 仅门店主账号可见;门店员工不可见
- 分红订单上**此按钮自动隐藏**

### 9.2 运营端
- 订单详情 → 账单期列表
- 每期账单显示"标记线下支付"按钮
- 分红订单上**此按钮自动隐藏**

### 9.3 商家 PC 端
- 同门店手机端,商家主账号可见

## 10. 数据模型

```sql
CREATE TABLE offline_payment_mark (
    mark_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_id BIGINT NOT NULL,
    bill_period INT NOT NULL,
    payment_amount DECIMAL(10,2) NOT NULL,
    order_type ENUM('store', 'platform') NOT NULL,

    operator_id BIGINT NOT NULL,
    operator_role ENUM('store', 'customer_service', 'admin') NOT NULL,

    verification_type VARCHAR(50),
    verification_note VARCHAR(500),

    fee_amount DECIMAL(10,2) NOT NULL,
    fee_target ENUM('store_wallet', 'funder_wallet') NOT NULL,
    fee_status ENUM('deducted', 'pending', 'failed') DEFAULT 'pending',

    notification_status VARCHAR(50),
    notification_sent_at TIMESTAMP NULL,

    objection_status ENUM('none', 'raised', 'handled') DEFAULT 'none',
    objection_deadline TIMESTAMP NULL,
    objection_handled_by BIGINT NULL,
    objection_result VARCHAR(500),

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_order (order_id, bill_period),
    INDEX idx_operator (operator_id, operator_role),
    INDEX idx_status (fee_status, objection_status)
);
```

## 11. 关联补偿动作

1. 抽佣扣款 → 从对应钱包扣 2%(失败挂账)
2. 门店通知(客服代标记时) + 24 小时异议倍计时
3. 账单状态变"线下已支付",不参与逾期违约金计算
4. 该期不计入逾期

## 12. 风险与边界

### 已知风险
- 门店虚报金额:V1 不阻断,抽佣金额异常告警(配合月度对账)
- 门店漏报:V1 不主动检测,靠对账发现

### V1 不在范围
- 凭证图片 OCR 自动审核
- 客户端"我已线下付款"按钮
- 线下支付金额自动校验
