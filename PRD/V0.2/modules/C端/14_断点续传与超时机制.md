# 断点续传与超时机制

> **来源**: P1-5

## 1. 业务背景

客户办单流程可能因网络异常 / 临时离开 / 客户主动中断 / 后台等待等被打断。系统支持断点续传 + 超时自动关单。

## 2. 断点续传

### 2.1 触发场景
客户在 1-6 流程中任何环节中断:
- 关闭小程序
- 网络异常
- 临时离开

### 2.2 系统行为
- 系统记住当前进度(写入 `order_c_progress` 表)
- 客户下次进入"我的订单":
  - 看到此订单的当前状态
  - 显示"继续办单"按钮
- 点击 → 从断点位置继续
- 已填资料 / 已上传照片 / 已签内容**不重做**

### 2.3 客户进度可视化
- 进度条:1/8 订单确认 → 2/8 实名 → 3/8 ...
- 客户清楚还需几步

## 3. 节点超时

### 3.1 客户长期不推进(待客户响应)
| 时间 | 行为 |
|---|---|
| 24 小时未推进 | 订单状态 → "待处理" + 推送提醒 |
| 72 小时未推进 | 订单自动关闭(未签合同免费,P1-3 关闭规则) |

### 3.2 后台发起后客户响应
| 节点 | 推送方式 | 时效 |
|---|---|---|
| 合同签署 | 小程序通知 + 短信 | 24 小时未签 → 客服跟进 |
| 公证签署 | 小程序通知 | 48 小时未签 → 客服核实 |
| 代扣绑卡 | 小程序通知 | 7 天未绑 → 客服跟进 |
| 收货确认 | 小程序通知 | 14 天未确认 → 客服跟进;30 天后默认收货 |

### 3.3 自动关单
未签合同情况 + 72 小时不推进:
- 系统自动关单(免费,P1-3)
- 释放预保留资源(设备 / 临时锁的资金)
- 推送客户

## 4. 客服干预

任何状态下客户卡住,系统自动:
- 推送给客服(运营端工作台高亮显示)
- 客服可主动联系客户协助
- 客服可标记"已沟通"或"客户放弃"

## 5. 客户主动取消

任何状态下客户主动取消:
- 在订单详情点"取消订单"
- 仅未签合同前免费取消
- 已签合同需走 P1-3 取消重下流程

## 6. 数据模型

`order_c_progress` 表(00 决策表中已定义):
- order_id / current_step / step_history / is_continuable
- last_active_at / timeout_at
- failed_attempts JSON
- intervention_required / intervention_reason

## 7. 推送渠道

- 支付宝小程序通知
- 微信小程序通知
- 短信(关键节点)
- 邮件(可选)
