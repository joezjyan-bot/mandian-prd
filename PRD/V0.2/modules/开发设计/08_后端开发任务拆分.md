# 后端开发任务拆分

> 所属：开发设计
> 目标：把 V0.2 PRD、数据模型、接口清单和 OpenAPI 草案拆成可排期、可分工、可验收的后端任务。这里不是最终技术方案，但必须避免只做页面、不打通数据。

---

## 1. 拆分原则

1. 后端按业务域拆模块，接口按业务动作落 controller。
2. 每个写接口必须同时考虑状态机、操作日志、幂等、权限和回调补偿。
3. 钱包、分账、提现、退款、冲正、对账不能散落在订单模块里，必须进入财务域统一处理。
4. 第三方回调只做验签、入库、排队和幂等处理，业务状态由 worker 消费后推进。
5. 商家订单、联营订单、平台订单共用订单底座，但审核主体、分账规则、发货责任和可见范围按订单类型分流。
6. 中控台不并入租赁系统，只保留后续接口对接边界。

| Assistant | 办单助手报价、二维码、价格快照、账单预览 | Product、Order、Finance |
| Order | 三类订单、客户资料、审核、补资料、改价、订单状态机 | Customer、Contract、Finance |
| Review | 审核工作台、内部资金来源分配、风控提示、黑名单匹配 | Order、Funder、Blacklist |
| Contract | 合同、补充合同、公证、授权待办、合同回调 | Order、Callback |
| Fulfillment | 发货、设备绑定、签收、归还验收、物流接口 | Device、Order |
| Device | 设备码、库存、短租锁定、出入库、监管锁预留 | Fulfillment |
| Finance | 账单、支付、分账、钱包、提现、退款、冲正、对账 | Order、Channel、Funder |
| Channel | 渠道、推广码、渠道归属、佣金、渠道端统计 | Merchant、Order、Finance |
| PostRent | 逾期、催收案件、外部催收同步、回款记录 | Order、Finance |
| Complaint | 支付宝投诉同步、投诉处理、订单上下文 | Order、Callback |
| Config | 链路配置、租赁模式、通道、审核、财务、版本灰度 | 全模块 |
| Callback | 第三方回调统一入口、验签、事件入库、重放 | Contract、Finance、Fulfillment |
| PermissionLog | 操作日志、敏感查看日志、配置版本日志 | 全模块 |

## 3. Controller 拆分

| Controller | 主要接口 |
|---|---|
| `AuthController` | 登录、刷新会话、退出登录、强制下线 |
| `MerchantApplicationController` | 创建入驻、上传资质、提交入驻、审核入驻 |
| `MerchantProfileController` | 店铺详情、资料变更、授权状态、员工账号 |
| `ProductController` | 创建商品、编辑规格、提交审核、复制给商家、上下架 |
| `AssistantController` | 生成报价、生成二维码、读取报价快照 |
| `CustomerOrderController` | 客户扫码创建订单、提交资料、查看待办、确认待办 |
| `OrderController` | 订单列表、订单详情、备注、关闭、改价、补资料 |
| `ReviewController` | 锁单、审核通过、审核拒绝、分配内部资金来源、黑名单提示 |
| `ContractController` | 发起合同、补充合同、公证、重发待办、查看附件 |
| `FulfillmentController` | 创建发货、记录长租设备识别码、上传交付材料、签收、归还验收；短租后续再接设备绑定 |
| `FinanceController` | 账单、支付收款码、提现、退款、冲正、对账批次 |
| `ChannelController` | 渠道、推广码、统计、佣金明细、渠道提现 |
| `PostRentController` | 租后案件、催收同步、回款记录、案件状态 |
| `ComplaintController` | 投诉列表、投诉详情、回复投诉、投诉日志 |
| `ConfigController` | 链路、租赁模式、通道、审核、财务配置版本 |
| `CallbackController` | 支付、打款、合同、公证、物流、投诉等统一回调入口 |
| `AuditLogController` | 操作日志、敏感查看日志、回调日志、导出日志 |

## 6. Job 和 Worker

| 异步任务 | 触发来源 | 处理目标 |
|---|---|---|
| `CallbackConsumeWorker` | `CallbackController` | 消费回调事件，推进支付、合同、公证、物流、投诉状态 |
| `PaymentReconcileJob` | 定时/运营发起 | 拉取通道流水，生成对账异常 |
| `BillDueJob` | 定时 | 推进入账单到期、逾期和租后案件 |
| `AllocationJob` | 支付成功 | 生成分账、平台抽佣、钱包入账和渠道佣金 |
| `WithdrawPayoutWorker` | 提现审核通过 | 调用打款通道并处理回调 |
| `ContractRetryJob` | 合同发起失败/超时 | 重试或进入人工补偿 |
| `NotaryRetryJob` | 公证发起失败/超时 | 重试或进入人工补偿 |
| `ShipmentSyncJob` | 物流单创建/物流回调 | 同步签收状态和异常物流 |
| `PostRentSyncJob` | 租后案件 | 同步外部催收系统和回传结果 |
| `ComplaintPullJob` | 定时/回调 | 拉取支付宝投诉并补齐订单上下文 |
| `ConfigPublishJob` | 配置发布 | 灰度、校验、回滚准备和生效通知 |
| `ExportTaskWorker` | 导出任务 | 生成文件、脱敏、记录下载日志 |

## 7. 开发阶段

### 7.1 Stage 1：基础底座

- 建表迁移：商家、商品、订单、客户、设备、财务、渠道、配置、回调、日志基础表。
- 登录和权限：运营端、商家 PC、门店手机端、渠道端基础登录和数据范围。
- 附件中心：资质、合同、照片证据、风控资料统一文件元数据。
- 配置中心：链路配置、租赁模式、通道配置、审核策略、财务配置版本。
- 操作日志：人工操作、系统操作、敏感查看、回调处理统一记录。

验收闸口：

- 关键表唯一约束和索引已建立。
- 所有写接口有幂等键和操作日志。
- 敏感配置不入库明文，不进入代码仓库。

### 7.2 Stage 2：入驻、商品、办单、下单、审核

- 商家入驻闭环：注册、资料、收款账户、签章授权、运营审核。
- 商品闭环：平台商品、商家商品、全新/二手规格、长短租、办单助手同步。
- 办单助手闭环：三入口、报价、二维码、扫码归属、价格快照。
- 客户下单闭环：扫码、资料、授权、创建订单、待办。
- 审核闭环：商家订单自审，联营订单和平台订单运营审核，内部资金来源分配。

验收闸口：

- 三类订单都能从办单助手生成二维码到后台看到订单。
- 订单列表能按订单类型、商家、门店、渠道、资方、客户、日期搜索。
- 价格快照和后台订单展示金额一致。

### 7.3 Stage 3：合同、公证、支付、履约、财务

- 合同公证：发起、客户待办、回调、附件归档、补充合同。
- 支付账单：首期、租金、服务费、公证费、部分支付收款码。
- 履约设备：门店默认发货、设备码绑定、人机/人车合照、签收、起租。
- 财务分账：平台抽佣、分红比例、资方/门店/渠道/平台钱包。
- 提现和对账：提现审核、打款回调、退款、冲正、三边对账。

验收闸口：

- 支付成功能自动核销账单并生成分账。
- 联营订单按比例分到门店和内部资金来源台账，并分别扣平台抽佣。
- 发货签收后设备状态、订单状态、起租时间一致。

### 7.4 Stage 4：运营扩展和交付能力

- 渠道端：推广码、入驻统计、联营订单和平台订单统计、佣金提现。
- 租后：逾期案件、催收动作、外部系统同步、回款关联。
- 客诉：投诉同步、订单上下文、回复处理、日志追溯。
- 报表导出：订单、财务、渠道、租后、客诉报表和字段脱敏。
- 操作手册：按模板生成商家入驻、门店办单、审核、合同公证、发货签收培训文档。

验收闸口：

- 渠道佣金能从订单归属一路追到钱包流水。
- 租后和客诉都能回到原订单详情。
- 所有导出有字段权限和下载日志。

## 8. 测试要求

| 测试类型 | 覆盖内容 |
|---|---|
| 单元测试 | 金额计算、分账、状态机、权限判断、幂等 |
| 接口测试 | OpenAPI 中所有写接口的成功、失败、重复提交 |
| 集成测试 | 入驻、商品、办单、下单、审核、合同、支付、发货、财务闭环 |
| 回调测试 | 验签失败、重复回调、乱序回调、超时重试 |
| 财务测试 | 金额平衡、分账平衡、退款冲正、提现冻结、对账异常 |
| 权限测试 | 运营、商家老板、店员、渠道、资方只读预留的数据范围 |
| 安全测试 | 敏感文件访问、身份证号脱敏、收款账户脱敏、导出审计 |

## 9. 最小可上线边界

第一版不追求所有扩展功能一次做完，但必须具备以下可用闭环：

1. 商家入驻、审核、登录和员工账号。
2. 商品新增、审核、复制给商家、同步办单助手。
3. 商家订单、联营订单、平台订单三入口办单和扫码下单。
4. 客户资料、授权、合同、支付、后台审核。
5. 门店默认发货、设备码绑定、交付照片、签收起租。
6. 账单、支付、分账、钱包、提现、退款、冲正和对账基础能力。
7. 渠道推广码、渠道归属、佣金明细。
8. 操作日志、回调日志、配置版本、敏感信息访问日志。

只要其中任一链路只能展示不能真实写入和联动，就不能视为完成。
