# 实名认证

> **来源**: P1-5
> **核心**: 必须客户本人,不可门店代替

## 1. 业务背景

客户首次办单或资料过期时,需做实名认证:
- 身份证 OCR 识别
- 人脸识别活体检测

## 2. 流程

```
1. 客户上传身份证正反面照片
   - 实拍 + 防翻拍检测
   - OCR 自动识别姓名 / 身份证号 / 有效期 / 地址
   - 校验身份证号格式 + 有效期
2. 人脸识别(活体检测)
   - 调用支付宝/微信/三方刷脸 SDK
   - 客户做指定动作(眨眼、张嘴、摇头等)
   - 服务方比对 OCR 出的身份证照与本人是否一致
3. 通过 → 客户实名状态变 verified
   通过失败 → 重试(最多 3 次)
   3 次失败 → 转人工(客服处理)
```

## 3. 必须客户本人

- 身份证拍照可门店帮举手机但客户拿身份证
- 人脸识别**必须客户本人**,不可代替

## 4. 中控台对接

- 身份证 OCR / 人脸识别都通过中控台对接(P0-4)
- 中控台异步计费
- 本地缓存 + 降级模式

## 5. 失败处理

| 失败类型 | 处理 |
|---|---|
| 身份证 OCR 失败 | 可重试 3 次,或手工填(客服核对) |
| 人脸识别不匹配 | 可重试 3 次,3 次后进"待客服介入" |
| 身份证过期 / 无效 | 提示客户更新身份证 |
| 黑名单(已被加黑) | "您当前无法办理此业务,请联系平台客服" |

## 6. 实名状态

- 客户实名一次,后续办单复用(不需每次重做)
- 实名信息有效期:身份证有效期内
- 身份证更换 / 过期:需重新实名

## 7. 隐私

- 实名信息默认脱敏展示
- 完整查看需权限 + 日志

## 8. 数据模型

- `customer.kyc_status` (verified / unverified / expired)
- `customer.kyc_verified_at`
- `customer.id_card_number`(加密存)
- `customer_kyc_log`(实名历史记录)
