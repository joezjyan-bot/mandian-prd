# 测试数据脱敏规范

> **来源**: P2-3
> **强制级别**: V1 必做

## 1. 业务背景

测试环境与开发环境**严禁使用真实用户数据**。所有测试数据必须从受控池随机生成,保证不泄露真实公民/客户信息。

## 2. 测试数据池

### 2.1 姓名
- 池来源:从虚构姓名表随机(如"张测试"、"李样本")
- 数量:1000+ 个
- 命名规则:可加前缀 _test_ 便于识别

### 2.2 身份证号
- 校验位计算正确(过校验)
- 但身份证号**整体是虚构的**(避开真实公民号段)
- 出生日期固定在特定范围(便于测试场景控制)

### 2.3 手机号
- 用 **180/170 段虚号**(部分号段已被三网定为测试号段)
- 1 个号段 + 8 位随机
- 不发真实短信(测试环境屏蔽短信通道)

### 2.4 银行卡号
- 用 **银联测试卡号**(各银行公布的官方测试卡)
- 不接真实银行通道
- 测试代扣 / 提现走通道方提供的 sandbox

### 2.5 地址
- 用虚构地址池(如"北京市朝阳区测试路 1 号")
- 不用真实小区 / 公司名

## 3. PRD 文档中的占位

所有 PRD 文档中:
- 订单号:用 `DZ20260525000XXX` 这种格式
- 客户姓名:用 "张三" / "李四" 等通用名
- 金额:用占位符 ¥XXX
- 时间:用占位符 `2026-MM-DD`

**不得使用**:
- 真实订单号
- 真实客户姓名 / 身份证 / 手机
- 真实金额(具体到分)

## 4. 测试环境部署

部署到测试环境时:
- 数据库自动加载测试数据池
- 不能从生产环境同步真实数据(即便脱敏后)
- 测试环境与生产环境数据库完全隔离

### 4.1 测试用例数据
- 测试用例中使用脱敏数据
- CSV / JSON 测试数据放代码仓库 `tests/fixtures/`
- 不上传真实数据到代码仓库

### 4.2 生产环境备份
- 备份文件加密
- 仅授权人可访问
- 不用于测试

## 5. 数据库迁移到测试

不允许:
- 把生产数据库 dump 后还原到测试环境

允许:
- 用脱敏池生成新的测试数据
- 用代码自动初始化测试场景

## 6. 测试通道

| 通道 | 测试环境配置 |
|---|---|
| 短信 | 屏蔽,所有短信改为日志记录 |
| 邮件 | 发到指定测试邮箱 |
| 支付 | sandbox(通联/支付宝/微信都有测试环境) |
| 实名 | sandbox |
| 风控 | sandbox |
| e 签宝 / 法大大 | sandbox |
| 公证 | sandbox |

## 7. 测试账号

各端预置测试账号:
- 客服账号(各角色)
- 商家账号
- 客户账号(实名 / 未实名各几个)
- 资方账号
- 渠道账号

## 8. 测试场景数据

- 正常订单(待付款 / 待审核 / 在租 / 已完成)各几个
- 异常订单(逾期 / 退款 / 关闭 / 争议)各几个
- 不同金额范围
- 不同租期长度
- 不同订单类型(门店 / 分红 / 平台)

## 9. 数据清理

- 测试环境每周清理一次
- 重置为基础测试场景
- 保留长期测试数据(标记为 _persist_)

## 10. 操作权限

- 仅测试 QA + 开发 + 运维可操作测试环境
- 不允许测试环境数据外泄
- 不允许测试环境向生产环境推数据
