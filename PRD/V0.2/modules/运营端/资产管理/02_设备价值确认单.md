# 设备价值确认单

> **所属端**:C 端(订单确认页勾选)+ 运营端/法务(合同附件、归档、证据包)
> **版本**:v1.0 / 2026-05-29
> **性质**:合规证据链核心。用于证明「客户独立、明示地确认了所租设备的型号、成色与价值」,支撑真实租赁定性、留购/违约赔偿的价值基准。
> **来源**:合规审查 P0 — 系统此前无设备价值确认能力。

> **落地方式(已定稿,三绑定)**:
> 1. **合同附件**:设备型号/配置/成色/SN-IMEI/确认价作为合同的**附件/附表**,与合同一起签署归档。**不写入主合同正文**(避免合同模板过重、每次改设备信息要动主合同)。
> 2. **订单确认页勾选**:在现有 `C端/02_订单确认页.md` 增加一个**独立勾选项**,客户在签约前确认设备价值。
> 3. **页面快照存证**:确认动作存页面快照(时间/IP/设备/hash),随合同一起进法诉证据包。

---

## 0. 设备价值的唯一来源(关键口径)

**设备价值 = 设备价格 = 设备成本,同一个数,不引入新数字。** 客户在订单确认页确认的,就是该订单已在使用的设备价格:

| 下单来源 | 设备价值取值 |
|---|---|
| 办单助手扫码下单 | 取办单助手填写的设备价格(`price`,商品配置带出或办单时填;见 门店手机端/办单助手 02 §1.1) |
| 非办单助手(客服后台下单) | 取客服后台填写的设备成本/设备价格 |

这样设备价值确认、计费(留购/账单)、违约赔偿基准用的是**同一个 `device_value_cent`**,不会出现「展示价 ≠ 计费价」的对账或口径冲突。

**展示方式 = A 方案(只读 + 勾选)**:客户侧只读展示该设备价值,客户**不可修改数字**,仅勾选「我已确认设备型号、成色及价值无误」。第三方估值价作为字段**预留**(后台可填、未来接估值接口),V1 客户侧只读展示设备价格 + 勾选。

## 1. 客户侧:订单确认页的设备价值确认(嵌入 C端/02)

### 1.1 位置

在 `C端/02_订单确认页.md` 的商品信息区(§2.2/§2.3)之下、协议勾选(§2.7)之中,增加「设备价值确认」区块与一个**独立必勾项**。

### 1.2 展示内容(只读)

```text
┌──────────────────────────────────────────┐
│ 设备信息确认                                │
│  型号:iPhone 15 Pro 256G                  │
│  成色:95 新                               │
│  设备价值:¥6,500                          │
│  (设备价值来源于本订单设备价格,仅供确认)   │
├──────────────────────────────────────────┤
│ ☐ 我已确认上述设备型号、成色及价值无误      │
└──────────────────────────────────────────┘
```

- 设备价值数字只读,来源见 §0,客户不可改。
- 该勾选项为**独立必勾项**,与租赁服务协议等其它协议分开,单独记录确认动作。
- 此步**尚无设备识别码(IMEI/SN/VIN)**——这是正确的:客户确认的是「这个型号/成色值多少钱」(SKU 层面),不是「具体哪一台」(实物层面)。识别码在交付时产生并回填(见 资产管理 01 §1.2)。

### 1.3 客户侧红线

- 不出现资方、资金来源、合作模式、风控、服务费拆分、商家结算价、平台利润等字眼。
- 「设备价值」用中性表述,不与「融资额」「贷款本金」等金融语义关联。

## 2. 三绑定的具体实现

### 2.1 合同附件

- 设备价值确认信息(型号/配置/成色/SN-IMEI/确认价)作为合同的附件/附表,在合同签署时一并签署、一并归档。
- 不进主合同正文;主合同正文引用附件即可。
- 交付前 SN-IMEI 为空,交付回填后,附件归档版本应包含最终识别码(或以补充归档方式补入)。

### 2.2 订单确认页勾选(见 §1)

- 客户勾选动作单独成一条确认记录,带时间/IP/设备/页面版本。

### 2.3 页面快照存证

- 订单确认页(含设备价值确认块与勾选状态)生成页面快照,纳入 `order_snapshot`,随合同进法诉证据包。

## 3. 字段

```text
device_value_confirmation
- confirmation_id     bigint PK
- order_id            bigint FK
- asset_id            bigint FK         -- 关联 device_asset(交付后绑定)
- brand_model         varchar           -- 型号
- spec                varchar           -- 配置
- condition_label     varchar           -- 成色
- device_identifier   varchar           -- SN/IMEI/VIN;确认时可空,交付回填
- store_price_cent    bigint            -- 设备价值 = 设备价格(办单助手填/客服后台填)
- platform_ref_price_cent bigint        -- 平台参考价(可选,预留)
- third_party_valuation_cent bigint     -- 第三方估值价(预留,V1 可空)
- confirmed_value_cent bigint           -- 客户确认价 = store_price_cent(A方案只读,等于设备价格)
- customer_confirmed   bool             -- 客户是否已勾选确认
- confirmed_at         datetime
- confirm_ip           varchar
- confirm_device_info  varchar
- page_version         varchar          -- 订单确认页版本
- snapshot_id          varchar          -- 页面快照关联
- snapshot_hash        varchar          -- 不可篡改标识
- consistency_status   enum             -- consistent/reconfirm_required(交付一致性校验结果)
- created_at / updated_at
```

## 4. 交付一致性校验(与资产中心联动)

1. 交付填 IMEI/SN/VIN 时,系统比对实物型号/成色与本确认记录。
2. 一致 → `consistency_status = consistent`,识别码回填,证据链闭合。
3. 不一致(如成色不符)→ `consistency_status = reconfirm_required`,触发客户**重新确认**,旧确认记录不得直接用于交付放行。
4. 重新确认走同一勾选流程,生成新确认记录与快照。

## 5. 权限与日志

| 动作 | 谁可操作 | 日志 |
|---|---|---|
| 生成设备价值确认(随订单) | 系统 | 创建日志 |
| 客户勾选确认 | 客户 | 确认记录 + 快照 + IP |
| 修改设备价值(=改设备价格) | 走办单助手/客服改价权限,改值纳入审计 | 改前后值 + 操作人 |
| 触发重新确认 | 系统/交付环节 | 一致性校验日志 |
| 填第三方估值 | 运营(预留) | 操作日志 |

设备价值确认记录与快照不可篡改,删除/修改受控并留痕。

## 7. 验收要点

1. 订单确认页展示设备价值(只读,来源 = 该订单设备价格),客户勾选后方可进入下一步。
2. 勾选时无设备识别码可正常完成(SKU 层面确认)。
3. 确认记录含时间/IP/设备/页面版本/快照/hash。
4. 设备价值确认作为合同附件归档,不在主合同正文。
5. 交付成色/型号不一致触发重新确认,旧记录不放行。
6. 客户确认价 = 设备价格,三处(确认/计费/违约基准)同一数值。
7. C 端无资方/风控/利润等字眼。
8. 设备价值确认页快照可随合同进法诉证据包。
