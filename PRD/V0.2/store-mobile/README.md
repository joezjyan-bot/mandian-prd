# 门店移动端页面审计

> 门店移动端是商家主体的手机/H5 工作台，和 PC 商家中心属于同一业务主体、同一店铺数据域。PC 端偏商品、经营配置和完整管理；移动端偏办单、待办、订单查看、钱包和拓店。

## 审计范围

| 模块 | 文档 | 覆盖页面 |
|---|---|---|
| 登录入驻 | `01-登录入驻/README.md` | 我的页入口、门店登录、门店入驻、目标入驻资料流程 |
| 工作台设置 | `02-工作台设置/README.md` | 工作台、设置、店铺信息修改、密码修改、员工账号缺口 |

## 角色边界

```text
同一商家主体
├─ 门店老板/管理员
│  ├─ PC 商家中心：商品、订单、资金、营销、权限、店铺资料
│  └─ 移动门店端：办单、订单、钱包、拓店、资料维护
└─ 门店员工
   ├─ 允许：办单助手、扫码办单、待办任务、本人经办订单
   └─ 禁止：钱包资金、提现充值、商品/营销配置、权限管理、完整客户敏感信息
```

## 关键补充规则

1. 同一手机/设备登录后应支持一段时间免登录，建议采用短期 access token + 长期 refresh token，默认 30 天有效；退出登录、改密码、账号禁用、权限变更、管理员强制下线时立即失效。
2. 员工账号由门店老板自行创建、禁用、重置密码和分配权限，不应由平台运营日常代管。
3. 所有涉及资金、审核、付费风控、生成客户下单二维码、保存店铺资料的动作都必须有二次确认、权限校验和审计日志。
4. 移动端文档只保留字段和交互，不保留真实手机号、身份证号、订单号、店铺名、账号和邀请码。
