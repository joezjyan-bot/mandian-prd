# 长租设备档案

> **V0.2 Peer Review 修订(2026-05-27)v1.0**:
> - 本文落实同行小程序能力吸收任务 5。
> - 长租只做设备档案,不做库存占用和短租状态机。
> - 设备码来源于发货/交付节点填写的 IMEI/SN/VIN。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 长租设备档案 |
| 所属端 | 运营端 / 商家端只读或按权限查看 |
| 入口路径 | 库存设备管理 > 长租设备档案 |
| 使用角色 | 运营客服、库存管理、商家老板、门店店长、售后 |
| 核心目标 | 用轻量表格沉淀长租订单交付后的设备信息、订单关系、履约状态和赎回/处置记录 |

---

## 2. 核心口径

1. 长租设备档案不是短租库存,不参与可租库存、预约保留、锁定、出租中等短租状态。
2. 长租下单不需要提前选择库存设备。
3. 长租在发货或门店交付时填写设备识别码,如 IMEI / SN / VIN。
4. 客户确认收货后,系统自动生成长租设备档案。
5. 档案用于资产追踪、售后、赎回登记、留购完成和历史查询。
6. 档案字段必须一页展示,客服日常操作不进入复杂多 tab。

## 4. 自动同步规则

```text
订单发货/交付
   ↓
填写设备码 IMEI/SN/VIN
   ↓
客户确认收货
   ↓
系统生成长租设备档案
   ↓
设备状态 = 履约中
```

后续状态:

| 订单事件 | 档案状态 |
|---|---|
| 客户确认收货 | 履约中 |
| 客户正常归还并验收完成 | 已归还 |
| 客户留购 / 提前结清完成 | 已留购 |
| 平台或门店赎回设备 | 已赎回 |
| 设备无法继续处置 | 已报废 |

---

## 5. 赎回登记

触发场景:客户私自变卖、门店/平台追回、平台回收后需要登记资产处置。

操作流程:

```text
长租设备档案 → 找到设备 → [赎回登记]
   ↓
填写:
  - 赎回价格
  - 处置方式
  - 备注
  - 附件/凭证
   ↓
提交后:
  - 设备状态改为已赎回
  - 写设备档案日志
  - 关联订单详情展示赎回记录
```

## 8. 与短租库存的边界

| 项 | 长租设备档案 | 短租车辆库存 |
|---|---|---|
| 是否下单前占库存 | 否 | 是 |
| 是否可反复出租 | 否,跟随订单履约 | 是,归还验收后可复租 |
| 是否有预约保留 | 否 | 是 |
| 是否需要时段调度 | 否 | 是 |
| 是否在发货时填设备码 | 是 | 已在库存中绑定 |
| 状态机 | 履约/归还/留购/赎回/报废 | 短租 9 状态 + 订单状态 |
