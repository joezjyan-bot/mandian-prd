# 店铺管理 · 03 商家入驻业务线勾选

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 商家入驻时勾选要做的业务线(安心用商家 / 体验租商家),可双选。
> 业务线勾选直接决定:商家可见的菜单 / C 端门店详情是否显示长租/短租 Tab / 商品维护范围 / 库存模型适用。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家入驻业务线勾选 |
| 所属端 | C 端入驻(商家自助)+ 运营端审核 |
| 入口路径 | 商家入驻表单(第 1 步) + 运营端 > 店铺管理 > 店铺审核 |
| 使用角色 | 商家(入驻人)、平台审核客服、店铺审核主管 |
| 核心目标 | 商家明确声明经营范围,平台审核后开通对应业务线菜单和数据隔离 |

---

## 2. 业务线选项

| 业务线 | 编码 | 含义 |
|---|---|---|
| ☐ **安心用商家** | `assurance_rent` | 长租业务(以租代购),订单走商家订单/联营订单/平台订单三类长租链路 |
| ☐ **体验租商家** | `experience_rent` | 短租业务,客户到店取还车,需要车辆库存和控车能力 |

**可双选**:一个商家可以同时勾选两个业务线,例如一家电动车店既做长租(以租代购)也做短租(按小时租)。

**必选 1 项**:至少勾 1 个,不能两个都不勾。

---

## 3. 入驻表单展示

### 3.1 字段位置

入驻表单第 1 步(选择经营范围),先选业务线再填资料。

```text
┌────────────────────────────────────────────────────────┐
│  欢迎入驻满点平台                                         │
├────────────────────────────────────────────────────────┤
│  请选择您的经营业务范围(可多选)*                          │
│                                                          │
│  ☐  安心用商家  ⓘ                                        │
│  ☐  体验租商家  ⓘ                                        │
│                                                          │
│  [下一步:填写商家资料]                                    │
└────────────────────────────────────────────────────────┘
```

### 3.2 ⓘ 解释弹窗(鼠标悬停 / 点击触发)

**安心用商家 ⓘ**:

> 客户长租为主,租期 6-24 个月。平台代收租金分期账单,签订三方租赁服务合同(您门店、客户、平台)。设备一般快递发货或门店当面交付,客户长期使用,到期可选留购。
>
> 适合:手机、数码、电脑、家电、智能穿戴等长期使用的设备。
>
> 您将获得:
> - 长租订单管理(商家订单 / 平台订单)
> - 长租商品管理(规格 / 套餐 / 期数)
> - 客户分期账单的钱包结算
> - 平台代收代付能力

**体验租商家 ⓘ**:

> 客户短租为主,租期 1 小时 - 1 个月。客户到店取车 / 还车,需要车辆库存管理和控车能力(开机 / 关机 / 定位)。保证金 / 免押由平台风控判断。
>
> 适合:电动车、共享设备、相机、户外装备等可短时使用的设备。
>
> 您将获得:
> - 体验租订单管理(待接单 / 待取车 / 在租 / 还车 / 保证金)
> - 车辆库存管理(VIN / SN / 设备 9 状态)
> - 中控控车接入
> - 保证金扣款与退款

## 4. 数据模型

```sql
merchant
- id
- merchant_name
- ...
- business_lines        -- JSON 数组: ["assurance_rent"] / ["experience_rent"] / ["assurance_rent", "experience_rent"]
- business_lines_status -- JSON: {"assurance_rent": "approved", "experience_rent": "pending_review"}
- ...
```

**business_lines_status 取值**:

| 值 | 含义 |
|---|---|
| `not_applied` | 未申请 |
| `pending_review` | 待审核 |
| `approved` | 已开通 |
| `rejected` | 审核驳回 |
| `suspended` | 已冻结(临时停用) |
| `revoked` | 已撤销(永久关闭) |

每条业务线独立审核状态,不互相影响。

---

## 5. 入驻后追加业务线

商家入驻后可在「商家 PC 端 > 商家资料 > 业务范围」追加业务线:

```text
当前已开通:
  ✅ 安心用商家(已开通 2026-03-15)
  
申请新增:
  ☐ 体验租商家  [申请开通]
```

追加业务线:
1. 商家提交申请 → 业务线状态 `pending_review`
2. 平台审核(可能要求补资质,例如体验租需要车辆库存证明)
3. 审核通过 → 业务线状态 `approved`,商家立即获得对应菜单

---

## 6. 审核流程

```text
商家入驻提交资料(选定业务线)
  ↓
运营端店铺审核客服分配单
  ↓
按业务线分别审核
  ├─ 安心用资质:营业执照 / 法人 / 签章授权
  └─ 体验租资质:营业执照 / 法人 / 签章授权 + 车辆所有权证明 + 经营场地证明
  ↓
全部业务线审核结果
  ├─ 全部通过 → 商家整体状态 approved,开通所有勾选业务线
  ├─ 部分通过 → 商家状态 partial_approved,只开通通过的业务线,其余在商家端显示"待补资料"
  └─ 全部驳回 → 商家状态 rejected,通知商家
```

## 7. 业务线开通后的影响

| 影响范围 | 安心用商家 | 体验租商家 |
|---|---|---|
| 商家 PC 端菜单 | 显示长租订单 / 长租商品 / 钱包 | 显示体验租订单 / 车辆库存 / 控车 / 保证金 |
| 门店手机端菜单 | 显示三类长租办单助手 | 显示体验租待办 / 取车核销 / 还车验收 |
| 商品维护 | 可创建长租商品 | 可创建体验租车型 |
| C 端门店详情 | 长租 Tab 可见 | 短租 Tab 可见 |
| 资金结算 | 长租钱包 | 短租钱包(保证金账户分离) |
| 合同模板 | 三方租赁合同 | 体验租服务协议 |

**双业务线商家**:同时显示所有菜单,两条业务线的钱包独立但**主体合并**。

## 8. 业务线冻结 / 撤销

| 场景 | 处理 |
|---|---|
| 临时合规问题 | 业务线 → `suspended`,商家暂时无法创建新订单,但可处理存量订单 |
| 严重违规 | 业务线 → `revoked`,商家完全无法操作该业务线 |
| 商家主动退出某条业务线 | 商家申请 → 平台审核 → `revoked`(需先处理完所有存量订单) |
| 单条业务线被处理 | 另一条业务线不受影响 |

## 9. C 端门店详情 Tab 显隐规则

依据商家业务线 + 门店开通状态共同决定(详见 `04_门店开通与城市管理.md`):

| 商家业务线 | 门店开通 | C 端门店详情 |
|---|---|---|
| 仅安心用 | 长租开通 | 只显示长租板块,不显示 Tab |
| 仅体验租 | 短租开通 | 只显示短租板块,不显示 Tab |
| 双业务线 | 双开通 | 显示长租/短租双 Tab |
| 双业务线 | 仅长租开通 | 只显示长租板块 |
| 双业务线 | 仅短租开通 | 只显示短租板块 |

**核心原则**:Tab 显隐 = (商家业务线已开通)AND(门店当前业务线已开通)。
