按你确认:**"还是保持已完成/已取消即可"**

| 子标签 | 触发场景 | 颜色 |
|---|---|---|
| 已完成 | COMPLETED(归还)/ RETAINED(留购/提前结清) | 中性灰 |

## 2. C 端订单详情页(v1.2 字眼统一)

### 2.1 "审核中"详情页

展示:商品基础信息 / 当前订单总额 / 联系客服 / 取消订单

不展示:商家主体 / 资金来源 / 资方 / 履约门店 / 做单客服 / 合作模式 / 预估时间

### 2.2 "待付款"详情页

展示:商品 + 订单总额总览 + **首期应付**明细(保证金/首期应付/设备管理费等)+ [立即支付] + 取消订单

### 2.3 "待签约"详情页

展示:待办列表(首期支付 ✓ / 电子合同签署 ⏳ / 代扣签约 ⏳)+ 商品 + 账单 + 取消订单(需客服同意)

#### 2.3.1 倒计时基础展示(v1.4)

页面顶部展示倒计时提示。倒计时默认 60 分钟,租户可配置:

```text
将在 23:45 后自动取消,请尽快完成签约/授权/支付。
[立即签约] [延长 30 分钟]
```

倒计时归零后:
- 未支付任何款项:订单自动取消。
- 已支付首期/保证金:订单进入自动取消 + 原路退款流程,退款进度在订单详情展示。
- 正在等待合同/代扣/支付回调:先进入回调容错窗口,不得直接关单。

#### 2.3.2 倒计时暂停机制

客户跳转到三方页面时,平台主倒计时暂停,改用对应三方动作的独立 SLA。客户从三方页面回到订单详情后,按原剩余时间恢复主倒计时。

| 场景 | 倒计时行为 | 默认 SLA |
|---|---|---|
| 客户在订单详情页停留 | 主倒计时正常减少 | 待签约自动取消时长 60 分钟 |
| 客户点击 [签约] 跳到 e 签宝 | 主倒计时暂停,进入合同签署 SLA | 30 分钟 |
| 客户点击 [代扣签约] 跳到银行/支付宝 | 主倒计时暂停,进入代扣签约 SLA | 15 分钟 |
| 客户点击 [立即支付] 跳到支付通道 | 主倒计时暂停,进入支付 SLA | 10 分钟 |
| 客户从三方页面返回订单详情 | 主倒计时恢复,从暂停前剩余时间继续 | - |

三方动作 SLA 超时后:
- 若三方已明确失败或客户取消,订单回到待签约详情页,主倒计时恢复。
- 若三方状态未知,订单详情展示"处理中,请稍后刷新",后台等待回调容错窗口。
- 系统不得因为客户正在三方页面阅读合同、办理代扣或支付而直接触发自动取消。

#### 2.3.3 客户主动延长机制

客户可在倒计时未归零前点击 [延长 30 分钟]。

| 规则 | 说明 |
|---|---|
| 单次延长时长 | 默认 30 分钟,租户可配置 |
| 最大延长次数 | 默认 2 次,即总共最多 +60 分钟 |
| 延长日志 | 每次延长记录客户、端、时间、延长前剩余时间、延长后到期时间 |
| 按钮隐藏 | 延长次数用完、倒计时已归零、订单已进入三方 SLA 时不展示 |
| 延长后归零 | 仍按自动取消规则处理,不可再延 |

延长成功后提示:

```text
已为您延长 30 分钟,请尽快完成签约/授权/支付。
```

#### 2.3.4 接近超时提醒

| 触发点 | C 端展示 | 触达 |
|---|---|---|
| 剩余 10 分钟 | 弹窗:"订单将在 10 分钟后自动取消,请尽快完成签约/支付" | 小程序订阅消息或短信(若客户授权) |
| 剩余 3 分钟 | 顶部红色横幅:"订单即将取消,请立即完成当前操作" | 可再次触达,避免频繁打扰 |

提醒只做触达和展示,不改变倒计时状态。多端同时打开同一订单时,提醒状态以服务端为准,避免重复弹窗。

#### 2.3.5 回调容错与边界处理

| 场景 | 处理 |
|---|---|
| 客户在 e 签宝刚签完合同但平台还未收到回调 | 倒计时归零后进入 5 分钟回调容错窗口;5 分钟内回调成功则正常进入下一步 |
| 客户在支付通道刚支付成功但平台还未收到回调 | 同上,以支付成功回调和主动查单结果为准 |
| 容错窗口 5 分钟内仍无回调 | 进入异常队列,客服处理;订单不直接关闭为退款态 |
| 客户在 e 签宝点击拒签 | 订单进入 CLOSED,可重新下单 |
| 客户多端登录(支付宝 + 微信同时打开同一订单) | 两端共享服务端倒计时,任一端操作后另一端通过轮询/推送同步 |
| 客户返回后连续刷新 | 只刷新状态,不得重复发起合同、代扣或支付 |

#### 2.3.6 租户配置项

| 配置 | 默认值 | 可配置 |
|---|---|---|
| 待签约自动取消时长 | 60 分钟 | 是 |
| e 签宝独立 SLA | 30 分钟 | 是 |
| 代扣独立 SLA | 15 分钟 | 是 |
| 支付独立 SLA | 10 分钟 | 是 |
| 允许客户延长次数 | 2 次 | 是 |
| 单次延长时长 | 30 分钟 | 是 |
| 第一次提醒时间 | 剩余 10 分钟 | 是 |
| 第二次提醒时间 | 剩余 3 分钟 | 是 |
| 回调容错窗口 | 5 分钟 | 是 |

#### 2.3.7 数据模型:`order_countdown`

```sql
order_countdown
- countdown_id           bigint PK
- order_id               bigint FK
- countdown_type         enum -- pending_sign / pending_pay / pending_auto_debit
- started_at             datetime
- expires_at             datetime
- paused_at              datetime
- pause_reason           enum -- contract_signing / payment_processing / auto_debit_signing
- resumed_at             datetime
- extended_times         int
- extended_total_minutes int
- final_status           enum -- completed / expired / extended / abandoned
- created_at             datetime
- updated_at             datetime
```

实现要求:
1. `expires_at` 以服务端计算为准,C 端只展示剩余时间。
2. 暂停期间不得继续扣减主倒计时,但必须记录三方动作 SLA。
3. `extended_times` 和 `extended_total_minutes` 用于控制客户延长次数和风控分析。
4. `final_status = abandoned` 表示客户进入三方页面后未完成且未返回,需结合 SLA 和回调容错处理。
5. 所有暂停、恢复、延长、超时、容错入队动作都写入订单操作日志。

**合同 PDF 内容按资金来源切换**(详见话术规范文档):
- 平台自营资金 → 《租赁服务协议》
- 蓝海银行 → 《蓝海银行赊销合同》

### 2.6 "已完成"详情页

展示:商品 + 设备信息 + 完整支付记录(首期应付 / 各期应付 / 逾期费用 / 留购或提前结清支付)

### 2.7 通用状态刷新

C 端订单详情页必须支持下拉刷新订单状态:

| 场景 | 刷新后行为 |
|---|---|
| 客户完成实名认证后返回 | 重新读取实名状态,仍停留在原商品/订单上下文 |
| 客户完成合同签署后返回 | 重新读取合同回调状态,更新待办列表 |
| 客户完成支付后返回 | 重新读取支付状态和账单状态 |
| 客户完成代扣签约后返回 | 重新读取代扣签约状态 |
| 第三方回调延迟 | 展示“处理中,请稍后下拉刷新”,不重复创建订单 |

刷新接口只更新展示状态,不得重复提交订单、重复发起合同、重复调起支付。

### 3.2 关键按钮按资方话术模板切换 ⭐ v1.2 核心

// 系统读取订单的 terminology_template_id 决定按钮文案

### 3.3 任意期可支付的逻辑(严格顺序 A,保留 v1.1)

核心规则:任何一期都可以点 [立即支付] 按钮查看金额,但实际支付时系统强制结清前期欠款。

| 状态 | 实际行为 |
|---|---|
| 已付清 | 显示 ✅,无支付按钮 |
| 当前期未付 | 显示 [立即支付] + 当期金额 |
| 第 N 期未付,客户点第 N+M 期 | 系统弹窗:"请先支付第 N 期(¥XXX)";不允许跳期 |
| 已逾期 | 显示红色逾期标签 + 累计天数 |

支付前的校验逻辑:

```
客户点 [立即支付] 第 N 期
   ↓
系统检查:第 1 到 第 N-1 期是否都已付清?
   ├─ 是 → 直接调起支付通道
   └─ 否 → 弹窗:"您的第 X 期尚未支付,需先支付才能支付第 N 期"
            [取消]  [立即支付第 X 期]
```

### 3.4 提前结清的流程(按资方话术差异化弹窗)

```
客户点 [申请留购] 或 [提前结清](按话术模板)
   ↓
检查前置条件:
  ✓ 当前期已付清?
  ✓ 之前所有期数已付清?(严格顺序 A)
  ✓ 无未结逾期费用?(留购前必清)
   ↓
  └─ 不满足 → 提示具体未结清项
              示例:"请先支付第 3 期(¥568.75)+ 逾期费用 ¥30.00"
  └─ 满足 → 弹出提前结清明细
   ↓
─── 标准租赁话术(平台自营资金)弹窗 ───
留购确认
   已付期数:    第 1-N 期
   留购价:      ¥X,XXX
   保证金抵扣:  -¥1,490
   ─────────────
   应付金额:    ¥X,XXX
   
   留购成功后,设备所有权将归您所有
   [取消]      [确认支付]

─── 蓝海银行赊销话术弹窗 ───
提前结清确认
   已付期数:    第 1-N 期
   剩余应付:    ¥X,XXX
   保证金抵扣:  -¥1,490
   ─────────────
   应付金额:    ¥X,XXX
   
   结清后,本订单已全部还清,不再产生费用
   [取消]      [确认支付]

   ↓
客户确认 → 调起支付通道 → 支付完成
   ↓
订单状态:履约中 → 已完成
设备归属处理:
  - 标准租赁:设备所有权转客户(留购完成)
  - 蓝海银行赊销:设备本就归客户,此为完成该资金方案项下全部剩余款项支付
```

### 3.5 严格顺序支付(保留 v1.1)

| 场景 | 处理 |
|---|---|
| 第 3 期未付,客户点第 5 期 [立即支付] | 不允许,系统弹窗提示先付第 3 期 |
| 客户第 3 期 + 第 4 期一起付 | V1 不支持,客户分别点 |
| 客户提前付了第 3 期(还没到期日)| 允许,系统记录;下一期顺序变为第 4 期 |
| 第 3 期支付失败 | 客户必须重新支付,第 4 期等等 |

### 3.6 剩余应付的计算规则(原"留购价")

按办单助手预计算,字段在合同模板里展示为:
- 租赁话术:**留购价**
- 赊销话术:**剩余应付本金**
- C 端瀑布流统一展示为:**剩余应付**

| 期数 | 本期应付 | 剩余应付 |
|---|---|---|
| 第 1 期 | ¥10.00 | ¥6,040.00 |
| 第 2 期 | ¥568.75 | ¥5,471.25 |
| 第 N 期 | ¥568.75 | 递减 |

### 3.7 逾期费用账单展示规则 ⭐ v1.2 字眼调整

逾期费用账单与主账单完全独立展示。详细规则见 `11_逾期费用账单与规则配置.md`(11 文档将同步调整字眼为"逾期费用账单")。

关键展示原则:
- 独立账单区块(在主账单瀑布流下方)
- 每笔逾期费用独立一条记录:来源期数、累计天数、原始金额、减免、实付
- 已减免的展示"已减免 ¥XX(运营调整)"标识
- 全额减免的展示"已免除"+ 灰色
- 客户可单独支付逾期费用(不强制同时付当期)
- 客户提前结清或归还前必须清完所有逾期费用

**主账单 vs 逾期费用账单**:

| 项目 | 主账单 | 逾期费用账单 |
|---|---|---|
| 强制顺序 | 严格顺序 A | 独立,无顺序约束 |
| 是否影响订单状态 | 是 | 否(不阻塞订单)|
| 提前结清前必清 | 是 | 是 |
| 后台可修改 | 否(已锁定)| 是(可修改原始金额 + 减免)|

## 4. 客户主动撤单

### 4.1 撤单按钮可点状态

| 订单状态 | [取消订单] 按钮 |
|---|---|
| 审核中 | ✅ 直接可取消 |
| 待付款(未付款)| ✅ 直接可取消 |
| 待付款(已付款)| ⚠️ 需要客服同意 |
| 待签约(已付款)| ⚠️ 需要客服同意 |
| 已发货 | ❌ 不可单方取消(走售后)|
| 待收货 | ❌ 不可单方取消(走售后)|
| 履约中 | ❌ 不可取消(走归还/提前结清)|

### 4.2 直接可取消的撤单流程

```
[取消订单] → 客户填取消原因(可选)→ 确认
   ↓
订单 → 已完成(已取消)
保证金/付款若有,按 §4.3 退款
```

### 4.3 退款规则(对接 07 文档撤单退款资金链路)

| 场景 | 退款 |
|---|---|
| 审核中撤单(未付款)| 无需退款 |
| 待付款撤单(未付款)| 无需退款 |
| 待付款撤单(已付首期)| 原路退回(1-3 工作日)|
| 待签约撤单(已付首期未签)| 客服同意后,原路退回 |
| 已签约撤单(分账已发生)| ⭐ 走线下退款流程(详见 `财务管理/07 §6` 和 `订单管理/10`)|

**关键变化**:已签约且分账已发生的订单撤单,**不再原路退**,而是走 07 文档的线下退款工单流程(门店转账到位再退给客户)。

### 4.4 需客服同意的撤单流程

```
[取消订单] → 提示"已付款订单需联系客服处理"
   ↓
客户填取消原因 → 提交申请
   ↓
进入运营端"取消申请"队列
   ↓
客服审核:
  ├─ 同意 → 系统判断是否已分账:
  │         ├─ 未分账 → 平台对公直接退给客户
  │         └─ 已分账 → 生成退款工单(走 07 文档 §6 流程)
  └─ 拒绝 → 通知客户(给出理由)→ 订单继续
```
