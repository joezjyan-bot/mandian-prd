# 资方主体与授信管理

> [V0.2.2 同步声明] 本目录历史命名“资方管理”保留为内部配置入口兼容。当前业务语义统一为**内部资金来源管理**;相关信息仅运营端、财务端、风控端可见,不得进入 C 端、商家端、客服话术或合同正文。

> P0 业务文档(2026-05-27)。
> 定义资方主体管理、授信额度管理、收款账户、对账规则、合同模板/话术模板绑定的完整 PRD。
>
> **核心边界**:
> - 资方对**门店和客户都不可见**(完全是平台后台的资金安排)
> - 门店看到的是"商家订单 / 联营订单 / 平台订单"三种合作模式
> - 客户看到的是统一中性话术 + 资方对应的合同模板
> - 平台后台运营/财务才感知"资金来源"(funding_source)

---

## 1. 资方分类

| 资方类型(`funder_type`)| 说明 | 当前/计划接入 |
|---|---|---|
| `platform_self` | 平台自有资金(默认主资金源) | ✅ MVP 接入 |
| `bank` | 银行(赊销融资,持牌)| ✅ 蓝海银行 |
| `trust` | 信托机构 | 预留 |
| `private` | 民间机构资方 | 预留 |
| `factoring` | 商业保理公司 | 预留 |

## 2. 数据模型

### 2.1 `funder` 资方主表

```sql
funder
- funder_id               bigint PK
- funder_code             varchar(32)   -- 系统编号(如 FD_PLATFORM_SELF / FD_BLUE_OCEAN)
- funder_name             varchar(64)   -- "平台自营资金" / "蓝海银行"
- funder_type             enum          -- platform_self / bank / trust / private / factoring
- legal_entity            varchar(128)  -- 法律主体名称
- legal_entity_credit_no  varchar(32)   -- 统一社会信用代码
- contact_person          varchar(32)   -- 对接联系人
- contact_phone           varchar(20)
- contact_email           varchar(64)

-- 授信额度
- credit_limit_total      decimal(18,2) -- 总授信额度
- credit_limit_used       decimal(18,2) -- 已用额度(SUM(outstanding_balance))
- credit_limit_left       decimal(18,2) -- 剩余额度

-- 资方合同与话术模板
- contract_template_id    bigint FK     -- 客户合同模板 ID(必填)
- terminology_template_id bigint FK     -- C 端话术模板 ID(必填)

-- 资方收款账户(撤单退款 / 应收账款回款用)
- repayment_account_name      varchar(64)   -- 户名
- repayment_account_no        varchar(32)   -- 账号
- repayment_account_bank      varchar(64)   -- 开户行
- repayment_account_bank_code varchar(16)   -- 银行联行号

-- 资方收益规则
- service_fee_rate        decimal(8,6)  -- 平台从该资方拿的服务费率(0-1)
- profit_share_rate       decimal(8,6)  -- 资方溢价分成比例(若适用)

-- 资方风控规则
- risk_rule_id            bigint        -- 关联的风控规则 ID
- min_order_amount        decimal(18,2) -- 该资方接单的最小订单金额
- max_order_amount        decimal(18,2) -- 该资方接单的最大订单金额
- max_period_months       int           -- 该资方接单的最长订单周期(月)
- min_customer_credit     int           -- 该资方接单的最低客户信用评分

-- 状态
- status                  enum          -- active / paused / closed
- paused_reason           text
- paused_at               datetime

- created_by              bigint
- created_at / updated_at

-- 索引:(funder_type, status) / (funder_code) 唯一约束
```

### 2.2 `funder_receivable_ledger` 资方应收账款台账

详见 `财务管理/07 §7`,核心字段:

```sql
funder_receivable_ledger
- ledger_id               bigint PK
- funder_id               bigint FK
- order_id                bigint FK
- order_no                varchar(32)
- cooperation_mode        enum          -- 合作模式快照
- transferred_amount      decimal(18,2) -- 资方支付的应收账款转让对价 / 联营出资款
- collected_amount        decimal(18,2) -- 已从客户租金回收的部分
- refunded_amount         decimal(18,2) -- 撤单时退回资方的金额
- outstanding_balance     decimal(18,2) -- 应收余额 = transferred - collected - refunded
- status                  enum          -- active / collected / refunded
- created_at / updated_at
```

### 2.3 `funder_log` 资方操作日志

```sql
funder_log
- log_id                  bigint PK
- funder_id               bigint FK
- action                  enum          -- create / update / pause / resume / close / credit_adjust / template_change
- operator_id             bigint
- operator_role           varchar
- previous_value          json
- new_value               json
- reason                  text
- created_at              datetime
```

### 2.4 `funder_credit_adjustment` 授信额度调整记录

```sql
funder_credit_adjustment
- adjustment_id           bigint PK
- funder_id               bigint FK
- adjustment_type         enum          -- increase / decrease / reset
- amount_before           decimal(18,2)
- amount_change           decimal(18,2)
- amount_after            decimal(18,2)
- reason                  text
- approval_id             bigint        -- 关联审批单
- operator_id             bigint
- created_at              datetime
```

## 4. 授信额度管理

### 4.1 自动增减规则

```
订单生效时(联营/平台订单监管锁回调):
  funder.credit_limit_used += transferred_amount
  funder.credit_limit_left -= transferred_amount
  funder_receivable_ledger.outstanding_balance = transferred_amount

客户每期应付时(联营/平台订单):
  funder_receivable_ledger.collected_amount += 资方应收份额
  funder_receivable_ledger.outstanding_balance -= 资方应收份额
  (注:授信额度不变,credit_limit_used 仍为投入总额,直到订单完成或撤单)

订单完成(履约完成):
  funder.credit_limit_used -= transferred_amount
  funder.credit_limit_left += transferred_amount
  funder_receivable_ledger.status = collected

订单撤单退款:
  funder.credit_limit_used -= refunded_amount
  funder.credit_limit_left += refunded_amount
  funder_receivable_ledger.refunded_amount += refunded_amount
  funder_receivable_ledger.status = refunded
```

### 4.2 手动调整授信

| 场景 | 操作 |
|---|---|
| 与资方达成扩额协议 | 财务主管 + 运营主管双审批后,credit_limit_total 增加 |
| 资方降额 | 财务主管 + 运营主管双审批后,credit_limit_total 减少 |
| 调整后剩余额度 < 0 | 系统阻止,需先暂停或归还订单 |

调整记录写入 `funder_credit_adjustment`。

### 4.3 额度预警

| 预警条件 | 行为 |
|---|---|
| credit_limit_used / credit_limit_total ≥ 80% | 告警财务主管 |
| credit_limit_used / credit_limit_total ≥ 95% | 告警 + 自动暂停新订单分配该资方 |
| credit_limit_left < 单笔订单上限 | 系统不再推荐该资方 |

## 5. 资方对账

### 5.1 资方台账对账原则

```
对每个资方,日终对账验证:
   credit_limit_used = SUM(outstanding_balance) (所有 active 状态台账)
   总投入 = SUM(transferred_amount) (所有台账)
   总回收 = SUM(collected_amount) + SUM(refunded_amount)
   应收余额 = SUM(outstanding_balance) = 总投入 - 总回收
```

## 6. 资方暂停 / 关闭

### 6.1 暂停场景

| 场景 | 操作 |
|---|---|
| 资方临时不接单 | 暂停 → 新订单不分配该资方,已有订单继续履约 |
| 资方授信额度用完 | 自动暂停 → 直到额度增加或订单完成释放 |
| 资方违约 / 合作问题 | 主管手动暂停 → 走法务流程 |

### 6.2 关闭场景

| 场景 | 操作 |
|---|---|
| 与资方合作结束 | 关闭 → 不再接单 |
| 但已有订单仍需继续履约 | 不影响在外订单 |
| 直到所有 outstanding_balance = 0 后,资方记录归档 | - |
