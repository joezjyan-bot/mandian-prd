# 操作日志查询页

> **来源**: P2-2
> **核心**: V1 必做,按订单/操作人/时间/操作类型多维筛选,可导出

## 1. 页面定位

运营端 · 权限日志 · 操作日志查询。运营管理员、财务主管、合规主管查询所有操作日志。

## 2. 日志记录范围

V1 必记的关键动作:

### 2.1 资金类
- 钱包扣费(订单抽佣、线下手续费、关单费用、退款扣减、佣金生成等)
- 提现申请 / 提现审核 / 提现打款
- 退款单创建 / 责任划分 / 执行退款
- 后台余额变动(对公转账入账等)
- 抽佣比例变更
- 渠道佣金生成 / 结算 / 退款冲正

### 2.2 订单类
- 订单创建 / 状态流转
- 价格调整 / 取消重下
- 关闭订单
- 修改还款日

### 2.3 审核类
- 风控审核通过 / 驳回 / 人工放行
- 监管锁下 / 解
- 客户加黑名单

### 2.4 合同类
- 合同发起 / 签署回调 / 作废
- 公证发起 / 完成

### 2.5 配置类
- `platform_config` 变更
- 合同模板变更
- 电子签通道切换

### 2.6 权限类
- 员工角色变更
- 员工启用 / 禁用
- 权限授予 / 撤销

## 3. 筛选维度

- 操作类型(下拉多选)
- 操作对象类型(订单 / 钱包 / 合同 / 设备 / 配置 / 员工)
- 操作对象 ID(订单号、钱包 ID 等)
- 操作人(用户名 / 角色 / 员工 ID)
- 操作来源(运营端 / 商家 PC / 门店手机 / C 端 / API / 系统自动 / Webhook)
- 时间范围(默认最近 7 天)
- 关键字模糊搜索(命中备注 / IP 等)
- 是否成功

## 4. 列表字段

- 日志 ID
- 操作时间
- 操作类型
- 操作对象类型 + ID(可跳转)
- 操作人(姓名 + 角色)
- 操作来源
- 简要描述(自动生成,如"客服 张三 关闭了订单 XX,扣除 1.5% 关单费用")
- IP
- 操作前 / 操作后 (摘要,详情看完整)
- 操作结果(成功 / 失败)

## 5. 详情视图

点击单条日志查看完整内容:
- 完整 JSON 操作前数据快照
- 完整 JSON 操作后数据快照
- 完整 diff(高亮变化)
- 关联信息(IP、User-Agent、设备类型)
- 操作备注

## 6. 导出

- 当前筛选条件下导出 Excel
- 大量数据(> 1 万)异步导出 + 邮件通知

## 7. 权限

- 运营管理员 + 财务主管 + 合规主管:可查看所有日志
- 普通客服:仅能查看自己操作的日志
- 主管:可查看下属操作的日志

## 8. 审计告警

- 高频操作(同一用户单日操作 >100 条):告警
- 查看敏感数据(展开身份证 / 银行卡 / 客户全信息):额外日志 + 告警
- 操作失败率高(单用户 1 小时失败 > 10):告警

## 9. 性能考虑

- 操作日志表分库分表(按月分表)
- 索引:operator_id / operation_type / created_at / target_type+target_id
- 大查询(>30 天)异步导出
- 历史日志归档:> 1 年的日志归档冷存储,查询仍可访问
