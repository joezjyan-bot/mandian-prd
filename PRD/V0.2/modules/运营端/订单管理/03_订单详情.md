## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 运营端订单详情 |
| 所属端 | 运营端(内部页面;资方字段允许在此展示) |
| 入口路径 | 订单管理 > 商家订单 / 联营订单 / 平台订单 > 订单详情 |
| 使用角色 | 平台管理员、审核客服、订单客服、财务、风控、租后客服、运营主管 |
| 核心目标 | 承载订单完整资料、审核、合同、支付、发货、账单、分账、日志和回调状态 |

订单详情是订单数据的完整事实页。订单列表负责让客服快速判断和批量处理,订单详情负责展示更深层资料和执行高风险动作。

---

## 2. 核心口径(v1.1 更新)

1. 详情页必须支持三种合作模式:`商家订单 / 联营订单 / 平台订单`。
2. 商家订单由商家或门店自审,运营端默认只做查看、监管、平台综合服务费、客诉、租后和异常介入。
3. 联营订单和平台订单由运营端主控审核、内部资金来源分配、合同、公证、支付、发货链路。
4. 详情页所有人工操作、系统回调、接口调用、状态变更都必须进入操作日志或回调日志。
5. 订单详情展示的是下单时价格快照,商品、费率、增值服务后续改动不能反向污染历史订单。
6. 敏感字段按权限脱敏展示;查看明文必须记录查看人、时间、原因和来源 IP。
7. 客服改套餐、改价、发起补充合同、关闭订单、退款、变更资方、变更发货主体均属于高风险动作,需要权限和二次确认。
8. **资金来源 / 资方字段允许在本运营端详情页展示**,但客户侧(C 端)详情页**完全不展示**资金来源 / 资方 / 合作模式信息(详见话术规范文档)。
9. C 端按统一中性话术展示(订单总额 / 每期应付 / 履约中 等),后台沿用业务/法律严谨术语。

## 4. 顶部订单头(v1.1 新增字段)

| 字段 | 说明 |
|---|---|
| 订单号 | 系统订单编号,支持复制 |
| **合作模式** ⭐ | 商家订单 / 联营订单(出资比例 X/Y)/ 平台订单 |
| **资金来源** ⭐(后台) | 平台自营 / 蓝海银行 / 其他资方 |
| **资方** ⭐(后台) | 具体内部资金来源主体名称(关联 funder_id) |
| **合同模板** ⭐(后台) | 客户合同模板版本(标准租赁 v1.0 / 蓝海银行赊销 v2.3 等) |
| **话术模板** ⭐(后台) | C 端话术模板(标准租赁话术 / 蓝海银行赊销话术) |
| 订单状态 | 待付款、待审核、待分配内部资金来源、待签约、待支付、待发货、待归还、已逾期、已完成、已留购、已关闭等 |
| 下一步待办 | 根据 `next_action` 计算 |
| 商家/门店 | 商家名称、门店名称、负责人、办单员工 |
| 渠道来源 | 渠道、推广码、渠道人员 |
| 发货主体 | 默认门店,可配置为商家、平台、资方、快递接口 |
| 风险标签 | 疑似黑名单、资料缺失、定位异常、历史逾期、回调异常等 |

**C 端订单详情**:**完全不展示** 合作模式 / 资金来源 / 资方 / 合同模板 / 话术模板。

## 6. 操作按钮(v1.1 更新)

| 按钮 | 显示条件 | 二次确认 | 结果 |
|---|---|---|---|
| 审核通过 | 联营/平台订单待审核,或商家订单异常介入 | - | 进入下一步待办 |
| 审核不通过 | 待审核 | - | 填写原因,通知商家/客户,按规则关闭或退回 |
| 要求补资料 | 资料不完整 | - | 生成客户或商家待办 |
| **分配内部资金来源** ⭐ | 联营/平台订单待分配内部资金来源 | - | 选择资方(系统推荐 + 手动确认)+ 自动绑定合同模板 / 话术模板 |
| **更换资方** ⭐ | 合同/支付前且有权限 | **2FA** | 记录变更原因,更新资方、合同模板、话术模板和分账规则 |
| 修改套餐/改价 | 有改价权限 | **2FA** | 生成新价格方案,保留旧方案 |
| 发起授权 | 风控/征信未授权 | - | 调用授权链路并同步客户待办 |
| 发起合同 | 待签合同 | - | 按订单的合同模板调用 e 签宝并同步客户待办 |
| 发起公证 | 配置需要公证 | - | 同步客户小程序办理入口 |
| 生成收款码 | 部分支付或补缴 | - | 调用通联收款码 |
| 发货/确认交付 | 满足发货条件 | - | 进入发货或照片上传流程 |
| 关闭订单 | 有关闭权限 | **2FA** | 二次确认并填写原因;生成退款工单(若已分账)|
| **退款** ⭐ | 有退款权限 | **2FA** | 走退款工单流程(已分账走线下退款) |
| 加入黑名单 | 有风控权限 | **2FA** | 写入黑名单库并记录原因 |

1. 运营端是否允许对商家订单做强制关闭、强制退款、强制改价;如果允许,需要单独权限和主管审批。
2. 资方变更的截止节点建议定为合同签署前;合同签署后变更需要补充合同或冲正流程。
3. 公证是否按订单类型强制配置,还是按商品/商家/资方链路配置决定。
4. 已分配内部资金来源但未发起合同时,运营更换资方,合同模板 / 话术模板自动跟随更换的逻辑确认。
