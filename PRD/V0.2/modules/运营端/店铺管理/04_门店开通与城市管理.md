# 店铺管理 · 04 门店开通与城市管理

> 管理城市开通(平台层面)和门店开通(商家层面)。
> **本模块从「前台配置中心」迁移过来**(用户决策:城市与门店属于业务实体,不属于 UI 装修)。
> 门店是体验租的核心 — 客户找门店 / 取还车 / 控车都依赖门店经纬度和开通状态。

---

## 3. 城市管理

### 3.1 城市字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| city_code | 文本 | 是 | 标准行政区划编码(如 `110100`=北京市) |
| city_name | 文本 | 是 | 城市名 |
| city_pinyin | 文本 | 是 | 拼音首字母(用于 C 端字母索引) |
| province_code | 文本 | 是 | 省级编码 |
| province_name | 文本 | 是 | 省份名 |
| is_hot | 开关 | 否 | 是否热门城市(C 端城市选择页置顶) |
| service_business_lines | 多选 | 是 | 该城市开通的业务线:assurance_rent / experience_rent |
| display_order | 数字 | 是 | 同省内排序 |
| service_phone | 文本 | 否 | 该城市客服电话(可选,覆盖平台默认) |
| status | 下拉 | 是 | active(开通中)/ paused(暂停)/ inactive(未开通) |

### 3.2 城市分组

```sql
city_group
- group_id
- group_name        -- 华北 / 华东 / 华南 / 西南 / 西北 / 东北 / 港澳台
- city_codes[]
- display_order
```

C 端城市选择页按「热门城市 → 字母索引 → 分组」展示。

### 3.3 城市开通流程

```text
平台管理员新增城市
  ↓
填写 city_code / city_name / 经纬度范围
  ↓
选择该城市开通的业务线(可勾安心用 / 体验租 / 两者)
  ↓
status = active
  ↓
C 端城市选择页可见
```

**关闭城市规则**:有活跃订单的城市不能直接关闭,需先处理完存量订单或转其他城市。

## 4. 门店管理

### 4.1 门店字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| store_id | 文本(系统) | 是 | - |
| merchant_id | 关联 | 是 | 所属商家 |
| store_name | 文本 | 是 | 门店名(对外展示) |
| store_code | 文本 | 是 | 门店编码(后台用) |
| city_code | 关联 | 是 | 所在城市 |
| district | 文本 | 是 | 区/县 |
| address | 文本 | 是 | 详细地址 |
| location_lat | 数字 | 是 | 纬度(GCJ-02 坐标系) |
| location_lng | 数字 | 是 | 经度 |
| business_hours | JSON | 是 | 营业时间(支持每日不同) |
| contact_phone | 文本 | 是 | 门店电话 |
| store_head_url | 图片 | 否 | 门店头图 |
| store_photos | 图片数组 | 否 | 门店相册 |
| store_introduction | 富文本 | 否 | 门店介绍 |
| business_lines_enabled | JSON | 是 | 该门店开通的业务线:["assurance_rent", "experience_rent"] |
| store_status | 下拉 | 是 | pending_review / active / paused / closed |

### 4.2 门店业务线开通规则

**前提**:门店开通某业务线,商家必须先开通该业务线(详见 `03_商家入驻业务线勾选.md`)。

| 商家业务线 | 门店可开通 |
|---|---|
| 仅安心用 | 仅可开通门店的安心用 |
| 仅体验租 | 仅可开通门店的体验租 |
| 双业务线 | 门店可自由选择开通一条或两条 |

**门店开通入口**:
1. 商家在「商家 PC 端 > 门店管理 > 门店开通」自助提交
2. 平台审核(尤其体验租门店需要审车辆库存能力)
3. 审核通过 → 该业务线在该门店启用

### 4.3 经纬度维护

| 来源 | 说明 |
|---|---|
| 商家入驻时填写 | 通过地图选点(高德/百度 API) |
| 商家修改 | 需要平台审核(防止恶意改成虚假位置) |
| 平台校验 | 入驻审核时人工核对地图位置与营业执照地址 |

**经纬度坐标系**:统一使用 GCJ-02(国测局火星坐标系)— 与小程序定位坐标系一致,避免偏移。

**C 端附近门店查询**:
```sql
SELECT * FROM store
WHERE city_code = :user_city
  AND store_status = 'active'
  AND :line IN (JSON_EXTRACT(business_lines_enabled, '$'))
  AND ST_DISTANCE(POINT(location_lng, location_lat), POINT(:user_lng, :user_lat)) <= :radius_km
ORDER BY distance ASC
LIMIT 50;
```

## 5. 门店分配(平台自营 / 平台货源-门店履约)

体验租 3 种经营模式(详见 `modules/短租业务/01_体验租业务总览.md` §4):

| 模式 | 资产归属 | 履约门店 | 配置位 |
|---|---|---|---|
| store_self_operated | 门店自己 | 门店自己 | 默认 |
| platform_self_operated | 平台 | 平台直营门店 / 平台指定仓店 | 本模块 |
| platform_asset_store_fulfillment | 平台 | 商家门店 | 本模块(平台分配) |

### 5.1 平台自营门店开通

平台管理员可直接创建"平台直营门店",merchant_id 关联到一个特殊的"平台主体"(系统预置)。

```sql
merchant_platform_self  -- 系统预置的平台主体
- id (固定 'PLATFORM_SELF')
- merchant_name = '满点平台自营'
- business_lines = ["assurance_rent", "experience_rent"]
```

### 5.2 平台货源-门店履约配置

平台拥有车辆 / 设备,但找商家门店做履约。配置:

```sql
store_fulfillment_assignment
- assignment_id
- asset_owner_type     -- 'platform' 或 'merchant'(特殊场景)
- asset_owner_id
- fulfillment_store_id -- 履约门店
- effective_at / expired_at
- fulfillment_fee_rate -- 履约服务费比例(平台给门店)
- fulfillment_fee_fixed-- 履约固定费用(可选)
- status
```

**订单生成时**:
- 客户在 C 端选择该门店
- 订单字段:`asset_owner_type = 'platform'`、`fulfillment_store_id = store_id`、`merchant_id = 平台主体`
- 履约门店赚劳务费(从平台分账)

## 7. 门店冻结 / 关闭

| 场景 | 处理 |
|---|---|
| 临时停业(春节)| store_status = paused,C 端显示"暂停营业",可处理存量订单 |
| 永久关闭 | store_status = closed,需先处理完所有存量订单(短租在租 / 长租在租) |
| 单条业务线停用 | business_lines_enabled 移除该条,C 端该业务线不再展示该门店 |

## 8. 与 C 端的数据联动

| C 端使用 | 对应字段 |
|---|---|
| 城市切换列表 | `city.status = 'active'` 全部城市 |
| 体验租首屏附近门店 | `store.business_lines_enabled` 含 `experience_rent` 且 `store_status = 'active'` |
| 安心用门店选择 | `store.business_lines_enabled` 含 `assurance_rent` 且 `store_status = 'active'` |
| 门店详情双 Tab | 看门店 `business_lines_enabled` 是否同时含两条 |
| 门店导航到店 | location_lat / location_lng |
