# IM 客服页面与接口规则

> 本文补齐 IM 客服基础能力的页面级和数据级设计。V1 不追求复杂机器人和完整工单系统,但必须可用、可归档、可带订单上下文、可通过 `transfer` 转主管,并能进入法诉证据包。

## 1. V1 页面结构

### 1.1 C 端入口

| 页面 | 入口名 | 自动带入上下文 |
|---|---|---|
| 首页 / 我的 | 客服 | `customer_id`、最近业务线、最近订单 |
| 安心用订单详情 | 联系客服 | 长租订单号、订单状态、商品、商家、当前待办 |
| 体验租订单详情 | 联系客服 | 短租订单号、门店、车辆、租期、剩余时间、控车状态 |
| 支付失败页 | 联系客服 | 支付单号、金额、通道、失败码 |
| 合同/公证异常页 | 联系客服 | 合同 ID、公证 ID、异常原因 |
| 保证金扣款争议页 | 联系客服 | 扣款单、证据、申诉状态 |

### 1.2 商家端入口

| 页面 | 入口名 | 队列 |
|---|---|---|
| 订单详情 | 联系平台客服 | 长租订单 |
| 我的钱包 / 提现 | 联系财务客服 | 财务 |
| 体验租订单 / 车辆 | 联系体验租客服 | 体验租订单 |
| 商品和价格方案 | 联系运营客服 | 运营 |

### 1.3 运营端客服工作台

客服工作台固定四栏:

1. 会话列表:队列、等待时长、客户/商家、业务线、订单号、优先级。
2. 消息区:文本、图片、视频、订单卡片、系统事件。
3. 上下文卡片:客户脱敏资料、订单状态、账单/支付摘要、门店、短租车辆、异常处理标记。
4. 操作区:快捷回复、转接、标记待内部处理、跳订单、跳退款、跳保证金、跳控车日志。

操作区只能跳转,不得在 IM 内直接退款、扣款、锁车、关单、改价、修改账户。

## 2. 会话状态

| 状态 | 含义 | 进入条件 | 退出条件 |
|---|---|---|---|
| `queued` | 排队中 | 客户发起会话但未分配 | 分配客服 |
| `assigned` | 已派单待接 | 系统按轮询派给某客服 | 客服接单 / 超时转派 / 主管改派 |
| `active` | 处理中 | 客服接入 | 客户离开/转主管/结束 |
| `waiting_customer` | 待客户回复 | 客服发出问题 | 客户回复 |
| `waiting_internal` | 待内部处理 | 已转业务模块或主管处理 | 内部处理完成 |
| `ticketed` | 已标记待工单 | V1 仅作为待内部跟进占位标记;完整工单系统 V2 实现 | V1 由主管处理后关闭或转回;V2 工单关闭 |
| `closed` | 已结束 | 客服关闭或超时关闭 | 重新咨询新建/重开 |

### 2.1 客服在线状态

| 状态 | 含义 | 能否被派单 |
|---|---|---|
| `off_duty` | 未上班,默认状态 | 否 |
| `on_duty_idle` | 已上班且未达到并发上限 | 是 |
| `on_duty_busy` | 已上班但达到并发上限 | 否 |
| `on_break` | 小休 | 否 |
| `off_duty_manual` | 手动下班 | 否 |

规则:
1. 客服进入工作台后必须点击【上班】才进入派单池。
2. 点击【下班】或【小休】立即退出派单池;已接会话按转派 / 主管处理规则处理。
3. `current_load >= max_concurrent` 时自动进入 `on_duty_busy`,不再派新单;降到阈值以下后恢复 `on_duty_idle`。
4. 掉线超过可配时长时,未接任务自动回收,处理中会话提醒主管处理。

### 2.2 派单字段

```text
customer_service_agent
- agent_id / name
- duty_status       enum -- off_duty/on_duty_idle/on_duty_busy/on_break/off_duty_manual
- max_concurrent    int  -- 并发上限,后台可配
- current_load      int  -- 当前处理中会话数
- on_duty_at / off_duty_at datetime
- last_assigned_at  datetime -- round-robin 排序参考
- missed_assign_count int -- 连续未接次数

im_session(在现有基础上补)
- assigned_agent_id bigint -- 当前归属客服,用于单人锁定
- assign_status     enum   -- queued/assigned/active/timeout_reassigned/supervisor_reassigned/closed
- assign_deadline   datetime -- 当前接单截止时间,默认派单后 +3 分钟
- miss_count        int    -- 本会话累计转派次数
- assign_log        json   -- 历次派给谁、是否接、超时/改派原因
```

## 3. 队列和 SLA

| 队列 | 首响 SLA | 升级规则 |
|---|---|---|
| 体验租用车中 / 控车异常 | 1 分钟 | 超时提醒主管 |
| 支付 / 保证金 / 退款 | 3 分钟 | 超时转财务主管 |
| 长租订单 | 3 分钟 | 超时转客服主管 |
| 客诉争议 | 5 分钟内标记待内部处理并转主管 | 自动标高优先级;完整工单 SLA V2 实现 |
| 售前咨询 | 5 分钟 | 超时轮转 |
| 商家财务 | 10 分钟 | 超时提醒财务 |

### 3.1 轮询派单算法

```text
客户下单 / 发起会话
  -> 会话进入 queued
  -> 系统从 on_duty_idle 且 current_load < max_concurrent 的客服中按 round-robin 选择
  -> 写入 assigned_agent_id,assign_status=assigned,assign_deadline=now+3分钟
  -> 被派客服在倒计时内点击接单
       -> active,单人锁定,仅该客服可回复/转主管/关闭
     未接
       -> 记录 miss_count,自动转派下一个 on_duty_idle 客服,重置倒计时
  -> 一轮无人接
       -> 提醒主管,任务保留队列顶部继续轮询,可人工强制改派
```

单人锁定:
1. `assigned` 或 `active` 状态下,非 `assigned_agent_id` 的普通客服只能只读查看,操作按钮置灰。
2. 主管可强制改派,必须填写原因,记录原客服、新客服、操作人、时间。
3. 被派客服下班 / 掉线 / 小休时,未接会话自动回收;已接会话提醒主管转派。
4. 转派、超时、接单、主管改派均写入 `assign_log` 和操作日志。

### 3.2 派单配置

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 接单超时转派时长 | 3 分钟 | 可配置,默认按业务补充需求执行 |
| 客服并发上限 | TODO【需运营确认】 | 不编造固定数值,按队列 / 角色可配置 |
| 连续未接自动小休阈值 | TODO【需运营确认】 | 达到阈值可自动转 `on_break` |
| 无人接单升级策略 | 提醒主管 | 后续可扩展企微 / 短信 / 站内待办 |
| 派单池维度 | 队列 + 技能组 | V1 先按队列轮询,预留技能组 |

## 4. 接口规则

| 动作 | 接口 | 幂等键 | 关键校验 |
|---|---|---|---|
| 打开会话 | `POST /im/sessions` | `customer_id + business_line + order_id + trigger_page + time_bucket` | 订单归属、端类型、敏感字段过滤 |
| 发送消息 | `POST /im/sessions/{session_id}/messages` | `session_id + client_message_id` | 合规词、附件权限、会话状态 |
| 上班 / 下班 / 小休 | `POST /im/agents/me/duty-status` | `agent_id + duty_status + time_bucket` | 角色有效、未完成会话处理策略、操作日志 |
| 接单 | `POST /im/sessions/{session_id}/accept` | `session_id + assigned_agent_id + assign_deadline` | 必须为当前被派客服;未超时;会话为 `assigned` |
| 转接 / 主管改派 | `POST /im/sessions/{session_id}/transfer` | `session_id + from + to + reason` | 目标客服在线/技能匹配;主管强制改派需原因 |
| 标记待工单 / 转主管 | V1 使用 `POST /im/sessions/{session_id}/transfer`;完整工单系统 V2 实现 | `session_id + from + to + reason` | V1 仅支持将会话标记 `ticketed` 占位并经 `transfer` 转主管;不得新增工单接口或驱动工单流转 |
| 归档 | 后台任务 | `session_id + archive_version` | 消息、附件、上下文卡片完整 |

## 5. 合规拦截

客服输入命中以下类型时必须提醒或拦截:

| 类型 | 示例 | 动作 |
|---|---|---|
| C 端禁用字段 | 资方、资金来源、合作模式、风控结论 | 阻断发送或主管审批 |
| 绝对承诺 | 包过、一定退款、一定免赔、保证放款 | 阻断发送 |
| 金融误导 | 借款、贷款、融资租赁、账单支付压力 | 提醒改写为订单/账单/支付口径 |
| 私下收款 | 加微信转账、线下打款给个人 | 阻断并上报 |
| 高危操作承诺 | 我直接帮你锁车/扣款/关单 | 提醒跳转业务模块 |

## 6. 与法诉证据包的关系

以下内容必须可按订单导出:

- 会话基础信息
- 消息全文或脱敏摘要
- 图片/视频/附件 hash
- 订单上下文卡片快照
- 转接记录
- 转派 / 待工单标记记录(V2 起包含完整工单记录,V1 仅含 `transfer` 转派记录)
- 合规命中记录
- 客户确认/申诉记录

法诉导出必须写审计日志,客服不能私自下载完整证据包。

## 7. V1 不做但要预留

| 能力 | 预留方式 |
|---|---|
| 机器人自动问答 | 预留 `im_bot_intents`,不阻塞 V1 |
| 智能质检 | 保留合规命中和消息归档,后续接质检 |
| 客户-门店私聊 | 先不开;后续必须先补归档、投诉、监管规则 |
| 电话录音接入 | V2 工单可上传录音附件;V1 作为会话附件归档 |
| App Push | 通过 `customer_external_accounts` 扩展 |

## 8. 验收标准

1. C 端安心用和体验租订单详情均能打开会话。
2. 会话自动带订单上下文,客服无需让客户重复报订单号。
3. 高危动作在 IM 内只能跳转,不能直接执行。
4. 会话可标记 `ticketed` 占位并经 `transfer` 转主管;完整工单责任方、证据、SLA 追踪 V2 实现。
5. 禁用词有拦截或提醒。
6. IM 消息、附件和上下文卡片能归入法诉证据包。
7. 客服未点击【上班】不进入派单池,不会收到新会话。
8. 新会话按 round-robin 自动派给在线空闲客服,3 分钟未接自动转派下一人。
9. 已派单 / 已接单会话只能由归属客服操作,其他客服只读;主管强制改派必须留痕。
10. 客服下班、掉线、小休时,未接任务自动回收重派。
