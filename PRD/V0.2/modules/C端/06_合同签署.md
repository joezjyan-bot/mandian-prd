# 合同签署

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> **来源**: P1-5 / P1-4

## 1. 业务背景

客户在订单详情看到"待签合同",点击进入电子签通道完成签署。

## 2. 触发节点

订单审核通过 + (联营/平台订单) 分配资方后,系统**后台发起**合同签署:
- 系统调电子签通道(e签宝/法大大),生成合同
- 推送 C 端通知客户去签

## 3. 客户签署流程

```
1. 客户在小程序订单详情看"去签合同"按钮(红点提醒)
2. 点击 → 跳转电子签通道页面
3. 电子签通道页面:
   - 完整展示合同 PDF
   - 客户阅读 + 滚动到底
   - 必须本人:人脸识别 + 输入验证码
4. 签署完成 → 通道回调
5. 系统更新订单状态:客户已签
6. 等待门店签 → 平台签(自动)
7. 全部签完 → 进入下一步(保证金/首期支付 / 发货)
```

## 4. 三方制(P1-4 详)

- 甲方:门店
- 乙方:客户
- 丙方:平台

## 5. 模板自动选择

- 商家订单 → 默认模板
- 联营/平台订单 → 资方专属模板(若有)或默认模板

## 6. 失败处理

| 场景 | 处理 |
|---|---|
| 签署超时(默 24h) | 客服核查通道方,重新发起 |
| 客户拒签 | 订单进"待客服处理" |
| 人脸不通过 | 客户可重试,3 次后客服介入 |
| 通道方故障 | 自动尝试备用通道 |
