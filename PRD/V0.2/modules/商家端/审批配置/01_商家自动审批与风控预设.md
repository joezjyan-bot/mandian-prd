# 商家自动审批与风控预设

> **V0.2 Peer Review 修订(2026-05-27)v1.0**:
> - 本文落实同行小程序能力吸收任务 2。
> - 商家可为**商家订单**配置自动审批规则。
> - “全部通过”保留,但必须高危二次确认并由商家自担风险。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家自动审批与风控预设 |
| 所属端 | 商家端 / 运营端只读与监管 |
| 入口路径 | 商家中心 > 配置管理 > 审批配置 |
| 使用角色 | 商家老板、商家管理员、运营监管 |
| 核心目标 | 让商家订单可按商家自身风险偏好自动进入下一环节,减少人工审核成本 |

---

## 2. 核心口径

1. 本配置**仅作用于商家订单**。
2. 联营订单 / 平台订单必须走平台统一风控和运营审核,商家不可关闭。
3. 商家订单可选择“依据风控审批 / 全部通过 / 手动审批”。
4. “全部通过”是商家自主经营权限,但需高危红字提示、二次确认、操作日志。
5. 平台底线拦截不允许关闭:身份不一致、平台黑名单、明显欺诈、合规强拦截等。
6. C 端不展示“风控”“商家审批规则”等内部字眼,客户只看到“审核中”。

---

## 3. 自动审批开关

| 配置项 | 说明 |
|---|---|
| 是否启用自动审批 | 开启后商家订单按规则自动处理 |
| 生效门店 | 全部门店或指定门店 |
| 生效商品 | 全部商品或指定商品/SKU |
| 生效时间 | 立即生效或指定时间 |
| 兜底规则 | 自动审批失败后进入人工审核 |

---

## 4. 审批规则

| 模式 | 说明 | 风险 |
|---|---|---|
| 依据风控审批 | 符合商家配置的风控条件后自动通过 | 中 |
| 全部通过 | 平台底线拦截外,商家订单全部自动通过 | 高 |
| 手动审批 | 所有商家订单由商家手动审核 | 低 |

“全部通过”确认弹窗:

```text
高危提示

您正在开启“全部通过”模式。
开启后,除平台黑名单、身份不一致、欺诈拦截等底线规则外,
商家订单将自动进入后续支付/签约环节。

由此产生的逾期、损失、客诉、设备追回等风险,
由商家自行承担,平台不为此类订单的经营风险背书。

[取消] [我已知晓风险,确认开启]
```

## 5. 风控条件预设

| 方案 | 说明 |
|---|---|
| 保守型 | 8 项条件全部启用 |
| 开放型 | 仅启用核心条件:平台黑名单、身份三要素、当前逾期 |
| 自定义 | 商家逐项配置 |

8 项条件:

| 条件 | 可选配置 |
|---|---|
| 在执行案件 | 无在执案件 / 不做要求 |
| 历史案件 | 无历史案件 / 不做要求 |
| 失信人/黑名单 | 非失信人/黑名单 / 不做要求 |
| 历史逾期 | 无历史逾期 / 不做要求 |
| 当前网贷逾期 | 无逾期 / 不做要求 |
| 租赁申请行为 | 正常 / 不做要求 |
| 身份三要素一致性 | 必须一致 / 不做要求 |
| 手机号状态 | 正常 / 不做要求 |

平台底线拦截:
1. 平台黑名单命中。
2. 实名身份与下单人不一致。
3. 风控通道返回明确欺诈/冒用。
4. 法务或监管配置的强制拦截规则。

---

## 6. 流程

```text
客户提交商家订单
   ↓
平台底线规则检查
   ├─ 命中底线拦截 → 进入人工审核/驳回
   └─ 未命中
        ↓
      读取商家审批配置
        ├─ 全部通过 → 自动通过
        ├─ 手动审批 → 商家待审核
        └─ 依据风控审批 → 按预设条件判断
```
