# 收货确认

> **来源**: P1-5

## 1. 业务背景

订单生效 + 门店发货后,客户收到设备并验收。客户在小程序确认收货。

## 2. 触发节点

门店标记发货 → 客户在订单详情看"确认收货"按钮 → 红点提醒

## 3. 收货确认流程

```
1. 客户在订单详情看"确认收货"
2. 点击 → 显示发货信息(快递单号或门店自送)
3. 客户验收设备:
   - 设备型号 / 序列号正确
   - 设备外观无明显损坏
   - 开机检测基本功能
4. 选项:
   - 确认收货:无问题
   - 有问题(申请争议):走异常流程
5. 确认收货 → 订单状态:出租中
   - 设备状态:出租中
   - 账单日起算
```

## 4. 必须客户本人

- 不允许门店代客户确认收货
- 客户必须用本人小程序账户点击

## 5. 长期未确认收货

- 7 天未确认 → 系统自动提醒
- 14 天未确认 → 推送客服跟进
- 30 天未确认 → 系统默认收货(防止恶意)

## 6. 收货时争议

如客户验收发现问题:
- 在订单详情点"申请争议"
- 选择争议类型(设备问题 / 配置不符 / 缺配件等)
- 上传照片证据
- 设备状态:争议中
- 进入异常流程(详见 13)

## 7. 数据模型

- `order.delivery_status`
- `order.received_at`(客户确认收货时间)
- `order.receive_status`(收货 / 争议 / 拒收)
