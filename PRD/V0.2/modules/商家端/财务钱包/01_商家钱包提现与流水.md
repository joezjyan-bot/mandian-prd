# 商家钱包提现与流水

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：商家 PC 端和门店手机端老板账号可以查看自己的收益、提现、接口扣费和对账明细；员工账号不可见钱包数据。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家钱包提现与流水 |
| 所属端 | 商家 PC 端、门店手机端 |
| 入口路径 | 商家端 > 财务钱包 / 门店手机端 > 我的 > 钱包 |
| 使用角色 | 商家老板、财务管理员、店长；员工账号不可见 |
| 核心目标 | 展示商家订单收益、联营订单收益、协作收益、提现、接口计费和冻结记录 |
| 运营联动 | 运营端财务管理、提现审核、对账中心、店铺详情 |

---

## 2. 权限边界

| 角色 | 可看余额 | 可申请提现 | 可看明细 | 可导出 | 说明 |
|---|---|---|---|---|---|
| 商家老板 | 是 | 是 | 是 | 是 | 主账号完整权限 |
| 财务管理员 | 是 | 可配置 | 是 | 是 | 由老板分配 |
| 店长 | 可配置 | 否 | 可配置 | 否 | 可只看经营数据 |
| 员工账号 | 否 | 否 | 否 | 否 | 只办单和处理待办 |

门店手机端登录保持可以按安全策略保留，但钱包入口在查看余额、提现、敏感流水时需要二次验证。

## 3. 钱包首页

| 模块 | 字段 |
|---|---|
| 余额卡片 | 可提现余额、待结算余额、冻结余额、提现中 |
| 收益概览 | 今日收益、本月收益、累计收益、累计提现 |
| 订单收益 | 商家订单收益、联营订单门店份额、平台订单协作收益 |
| 扣费概览 | 平台抽佣、接口计费、公证/信用/信息调用费用、退款冲正 |
| 快捷入口 | 提现、流水明细、提现记录、收款账户、接口计费 |

规则：

1. 钱包首页所有金额都必须能下钻到明细。
2. 余额不足、账户未实名、授权未完成、店铺冻结时，提现按钮置灰并展示原因。
3. 平台抽佣默认 2%，但商家端只展示实际结算结果和规则版本。

## 4. 收益构成

| 来源 | 入账规则 | 展示说明 |
|---|---|---|
| 商家订单 | 客户账单结清后扣平台抽佣，剩余入商家钱包 | 商家自营收益 |
| 联营订单 | 按订单锁定比例拆分，门店份额扣抽佣后入账 | 展示门店占比和平台资金占比 |
| 平台订单 | 默认不产生门店出资收益，如配置协作服务费则入账 | 展示协作收益来源 |
| 渠道佣金 | 如果该商家同时是推广主体，按渠道规则入账 | 与渠道账户可合并或独立配置 |
| 人工调账 | 运营端审批后生成 | 必须展示原因和审批单 |

## 6. 提现

### 6.1 提现申请

| 字段 | 类型 | 规则 |
|---|---|---|
| 可提现余额 | 只读 | 已过结算条件且未冻结 |
| 提现金额 | 金额输入 | 不得超过可提现余额;单笔 500-20000 元 |
| 收款账户 | 只读/选择 | 需实名一致或已完成授权 |
| 手续费 | 只读 | 0.2% + 0.5 元,按运营端配置展示 |
| 备注 | 文本 | 可选 |
| 二次验证 | 验证码/密码 | 按安全策略配置 |

提现限制:

1. 普通提现单笔 500-20000 元。
2. 同一商家每日最多提交 3 次提现申请。
3. 单笔超过 20000 元进入人工审核,不得由前端直接放行。
4. 提现金额、手续费、到账金额必须在提交前二次确认。

### 6.2 提现状态

| 状态 | 说明 |
|---|---|
| 待审核 | 已提交运营端财务 |
| 已驳回 | 展示驳回原因 |
| 打款中 | 财务已通过，等待通道结果 |
| 打款成功 | 到账完成，生成提现流水 |
| 打款失败 | 退回可提现余额，展示失败原因 |

提现提交后：

1. 提现金额从可提现余额转入提现中。
2. 财务驳回或打款失败时退回可提现余额。
3. 成功后计入累计提现。
4. 所有节点同步通知商家老板。

## 7. 接口计费

| 费用项 | 来源 |
|---|---|
| 风控查询 | 下单审核、补充风控报告 |
| 大数据报告 | 商家配置或运营发起 |
| 合同签署 | 合同发起和补充合同 |
| 公证服务 | 公证办理 |
| 短信通知 | 客户通知、商家通知 |
| 其他接口 | 后续外部能力 |

规则：

1. 接口计费按运营端配置执行，可按次、按订单或按套餐。
2. 商家订单调用平台能力时也可产生接口扣费。
3. 扣费失败或余额不足时，按配置限制继续调用。
4. 每笔扣费必须关联订单、客户操作或系统任务。

---

## 8. 冻结与争议

| 冻结原因 | 示例 |
|---|---|
| 退款争议 | 客户申请退款或支付异常 |
| 客诉争议 | 支付宝投诉处理中 |
| 租后异常 | 逾期、拒还、验收争议 |
| 风控异常 | 黑名单命中、资料异常 |
| 财务异常 | 对账不平、通道回调冲突 |

冻结记录展示：冻结金额、原因、关联订单、冻结时间、预计解除条件、处理人。

---

## 9. 对账与导出

商家端提供轻量对账：

| 功能 | 说明 |
|---|---|
| 按订单导出 | 导出订单收益、抽佣、扣费、提现状态 |
| 按流水导出 | 导出钱包流水 |
| 按月份汇总 | 月度收入、支出、提现、冻结 |
| 异常提示 | 分账失败、提现失败、退款冲正 |

导出文件不包含客户完整身份证、银行卡、手机号等敏感信息。

## 10. 与运营端联动

| 运营端动作 | 商家端表现 |
|---|---|
| 提现审核通过 | 提现记录进入打款中 |
| 提现驳回 | 展示驳回原因，余额退回 |
| 钱包冻结 | 余额冻结，提现受限 |
| 接口扣费配置变更 | 展示新规则生效时间 |
| 退款冲正 | 生成负向流水 |
| 收款账户审核驳回 | 提现入口限制 |
