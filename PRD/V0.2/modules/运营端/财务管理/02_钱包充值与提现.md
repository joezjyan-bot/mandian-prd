> **来源**: P0-1
> **位置**: 运营端 · 财务管理

## 1. 页面定位

运营端管理所有主体(商家/门店/资方/渠道)钱包余额、充值记录、提现申请、提现审核。

## 2. 主要入口

- 运营端 · 财务管理 · 钱包总览
- 运营端 · 财务管理 · 提现审核列表

## 3. 钱包总览页

### 3.1 顶部指标
- 总余额(所有钱包合计)
- 今日充值金额
- 今日提现金额
- 待审核提现数
- 门店欠款总额
- 资方待扣费总额

### 3.2 钱包列表

筛选:钱包类型 / 关联主体 / 余额范围 / 状态

字段:
- 钱包 ID
- 钱包类型(商家主体钱包/门店结算账户/资方账户/渠道钱包)
- 关联主体
- 当前余额
- 待扣费金额
- 可提现金额 = 余额 - 待扣费
- 总充值 / 总提现
- 状态(正常/冻结)
- 操作:查看流水 / 手工调账(跳后台余额变动)

### 3.3 流水详情

字段:
- 时间、变动金额(+/-)、变动后余额
- 业务类型(订单抽佣/线下手续费/关单费用/退款冲正/充值/提现/后台调账/逾期费用)
- 关联订单/账单期
- 备注(按措辞规范)
- 操作人、状态

## 4. 提现审核列表

### 4.1 筛选
- 申请人主体类型
- 金额范围
- 状态:pending / approved / rejected / processing / success / failed
- 申请时间
- 是否需人工审核(>2万)

### 4.2 列表字段
- 提现号、申请主体、钱包类型
- 申请金额、手续费、实际到账
- 到账档位(快速/普通/慢速)
- 通道(通联/直付通)
- 是否需审核
- 状态、申请时间
- 操作:详情/通过/拒绝/手动打款

### 4.3 审核操作
- 通过:二次确认 → 调通道发起打款 → 状态"打款中"
- 拒绝:填备注 → 状态"已拒绝" → 钱包不扣 → 通知申请人
- 任何操作留日志

## 5. 提现规则

| 项 | 规则 |
|---|---|
| 最低 | 500 元 |
| 最高 | 2 万 |
| 日次数 | ≤ 3 |
| 手续费 | 千分之 2 + 5 毛 |
| 审核 | >2万 人工审核 |
| 错账号 | 退回钱包 |

## 6. 充值通道

- 三方支付(支付宝/微信/银行卡):实时,钱包页"充值"按钮
- 对公转账:联系客服 → 后台余额变动页手工入账

## 7. 状态机

pending → approved/rejected
approved → processing → success/failed
failed → 等重试或返钱包

## 8. 权限

- 运营管理员/财务主管:全能力
- 普通客服:可查不可审
- 商家/资方:仅看自己钱包(各自端入口)

## 9. 数据模型

见 `00_核心决策表.md` `withdrawal_request`。
