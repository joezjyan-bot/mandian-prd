# 渠道 H5 数据统计与提现

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：为渠道推广人员提供独立 H5，查看推广入驻、联营订单、平台订单、佣金明细和提现进度。

> **⚠️ v1.1 修订(2026-05-29)**：§9.2 渠道佣金提现补全限额/次数/费率/人工审核阈值，套用与门店结算账户提现一致的规则;§13 待确认第 4 项(渠道提现规则)已定稿,从待确认移除。

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 渠道 H5 数据统计与提现 |
| 所属端 | 渠道端 H5 |
| 入口路径 | 渠道 H5 登录页 > 数据统计 / 佣金明细 / 提现 |
| 使用角色 | 渠道推广人员、渠道团队负责人、二级推广人 |
| 核心目标 | 让渠道看到自己推广带来的商家入驻、联营订单、平台订单和佣金，不展示门店自营订单 |

---

## 2. 核心口径

1. 渠道推广码扫码后跳转门店入驻页面。
2. 入驻审核通过后，商家和渠道建立推广归属。
3. 渠道只统计推广商家产生的联营订单和平台订单。
4. 商家订单默认不进入渠道统计和佣金。
5. 佣金规则由运营端配置，渠道端只查看结果和提现。
6. 支持一级、二级推广关系，但不做无限层级。
7. 渠道端所有金额要能下钻到佣金明细，不能只展示总数。

| 角色 | 数据范围 |
|---|---|

## 9. 提现

### 9.1 提现首页

| 字段 | 说明 |
|---|---|
| 可提现余额 | 满足结算条件 |
| 提现中 | 已申请未到账 |
| 已提现 | 历史成功提现 |
| 冻结金额 | 逾期、退款、争议冻结 |
| 收款账户 | 脱敏展示 |

### 9.2 提现申请

| 字段 | 类型 | 说明 |
|---|---|---|
| 可提现余额 | 只读 | 已过结算条件且未冻结 |
| 提现金额 | 金额 | 不得超过可提现余额;单笔 ¥500-¥20,000 |
| 收款账户 | 只读/选择 | 需已实名或已授权 |
| 手续费 | 只读 | 0.2%(千二)+ ¥0.5,后台可配,展示规则 |
| 备注 | 文本 | 可选 |
| 确认 | 按钮 | 提交运营端财务审核 |

**提现规则(套用门店结算账户提现同一套规则,详见 财务管理/07 §9.2、财务管理/02、决策表 P0-1)**:

- 单笔最低提现金额:¥500
- 单笔最高提现金额:¥20,000
- 每日最多 3 次
- 手续费:千分之 2 + ¥0.5,后台可配
- 超过 ¥20,000 或命中异常规则进入人工审核,不直接提交通道
- 审核通过后通过通联代付出款;失败退回可提现余额

> 口径说明:渠道佣金提现与门店结算账户提现采用同一套限额/次数/费率/人工审核规则,以便复用通联代付通道、风控与对账逻辑,全平台提现规则统一。佣金提现的资金来源为渠道佣金账户(wallet_entry / withdraw,渠道账户用),与门店结算账户(settlement_*)分属不同账户体系,但提现限额口径一致。
