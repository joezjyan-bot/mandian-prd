# 资方分配工作台

> **来源**: P1-2
> **核心**: 系统推荐 + 人工确认;2 小时超时进待人工介入

## 1. 页面定位

运营端 · 订单管理 · 待分配资方工作台(分红/平台订单核心节点)。

## 2. 业务背景

分红订单 + 平台订单审核通过后,需要分配资方出资。V1 用"系统推荐 + 人工确认"。

## 3. 工作台布局

### 3.1 待分配队列(左侧)
- 默认按创建时间从早到晚
- 高亮:超时(> 2 小时未分配)
- 顶部计数:待分配总数 + 超时数

### 3.2 订单列表
每个订单卡片显示:
- 订单号
- 订单类型(分红/平台)
- 商品 + 设备款
- 客户姓名(脱敏)
- 创建时间 + 已等待时长
- 状态(待分配 / 超时待人工)

### 3.3 订单详情区(右侧,点击订单展开)
- 订单基本信息
- 客户风控报告快照
- 系统推荐资方列表(1-3 个)
- 客服选择区

## 4. 系统推荐逻辑

### 4.1 筛选规则
```
1. 品类匹配:订单品类 ∈ 资方 allowed_categories
2. 单笔限额够:订单设备款 ≤ 资方 max_per_order
3. 总额限额够:订单设备款 + 资方当前累计 ≤ 资方 max_total_active
4. 余额够:资方钱包余额 ≥ 订单需出资金额
5. 资方状态:active

排序:按"剩余可放款额度"从高到低
推荐:前 3 个
```

### 4.2 推荐卡片显示
- 资方名称
- 剩余可放款额度
- 当前在租累计 / 总额限额
- 钱包余额
- 历史合作订单数
- 最近 30 天平均放款金额

## 5. 客服分配操作

### 5.1 选定资方
- 点击候选资方卡片"选择"按钮
- 二次确认弹窗显示关键信息
- 必填"选择理由"(简要,如"额度充足/合作关系紧密/客户匹配")
- 提交后:
  - 创建 funder_assignment 记录
  - 订单状态:已分配资方
  - 锁定合同模板(资方有专属用专属,否则默认)
  - 自动推动订单进入"待合同签署"

### 5.2 候选资方都不满足
- 客服可点击"展开全部资方"看完整列表
- 手动指定资方(可能临时让运营临时充值资方钱包)
- 留备注

### 5.3 候选资方都拒接(实际场景少见)
- 客服联系客户改方案或取消订单

## 6. 超时机制

### 6.1 超时定义
订单进入待分配 ≥ 2 小时未分配

### 6.2 超时处理
- 队列高亮(红色)
- 推送告警(运营主管 + 客服)
- 主管协助协调

### 6.3 长时间超时
> 12 小时仍未分配:升级告警,联系客户协商方案

## 7. 历史分配记录

工作台旁边 Tab:"已分配资方"
- 列出所有已分配订单
- 可筛选 / 导出
- 可查看每单的资方选择记录

## 8. 权限

- 资方管理客服:可分配
- 普通客服:可查看,不可分配
- 主管:可分配 + 改派(已分配但客户未签合同前)

## 9. 改派(罕见场景)

订单已分配资方,但未签合同前发现资方钱包不足或政策调整:
- 主管可发起"改派"
- 重新走分配流程
- 留完整改派日志
