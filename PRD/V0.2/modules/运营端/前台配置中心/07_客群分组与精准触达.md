## 3. 客群规则

### 3.1 系统预置客群(平台默认)

| segment_code | 客群名 | 规则 |
|---|---|---|
| `new_user` | 新客户 | 注册时间 ≤ 7 天 且 订单数 = 0 |
| `repeat_user` | 老客户 | 完成订单数 ≥ 1 |
| `high_value_user` | 高价值客户 | 累计 GMV ≥ ¥10,000 |
| `student` | 校园学生 | 实名年龄 18-24 + 手动认证或学校 GPS |
| `commuter` | 通勤白领 | 工作日 7-9 点、17-19 点订单占比 ≥ 60% |
| `food_delivery` | 外卖骑手 | 行业实名认证为外卖骑手 / 月租订单占比 ≥ 50% |
| `tourist` | 旅游游客 | GPS 与实名地址不同省 + 订单租期 ≤ 7 天 |
| `inactive_user` | 沉睡用户 | 注册时间 ≥ 30 天 且 最近 30 天无订单 |

### 3.2 客群字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| segment_id | 文本(系统) | 是 | - |
| segment_code | 文本 | 是 | 客群编码(英文,代码引用) |
| segment_name | 文本 | 是 | 客群中文名 |
| segment_type | 下拉 | 是 | `system`(系统预置)/ `custom`(自定义) |
| description | 富文本 | 否 | 客群说明 |
| rules | JSON | 是 | 规则定义(支持 AND / OR 组合) |
| is_enabled | 开关 | 是 | 是否生效 |
| estimated_user_count | 数字(系统) | 是 | 系统每日重新计算 |
| created_by | 关联 | 是 | 创建人 |

### 3.3 规则字段(rules JSON 结构)

支持的规则维度:

| 维度 | 字段 | 操作符 |
|---|---|---|
| 用户属性 | 注册时间 / 实名年龄 / 性别 / 实名城市 | =, >, <, >=, <=, between, in |
| 订单行为 | 累计订单数 / 累计 GMV / 最近订单时间 | 同上 |
| 业务线偏好 | 安心用订单数 / 体验租订单数 | 同上 |
| 时段偏好 | 工作日订单占比 / 时段订单占比 | 同上 |
| GPS | 与实名地址同省 / 异省 | =, != |
| 标签认证 | 学生 / 骑手 / 司机(需运营手动打标) | =, != |

**JSON 示例**:

```json
{
  "operator": "AND",
  "conditions": [
    {"field": "registered_days_ago", "op": "<=", "value": 7},
    {"field": "order_count", "op": "=", "value": 0}
  ]
}
```

## 4. 客群与内容元素绑定

### 4.1 客群-Banner 绑定(在 Banner 投放页配置)

Banner 字段 `target_audience_id` 引用本模块的 `segment_id`。

### 4.2 客群-商品推荐绑定

| 字段 | 类型 | 说明 |
|---|---|---|
| binding_id | 文本(系统) | - |
| segment_id | 关联 | 客群 |
| business_line | 下拉 | assurance_rent / experience_rent |
| recommend_position | 下拉 | 首页橱窗 / 品类页推荐位 / 商品详情底部 |
| product_ids | 数组 | 推荐商品 ID 列表(运营选品) |
| sort_order | 数字 | 同一位置多客群冲突时按优先级 |
| effective_at / expired_at | 时间 | - |

**冲突解决**:
- 同一用户命中多个客群时,按 `sort_order` 取优先级最高的客群的推荐
- 全部未命中时,降级到默认推荐(由"装修管理"配置)

### 4.3 客群-套餐主推绑定(体验租特有)

| 字段 | 类型 | 说明 |
|---|---|---|
| segment_id | 关联 | 客群 |
| featured_cycle | 多选 | 主推的租赁周期:1h / 2h / 4h / 当日还 / 日租 / 周租 / 月租 |
| product_id_or_vehicle_no | 关联 | 应用到哪些车型(空 = 全部) |

**示例**:
- 学生客群 → 主推日租 / 周租
- 骑手客群 → 主推月租
- 通勤客群 → 主推时租
- 游客客群 → 主推日租

### 4.4 客群-弹窗绑定

弹窗字段 `target_audience_id` 引用 `segment_id`。

### 4.5 客群-品类排序(V2)

V1 不实现。V2 支持按客群定制品类宫格顺序(例如骑手客群把电动车排第一,学生客群把手机排第一)。

## 5. 客群命中计算

### 5.1 触发时机

- 用户登录 → 计算一次客群归属(写入缓存,1 小时刷新)
- 每日凌晨 2 点 → 全量批跑(更新所有用户的客群归属)
- 关键行为触发(下单成功 / 实名通过 / 学生认证)→ 即时重算

### 5.2 数据表

```sql
user_segment_membership
- id
- user_id
- segment_id
- matched_at        -- 命中时间
- expires_at        -- 失效时间(配合实时性)
- source            -- batch / login / event
- created_at
- updated_at
```

### 5.3 命中多个客群

一个用户可以同时属于多个客群(例如同时是"新客"+"通勤白领")。
绑定内容时按各内容自身的 `sort_order` 决定最终展示。

V1 提供基础数据(详细看板 V2 做):

|---|---|

## 7. 隐私与合规

- 客群标签**不向用户暴露**(用户看不到自己被归到哪个客群)
- 学生 / 骑手等敏感标签需要用户**主动认证**才能命中,不可单纯通过 GPS 推断
- 客群规则不涉及种族 / 宗教 / 政治倾向
- 客群计算用到的数据(GPS / 订单)按 `personal_info_handling_policy` 处理
