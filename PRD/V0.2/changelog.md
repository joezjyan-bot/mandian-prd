## 2026-05-28:V0.2.2 PRD 全量整改

**背景**:基于多角色审计和老板补充确认,对 V0.2.1 开发冻结版继续收口。V0.2.2 不修改数据库字段、API 路径、英文枚举或业务代码,只统一 PRD、流程、状态、财务、权限、OpenAPI 文档、数据模型和测试验收口径。

### V0.2.2 当前执行口径

- 一期范围明确为:安心用长租 + 体验租短租 V1 + IM 客服基础能力。
- 前端、客服、商家端、合同说明、财务报表统一展示为:商家订单 / 联营订单 / 平台订单;技术内部枚举可继续保留兼容命名。
- 结算触发条件统一为:客户签收确认 + 监管锁显示已上锁 + 激活锁状态已激活。仅苹果手机等配置为需监管锁回调的品类强制走锁状态校验;其它无锁品类按签收确认 + 设备识别码归档 + 交付证据完整 + 前置检测完成结算。
- 退款责任统一为:商家订单由商家承担;联营订单 / 平台订单线下协商方案;系统支持指定退款金额并记录责任方、审批人、凭证、冲正对象和流水。
- 财务口径升级为四账分离:订单业务账、支付流水账、钱包 / 商家账户账、财务总账 / 分录账。
- 服务费默认 2% 且可配置;短租商家自营订单客户缴费后分账给对应商家并扣服务费,平台自营短租不进入商家钱包。
- C 端禁止暴露资方、资金来源、合作模式、风控结论、内部审核备注、商家结算价、平台利润、返点规则等内部字段。
- 高危资金和状态操作拆细到角色权限、二次确认、二人复核和审计日志。
- 催收系统暂不自建,后续通过第三方系统 API 对接;系统保留催收记录和法诉证据包导出能力。
