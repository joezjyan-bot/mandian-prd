# 商品选择器与办单助手同步字段

> 页面级 PRD 草案。
> 目标：明确运营端、商家端、门店手机端办单助手选择商品时需要同步哪些字段，避免商品库和办单助手价格方案各算各的。

> **⚠️ V0.2 Peer Review 修订(2026-05-27)v1.1**:
> - 办单助手同步“设备指导价区间”,仅用于设备价校验。
> - 指导价规则详见 `04_商品指导价管理.md`。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商品选择器与办单助手同步字段 |
| 所属端 | 运营端、商家端、门店手机端 |
| 入口路径 | 商品管理、办单助手、审核改价 |
| 使用角色 | 商品运营、商家老板、审核客服、门店员工 |
| 核心目标 | 统一商品选择、规格、长租办单、增值服务、设备识别码和二维码下单字段 |

---

## 2. 核心口径

1. 同一商品可以同时有全新、二手等规格，不需要重复建两个商品。
2. 商品可配置是否同步办单助手。
3. 门店订单使用商家自己的商品和配置；分红订单、平台订单使用运营端配置或平台同步商品。
4. 商品同步到办单助手时，第一版只同步长租可用规格、价格参数和增值服务范围。
5. 客户扫码下单后，价格快照必须带回后台订单。
6. 短租不在当前三个办单助手入口展示，也不在本版继续细化；后续由完整短租需求包嫁接进来。

---

## 3. 商品选择器字段

| 字段 | 说明 |
|---|---|
| 商品名称 | 搜索展示 |
| 类目 | 手机、电动车、电脑等 |
| 品牌/型号 | 可选 |
| 规格 | 全新、二手、容量、颜色、电池等 |
| 租赁模式 | 当前同步长租；短租后续独立需求包接入 |
| 计费单位 | 当前按期/月；短租计费单位后续需求包定义 |
| 是否要求设备识别码 | 长租交付/发货时填写；短租后续需求包定义库存设备 |
| 是否同步办单助手 | 开关 |
| 可用订单类型 | 门店订单、分红订单、平台订单 |
| 增值服务 | 可选服务范围 |

---

## 4. 同步字段

| 字段组 | 字段 |
|---|---|
| 商品基础 | 商品 ID、名称、类目、图片、规格 |
| 租赁配置 | 长租可选租期/套餐；短租配置后续需求包接入 |
| 价格配置 | 设备价、首期、后期账单、押金、服务费、公证费、留购价 |
| 指导价校验 | 设备指导价下限、设备指导价上限、超上限处理规则 |
| 分红配置 | 可选配资比例、默认比例、资方范围 |
| 增值服务 | 服务项、金额、是否必选、是否可改 |
| 设备识别 | 长租是否要求发货/交付填写设备识别码；短租后续库存状态和可用门店 |
| 链路配置 | 合同、公证、支付、发货、审核策略版本 |

---

## 5. 办单助手回传字段

客户扫码下单后回传：

| 字段 | 说明 |
|---|---|
| 商品和规格快照 | 下单时展示内容 |
| 订单类型 | 门店订单、分红订单、平台订单 |
| 租赁方案 | 长租租期/套餐、起止时间 |
| 费用明细 | 首期、后期账单、押金、服务费、公证费、留购价 |
| 设备价校验快照 | 指导价区间、是否超上限、确认人和确认时间 |
| 增值服务 | 客户选择或默认项 |
| 配资比例 | 分红订单内部字段，客户侧不展示 |
| 设备识别码 | 长租发货/交付时回传 IMEI/SN/VIN；短租后续需求包单独处理库存设备 |
| 办单员工 | 员工账号 |
| 二维码来源 | 办单助手、商品二维码 |

---

## 6. 审核改价联动

审核客服改价时必须基于办单助手快照生成新版本。新版本要同步账单、合同变量、分账和客户确认待办。

---

## 7. 异常

| 场景 | 处理 |
|---|---|
| 商品下架 | 历史订单保留快照，新办单不可选 |
| 规格停用 | 已生成二维码可按配置失效 |
| 增值服务变更 | 历史订单不变，新订单用新版本 |
| 设备识别码缺失 | 长租可先生成报价，但发货/交付前必须填写 IMEI/SN/VIN |
| 配置缺失 | 办单助手不展示该商品或规格 |
