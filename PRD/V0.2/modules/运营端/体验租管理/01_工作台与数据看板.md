# 体验租管理 · 01 工作台与数据看板

> 运营人员每天进入「体验租管理」的第一屏。集中展示**待办任务、核心指标、预警告警**,让运营人员能在一屏内判断今日工作优先级。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 体验租工作台 |
| 入口路径 | 运营端 > 体验租管理 > 工作台与数据看板(默认页) |
| 使用角色 | 体验租运营、体验租运营主管、客服、平台管理员 |
| 核心目标 | 让运营一屏掌握当日待办 + 关键指标 + 异常预警 |
| 刷新频率 | 待办与预警实时;指标按时间窗口聚合(默认 5 分钟刷新) |

## 3. 待办卡片(顶部 5 个)

每个卡片显示:**类型 + 待办数量 + 点击进入对应处理列表**。

| 卡片 | 数据源 | 跳转目标 | 角色权限 |
|---|---|---|---|
| 待接单审(代接单)| 短租订单 status=SR_PENDING_CONFIRM 且 SLA 剩余 < 15 分钟 | 02 订单管理 > 筛选"待接单超时风险" | 体验租运营 |
| 押金大额审批 | short_rent_deposit_deduction status=pending_review 且 level=large | 06 押金管理 > 大额待审 | 体验租运营主管 |
| 违约金免除待审 | short_rent_overdue_event 客户申请免除待审 | 10 逾期与违约金 > 免除审批 | 体验租运营主管 |
| 长时间逾期 | short_rent_order status=SR_OVERDUE 且 逾期 > 24 小时 | 10 逾期与违约金 > 长时间逾期 | 体验租运营主管 |
| 控车异常工单 | short_rent_control_exception_ticket status=pending | 07 控车与监控 > 异常工单 | 体验租运营 |

**优先级标识**:
- 数字 > 0 时显示红色,数字 = 0 时显示灰色
- 鼠标悬浮显示明细预览(最近 3 条)

## 4. 核心指标(中部)

时间窗口可切:今日 / 本周 / 本月 / 自定义。

### 4.1 指标定义

| 指标 | 计算口径 | 同比对比 |
|---|---|---|
| 订单量 | 时间窗口内 short_rent_order 创建数(含已取消)| 与上一周期对比 |
| GMV | 时间窗口内 short_rent_order 实际成交租金合计(含押金不计)| 同上 |
| 接单率 | 时间窗口内 SLA 达标接单数 / 总接单需求数 | 同上 |
| 平均接单时长 | 时间窗口内 SR_PENDING_CONFIRM → SR_PENDING_PICKUP 平均耗时(分钟)| 同上 |
| 逾期率 | 时间窗口内进入 SR_OVERDUE 状态订单数 / 进入 SR_IN_USE 订单数 | 同上 |
| 押金扣款率 | 时间窗口内发生扣款的订单数 / 已完成订单数 | 同上 |
| 续租率 | 时间窗口内发生续租的链数 / 完成链数 | 同上 |
| 客诉率 | 时间窗口内 SR_DISPUTE 状态订单数 / 总订单数 | 同上 |
| 控车异常率 | 时间窗口内 CONTROL_FAIL + CONTROL_TIMEOUT / 总控车操作数 | 同上 |

### 4.2 同比对比颜色规则

| 指标性质 | 上升 | 下降 |
|---|---|---|
| 正向指标(订单量/GMV/接单率/续租率)| 绿色 ↑ | 红色 ↓ |
| 负向指标(逾期率/扣款率/客诉率/控车异常率)| 红色 ↑ | 绿色 ↓ |

## 5. 预警告警(实时)

实时滚动展示需要立即关注的事项。

### 5.1 预警类型

| 类型 | 触发条件 | 等级 |
|---|---|---|
| SLA 接单超时 | 订单 SR_PENDING_CONFIRM 且超过 SLA 时长 80%(默认 48 分钟)| 中 |
| SLA 接单超时严重 | 已超过 SLA 时长(默认 60 分钟)| 高 |
| 押金多次申诉 | 同一订单 30 天内 ≥ 3 次申诉 | 中 |
| 商家扣款率异常 | 商家 7 天扣款率 > 30% | 中 |
| 商家客诉率异常 | 商家 7 天客诉率 > 10% | 高 |
| 中控台 API 异常 | 近 5 分钟控车失败率 > 10% | 高 |
| 长时间逾期新增 | 新出现逾期 > 24 小时的订单 | 高 |
| 押金大额待审堆积 | pending_review 状态超过 4 小时未处理 | 中 |
| 单门店库存预警 | 门店可租车型 < 20% 总库存 | 低 |

### 5.2 预警卡片格式

```text
[图标] [等级颜色条] [预警内容] [发生时间] [操作按钮]
```

操作按钮:**查看**(跳转详情)/ **认领**(标记自己处理中)/ **忽略**(标记为已知)

## 7. 筛选与个性化

### 7.1 全局筛选

| 筛选项 | 说明 |
|---|---|
| 时间窗口 | 今日 / 本周 / 本月 / 自定义日期段 |
| 城市 | 多选;默认全部 |
| 业务模式 | store_self_operated / platform_self_operated / platform_asset_store_fulfillment / 全部 |
| 商家 | 多选;搜索 |
| 门店 | 多选;搜索 |

### 7.2 个性化(V2)

- 待办卡片可拖拽顺序
- 指标卡片可选择关注哪几个
- 预警可订阅特定类型推送(短信/邮件/IM)

V1 不开放个性化,所有用户看到的工作台一致。

## 8. 数据接口

| 接口 | 用途 | 缓存 |
|---|---|---|
| `GET /api/v1/admin/exp-rent/workbench/todos` | 待办卡片数据 | 30 秒 |
| `GET /api/v1/admin/exp-rent/workbench/metrics?range=today` | 核心指标 | 5 分钟 |
| `GET /api/v1/admin/exp-rent/workbench/alerts` | 预警实时数据 | 不缓存(WebSocket 推送)|
| `GET /api/v1/admin/exp-rent/workbench/trends?days=7` | 趋势图 | 1 小时 |

**性能要求**:整页首屏加载 < 2 秒;关键指标查询 < 500ms(走数据看板服务,不直接打业务库)。
