# 运营端链路配置中心

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目的：把支付、合同、风控、发货、分账、办单助手同步等关键链路做成可配置、可继承、可回滚、可追溯的规则中心，避免后续每换一个通道或规则都重新改接口。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 链路配置中心 |
| 所属端 | 运营端 |
| 入口路径 | 配置管理 > 链路配置中心 |
| 使用角色 | 平台管理员、运营配置人员、风控负责人、财务负责人、技术支持 |
| 核心目标 | 按平台默认、订单类型、商户覆盖、商品覆盖四层配置订单全链路规则 |
| 不开放给 | 普通商家员工、门店员工、渠道账号、C 端用户 |

## 2. 核心口径

链路配置中心只管理“平台级和平台主控链路”，不是让商家随意改平台规则。

| 层级 | 维护端 | 说明 |
|---|---|---|
| 平台默认 | 运营端 | 全平台兜底规则，例如默认发货主体、默认合同服务、默认风控 |
| 订单类型覆盖 | 运营端 | 商家订单、联营订单、平台订单分别走不同审核、合同、支付、分账链路 |
| 商户覆盖 | 运营端为主 | 针对单个商户设置特殊链路，例如抽佣、合同模板、发货地址、可用订单类型 |
| 商品/规格覆盖 | 运营端/商家端按权限 | 控制商品是否同步办单助手、适配长租/短租、可用增值服务 |

继承优先级：

```text
商品/规格覆盖 > 商户覆盖 > 订单类型覆盖 > 平台默认
```

每个配置项必须展示当前来源：

- 使用平台默认
- 使用订单类型配置
- 使用商户覆盖
- 使用商品覆盖

## 4. 顶部筛选与对象选择

| 控件 | 类型 | 说明 |
|---|---|---|
| 配置对象 | 下拉 | 平台默认、商家订单、联营订单、平台订单、指定商户、指定商品规格 |
| 商户 | 搜索选择 | 选择指定商户时必填 |
| 商品/规格 | 搜索选择 | 选择指定商品规格时必填 |
| 配置域 | 多选 | 订单、支付、合同、风控、接口计费、发货、财务、办单助手、回调 |
| 状态 | 下拉 | 草稿、待发布、已启用、已停用、已回滚 |
| 最近修改人 | 搜索 | 按操作人筛选 |
| 最近修改时间 | 日期范围 | 查找配置变更 |

规则：

1. 进入页面默认展示 `平台默认` 配置。
2. 选择 `商家订单 / 联营订单 / 平台订单` 时，展示该订单类型的覆盖规则。
3. 选择指定商户时，必须能看到它继承了哪些平台规则、覆盖了哪些规则。
4. 选择指定商品规格时，只展示商品相关链路，例如办单助手同步、租赁模式、增值服务、短租库存。

## 5. 配置域

### 5.1 订单链路配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 是否启用该订单类型 | 开关 | 控制前端和办单助手是否展示 |
| 审核主体 | 下拉 | 门店/商家、运营平台 |
| 是否需要平台审核 | 开关 | 商家订单默认否，联营/平台订单默认是 |
| 是否允许客服改价 | 开关 | 联营/平台订单默认是，商家订单按商家权限 |
| 是否允许分配资方 | 开关 | 联营/平台订单是，商家订单否 |
| 是否允许联系客服 | 开关 | 平台订单默认是，待审核列表可调起 IM |
| 是否允许补资料 | 开关 | 审核不完整时可退回补资料 |
| 订单关闭策略 | 配置 | 超时未支付、审核拒绝、客户取消、风控拒绝 |

默认建议：

| 订单类型 | 审核主体 | 平台审核 | 资方分配 | 客服改价 |
|---|---|---|---|---|
| 商家订单 | 门店/商家 | 否 | 否 | 商家可配置 |
| 联营订单 | 运营平台 | 是 | 是 | 是 |
| 平台订单 | 运营平台 | 是 | 是 | 是 |

### 5.2 支付代扣配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 下单后是否唤起支付 | 开关 | 控制客户扫码下单后是否立即支付 |
| 待付款支付拦截 | 开关 | 是否需要审核/确认后才能支付 |
| 支付前置条件 | 多选 | 实名、合同、授权、风控资料、审核通过 |
| 支付渠道 | 多选 | 支付宝小程序、微信小程序、银行卡、线下支付 |
| 代扣方式 | 多选 | 租赁行业代扣、直付通、银联/银行卡代扣 |
| 代扣费率承担方 | 下拉 | 平台、商户、资方、按订单类型 |
| 支付失败处理 | 下拉 | 待跟进、重新发起、关闭订单、人工处理 |
| 是否允许线下收款 | 开关 | 仅特定角色可用，必须上传凭证 |

规则：

1. 商家订单可按商户配置支付链路，但平台保留监管和日志。
2. 联营订单和平台订单默认由运营端主控支付和代扣。
3. 支付、代扣、退款、线下收款都必须进入财务对账。

### 5.3 合同签署配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 是否启用合同 | 开关 | 控制该链路是否需要签署合同 |
| 合同服务商 | 下拉 | e 签宝、其他电子签服务、无合同 |
| 合同模板 | 选择器 | 商家订单模板、联营订单模板、平台订单模板 |
| 是否自动发起 | 开关 | 审核通过后是否自动发起合同 |
| 是否允许人工发起 | 开关 | 审核客服可手动发起 |
| 是否启用补充合同 | 开关 | 订单详情可发起补充合同 |
| 签署方式 | 多选 | 短信、扫码、H5、支付宝/微信授权 |
| 签署失败处理 | 下拉 | 重发、人工处理、退回待签约 |
| 商户签章授权要求 | 下拉 | 必须完成、可后补、不需要 |

订单类型口径：

| 订单类型 | 合同主控 |
|---|---|
| 商家订单 | 商家可选启用商家模板或平台模板，平台不默认成为出租方 |
| 联营订单 | 运营端主控，需结合资方和分账协议 |
| 平台订单 | 运营端主控，需结合资方/平台主体和客户签约 |

### 5.4 风控授权配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 是否启用黑名单匹配 | 开关 | 下单或审核时匹配脱敏黑名单 |
| 是否启用实名校验 | 开关 | 二要素/三要素/人脸等 |
| 是否启用风控报告 | 开关 | 可对接中控台条数消耗 |
| 是否启用流水报告 | 开关 | 审核客服可发起 |
| 是否启用信用/信息授权签署 | 开关 | 客户授权后再调用 |
| 风控触发节点 | 多选 | 下单、实名后、提交审核、人工发起 |
| 命中处理 | 下拉 | 弹窗提示、强制严格审核、自动拒绝、人工复核 |
| 报告有效期 | 数字 | 过期后需重新发起 |

规则：

1. 联营订单和平台订单默认启用平台风控。
2. 商家订单可由商家选择是否购买/启用平台风控服务。
3. 中控台仍是租赁系统外部系统，只通过接口调用、条数、报告状态打通。

### 5.5 接口计费配置

接口计费配置用于控制商家/门店调用平台能力时的费用归属，详细页面见 `03_商家接口计费配置.md`。

| 字段 | 类型 | 说明 |
|---|---|---|
| 是否启用接口计费 | 开关 | 控制当前对象是否启用接口计费 |
| 适用接口 | 多选 | 实名、人脸、OCR、大数据报告、风控报告、合同、公证、短信、收款码 |
| 默认费用承担方 | 下拉 | 商家、平台、资方、客户、按订单类型 |
| 扣费账户 | 下拉 | 商家钱包、平台成本账户、资方账户 |
| 余额不足处理 | 下拉 | 禁止调用、进入审批、平台垫付、仅提醒 |
| 允许欠费额度 | 金额 | 可按商家覆盖 |
| 调用统计 | 开关 | 是否进入接口调用统计和商家账单 |

规则：

1. 商家订单默认允许向商家计费，尤其是大数据、风控、合同、公证等平台能力。
2. 联营订单、平台订单的接口费用承担方由运营端按订单类型配置。
3. 平台人员代商家发起接口调用时，必须确认费用承担方并写入订单日志。
4. 每次接口调用都要生成调用流水，扣费成功后生成钱包变动。

### 5.6 发货履约配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 默认发货主体 | 下拉 | 门店、商家、平台、资方 |
| 可选发货方式 | 多选 | 门店自提、门店配送、快递发货、顺丰线上发货 |
| 发货地址来源 | 多选 | 平台默认、商户默认、门店默认、订单覆盖 |
| 退货地址来源 | 多选 | 平台默认、商户默认、门店默认、订单覆盖 |
| 是否需要发货凭证 | 开关 | 图片/视频凭证 |
| 是否启用发货提醒 | 开关 | 按订单状态和时效提醒 |
| 是否允许线上快递接口 | 开关 | 顺丰等第三方接口 |
| 是否要求设备识别码 | 开关 | 长租发货/交付必填；短租后续按库存设备要求配置 |

默认建议：

1. 全平台默认 `门店发货`。
2. 平台订单可配置为门店发货、平台发货或资方发货。
3. 短租交付必须和设备库存打通，设备码进入订单详情。
4. 发货凭证、签收凭证、归还验收凭证都要进入订单详情。

### 5.7 财务分账配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 平台抽佣比例 | 数字 | 默认 2%，可按订单类型/商户覆盖 |
| 抽佣顺序 | 固定/下拉 | 联营订单默认先按比例分账，再分别扣抽佣 |
| 商家订单入账规则 | 配置 | 平台抽佣后进入门店钱包 |
| 联营订单分账规则 | 配置 | 门店/资方按出资比例分账 |
| 平台订单分账规则 | 配置 | 资方主收益、平台抽佣、渠道佣金 |
| 渠道佣金是否启用 | 开关 | 仅统计联营订单和平台订单 |
| 提现审核规则 | 配置 | 商家、渠道、资方提现流程 |
| 异常处理队列 | 开关 | 分账失败、打款失败、退款失败进入异常队列 |

规则：

1. 每笔收款都必须生成账单、分账记录、钱包变动记录。
2. 每笔钱包变动必须能追溯到订单、账单、期数、规则版本。
3. 普通门店端不保留资金充值入口。
4. 资方授信、出资、回款和提现账户如需保留，先放在运营端资方管理，资方端后置。

### 5.8 办单助手同步配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 是否同步办单助手 | 开关 | 商品/规格维度 |
| 同步到哪个入口 | 多选 | 商家订单、联营订单、平台订单 |
| 数据来源 | 只读 | 商家配置/运营配置 |
| 可用租赁模式 | 多选 | 长租、短租 |
| 可用计费单位 | 多选 | 小时、天、周、月 |
| 可用增值服务 | 多选 | 平台或商家配置的增值服务 |
| 是否要求价格完整 | 开关 | 无价格不允许同步 |
| 是否要求库存/设备可用 | 开关 | 短租无设备不允许生成可下单二维码 |

数据来源规则：

| 助手入口 | 商品/费率/增值服务来源 |
|---|---|
| 商家订单 | 商家自己的商品、规格、费率、增值服务 |
| 联营订单 | 运营平台配置的商品、费率、资方规则、平台增值服务 |
| 平台订单 | 运营平台配置的商品、费率、平台增值服务 |

### 5.9 回调与日志配置

| 字段 | 类型 | 说明 |
|---|---|---|
| 回调类型 | 多选 | 支付、合同、风控、物流、监管锁、分账、投诉 |
| 回调失败重试 | 配置 | 次数、间隔、是否进入异常队列 |
| 是否记录请求摘要 | 开关 | 不记录敏感明文 |
| 是否记录响应摘要 | 开关 | 不记录敏感明文 |
| 日志保留周期 | 数字 | 按合规要求配置 |
| 异常通知对象 | 多选 | 技术、运营、财务、风控 |

规则：

1. 所有外部接口回调都要有唯一回调流水号。
2. 回调不能直接覆盖人工结果，冲突时进入异常队列。
3. 敏感信息只保留脱敏摘要。

## 6. 发布、版本和回滚

链路配置不能点保存就直接影响线上，必须有发布流程。

```mermaid
flowchart LR
    A["编辑配置"] --> B["保存草稿"]
    B --> C["校验规则"]
    C --> D{"校验通过"}
    D -->|否| E["提示冲突/缺失"]
    D -->|是| F["发布确认"]
    F --> G["生成配置版本"]
    G --> H["按生效时间启用"]
    H --> I["写入操作日志"]
    I --> J["可查看历史/回滚"]
```

发布规则：

1. 配置必须先保存草稿。
2. 发布前校验必填项、冲突项和影响范围。
3. 发布时必须填写变更说明。
4. 支持立即生效和指定时间生效。
5. 已生效配置生成不可变版本。
6. 回滚只能回滚到历史版本，并产生新的版本号。

## 7. 配置校验规则

| 场景 | 校验 |
|---|---|
| 联营订单关闭资方分配 | 不允许，联营订单必须有资方分配能力 |
| 平台订单关闭平台审核 | 不允许，平台订单必须运营审核 |
| 商家订单强制平台合同 | 允许但需二次确认，避免平台错误成为出租方 |
| 启用短租但关闭设备绑定 | 高风险提示，正式短租模式不允许 |
| 启用合同但无合同模板 | 不允许发布 |
| 启用支付但无支付渠道 | 不允许发布 |
| 启用渠道佣金但无佣金规则 | 不允许发布 |
| 启用风控报告但中控台不可用 | 允许保存草稿，不允许发布 |
| 发货主体为资方但订单无资方 | 不允许进入发货 |

## 8. 权限控制

| 权限 | 可操作内容 |
|---|---|
| 平台超级管理员 | 查看、编辑、发布、回滚全部配置 |
| 运营配置人员 | 编辑订单、合同、发货、办单助手配置，可提交发布 |
| 财务负责人 | 编辑和发布财务分账、提现、抽佣配置 |
| 风控负责人 | 编辑和发布风控、黑名单、报告配置 |
| 技术支持 | 查看回调、接口、异常队列，不默认允许改业务规则 |
| 商家老板 | 仅可查看自己商户已生效配置，部分商家订单低风险配置可在商家端维护 |
| 门店员工 | 不可见链路配置中心 |

敏感操作必须二次验证：

- 发布配置
- 回滚配置
- 修改抽佣比例
- 修改分账规则
- 修改支付/代扣通道
- 修改合同服务商或模板
- 修改发货主体
- 启用/停用风控

|---|---|
| 商品管理 | 读取办单助手同步规则、租赁模式、增值服务配置 |
| 办单助手 | 读取订单类型、商品、费率、增值服务、支付前置条件 |
| 订单管理 | 读取审核、合同、支付、发货、分账规则；写入链路执行结果 |
| 店铺管理 | 读取商户覆盖配置、应用权限、签章授权状态 |
| 资方管理 | 读取资方分配规则、平台订单/联营订单资金规则 |
| 财务管理 | 读取抽佣、分账、提现规则；写入分账与钱包流水 |
| 渠道管理 | 读取渠道佣金规则；写入推广订单和佣金明细 |
| 客诉管理 | 读取投诉处理链路，写入投诉状态和处理日志 |
| 中控台 | 作为外部系统提供风控/报告调用能力，回写调用状态和条数消耗 |
| 操作日志 | 记录配置变更、发布、回滚、回调、异常处理 |

## 10. 失败态与异常

| 场景 | 页面处理 |
|---|---|
| 配置缺必填项 | 阻止发布，定位到缺失字段 |
| 配置互相冲突 | 阻止发布，列出冲突规则 |
| 外部通道未配置 | 阻止发布或仅允许保存草稿 |
| 回调地址不可用 | 阻止发布，需要技术确认 |
| 发布后执行失败 | 自动进入异常队列，保留旧版本兜底 |
| 配置被其他人修改 | 提示版本已变化，要求刷新后再编辑 |
| 回滚失败 | 保留当前版本，记录失败原因 |
| 无权限访问 | 隐藏菜单或提示无权限 |

## 11. V1 和后续边界

V1 先做最关键的配置闭环：

1. 平台默认配置。
2. 三类订单的订单链路配置。
3. 支付代扣、合同签署、风控授权、发货履约、财务分账基础配置。
4. 商户覆盖配置的查看和运营端维护。
5. 配置版本、发布、回滚、操作日志。

V1 可先不做：

1. 商家端自助修改平台链路。
2. 复杂 A/B 链路实验。
3. 多级渠道链路覆盖。
4. 资方端自助配置。
5. 短租完整仓库自动化，只先保留链路配置项和字段。
