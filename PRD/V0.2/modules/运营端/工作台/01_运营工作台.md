# 运营工作台

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：把运营端首页做成真实待办工作台，聚合订单、店铺、商品、财务、客诉、租后、链路异常，不做只展示数字的空首页。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 运营工作台 |
| 所属端 | 运营端 |
| 入口路径 | 首页 / 工作台 |
| 使用角色 | 平台管理员、审核客服、订单客服、财务、租后客服、商品运营、客服主管 |
| 核心目标 | 让不同角色进入系统后直接看到自己要处理的待办、异常、预警和关键数据 |

工作台不是数据大屏，也不是营销首页。它的目标是减少漏单、漏审核、漏回调、漏投诉、漏打款。

---

## 2. 核心口径

1. 工作台按角色展示，不同角色看到的卡片和待办不同。
2. 工作台所有数字必须能点击进入对应列表，不能只展示不可追溯指标。
3. 订单相关待办按三类订单拆分，不能把商家订单、联营订单、平台订单混在一起。
4. 待办来源必须来自真实业务状态、回调状态、超时规则、人工分配，不允许手工虚报。
5. 钱相关待办只进入财务页面处理，工作台不直接执行最终打款、退款、确认到账。
6. 异常卡片优先级高于普通统计，避免回调失败、投诉超时、黑名单命中被埋掉。
7. 工作台要和权限系统联动，用户没有权限的模块不展示或只展示汇总不展示详情。

---

## 3. 页面结构

```text
运营工作台
├─ 顶部：日期、当前角色、快捷搜索、通知
├─ 我的待办：待审核、待分配、待补资料、待签约、待支付、待发货、待处理投诉
├─ 异常预警：回调失败、投诉临近超时、黑名单命中、支付异常、打款失败、库存异常
├─ 订单概览：商家订单 / 联营订单 / 平台订单
├─ 财务概览：待分账、待提现审核、打款失败、退款/冲正待处理
├─ 商家与商品：店铺待审核、商品待审核、复制商品待确认
├─ 租后概览：已逾期、催收跟进、外部催收同步、回款异常
└─ 最近操作：我的处理记录、系统回调记录、主管关注记录
```

## 4. 顶部快捷区

| 模块 | 说明 |
|---|---|
| 全局搜索 | 支持订单号、客户姓名、手机号后四位、商家、设备码 |
| 我的角色 | 展示当前账号角色和数据权限范围 |
| 快捷入口 | 新建商品、待审核订单、店铺审核、客诉、财务审核 |
| 通知 | 待办提醒、系统异常、主管分配、接口失败 |

搜索结果应按类型分组：订单、商家、客户、设备、投诉、账单。

## 5. 我的待办

### 5.1 待办卡片

| 待办 | 来源 | 点击后 |
|---|---|---|
| 待审核订单 | 联营订单、平台订单提交后未审核 | 进入待审核队列 |
| 待分配内部资金来源 | 联营/平台订单未绑定资方 | 进入内部资金来源分配队列 |
| 待补资料 | 审核要求客户或商家补资料 | 进入补资料队列 |
| 待客户授权 | 风控、合同、代扣、公证未完成 | 进入订单列表筛选 |
| 待支付 | 首期、服务费、公证费、补缴未完成 | 进入支付待办 |
| 待发货 | 审核链路完成但未发货 | 进入待发货列表 |
| 待处理客诉 | 新投诉、待回复、临近超时投诉 | 进入客诉管理 |
| 待财务审核 | 提现、退款、冲正、打款异常 | 进入财务待办 |

### 5.2 三类订单拆分

| 订单类型 | 工作台展示重点 |
|---|---|
| 商家订单 | 商家自审待办、平台抽佣异常、客诉、租后、发货证据缺失 |
| 联营订单 | 待审核、待分配内部资金来源、待合同、待支付、分账异常 |
| 平台订单 | 待审核、待内部资金来源、渠道来源、客服 IM、内部资金来源回款、发货主体 |

## 6. 异常预警

| 异常 | 触发条件 | 处理入口 |
|---|---|---|
| 回调失败 | 支付、合同、公证、风控、分账、物流回调失败 | 回调日志/订单详情 |
| 投诉超时 | 支付宝投诉临近回复截止或已超时 | 客诉管理 |
| 疑似黑名单 | 客户命中内部或外部黑名单 | 订单审核/黑名单详情 |
| 支付异常 | 支付金额不匹配、部分支付未入账、代扣失败 | 财务/订单详情 |
| 打款失败 | 提现或分账通道失败 | 财务打款异常 |
| 退款冲正异常 | 退款失败、重复退款、分账已发生需冲正 | 退款冲正页 |
| 库存异常 | 短租设备被重复锁定、设备码缺失 | 库存设备管理 |
| 发货证据缺失 | 已发货但未上传人车合照/签收照片 | 订单交付证据 |

异常预警必须支持按严重程度排序：高风险、临近超时、普通异常。

## 7. 订单概览

### 7.1 指标

| 指标 | 说明 |
|---|---|
| 今日新增订单 | 按订单类型拆分 |
| 待审核 | 仅统计当前角色可处理订单 |
| 待分配内部资金来源 | 联营订单、平台订单 |
| 待签约/待支付 | 客户侧未完成节点 |
| 待发货/待签收 | 默认门店发货，按发货主体拆分 |
| 在租订单 | 当前正常履约订单 |
| 已逾期 | 已超过应还/应付节点 |
| 今日到期/三日到期 | 租后提醒 |

### 7.2 点击规则

每个指标点击后进入订单列表，并自动带上筛选条件：

```text
订单类型 + 状态 + 时间范围 + 当前用户权限
```

例如点击「联营订单 / 待分配内部资金来源」进入订单管理 > 联营订单，并筛选内部资金来源状态 = 未分配。

## 8. 财务概览

| 模块 | 展示内容 |
|---|---|
| 待分账 | 已收款但未完成分账的订单 |
| 待提现审核 | 商家、渠道、资方提现申请 |
| 打款失败 | 通道返回失败、账户异常、需复核 |
| 退款待处理 | 客户退款申请、订单关闭退款 |
| 冲正待处理 | 已分账后退款、错误入账、佣金扣回 |
| 对账异常 | 支付/分账/提现通道与系统账不一致 |

财务概览只做入口和状态展示，不在工作台执行打款、退款、冲正确认。

## 9. 商家与商品

| 模块 | 展示内容 |
|---|---|
| 店铺待审核 | 新入驻、资料补充、收款账户待确认、e 签宝授权未完成 |
| 商品待审核 | 商家新增商品、修改商品、复制商品确认 |
| 商品同步异常 | 同步办单助手失败、规格缺失、增值服务冲突 |
| 平台复制待确认 | 平台复制给商家但商家未确认 |
| 授权状态异常 | 商家合同授权、收款账户实名、授权签章未完成 |

## 10. 客诉与租后

### 10.1 客诉

| 指标 | 说明 |
|---|---|
| 新投诉 | 新同步的支付宝投诉 |
| 待回复 | 已分配但未回复 |
| 临近超时 | 距截止时间小于配置阈值 |
| 已超时 | 超过平台或支付宝要求时限 |
| 高风险投诉 | 标记主管关注、涉及退款/扣款争议 |

### 10.2 租后

| 指标 | 说明 |
|---|---|
| 今日到期 | 今天应还/应付 |
| 三日到期 | 未来三天到期 |
| 已逾期 | 按逾期天数分层 |
| 催收跟进 | 有待跟进记录 |
| 外部催收同步异常 | 推送外部催收系统失败 |

## 11. 最近操作

| 内容 | 说明 |
|---|---|
| 我的审核记录 | 最近通过、拒绝、补资料、分配内部资金来源 |
| 我的财务处理 | 退款、冲正、提现审核的非敏感摘要 |
| 我的客诉处理 | 最近回复、备注、分配 |
| 系统回调 | 与自己处理订单相关的最新回调 |
| 主管关注 | 被主管标记或分配给我的事项 |

最近操作用于快速回到刚处理过的业务，不能展示敏感明文。

## 12. 权限与个性化

| 角色 | 默认工作台 |
|---|---|
| 审核客服 | 待审核、待分配内部资金来源、待补资料、风控异常 |
| 财务 | 待分账、提现审核、退款冲正、打款失败、对账异常 |
| 客服 | 客诉、订单备注、客户待办、发货签收 |
| 租后客服 | 已逾期、催收跟进、外部同步、回款异常 |
| 商品运营 | 商品审核、商品同步、复制商品、增值服务异常 |
| 管理员 | 全部模块汇总和异常预警 |

用户可以保存常用卡片排序，但不能看到超出权限的数据。
