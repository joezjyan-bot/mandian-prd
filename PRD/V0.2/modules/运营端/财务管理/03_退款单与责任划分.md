# 退款单与责任划分

> - 同步展示用语:商家订单 / 联营订单 / 平台订单 / 履约中 / 逾期费用。
> - 底层字段、接口、枚举不变;资金账户、退款工单、五边对账以 Stage 6 核心财务文档为准。

> **来源**: P0-3
> **核心**: "分账有出,退款必入";责任方钱包必须充值到位

## 1. 页面定位

运营端 · 财务管理 · 退款单列表与详情。客服发起退款、填责任划分、等责任方钱包充值、执行退款。

## 2. 发起入口

- 订单详情 · "发起退款"
- 财务管理 · 退款单列表 · "新建"

## 3. 发起表单

### 3.1 基础
- 关联订单(锁定)
- 订单类型(锁定)
- 客户(锁定,脱敏)

### 3.2 退款项目明细(多行表格)

| 项目类型 | 扣款对象 | 金额 |
|---|---|---|
| 设备款 | store_wallet / funder_wallet | 手工 |
| 押金 | platform_deposit_account | 手工 |
| 租金某期 | 按比例拆 store/funder | 手工 |
| 平台抽佣 | platform_account | 手工 |
| 服务费/公证费 | 按原归属 | 手工 |
| 线下手续费冲回 | store/funder | 手工 |
| 补充说明 | (仅备注) | - |

总额 = 各项之和

### 3.3 责任方
下拉:门店 / 平台 / 资方 / 混合责任
混合需填拆分明细

### 3.4 客户补偿状态
7天内:是否需收1.5%关单费;门店是否已收

### 3.5 退款原因(必填下拉)
商品质量 / 审核失误 / 资方违约 / 客户单方面 / 混合 / 其他

### 3.6 原路退回通道
自动识别原通道(通联/直付通)

## 4. 提交后状态

| 情况 | 状态 |
|---|---|
| 责任方余额≥应扣 | pending |
| 责任方余额不足 | pending_topup |
| 1.5%补偿未到位 | pending_compensation |

### 4.1 已分账退款工单

如订单已经触发结算穿透、资方放款或门店提现冻结,退款单提交后必须同步生成 `refund_workflow` 工单:

| 工单状态 | 触发 |
|---|---|
| `pending` | 退款责任划分完成,等待责任方资金处理 |
| `merchant_paid` | 门店结算账户扣减成功或线下转账确认 |
| `refunded_customer` | 客户原路退款 / 人工退款完成 |
| `refunded_funder` | 资方台账还原 / 资方退款完成 |
| `completed` | 客户、资方、平台内部流水全部闭环 |

工单未完成前,订单关闭、监管锁、合同作废、重新下单等动作按业务锁规则限制。

## 5. 业务流程锁

退款未完成时该订单以下动作锁:
- 关闭订单
- 下监管锁
- 解锁设备
- 状态流到"已完成"

解锁:退款 status=success

## 6. 执行扣减

- 二次确认
- 各责任方钱包按明细扣
- 从平台押金账户退客户
- 调通道原路退回
- 状态:退款中

## 7. 通道回调

- 成功:已退款 + 解锁业务
- 失败:退款失败 + 人工处理

## 8. 退款单列表

筛选:订单号 / 状态 / 发起客服 / 责任方 / 金额范围 / 发起时间
字段:退款单号 / 关联订单 / 责任方 / 总金额 / 状态 / 发起人 / 时间 / 操作

## 9. 权限

- 运营管理员/财务主管:全能力
- 普通客服:可发起,V2 大额需主管审
- 商家/资方:不可发起后台退款单

## 10. 数据模型

见 `00_核心决策表.md` `refund_request` / `order_downstream_lock`。
