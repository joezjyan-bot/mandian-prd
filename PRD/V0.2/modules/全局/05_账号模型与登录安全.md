# 账号模型与登录安全

> 全局 PRD 草案。
> 目标：统一运营端、商家 PC 端、门店手机端、渠道端、C 端和中控台的账号主体、登录保持、员工权限、二次验证和敏感操作安全策略。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 账号模型与登录安全 |
| 所属端 | 全局基础能力 |
| 入口路径 | 系统设置 > 账号安全 / 权限管理 / 登录配置 |
| 使用角色 | 平台管理员、商家老板、渠道管理员、系统安全配置人员 |
| 核心目标 | 解决多端登录、员工账号、免登录时长、强制下线、二次验证和敏感权限 |

---

## 2. 账号主体

| 主体 | 说明 | 登录端 |
|---|---|---|
| 平台员工 | 运营、审核、财务、租后、客服、管理员 | 运营端 |
| 商家老板 | 入驻通过后的商家主账号 | 商家 PC 端、门店手机端 |
| 商家员工 | 老板创建的店员或店长 | 门店手机端、可选商家 PC |
| 渠道账号 | 渠道推广人员或团队负责人 | 渠道 H5 |
| C 端客户 | 扫码下单客户 | 小程序/H5/APP |
| 中控台账号 | 中控台管理员和客户管理角色 | 中控台 |

中控台是租赁系统外的系统，账号体系可打通但不能混为一个业务后台。

---

## 3. 商家与门店账号关系

1. 门店入驻审核通过后，生成商家主体、门店主体、商家老板账号。
2. 商家老板账号同时可登录商家 PC 端和门店手机端。
3. 老板可以在商家端分配员工账号。
4. 员工账号只能在授权门店范围内办单、处理交付、查看待办。
5. 员工账号默认不可见钱包、提现、分账、抽佣、接口扣费、收款账户。

---

## 4. 员工角色建议

| 角色 | 典型权限 |
|---|---|
| 店员 | 办单助手、扫码下单、交付照片、签收、归还验收 |
| 店长 | 店员权限 + 门店订单自审、设备库存查询、部分售后 |
| 财务管理员 | 钱包查看、提现申请、流水导出，需老板授权 |
| 商品管理员 | 商品新增、增值服务配置、办单助手同步 |
| 商家管理员 | 接近老板权限，但高风险动作仍需老板或二次验证 |

角色可以作为模板，实际权限通过权限矩阵配置。

---

## 5. 登录方式

| 端 | 登录方式 |
|---|---|
| 运营端 | 密码、验证码、扫码，按配置 |
| 商家 PC 端 | 手机号验证码、密码、扫码 |
| 门店手机端 | 手机号验证码、密码 |
| 员工账号 | 手机号 + 密码或验证码 |
| 渠道 H5 | 手机号验证码或密码 |
| C 端客户 | 小程序授权、手机号验证 |

无权限的登录方式不展示，登录失败必须提示明确但不能泄露账号是否存在的敏感信息。

---

## 6. 登录保持

| 场景 | 默认建议 | 说明 |
|---|---|---|
| 运营端 PC | 7 天 | 高风险后台，不建议过长 |
| 商家 PC 端 | 7 天 | 可配置 |
| 门店手机端老板账号 | 30 天 | 同一手机登录后减少频繁登录 |
| 门店手机端员工账号 | 7 天 | 老板可强制下线 |
| 渠道 H5 | 30 天 | 低频查看数据 |
| C 端客户 | 按小程序授权策略 | 支付、签约仍需平台能力校验 |

规则：

1. 登录保持绑定设备、端类型和账号。
2. 换设备、换浏览器、清缓存后需要重新登录。
3. 修改密码、停用账号、权限变更、店铺暂停后应强制相关端重新登录。
4. 高风险操作即使在免登录期内，也需要二次验证。

---

## 7. 二次验证

| 操作 | 是否需要 |
|---|---|
| 查看完整身份证、银行卡、收款账号 | 需要 |
| 修改收款账户 | 需要 |
| 提现申请或审核 | 需要 |
| 打款、退款、冲正 | 需要 |
| 修改抽佣、分账、资方规则 | 需要 |
| 暂停店铺、关闭订单、审核拒绝 | 可配置 |
| 员工创建和权限调整 | 需要 |

二次验证方式：短信验证码、登录密码、扫码确认、管理员复核，按端和角色配置。

---

## 8. 强制下线与设备管理

| 功能 | 说明 |
|---|---|
| 登录设备列表 | 展示账号最近登录设备、时间、端类型 |
| 单设备下线 | 强制某设备退出 |
| 全部下线 | 修改密码或风险处理 |
| 员工下线 | 老板可对员工执行 |
| 风险锁定 | 连续失败、异常地区、疑似盗号 |

设备列表不展示完整 IP 或精确定位给普通角色，只展示必要摘要。

---

## 9. 安全日志

必须记录：

| 日志 | 字段 |
|---|---|
| 登录成功/失败 | 账号、端、设备、时间、失败原因 |
| 验证码发送 | 手机号脱敏、用途、发送状态 |
| 权限变更 | 操作人、被操作人、变更前后 |
| 强制下线 | 操作人、原因、设备 |
| 二次验证 | 操作场景、验证结果 |
| 敏感查看 | 查看对象、字段类型、原因 |

安全日志不可由普通运营删除，只能归档。
