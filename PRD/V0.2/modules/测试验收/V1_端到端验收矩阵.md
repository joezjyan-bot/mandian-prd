# V1 端到端验收矩阵

> **V0.2.1 开发冻结同步(2026-05-27)**: 本文档验收安心用长租、体验租短租和 IM 客服基础能力。所有 C 端页面和通知不展示资方、资金来源、合作模式、风控字眼。

---

## 1. 验收范围

| 范围 | 是否进入 V1 | 说明 |
|---|---|---|
| 商家订单 | 是 | 商家/门店自主营销、自主审核、门店发货履约 |
| 联营订单 | 是 | 平台、商家、内部资金来源参与后台结算，客户侧不展示资金来源 |
| 平台订单 | 是 | 平台主导商品、审核、履约和结算 |
| 体验租/短租 | 是 | 独立入口、独立车辆库存、独立订单状态、独立取还车核销 |
| IM 客服基础能力 | 是 | 统一入口、上下文卡片、会话分配、转工单、归档 |
| 电子确认单签收 | 是 | V1 默认签收确认方案 |
| AI 问答签收视频 | 配置预留 | 只验收配置、状态、记录字段和异常入口，不验收真实 AI 能力 |

---

## 2. 商家订单验收路径

| 步骤 | 触发端 | 必验结果 |
|---|---|---|
| 1. 商家/门店办单 | 门店手机端/商家端 | 选择商品、规格、设备价、保证金、首期应付、订单周期，生成商家订单草稿 |
| 2. 客户扫码确认 | C 端 | 页面展示商品、每期应付、订单总额、保证金、订单周期，不展示合作模式/资金来源 |
| 3. 客户资料和实名 | C 端 | 完成资料、实名、人脸、信用/信息授权，状态可断点续传 |
| 4. 商家自主审核 | 商家端 | 商家老板/管理员可审核通过/驳回；普通员工无高危审核权限 |
| 5. 合同发起 | 商家端/运营端 | 默认三方租赁合同，可选择模板；资方不出现在合同 |
| 6. 支付和绑卡 | C 端 | 支付首期/保证金，完成代扣签约或账单支付授权 |
| 7. 发货 | 门店端/商家端 | 填写设备识别码 `IMEI/SN/VIN`，上传交付证据；长租不占用库存 |
| 8. 签收 | C 端 | 默认电子确认单签收，通过后进入履约中并生成账单计划 |
| 9. 履约账单 | C 端/商家端 | 客户可查看账单支付；商家可查看收款、履约、订单结算款 |
| 10. 留购/提前结清 | C 端 | 租赁口径展示留购；银行/应收内部模式可按模板展示提前结清，但不展示资方 |

## 3. 联营订单验收路径

| 步骤 | 触发端 | 必验结果 |
|---|---|---|
| 1. 联营办单 | 门店手机端/运营端 | 字段与商家订单基本一致，额外选择配资额度/内部资金来源配置 |
| 2. 客户确认 | C 端 | 客户只看到租赁商品、账单、保证金、合同，不看到配资、分红、资金来源 |
| 3. 审核 | 商家端/运营端 | 商家可自主审核；平台可按权限复核或接管异常 |
| 4. 内部资金分配 | 运营端 | 记录 `funding_source`、`funder_id`、内部台账；不改变客户合同链路 |
| 5. 合同 | C 端/运营端 | 仍为甲方商家、乙方客户、丙方平台三方租赁合同 |
| 6. 支付 | C 端 | 支付和绑卡流程与商家订单一致 |
| 7. 发货签收 | 门店端/运营端/C 端 | 可门店发货或内部代发；C 端统一展示发货进度，签收通过后履约中 |
| 8. 分账结算 | 财务后台 | 客户支付后平台代收，按规则形成订单结算款、平台服务费、内部台账 |
| 9. 钱包和提现 | 商家端 | 商家看到我的钱包；财务人员可被授权提现但不能改结算账户 |
| 10. 售后/退款 | 运营端/商家端 | 责任划分、钱包扣减/挂账、原路退款、冲正闭环 |

---

## 4. 平台订单验收路径

| 步骤 | 触发端 | 必验结果 |
|---|---|---|
| 1. 平台商品下单 | C 端 | 在安心用/平台长租板块选择商品、规格、套餐并下单 |
| 2. 平台审核 | 运营端 | 待审核列表可查看客户资料、授权状态、合同、公证、保证金、账单、备注和回调 |
| 3. 合同模板选择 | 运营端 | 可使用默认模板或切换业务模板后发起签署 |
| 4. 公证配置 | 运营端/C 端 | 可选发起公证，客户在订单页完成 |
| 5. 支付 | C 端 | 首期、保证金、绑卡按配置完成 |
| 6. 发货 | 运营端/平台仓/内部代发 | 选择发货方式，填写设备识别码，上传证据 |
| 7. 签收 | C 端 | 电子确认单签收通过后起租；异常进入人工复核 |
| 8. 履约 | C 端/运营端 | 订单状态为履约中，账单计划生效 |
| 9. 资金 | 财务后台 | 平台代收，生成平台收入、商家应结算款或内部应收台账 |
| 10. 留购/提前结清 | C 端 | 按业务模板展示中性按钮和弹窗，不暴露内部资金身份 |

## 5. 资金链路验收

| 场景 | 必验结果 |
|---|---|
| 客户支付 | 支付记录、订单账单、钱包/台账、操作日志全部一致 |
| 平台代收 | 客户资金进入平台代收链路，生成可对账流水 |
| 订单结算款 | 商家订单和联营订单可形成商家订单结算款 |
| 我的钱包 | 商家端展示我的钱包，不使用软钱包文案 |
| 提现 | 单笔 500-20000，每日最多 3 次；超 20000 进入人工审核 |
| 财务人员提现 | 可被授权查看钱包和发起提现；不能修改结算账户 |
| 修改结算账户 | 仅老板/管理员高权限，必须二次确认和日志 |
| 退款 | 出退款必须进入钱包/分账冲正；余额不足挂账并锁定相关流程 |
| 内部资金台账 | `funding_source`、`funder_id`、内部应收台账只后台可见 |

## 6. 体验租短租验收路径

| 步骤 | 触发端 | 必验结果 |
|---|---|---|
| 1. 进入体验租 | C 端 | 首页展示安心用/体验租分区;体验租按城市、品类、租期类型进入 |
| 2. 选择门店 | C 端 | 展示附近有可租车辆的门店、距离、营业时间、联系商家、导航到店 |
| 3. 选择车辆/车型 | C 端 | 展示车型、图片、短租价格、可租租期,商品标准名称不可由门店随意改名 |
| 4. 确认租期方案 | C 端 | 支持小时租、日租、周租、月租等后台可配置方案;显示取还时间和合计金额 |
| 5. 骑行人信息 | C 端 | 新增/选择骑行人,完成实名、人脸、证件、手机号、紧急联系人等校验 |
| 6. 协议和保证金 | C 端 | 客户确认短租协议、保证金/免押规则、扣款规则和逾期费用规则 |
| 7. 支付下单 | C 端 | 支付成功后生成体验租订单,进入商家接单倒计时 |
| 8. 商家接单 | 商家端/门店端 | 商家可接单/拒单;超时进入异常或自动取消规则 |
| 9. 取车核销 | 门店端/C 端 | 核销码或扫码绑车,绑定唯一车辆码,车辆状态进入出租中 |
| 10. 用车中 | C 端/运营端 | 显示剩余时间、到期时间、联系商家、导航、续租、联系客服、控车状态 |
| 11. 续租 | C 端 | 支持按可配置方案续租,生成父子订单链和补充支付记录 |
| 12. 还车验收 | 门店端 | 门店验车,上传证据,确认正常还车/超时/损坏/争议 |
| 13. 费用结算 | 财务后台/C 端 | 逾期费用、损坏扣款、保证金释放/退款/扣款审批闭环 |
| 14. 控车异常 | 运营端/客服中心 | 控车失败、定位失败、无法取还车进入异常工单并关联 IM 会话 |

## 7. IM 客服验收路径

| 步骤 | 触发端 | 必验结果 |
|---|---|---|
| 1. C 端入口 | C 端 | 底部 Tab、订单详情、支付失败、合同异常、体验租用车中等页面可进入客服 |
| 2. 商家/门店入口 | 商家端/门店端 | 订单异常、财务结算、体验租控车异常可联系平台客服 |
| 3. 上下文注入 | 系统 | 会话带入客户、业务线、订单、状态、商品/车辆、门店、当前页面和触发原因 |
| 4. 会话分配 | 运营端 | 按长租、体验租、财务、客诉等队列分配;支持在线/忙碌/小休/离线 |
| 5. 客服处理 | 运营端 | 客服可查看上下文卡片、快捷回复、历史会话、脱敏资料 |
| 6. 转工单 | 运营端 | 无法即时解决的问题可转工单,保留会话、订单、证据和 SLA |
| 7. 高危动作边界 | 运营端 | IM 内不能直接退款、关单、扣款、锁车、改账户;只能跳转原业务模块 |
| 8. 会话归档 | 系统 | 消息、附件、订单卡片、转接、工单事件可归档和检索 |
| 9. 合规话术 | 系统 | 敏感词、资方/风控外露、违规承诺有提醒或拦截记录 |

## 8. 权限与安全验收

| 角色 | 必验权限边界 |
|---|---|
| 商家老板 | 可管理员工、钱包、结算账户、审核、发货、订单 |
| 商家管理员 | 可按授权管理订单、员工、商品、发货，但修改结算账户需高权限 |
| 商家财务 | 可查看钱包和发起提现，不能修改结算账户、不能改审核规则 |
| 店长 | 可办单、发货、查看本门店订单，不可操作资金账户 |
| 普通员工 | 可办单和基础跟进，不可审核高风险订单、提现、改账户 |
| 运营管理员 | 可配置平台规则、高危操作需二次确认和日志 |
| 客服 | 可查看脱敏资料、备注、跟进，敏感操作受权限限制 |

## 9. C 端合规文案验收

| 检查项 | 必验结果 |
|---|---|
| 禁用字眼 | 正式页面不出现资方、资金来源、合作模式、风控 |
| 授权文案 | 使用信用/信息授权 |
| 保证金 | 全部展示为保证金 |
| 账单 | 展示账单支付、每期应付、首期应付 |
| 履约状态 | 展示履约中，不展示租用中 |
| 留购/提前结清 | 根据业务模板展示，不泄露内部资金模式 |
| 合同主体 | 客户合同中不出现资金方身份 |

## 10. 发货签收验收

| 场景 | 必验结果 |
|---|---|
| 门店自提 | 发货方式可选门店自提，填写设备识别码，上传交付证据 |
| 物流发货 | 可选择物流方式，填写/生成物流单号 |
| 设备识别码 | 长租发货必须填 `IMEI/SN/VIN` 之一或配置的识别码类型 |
| 电子确认单 | 客户签署后订单进入履约中，账单计划生效 |
| AI 问答预留 | 后台有问题配置、状态、记录字段和异常人工复核入口 |
| 异常签收 | 拒收、未签、AI 异常、物流异常均进入人工复核，不自动起租 |

## 11. 不通过条件

1. C 端任一正式页面暴露资方、资金来源、合作模式、风控字眼。
2. 长租订单必须先占用短租车辆库存才能发货。
3. 商家财务人员可以修改结算账户。
4. 提现规则不是 500-20000、每日最多 3 次、超 20000 人工审核。
5. 退款没有钱包/分账冲正或余额不足挂账机制。
6. 合同主体出现资方或内部资金来源。
7. 发货后没有设备识别码和交付证据留痕。
8. 签收异常仍自动起租或生成账单。
9. 体验租订单和长租订单混用同一套库存/状态机导致状态污染。
10. 体验租车辆没有唯一设备码或取车核销后未绑定车辆。
11. IM 客服无法带入订单上下文,客服还需要客户手工报订单号。
12. IM 内绕过权限直接执行退款、扣款、锁车、关单等高危动作。
