# 发货签收手册模板

> 适用对象：门店店员 / 商家运营 / 平台运营

---

## 1. 操作入口

```text
门店手机端
  -> 待办任务
  -> 待发货 / 待交付
```

或：

```text
商家 PC / 运营端
  -> 订单详情
  -> 发货履约
```

---

## 2. 发货方式

| 方式 | 说明 |
|---|---|
| 门店发货 | 默认方式，门店上传交付材料 |
| 物流发货 | 可对接顺丰、其他物流等接口或手工录入 |
| 平台发货 | 平台统一处理 |
| 资方发货 | 后续可配置 |

---

## 3. 正常流程

```text
订单满足发货条件
  -> 门店收到待办
  -> 选择发货方式并填写设备识别码（IMEI/SN/VIN）
  -> 上传设备照片 / 人机或人车合照 / 配件清单
  -> 确认发货或交付
  -> 订单进入待收货
  -> 客户按配置确认收货（普通确认 / 电子签收单 / AI 问答签收视频）
  -> 订单起租
```

---

## 4. 长租与短租设备要求

当前长租不设置独立设备扫描入口，不走库存状态。发货或交付时只要求按类目填写设备识别码，例如 IMEI、SN、VIN，并将识别码同步到订单详情、发货记录、合同签收证据和操作日志。

短租后续接入设备库存后必须：

- 绑定唯一设备码。
- 设备状态从在库变为已锁定。
- 交付后变为出租中。
- 归还后进入待验收。
- 验收通过后回到在库。

---

## 5. 页面截图占位

1. 门店待办列表。
2. 发货方式和设备识别码填写页面。
3. 上传照片页面。
4. 配件清单确认页面。
5. 发货确认页面。
6. 客户签收页面，包括电子签收单或 AI 问答签收视频入口。
7. 运营端订单详情发货记录。
8. 长租设备识别码记录；短租后续设备库存状态变化记录。

---

## 6. 异常处理

| 异常 | 处理 |
|---|---|
| 设备识别码缺失 | 长租不允许确认发货或交付 |
| 设备识别码重复 | 提示历史订单，进入人工复核 |
| 短租设备已出租 | 禁止绑定，提示当前订单 |
| 照片缺失 | 不允许确认交付 |
| 客户拒签 | 进入发货异常 |
| 归还损坏 | 进入验收争议或维修 |

---

## 7. 数据联动

发货签收会影响：

- 订单发货状态
- 长租设备识别码记录
- 短租后续设备库存状态
- 起租时间
- 账单计划
- 附件证据链
- 操作日志
- 租后逾期跟进
