# C 端前台总入口与页面架构

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、申请购买、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 本文定义客户小程序/App/H5 的前台入口。安心用长租和体验租短租在同一 C 端内展示，但业务链路和状态机分开。

---

## 1. 底部导航

| 导航 | 页面定位 | 核心内容 |
|---|---|---|
| 首页 | 综合入口 | 搜索、定位、活动、业务入口、推荐商品、风险提示 |
| 安心用 | 长租入口 | 分类、商品列表、商品详情、扫码办单 |
| 体验租 | 短租入口 | 城市、品类、附近门店、车型、短租方案 |
| 订单 | 订单中心 | 安心用订单、体验租订单、待办筛选 |
| 我的 | 账户中心 | 实名、授权、地址、协议、客服、设置 |

底部导航文案固定使用：

- 安心用：长租以租代购，不展示短租小时租内容。
- 体验租：短期租赁，重点展示门店库存、到店取还、小时/天/周/月方案。

---

## 2. 首页

首页不是单一业务线页面，而是综合入口。

### 2.1 页面模块

- 城市/定位授权。
- 搜索框：支持商品、门店、品牌、订单号。
- 业务入口：安心用、体验租。
- 品类导航：手机、电动车、电脑平板、数码设备、智能穿戴、出行设备等。
- 活动 banner：可跳转指定业务线、品类、商品或门店。
- 推荐商品：必须显示业务线标签。
- 官方提醒：反诈提醒、官方客服提醒、合同签署提醒。
- 悬浮客服：进入时带当前页面来源。

### 2.2 首页推荐规则

1. 客户未授权定位时，默认展示平台配置城市和全国可办商品。
2. 客户授权定位后，体验租优先展示附近有库存门店。
3. 安心用推荐按品类、销量、门店活动、信用评估可办范围排序。
4. 首页不得直接展示“联营订单”“后台内部规则”“收益”等内部概念。

## 3. 安心用长租路径

```text
首页/安心用
  -> 分类/搜索
  -> 商品详情
  -> 选择规格/长租套餐
  -> 订单确认
  -> 登录授权
  -> 实名认证
  -> 资料补充
  -> 信用/信息授权
  -> 合同/公证
  -> 保证金/首期/代扣
  -> 发货/收货确认
  -> 在租/续租/申请购买/归还
```

安心用商品详情必须展示：

- 商品图、商品名、规格、成色、设备价。
- 商家/门店/平台服务主体。
- 首期、每期、期数、保证金、服务费、公证费、申请购买参考金额。
- 服务保障、发货方式、签收方式。
- 租赁须知、逾期费用、归还/续租/申请购买说明。
- 反诈提醒：不向非平台账户转账，不相信非官方回收/退款电话。

---

## 4. 体验租短租路径

```text
首页/体验租
  -> 选择城市/授权定位
  -> 选择品类
  -> 选择门店
  -> 选择车型/商品
  -> 选择短租方案
  -> 填写骑行人/使用人
  -> 实名/人脸
  -> 支付租金/保证金或免押授权
  -> 商家接单
  -> 到店取车/核销/绑车
  -> 使用中/控车/倒计时
  -> 续租或还车
  -> 门店验收
  -> 保证金处理/完成
```

体验租页面必须展示：

- 城市、门店、距离、地址、营业时间。
- 联系商家、导航到店。
- 品类与租赁周期：时租、日租、周租、月租。
- 车型卡片：车型图、车型名、卖点、最低价、库存状态。
- 取还信息：第一版默认到店取还。
- 骑行人信息：证件、姓名、手机号、紧急联系人可配置。
- 核销码：支付后生成，门店取车时核销并绑定唯一设备码。
- 用车控件：开机、关机、闪灯鸣笛、定位、续租、还车。

## 5. 订单中心

订单中心顶部增加业务线筛选：

| 筛选 | 内容 |
|---|---|
| 全部 | 两条业务线订单混排，但卡片必须标明安心用/体验租 |
| 安心用 | 长租订单状态和待办 |
| 体验租 | 短租订单状态和待办 |

安心用状态重点：

- 待资料
- 待审核
- 待签约
- 待支付
- 待发货
- 待收货
- 履约中
- 已完成
- 已关闭

体验租状态重点：

- 待支付
- 待确认
- 待取车
- 已取车
- 待还车确认
- 已完成
- 已取消

---

## 6. 我的

我的页面包含：

- 登录状态和实名状态。
- 实名认证、信用/信息授权、代扣授权、协议中心。
- 地址管理：安心用发货地址。
- 骑行人/使用人管理：体验租使用。
- 客服 IM、工单、常见问题。
- 优惠券/活动权益。
- 设置、隐私、注销。

---

## 7. 告知内容统一要求

所有 C 端关键页面必须把“客户要承担什么、平台要做什么、门店要做什么”讲清楚。

| 页面 | 必须展示 |
|---|---|
| 首页 | 用户协议、隐私政策、定位用途、反诈提醒 |
| 商品详情 | 费用口径、服务保障、租赁须知、逾期费用 |
| 订单确认 | 费用快照、合同主体、支付/保证金/代扣说明 |
| 实名/人脸 | 身份和人脸数据用途、授权协议 |
| 合同/公证 | 合同模板、公证是否必选、费用和效力 |
| 安心用收货 | 发货方式、设备码、签收确认方式 |
| 体验租取车 | 核销码、绑车、控车定位授权 |
| 体验租用车 | 到期倒计时、续租、还车、逾期处理 |
| 退款/取消 | 退款条件、保证金释放、到账时效 |

---

## 8. 数据互通

C 端每一步必须写入统一订单快照：

- 业务线：安心用/体验租。
- 来源：平台自然流量、门店二维码、渠道二维码、活动页。
- 商家、门店、员工、渠道。
- 商品、规格、价格方案。
- 协议版本和勾选记录。
- 实名、人脸、信用评估、合同、公证、支付、发货/核销状态。

后台订单、财务、合同、公证、客服、渠道统计必须消费同一份快照，不能各自重新计算。
