# 话术模板与合同模板管理

> [V0.2.2 同步声明] 本目录历史命名“资方管理”保留为内部配置入口兼容。当前业务语义统一为**内部资金来源管理**;相关信息仅运营端、财务端、风控端可见,不得进入 C 端、商家端、客服话术或合同正文。

> P0 业务文档(2026-05-27)。
> 定义资方关联的两种模板的完整设计:**话术模板**(C 端 UI 差异化字眼)+ **合同模板**(客户合同 PDF)。

---

## 1. 模板分类总览

| 模板 | 用途 | 影响范围 |
|---|---|---|
| **话术模板**(terminology_template)| C 端 UI 按钮 / 弹窗 / 推送的差异化字眼 | C 端展示层 |
| **合同模板**(contract_template)| 客户合同 PDF 全文 | 客户签约时的合同 |

## 2. 话术模板

### 2.1 数据模型

```sql
terminology_template
- template_id              bigint PK
- template_name            varchar(64)   -- "标准租赁话术" / "蓝海银行赊销话术"
- description              varchar(255)
- applicable_scenario      varchar(255)  -- 适用场景说明

-- 差异化字眼配置(MVP 阶段共 5 个差异点)
- term_early_settle_button    varchar(32)  -- 提前结清按钮文案 ⭐
- term_early_settle_title     varchar(32)  -- 弹窗标题 ⭐
- term_early_settle_desc      text         -- 弹窗说明 ⭐
- term_completion_msg         varchar(255) -- 完成订单提示 ⭐
- term_overdue_push           varchar(255) -- 逾期推送话术
- term_due_reminder_push      varchar(255) -- 到期提醒推送

- status                   enum            -- active / draft / archived
- created_by               bigint
- created_at / updated_at

-- 索引:(status, template_name)
```

## 3. 合同模板

### 3.1 数据模型

```sql
contract_template
- template_id             bigint PK
- template_name           varchar(128)  -- "标准三方租赁服务协议 v2.5" / "蓝海银行赊销合同 v2.3"
- template_type           enum          -- lease(租赁)/ bank_credit_assignment(银行赊销)/ joint_venture(联营合作)
- description             text          -- 模板说明 / 法律性质说明
- template_file_url       varchar(512)  -- 合同模板文件 URL(PDF / Word 母版)
- template_html_content   text          -- HTML 版本(用于在线生成)
- version_no              varchar(16)   -- 版本号 v2.3
- effective_at            datetime      -- 生效时间
- expire_at               datetime      -- 失效时间(空 = 长期有效)

-- 法律审查
- legal_reviewed          boolean
- legal_reviewer          varchar(64)
- legal_review_at         datetime

-- 状态
- status                  enum          -- draft / pending_review / active / archived
- created_by              bigint
- created_at / updated_at

-- 索引:(template_type, status) / (version_no)
```

### 3.2 MVP 预设模板

| 模板 | 类型 | 适用资方 | 备注 |
|---|---|---|---|
| 标准三方租赁服务协议 v2.5 | lease | 平台自营资金 | 沿用现有 Hudson 已起草的版本 |
| 应收账款转让确认书 | lease(附件)| 平台自营资金 + 平台订单 | 跟随主合同 |
| 蓝海银行赊销合同 v2.3 | bank_credit_assignment | 蓝海银行 | 预留模板槽,等银行对接定稿 |
| 门店入驻及合作框架协议 | joint_venture(附件)| 所有商家 | 后期 Hudson 配置 + 法务定稿(本轮不做)|

### 3.4 客户合同生成流程

```
订单生效流程:
  ① 订单审批通过 → 分配内部资金来源
  ② 系统读取 funder.contract_template_id
  ③ 调用 contract_service.generate_contract(order_id, template_id):
     - 加载合同模板
     - 替换动态字段({{...}})
     - 生成 PDF
     - 上传到 e签宝
     - 客户、门店、平台依次签字
  ④ 签字完成后,合同 PDF 归档
  ⑤ 订单进入下一阶段(待付款 / 待发货)
```

## 4. 模板版本管理

### 4.1 版本切换原则

```
新版本发布:
  ① 法务定稿新版模板
  ② 后台新建模板,status = pending_review
  ③ 主管审核 → status = active,设置 effective_at
  ④ 新订单使用新版本(根据 effective_at)
  ⑤ 老订单沿用签约时的版本(已嵌入订单数据)
```

### 4.2 订单字段中的模板版本快照

```sql
order
- contract_template_id        -- 合同模板 ID
- contract_template_version   -- 合同模板版本号快照(防止后台改了模板影响已有订单)
- terminology_template_id     -- 话术模板 ID
- terminology_template_version -- 话术模板版本号快照(同上)
```

**关键设计**:订单一旦签约,合同 + 话术模板版本号锁定,后台后续更新模板**不影响已有订单**。
