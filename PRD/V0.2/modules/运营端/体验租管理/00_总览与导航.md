# 体验租管理 · 00 总览与导航

> 运营端「体验租管理」一级模块的入口总览。本模块是 P0 体验租短租业务的**运营后台主控**。
> **本目录与 `modules/短租业务/` 的边界**:
> - `modules/短租业务/` 是横切文档(C 端、商家端、运营端、数据模型都在同一目录,按业务领域组织)
> - `modules/运营端/体验租管理/` 是垂直文档(只看运营端这一端的页面级 PRD,按菜单组织)
> - 两者**互相引用**,不重复内容

---

## 1. 模块定位

| 项 | 内容 |
|---|---|
| 模块名称 | 体验租管理 |
| 所属端 | 运营端(总后台) |
| 入口路径 | 运营端左侧导航 > 体验租管理 |
| 使用角色 | 平台管理员、体验租运营、客服、风控、财务 |
| 核心目标 | 短租业务的运营后台主控:订单管理、车辆库存、价格方案、押金扣款、控车日志、异常处置、收益对账 |
| 不做什么 | C 端装修(在前台配置中心)、城市/门店开通(在店铺管理)、商品基础资料(在商品管理) |

## 6. 工作流(典型场景)

### 6.1 客诉:押金扣款不合理

```
客户在 C 端发起申诉
  ↓
路由到客诉管理(全局)
  ↓
客服判定属于体验租 → 转本模块 06 押金扣款异常
  ↓
体验租运营查看扣款证据 + 控车日志(本模块 07)
  ↓
判定:扣款合理 / 部分退还 / 全额退还
  ↓
完成处理(本模块 06 退款),回写客诉管理
```

### 6.2 长时间逾期处置

```
系统自动告警(订单逾期 24 小时)
  ↓
本模块 10 逾期事件列表自动出现
  ↓
体验租运营主管接单
  ↓
执行:远程锁车(本模块 07)→ 外呼客户 + 紧急联系人 → 通知门店上门找回
  ↓
处理结果记录在订单(本模块 02)
```

### 6.3 SLA 商家接单超时

```
系统监测到订单 60 分钟未接单
  ↓
本模块 01 工作台预警 + 02 订单管理标记
  ↓
体验租运营介入:
  ├─ 联系商家催接 → 订单恢复正常
  └─ 商家无响应 → 代接单(运营受理)或自动取消退款
```
