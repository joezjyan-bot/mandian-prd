# 店铺详情与商户配置

> 页面级 PRD 草案。
> 目标：店铺审核通过后，运营端可以在店铺详情里查看资料、授权、渠道来源、订单能力、钱包、接口计费和链路覆盖配置，而不是只能在审核页看一次资料。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 店铺详情与商户配置 |
| 所属端 | 运营端 |
| 入口路径 | 店铺管理 > 店铺列表 > 店铺详情 |
| 使用角色 | 平台管理员、店铺运营、财务、风控、客服主管 |
| 核心目标 | 管理已入驻商家/门店的资料、授权状态、经营权限、订单配置、渠道归属和财务状态 |
| 关联页面 | 店铺审核、商家工作台、商家商品、商家钱包、渠道管理、链路配置中心 |

---

## 2. 核心口径

1. 门店入驻审核通过后，入驻主体即成为商家主体，可同时使用商家 PC 端和门店手机端。
2. 店铺详情不重复做审核页的大表单，而是保留资料快照、配置入口、授权状态和变更记录。
3. 商家可自行完成部分授权，运营端必须能同步查看状态、回调时间和失败原因。
4. 订单能力按三类订单分别开关：门店订单、分红订单、平台订单。
5. 链路配置默认继承平台配置，必要时运营端可给单个商家做覆盖。
6. 渠道来源、推广码、渠道人员必须沉淀到店铺详情，后续渠道统计和佣金依赖这里。
7. 店铺详情内所有配置变更必须保留操作日志和配置版本。

## 4. 概览页签

| 模块 | 展示内容 |
|---|---|
| 基础信息 | 店铺名称、主体名称、主体类型、联系人、脱敏手机号、所在城市 |
| 当前状态 | 正常、待补授权、暂停、冻结、注销 |
| 订单能力 | 门店订单、分红订单、平台订单开通状态 |
| 授权状态 | e 签宝授权、收款账户授权、合同签章授权、风控授权 |
| 财务概览 | 可提现、冻结、提现中、接口计费欠费、最近提现 |
| 渠道来源 | 渠道名称、推广码、渠道人员、绑定时间 |
| 风险标签 | 法人不一致、收款账户授权、投诉异常、租后异常、黑名单命中 |
| 快捷操作 | 暂停店铺、调整能力、发起授权、查看钱包、查看商品、查看订单 |

概览只展示摘要，敏感资料和完整账号按权限进入对应页签查看。

## 5. 资料档案

| 分类 | 字段 |
|---|---|
| 营业执照 | 主体名称、统一社会信用代码脱敏、法定代表人、注册地址、有效期、OCR 状态 |
| 法人身份证 | 法人姓名、身份证号脱敏、有效期、正反面附件、OCR 状态 |
| 门店地址 | 省市区、详细地址、地图定位、门头照片、店内照片 |
| 收款资料 | 收款方式、账户名、账号脱敏、实名一致性、授权文件 |
| 入驻资料 | 入驻时间、提交端、审核人、审核结论、最近补资料记录 |

资料变更规则：

1. 商家端可提交资料变更申请。
2. 涉及主体、法人、收款账户、门店地址的变更必须进入运营审核。
3. 变更通过后生成新资料版本，旧版本归档。
4. 审核中的资料不能直接覆盖线上生效资料。

## 6. 授权状态

| 授权项 | 状态 | 说明 |
|---|---|---|
| e 签宝签章授权 | 未发起、授权中、已完成、失败、过期 | 用于平台代商家发起合同或补充合同 |
| 收款账户授权 | 无需授权、待授权、已提交、已通过、驳回 | 非法人账户必须有授权材料 |
| 合同模板确认 | 未确认、已确认、需重签 | 商家自营门店订单可使用商家模板 |
| 风控/征信授权 | 未开通、已开通、停用 | 商家调用平台风控能力时使用 |
| 接口计费协议 | 未确认、已确认、欠费限制 | 大数据、合同、公证等按次计费 |

操作：

| 操作 | 结果 |
|---|---|
| 发起授权 | 生成授权任务，推送商家端或门店手机端 |
| 重发授权 | 保留上次失败记录，重新生成任务 |
| 查看凭证 | 按权限查看授权文件 |
| 同步状态 | 手动拉取第三方状态，写入回调日志 |

## 7. 订单能力配置

| 能力 | 配置项 | 说明 |
|---|---|---|
| 门店订单 | 开启/关闭、自审权限、抽佣比例、门店订单办单助手 | 门店自有订单由商家审核 |
| 分红订单 | 开启/关闭、可选配资比例、资方范围、平台审核 | 需要运营端审核和资方分配 |
| 平台订单 | 开启/关闭、平台审核、是否允许联系客服 | 门店不出资，运营端主控 |
| 商品能力 | 自建商品、平台商品复制、商品审核要求、同步办单助手 | 商品必须进入审核或配置范围 |
| 发货能力 | 门店发货、平台发货、资方发货、物流接口 | 默认门店发货 |
| 员工办单 | 是否允许创建员工、员工可用入口 | 员工不可看钱包和敏感财务 |

能力变更规则：

1. 关闭某类订单能力后，不影响历史订单继续履约。
2. 新订单创建时读取当前生效配置版本。
3. 员工账号能力不能高于店铺主能力。
4. 暂停店铺时默认禁止新增订单和提现，历史订单仍可处理还款、归还和售后。

## 8. 链路配置覆盖

商家默认继承平台链路配置，单店覆盖只记录差异项。

| 链路 | 可覆盖内容 |
|---|---|
| 支付链路 | 首付、账单支付、部分支付收款码、代扣通道 |
| 合同链路 | 合同模板、补充合同、签署主体、失败重试 |
| 公证链路 | 是否使用、公证费、发起节点、客户办理入口 |
| 风控链路 | 资料要求、报告类型、黑名单匹配、补资料规则 |
| 发货链路 | 默认发货方、物流接口、签收照片要求 |
| 分账链路 | 抽佣比例、结算账期、提现限制、渠道佣金 |
| 办单助手 | 入口可用性、商品同步范围、增值服务范围 |

覆盖规则：

1. 单店覆盖必须显示“继承平台配置/已覆盖”。
2. 覆盖配置需要版本号、操作人、原因和生效时间。
3. 历史订单使用订单创建时的配置快照。
4. 高风险覆盖项需要主管复核，例如抽佣比例、提现限制、支付通道。

---

## 9. 财务与接口计费

| 模块 | 内容 |
|---|---|
| 钱包摘要 | 待结算、可提现、冻结、提现中、累计收入、累计提现 |
| 收款账户 | 实名状态、授权状态、账户变更记录 |
| 接口计费 | 风控、大数据、合同、公证、短信等调用次数和费用 |
| 欠费限制 | 欠费时限制新增调用或限制特定订单能力 |
| 提现限制 | 正常、风控冻结、资料异常、收款账户异常 |

接口计费不直接在店铺详情里扣改金额，只展示摘要并跳转财务明细。

---

## 10. 渠道归属

| 字段 | 说明 |
|---|---|
| 渠道名称 | 推广来源 |
| 渠道人员 | 具体推广人或团队 |
| 推广码 | 入驻时绑定的推广码 |
| 归属层级 | 一级、二级 |
| 绑定时间 | 首次绑定时间 |
| 变更记录 | 运营手动调整时记录原因 |

规则：

1. 渠道归属影响分红订单和平台订单佣金。
2. 门店订单默认不进入渠道佣金。
3. 渠道归属调整不追溯已结算佣金，除非走财务冲正流程。
4. 渠道归属变更必须有权限和日志。

---

## 11. 操作日志

必须记录：

| 日志类型 | 示例 |
|---|---|
| 资料变更 | 地址、收款账户、法人资料、门头照片 |
| 授权回调 | e 签宝授权、收款授权、合同确认 |
| 能力变更 | 开关订单类型、暂停店铺、恢复店铺 |
| 链路覆盖 | 支付、合同、公证、发货、分账配置 |
| 财务限制 | 冻结钱包、解除冻结、提现限制 |
| 渠道归属 | 绑定、解绑、调整推广人 |

日志字段：操作时间、操作人、角色、操作端、变更前、变更后、原因、关联单据。
