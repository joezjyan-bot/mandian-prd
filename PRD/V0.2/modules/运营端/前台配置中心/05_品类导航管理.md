# 前台配置中心 · 05 品类导航管理

> 管理 C 端品类树:安心用(多品类)+ 体验租(V1 只电动车,V2 扩展)。
> 品类与商品基础资料的关系:商品在「商品管理」维护,本模块只管品类树的**展示**与**前端排序**。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 品类导航管理 |
| 入口路径 | 前台配置中心 > 品类导航管理 |
| 使用角色 | 平台管理员、内容运营 |
| 核心目标 | 维护品类树、品类与商品关联、品类在 C 端的展示样式 |

---

## 2. 模块结构

```text
品类导航管理
├─ 2.1 安心用品类树
│  ├─ 一级品类(手机/电动车/电脑数码/智能穿戴...)
│  ├─ 二级品类
│  └─ 品类与商品的关联
└─ 2.2 体验租品类
   ├─ 一级品类(V1:电动车;V2:手机、相机、户外...)
   ├─ 二级品类(车型分类:踏板车/小型车/共享自行车...)
   └─ 品类与车型的关联
```

## 3. 安心用品类树

### 3.1 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| category_id | 文本(系统) | 是 | 品类 ID |
| parent_id | 关联 | 否 | 父品类(空 = 一级品类) |
| business_line | 下拉 | 是 | 固定 `assurance_rent` |
| standard_name | 文本 | 是 | 品类标准名(后台用) |
| display_name | 文本 | 是 | C 端显示名称 |
| icon_url | 图片上传 | 否 | 品类小图标(40×40 推荐) |
| sort_order | 数字 | 是 | 同级品类内顺序 |
| is_enabled | 开关 | 是 | 是否在 C 端显示 |
| show_label | 文本 | 否 | 角标文字(如"NEW"/"热门") |
| show_label_color | 颜色 | 否 | 角标颜色 |
| product_count | 数字(系统) | 是 | 当前关联商品数(只读) |

### 3.2 一级品类示例(平台默认)

| 顺序 | 品类 | 图标 | 关联商品类型 |
|---|---|---|---|
| 1 | 手机 | ti-device-mobile | iPhone / Android |
| 2 | 电动车 | (品类图,非 Tab 图标,允许具象) | 摩托/单车/电动车 |
| 3 | 电脑数码 | ti-device-laptop | 笔记本/平板/相机 |
| 4 | 智能穿戴 | ti-watch | 手表/耳机 |
| 5 | 出行设备 | ti-bike | 自行车/平衡车/滑板车 |

注:品类小图标允许使用具象图标(电动车 / 单车 / 手机)— 这跟"Tab 不锁品类"是两件事。Tab 是全局导航,要中性;品类宫格是具体业务,要直观。

## 4. 体验租品类

### 4.1 V1 范围

V1 仅启用电动车品类。其他品类(手机短租、相机短租、户外装备短租)做**框架预留**,设为 `is_enabled = false`。

### 4.2 字段(同安心用,差异)

| 字段 | 差异 |
|---|---|
| business_line | 固定 `experience_rent` |
| product_count | 改为 `vehicle_model_count`(车型数,而非商品数) |
| 新增 `time_unit_support` | 数组:`[hour, day, week, month]` 该品类支持的计费单位 |

### 4.3 V1 电动车二级品类(建议)

| 顺序 | 二级品类 | 用途 |
|---|---|---|
| 1 | 通勤代步 | 适合白领、学生通勤 |
| 2 | 长续航 | 适合外卖骑手、长距离 |
| 3 | 共享单车 | 适合短时使用 |
| 4 | 摩托车 | 适合驾照客户 |

## 5. 品类与商品/车型关联

### 5.1 关联规则

- **正向**:商品/车型在「商品管理」维护时,选定一个或多个品类
- **反向**:本模块的品类列表可以看到关联的商品/车型数量,但**不能在此页面增删商品**(只能链接跳转到商品管理)
- **多品类**:一个商品可挂多个品类(例如某商品既属于"手机"也属于"618 大促"促销品类 — 后者属于运营品类)

### 5.2 运营品类(虚拟品类)

除了基础品类(手机/电动车...),允许创建**运营品类**:

| 字段 | 说明 |
|---|---|
| category_type | `basic`(基础)/ `campaign`(运营) |
| 运营品类示例 | "618 大促"、"新人专享"、"学生优惠" |
| 关联规则 | 手动关联商品,不影响商品的基础品类归属 |
| 时间生效 | 运营品类可设 effective_at / expired_at |

## 6. C 端展示规则

### 6.1 品类宫格(C 端)

由「装修管理(安心用/体验租)」的"品类宫格"楼层引用,通过`category_id` 列表决定展示哪几个品类、顺序。

### 6.2 品类页(点击宫格进入)

| 模块 | 配置 |
|---|---|
| 品类页头部 | 品类名 + 副标题(可选) |
| 筛选器 | 二级品类(横滑标签)+ 品牌 + 价格区间 + 成色 |
| 商品/车型列表 | 默认按销量降序 |
| 空状态 | "该品类暂无可租商品"(可改) |

---

## 7. 三层覆盖(多租户)

| 层 | 可改 |
|---|---|
| 平台默认 | 全部字段 |
| 租户级 | 可改:display_name / icon_url / sort_order / is_enabled |
| 城市级 | 可改:is_enabled(某城市暂未开通某品类) |

## 8. 权限与日志

| 动作 | 权限 | 二次确认 | 日志 |
|---|---|---|---|
| 新增一级品类 | 平台管理员 | ✅ | 完整字段 |
| 删除品类 | 平台管理员 | ✅✅ | 仅可删除无关联商品的品类 |
| 改 display_name | 内容运营 | - | 前后值 |
| 改图标 / 排序 | 内容运营 | - | - |
| 启停品类 | 内容运营 | ✅ | 影响商品数 |
| 创建运营品类 | 内容运营 | - | - |
