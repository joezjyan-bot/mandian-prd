# 体验租 · 11 风险责任规则表

> 本文把体验租事故、违章、损坏、丢车、超时、保证金扣款从“原则描述”补成 V1 可配置规则表。老板已确认:平台不默认兜底,按保险、协议和证据责任划分;保证金扣款必须有规则、证据、审批、客户通知和申诉。

## 1. 统一处理原则

1. 体验租风险事件必须先建异常工单,再处理扣款、退款、锁车、法务或保险。
2. 责任方默认 `pending`,不可在证据不足时直接判定客户责任。
3. 保证金扣款按金额分级:小额 ≤5000,中额 5000-20000,大额 >20000。
4. 大额事件必须进入平台主管/法务复核。
5. 客户享有通知和申诉入口;申诉未结案前,除法定或协议允许的紧急处置外,不得直接关闭争议。
6. 所有证据必须归入订单证据链,可被法诉证据包引用。

| 场景 | 触发入口 | 必备证据 | 默认责任判断 | 保证金动作 | 审批 | 客户动作 | 订单状态 |
|---|---|---|---|---|---|---|---|
| 超时未还 | 系统到期检测/门店上报 | 订单租期、还车记录、续租记录、IM 通知 | 客户未续租且未按时还车时倾向客户责任 | 按超时费规则扣款或补收 | ≤5000 门店二次确认;更高走平台审核 | 可续租、还车、申诉 | `overtime_processing` |
| 普通损坏 | 还车验收 | 取车前照片、还车照片、损坏部位、维修报价 | 证据能证明使用期间产生则客户责任;证据不充分转争议 | 冻结保证金,审批后扣款 | 按金额分级 | 确认/申诉 | `damage_processing` |

| 还车争议 | 客户与门店意见不一致 | 还车照片、视频、IM、门店验收记录 | 进入争议,证据不足不得自动扣款 | 冻结争议金额 | 平台客服/主管 | 申诉/补充材料 | `return_dispute` |

## 3. 保证金扣款配置项

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `deduction_small_limit` | 500000 分 | 小额上限 |
| `deduction_medium_limit` | 2000000 分 | 中额上限 |
| `customer_appeal_window_hours` | 72 | 客户申诉窗口 |
| `auto_refund_after_normal_return_hours` | 1 | 正常还车后保证金退还/释放时效 |
| `large_case_legal_required` | true | 大额是否强制法务复核 |
| `insurance_policy_required` | true | 有保险时是否必须展示/归档保单 |
| `deposit_freeze_on_exception` | true | 异常工单创建后是否冻结保证金 |

## 4. C 端告知文案要求

1. 下单前必须展示租期、还车时间、保证金/免押、超时费、事故/违章/损坏/丢车处理原则。
2. 扣款前通知必须展示扣款原因、金额、证据入口、申诉入口、预计处理时效。
3. 不得使用“平台无条件承担”“保证不扣款”“一定免赔”等绝对承诺。
4. 保险信息展示以实际保险公司字段为准,但至少应有保障名称、保单状态、保障摘要和理赔入口。

## 5. 后台操作闭环

```text
异常触发
  -> 创建 short_rent_exception_ticket
  -> 冻结保证金争议金额
  -> 收集证据
  -> 判断责任方
  -> 审批扣款/退款/补收/理赔/法务
  -> 客户通知和申诉
  -> 四账入账或冲正
  -> 证据链归档
  -> 关闭异常工单
```

## 6. 验收断言

| 用例 | 必须断言 |
|---|---|
| 短租正常还车 | 保证金释放/退款,车辆回可租或待整备,四账无异常差额 |
| 短租损坏扣款 | 扣款申请、证据、审批、客户通知、钱包/支付/总账冲正完整 |
| 短租违章 | 违章证据、骑行人绑定、用车时间匹配,客户可申诉 |
| 短租丢车 | 控车/定位/报警/法务材料进入证据包 |
| 控车失败 | 不直接改变订单完成状态,进入异常工单 |
| 客户申诉 | 申诉期内状态可追踪,客服可查看证据和责任判断 |
