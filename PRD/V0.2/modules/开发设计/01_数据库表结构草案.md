# 数据库表结构草案

> 所属：开发设计
> 目标：按业务对象先给出表结构边界，后续开发可继续细化字段类型、索引、迁移脚本和分库分表策略。

---

## 1. 设计原则

1. 订单主表只放订单主信息，不承载所有子状态和流水。
2. 金额变动必须落流水，余额可由流水重算。
3. 所有第三方回调单独存表，不直接覆盖业务表。
4. 配置变更生成版本，订单保存版本引用。
5. 附件只存文件元数据和访问策略，不把文件放数据库。
6. 敏感字段单独标记，查询和导出按权限脱敏。

## 2. 主体与账号

| 表 | 用途 |
|---|---|
| `merchants` | 商家主体 |
| `stores` | 门店主体，可与商家一对多 |
| `merchant_qualifications` | 营业执照、法人、收款账户、授权状态 |
| `staff_users` | 商家老板、店长、店员账号 |
| `roles` | 角色 |
| `role_permissions` | 菜单、按钮、字段权限 |
| `login_sessions` | 登录保持、设备、过期时间 |
| `security_logs` | 登录、二次验证、强制下线 |

关键关联：

- `stores.merchant_id`
- `staff_users.merchant_id`
- `staff_users.store_id`

---

## 3. 商品与办单助手

| 表 | 用途 |
|---|---|
| `products` | 商品基础信息 |
| `product_skus` | 全新、二手、容量、颜色、设备指导价 |
| `product_prices` | 不同租期、模式、方案的价格配置 |
| `value_added_services` | 增值服务库 |
| `product_service_bindings` | 商品和增值服务绑定 |
| `assistant_product_pool` | 办单助手商品池 |
| `quote_snapshots` | 办单助手报价快照 |
| `quote_qrcodes` | 二维码、有效期、签名 |

关键口径：

- 商品归属支持平台商品和商家商品。
- SKU 独立控制是否同步办单助手。
- 报价快照保存商品、价格、服务费、公证费、留购价、公式版本。

## 4. 设备库存

| 表 | 用途 |
|---|---|
| `devices` | 唯一设备 |
| `warehouses` | 仓库 / 门店库存点 |
| `device_stock_logs` | 入库、出库、调拨、盘点 |
| `device_order_bindings` | 订单和设备绑定 |
| `device_inspections` | 归还验收、维修、争议 |
| `lock_devices` | 监管锁设备 |
| `lock_operation_logs` | 锁机、解锁、回调日志 |

关键口径：

- 短租必须绑定 `device_id`。
- 设备状态不能只靠库存数量表达。
- 设备出入库和订单状态联动。

---

## 5. 订单与客户

| 表 | 用途 |
|---|---|
| `customers` | 客户基础资料 |
| `customer_profiles` | 身份证、联系人、地址、认证状态 |
| `customer_risk_reports` | 风控、征信、授权报告 |
| `orders` | 订单主表 |
| `order_snapshots` | 商品、价格、配置、渠道、分账规则快照 |
| `order_status_logs` | 订单和子状态变更 |
| `order_reviews` | 审核记录、补资料、拒绝原因 |
| `order_funder_assignments` | 内部资金来源分配、出资比例 |
| `order_notes` | 商户备注、运营备注、系统备注 |
| `order_tasks` | 客户、商家、运营待办 |

订单主表建议字段：

- `order_id`
- `order_no`
- `order_type`
- `merchant_id`
- `store_id`
- `customer_id`
- `product_id`
- `sku_id`
- `main_status`
- `review_status`
- `payment_status`
- `contract_status`
- `ship_status`
- `created_from`
- `channel_id`
- `quote_snapshot_id`

## 6. 合同、公证、附件

| 表 | 用途 |
|---|---|
| `contract_templates` | 合同模板 |
| `contracts` | 合同实例 |
| `contract_signers` | 签署人 |
| `notary_cases` | 公证实例 |
| `authorization_records` | 风控、征信、签章等授权 |
| `attachments` | 附件元数据 |
| `attachment_access_logs` | 附件查看、下载、导出日志 |

关键口径：

- 合同、公证、授权都要有发起记录和回调状态。
- 附件中心统一承载身份证、营业执照、合同、照片证据、风控报告。

## 7. 财务

| 表 | 用途 |
|---|---|
| `bills` | 账单 |
| `payment_records` | 支付流水 |
| `refund_records` | 退款单 |
| `allocation_records` | 分账流水 |
| `wallet_accounts` | 钱包账户 |
| `wallet_entries` | 钱包流水 |
| `withdraw_requests` | 提现单 |
| `transfer_records` | 打款流水 |
| `reversal_records` | 冲正单 |
| `reconciliation_batches` | 对账批次 |
| `reconciliation_items` | 对账明细 |

关键口径：

- `wallet_accounts.balance` 可以做缓存，但必须能由 `wallet_entries` 重算。
- 退款、冲正不删除原流水。
- 联营订单按出资比例拆分后分别扣平台抽佣。

## 8. 资方与渠道

| 表 | 用途 |
|---|---|
| `funders` | 资方主体 |
| `funder_credit_logs` | 授信和额度变更 |
| `funder_statement_items` | 资方账单 |
| `channels` | 渠道主体 |
| `channel_invite_codes` | 推广码 |
| `channel_relations` | 一级、二级关系 |
| `channel_commission_rules` | 佣金规则 |
| `channel_commission_records` | 佣金明细 |

关键口径：

- 渠道佣金只统计联营订单和平台订单。
- 资方 V1 由运营端管理，后续只读端读取同一批数据。
