> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 解决 P0 问题 Q9(逾期费用规则未定义)+ Q10(接单超时规则描述不一致)。
> 本文统一定义:商家接单 SLA、客户逾期费用、控车权降级、保证金兜底、营业时间外订单规则。

## 1. 商家接单 SLA(Q10 解决方案)

### 1.1 默认 SLA

| 场景 | 默认超时 | 超时动作 |
|---|---|---|
| 标准订单(白天工作时段) | 60 分钟 | 自动取消 → 退款 |
| 营业时间外订单 | 营业开始后 30 分钟内 | 同上 |
| 节假日 / 高峰期 | 90 分钟(平台可配)| 同上 |

### 1.2 配置维度

SLA 可在 3 层配置(优先级:门店 > 商家 > 平台默认):

| 层级 | 字段 | 配置位 |
|---|---|---|
| 平台默认 | `default_accept_sla_minutes` | 配置管理 > 体验租周期 |
| 商家级 | `merchant_accept_sla_minutes` | 商家 PC 端 > 配置管理 |
| 门店级 | `store_accept_sla_minutes` | 商家 PC 端 > 门店资料 |

### 1.3 SLA 计算规则

```
SLA 起算时间 = 订单进入 pending_confirm 状态的时间
SLA 计时暂停 = 当超出门店营业时间时,SLA 时钟暂停
SLA 恢复 = 营业时间开始时
```

**示例**:
- 客户 22:30 下单(门店营业 9:00-21:00)
- SLA 时钟暂停,等待次日 9:00 营业开始
- 9:00 启动 60 分钟倒计时
- 10:00 仍未接单 → 自动取消

### 1.4 超时后行为(可配置)

| 行为 | 含义 | 适用 |
|---|---|---|
| `auto_cancel` | 自动取消订单,退款客户 | 默认 |
| `notify_operation` | 转运营人工介入 | 重要客户 / 高 GMV 订单 |
| `auto_reassign_v2` | 自动转其他可履约门店 | V2 实现 |

### 1.5 接单超时的客户告知

```
[订单状态 - 商家接单中]
┌─────────────────────────────────┐
│ ⏳ 商家接单中                     │
│ 剩余 12:30                       │
│                                  │
│ 提示:商家可能正在忙,请耐心等待   │
│       若 60 分钟未接单,我们将   │
│       自动取消订单并退款。        │
│                                  │
│ [联系商家] [取消订单]             │
└─────────────────────────────────┘
```

## 2. 客户逾期费用(Q9 解决方案)

### 2.1 核心定义

| 概念 | 说明 |
|---|---|
| **逾期开始** | 超过订单 `rent_end_at`(应还时间)之后 |
| **宽限期** | 默认 15 分钟,可配置(平台/商家级)|
| **逾期费用** | 客户超过宽限期未还车时产生的费用 |

**禁止用词**:在 C 端展示时**统一使用"逾期费用"**,不使用"违约金" / "罚息" / "滞纳金"(合规要求,P0-3 决策延续)。

### 2.2 逾期费用计算公式

```
逾期费用 = 已逾期小时数 × 单价 × 倍数

其中:
- 单价 = 该订单所属租赁周期的单价(如 1 小时套餐 = 9.9 元 / 时)
- 倍数 = 默认 1.5 倍(可配)
- 计费单位 = 1 小时(不足按 1 小时计)
```

**示例**:
- 客户租的是 9.9 元/小时
- 应还时间:18:00
- 实际还车:19:45
- 逾期时间:1 时 45 分(扣除 15 分钟宽限,实际计费 1.5 小时)
- 不足 1 小时按 1 小时算 → 计费 2 小时
- 逾期费用 = 2 × 9.9 × 1.5 = 29.7 元

### 2.3 逾期费用封顶

为避免极端情况(客户失联多天):

| 限额 | 默认值 |
|---|---|
| **单日逾期费用封顶** | 当日租赁基础租金 × 3 倍(防止 24 小时逾期费用高于车辆价值) |
| **总逾期费用封顶** | 保证金 × 2(允许超出保证金,但有上限) |

超过封顶后:
- 不再累计逾期费用
- 转入运营人工介入
- 系统触发"长时间逾期"工单

### 2.4 逾期费用扣款来源

```
逾期费用应付 = X 元
  ↓
扣款顺序:
1. 先扣保证金账户(若保证金充足)
2. 保证金不足 → 触发线下补收(运营介入)
3. 严重时 → 起诉客户(走司法流程,V2)
```

### 2.5 逾期费用免除场景

| 场景 | 免除规则 |
|---|---|
| 不可抗力(自然灾害/疫情封控)| 凭证 + 运营审核 → 全免 |
| 平台原因(系统故障无法还车)| 自动免除 |
| 商家原因(门店未营业/拒不接收)| 自动免除 |
| 客户首次少量逾期(< 30 分钟)| 商家级配置,可减免 |

## 3. 控车权降级(逾期联动)

### 3.1 降级阶梯

| 逾期时长 | 控车权状态 | 客户感知 |
|---|---|---|
| 0 - 15 分钟(宽限期)| 全部能力 | 仅 IM 友好提醒"快到期了,要续租或还车吗" |
| 15 分钟 - 2 小时 | 全部能力 | 红色横幅"已逾期,请尽快还车";逾期费用开始计 |
| 2 小时 - 6 小时 | 部分降级:关机 / 限速 / 定位仍开放,**禁用"开机"** | "车辆已限速,请立即还车" |
| 6 小时 - 24 小时 | 强降级:仅"查看定位"开放;远程锁车 | "车辆已被远程锁定" |
| > 24 小时 | 完全锁定 + 报警 | "请立即联系商家,长时间未还可能触发司法程序" |

### 3.2 降级前的告知

**强制规则**:每次降级前 15 分钟,系统必须推送告知:

```
⚠️ 即将进入限速模式

您的订单已逾期 1 小时 45 分钟。
为保障车辆安全,15 分钟后将进入限速模式。

请尽快前往最近门店还车,或选择续租。

[联系商家] [立即续租] [立即还车]
```

### 3.3 解除降级

| 场景 | 解除 |
|---|---|
| 客户补缴逾期费用并续租 | 立即恢复 |
| 客户补缴逾期费用并还车 | 自动解锁 → 进入还车流程 |
| 客户失联 → 门店主动找回 | 门店扫码核销 → 解锁 → 处置 |

## 4. 长时间逾期处置(> 24 小时)

### 4.1 触发动作

| 步骤 | 时机 | 动作 |
|---|---|---|
| 1 | 逾期 24 小时 | 系统自动远程锁车(参见 `08_控车日志与异常处置.md` §4) |
| 2 | 逾期 24 小时 | 系统创建"长时间逾期"工单,推送平台运营 |
| 3 | 逾期 24-48 小时 | 平台客服电话外呼客户 + 紧急联系人 |
| 4 | 逾期 48 小时 | 上报商家;门店根据车辆 GPS 上门找回(可选) |
| 5 | 逾期 72 小时 | 进入"疑似盗车"流程;平台启动黑名单+司法预备 |
| 6 | 逾期 7 天 | 报警 + 起诉(由商家与平台协商) |

### 4.2 客户告知

逾期 24 小时后的 C 端展示:

```
[订单严重异常]
┌─────────────────────────────────┐
│ ⛔ 车辆已被远程锁定                │
│                                  │
│ 您的订单已逾期超过 24 小时,        │
│ 累计逾期费用 ¥X(继续计算中)。     │
│                                  │
│ 请立即联系商家或平台客服:         │
│ 商家电话:****                    │
│ 平台客服:400-XXX-XXXX            │
│                                  │
│ 长时间未处理可能触发:              │
│ - 信用记录扣分                    │
│ - 黑名单标记                      │
│ - 司法追讨                        │
└─────────────────────────────────┘
```

## 5. 营业时间外的订单处理

### 5.1 客户下单时机

| 场景 | 处理 |
|---|---|
| 客户在门店营业时间内下单 | 标准 SLA |
| 客户在营业时间外下单,但选择"立即取车" | 提示"该门店当前未营业,预约下一个营业时间"或取消 |
| 客户在营业时间外下单,选择"预约次日取车" | 订单创建,SLA 在次日营业开始计算 |
| 客户在临近闭店前下单 | 警告"门店将在 X 分钟后闭店,请尽快赶到";正常下单 |

### 5.2 配置项

| 字段 | 说明 |
|---|---|
| `allow_overnight_booking` | 是否允许营业时间外预约下单 |
| `min_advance_booking_minutes` | 预约最小提前时长(默认 30 分钟) |
| `max_advance_booking_days` | 预约最大提前天数(默认 7 天) |

## 6. 跨天租赁的特殊规则

### 6.1 跨天租赁的还车时间

| 套餐 | 默认还车 |
|---|---|
| 1/2/4 小时 | 取车时间 + N 小时 |
| 当日还 | 取车当日的指定截止时间(默认 23:00,可改 |
| 日租 | 取车时间 + 24 小时(整 24 小时,不按自然日)|
| 周租 | 取车时间 + 7×24 小时 |
| 月租 | 取车时间 + 30×24 小时 |

### 6.2 跨天租赁的逾期费用计算

跨天订单的逾期,逾期费用单价以**该订单的小时均价**为基准:

```
日租 ¥48 / 24 小时 = 2 元/小时
逾期费用 = 逾期小时数 × 2 × 1.5
```

## 8. 数据落地

### 8.1 `short_rent_overdue_event` 逾期事件表

```sql
short_rent_overdue_event
- event_id                 bigint PK
- order_id                 bigint FK
- rental_chain_id          bigint FK
- vehicle_id               bigint FK
- customer_id              bigint FK
- overdue_started_at       datetime    -- 逾期起始
- overdue_ended_at         datetime    -- 逾期结束(还车时)
- overdue_minutes          int         -- 累计逾期分钟
- penalty_amount           decimal     -- 累计逾期费用
- penalty_deducted_from    enum        -- deposit / offline / forgiven
- exemption_reason         varchar(255)-- 免除原因(若有)
- escalation_level         enum        -- normal / lockdown / dispatch / legal
- status                   enum        -- active / resolved / written_off
- created_at / updated_at
```

### 8.2 `short_rent_sla_violation` SLA 违规表(接单超时)

```sql
short_rent_sla_violation
- violation_id             bigint PK
- order_id                 bigint FK
- store_id                 bigint FK
- merchant_id              bigint FK
- sla_type                 enum        -- accept_timeout / pickup_no_show / return_overdue
- expected_at              datetime    -- 应完成时间
- actual_at                datetime    -- 实际完成时间(若有)
- timeout_action           enum        -- auto_cancelled / manual_intervention / auto_reassigned
- created_at
```

商家级 SLA 违规率会影响商家信用分(V2)。

## 9. C 端用户感知

### 9.1 不展示给客户的内容

- "逾期费用" / "滞纳金"
- "平台分账"
- "门店超时罚款"
- 商家维度的 SLA 违规数据

- 接单倒计时(剩余 X 分钟)

- 控车权降级提醒(降级前 15 分钟)
