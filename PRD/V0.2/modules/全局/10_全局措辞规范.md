# 全局措辞规范

> **版本**: v2.0
> **最后修订**: 2026-05-27
> **适用范围**: PRD 正式文档、C 端文案、商家端文案、运营端文案、合同模板、操作手册、短信/通知/客服话术。
> **强制级别**: 必守。V0.2.1 开发冻结版以本文为唯一术语标准。

---

## 1. 核心原则

1. **客户侧不暴露内部资金和合作结构**:C 端不得展示资方、资金来源、合作模式、分账、平台抽佣等内部概念。
2. **租赁业务不使用借贷口径**:C 端和商家端避免"借款、贷款、还款、罚息"等金融语义。
3. **数据库字段 / API 路径 / 英文枚举保持兼容**:本文只约束中文展示和文档正式口径;已确认保留的英文枚举不因 UI 术语调整而改名。
4. **历史文档如出现旧词,不得作为开发依据**:以 `00_V0.2.2_开发冻结版总PRD.md`、本文和状态字典为准。

## 3. 分端用语边界

### 3.1 C 端

C 端只允许展示客户能理解的业务语义:

- 订单、商品、套餐、保证金、每期应付、账单支付、合同、收货、履约中、留购、提前结清、归还、客服。
- 不展示:资方、资金来源、合作模式、联营、应收账款受让、平台抽佣、分账、风控结论。

### 3.2 商家端

商家端允许展示经营和财务语义:

- 商家订单、联营订单、平台订单、我的钱包、订单结算款、提现、资金明细、账单、员工权限。
- 不允许把资方资金描述为"平台借款"或"采购款"。

### 3.3 运营端

运营端允许展示内部控制字段:

- 合作模式、资金来源、资方、话术模板、合同模板、放款条件、资方台账。
- 面向客户的文案配置必须同步检查 C 端敏感词。

### 3.4 合同与客服

合同和客服话术遵循客户侧边界:

- 默认三方租赁合同:甲方商家、乙方客户、丙方平台。
- 资方默认不出现在客户合同和客服话术中。
- 银行赊销类特例如启用,必须由 `funding_source + terminology_template_id` 控制按钮和弹窗话术。

## 4. 按业务模式的话术模板

| 业务模式 | 客户侧按钮 | 弹窗核心提示 |
|---|---|---|
| 默认租赁模式 | 留购 | 支付剩余应付款后,可按合同约定取得设备留购权益 |
| 银行赊销/提前结清模板 | 提前结清 | 支付剩余应付款后,本订单账单结清 |
| 归还模式 | 申请归还 | 按合同约定完成归还和验收 |
| 续租模式 | 申请续租 | 按续租方案继续履约 |

---

## 5. 执行规则

1. 新建文档必须直接使用标准口径。
2. 修订旧文档时,优先修正式开发入口文档;历史参考文档可标注"历史口径,不作为开发依据"。
3. 代码、数据库、接口层的英文兼容字段不在本次文档冻结中强制重命名。
4. C 端目录每次修订后必须扫描敏感词:资方、资金来源、合作模式、风控。
5. 文档如确需解释旧词,必须标注"历史称呼"或"禁止用语",不能作为正式业务口径。
