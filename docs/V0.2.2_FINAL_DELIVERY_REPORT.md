## 六、已确认但仍需制度落地

以下事项已并入总 PRD、最终文档地图和各模块主文档,不再单独保留确认清单。重点包括:

1. 无监管锁品类替代验收制度。
2. 短租事故、违章、损坏、丢车责任细则。
3. 保证金扣款和申诉制度。
4. 大额退款、坏账核销、法诉导出审批制度。
5. 合同、确认单、AI 问答文本最终法务盖章版。
6. 财务总账科目编码和供应商联调字段。

上述事项已按推荐方案确认,不阻塞研发拆任务;但需要财务、法务、运营或供应商联调把制度、文本、字段和验收配置固化后,才能通过对应模块上线验收。

## 七、是否可以进入研发任务拆分

可以。建议按以下顺序:

1. 基础设施:枚举、状态机、权限点、审计日志、幂等键、数据表。
2. 长租主链路:下单、审核、签约、支付、交付、签收、监管锁校验、结算、账单、退款、留购/提前结清。
3. 短租 V1:体验租入口、门店/车型/库存、短租订单、核销绑车、取还车、续租、保证金、异常工单。
4. 财务闭环:四账、分账、退款、提现、返点、坏账回收、对账。
5. IM/法诉/报表:会话、工单、证据包、经营看板、第三方催收 API 预留。
