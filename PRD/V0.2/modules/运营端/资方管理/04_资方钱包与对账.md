# 资方钱包与对账

> **来源**: P1-2 / P0-1

## 1. 页面定位

运营端 · 资方管理 · 资方详情 · 钱包页签。资方钱包余额、流水、充值、提现、待扣费队列。

## 2. 顶部指标

- 当前余额
- 总充值金额
- 总提现金额
- 总放款金额(累计出资)
- 总分账收入(累计收益)
- 待扣费金额(pending_fee_queue)
- 可用余额 = 余额 - 待扣费

## 3. 钱包流水

### 3.1 筛选
- 业务类型:充值 / 放款扣款 / 客户支付分账 / 线下手续费扣款 / 退款冲正 / 提现 / 后台调账 / 待扣费扣除
- 关联订单
- 金额范围
- 时间范围

### 3.2 字段
- 时间
- 业务类型
- 金额(+/-)
- 变动后余额
- 关联订单(可跳)
- 备注(按措辞规范)
- 操作人

## 4. 充值

资方充值通道:
- 三方支付(向资方提供平台收款账户)
- 对公转账 + 后台调账(走 04_后台余额变动)

资方钱包不开 C 端入口(资方端 V1 不开放)。
所有资方钱包充值都通过运营客服在运营端代操作。

## 5. 提现

资方提现:
- 提交提现申请(运营客服代操作)
- 走 02_钱包充值与提现 的提现流程
- 单笔/日额度同 P0-1 规则

## 6. 待扣费队列

### 6.1 概念
当订单需要扣资方钱包(如线下支付手续费、退款扣回)但余额不足时,挂入待扣费队列。

### 6.2 队列显示
- 待扣订单号
- 待扣业务类型
- 待扣金额
- 挂账时间
- 状态

### 6.3 自动扣除
资方下次充值/收入到钱包后,系统优先扣除待扣费队列(按时间从早到晚)。

### 6.4 长期待扣
> 7 天仍未补 → 告警,联系资方处理。

## 7. 资方对账(月度)

### 7.1 对账内容
每月生成对账单:
- 月初余额
- 总充值
- 总放款扣款
- 总分账收入
- 总线下手续费扣款
- 总退款冲正
- 总提现
- 总待扣费扣除
- 月末余额(应=月初+各项)

### 7.2 对账输出
- 可下载 Excel
- 包含每条流水明细
- 发送给资方主体确认(邮件)

### 7.3 资方确认
- 资方对账后确认:状态"已对账"
- 资方异议:运营介入核实

## 8. 数据隔离

- 资方钱包流水仅运营管理员 + 财务主管 + 资方管理客服可见
- 资方主体本身 V1 不开放访问

## 9. 操作日志

任何钱包变动 + 对账操作:留完整日志

## 10. 数据模型

- `funder.wallet_id`
- `pending_fee_queue`
- `wallet_transaction`(钱包流水统一表)
- 月度对账记录(后续补)
