# 满点租赁系统 V0.2.2 PRD 全量整改变更报告

> 本报告记录本轮 V0.2.2 PRD 全量整改结果。V0.2.2 是当前最新产品口径；旧文档、旧任务说明、旧术语与本报告和 `docs/V0.2.2_P0_FIXUP_TASK.md` 冲突时，以 V0.2.2 为准。
> 本轮未修改业务代码、数据库 migration、接口实现或前端页面代码。

## 零、二次收口结论(2026-05-28)

本轮在首次 V0.2.2 全量整改基础上继续补强 5 个容易导致研发误读的点:

1. 新增 `PRD/V0.2/00_V0.2.2_开发冻结版总PRD.md` 作为当前显式入口,旧 `00_V0.2.1_开发冻结版总PRD.md` 已归档到 `PRD/V0.2/archive/root_history/`,不再作为研发入口。
2. 财务文档中“资方账户 / 资方回款 / 退给资方 / 五边对账”等表达已净化为“内部资金来源台账 / 内部资金来源清算映射”,避免研发误做客户侧或商家侧资方账户体系。
3. 权限矩阵旧简表已降级为“默认角色模板”,并补充高危资金、账单、监管锁、退款、坏账、法诉导出操作的二次确认、二人复核和审计要求。
4. OpenAPI YAML 已升级到 V0.2.2,补充 C 端敏感字段隔离声明、三类订单新枚举、内部资金来源兼容说明和提现规则版本说明。
5. 验收矩阵新增可执行用例编号,覆盖四账分离、结算三条件、退款冲正、账单锁定、短租取还车、IM 证据沉淀、法诉证据包、高危权限和幂等。

## 二、已解决的 P0 问题

1. 三类订单命名统一：展示层统一为商家订单 / 联营订单 / 平台订单，内部枚举兼容保留。
2. 结算触发条件统一：签收确认 + 监管锁已上锁 + 激活锁已激活，三项缺一不得自动结算。
3. 退款责任统一：商家订单商家承担；联营订单 / 平台订单线下协商；支持指定退款金额并记录冲正对象。
4. 订单状态机补齐：覆盖草稿、资料、审核、签约、支付、交付、签收、锁校验、结算、履约、退款、逾期、法诉、坏账、完成、关闭。
5. 账单状态机补齐：覆盖未出账、待支付、部分支付、已支付、逾期、减免、冲正、退款、关闭、法诉、核销、回收完成。
6. 四账分离落地：订单业务账、支付流水账、钱包 / 商家账户账、财务总账 / 分录账分开设计。
7. C 端敏感字段隔离：C 端禁止返回资方、资金来源、合作模式、风控结论、内部审核备注、平台利润、返点规则等字段。
8. 法诉证据包补齐：订单、实名、合同、签收、AI 视频、交付照片、设备价值、监管锁、账单、支付、催收、IM、退款、日志可按订单导出。
9. 短租 V1 主流程补齐：C 端入口、商家接单、核销绑车、取车、用车、续租、超时、还车、保证金、事故结构进入一期。
10. 幂等与防重复补齐：支付回调、打款、退款、返点、提现必须有业务单号、幂等键、补偿任务和审计日志。

## 三、已解决的 P1 问题

1. 商家改单 / 客服改价边界：仅允许账单生成前由客服改价；账单生成后任何人不得直接修改账单。
2. 电子确认单 + AI 问答签收：作为可配置方案进入文档，默认方案和手动触发方案可后台配置。
3. 联营 / 平台订单退款协商单：补入协商金额、审批人、凭证、责任方、冲正对象。
4. 返点回滚：退款关闭按比例回滚，具体算法进入财务和测试矩阵。
5. 推广员离职归属：历史归属快照保留，新单不再归属离职推广员，未结佣金进入结算或冻结复核。
6. IM 客服基础能力：进入 V1，作为会话、工单、业务跳转、证据沉淀入口，高危动作不得直接在 IM 内执行。
7. 租后 / 催收边界：自建复杂催收后置，V1 保留状态、记录和第三方 API 对接。
8. 监管锁异常流程：允许人工放行，但必须有权限、审批、原因、凭证和审计日志。
9. 测试矩阵增加资金断言：订单业务账、支付流水账、钱包账、总账分录、退款冲正、返点回滚、提现失败回滚均需校验。

## 四、暂未处理的 P2 问题

1. 门店处罚系统化：本轮只保留备注、冻结、停用；处罚规则、扣款规则、评级规则后置。
2. 老板经营看板增强：GMV、回款率、逾期迁徙率、坏账率、Vintage、利润率等可在报表专项中继续细化。
3. 操作手册体系：Wujiezu 风格培训文档后续单独产出。
4. IM 智能质检：本轮只做基础客服能力，不做智能质检。
5. 多租户 SaaS 增强：独立部署、分品牌、分城市、租户隔离继续保留工程预留。

## 十、建议研发拆任务方式

1. 第一批:全局基础设施。状态机、枚举、DTO、权限点、审计日志、幂等键、基础数据表。
