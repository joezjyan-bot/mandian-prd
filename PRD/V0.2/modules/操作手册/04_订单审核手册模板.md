# 订单审核手册模板

> 适用对象：运营审核客服 / 商家老板 / 店长

---

## 1. 操作入口

```text
运营端 / 商家端
  -> 订单管理
  -> 待审核
  -> 订单详情
```

---

## 2. 审核边界

| 订单类型 | 审核方 |
|---|---|
| 门店订单 | 商家 / 门店 |
| 分红订单 | 运营端 |
| 平台订单 | 运营端 |

---

## 3. 审核前检查

需要查看：

- 客户实名资料
- 联系人和地址
- 风控/授权状态
- 历史订单
- 黑名单匹配提示
- 商品和价格快照
- 合同、公证、支付状态
- 订单备注和商家备注
- 分红订单资方和出资比例

---

## 4. 正常流程

```text
打开待审核列表
  -> 筛选订单类型
  -> 打开订单详情
  -> 查看客户资料和风控建议
  -> 必要时发起补资料或联系客服
  -> 分红/平台订单分配资方
  -> 审核通过或拒绝
```

---

## 5. 高风险动作

手册采集时不得点击最终确认：

- 审核通过
- 审核拒绝
- 关闭订单
- 加黑名单
- 改价确认
- 发起退款

正式系统需要：

- 二次确认
- 权限校验
- 操作日志
- 原因必填

---

## 6. 页面截图占位

1. 待审核订单列表，标注搜索条件和订单卡片。
2. 订单详情顶部，标注订单类型、状态、商家、客户。
3. 风控资料区，标注认证、定位、黑名单提示。
4. 价格账单区，标注已付、未付、押金、留购。
5. 操作区，标注补资料、改价、分配资方、审核按钮。
6. 操作日志区，标注人工和系统记录。

---

## 7. 数据联动

审核动作会影响：

- 订单主状态
- 审核状态
- 客户待办
- 资方额度占用
- 合同、公证、支付待办
- 发货条件
- 操作日志
