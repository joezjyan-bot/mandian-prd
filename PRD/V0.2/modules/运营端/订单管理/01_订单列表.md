## 2. 订单范围与页面拆分(v1.1 命名统一)

| 订单类型 | 是否显示 | 说明 |
|---|---|---|
| **商家订单**(self_operate)| 显示 | 平台可查看与综合服务费,但审核由门店自审 |
| **联营订单**(joint_venture)| 显示 | 平台审核、分配资方、处理合同/支付/授权/分账 |
| **平台订单**(receivables_assignment)| 显示 | 由平台匹配资方并主控审核财务 |

订单管理拆成四个入口:

```
订单管理
├─ 商家订单
├─ 联营订单
├─ 平台订单
└─ 全部订单
```

| 页面 | 定位 | 日常使用方式 |
|---|---|---|
| 商家订单 | 门店自营订单监管与综合服务费查看 | 平台主要看商户、发货、客诉、逾期、平台综合服务费和门店结算账户入账 |
| 联营订单 | 门店 + 平台共同出资的订单工作台 | 平台重点处理资方分配、审核、合同、风控、分账 |
| 平台订单 | 平台/资方主控订单工作台 | 平台重点处理资方、审核、合同、渠道佣金、发货主体和资金进度 |
| 全部订单 | 跨类型全局搜索与汇总 | 只作为查询和汇总入口,不作为日常主工作台 |

默认不进入「全部订单」。运营端可根据角色配置默认入口。

### 2.1 三类列表展示重点(v1.1 更新)

| 列表 | 展示重点 | 操作重点 |
|---|---|---|
| 商家订单 | 商户/门店、办单员工、客户资料、商品/订单周期/金额、商家审核状态、发货状态、平台综合服务费、门店结算账户入账、逾期/客诉 | 查看、备注、综合服务费查看、客诉处理、租后查看 |

### 4.3 搜索规则

1. 默认只展示基础筛选,避免页面过重。
2. 展开高级筛选后,用户可保存为个人常用筛选方案。
3. 订单类型页只查询当前类型,全部订单页可跨类型查询。
4. 手机号、身份证、银行卡等敏感字段只用于权限内查询,列表仍脱敏展示。
5. 查询结果必须可导出,但导出字段按角色权限控制。

## 5. 状态标签(v1.1 字眼更新)

| 标签 | 说明 |
|---|---|
| 全部 | 当前权限下可见订单总数 |
| 待付款 | 客户已提交但未完成首笔支付/冻结 |
| 待审核 | 需要平台或门店审核 |
| **待分配资方** | 联营/平台订单需要确认资方 |
| **待分配门店** | 平台订单资方分配后未分配履约门店 |
| 待签约 | 合同、授权、公证等未完成 |
| 待支付 | 客户待支付首期、服务费、公证费等 |
| 待发货 | 审核/签约/支付完成,等待门店或平台发货 |
| 待收货 | 已发货未确认收货 |
| **履约中** ⭐ v1.1 | 租期中(原"待归还"已统一改为"履约中") |
| 已逾期 | 已超过应还/应付时间(详情页显示"逾期费用"标签)|
| 归还中 | 已发起归还流程 |
| 已完成 | 订单正常完结(含归还/留购/提前结清) |
| 已关闭 | 取消、审核拒绝、退款关闭等 |
| **待退款工单** ⭐ v1.1 新增 | 财务专属:撤单已发起,等门店线下转账 |

**注**:原"已购买"统一改"已完成"(对齐 09 文档 v1.2)。

## 8. 关键交互

```
[查询] ──> 按筛选条件刷新订单卡片
[展开] ──> 显示更多筛选项
[状态标签] ──> 切换订单状态并刷新数量
[订单详情] ──> 打开详情页/抽屉,查看完整资料
[审核资料] ──> 打开资料抽屉,支持身份证、营业执照、授权文件等预览
[分配资方] ──> 系统推荐 + 运营确认 ──> 生成资方占比/分账规则/锁定合同+话术模板
[联系客服] ──> 调起 IM ──> 自动推送订单摘要给客服
[审核通过] ──> 二次确认 ──> 进入签约/支付/发货下一步
[审核不通过] ──> 填拒绝原因 ──> 订单关闭或退回
[取消订单(已分账)] ──> 二次确认 ──> 生成退款工单(refund_workflow)
[退款工单确认转账到位] ──> 触发客户退款 + 资方还原
```

## 10. 敏感信息规则

1. 列表默认脱敏展示姓名、手机号、身份证号、地址、IP。

3. 导出订单需要单独权限,导出文件记录操作人、时间、筛选条件。
4. 文档、测试数据、演示环境不得使用真实生产订单样例。
