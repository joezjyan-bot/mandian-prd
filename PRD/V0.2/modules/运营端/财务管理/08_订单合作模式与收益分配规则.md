# 订单合作模式与收益分配规则

> P0 业务文档(2026-05-27)。
> 配套 `财务管理/07_门店结算账户与资金穿透架构.md` 的核心算法补充。
>
> 定义三种合作模式的法律性质、出资模型、收益分配算法、会计科目映射、税务处理建议。
>
> **本文档是合规架构的核心**,必须与《门店入驻及合作框架协议》合同条款保持一致。

---

## 1. 三种合作模式总览

| 模式 | `cooperation_mode` | 业务含义 | 法律性质 |
|---|---|---|---|
| 自营订单 | `self_operate` | 门店全额出资 + 自主经营 | 平台综合服务合同(服务费收入) |
| 联营订单 | `joint_venture` | 双方按比例共同出资 | 联营合作(投资收益)+ 应收账款部分受让 |
| 应收账款受让订单 | `receivables_assignment` | 平台支付受让对价 | 应收账款受让(受让溢价收益)|

**核心特征**:三种模式法律性质独立,会计科目独立,税务处理独立,系统按 `cooperation_mode` 字段自动适用对应规则。

## 2. 模式 1:自营订单

### 2.1 业务实质

```
门店:全额出资采购设备 + 自主出租 + 履约服务
平台:提供综合服务(技术 / 风控 / 订单管理 / 代收代付 / 平台支持)
```

### 2.2 出资模型

| 出资方 | 出资比例 | 出资形式 |
|---|---|---|
| 门店 | 100% | 现金(采购设备) |
| 平台 | 0% | 仅提供服务(不出现金)|

### 2.3 收益分配

```
客户支付 ¥500 月供
   ↓
平台综合服务费 ¥10(留对公户,平台收入)
   ↓
设备使用费 ¥490 → 全部进门店结算账户
```

| 月度合计 | 平台 | 门店 |
|---|---|---|
| 收入 | ¥10 | ¥490 |
| 法律性质 | 平台综合服务费(服务收入)| 应收租金 |

### 2.4 风险承担

| 风险类型 | 承担方 |
|---|---|
| 客户违约 / 坏账 | 门店 100% |
| 设备损坏 | 门店 100% |
| 履约风险 | 门店 100% |
| 系统 / 通道风险 | 平台 |

### 2.5 会计科目

#### 平台账上

```
T0:无入账(自营订单平台不出资)

T1~Tn 客户每月付 ¥500:
借:银行存款                           ¥500
贷:其他业务收入 - 平台综合服务费       ¥9.43 (¥10/1.06)
贷:应交税费 - 增值税(销项)          ¥0.57
贷:其他应付款 - 门店结算款             ¥490

资金穿透:
借:其他应付款 - 门店结算款             ¥490
贷:银行存款                           ¥490 (充值通联备付金)
```

#### 门店账上

```
T1~Tn 月度收到 ¥490:
借:银行存款(或备付金台账)             ¥490
贷:主营业务收入 - 租赁收入             ¥449.54 (¥490/1.09)
贷:应交税费 - 增值税(销项)          ¥40.46
```

### 2.6 税务处理

| 项目 | 平台 | 门店 |
|---|---|---|
| 收入类型 | 服务费收入 | 租赁收入 |
| 增值税税率 | 6%(现代服务 / 商务辅助服务)| 9%(有形动产租赁服务)或小规模 3% |
| 所得税 | 25% | 25%(或小微优惠) |
| 发票开具 | 平台向门店开 6% 增值税专票 | 门店向客户开 9% 增值税普票 |

## 3. 模式 2:联营订单

### 3.1 业务实质

```
门店:出资 X%(现金,采购部分设备)+ 履约服务
平台:出资 (100-X)%(现金,补足设备采购)+ 综合服务
共同对该订单经营成果有决策权,共担风险,按出资比例分润
```

### 3.2 出资模型

| 出资方 | 出资比例 | 出资形式 |
|---|---|---|
| 门店 | X%(20-80%,门店选择) | 现金(采购对应比例的设备价值) |
| 平台 | (100-X)% | 现金(补足设备采购) |

### 3.3 出资比例可选档位

| 档位 | 门店:平台 |
|---|---|
| 高门店配资 | 80:20 |
| 偏门店 | 60:40 |
| 平均 | 50:50 |
| 偏平台 | 40:60 |
| 高平台配资 | 20:80 |

由门店在办单系统下单时选择,经平台审核确认后锁定。

### 3.4 收益分配(以 50:50 为例)

#### 阶段 1:订单生效时(监管锁回调)

```
平台支付应收账款转让对价 ¥2,500(50% × ¥5,000)→ 门店结算账户
门店自己也实际投入 ¥2,500(采购设备)
合计:门店与平台共同支撑了设备采购 ¥5,000
```

#### 阶段 2:客户每月支付 ¥500

```
客户支付 ¥500 → 平台对公账户
   ↓
拆分:
   ├─ 平台综合服务费 ¥10(留对公户)
   └─ 设备使用费 ¥490(穿透通联备付金)
   ↓
按出资比例分账(50:50):
   ├─ 门店 50% × ¥490 = ¥245 → 门店结算账户
   └─ 平台 50% × ¥490 = ¥245 → 通联子台账(平台台账)
```

#### 阶段 3:订单完成总收益核算(以 12 期 / 总收入 ¥6,000 为例)

| 总账 | 门店 | 平台 |
|---|---|---|
| 出资 | ¥2,500 | ¥2,500 |
| 月度分账累计 | ¥245 × 12 = ¥2,940 | ¥245 × 12 = ¥2,940 |
| **本金回收** | ¥2,500 | ¥2,500 |
| **利润分成** | ¥440 (¥2,940 - ¥2,500) | ¥440 (¥2,940 - ¥2,500) |
| 平台综合服务费 | - | ¥120 (¥10 × 12) |
| **总收益** | ¥440 | ¥560 |

### 3.5 风险承担

| 风险类型 | 承担方 |
|---|---|
| 客户违约 / 坏账 | 按出资比例分担(50:50)|
| 设备损坏 | 门店 100%(设备由门店保管) |
| 履约风险 | 门店 100% |
| 平台综合服务质量 | 平台 100% |

### 3.6 会计科目

#### 平台账上

##### 方案 A:按"联营投资"处理(争取免增值税,需税务师确认)

```
T0 平台支付应收账款转让对价 ¥2,500:
借:长期股权投资 - 联营投资 - 订单 OD001    ¥2,500
贷:银行存款                                ¥2,500

T1~Tn 客户每月付 ¥500:
借:银行存款                                ¥500
贷:其他业务收入 - 平台综合服务费(¥10)      ¥9.43
贷:应交税费 - 增值税(销项)              ¥0.57
贷:其他应付款 - 门店结算款(¥245)          ¥245
贷:投资收益 - 联营投资收益(¥245)          ¥245

订单完成总结(平台累计):
- 综合服务费收入(交 6% 增值税):¥120
- 联营投资本金回收:¥2,500
- 联营投资收益:¥440(平台分润,不交增值税,只交 25% 所得税)
```

##### 方案 B:按"应收账款部分受让"处理(税务保守方案)

```
T0 平台支付应收账款转让对价 ¥2,500:
借:应收账款 - 受让应收 - 订单 OD001(账面)  ¥2,940 (¥2,500 + 预期 ¥440 溢价)
贷:银行存款                                ¥2,500
贷:递延收益 - 应收账款受让溢价             ¥440

T1~Tn 客户付款时:
- 本金回收冲减应收账款
- 溢价按比例确认收入,交 6% 增值税
```

**实际选用方案 A 还是 B,由税务师最终确认**。系统设计上两种方案的数据流相同,只是会计科目不同。

#### 门店账上

```
T0 门店收到平台联营出资款 ¥2,500:
借:银行存款(或备付金台账)                   ¥2,500
贷:其他应付款 - 联营款 - 平台 - 订单 OD001     ¥2,500

T0 门店自己也投入 ¥2,500 采购设备(并入设备成本)

T1~Tn 月度收到 ¥245:
借:银行存款                                ¥245
贷:其他应付款 - 联营款 - 平台 - 订单 OD001     ¥208.33(本金回收)
贷:投资收益 - 联营投资收益                   ¥36.67(分润,不交增值税)
```

### 3.7 税务处理

| 项目 | 平台 | 门店 |
|---|---|---|
| 出资款收支 | 不计收入(投资行为) | 不计收入(投资款) |
| 月度分账本金部分 | 不计税(本金回收) | 不计税(本金回收) |
| 月度分账分润部分 | 联营投资收益,**不交增值税**,交 25% 所得税 | 联营投资收益,**不交增值税**,交 25% 所得税 |
| 平台综合服务费 | 6% 增值税 + 25% 所得税 | - |

### 3.8 必要的合规设计

要让税务认可"联营投资",必须做到:

| 要求 | 实施 |
|---|---|
| 共同出资真实 | 双方各自有出资记录 + 银行流水 |
| 共同经营真实 | 平台真实参与:风控审核、订单授权、客户管理 |
| 共担风险真实 | 坏账损失双方按比例真实承担 |
| 不收固定回报 | 按实际经营成果分,不保底 |
| 合同形式合规 | 联营合作协议 + 单笔订单出资确认 |

详细合同条款见《门店入驻及合作框架协议》第三章。

## 4. 模式 3:应收账款受让订单

### 4.1 业务实质

```
平台:支付应收账款转让对价(全额设备价)+ 综合服务
门店:全额接受平台支付的对价,但仍是出租方,负责履约 + 设备采购 + 售后
   ↓
平台 100% 受让门店的应收账款权益
客户租金全部归平台
门店赚"采购差价"(进货成本与平台付款的差额,门店自己消化)
```

### 4.2 出资模型

| 出资方 | 出资比例 | 出资形式 |
|---|---|---|
| 门店 | 0% | 不出资(只履约) |
| 平台 | 100% | 应收账款转让对价 |

### 4.3 收益分配

#### 阶段 1:订单生效时(监管锁回调)

```
平台支付应收账款转让对价 ¥5,000(100% × ¥5,000)→ 门店结算账户
门店实际采购成本 ¥4,000(假设,因供应链优势)
门店赚:¥5,000 - ¥4,000 = ¥1,000(采购差价,门店内部消化)
```

#### 阶段 2:客户每月支付 ¥500

```
客户支付 ¥500 → 平台对公账户
   ↓
拆分:
   ├─ 平台综合服务费 ¥10(留对公户)
   └─ 设备使用费 ¥490(穿透通联备付金,全部归平台台账)
   ↓
门店 0%,平台 100%
   ↓
平台子台账:¥490(应收账款受让回款)
```

#### 阶段 3:订单完成总收益核算(以 12 期 / 总收入 ¥6,000 为例)

| 总账 | 门店 | 平台 |
|---|---|---|
| 应收账款转让对价 | +¥5,000(已收) | -¥5,000(已付) |
| 月度回款累计(设备使用费部分) | ¥0 | ¥490 × 12 = ¥5,880 |
| 平台综合服务费 | - | ¥10 × 12 = ¥120 |
| **平台应收账款受让本金回收** | - | ¥5,000 |
| **平台应收账款受让溢价收益** | - | ¥880 (¥5,880 - ¥5,000) |
| 门店采购差价 | ¥1,000(自己消化)| - |
| **总收益** | ¥1,000 | ¥1,000 (¥880 + ¥120) |

### 4.4 风险承担

| 风险类型 | 承担方 |
|---|---|
| 客户违约 / 坏账 | 平台 100%(已受让应收账款)|
| 设备损坏 | 门店 100% |
| 履约风险 | 门店 100% |
| 客户合同纠纷 | 门店(作为出租方)+ 平台(应收账款受让方)按合同协商 |

### 4.5 会计科目

#### 平台账上

```
T0 平台支付应收账款转让对价 ¥5,000:
借:应收账款 - 受让应收 - 订单 OD001(账面)  ¥5,880 (¥5,000 + 预期 ¥880 溢价)
贷:银行存款                                ¥5,000
贷:递延收益 - 应收账款受让溢价             ¥880

T1~Tn 客户每月付 ¥500:
借:银行存款                                ¥500
贷:其他业务收入 - 平台综合服务费(¥10)      ¥9.43
贷:应交税费 - 增值税(销项)              ¥0.57
贷:应收账款 - 受让应收 - 订单 OD001         ¥490

按比例确认溢价收益(每期 ¥73.33,12 期均摊):
借:递延收益 - 应收账款受让溢价              ¥73.33
贷:其他业务收入 - 受让溢价收益(¥73.33)     ¥69.18
贷:应交税费 - 增值税(销项)              ¥4.15
```

#### 门店账上

```
T0 门店收到应收账款转让对价 ¥5,000:
借:银行存款(或备付金台账)                  ¥5,000
贷:应收账款 - 客户                          ¥5,000(同时把对客户的应收账款移交平台)

T1~Tn 月度无入账(应收账款已转让)
```

### 4.6 税务处理

| 项目 | 平台 | 门店 |
|---|---|---|
| 应收账款转让对价 | 计入"应收账款"(资产),不交税 | 不计收入,但需要做应收账款转让流转 |
| 客户月度还款本金部分 | 冲减应收账款,不交税 | - |
| 应收账款受让溢价 | 6% 增值税(金融服务 / 商务辅助服务)+ 25% 所得税 | - |
| 平台综合服务费 | 6% 增值税 + 25% 所得税 | - |
| 门店采购差价 | - | 业务收入,9% 增值税 + 25% 所得税 |

### 4.7 必要的合规设计

| 要求 | 实施 |
|---|---|
| 应收账款真实存在 | 真实的客户租赁合同 + 客户身份验证 |
| 转让对价合理 | 不显著偏离应收账款账面价值 |
| 转让真实有效 | 客户签署《应收账款转让确认书》|
| 转让流程合规 | 合同 + 通知 + 系统留痕 |

## 5. 三种模式的对比总览

### 5.1 业务对比

| 维度 | 自营订单 | 联营订单(50/50) | 受让订单 |
|---|---|---|---|
| 门店出资 | 100% | 50% | 0% |
| 平台出资 | 0% | 50% | 100% |
| 客户合同 | 门店出租 | 门店出租 | 门店出租 |
| 设备所有权 | 门店 | 门店 | 门店(留购转客户) |
| 客户违约风险 | 门店全担 | 双方按比例 | 平台全担 |

### 5.2 收益对比(以设备价 ¥5,000 / 客户总租金 ¥6,000 为例)

| 维度 | 自营订单 | 联营订单(50/50) | 受让订单 |
|---|---|---|---|
| 门店应收账款转让对价 | ¥0 | ¥2,500 | ¥5,000 |
| 客户租金总额 | ¥6,000 | ¥6,000 | ¥6,000 |
| 平台综合服务费(平台收入)| ¥120 | ¥120 | ¥120 |
| 月度分账门店份额累计 | ¥5,880 | ¥2,940 | ¥0 |
| 月度分账平台份额累计 | ¥0 | ¥2,940 | ¥5,880 |
| **门店总收益** | ¥5,880(全部) | ¥5,440(¥2,500 + ¥2,940) | ¥5,000 + 采购差价 |
| **平台总收益** | ¥120 | ¥560(¥120 + ¥440 分润) | ¥1,000(¥120 + ¥880 溢价) |
| **门店净利润**(扣除门店成本 ¥4,000 进货价后) | ¥1,880 | ¥1,440 | ¥1,000 |
| **平台净利润** | ¥120 | ¥560 | ¥1,000 |

**核心洞察**:出资越多的一方,获得的"溢价收益"也越多。三种模式都体现"出资 = 收益"的对等关系。

### 5.3 税务对比

| 项目 | 自营订单 | 联营订单 | 受让订单 |
|---|---|---|---|
| 平台增值税 | 6%(综合服务费部分) | 6%(综合服务费部分)+ 投资收益不交 | 6%(综合服务费 + 受让溢价) |
| 平台所得税 | 25% | 25% | 25% |
| 门店增值税 | 9%(租赁收入) | 9%(应收租金部分)+ 投资收益不交 | 不涉及(已转让) |
| 门店所得税 | 25% | 25% | 25% |

### 5.4 法律风险对比

| 风险点 | 自营订单 | 联营订单 | 受让订单 |
|---|---|---|---|
| 被认定为放贷 | 低 | 中(需严密设计联营条件)| 低 |
| 被认定为变相借贷 | 低 | 低 | 低 |
| 税务穿透认定 | 低 | 中(需真实共担风险)| 低 |
| 监管牌照风险 | 低 | 低 | 低 |

## 6. 系统实现要点

### 6.1 订单字段(`order` 表新增)

```sql
order
- cooperation_mode              enum    -- self_operate / joint_venture / receivables_assignment
- merchant_funding_ratio        decimal -- 门店出资比例(0-100%)
- platform_funding_ratio        decimal -- 平台出资比例 = 100% - merchant_funding_ratio
- platform_service_fee_rate     decimal -- 平台综合服务费率(下单时锁定快照)
- device_price                  decimal -- 设备价
- total_rent                    decimal -- 客户租金总额
- expected_profit               decimal -- 预期总利润 = total_rent - device_price
- merchant_expected_profit      decimal -- 门店预期分润
- platform_expected_profit      decimal -- 平台预期分润 + 综合服务费
```

### 6.2 办单时的合作模式选择 UI(运营端 / 商家端)

```
┌──────────────────────────────────────────────────┐
│ 办单助手 - 选择本订单的合作模式                       │
├──────────────────────────────────────────────────┤
│ 设备价:¥5,000                                     │
│ 客户租金总额:¥6,000                                │
│                                                    │
```

### 6.4 分账算法(系统核心)

```python
def calculate_monthly_split(payment_amount, order):
    """客户每期租金支付后,按合作模式分账"""
    
    # 步骤 1:计算平台综合服务费(留对公户)
    platform_service_fee = payment_amount * order.platform_service_fee_rate
    
    # 步骤 2:计算设备使用费部分(穿透通联备付金)
    device_use_fee = payment_amount - platform_service_fee
    
    # 步骤 3:按合作模式分账
    if order.cooperation_mode == 'self_operate':
        merchant_share = device_use_fee   # 自营订单门店拿全部
        platform_share = 0
    elif order.cooperation_mode == 'joint_venture':
        merchant_share = device_use_fee * order.merchant_funding_ratio
        platform_share = device_use_fee * order.platform_funding_ratio
    else:  # receivables_assignment
        merchant_share = 0  # 应收账款已转让
        platform_share = device_use_fee
    
    return {
        'platform_service_fee': platform_service_fee,  # 留对公户
        'merchant_share': merchant_share,              # 穿透 → 门店子台账
        'platform_share': platform_share,              # 穿透 → 平台子台账
    }
```
