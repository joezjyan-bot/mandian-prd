# 在租期间(还款 / 留购 / 归还 / 续租)

> **来源**: P1-5

## 1. 在租期间的客户操作

订单状态为"出租中"时,客户可在订单详情进行:
- 查看账单 / 主动还款
- 申请留购(P1-5 重命名)
- 申请归还(到期前)
- 申请续租(到期前)

## 2. 账单

### 2.1 账单列表
- 按期数排序
- 每期账单:期数 / 还款日 / 应还金额 / 实还金额 / 状态(待还 / 已还 / 逾期)
- 已还期数(灰色),待还期数(高亮)
- 当前期(红框)

### 2.2 还款日提醒
- 还款日前 3 天:推送提醒
- 还款日当天:推送
- 逾期 1 天:推送 + 短信
- 逾期 3 天:推送 + 短信 + 客服跟进

### 2.3 主动还款
客户可主动点击"立即还款":
- 自动调起代扣卡支付
- 也可选其他支付方式
- 支付成功 → 入账 → 分账

### 2.4 部分支付
- 客户可还款一部分(支持)
- 系统按原比例拆账(P0-3)

## 3. 申请留购

### 3.1 触发节点
任何时候(在租期间),客户可主动留购

### 3.2 留购计算
- 留购价格 = 价格方案中的"留购价格"字段
- **保证金可抵扣**留购价
- 客户实付 = 留购价 - 保证金
- 还有未到期的期数:可选"还清剩余后留购"或"一次性付清留购"

### 3.3 流程
```
1. 点"申请留购"
2. 系统计算:留购价 / 保证金抵扣 / 实付金额
3. 二次确认弹窗:"留购后设备归您所有,所有租期账单结清"
4. 调起支付
5. 支付成功 → 订单状态:已完成(留购)
6. 设备状态:已下架(下架原因: liugou = 已售给客户)
7. 解锁监管锁(如有)
```

### 3.4 部分场景需提示
- 客户实付 ≤ 0(保证金 > 留购价):系统自动退差额
- 保证金 < 留购价:补差额

## 4. 申请归还

到期前客户可主动申请:
- 走门店归还流程
- 设备验收
- 保证金按验收结果退还

## 5. 申请续租

到期前客户可申请:
- 续租期数(可配)
- 续租价格(按平台政策)
- 续租走新一轮合同流程(简化)

## 6. 逾期处理

- 计算逾期逾期费用:0.05%/日,最低 10 元/日(P0-3 规则)
- 全部归平台,不参与分账
- 客服跟进 + 推送提醒
- 长期逾期 → 监管锁触发 → 法务跟进

## 7. 数据模型

- `bill` 账单表
- `order.purchase_applied_at`(留购申请时间)
- `order.return_request_id`(归还申请)
- `order.renewal_request_id`(续租申请)
- `overdue_penalty` 逾期逾期费用记录
