# 押金 / 首期支付

> **来源**: P1-5 / P0-1

## 1. 业务背景

合同签署完成后,客户支付押金 + 首期。这是订单生效的最后一道资金关卡。

## 2. 支付内容

- 押金:¥XXX
- 首期金额:¥XXX
- 总计:¥XXX

## 3. 支付通道

V1:
- 支付宝小程序 → 支付宝(直付通,可分账,V2 启用)
- 微信小程序 → 微信支付
- H5 → 通联(银行卡)+ 支付宝/微信 H5 支付

后续 V2:全部走分账支付通道(支付时直接拆账)

## 4. 流程

```
1. 订单详情看"去支付"按钮
2. 点击 → 显示支付汇总(押金 + 首期 + 总计)
3. 选支付通道
4. 调起支付 SDK
5. 客户完成支付
6. 通道回调 → 系统:
   - 资金到平台备付金账户
   - 抽佣 → 平台账户
   - 剩余 → 责任方(门店/资方)钱包
7. 订单状态:已生效 → 进入"待发货"
```

## 5. 失败处理

| 场景 | 处理 |
|---|---|
| 支付被风控拦截 | 提示客户选其他通道 |
| 支付通道异常 | 自动尝试备用通道 |
| 客户取消支付 | 订单回到"待支付",可重试 |
| 客户长期不支付(72 小时) | 订单自动关单(未签合同免费) |

## 6. 重要规则

- **必须客户本人**(用客户账户支付,不可门店代付)
- 不允许使用平台账户代付
- 支付完成后立即分账(P0-1 实时分账)

## 7. 数据模型

- `order.payment_status`
- `order.first_period_paid_at`
- `payment_record` 流水
- 触发分账流程,见 P0-3
