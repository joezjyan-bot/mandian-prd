# 前台配置中心 · 06 弹窗与公告

> 管理 C 端所有弹窗、浮窗、公告栏、强制更新提示。
> 核心约束:弹窗会打断用户操作,必须配置触发条件、展示频次、客群定向、关闭逻辑,不能滥用。

弹窗与公告

## 3. 启动弹窗

App / 小程序冷启动时弹出。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| popup_id | 文本(系统) | 是 | - |
| popup_name | 文本 | 是 | 后台名称 |
| popup_type | 下拉 | 是 | image / image_with_button / multi_step(多步引导) |
| asset_id | 关联 | 是 | 主图素材 |
| title_copy | 文本 | 否 | 标题文案 |
| body_copy | 文本 | 否 | 正文文案 |
| button_text | 文本 | 否 | 按钮文字 |
| button_link_id | 关联 | 否 | 按钮跳转 |
| close_behavior | 下拉 | 是 | 立即可关 / 倒计时 3 秒后可关 / 必须点按钮 |
| effective_at / expired_at | 时间 | 是 | 生效时间 |
| target_audience_id | 关联 | 否 | 定向客群 |
| target_cities | 多选 | 否 | 定向城市 |
| frequency_rule | 下拉 | 是 | 仅展示一次 / 每天首次启动 / 每次启动 / 自定义间隔(N 天 1 次) |
| priority | 数字 | 是 | 多个启动弹窗同时生效时排序 |

**展示规则**:
- 同一冷启动**最多展示 1 个**启动弹窗(高优先级)
- 用户关闭后,记录到 `popup_dismiss_log`,按 `frequency_rule` 决定下次是否再展示
- 强制更新弹窗优先级最高,会覆盖启动弹窗

## 4. 活动浮窗

页面右下角常驻的小图标(可关闭),点击进入活动页。

| 字段 | 类型 | 说明 |
|---|---|---|
| float_id | 文本(系统) | - |
| float_name | 文本 | - |
| icon_asset_id | 关联 | 浮窗图标 |
| link_id | 关联 | 跳转目标 |
| display_pages | 多选 | 在哪些页面显示(安心用首页 / 体验租首页 / 商品详情 ...) |
| position | 下拉 | 浮窗位置:右下 / 右中 / 左下 |
| dismissible | 开关 | 用户能否关闭 |
| dismiss_persist | 开关 | 用户关闭后是否记住(默认本会话内不再展示) |
| effective_at / expired_at | 时间 | - |
| target_audience_id | 关联 | - |

---

## 5. 公告栏

顶部一行跑马灯文字(可关闭),常用于系统公告、限时活动提示。

| 字段 | 类型 | 说明 |
|---|---|---|
| notice_id | 文本(系统) | - |
| notice_content | 文本 | 公告内容(最长 60 字) |
| notice_color | 下拉 | 信息蓝 / 警告黄 / 危险红 / 中性灰 |
| display_pages | 多选 | 在哪些页面顶部展示 |
| link_id | 关联 | 点击跳转(可选) |
| dismissible | 开关 | 用户能否关闭 |
| scroll_speed | 数字 | 跑马灯滚动速度(秒/字) |
| effective_at / expired_at | 时间 | - |

## 6. 反诈提示弹窗

满足触发条件时强制弹出,**不可由运营完全关闭**(合规要求)。

### 6.1 默认触发场景(平台预置)

| 场景 | 默认弹窗内容(运营可微调话术,但不可删除) |
|---|---|
| 首次进入 App | 反诈基础提示(不向非平台转账;官方客服只通过 App 内联系) |
| 进入"客服 Tab" | 提示"满点官方客服只通过 App 内联系,任何要求微信/QQ 私聊的都是骗子" |
| 进入合同签署页 | 提示合同主体三方(门店/客户/平台),签署前确认费用快照 |
| 进入支付页 | 提示不向私人账号转账 |
| 进入留购支付页 | 提示留购流程仅在 App 内完成 |

### 6.2 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| scenario | 下拉(系统) | 触发场景(系统预置) |
| content_copy_id | 关联 | 文案(运营可改但不可空) |
| can_skip_today | 开关 | 是否允许"今日不再提示"(部分场景必须每次提示) |
| highlight_keywords | 多选 | 高亮关键词列表 |
| frequency_rule | 下拉 | 每次进入 / 每天首次 / 仅首次进入 App |

**校验**:
- 反诈弹窗**不能整体关闭**,只能调整文案
- 文案修改也需走审核流程(防止运营误改削弱反诈效果)

## 7. 强制更新弹窗

App 版本低于最低支持版本时强制弹出,无法关闭。

| 字段 | 类型 | 说明 |
|---|---|---|
| min_version | 文本 | 最低支持版本(如 `1.5.0`) |
| current_recommended_version | 文本 | 推荐版本 |
| force_update | 开关 | true = 不更新无法使用 / false = 可跳过 |
| update_message | 富文本 | 更新说明 |
| update_link | 文本 | 应用市场链接 |

---

## 8. 全局弹窗优先级

同一时刻可能多个条件命中时,按优先级展示:

```text
1. 强制更新(最高) ⭐ 不可跳过
2. 反诈提示(场景触发)
3. 启动弹窗
4. 活动浮窗(始终常驻,不抢占)
5. 公告栏(始终常驻,不抢占)
```

同一启动序列内,**只弹一个弹窗**(避免连续打断)。

---

## 9. 弹窗频次控制

| 频次规则 | 实现 |
|---|---|
| 仅首次 | 写入 `popup_dismiss_log` 永久标记 |
| 每天首次 | 按 date(用户本地时区)判断 |
| 每次启动 | 不记录 |
| 自定义 N 天 | 记录上次展示时间,N 天后再次展示 |
| 关闭后立即不再展示(本会话) | 内存标记,App 杀进程后重置 |

## 10. 权限与日志

| 动作 | 权限 | 二次确认 | 日志 |
|---|---|---|---|
| 创建启动弹窗 | 内容运营 / 市场运营 | - | 完整配置 |
| 创建反诈弹窗 | 平台管理员 | ✅ | 影响范围 |
| 修改反诈文案 | 内容审核 | ✅ | 前后值 |
| 关闭反诈弹窗 | **禁止** | - | - |
| 发布强制更新 | 平台管理员 | ✅✅ | 影响版本 |
| 紧急下线弹窗 | 平台管理员 | ✅ | 下线原因 |
