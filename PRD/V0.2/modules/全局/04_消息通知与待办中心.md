# 消息通知与待办中心

> 全局 PRD 草案。
> 目标：统一运营端、商家端、门店手机端、渠道端和 C 端的待办、通知、短信、IM 和系统提醒，避免每个模块各自发消息导致漏处理、重复处理和不可追溯。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 消息通知与待办中心 |
| 所属端 | 全局基础能力 |
| 入口路径 | 各端顶部消息、工作台待办、订单详情消息记录 |
| 使用角色 | 运营、商家老板、员工、客户、渠道、财务、租后人员 |
| 核心目标 | 统一生成、分配、展示、处理和归档所有待办与通知 |

---

## 2. 核心口径

1. 待办是需要用户处理的任务，通知是信息提醒，两者必须区分。
2. 所有待办必须有关联对象：订单、账单、店铺、商品、投诉、提现、租后案件等。
3. 订单详情必须展示与该订单相关的消息和待办处理记录。
4. 短信、站内信、IM、小程序通知都通过统一通知模型生成。
5. 同一个业务事件不能重复创建多个未处理待办，必须有幂等规则。
6. 敏感消息按接收角色脱敏，员工账号不接收钱包和提现类通知。
7. 通知失败不影响主流程，但必须进入异常日志；待办超时会影响工作台预警。

---

## 3. 待办类型

| 类型 | 接收端 | 示例 |
|---|---|---|
| 审核待办 | 运营端、商家端 | 订单待审核、商品待审核、店铺待审核 |
| 资料待办 | C 端、商家端 | 客户补资料、商家补授权、收款账户补材料 |
| 签约待办 | C 端、商家端 | 合同待签、补充合同、e 签宝授权 |
| 支付待办 | C 端、财务 | 首付待支付、部分支付、线下收款待确认 |
| 发货待办 | 门店手机端、商家端、运营端 | 待发货、上传交付照片、签收异常 |
| 归还待办 | C 端、门店端、运营端 | 申请归还、归还验收、争议处理 |
| 财务待办 | 运营端、商家端 | 提现审核、打款失败、对账异常 |
| 租后待办 | 运营端 | 下次跟进、承诺支付到期、外部回调异常 |
| 客诉待办 | 运营端 | 支付宝投诉待处理、补充凭证 |
| 系统异常 | 运营端 | 回调失败、链路配置缺失、分账失败 |

---

## 4. 待办字段

| 字段 | 说明 |
|---|---|
| 待办编号 | 系统生成 |
| 待办类型 | 审核、支付、发货、租后等 |
| 关联对象 | 订单、账单、店铺、提现单等 |
| 订单类型 | 商家订单、联营订单、平台订单；非订单待办为空 |
| 优先级 | 普通、重要、紧急 |
| 状态 | 待处理、处理中、已完成、已取消、已超时 |
| 接收角色 | 运营、商家老板、员工、客户、渠道 |
| 接收人 | 具体账号或角色队列 |
| 创建来源 | 人工、系统、回调、定时任务 |
| 截止时间 | 超时预警依据 |
| 处理入口 | 跳转页面和按钮 |
| 处理结果 | 通过、驳回、已补资料、已支付等 |

## 5. 通知渠道

| 渠道 | 用途 | 说明 |
|---|---|---|
| 站内信 | 运营端、商家端、渠道端 | 可追溯，默认启用 |
| 门店手机端推送 | 门店待办、发货、归还、授权 | 适合老板和店员 |
| 小程序通知 | 客户支付、签署、补资料、归还 | 按平台能力配置 |
| 短信 | 关键节点和租后提醒 | 需模板和计费 |
| IM | 平台订单联系客服、审核沟通 | 后续对接客服系统 |
| 邮件 | 对账、导出、系统通知 | 可选 |

同一事件可配置多个通知渠道，但待办只保留一条主任务。

---

## 6. 通知模板

| 字段 | 说明 |
|---|---|
| 模板名称 | 业务可读名称 |
| 适用事件 | 订单审核、合同签署、提现结果等 |
| 接收角色 | 客户、商家、运营、渠道 |
| 渠道 | 短信、站内信、IM、小程序通知 |
| 内容变量 | 订单类型、商品、金额、截止时间、跳转链接 |
| 脱敏规则 | 手机号、姓名、证件号、银行卡 |
| 启用状态 | 启用、停用 |
| 审核状态 | 草稿、待审核、已发布 |

模板规则：

1. 高风险模板需要审核后发布。
2. 模板变量必须来自白名单，不允许拼接敏感明文。
3. 短信模板和站内信模板可以不同，但业务事件一致。
4. 模板版本要保留，历史消息使用发送时版本。

## 7. 待办生成规则

| 事件 | 待办 |
|---|---|
| 客户提交联营订单 | 运营端生成订单审核待办 |
| 客户提交平台订单 | 运营端生成订单审核和资方分配待办 |
| 商家提交商家订单 | 商家端生成自审待办 |
| 审核要求补资料 | C 端或商家端生成补资料待办 |
| 合同发起成功 | C 端生成签署待办 |
| 公证发起成功 | C 端生成公证办理待办 |
| 订单待发货 | 门店端或配置的发货方生成发货待办 |
| 归还申请 | 门店端/运营端生成验收待办 |
| 账单逾期 | 租后人员生成跟进待办 |
| 提现申请 | 运营财务生成提现审核待办 |
| 客诉同步 | 客诉人员生成处理待办 |

---

## 8. 幂等与关闭

| 场景 | 规则 |
|---|---|
| 同一订单重复回调 | 不重复创建待办，只更新状态和日志 |
| 客户已完成支付 | 自动关闭支付待办 |
| 合同过期后重发 | 关闭旧待办，生成新待办 |
| 订单关闭 | 关闭该订单未处理的非必要待办 |
| 手动取消 | 必须填写原因 |
| 超时 | 状态变为已超时，并通知负责人或主管 |

## 9. 各端展示

### 9.1 运营端

| 入口 | 内容 |
|---|---|
| 顶部消息 | 未读通知、异常提醒 |
| 工作台 | 待审核、待发货、财务异常、客诉、租后 |
| 订单详情 | 当前订单所有待办和通知记录 |
| 待办中心 | 全量待办列表、分配、转办、关闭 |

### 9.2 商家端

| 入口 | 内容 |
|---|---|
| 工作台 | 商家订单自审、商品审核结果、发货归还、提现结果 |
| 订单详情 | 分红/平台订单进度、补资料、联系客服 |
| 钱包 | 提现状态、扣费通知，员工不可见 |

### 9.3 门店手机端

| 入口 | 内容 |
|---|---|
| 首页待办 | 发货、交付照片、签收、归还验收、补授权 |
| 订单中心 | 单订单处理入口 |
| 我的消息 | 审核结果、提现结果、授权提醒 |

### 9.4 C 端客户

| 入口 | 内容 |
|---|---|
| 订单详情 | 补资料、扫脸、签合同、公证、支付、签收、归还 |
| 小程序通知 | 关键节点提醒 |

---

## 10. 与日志和计费联动

| 模块 | 联动 |
|---|---|
| 操作日志 | 待办创建、处理、取消、超时都写日志 |
| 回调日志 | 外部回调触发的待办保留回调原始结果摘要 |
| 接口计费 | 短信、风控、合同、公证等通知或动作可生成计费 |
| 数据看板 | 待办数量、处理时长、超时率、异常率 |
| 权限系统 | 待办可见范围、按钮权限、敏感通知脱敏 |
