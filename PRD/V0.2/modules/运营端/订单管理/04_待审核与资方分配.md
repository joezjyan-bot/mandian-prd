## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 运营端待审核与资方分配 |
| 所属端 | 运营端 |
| 入口路径 | 订单管理 > 待审核 / 待分配资方 / 待分配门店 |
| 使用角色 | 审核客服、风控客服、运营主管、财务、平台管理员 |
| 核心目标 | 集中处理分红订单、平台订单的资料审核、风控判断、资方分配、门店分配、调整套餐、确认订单进入支付 |

该页面是分红订单和平台订单的主工作台。门店订单默认不进入平台审核队列,除非发生客诉、异常介入、黑名单命中、平台抽佣异常或平台管理员手动转入。

## 2. 订单范围

| 订单类型 | 是否进入待审核 | 调整套餐 | 分配门店 | 说明 |
|---|---|---|---|---|
| 门店订单 | 默认不进入 | ❌(商家用办单助手) | ❌(扫码即门店) | 商家/门店自审;运营端只监管和异常介入 |
| 分红订单 | 进入 | ✅ | ❌(扫码即门店) | 门店出设备或部分资金,平台分配资方补足资金并审核 |
| 平台订单 | 进入 | ✅ | ✅ | 门店把订单推给平台,平台匹配资方并主控审核财务 |

新系统页面、接口和文案均使用「平台订单」。

## 4. 顶部指标

| 指标 | 说明 |
|---|---|
| 待审核 | 已提交但未给出审核结论 |
| 待分配资方 | 分红/平台订单尚未绑定资方 |
| **待分配门店** ⭐ | **仅平台订单**:资方分配后(或无资方需求时审核通过后)未分配履约门店 |
| 待补资料 | 客户或商家需补充资料 |
| 待客户授权 | 风控/征信/代扣/合同已发起但客户未完成 |
| 待合同 | 已确认订单进入支付但合同未发起或未签署 |
| 待支付 | 客户未完成首期、增值服务或补缴 |
| 异常 | 回调失败、黑名单命中、支付不匹配、资料异常 |

指标必须可点击跳转到对应队列,且数量按当前用户权限实时统计。

## 6. 审核队列

| 队列 | 进入条件 | 主操作 |
|---|---|---|
| 待接单 | 订单提交后未分配审核人员 | 接单、分配审核人员(接单即标记做单客服) |
| 待资料审核 | 客户资料已提交 | 查看资料、调整套餐、补资料、拒绝 |
| 待分配资方 | 分红/平台订单未绑定资方 | 选择资方、生成分账规则 |
| **待分配门店** ⭐ | 平台订单(资方分配后或无资方)未分配履约门店 | 搜索匹配门店、工号二次确认(详见 07 文档) |
| 待风控 | 需要发起或查看风控 | 发起授权、查看报告、写审核意见 |
| 待补资料 | 客户或商家需补资料 | 查看补充状态、催办、重新审核 |
| 待签约支付 | 已"确认订单进入支付"但合同/支付未完成 | 系统自动串行;客服可触发银行卡代扣 |
| 异常队列 | 黑名单、回调失败、支付不匹配、资料冲突 | 主管复核、重试回调、转人工 |

队列不是最终订单状态,而是运营工作台视角的待办归类。

## 8. 审核流程(完整 v1.1)⭐

```
订单提交
  ↓
进入待审核队列
  ↓
接单(系统自动标记"做单客服 XXX")
  ↓
查看客户资料 + 商品价格方案 + 风控摘要
  ↓
判断:是否需要调整套餐?
  ├─ 门店订单 → 不开放(只能核对)
  └─ 分红/平台订单 → 客服可点 [调整套餐]
       ↓
       弹出"类办单助手"界面:
         - 平台订单必须分配履约门店(见 07 文档)
         - 设备价 / 首付 / 期数 / 月供 / 留购价 / 押金
         - 增值服务多选(公证费/设备管理费/碎屏险等,后台配置)
         - 资方与出资比例(分红/平台)
         - 首期一笔支付汇总
       ↓
       [生成办单图片]
         - 系统生成订单图
         - 客服下载图片
         - 通过任意渠道(微信/钉钉)发给商家
         - 商家与客户确认费用
       ↓
       [提交]
         - 订单信息写库
         - 状态仍为 PENDING_REVIEW
         - 等待客户在 IM 回复"确认"
  ↓
客服肉眼判定客户已确认
  ↓
[确认订单进入支付]
  ↓
订单状态:PENDING_REVIEW → PENDING_PAYMENT
后续:系统自动串行(支付 → 合同 → 代扣 → 待发货 → 待收货 → 待归还)
```

**关键约束**:
- 平台订单未分配门店,**不可点"确认订单进入支付"**(合同需带商家主体)
- 客户未确认账单,**不可点"确认订单进入支付"**(避免改价后客户不认)
- 完整 6 阶段流程详见 `08_长租订单全生命周期与客服操作.md`

## 9. 资方分配规则

### 9.1 资方前置

资方必须先在 `资方管理` 中配置完成,至少包括:

| 字段 | 说明 |
|---|---|
| 资方名称 | 展示和筛选使用 |
| 资金账户 | 分账和提现账户 |
| 支持订单类型 | 分红订单、平台订单或两者 |
| 支持类目 | 可投放的商品类目 |
| 可用额度 | 当前可分配额度 |
| 分账规则 | 默认分账账户和结算周期 |
| 状态 | 启用、停用、冻结 |

### 9.2 分红订单分配

分红订单由门店填写设备价和需求配资比例。例如设备价 5000,需求配资比例 80%,则资方出资 4000,**门店出资 1000**,后续客户每一笔还款按 80/20 先分账,再分别扣平台抽佣。

| 字段 | 说明 |
|---|---|
| 设备价 | 办单助手传入,可审核时调整 |
| 资方出资比例 | 20% 到 80% 下拉选择,默认使用办单助手提交值 |
| 门店出资比例 | 100% 减资方出资比例 |
| 资方出资金额 | 设备价 × 资方出资比例 |
| **门店出资金额** | 设备价 × 门店出资比例(措辞统一:"门店出资"不用"等效出资") |
| 平台抽佣 | 默认 2%,可按商家/资方/订单配置 |
| 增值服务 | 多选(公证费/设备管理费/碎屏险等,后台配置)|

### 9.3 平台订单分配

平台订单由平台或资方全额承接资金和审核主控。门店只作为**履约主体**(发货 + 售后),不参与出资。

| 字段 | 说明 |
|---|---|
| 资方 | 可由审核客服选择 |
| 出资比例 | 默认 100% 资方/平台资金 |
| **履约门店** | 审核通过后由客服分配(见 07 文档),分配后不可改派 |
| 门店收益 | 默认无资金收益,可按渠道或服务规则给佣金 |
| 渠道佣金 | 如果来自渠道推广,按渠道规则结算 |
| 发货主体 | 分配的履约门店 |

### 9.4 资方变更

| 节点 | 是否允许变更 | 规则 |
|---|---|---|
| 合同发起前 | 允许 | 普通资方分配权限可操作 |
| 合同已发起未签 | 谨慎允许 | 需要重新发起合同或补充确认 |
| **合同已签署** | **原则不允许** | 如必须变更,需要主管审批和补充合同 |
| **已支付/已分账** | **不允许普通变更** | 走冲正、退款或财务调整流程 |

## 10. 客服 IM 联动 + 办单图片传递

商家 PC 端可以查看自己发起的平台订单进度,并在待审核或补资料节点点击「联系客服」。

| 场景 | 规则 |
|---|---|
| 商家联系平台客服 | 调起 IM,并自动推送订单摘要 |
| 审核客服联系商家 | 从审核台发起会话,带订单号、缺失资料、审核意见 |
| 客服联系客户 | 通过客户侧小程序/短信/IM 触达,记录触达日志 |
| **办单图片传递** ⭐ | 客服 [生成办单图片] → 下载到本地 → 通过微信/钉钉等任意渠道发给商家 → 商家与客户确认费用 → 客户在 IM 回复"确认" → 客服肉眼判定 |

推送给 IM 的订单摘要应包括订单号、订单类型、商家门店、客户脱敏信息、商品、价格方案、当前待办、需要处理的问题,**不推送身份证号、银行卡号等敏感明文**。

**办单图片设计**:
- 系统统一生成(类似商家端办单助手成图)
- 不通过 IM 自动推送(给客服灵活操作空间)
- 客户在 C 端订单中心可见完整订单详情,无需重复推送

## 11. 审核结果

| 结果 | 必填 | 后续动作 |
|---|---|---|
| **确认订单进入支付** ⭐ | 审核意见 | 订单进入 PENDING_PAYMENT,系统自动串行支付/合同/代扣 |
| 审核不通过 | 拒绝原因、通知对象 | 通知客户/商家,按规则关闭或退回 |
| 要求补资料 | 补充资料项、截止时间 | 生成客户或商家待办 |
| 转主管复核 | 复核原因 | 进入异常队列 |
| 暂挂 | 暂挂原因、预计处理时间 | 保持待审核但标记超时规则 |

拒绝原因要结构化,便于后续统计,例如资料不符、风控拒绝、黑名单命中、资方拒绝、客户放弃、商家撤回、重复下单。

## 13. 异常处理

| 异常 | 处理 |
|---|---|
| 资方额度不足 | 禁止分配,提示更换资方或调整额度 |
| 资方被停用 | 已分配未签约订单进入异常队列 |
| **平台订单未分配门店** ⭐ | **阻止"确认订单进入支付"按钮**(合同需带商家主体)|
| **门店收款账户被冻结** | 系统阻止该门店出现在候选列表 |
| 风控报告未返回 | 可重试或人工暂挂 |
| 客户补资料超时 | 提醒商家/客户,超时后可关闭 |
| 支付已完成但合同未签 | 禁止发货,提示补签或退款处理 |
| 合同已签但支付失败 | 保留合同状态,订单待支付 |
| 黑名单命中 | 必须主管复核或强制填写审核意见 |
| **客服调整套餐后客户未确认** | **禁止"确认订单进入支付"**(防止客户后续否认) |
| **AI 问答 / 电子合同收货验收失败** | 客服可手动切换另一种方式(详见 08 §6.2) |
