# 附件中心与敏感文件

> 全局 PRD 草案。
> 目标：统一身份证、营业执照、合同、公证、交付照片、归还验收、风控报告、授权文件等附件的存储、权限、脱敏、引用和日志。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 附件中心与敏感文件 |
| 所属端 | 全局基础能力 |
| 入口路径 | 不单独给普通用户开放，嵌入订单、店铺、商品、设备、客诉、租后等页面 |
| 使用角色 | 运营、审核、商家、门店员工、客户、系统服务 |
| 核心目标 | 所有文件统一存储和鉴权，业务页面只引用附件 ID，不各自保存文件逻辑 |

---

## 2. 核心口径

1. 附件中心是全系统文件总账，业务表只保存附件引用。
2. 身份证、银行卡、营业执照、合同、风控报告等敏感文件必须按权限查看。
3. 图片和文件要保存原件、压缩预览、缩略图、OCR 结果和审核状态。
4. 查看、下载、导出敏感附件必须记录日志。
5. 文件不能只存在第三方临时链接，必须有平台可控引用和有效期策略。
6. 客户、商家、门店、运营上传的附件要记录上传主体和业务场景。
7. 附件删除原则上做逻辑删除或废弃标记，不物理删除业务证据。

## 3. 附件类型

| 类型 | 示例 | 敏感等级 |
|---|---|---|
| 身份证 | 人像面、国徽面 | 高 |
| 营业执照 | 执照照片 | 高 |
| 银行卡/收款账户 | 银行卡照片、授权文件 | 高 |
| 合同文件 | 主合同、补充合同、入驻合同 | 高 |
| 授权文件 | e 签宝授权、风控授权、收款授权 | 高 |
| 风控报告 | 大数据、流水、征信授权结果 | 高 |
| 商品图片 | 商品主图、详情图 | 低 |
| 交付照片 | 设备照、人机合照、人车合照 | 中 |
| 签收照片 | 客户签收、回执 | 中 |
| 归还验收 | 归还照片、损坏照片、配件照 | 中 |
| 客诉附件 | 投诉图片、客户补充材料 | 中/高 |
| 维修附件 | 维修照片、维修单 | 中 |
| 导入文件 | 黑名单导入、设备导入 | 高 |

## 4. 附件字段

| 字段 | 说明 |
|---|---|
| attachment_id | 附件 ID |
| business_type | 业务类型：订单、店铺、设备、客诉等 |
| business_id | 业务对象 ID |
| file_type | 图片、PDF、视频、表格、其他 |
| attachment_type | 身份证、合同、交付照片等 |
| sensitive_level | 低、中、高 |
| uploader_type | 客户、商家、门店员工、运营、系统 |
| uploader_id | 上传人 |
| original_name | 原始文件名 |
| storage_key | 存储路径或对象 key |
| preview_key | 预览图 |
| thumbnail_key | 缩略图 |
| file_hash | 文件哈希，去重和防篡改 |
| file_size | 文件大小 |
| ocr_status | 未识别、识别中、成功、失败 |
| audit_status | 待审核、通过、驳回 |
| status | 正常、废弃、删除 |
| created_at | 上传时间 |

## 6. 权限模型

| 角色 | 默认权限 |
|---|---|
| 平台管理员 | 可按权限查看全部，敏感下载记录日志 |
| 审核客服 | 查看审核所需附件，敏感字段脱敏 |
| 财务 | 查看收款、合同、退款相关附件 |
| 商家老板 | 查看自己商家的订单和店铺附件 |
| 商家员工 | 只看自己权限内订单的交付/归还附件 |
| 客户 | 只看自己订单相关合同、凭证、验收结果 |
| 渠道 | 默认不看客户附件 |
| 资方 | 后续只看自己订单必要合同和账单附件 |

## 7. 敏感控制

| 能力 | 规则 |
|---|---|
| 预览 | 按权限展示，敏感图可加水印 |
| 下载 | 高权限，必须记录下载日志 |
| 分享链接 | 必须有短有效期和签名 |
| 脱敏展示 | 身份证、银行卡、手机号默认脱敏 |
| 水印 | 敏感附件预览叠加账号和时间水印 |
| 防枚举 | 附件 URL 不能用连续 ID 直接访问 |
| 过期 | 临时访问链接自动过期 |
| 导出 | 导出敏感附件需审批或高权限 |

## 8. OCR 与识别

| 附件 | OCR 字段 |
|---|---|
| 营业执照 | 主体名称、统一社会信用代码、法人、地址、有效期 |
| 身份证 | 姓名、身份证号、有效期、地址 |
| 银行卡 | 卡号、银行、姓名 |
| 物流凭证 | 物流单号、公司 |
| 设备照片 | SN/IMEI/VIN 可选识别 |

OCR 结果要保存原始识别值、用户修正值、修正人、修正时间。

---

## 9. 附件审核

| 状态 | 说明 |
|---|---|
| 待审核 | 上传后待人工或系统审核 |
| 通过 | 可用于业务流程 |
| 驳回 | 展示驳回原因，可重传 |
| 需补充 | 不完全驳回，要求补充 |
| 废弃 | 历史附件保留但不再使用 |

驳回后新附件和旧附件都要保留引用，方便追溯。
