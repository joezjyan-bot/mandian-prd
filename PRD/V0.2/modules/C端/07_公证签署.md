# 公证签署

> **来源**: P1-4 / P1-5

## 1. 业务背景

合同签署完成后,可选择是否发起公证。公证需要客户本人在电子签通道完成公证签署。

## 2. 触发节点

- 合同签署完成后,客服 / 门店发起公证(详见 02_公证服务管理)
- 系统调公证服务接口生成公证文书
- 推送 C 端通知客户去签

## 3. 客户签署流程

```
1. 客户在小程序订单详情看"去签公证"按钮
2. 点击 → 跳转公证通道
3. 阅读公证书内容
4. 人脸识别 + 验证码
5. 签署完成 → 公证机构出具公证书 PDF
6. 系统存档(随合同副本一起)
```

## 4. 客户感知

- 不强制(可选环节)
- 加强法律保障
- 如客户拒签,订单仍可继续(合同有效,只是无公证)

## 5. 失败处理

| 场景 | 处理 |
|---|---|
| 公证服务方拒绝 | 客服核查原因 |
| 公证超时 | 重新发起或人工跟进 |
| 客户拒签 | 订单继续(无公证) |

## 6. 数据模型

订单合同记录中:
- `notary_status`
- `notary_initiator`
- `notary_completed_at`
- `notary_document_url`
