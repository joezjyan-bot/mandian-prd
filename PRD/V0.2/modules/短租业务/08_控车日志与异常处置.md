# 体验租 · 08 控车日志与异常处置

> 解决 P0 问题 Q8:控车操作散落在多份文档,但**没有独立日志表 + 失败兜底设计**,导致客户投诉时无证据可查、接口异常时订单状态错乱。
> 本文统一定义控车数据表、操作权限、失败处理、异常处置流程。

---

## 1. 控车能力清单

| 能力 | action 编码 | 适用车辆 | 风险等级 |
|---|---|---|---|
| 开机 | `power_on` | 全部 | 低 |
| 关机 | `power_off` | 全部 | 中(在租中关机影响客户) |
| 闪灯鸣笛(找车) | `find_vehicle` | 电动车 | 低 |
| 查询定位 | `locate` | 全部 | 低(高频) |
| 锁定座桶 | `lock_seat` | 部分电动车 | 低 |
| 解锁座桶 | `unlock_seat` | 部分电动车 | 低 |
| 远程锁车(禁骑) | `lockdown` | 全部 | **高**(强制停止客户使用) |
| 解除锁车 | `unlock` | 全部 | 中 |
| 限速 | `speed_limit` | 部分电动车 | 中 |
| 解除限速 | `unlock_speed` | 部分电动车 | 中 |
| 切断电源(终极) | `kill_switch` | 部分电动车 | **极高**(运营级审批) |

## 2. 控车日志表

### 2.1 `short_rent_control_log`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| log_id | bigint | 是 | 主键 |
| order_id | bigint | 否 | 关联订单(空时为后台主动控车) |
| vehicle_id | bigint | 是 | 关联车辆 |
| device_no | varchar(64) | 是 | 中控设备号(冗余,便于排障) |
| action | enum | 是 | §1 中的 action 编码 |
| risk_level | enum | 是 | low / medium / high / extreme |
| operator_type | enum | 是 | customer / store_staff / merchant_admin / platform_operation / system |
| operator_id | bigint | 否 | 操作人 ID(operator_type=system 时为空) |
| trigger_source | enum | 是 | manual(人工)/ auto_overdue(逾期自动)/ auto_geofence(围栏自动)/ api_callback(回调) |
| operation_reason | varchar(255) | 否 | 操作原因(高/极高风险必填) |
| client_ip | varchar(45) | 否 | 操作人 IP |
| user_agent | varchar(255) | 否 | 操作端 UA |
| channel_request_id | varchar(64) | 否 | 调用中控台的请求 ID |
| channel_response_status | enum | 是 | success / fail / timeout / unknown / pending_callback |
| channel_response_code | varchar(32) | 否 | 中控台返回码 |
| channel_response_msg | text | 否 | 中控台返回原始信息 |
| channel_response_payload | json | 否 | 中控台返回完整 payload |
| vehicle_status_before | varchar(32) | 否 | 操作前车辆状态 |
| vehicle_status_after | varchar(32) | 否 | 操作后车辆状态 |
| retried_count | int | 是 | 重试次数 |
| retry_of_log_id | bigint | 否 | 若为重试,关联首次日志 |
| approved_by_id | bigint | 否 | 二次确认人(高/极高风险) |
| approved_at | datetime | 否 | 二次确认时间 |
| created_at | datetime | 是 | 操作时间 |

### 2.2 索引建议

- (vehicle_id, created_at) — 车辆维度时间线查询
- (order_id, created_at) — 订单维度
- (operator_id, created_at) — 操作人维度
- (channel_response_status, created_at) — 故障排查

### 2.3 日志保留期

- **永久保留**(用于审计、客诉、监管)
- 1 年内热数据;1 年以上转冷存储但可查

## 4. 远程锁车(高风险动作)详细规则

### 4.1 触发场景

| 场景 | 主体 | 自动 / 人工 |
|---|---|---|
| 客户严重逾期(超过 24 小时)| 系统自动 | auto_overdue |
| 客户进入禁行区域 | 系统自动 | auto_geofence |
| 客户失联且超时 | 门店 / 运营 | manual |
| 车辆被盗报警 | 运营 | manual |
| 客户违规驾驶投诉 | 运营 | manual |

### 4.2 锁车前的告知

**强制要求**:除"车辆被盗"场景外,锁车前必须先告知客户。

| 步骤 | 时序 |
|---|---|
| 1. IM + 短信 + 推送 通知客户 | 锁车前 15 分钟 |
| 2. 客户响应 / 处理订单 | 15 分钟窗口 |
| 3. 仍未处理 → 执行锁车 | 锁车后再次推送告知 |

**例外**:车辆被盗场景跳过告知,直接锁车。

### 4.3 锁车后的客户文案

C 端订单详情:

```
⚠️ 车辆已被远程锁定
原因:订单已逾期 25 小时,请立即联系商家处理
锁定时间:2026-05-25 14:30
联系商家:[商家电话]
联系平台客服:[平台电话]
```

不展示锁车操作员姓名;客户感知统一是"平台已锁车"。

## 5. 失败兜底与重试

### 5.1 中控接口失败处理

| 失败类型 | 处理 |
|---|---|
| HTTP 超时(无响应)| 标记 `channel_response_status = timeout`;**不改订单状态**;3 秒后重试 1 次 |
| HTTP 5xx | 同上,5 秒后重试 |
| HTTP 4xx(参数错误)| 标记 `fail`,**不重试**,记录详细错误,人工介入 |
| 中控返回业务失败码 | 标记 `fail`,**不重试**,前端提示具体原因 |
| 重试 3 次仍失败 | 进入异常工单,通知运营 |

### 5.2 状态机的"失败一致性"原则

**核心原则**:中控接口失败 ≠ 订单状态变更失败。

| 场景 | 错误做法 ❌ | 正确做法 ✅ |
|---|---|---|
| 客户点"开机"但中控失败 | 订单状态变成 in_use | 订单状态保留 pending_pickup;提示客户"开机失败,请联系商家" |
| 客户点"还车"但锁车失败 | 直接进入 pending_return_check | 保留 in_use 状态;通知门店人工介入 |
| 平台自动锁车失败 | 订单标记已锁车 | 不标记;转人工工单 |

### 5.3 异步回调

中控台部分动作(如开机)是异步的:发起请求 → 中控立即返回 `pending_callback` → 中控完成动作后回调平台。

| 字段 | 处理 |
|---|---|
| 发起时记录 | channel_response_status = pending_callback |
| 回调到达时更新 | channel_response_status = success / fail |
| 回调超时(默认 5 分钟)| 主动查询中控状态 → 更新结果 |
| 查询仍未果 | 进异常工单 |

## 6. 异常处置流程

### 6.1 异常类型

| 异常 | 触发 | 处置 |
|---|---|---|
| 控车接口连续失败 | 重试 3 次 | 进异常工单 |
| 车辆离线 > 1 小时 | 中控心跳丢失 | 通知门店检查 |
| GPS 异常(漂移/无信号)| 定位失败 | 标记 GPS 异常,不影响其他控车 |
| 车辆未授权区域 | 地理围栏(V2) | 推送客户提示,人工介入 |
| 控车日志与订单状态不一致 | 系统校验 | 进异常工单,人工核对 |

### 6.2 异常工单

```sql
short_rent_control_exception_ticket
- ticket_id           bigint PK
- log_id              bigint FK -- 关联控车日志
- order_id            bigint FK
- vehicle_id          bigint FK
- exception_type      enum
- detail              text
- priority            enum -- urgent / high / normal / low
- assigned_to         bigint
- status              enum -- pending / processing / resolved / closed
- resolution          text
- resolved_at         datetime
- created_at / updated_at
```

工单进入运营端「客诉管理 / 异常处置」模块。

## 9. 与中控台对接

中控台是独立 Java 服务(通过 HTTP API + Webhook 对接 PHP 租赁系统)。本模块约束:

| 对接点 | 说明 |
|---|---|
| 调用方式 | PHP 通过 Guzzle 调用中控台 HTTP API |
| 鉴权 | API Key + 签名 + 时间戳 |
| 超时 | 默认 5 秒,可配置 |
| 异步回调 | 中控台通过 Webhook 回调到 PHP `/api/v1/control/callback` |
| 幂等 | 每个调用带 `channel_request_id`,中控台保证幂等 |

详细对接见中控台 API 文档(独立仓库)。
