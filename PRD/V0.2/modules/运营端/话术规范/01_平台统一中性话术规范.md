# 平台统一中性话术规范

> P0 业务文档(2026-05-27)。
> 定义满点租赁平台 C 端 / 商家端 / 后台的统一话术规范,确保:
> - **C 端客户感知层用统一中性话术**(不区分租赁/赊销/三模式),技术实现简单,客户体验一致
> - **关键金融动作按内部业务话术模板差异化**(留购 vs 提前结清),符合对应合同要求
> - **合同 PDF 按资金来源完全切换**(租赁合同 / 银行赊销合同),法律严谨
> - **后台财务保留法律严谨术语**(应收账款受让 / 服务费 / 投资收益),便于审计

---

## 1. 话术分层架构

```
┌────────────────────────────────────────────────────────────┐
│ 第 1 层 客户合同 PDF(法律严谨)                              │
│   - 按 funding_source 切换合同模板                          │
│   - 平台自营 → 《租赁服务协议》(租金/出租方/承租方)         │
│   - 蓝海银行 → 《分期付款销售合同》(按银行模板展示合同主体) │
├────────────────────────────────────────────────────────────┤
│ 第 2 层 后台财务记账(法律严谨)                              │
│   - 科目:租赁收入 / 服务费收入 / 投资收益 / 应收账款受让    │
│   - 流水严格区分本金 / 利息 / 服务费                        │
├────────────────────────────────────────────────────────────┤
│ 第 3 层 商家端 UI(业务通用术语 + 统一中性)                  │
│   - 我的钱包 / 订单结算款 / 月度分账                        │
│   - 三种合作模式:商家订单 / 联营订单 / 平台订单            │
│   - 商家不感知内部业务话术模板差异                          │
├────────────────────────────────────────────────────────────┤
│ 第 4 层 客户 C 端 UI(统一中性话术)⭐                       │
│   - 基础字眼完全统一(订单总额/每期应付/履约中等)            │
│   - 关键按钮按内部业务话术模板差异化(留购 vs 提前结清)       │
│   - 弹窗说明、推送话术按内部业务话术模板差异化               │
└────────────────────────────────────────────────────────────┘
```

## 2. 客户 C 端统一中性话术(完整对照表)

### 2.1 已锁定的中性词(三模式 + 多内部业务场景全部适用)

| 业务概念 | 统一中性词 | 备注 |
|---|---|---|
| 总款项 | **订单总额** | - |
| 每期款项 | **每期应付** | - |
| 首笔款项 | **首期应付** | 包含保证金 + 首期 + 服务费等 |
| 设备 | **设备** | 不用"租赁物"也不用"已购设备" |
| 期限 | **订单周期** | 不用"租期"也不用"账单支付期" |
| 账单 | **账单** | 不区分"租金账单"和"账单支付账单" |
| 保证金 | **保证金** | 中性,两种场景都适用 |
| 到期日 | **到期日** | 客户最熟悉,无歧义 |
| 逾期罚款 | **逾期费用** | 直接,不带"逾期费用 / 利息"等术语 |
| 设备保管 | **设备使用** | - |

### 2.2 订单状态(C 端展示)

| 状态 | 统一中性词 |
|---|---|
| 创建中 / 待审核 | **审核中** |
| 待付款 | **待付款** |
| 待签约 | **待签约** |
| 已发货 | **已发货** |
| 待收货 | **待收货** |
| 履约中(在租中 / 账单支付中)| **履约中** |
| 已完成 | **已完成** |
| 已取消 | **已取消** |

**完整 C 端 8 状态详见** `订单管理/09_C端订单状态与账单支付.md`。

### 2.3 后台财务术语 vs 商家端 UI vs C 端 UI 对照

| 业务概念 | 后台财务(法律严谨)| 商家端 UI(业务通用)| C 端 UI(统一中性) |
|---|---|---|---|
| 客户每月支付的钱 | 客户租金(=设备使用费+平台服务费) | - | 每期应付 |
| 平台收的服务费 | 平台综合服务费收入 | - | (不展示,包含在每期应付里) |
| 联营订单平台出资款 | 联营投资款 / 应收账款转让对价 | 订单结算款 | (后台动作,客户不感知) |
| 平台订单应收账款转让对价 | 应收账款受让对价 | 订单结算款 | 同上 |
| 客户每月分给门店的钱 | 设备使用费回款 | 月度分账 | 同上 |
| 联营订单平台获得的份额 | 联营投资收益 / 应收账款受让回款 | (商家不可见,后台展示)| 同上 |
| 内部资金来源对应收账款的支付 | 应收账款受让方付款 | (商家不可见,后台展示) | 同上 |
| 撤单后门店退还平台的钱 | 应收账款转让对价退回 | 撤单退款扣减 | 同上 |
| 设备所有权转移给客户 | 留购(租赁)/ 已结清(赊销) | - | 见 §3 按内部业务话术模板差异化 |

## 3. 关键按钮 + 弹窗按内部业务话术模板差异化

C 端 UI 基础字眼统一,不展示资金来源、合作模式或内部资金来源主体;但**4 个关键差异点**需要按 `order.terminology_template_id` 切换:

### 3.1 差异化字眼总表

| 业务点 | 默认(平台自营/租赁话术) | 蓝海银行(赊销话术) |
|---|---|---|
| 提前结清按钮 | **申请留购** | **提前结清** |
| 提前结清弹窗标题 | **留购确认** | **提前结清确认** |
| 提前结清弹窗说明 | 留购成功后,设备所有权将归您所有 | 结清后,本订单已全部还清,不再产生费用 |

## 4. 内部业务话术模板机制

### 4.1 数据库设计

```sql
terminology_template
- template_id             bigint PK
- template_name           varchar(64)   -- "标准租赁话术" / "蓝海银行赊销话术"
- description             varchar(255)
-- 差异化字眼配置
- term_early_settle_button   varchar(32)  -- "申请留购" / "提前结清"
- term_early_settle_title    varchar(32)  -- "留购确认" / "提前结清确认"
- term_early_settle_desc     text         -- 弹窗说明
- term_completion_msg        varchar(255) -- 完成订单提示
- term_overdue_push          varchar(255) -- 逾期推送话术
- term_due_reminder_push     varchar(255) -- 到期提醒推送
- status                  enum            -- active / draft / archived
- created_by              bigint
- created_at / updated_at
```

### 4.2 MVP 预设两套模板

| 模板 | 适用场景 | 关键字眼 |
|---|---|---|
| **标准租赁话术** | 平台自营资金的所有订单 | 申请留购 / 留购确认 / 设备所有权归您 |
| **蓝海银行赊销话术** | 蓝海银行内部业务模板订单 | 提前结清 / 提前结清确认 / 已全部还清 |

### 4.3 模板绑定流程

```
内部资金来源管理 > 添加/编辑主体
   ↓
[必填] 选择话术模板(下拉:租赁话术 / 蓝海银行赊销话术 / ...)
   ↓
保存内部资金来源主体
   ↓
该内部资金来源主体关联的订单自动继承 terminology_template_id
   ↓
C 端 UI 渲染时根据 terminology_template_id 查询对应字眼
```

## 5. 商家端 UI 话术(完全统一,不按资方切换)

### 5.1 核心原则

**商家不需要感知客户合同是租赁还是赊销** — 商家的关心点是:
- 这单我需要出多少钱(合作模式决定)
- 这单我能拿多少钱(分账规则决定)
- 钱什么时候到账(通联代付决定)

**内部资金来源不影响商家收益展示**(只影响后台合同/话术模板),所以**商家端 UI 全部用统一中性词**。

### 5.2 商家端用语统一规范

| 业务概念 | 商家端 UI 用语 |
|---|---|
| 商家钱包 | **我的钱包** |
| 月度租金回款 | **月度分账** |
| 联营/平台订单出资款入账 | **订单结算款** |
| 平台逾期费用扣减 | **平台扣款** |
| 撤单退款扣减 | **撤单退款扣减** |
| 三种合作模式 | **商家订单 / 联营订单 / 平台订单** |
| 客户每期支付的钱 | **客户支付** / **每期应付** |
| 商家应得部分 | **本期分账金额** |

### 5.3 商家端资金相关展示边界

商家端不展示内部资金来源主体、内部资金来源主体名称、资金来源、平台利润和返点规则。商家只需要看到订单类型、结算状态、应结算金额、已结算金额、扣减/冲正原因和钱包流水。

```
订单详情(商家端):
┌────────────────────────────────────────┐
│ 订单 OD2026001 - 平台订单                │
│ 客户:张** | 设备:iPhone 17 Pro Max     │
│ 订单总额:¥5,940                         │
│ 结算状态:待结算 / 已结算 / 冲正中       │
│ ...                                      │
└────────────────────────────────────────┘
```

## 6. 后台财务术语规范(法律严谨)

### 6.1 财务科目对照

| 业务行为 | 后台会计科目 |
|---|---|
| 平台收取客户综合服务费 | 主营业务收入 - 平台综合服务费 |
| 联营订单平台出资款 | 长期股权投资 - 联营投资(方案 A)|
| 联营订单平台分得收益 | 投资收益 - 联营投资收益 |
| 平台订单应收账款受让对价 | 应收账款 - 受让应收 |
| 平台订单应收账款受让溢价 | 主营业务收入 - 受让溢价收益 |
| 内部资金来源清算冲正 | 应收账款 - 受让应收(冲销)|
| 平台向商家代付订单结算款 | 其他应付款 - 商家结算款 |

### 6.2 流水严格区分

每笔流水必须有:
- `cooperation_mode`(合作模式)
- `funding_source`(资金来源)
- `funder_id`(若涉及内部资金来源主体,仅后台可见)
- `entry_type`(财务对象类型)

便于审计时按维度统计。
