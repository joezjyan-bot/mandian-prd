# 门店移动端：钱包与对账

## 入口与路由

| 页面 | 入口 | 路由 |
|---|---|---|
| 我的钱包 | 工作台 `我的钱包` | `/pages/shopManage/wallet` |
| 余额/佣金/充值/配资明细 | 钱包内明细入口 | `/pages/shopManage/walletDetail` |

## 点击反馈

| 控件 | 点击结果 | 本次动作 |
|---|---|---|
| 分成余额提现 | 余额为 0 时 Toast：当前暂无可提现余额 | 未进入提现提交 |
| 佣金余额提现 | 余额为 0 时 Toast：当前暂无可提现余额 | 未进入提现提交 |
| 在线充值 | 打开底部弹窗 `配资余额充值` | 关闭，未确认 |
| 余额明细 | 进入明细页，Tab 默认或切到对应账户 | 只读查看 |
| 佣金明细 | 进入明细页，展示佣金相关流水 | 只读查看 |
| 充值明细 | 进入明细页，展示配资充值/扣减流水 | 只读查看 |
| 提现明细 | 进入提现记录页 | 只读查看 |
| 配资额度明细 | 进入配资账户流水 | 只读查看 |

## 充值弹窗

```text
配资余额充值
├─ 当前配比：充值金额将按比例放大入账
├─ 充值金额输入框
└─ 确认充值
```

新系统要求：

1. 确认充值前展示充值金额、放大后额度、支付方式、手续费、到账规则。
2. 创建充值订单后跳转支付，不应仅靠前端金额。
3. 充值成功回调后入账，失败/超时保持待支付或关闭。
4. 配资额度变动必须生成资金流水。

## 资金权限

| 角色 | 可看余额 | 可提现 | 可充值 | 可看流水 |
|---|---|---|---|---|
| 门店老板 | 是 | 是 | 是 | 是 |
| 店长 | 可配置 | 否 | 否 | 可配置 |
| 办单员 | 否 | 否 | 否 | 否 |

## 风险控制

1. 提现必须校验收款账号实名、店铺状态、可提现余额、冻结金额和风控限制。
2. 充值、提现、配资扣减、分成入账都要有资金流水和审计日志。
3. 员工账号默认看不到钱包入口。
4. 对账单导出如后续增加，应进入统一导出中心并脱敏。
