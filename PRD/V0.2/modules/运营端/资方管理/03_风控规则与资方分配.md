# 风控规则与资方分配

> [V0.2.2 同步声明] 本目录历史命名“资方管理”保留为内部配置入口兼容。当前业务语义统一为**内部资金来源管理**;相关信息仅运营端、财务端、风控端可见,不得进入 C 端、商家端、客服话术或合同正文。

> P0 业务文档(2026-05-27)。
> 定义订单审批环节的资方自动分配逻辑:风控规则定义 + 系统推荐 + 运营审核确认。
>
> **核心原则**:
> - 商家提交订单时**只选择合作模式**(商家/联营/平台订单)
> - 资方分配是**平台后台运营的决策**,商家不感知具体资方
> - 系统按风控规则自动推荐资方,运营审核员可调整最终分配

## 1. 资方分配总体流程

```
┌────────────────────────────────────────────────────────────────┐
│ Step 1  门店提交订单                                              │
│   - 选择合作模式:商家订单 / 联营订单 / 平台订单                  │
│   - 上传客户资料(身份证 / 信用授权 等)                          │
│   - 提交审核                                                      │
├────────────────────────────────────────────────────────────────┤
│ Step 2  系统执行风控审核                                          │
│   - 客户身份验证                                                  │
│   - 客户信用评分查询                                              │
│   - 设备价 / 周期 等订单参数合规检查                              │
│   - 反欺诈检查                                                    │
├────────────────────────────────────────────────────────────────┤
│ Step 3  系统按风控规则推荐资方(仅联营订单 / 平台订单)             │
│   - 商家订单:不涉及资方分配,跳过                                 │
│   - 联营订单:默认平台自营资金                                     │
│   - 平台订单:按风控规则匹配最优资方(优先级排序)                  │
├────────────────────────────────────────────────────────────────┤
│ Step 4  运营审核员审核                                            │
│   - 查看系统推荐                                                  │
│   - 可调整资方分配                                                │
│   - 通过 / 驳回订单                                                │
├────────────────────────────────────────────────────────────────┤
│ Step 5  订单生效                                                  │
│   - 锁定 funder_id / contract_template_id / terminology_template_id│
│   - 进入待付款阶段                                                │
└────────────────────────────────────────────────────────────────┘
```

## 2. 数据模型

### 2.1 `risk_rule` 风控规则

```sql
risk_rule
- rule_id                 bigint PK
- rule_name               varchar(64)   -- "蓝海银行风控规则" / "平台自营风控规则"
- rule_code               varchar(32)   -- 规则编码
- funder_id               bigint FK     -- 关联资方(一对一)
- priority                int           -- 推荐优先级(越小越优先)

-- 客户准入条件
- min_customer_age        int           -- 最小年龄(默认 18)
- max_customer_age        int           -- 最大年龄(默认 60)
- min_credit_score        int           -- 最低信用评分(默认 600)
- required_id_type        enum          -- mainland_id / hk_passport / passport

-- 订单参数限制
- min_order_amount        decimal(18,2) -- 最小订单金额
- max_order_amount        decimal(18,2) -- 最大订单金额
- min_period_months       int           -- 最短订单周期
- max_period_months       int           -- 最长订单周期
- allowed_device_types    json          -- 允许的设备类型(空 = 全部)

-- 反欺诈规则
- blacklist_check         boolean       -- 是否检查黑名单
- multi_device_per_customer_limit  int  -- 同一客户最大同时持有订单数
- multi_customer_per_device_limit  int  -- 同一设备最大已售订单数(防止重复销售)

-- 状态
- status                  enum          -- active / paused / archived
- created_by              bigint
- created_at / updated_at

-- 索引:(funder_id) 唯一约束 / (status, priority)
```

### 2.2 `order_audit` 订单审核记录

```sql
order_audit
- audit_id                bigint PK
- order_id                bigint FK
- audit_type              enum          -- system_risk_check(系统风控)/ funder_match(资方匹配)/ manual_review(人工审核)

-- 系统风控结果
- risk_check_passed       boolean
- risk_check_details      json          -- 风控通过的条件清单 + 任何失败项
- customer_credit_score   int

-- 资方推荐(若适用)
- recommended_funder_id   bigint        -- 系统推荐的资方
- recommended_reason      text          -- 推荐理由(满足哪些规则 / 额度足够等)
- alternative_funders     json          -- 备选资方列表

-- 人工审核
- assigned_funder_id      bigint        -- 最终分配的资方(可能 = 推荐,可能调整)
- audit_result            enum          -- approved / rejected
- audit_comment           text
- auditor_id              bigint
- audited_at              datetime

- created_at / updated_at
```

## 4. 系统资方推荐算法

### 4.1 推荐流程

```python
def recommend_funder(order):
    """系统为订单推荐最优资方"""
    
    # Step 1:按 cooperation_mode 判断是否需要资方
    if order.cooperation_mode == 'self_operate':
        return None  # 商家订单不需要资方
    
    if order.cooperation_mode == 'joint_venture':
        # 联营订单默认平台自营资金(MVP 阶段)
        return get_platform_self_funder()
    
    # Step 2:平台订单 — 按优先级匹配资方
    candidate_funders = []
    
    # 查询所有 active 状态的资方
    active_funders = Funder.query.filter(
        Funder.status == 'active'
    ).order_by(
        Funder.priority.asc()  # 优先级排序
    )
    
    for funder in active_funders:
        rule = funder.risk_rule
        
        # 风控规则匹配
        if not match_risk_rule(order, rule):
            continue
        
        # 授信额度检查
        if funder.credit_limit_left < order.device_price:
            continue
        
        candidate_funders.append({
            'funder': funder,
            'reason': '满足风控规则 + 授信额度充足'
        })
    
    if not candidate_funders:
        # 没有匹配的外部资方 → 降级到平台自营
        return {
            'funder': get_platform_self_funder(),
            'reason': '无外部资方匹配,降级平台自营',
            'alternative': []
        }
    
    # 返回最高优先级的资方 + 备选列表
    return {
        'funder': candidate_funders[0]['funder'],
        'reason': candidate_funders[0]['reason'],
        'alternative': candidate_funders[1:5]  # 最多 5 个备选
    }


def match_risk_rule(order, rule):
    """检查订单是否满足风控规则"""
    customer = order.customer
    
    # 年龄
    if not (rule.min_customer_age <= customer.age <= rule.max_customer_age):
        return False
    
    # 信用评分
    if customer.credit_score < rule.min_credit_score:
        return False
    
    # 订单金额
    if not (rule.min_order_amount <= order.device_price <= rule.max_order_amount):
        return False
    
    # 订单周期
    if not (rule.min_period_months <= order.period_months <= rule.max_period_months):
        return False
    
    # 设备类型
    if rule.allowed_device_types and order.device_type not in rule.allowed_device_types:
        return False
    
    # 黑名单 + 反欺诈
    if rule.blacklist_check and is_blacklisted(customer):
        return False
    
    if customer.active_order_count >= rule.multi_device_per_customer_limit:
        return False
    
    return True
```

### 5.3 审核后的处理

#### 按推荐分配

```
点击 [按推荐分配] → 系统执行:
   1. order.funder_id = 蓝海银行
   2. order.contract_template_id = 蓝海银行赊销合同 v2.3
   3. order.terminology_template_id = 蓝海银行赊销话术
   4. order.funding_source = 'funder_blue_ocean'
   5. order.status = APPROVED → 进入待付款
   6. funder_receivable_ledger 预创建(状态 pending)
   7. 商家端 IM 推送:"订单已通过审核"
```

#### 手动选择其他资方

```
点击 [手动选择其他资方] → 弹窗显示可选资方列表
审核员选定 → 同上流程,但 funder_id 改为审核员选定的资方
```

#### 驳回订单

```
点击 [驳回订单] → 必填驳回原因 → 订单状态 REJECTED
   商家端 IM 推送:"订单未通过审核,原因:XXX"
```

## 6. 资方降级策略

### 6.1 自动降级场景

| 场景 | 降级动作 |
|---|---|
| 推荐资方授信额度不足 | 降级到下一优先级资方,或平台自营 |
| 推荐资方临时暂停 | 同上 |
| 推荐资方风控规则被关闭 | 同上 |
| 所有外部资方都不匹配 | 默认平台自营资金 |

### 6.2 手动驳回 + 重新分配

如果运营审核员认为系统推荐不合适,可:
1. 手动选择其他资方
2. 改成平台自营资金
3. 驳回订单(让门店重新提交)

## 8. 异常处理

| 异常 | 处理 |
|---|---|
| 系统风控审核失败(任一条件不满足)| 订单不进入资方推荐,直接进入"系统风控待审"队列,需主管审核 |
| 没有任何资方匹配 | 降级到平台自营资金,提示审核员 |
| 资方授信额度刚好被其他订单占用 | 系统重新计算,降级或推迟分配 |
| 同一订单重复审核 | 系统幂等,只处理首次 |
