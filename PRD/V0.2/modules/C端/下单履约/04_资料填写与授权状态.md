# C 端资料填写与授权状态

> 页面级 PRD 草案。
> 目标：把客户下单过程中的实名、身份证、联系人、地址、定位、人脸、信用/信息授权、征信授权、代扣绑卡等资料节点拆成独立页面和状态。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 资料填写与授权状态 |
| 所属端 | C 端小程序/H5/APP |
| 入口路径 | 扫码下单 > 资料填写；订单详情 > 待补资料/待授权 |
| 使用角色 | C 端客户 |
| 核心目标 | 客户按后台配置补齐资料和授权，后台审核可明确看到每个资料项状态和失败原因 |

---

## 2. 核心口径

1. 资料项由后台按订单类型、商品类目、审核策略配置，不在前端写死。
2. 客户资料填写状态要拆成“未填、草稿、已提交、需补充、已通过、已驳回”。
3. 身份证、银行卡、手机号等敏感数据前端和日志默认脱敏。
4. 人脸、信用/信息授权、征信授权、流水报告授权都作为客户待办，完成后回写订单。
5. 客户补资料后，订单回到对应审核队列，不能直接跳过审核。
6. 资料和授权状态要展示给客户，但不展示内部信用评估结论和后台内部规则规则。

---

## 3. 页面结构

```text
资料填写
├─ 实名信息
├─ 身份证上传
├─ 联系方式
├─ 地址与定位
├─ 紧急联系人
├─ 补充资料
├─ 人脸识别
├─ 信用评估/征信授权
├─ 代扣绑卡
└─ 提交审核
```

订单详情待办中可重新进入某个未完成或被驳回的资料项。

---

## 4. 实名信息

| 字段 | 类型 | 说明 |
|---|---|---|
| 姓名 | 文本 | 必填 |
| 身份证号 | 文本 | 必填，输入后脱敏展示 |
| 手机号 | 文本 | 默认当前登录手机号 |
| 验证码 | 文本 | 如需验证手机号 |
| 实名校验状态 | 状态 | 未校验、校验中、通过、失败 |

规则：

1. 实名失败可允许重试，超过次数转人工。
2. 实名信息变更后，已完成的人脸、授权、信用评估可能需要重新发起。
3. 实名通过结果写入订单资料快照。

---

## 5. 身份证上传

| 字段 | 类型 | 说明 |
|---|---|---|
| 身份证人像面 | 图片上传 | 可拍照/相册 |
| 身份证国徽面 | 图片上传 | 可拍照/相册 |
| OCR 姓名 | 识别/确认 | 客户确认 |
| OCR 身份证号 | 识别/确认 | 脱敏展示 |
| 有效期 | 识别/确认 | 过期提示 |
| OCR 状态 | 状态 | 成功、失败、人工填写 |

OCR 失败允许人工填写，但运营端审核资料页必须标记“人工填写/识别失败”。

---

## 6. 联系人与地址

| 字段 | 类型 | 说明 |
|---|---|---|
| 联系手机号 | 文本 | 默认登录手机号 |
| 收货地址 | 地址选择/文本 | 默认门店发货也可用于信用评估和配送 |
| 使用地址 | 地址选择/文本 | 短租或车辆使用地 |
| 地图定位 | 授权/选择 | 获取客户当前位置或使用地 |
| 邮箱 | 文本 | 合同通知可选 |
| 紧急联系人 | 文本 | 按信用评估配置 |
| 紧急联系人手机号 | 文本 | 按信用评估配置 |

定位失败时允许继续提交还是强制补齐，由审核策略配置。

---

## 7. 补充资料

补充资料由后台配置，支持订单审核时动态发起。

| 类型 | 示例 |
|---|---|
| 工作资料 | 工牌、收入证明、社保、公积金 |
| 学生资料 | 学生证、学信证明 |
| 居住资料 | 租房合同、水电账单 |
| 车辆资料 | 驾驶证、骑行证件 |
| 其他 | 客服要求补充的图片或说明 |

每个补充资料项字段：

| 字段 | 说明 |
|---|---|
| 资料名称 | 客户看到的名称 |
| 是否必填 | 必填/选填 |
| 上传要求 | 图片、PDF、文字 |
| 驳回原因 | 被驳回时展示 |
| 状态 | 未上传、已上传、通过、驳回 |

---

## 8. 人脸识别

| 字段 | 说明 |
|---|---|
| 发起状态 | 未发起、待识别、识别中、通过、失败 |
| 发起来源 | 下单流程、后台审核、重新发起 |
| 失败原因 | 客户可见摘要 |
| 重试次数 | 按配置限制 |
| 有效期 | 识别结果有效期 |

人脸识别失败后，客户可重试或联系客服；后台可转人工复核。

---

## 9. 信用评估/征信授权

| 授权类型 | 客户动作 | 后台结果 |
|---|---|---|
| 信用/信息授权 | 签署授权书 | 可发起信用评估结果 |
| 征信授权 | 签署征信授权 | 可查询授权结果 |
| 流水报告授权 | 客户确认授权 | 可生成流水报告 |
| 黑名单告知 | 勾选确认 | 仅记录告知，不展示命中结论 |

授权状态：

| 状态 | 说明 |
|---|---|
| 未发起 | 后台未发起 |
| 待客户授权 | 客户订单页出现待办 |
| 授权中 | 第三方处理中 |
| 已完成 | 可进入后续审核 |
| 已拒绝 | 客户拒绝授权 |
| 已过期 | 超过有效期 |
| 失败 | 接口或签署失败 |

---

## 10. 代扣绑卡

| 字段 | 类型 | 说明 |
|---|---|---|
| 银行卡号 | 文本/识别 | 脱敏展示 |
| 持卡人 | 只读/文本 | 默认实名姓名 |
| 预留手机号 | 文本 | 银行校验 |
| 验证码 | 文本 | 银行/通道验证码 |
| 代扣协议 | 勾选/签署 | 客户确认 |
| 签约状态 | 状态 | 未签、签约中、已签、失败、解约 |

代扣绑卡成功后回写订单详情和账单账单支付页。

---

## 11. 状态汇总

客户资料页顶部展示进度：

| 状态 | 说明 |
|---|---|
| 待完善 | 还有必填项未完成 |
| 已提交 | 等待审核 |
| 需补充 | 审核驳回部分资料 |
| 审核中 | 客服正在审核 |
| 已通过 | 资料通过 |
| 已失效 | 资料过期需重新提交 |

---

## 12. 后台联动

| 模块 | 联动内容 |
|---|---|
| 订单审核 | 资料状态、补资料项、驳回原因 |
| 附件中心 | 身份证、补充资料、授权文件 |
| 信用评估 | 实名、人脸、授权、报告 |
| 黑名单 | 实名后触发匹配 |
| 合同公证 | 授权完成后进入签约 |
| 财务 | 代扣绑卡、支付、部分账单支付 |
| 日志 | 客户提交、修改、授权、失败 |
