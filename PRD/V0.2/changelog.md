## 2026-05-28:V0.2.2 PRD 全量整改

### V0.2.2 当前执行口径

- 一期范围:安心用长租 + 体验租短租 V1 + IM 客服基础能力。
- 展示层统一为商家订单 / 联营订单 / 平台订单;技术内部枚举可继续保留兼容命名。
- 长租结算触发条件:客户签收确认 + 监管锁显示已上锁 + 激活锁状态已激活。监管锁仅对配置品类强制校验。
- 退款责任:商家订单由商家承担;联营订单 / 平台订单线下协商方案;系统支持指定退款金额并记录责任方、审批人、凭证、冲正对象和流水。
- 财务口径:订单业务账、支付流水账、钱包 / 商家账户账、财务总账 / 分录账四账分离。
- 服务费默认 2% 且可配置。
- C 端禁止暴露内部资金来源、合作模式、风控结论、内部审核备注、商家结算价、平台利润、返点规则等内部字段。
- 高危资金和状态操作拆细到角色权限、二次确认、二人复核和审计日志。
- 催收系统暂不自建,后续通过第三方系统 API 对接;系统保留催收记录和法诉证据包导出能力。

## 2026-05-25:V0.2 核心决策定稿

- 资金链路:平台代收、钱包结算、提现,通联 + 直付通双通道。
- 商家与门店:商家 1:N 门店,四级角色,订单冻结门店归属快照。
- 分账细节:分账有出退款必入;责任方钱包充值到位才能扣减;平台不默认兜底客户。
- 中控台边界:直连第三方 + 中控台异步计费 + 本地缓存降级。
- 设备状态机、内部资金来源、价格调整、合同链路、C 端核心流程、多租户部署等关键底座定稿。
