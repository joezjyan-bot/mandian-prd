# 合同签署流程

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> **来源**: P1-4
> **核心**: 三方制 + 模板分层 + 电子签必走

## 1. 流程概览

```
1. 客户扫码下单 → 待付款 / 待审核
2. 客服审核通过
3. (联营/平台订单) 分配资方
4. 系统锁定合同模板:
   - 商家订单 → 默认模板
   - 联营/平台订单 → 资方专属模板(若有)或默认模板
5. 系统通过电子签通道发起合同签署
6. 三方依次签署
7. 全部签署完成 → 平台下载 PDF 副本存档
8. (可选) 客服/门店决定是否发起公证
9. 合同生效,订单进入待发货
```

## 2. 三方主体

固定:
- 甲方:门店(实际出租人/服务方)
- 乙方:客户(承租人)
- 丙方:平台(居间/技术服务/代收代付)

资方在合同中**不出现**(避免融资租赁定性)。

## 3. 模板选择(详见 P1-4)

| 订单类型 | 默认情况 | 资方有专属时 |
|---|---|---|
| 商家订单 | 默认模板 | (无资方,固定默认) |
| 联营订单 | 默认模板 | 资方专属模板 |
| 平台订单 | 默认模板 | 资方专属模板 |

## 4. 发起签署

系统调用电子签通道:
- 通道按平台配置(e签宝 / 法大大等)
- 推送合同链接到三方
- 设定签署顺序(平台 → 门店 → 客户 / 或客户 → 门店 → 平台)
- 设定签署超时(默认 24 小时)

## 5. 各方签署

### 5.1 客户签署(C 端)
- 客户在小程序订单详情看"去签合同"
- 跳转电子签通道完成签署(身份证人脸验证 + 输入验证码)
- 必须客户本人完成,不可代签
- 签署成功 → 状态更新

### 5.2 门店签署
- 门店在订单详情或合同消息中收到通知
- 门店主账号点击"去签合同"
- 跳转电子签通道完成签署

### 5.3 平台签署
- 系统自动签署(平台有预先备案的电子印章)
- 部分场景需运营管理员手动签

## 6. 签署完成

全部签署完成 → 通道回调:
- 系统更新合同状态:已签署
- 自动下载 PDF 副本存档(见 03 合同存档查阅)
- 通知订单进入下一节点(发货 / 公证)

## 7. 异常处理

### 7.1 签署超时
- 默认 24 小时未签 → 暂停 + 告警
- 客服联系客户/门店核实
- 可重新发起(新的签署超时计时)

### 7.2 签署失败
- 客户人脸不通过:可重试(累计 3 次后客服介入)
- 通道方故障:自动尝试备用通道
- 客户拒签:订单进入"待客服处理"
