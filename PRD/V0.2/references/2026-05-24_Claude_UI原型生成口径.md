# Claude UI 原型生成口径

> 用途：交给 Claude 生成租赁系统 UI 原型时使用。  
> 原则：不要自由发挥视觉风格，先复用已确认参考，再落完整页面截面图。

## 1. 设计基准

1. 整体后台风格参考 `joezjyan-bot/finance-zhongtai` 的 `static/admin-v2`：企业中台、克制、干净、适合长期运营使用。
2. 登录页参考无界租后台登录页：浅蓝背景、居中白色卡片、扫码登录与密码/验证码登录并列。
3. 订单列表参考无界租后台订单列表的信息密度：列表内直接展示订单、商品、用户、金额、风控、备注、回调和操作。
4. 门店端办单助手参考 `joezjyan-bot/calculator` 仓库里的 `phone-rent`，不要使用当前系统计算器逻辑。

## 2. 不允许的输出

1. 不要只画文字流程图。
2. 不要做营销页、大首屏、装饰性大卡片。
3. 不要把所有订单塞进一个无差别大列表。
4. 不要使用真实生产订单、姓名、手机号、身份证、地址样例。
5. 不要把敏感操作裸露在第一层，例如退款、关闭订单、锁机、解锁、改价、加黑名单。

## 3. 订单列表拆分定案

订单管理拆成四个入口：

```text
订单管理
├─ 门店订单
├─ 分红订单
├─ 平台订单
└─ 全部订单
```

| 页面 | 定位 |
|---|---|
| 门店订单 | 门店自营订单监管、抽佣查看、客诉/租后跟进 |
| 分红订单 | 平台审核、资方分配、合同、风控、分账主工作台 |
| 平台订单 | 平台/资方主控、渠道佣金、发货主体、回款/代扣主工作台 |
| 全部订单 | 跨类型搜索和汇总，不作为默认日常入口 |

## 4. 优先生成页面

先生成运营端完整 UI 截面图：

1. 登录页
2. 首页数据看板
3. 门店订单列表
4. 分红订单列表
5. 平台订单列表
6. 订单详情
7. 店铺审核/商户详情
8. 商品管理/商品编辑
9. 链路配置中心
10. 财务分账/钱包流水

再生成：

1. 商家 PC 端：首页、商品、订单、员工、钱包、门店订单配置。
2. 门店手机端：门店登录、门店入驻、审核状态、三个办单助手入口、计算页、报价二维码页、待办任务、订单进度。
3. 渠道 H5：登录、数据首页、推广商户、订单统计、佣金明细、提现记录。

## 5. 输出要求

每个页面必须包含：

1. 页面标题、导航位置、筛选区、主体列表/表单、状态标签、按钮区。
2. 真实业务字段，不要空泛占位。
3. 弹窗/抽屉入口和点击路径。
4. 空状态、加载态、异常态的简要设计。
5. 权限差异和敏感字段脱敏。
6. 危险操作二次确认和操作日志入口。

## 6. 业务基线

1. 订单类型只保留门店订单、分红订单、平台订单。
2. 原低费率订单全部改为平台订单。
3. 默认门店发货，平台订单可配置门店发货、平台发货、资方发货。
4. 门店端配资充值先取消。
5. 分红订单先按比例分账，再分别扣平台 2% 默认抽佣。
6. 渠道佣金只统计分红订单和平台订单，门店订单只记录来源。
7. 链路配置中心只放运营端，商家不直接配置平台链路。
8. 租后 V1 轻量化，后期对接外部催收系统。
