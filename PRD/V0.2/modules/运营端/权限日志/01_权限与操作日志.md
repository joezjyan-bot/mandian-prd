# 权限与操作日志

> 页面级 PRD 草案。
> 目标：把三端账号、角色、菜单、按钮、数据权限、敏感查看、操作日志、回调日志统一成基础能力。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 权限与操作日志 |
| 所属端 | 运营端，商家端和门店端引用同一权限模型 |
| 入口路径 | 权限管理 > 角色权限；日志审计 > 操作日志 / 回调日志 |
| 使用角色 | 平台管理员、运营主管、审计人员、技术支持 |
| 核心目标 | 控制谁能看、谁能点、谁能导出、谁能改钱和状态，并保证全链路可追溯 |

---

## 2. 核心口径

1. 权限必须同时控制菜单、页面、按钮、数据范围、字段明文、导出和接口。
2. 前端隐藏按钮不等于权限控制，后端接口必须校验。
3. 所有人工操作、系统操作、接口回调、配置变更都必须记录日志。
4. 钱、合同、审核、发货、退款、提现、改价、资方、敏感资料查看都属于高风险动作。
5. 高风险动作需要二次确认，部分动作需要主管审批或二次验证。
6. 日志不能被普通管理员删除；只能按合规策略归档。

---

## 3. 账号体系

| 账号类型 | 所属端 | 说明 |
|---|---|---|
| 平台员工 | 运营端 | 审核客服、财务、商品运营、租后、管理员 |
| 商家老板 | 商家 PC、门店手机端 | 入驻主体主账号 |
| 商家管理员 | 商家 PC、门店手机端 | 老板授权管理账号 |
| 门店店员 | 门店手机端 | 只办单、看待办、上传材料 |
| 渠道账号 | 渠道 H5 | 看推广、订单统计、佣金、提现 |
| 资方账号 | 后续资方端 | V1 可先不开放或只读 |
| C 端客户 | 小程序/H5/APP | 下单、签约、支付、还款 |

---

## 4. 权限维度

| 维度 | 说明 |
|---|---|
| 菜单权限 | 是否显示一级/二级菜单 |
| 页面权限 | 是否可进入页面 |
| 按钮权限 | 是否可执行审核、退款、提现等动作 |
| 数据范围 | 全平台、指定商家、指定门店、本人订单 |
| 字段权限 | 是否可看成本价、身份证、银行卡、手机号明文 |
| 导出权限 | 是否可导出订单、财务、客户资料 |
| 配置权限 | 是否可修改链路、租赁模式、增值服务 |
| 审批权限 | 是否可审批退款、提现、冲正、强制关闭 |

---

## 5. 角色建议

| 角色 | 核心权限 |
|---|---|
| 超级管理员 | 全部权限，仍需记录日志 |
| 运营主管 | 审核、分配资方、异常处理、强制关闭审批 |
| 审核客服 | 待审核、资料、风控、合同发起、补资料 |
| 财务 | 钱包、提现、对账、退款、冲正、收款账户 |
| 商品运营 | 商品、规格、增值服务、同步办单助手 |
| 租后客服 | 逾期、催收、归还、售后跟进 |
| 客诉客服 | 投诉接口、客诉处理、协商记录 |
| 商家老板 | 本商家商品、订单、员工、钱包、配置 |
| 门店店员 | 办单助手、本人待办、交付材料 |
| 渠道账号 | 推广统计、佣金明细、提现申请 |

## 6. 高风险动作清单

| 动作 | 要求 |
|---|---|
| 审核通过/拒绝 | 原因或意见必填 |
| 改套餐/改价 | 生成价格版本，必要时客户重新确认 |
| 分配/更换资方 | 记录资方、比例、金额、原因 |
| 发起合同/公证/授权 | 记录接口请求和返回 |
| 关闭订单 | 二次确认，原因必填 |
| 退款/冲正 | 财务权限，生成流水 |
| 提现审核/打款 | 财务高权限，二次确认 |
| 修改收款账户 | 老板/财务权限，敏感校验 |
| 查看身份证/银行卡明文 | 敏感查看日志 |
| 导出订单/客户/财务 | 导出日志，记录筛选条件 |
| 强制同步商品 | 展示影响范围，记录版本 |
| 监管锁锁机/解锁 | 记录设备、订单、接口结果 |

## 7. 操作日志字段

| 字段 | 说明 |
|---|---|
| log_id | 日志编号 |
| operator_id | 操作人 |
| operator_role | 操作角色 |
| operator_type | 平台、商家、门店、渠道、系统 |
| action | 操作动作 |
| module | 模块 |
| target_type | 订单、商品、钱包、合同、配置等 |
| target_id | 目标 ID |
| before_value | 操作前值 |
| after_value | 操作后值 |
| reason | 操作原因 |
| result | 成功、失败、处理中 |
| ip / device | 来源 IP 和设备 |
| created_at | 操作时间 |

敏感字段日志只记录脱敏摘要或哈希，不在日志里保存身份证、银行卡、密钥明文。

---

## 8. 回调日志字段

| 字段 | 说明 |
|---|---|
| callback_id | 回调编号 |
| provider | 支付、合同、公证、风控、物流、分账等服务商 |
| event_type | 回调事件 |
| order_id | 关联订单 |
| request_id | 第三方请求编号 |
| status | 成功、失败、待重试 |
| payload_digest | 回调摘要，不存敏感明文 |
| error_message | 失败原因 |
| retry_count | 重试次数 |
| next_retry_at | 下一次重试时间 |
| created_at | 接收时间 |

回调失败不能靠人工直接改最终状态，必须保留失败记录和重试记录。

---

## 9. 日志查询页

| 筛选 | 说明 |
|---|---|
| 操作人 | 平台员工、商家、店员、系统 |
| 模块 | 订单、商品、财务、合同、配置、登录 |
| 动作 | 审核、改价、退款、提现、查看、导出 |
| 目标对象 | 订单号、商品、钱包、配置 |
| 时间范围 | 操作时间 |
| 结果 | 成功、失败、处理中 |
| 风险等级 | 普通、高风险、敏感查看 |

日志详情展示操作前后值、原因、来源、关联订单和相关回调。

---

## 10. 数据脱敏

| 字段 | 默认展示 |
|---|---|
| 手机号 | 中间脱敏 |
| 身份证号 | 中间脱敏 |
| 银行卡 | 只展示后四位 |
| 收款账号 | 脱敏 |
| 地址 | 列表摘要脱敏，详情按权限 |
| 成本价 | 无权限不展示 |
| 接口密钥 | 永不前端展示 |

查看明文需要权限、二次确认和查看原因。
