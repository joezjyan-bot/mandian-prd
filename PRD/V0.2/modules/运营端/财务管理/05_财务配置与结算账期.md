# 财务配置与结算账期

> **⚠️ V0.2 Stage 6 同步修订(2026-05-27)v1.1**:
> - 同步展示用语:商家订单 / 联营订单 / 平台订单 / 履约中 / 逾期费用。
> - 底层字段、接口、枚举不变;资金账户、退款工单、五边对账以 Stage 6 核心财务文档为准。

> 页面级 PRD 草案。
> 目标：统一平台抽佣、结算账期、冻结规则、提现阈值、手续费和分账时点，供商家订单、联营订单、平台订单共同使用。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 财务配置与结算账期 |
| 所属端 | 运营端 |
| 入口路径 | 财务管理 > 财务配置 |
| 使用角色 | 财务主管、平台管理员 |
| 核心目标 | 配置抽佣、分账、账期、提现、冻结、手续费和异常处理规则 |

---

## 2. 核心口径

1. 商家订单默认平台抽佣 2%，可按商家或订单覆盖。
2. 联营订单先按门店/资方比例拆分，再分别扣平台抽佣。
3. 平台订单默认资方或平台主收益，门店协作收益可配置。
4. 财务配置变更只影响新订单或新账期，历史流水不重算。
5. 高风险财务配置需要审批、版本和回滚。

---

## 3. 配置项

| 配置 | 字段 |
|---|---|
| 抽佣配置 | 默认比例、商家覆盖、资方覆盖、订单覆盖 |
| 结算账期 | T+0、T+1、T+N、订单完成后、账单结清后 |
| 提现配置 | 最低提现、单笔上限、日上限、审核方式 |
| 冻结规则 | 逾期、退款、客诉、租后、风控异常 |
| 手续费 | 支付手续费、打款手续费、承担方 |
| 渠道佣金 | 结算触发、冻结、冲正 |
| 接口扣费 | 扣费账户、余额不足处理 |

---

## 4. 抽佣规则

| 订单类型 | 默认规则 |
|---|---|
| 商家订单 | 客户实收扣平台抽佣后进入商家钱包 |
| 联营订单 | 门店份额和内部资金来源份额分别扣平台抽佣 |
| 平台订单 | 资方/平台收益扣抽佣或服务费 |

抽佣覆盖优先级：订单覆盖 > 商家覆盖 > 资方覆盖 > 订单类型默认 > 平台默认。

---

## 5. 结算账期

| 触发 | 说明 |
|---|---|
| 账单结清 | 当期账单支付完成后进入待结算 |
| 订单完成 | 归还或留购完成后统一结算 |
| 审核通过 | 特殊服务费可用 |
| 人工确认 | 异常订单 |

账期到达后从待结算转为可提现；如果期间发生退款、逾期、客诉，按冻结规则处理。

---

## 6. 冻结规则

| 场景 | 冻结对象 |
|---|---|
| 客户退款 | 相关分账和佣金 |
| 客诉处理中 | 订单收益 |
| 租后逾期 | 联营/平台订单相关收益 |
| 风控异常 | 商家钱包或订单收益 |
| 对账不平 | 相关账户提现 |

解冻必须由系统条件或人工审核触发，并生成日志。

---

## 7. 提现规则

| 字段 | 说明 |
|---|---|
| 最低提现金额 | 小于阈值不可提交 |
| 单笔上限 | 控制单次提现 |
| 日累计上限 | 控制风险 |
| 审核方式 | 自动、人工、金额分级 |
| 打款通道 | 默认通道和备用通道 |
| 到账失败处理 | 退回余额或重试 |

商家员工不能提现，渠道提现和商家提现共用财务审核模型。

---

## 8. 版本与审批

财务配置发布必须记录版本、发布人、审批人、生效范围、生效时间和回滚版本。抽佣、提现、冻结、打款通道属于高风险配置。
