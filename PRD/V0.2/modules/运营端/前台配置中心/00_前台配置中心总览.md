# 前台配置中心 · 00 总览

> V0.2 新增一级模块。
> 目标:把所有 C 端可展示的元素(Tab / Banner / 装修 / 品类 / 弹窗 / 客群定向 / 版本发布)集中到一个后台模块统一管理,不让代码里写死任何 C 端展示项。
> **不包含**:城市与门店开通(放在「店铺管理」)、业务配置如费率/合同模板/链路(放在「配置管理」)、订单/财务/商品基础资料(各自独立模块)。

## 3. 多租户与三层覆盖架构

```
配置生效优先级(从高到低):
  城市级覆盖
    ↓ 继承
  租户级覆盖
    ↓ 继承
  平台默认装修
```

**核心规则**:

1. **平台默认装修**:满点产品团队提供的"通用装修模板",所有租户开通后立即可用
2. **租户级覆盖**:每个租户(品牌,如鼎租/惠讯)可基于平台默认装修做自己的覆盖(改 Banner / 改主推商品 / 改品类排序)
3. **城市级覆盖**:租户可在某个具体城市做差异化(例如鼎租上海做活动,只在上海覆盖 Banner)
4. **继承机制**:每一层可"完全继承上层(默认)" / "覆盖部分模块(例如只改 Banner)" / "完全自定义(全部重做)"

**字段(数据库)**:

```sql
ui_config
- config_id
- tenant_id        -- NULL=平台默认 / 具体租户 ID=租户配置
- city_code        -- NULL=全部城市 / 具体城市=城市级覆盖
- business_line    -- assurance_rent / experience_rent / common
- module_type      -- banner / floor / popup / category / tab ...
- module_key       -- 具体模块标识(如 banner_slot=home_top)
- config_json      -- 配置内容
- inherit_from     -- 继承自哪一层
- override_fields  -- 覆盖了哪些字段(JSON 数组)
- status           -- draft / pending_review / published / archived
- version
- effective_at / expired_at
- created_by / updated_by
```

## 4. 与其他模块的边界

| 模块 | 边界 |
|---|---|

## 5. 关键 API(C 端消费)

C 端启动后或进入特定页面时拉取以下 API:

| API | 用途 | 缓存策略 |
|---|---|---|
| `GET /api/v1/ui/tabs` | Tab 配置 | 进程内缓存 5 分钟 |
| `GET /api/v1/ui/business-line/:line` | 业务线装修(安心用/体验租) | 进程内缓存 5 分钟 |
| `GET /api/v1/ui/banners?slot=xxx` | Banner(带客群匹配) | 进程内缓存 2 分钟 |
| `GET /api/v1/ui/categories?line=xxx` | 品类导航 | 进程内缓存 5 分钟 |
| `GET /api/v1/ui/popups?scene=launch` | 弹窗 | 启动一次 |

**工程要求**:
- C 端:CDN 缓存图片/视频 + 本地缓存 JSON
- 降级:后端挂时 C 端有兜底配置(不白屏)
- A/B 预留:同一位置可推两套配置,按客群分流
- 埋点:每个 Banner / 推荐位 / 楼层必须有曝光 + 点击埋点
