# 04 UI 风格与页面原型规范

> V0.2 后续页面级 PRD 必须带完整 UI 图或低保真原型图。本文先固定 UI 风格、页面结构和输出格式。

---

## 1. UI 基准项目

默认参考 GitHub 仓库：

- `joezjyan-bot/finance-zhongtai`
- 重点目录：`static/admin-v2/`
- 关键文件：
  - `static/admin-v2/index.html`
  - `static/admin-v2/css/tokens.css`
  - `static/admin-v2/js/app.js`

该项目是企业财务中台后台，适合作为满点租赁系统运营端和商家端的视觉基准。

---

## 2. 视觉风格关键词

| 方向 | 说明 |
|---|---|
| 企业中台 | 不做营销页，不做花哨大卡片，以工作效率为主 |
| 财务/运营平台 | 适合订单、资金、审核、配置、日志等复杂后台 |
| 克制色彩 | 蓝色作为主色，灰白作为页面底色，状态色用于提醒 |
| 高信息密度 | 表格、筛选、抽屉、分页、标签页为主 |
| 强可追溯 | 详情页必须保留操作日志、状态流转、资金明细 |
| 多角色权限 | 同一页面根据运营/商家/员工角色显示不同按钮 |

---

## 3. 页面基础布局

### 3.1 PC 后台

适用于运营端、商家 PC 端。

```
┌──────────────────────────────────────────────────────────────┐
│ 顶部栏：系统名 / 当前主体 / 通知 / 用户 / 退出                │
├──────────────┬───────────────────────────────────────────────┤
│ 左侧导航      │ 面包屑 / 页面标题 / 操作按钮                  │
│              ├───────────────────────────────────────────────┤
│ 一级菜单      │ 筛选区                                        │
│ 二级菜单      ├───────────────────────────────────────────────┤
│              │ 表格 / 卡片列表 / 配置表单                     │
│              ├───────────────────────────────────────────────┤
│              │ 分页 / 批量操作                                │
└──────────────┴───────────────────────────────────────────────┘
```

常用组件：

- 左侧导航
- 顶部状态区
- 面包屑
- 筛选表单
- 表格
- 状态标签
- 分页
- 右侧详情抽屉
- 新增/编辑弹窗
- 二次确认弹窗
- 操作日志时间线
- 标签页

### 3.2 H5 门店端

适用于门店手机端、渠道端、资方端。

```
┌────────────────────┐
│ 顶部标题 / 返回     │
├────────────────────┤
│ 当前主体 / 状态提示 │
├────────────────────┤
│ 核心操作入口        │
├────────────────────┤
│ 列表 / 表单 / 卡片   │
├────────────────────┤
│ 底部 Tab 或主按钮    │
└────────────────────┘
```

常用组件：

- 顶部导航
- 底部 Tab
- 大按钮入口
- 表单输入
- 图片上传
- OCR 识别结果确认
- 地图选点
- 价格明细卡
- 二维码区域
- 待办列表
- 操作确认弹窗

## 4. 页面级 PRD 输出模板

每个页面必须按下面结构输出。

### 4.1 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 例如：运营端商品列表 |
| 所属端 | 运营端 / 商家 PC / 门店 H5 / C 端 / 渠道端 / 资方端 |
| 入口路径 | 从哪个菜单进入 |
| 使用角色 | 运营、客服、商家老板、门店员工等 |
| 核心目标 | 这个页面解决什么业务问题 |

### 4.2 UI 图

必须包含完整界面图，至少要表现：

- 页面标题
- 筛选区
- 主表格/主内容
- 操作按钮
- 状态标签
- 分页
- 弹窗/抽屉入口
- 关键字段说明
- 点击箭头

示例标注方式：

```
[查询] ──> 刷新列表
[新增商品] ──> 打开新增商品弹窗
[编辑] ──> 打开右侧详情抽屉
[同步办单助手] ──> 打开同步确认弹窗
```

### 4.3 字段说明

| 字段 | 类型 | 必填 | 说明 | 校验/失败提示 |
|---|---|---|---|---|
| 商品名称 | 文本 | 是 | 商品展示名称 | 不能为空 |

### 4.4 按钮与交互

| 按钮/控件 | 点击后 | 成功结果 | 失败结果 |
|---|---|---|---|
| 查询 | 按筛选条件拉取列表 | 表格刷新 | 提示接口错误 |
| 新增 | 打开弹窗 | 可填写表单 | 无权限则隐藏或置灰 |

### 4.5 权限

| 角色 | 可见 | 可操作 |
|---|---|---|
| 运营管理员 | 全部 | 全部 |
| 普通客服 | 订单相关 | 不可改财务配置 |

### 4.6 状态流转

用流程图或状态表描述。

### 4.7 数据互通

说明本页面读写哪些数据，以及影响哪些模块。

## 6. UI 设计约束

1. 不做官网式大首屏。
2. 不用大面积渐变背景。
3. 不用营销卡片堆叠替代业务表格。
4. 表格必须支持筛选、分页、状态标签。
5. 详情页优先用右侧抽屉，避免频繁跳页。
6. 钱相关操作必须有二次确认和操作日志。
7. 危险操作按钮不能和普通按钮放得过近。
8. H5 表单必须考虑上传失败、OCR 失败、验证码失败、实名不一致。
9. 员工账号登录后隐藏财务和配置入口。
10. 所有页面要考虑空状态、加载态、错误态。

## 7. 后续生成 UI 图的工作方式

后续每做一个模块，按以下顺序：

1. 先列页面清单。
2. 再写页面级 PRD。
3. 再生成完整 UI 图。
4. 再让业务确认交互和字段。
5. 再交给 Claude/Codex 继续扩写开发文档。

页面 UI 图需要尽量接近真实后台，不只画流程框。
