# 满点租赁系统 V0.2.2 全流程会议对照总文档

> 文档定位:这是一份用于公司会议逐环节对系统的“单一总文档”。它不是分散索引,而是把系统架构、端到端流程、模块操作流程、关键规则、会议讨论点放在一个文档里。  
> 使用方式:开会时从“系统总览”开始,按目录逐个模块往下过;每个模块都有“操作流程”和“会议要对的问题”。  
> 版本口径:V0.2.2,一期范围为安心用长租 + 体验租短租 V1 + IM 客服基础能力。

## 0. 会议使用说明

- 如果只想从头到尾过业务,按本文顺序看即可。
- 如果某个模块需要展开细节,看该模块末尾的“对应细分文档”。
- 本文优先讲业务、流程、规则和会议讨论点;接口字段、数据库字段、枚举值留在工程文档。
- 客户侧不得展示内部资金来源、合作模式、风控结论、商家结算价、平台利润、返点规则等内部信息。

## 3. 三类订单模式

| 类型 | 谁主导 | 商家做什么 | 平台做什么 | 退款责任 |
|---|---|---|---|---|
| 商家订单 | 商家自营 | 获客、办单、审核或履约、承担主要责任 | 提供系统、支付、合同、客服、财务工具 | 商家自行承担 |
| 联营订单 | 平台+商家共同经营 | 参与履约、交付、服务 | 主控规则、审核、结算、证据链 | 线下协商退款方案,系统记录 |
| 平台订单 | 平台主导 | 只处理授权履约动作 | 主控获客、审核、合同、财务、售后 | 线下协商退款方案,系统记录 |

客户侧不展示三类订单背后的内部资金来源和合作细节。

## 4. 端到端总流程

### 4.1 安心用长租总流程

```text
客户扫码/选品 → 订单确认 → 资料提交 → 信用/信息授权 → 审核
→ 合同签署 → 首期/保证金支付 → 商家交付 → 客户签收
→ 监管锁或替代验收条件校验 → 平台结算 → 履约中账单
→ 到期三选一:归还设备 / 继续使用 / 申请购买
→ 正常结清 / 提前归还 / 退款 / 逾期 / 法诉 / 坏账回收
```

### 4.2 体验租短租总流程

```text
客户进入体验租 → 选城市/门店/车辆/租期 → 确认订单
→ 支付租金/处理保证金 → 商家接单 → 到店核销
→ 绑定实际车辆 → 取车确认 → 用车中 → 续租/超时
→ 还车预约 → 还车验收 → 保证金扣款/退还 → 订单完成/争议处理
```

### 4.3 财务资金总流程

```text
客户付款 → 支付流水 → 订单业务账 → 平台服务费/商家结算款
→ 商家账户/钱包 → 提现 → 提现成功或失败回滚
→ 若退款:责任方确认 → 指定金额退款 → 冲正订单/钱包/总账/返点
→ 若逾期/法诉:逾期费用、坏账核销、后续回收款回原订单处理
```

### 4.4 法诉证据链总流程

```text
订单快照 → 设备价值确认 → 实名资料 → 合同/电子签
→ 交付照片/人设备合照 → 签收确认/视频确认 → 监管锁记录
→ 账单明细 → 支付流水 → IM聊天记录 → 逾期记录
→ 退款/冲正记录 → 操作日志/审计日志 → 法诉证据包导出
```

## 01 C端总入口与订单确认

**牵头部门:** 产品 / 前端 / 法务 / 客服

**模块定位:** 让客户从安心用长租或体验租短租入口进入,完成选品、订单确认、协议确认、支付和后续履约操作。

### 入口和页面
- 小程序/H5 首页:安心用、体验租、个人中心
- 订单确认页:商品、规格、费用、最低服务期、设备价值、协议勾选
- 在租期间页:账单、归还、继续使用、申请购买、客服入口

### 操作流程
1. 客户进入首页,先区分安心用长租和体验租短租。
2. 安心用客户选择商品或扫码进入订单确认页。
3. 客户确认设备型号、成色、设备价值、保证金、首期应付、每期应付和最低服务期。
4. 客户勾选协议、隐私、授权、设备价值确认等必要确认项。
5. 支付成功后进入签约、交付、签收和履约链路。

### 关键规则
- C端不展示资方、资金来源、合作模式、风控结论、商家结算价、平台利润、返点规则。
- 普通租赁客户侧统一用“申请购买”,不使用会让客户误解为默认购买的表达。
- 安心用最低服务期统一3个月起步,下单页显著展示并留痕。
- 申请购买价一期按A口径:剩余应付租金+保证金;折旧余值口径后期再做。

### 会议要对的问题
- 客户是否能一眼看懂自己要付什么、什么时候付、到期有哪些选择。
- 设备价值确认和最低服务期是否展示足够醒目。
- 普通客户侧是否还有内部词、资金词、利润词。

## 02 安心用长租订单全生命周期

**牵头部门:** 产品 / 运营 / 客服 / 风控 / 技术 / 测试

**模块定位:** 把长租订单从下单到审核、签约、支付、交付、签收、履约、到期、归还、申请购买、逾期、法诉完整闭环。

### 入口和页面
- 运营后台订单列表
- 订单详情页
- 客服操作区
- 商家履约视图
- C端订单详情

### 操作流程
1. 客户下单并提交资料。
2. 平台或商家按订单类型审核。
3. 审核通过后发起合同签署和必要公证。
4. 客户支付首期和保证金。
5. 商家交付设备并上传交付证据。
6. 客户签收确认,系统校验监管锁或替代验收条件。
7. 满足结算条件后进入平台结算。
8. 订单进入履约中,按账单周期生成应付。
9. 到期前同等展示归还设备、继续使用、申请购买。
10. 异常分流到退款、提前归还、逾期、法诉或坏账回收。

### 关键规则
- 结算前必须满足客户签收确认、监管锁要求品类的锁状态、激活状态或无锁品类替代验收条件。
- 监管锁适用苹果类手机、平板、手表产品;其它无锁品类以前序检测、设备识别码归档和交付证据完整为结算条件。
- 账单生成前客服可按权限改价;账单生成后任何人不能直接改账单,只能走冲正、退款或调整流程。
- 订单关闭、退款、提前归还、申请购买都必须留流水和操作日志。

### 会议要对的问题
- 每个状态是否都有明确下一步和负责人。
- 客户支付成功但签约/交付/锁状态异常时如何处理。
- 未签收、未满足锁状态、设备型号不一致时财务是否绝对不能打款。

## 03 体验租短租V1

**牵头部门:** 产品 / 商家运营 / 客服 / 风控 / 测试

**模块定位:** 支持小时租、日租、周租等短租场景,让客户按城市、门店、车型、租期下单,商家完成接单、核销、取车、还车和验收。

### 入口和页面
- C端体验租入口
- 选择城市/门店/车辆页
- 短租订单确认页
- 商家端体验租订单
- 运营端体验租管理

### 操作流程
1. 客户进入体验租,选择品类、城市、门店和租期。
2. 客户选择车辆方案并确认订单。
3. 系统处理短租支付、保证金或预授权。
4. 商家接单,客户到店核销。
5. 商家绑定实际车辆,完成取车确认。
6. 用车中支持续租、查看剩余时间、联系客服。
7. 客户预约还车或到店还车。
8. 商家验收车辆,记录照片、里程、电量、损坏、违章、超时等。
9. 系统按验收结果退还或扣减保证金,订单完成。

### 关键规则
- 短租事故、违章、损坏、丢车,只要不是车辆质量问题,责任归承租人。
- 保证金冻结、扣款、退还必须有凭证、责任原因和操作日志。
- 超时、续租、还车验收都要有状态和费用记录。
- 争议由客服/运营介入,系统保留证据和备注。

### 会议要对的问题
- 短租各类责任是否写进协议和页面提示。
- 门店接单超时、客户不到店、车辆不可用时如何自动/人工处理。
- 保证金扣款证据是否足够。

## 04 门店手机端办单助手

**牵头部门:** 产品 / 运营 / 商家 / 财务 / 技术

**模块定位:** 让门店快速配置或使用办单方案,生成报价和“请扫码下单”二维码,客户扫码后进入C端确认。

### 入口和页面
- 门店手机端办单助手
- 运营端办单助手配置
- 商品/费率/期数/首付/增值服务配置

### 操作流程
1. 运营或商家按权限配置商品、期数、首付成数、费率和增值服务。
2. 门店员工选择商品和方案,系统自动计算报价。
3. 办单助手生成报价快照和二维码。
4. 客户扫码进入C端订单确认页。
5. 后续合同、账单、申请购买和客服解释都以报价快照为准。

### 关键规则
- 商家订单方案可由商家在权限内配置;联营订单和平台订单方案由运营配置。
- 报价必须固化快照,不能因为后续配置变化影响已生成订单。
- 电动车价格由后期商品配置填写,不在PRD里编造价格。
- 低首付不是全局固定值,按门店配置禁用具体首付成数。

### 会议要对的问题
- 门店员工是否能在不培训太多的情况下算清楚报价。
- 哪些配置允许商家改,哪些必须运营改。
- 报价单上客户可见字段是否足够清楚。

## 05 商品、设备与资产管理

**牵头部门:** 产品 / 运营 / 商家 / 法务 / 技术

**模块定位:** 区分商品展示、长租交付设备、短租车辆库存和订单后资产档案,避免商品、库存、资产混乱。

### 入口和页面
- 运营端商品管理
- 设备资产中心
- 设备价值确认单
- 短租车辆库存

### 操作流程
1. 运营或商家创建商品,配置规格、价格、期数和可用场景。
2. 长租商品不按库存出租,在发货/交付时绑定设备识别码。
3. 短租商品需要车辆/设备库存支持,每台设备有可租、已租、维修等状态。
4. 长租订单交付后生成设备资产档案。
5. 设备价值确认进入订单快照、合同附件和法诉证据链。

### 关键规则
- 长租无需库存锁定,只在交付节点录入IMEI/SN/VIN等设备识别码。
- 短租必须有库存和车辆状态机。
- 适配短租的商品名称应统一且不可随意修改,避免后续配置错乱。
- 设备价值确认要可追溯,客户侧确认看到的是设备价值而不是内部采购或资金口径。

### 会议要对的问题
- 商品规格是否足够灵活,例如成色、颜色、内存、电池等。
- 长租和短租库存边界是否清楚。
- 设备价值确认如何与合同和签收同步。

## 06 商家、门店与员工权限

**牵头部门:** 运营 / 商家 / 财务 / 产品

**模块定位:** 管理商家入驻、店铺审核、门店资料、员工账号、商家钱包和门店风险指标。

### 入口和页面
- 运营端商家/店铺管理
- 商家端员工管理
- 商家端财务钱包
- 门店风险数据区块

### 操作流程
1. 商家提交入驻资料。
2. 平台审核商家和门店。
3. 商家配置员工、角色和权限。
4. 门店处理订单、交付、短租取还车。
5. 商家查看钱包、提现、结算和风险指标。
6. 运营可按权限停单、关低首付、冻结或停用门店。

### 关键规则
- 商家财务人员可被授权查看钱包和发起提现,但不能修改结算账户。
- 门店风险指标包括静默率、逾期率等,用于审核参考和手动管控。

## 07 交付、签收、监管锁与验收

**牵头部门:** 运营 / 商家 / 风控 / 财务 / 法务

**模块定位:** 确保设备真实交付、客户本人确认、监管锁或替代验收条件达成后再进入结算。

### 入口和页面
- 发货履约
- 签收确认配置
- 交付照片证据
- 监管锁管理
- 订单详情结算条件区

### 操作流程
1. 商家选择发货方式或门店自提。
2. 录入设备识别码并上传交付照片。
3. 客户通过电子确认单或AI问答确认签收。
4. 需要监管锁的苹果类手机/平板/手表等待锁状态回调。
5. 无锁品类按签收确认、设备识别码归档、交付证据完整和前置检测完成校验。
6. 满足条件后财务结算按钮可用;不满足则显示阻断原因。

### 关键规则
- 未签收不能结算。
- 监管锁适用品类未上锁或未激活不能结算。
- 无锁品类不得因没有监管锁本身阻断业务,以前序验收条件为准。
- 人设备合照AI核验只存比对结果、评分和流水号,不建可检索人脸库。

### 会议要对的问题
- 哪些品类必须走监管锁,目前确认苹果手机、平板、手表。
- AI问答确认和电子确认单默认用哪个,哪些订单需要人工复核。
- 财务打款前能否清晰看到缺哪个条件。

## 08 财务、钱包、结算、退款与总账

**牵头部门:** 财务 / 老板 / 产品 / 技术 / 测试

**模块定位:** 把客户付款、平台服务费、商家结算、钱包提现、退款冲正、返点、坏账和总账全部拆清楚。

### 入口和页面
- 运营端财务管理
- 商家钱包
- 渠道佣金提现
- 支付回调
- 退款和冲正
- 总账报表

### 操作流程
1. 客户付款进入支付流水。
2. 订单业务账记录应收、已收、待收。
3. 系统按规则确认平台服务费和商家结算款。
4. 商家账户余额变化,可提现余额和冻结余额分开。
5. 商家或渠道发起提现,系统校验限额、次数、手续费和审核阈值。

- 支付回调、打款、退款、返点、提现必须幂等,防重复。
- 商家订单退款由商家承担;联营订单和平台订单线下协商退款方案,系统支持指定金额退款。
- 普通提现单笔500-20000元,每日最多3次,手续费0.2%+0.5元,超2万人工审核。

### 会议要对的问题
- 客户每笔钱财务能否追到支付流水、订单账、钱包账、总账。
- 退款后返点和服务费是否同步冲正。
- 提现失败后余额是否自动回滚。
- 财务报表是否能按门店、日期、渠道、业务员、支付方式对账。
