# 满点租赁系统重构 V0.2.2 开发冻结版总 PRD

> **当前唯一开发入口声明**: V0.2.2 是当前最新产品口径。开发、测试、评审先读本文和 `README.md` 指定主文档。早期 V0.2 / V0.2.1 文档已移入 `archive/`,仅作背景追溯。

## 1. 必须执行的 V0.2.2 规则

1. 一期范围为 **安心用长租 + 体验租短租 V1 + IM 客服基础能力**。
2. 展示层、客服、商家端、合同说明、财务报表统一使用 **商家订单 / 联营订单 / 平台订单**;技术内部枚举可兼容保留。
3. 长租结算必须满足 **客户签收确认 + 监管锁已上锁 + 激活锁已激活**;无监管锁品类按已确认替代验收条件执行,最终制度文本和供应商字段按后续盖章/联调固化。
4. 退款责任:商家订单商家承担;联营订单/平台订单按线下协商方案执行,系统必须记录责任方、金额、审批、凭证、冲正对象和流水。
5. 资金必须四账分离:订单业务账、支付流水账、钱包/商家账户账、财务总账/分录账。
6. C 端不得展示资方、资金来源、合作模式、风控结论、内部审核备注、服务费拆分、商家结算价、平台利润、返点规则、黑灰名单结论。
7. 权限必须按操作点拆细;高危资金/法务/资料动作必须二次确认、必要时二人复核,并写审计日志。
8. 历史文档如与本文、状态字典、财务规则、权限矩阵、OpenAPI 契约冲突,以 V0.2.2 口径为准。

## 3. 主流程摘要

### 3.1 安心用长租

客户下单 → 资料提交 → 信用/信息授权 → 风控审核 → 合同/可选公证 → 首期/保证金支付 → 商家交付 → 客户签收 → 监管锁校验 → 平台结算 → 账单生成 → 履约中 → 正常结清 / 留购或提前结清 / 逾期 / 法诉 / 坏账核销。

约束:

1. 未签收、未上锁、激活锁未激活时不得自动结算。
2. 账单生成前可按权限改价;账单生成后任何人不得直接改账单,只能走冲正、减免、补收或退款。
3. C 端只展示客户应付、协议、签收、账单、状态和客服入口,不得展示内部资金来源、合作模式、风控结论、平台利润和返点。

### 3.2 体验租短租 V1

C 端进入体验租 → 选择城市/门店/车型/租期 → 下单 → 商家接单 → 保证金处理 → 核销绑车 → 取车确认 → 用车中 → 续租/超时 → 还车预约 → 还车验收 → 保证金扣款或退还 → 事故/违章/损坏/丢车处理。

约束:

1. 短租车辆库存状态机只服务体验租,不得污染长租主链路。
2. 事故、违章、损坏、丢车按保险、协议和证据责任划分,平台不默认兜底。
3. 保证金扣款必须有规则、证据、审批、客户通知和申诉记录。

### 3.3 财务四账

客户支付进入支付流水;订单业务账记录应收、已收、待收和状态;钱包/商家账户账记录商家可提现、冻结、提现中和回滚;财务总账/分录账记录收入、成本、负债、冲正、坏账和回收。

任何支付、退款、打款、返点、提现、冲正、坏账回收、法诉回款都必须有业务来源、流水、审计、幂等键和失败补偿。

对外融资尽调报表独立于资金流水和总账分录。账龄分析与 Vintage 报表只做只读聚合和对外导出,用于银行/合作方评估应收质量,不得反向驱动订单状态、钱包余额或财务分录;其本金、逾期桶、世代口径以 [账龄分析与 Vintage 报表](./modules/运营端/财务管理/09_账龄分析与Vintage报表.md) 为准。

## 4. 交付验收结论

1. 页面级字段、按钮、弹窗、禁用态、空状态、权限态已通过 `19_V0.2.2_页面级字段按钮弹窗总表.md` 收口,UI 和研发不得再凭历史页面自行猜测。
2. OpenAPI 已从“边界草案”推进到“最终契约验收清单”:核心路径、DTO 白名单、敏感字段隔离、幂等键、错误码、回调和审计要求已可逐项评审。
3. ERD 已补核心表关系、关键字段、唯一键和幂等索引;技术可据此拆 migration 方案,但本 PRD 仓库不创建 migration。
4. 财务已补总账科目、分录模板和报表字段:订单业务账、支付流水账、钱包账、总账分录四账边界明确。
5. 财务已补账龄分析与 Vintage 报表:按对外融资尽调报表标准设计账龄分布、Vintage 世代分析、导出盖章、下钻核验和总账勾稽,但不产生资金流水。
6. 法务已补合同、电子确认单、AI 问答、保证金扣款、退款通知、留购/提前结清和法诉导出默认文本清单。
7. 供应商适配已补支付、退款、电子签、公证、监管锁、保险、IM、短信、OCR/人脸、物流、三方催收字段映射。
8. 短租 V1 已补风险责任规则表:事故、违章、损坏、丢车、超时、保证金扣款、保险和证据留存具备开发评审基础。
9. IM 客服已补页面与接口规则:V1 做统一会话、工单、业务跳转和证据沉淀,IM 内不得直接执行退款、扣款、锁车、关单、改价等高危动作。
10. 测试验收已补可执行用例编号和资金断言:长租、短租、四账、权限、OpenAPI、证据包均可进入测试用例拆分。

本结论为内部交付自查,不做自评分。最终评分、上线结论和剩余风险等级以第三方评审、法务书面意见和研发评审会结论为准。

## 5. 执行阶段外部依赖

以下事项不再视为 PRD 缺口,而是研发、财务、法务或供应商执行工作:

1. 技术根据 API 验收清单把 OpenAPI 转为代码级接口定义、SDK、mock 和自动化契约测试。
2. 技术根据 ERD 和总账分录模板拆 migration、索引、事务、补偿任务和多租户隔离字段。
3. 设计根据页面级契约输出高保真 UI,不得改变字段、按钮权限和 C 端敏感字段隔离规则。
4. 财务根据总账科目模板终定科目编码和报表格式,不得改变四账分离原则及已确认资金口径。
5. 法务根据法务文本清单终审合同正文和确认话术,不得改变已确认业务责任和证据链字段。
6. 供应商联调按第三方适配字段映射落地,不得改变主状态机和 C 端展示边界。
