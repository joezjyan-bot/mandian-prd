# 接口失败示例与字段校验

> [历史参考 / V0.2.2 口径覆盖] 本文保留早期失败码和字段校验样例。V0.2.2 后接口契约以 `openapi/core-api.v0.2.yaml` 和 `06_OpenAPI草案说明.md` 的 V0.2.2 敏感字段隔离、三类订单展示名、幂等、权限和审计要求为准。旧订单命名和“资方”表达仅作历史兼容。

> 所属：开发设计
> 目标：补足 OpenAPI 草案之外的失败返回、字段校验和前端处理口径，避免接口只写成功示例。

---

## 1. 通用失败响应

字段级校验失败：

```json
{
  "code": "VALIDATION_FAILED",
  "message": "参数校验失败",
  "request_id": "req_demo",
  "data": {
    "field_errors": [
      {
        "field": "mobile",
        "reason": "手机号格式不正确"
      }
    ]
  },
  "warnings": []
}
```

状态冲突：

```json
{
  "code": "STATUS_CONFLICT",
  "message": "当前状态不允许执行该操作",
  "request_id": "req_demo",
  "data": {
    "current_status": "reviewing",
    "allowed_actions": ["request_more_materials", "reject"]
  },
  "warnings": []
}
```

业务规则阻断：

```json
{
  "code": "BUSINESS_RULE_BLOCKED",
  "message": "业务规则阻断",
  "request_id": "req_demo",
  "data": {
    "blocked_reason": "收款账户实名未完成",
    "next_action": "complete_receive_account_auth"
  },
  "warnings": []
}
```

## 2. 通用字段校验

| 字段类型 | 校验规则 | 失败码 |
|---|---|---|
| `mobile` | 手机号格式、短信验证码匹配、同一租户内唯一 | `VALIDATION_FAILED` |
| `amount` | 正整数，单位分，不允许浮点 | `VALIDATION_FAILED` |
| `order_type` | 仅允许 store / dividend / platform | `VALIDATION_FAILED` |
| `rent_mode` | 仅允许 long / short | `VALIDATION_FAILED` |
| `billing_unit` | 仅允许 hour / day / week / month | `VALIDATION_FAILED` |
| `term` | 大于 0，且符合租赁模式配置 | `BUSINESS_RULE_BLOCKED` |
| `funding_ratio` | 联营订单必填，范围按配置 | `VALIDATION_FAILED` |
| `file_id` | 必须存在、未删除、当前角色可访问 | `RESOURCE_NOT_FOUND` |
| `idempotency_key` | 写接口必填，同一业务动作不可复用不同参数 | `IDEMPOTENCY_CONFLICT` |
| `config_version_id` | 必须存在且已发布 | `BUSINESS_RULE_BLOCKED` |

## 3. 商家入驻失败示例

| 场景 | 错误码 | 前端处理 |
|---|---|---|
| 短信验证码错误 | `VALIDATION_FAILED` | 停留注册页，提示重新输入 |
| 邀请码不存在或失效 | `CHANNEL_ATTRIBUTION_INVALID` | 可继续入驻，但不绑定渠道 |
| 营业执照 OCR 失败 | `OCR_FAILED` | 允许手动填写，并标记人工复核 |
| 法人身份证缺少反面 | `QUALIFICATION_INCOMPLETE` | 回到资料上传页 |
| 收款账户非实名 | `RECEIVE_ACCOUNT_NOT_VERIFIED` | 进入授权资料流程 |
| e 签宝授权未完成 | `ESIGN_AUTH_NOT_COMPLETED` | 显示授权待办和重发入口 |

失败响应：

```json
{
  "code": "RECEIVE_ACCOUNT_NOT_VERIFIED",
  "message": "收款账户实名未完成",
  "request_id": "req_demo",
  "data": {
    "next_action": "upload_authorization_material"
  },
  "warnings": []
}
```

## 11. 前端统一处理规则

1. `VALIDATION_FAILED` 展示字段级错误，不弹通用失败。
2. `STATUS_CONFLICT` 必须刷新当前对象状态。
3. `BUSINESS_RULE_BLOCKED` 展示阻断原因和下一步动作。
4. `PERMISSION_DENIED` 隐藏按钮或提示无权限，不允许继续提交。
5. `CHANNEL_UNAVAILABLE` 显示重试、切通道或人工处理入口。
6. `CALLBACK_VERIFY_FAILED` 只在运营端回调日志展示，不给客户侧展示技术原因。
7. 金额类失败必须进入财务异常，不允许前端自行调整金额绕过。
