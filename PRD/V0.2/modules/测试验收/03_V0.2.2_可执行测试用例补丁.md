# V0.2.2 可执行测试用例补丁

> 本文把验收矩阵进一步拆成研发和 QA 可直接转测试用例的 case。所有资金类 case 必须同时校验订单业务账、支付流水账、钱包账、财务总账。

## 1. 长租主流程

| Case ID | 场景 | 前置条件 | 操作步骤 | 必须断言 |
|---|---|---|---|---|
| LR-001 | 商家订单完整闭环 | 商家商品、门店、报价有效 | 下单→资料→商家审核→签约→首期→交付→签收→锁校验→结算→履约 | C 端无内部字段;商家钱包扣 2% 服务费后入账;订单状态流完整 |
| LR-002 | 联营订单完整闭环 | 联营配置、内部资金来源可用 | 下单→运营审核→内部资金来源分配→合同→支付→交付→结算 | C 端/商家端不展示内部资金来源;商家端只看履约视图 |
| LR-003 | 平台订单完整闭环 | 平台商品和审核配置有效 | 客户下单→平台审核→签约→支付→交付→结算 | 平台订单展示名正确;商家端不可操作非授权动作 |
| LR-004 | 未满足结算三条件 | 订单已签收但锁未上锁或激活锁未激活 | 财务点击结算 | 按钮禁用或进入人工复核;返回 blocker;无打款流水 |
| LR-005 | 无监管锁替代验收 | 无锁品类订单 | 上传设备识别码、交付照片、签收确认 | 替代验收通过后才可结算;证据进证据包 |
| LR-006 | 账单生成前改价 | 客服有改价权限,账单未生成 | 客服改价→客户确认→生成账单 | 账单按新快照生成;改价日志、客户确认、审批记录完整 |
| LR-007 | 账单生成后改价拦截 | 账单已生成 | 客服尝试改账单金额 | 被拒绝;提示走冲正/补收/减免/退款/核销 |

## 2. 体验租短租

| Case ID | 场景 | 前置条件 | 操作步骤 | 必须断言 |
|---|---|---|---|---|
| SR-001 | 正常短租闭环 | 门店有可租车辆和价格方案 | C 端选门店/车型/方案→骑行人→支付→商家接单→核销绑车→用车→还车验收 | 车辆状态 reserved→in_use→available/inspection;保证金释放;四账平衡 |
| SR-002 | 商家拒单自动退款 | 客户已支付,商家拒单 | 商家拒单→系统发起退款 | 退款金额正确;钱包无入账;支付和总账有冲正 |
| SR-003 | 取车车辆不一致 | 核销车辆和订单车型不同 | 门店扫码绑车 | 返回 `SHORT_RENT_VEHICLE_MISMATCH`;不得进入用车中 |
| SR-004 | 续租 | 订单用车中且未逾期 | 客户发起续租并支付 | 父子订单/续租记录关联;到期时间延长;不重复冻结保证金 |
| SR-005 | 超时未还 | 到期后未还车 | 系统到期扫描 | 进入超时状态;生成超时费;通知客户;IM 可介入 |
| SR-006 | 损坏扣款 | 还车验收发现损坏 | 门店上传证据→发起扣款→平台审批 | 保证金冻结/扣款;客户通知和申诉入口;四账冲正完整 |
| SR-007 | 丢车 | 用车中车辆丢失 | 上报丢车→锁车/定位→报警材料上传 | 异常工单、控车日志、报警材料进入证据包;不得自动平台兜底 |
| SR-008 | 控车失败 | 中控接口超时 | 客户点开机/还车锁车 | 不错误推进订单状态;进入异常工单;可重试 |

## 3. 财务四账

| Case ID | 场景 | 操作 | 必须断言 |
|---|---|---|---|
| FIN-001 | 客户首期支付 | 支付回调成功 | 支付流水 success;订单业务账核销;总账借贷平衡;不提前结算 |
| FIN-002 | 客户账单支付 | 周期账单支付 | 账单状态 paid;支付流水唯一;总账平衡 |
| FIN-003 | 商家结算 | 满足结算三条件 | 商家结算单、钱包入账、服务费分录、总账分录均生成 |
| FIN-004 | 提现成功 | 商家发起提现 | 钱包冻结→提现中→成功扣减;总账分录平衡 |
| FIN-005 | 提现失败 | 通道返回失败 | 冻结释放;提现失败流水;总账冲正/调整完整 |
| FIN-006 | 部分退款 | 指定退款金额 | 退款记录、支付退款、账单冲正、钱包冲正、总账冲正、返点按比例回滚 |
| FIN-007 | 重复支付回调 | 同一通道单号重复通知 | 不重复入账;回调日志标记 replay |
| FIN-008 | 重复打款请求 | 同一结算批次重复提交 | 返回首次结果;不重复生成钱包/总账流水 |

## 4. 权限和审计

| Case ID | 场景 | 操作 | 必须断言 |
|---|---|---|---|
| SEC-001 | 商家财务提现 | 财务角色发起提现 | 可发起;不可修改结算账户 |
| SEC-002 | 普通员工查看钱包 | 普通员工访问钱包 | 拒绝或脱敏无余额 |
| SEC-003 | 客服改价 | 客服改账单生成前价格 | 允许;写审批/操作日志;超过指导价拦截 |
| SEC-004 | 账单生成后修改 | 任意角色改已生成账单 | 拒绝;只能走财务调整流程 |
| SEC-005 | 法诉证据包导出 | 无权限导出 | 拒绝;有权限导出必须二次确认和审计日志 |
| SEC-006 | IM 内高危动作 | 客服在 IM 直接退款/扣款 | 返回 `IM_FORBIDDEN_ACTION`;只能跳转业务模块 |

## 5. OpenAPI 和字段隔离

| Case ID | 场景 | 必须断言 |
|---|---|---|
| API-001 | OpenAPI YAML 校验 | YAML 可解析;version=0.2.2 |
| API-002 | C 端订单详情 DTO | 不包含资方、资金来源、合作模式、风控结论、内部审核备注、商家结算价、平台利润 |
| API-003 | 商家端联营/平台订单 DTO | 只展示履约视图;无内部资金来源和平台利润 |

| API-005 | 写接口幂等 | 重复 `X-Idempotency-Key` 返回首次结果或明确冲突 |

## 6. 法诉证据包

| Case ID | 场景 | 必须断言 |
|---|---|---|
| LEGAL-001 | 长租证据包导出 | 合同、实名、签收、AI 视频、交付照片、设备价值、监管锁、账单、支付、IM、操作/审计日志完整 |
| LEGAL-002 | 短租证据包导出 | 骑行人、取车核销、车辆码、控车日志、还车证据、保证金扣款、异常工单完整 |
| LEGAL-003 | 证据缺失 | 返回缺失项,允许补证;导出留痕 |
| LEGAL-004 | 无权限导出 | 拒绝并写安全日志 |

## 7. 通过标准

1. P0 case 全部通过才允许进入上线验收。
2. 资金 case 任一账不平则不得上线。
3. C 端字段隔离失败视为合规 P0。
4. 体验租风险 case 必须至少覆盖超时、损坏、丢车、控车失败。
5. IM 必须证明“带上下文”和“高危动作跳转不执行”。
