# PRD V0.2 页面级补充

> 目标：在 V0.1 总体大纲基础上，补齐可直接进入开发的页面级 PRD、UI 结构、交互路径、按钮反馈、异常状态和权限边界。

## 页面审计原则

每个旧系统页面必须按下面顺序记录，未实测的内容不能写成已完成。

1. 页面入口：菜单路径、路由、页面标题、面包屑。
2. 页面布局：顶部、左侧菜单、查询区、表格区、详情区、弹窗区。
3. 所有控件：输入框、下拉框、日期框、Tab、表格列、分页、图标按钮、文字链接。
4. 点击结果：每个按钮/链接/Tab/下拉都必须逐项点击并记录点击后出现的页面、弹窗、Toast、加载态或无反应。
5. 滚动检查：页面纵向滚动到底，表格横向滚动到最右，Tab 容器左右切换全部检查；左侧菜单、弹窗、下拉面板如有独立滚动也必须检查。
6. 状态覆盖：空数据、有数据、禁用、loading、成功、失败、无权限、校验失败。
7. 风险边界：删除、审核、打款、发货、退款、代扣、上传、保存、提交、导出等高风险操作只记录入口，最终确认前必须人工确认；跳过的原因必须写清楚。
8. 敏感数据：公开文档中不保留真实姓名、手机号、身份证号、银行卡、订单号等样例值，只保留字段名和脱敏示例。

## 当前优先级

1. 运营端订单管理：订单列表、订单详情、关闭退货、到期未归还、买断、续租、分红、门店、电审。
2. 运营端资方管理：数据统计、订单管理、账单管理、用户管理、打款记录、资金账户。
3. 财务管理：租金、线下还款、提现、押金、商家资金、资方提现、佣金审批、结算明细。
4. 商品/店铺/采购账户审核。
5. 营销、佣金、渠道、配置、信审、数据、权限。
