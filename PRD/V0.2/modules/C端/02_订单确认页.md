# 订单确认页

> **来源**: P1-5
> **核心**: 办单二维码扫码后的入口

## 1. 页面定位

C 端 · 客户扫办单二维码后进入此页。锁定门店给定的方案,客户确认后下单。

## 2. 页面布局

### 2.1 顶部
- 门店名称 + 店内地址
- 门店 Logo(如配)
- 客服电话(联系门店按钮)

### 2.2 商品信息区
- 商品图(主图 + 多图轮播)
- 商品名称
- 商品规格(颜色、容量等)
- 设备型号说明
- 设备款(参考价,非客户实付)

### 2.3 价格方案区(锁定)
- 押金:¥XXX
- 首期金额:¥XXX
- 每期金额:¥XXX × XX 期
- **留购价格(押金可抵扣)**:¥XXX
- 总金额:¥XXX
- 租期:XX 个月

### 2.4 价格双层显示(P1-3)
"实际您要付:¥1000(总金额 ¥1080 - 押金 ¥80)"
小字辅助说明,避免误解

### 2.5 服务说明
- 服务期内可申请:留购 / 归还 / 续租
- 逾期违约金:0.05%/日,最低 10 元
- 客户须知(展开可读)

### 2.6 主 CTA
"立即下单"按钮(大按钮)

### 2.7 二级
- 客服咨询入口
- 商品详情(展开看更多说明)

## 3. 点击"立即下单"

```
1. 检测客户登录状态
   - 未登录 → 拉起支付宝/微信授权登录
   - 已登录 → 继续
2. 检测客户实名状态
   - 已实名 → 跳"资料补充"页(03)
   - 未实名 → 跳"实名认证"页(04)
3. 系统在后台创建订单 draft(状态: pending_info)
   - 关联门店、商品、客户、价格方案、渠道(如有)
```

## 4. 异常处理

| 异常 | 提示 |
|---|---|
| 二维码失效 | "此办单链接已失效,请联系门店" + 联系按钮 |
| 商品已下架 | "该商品已下架" |
| 门店暂停 | "门店暂停营业,请联系门店或选其他门店" |
| 客户在黑名单 | "您当前无法办理此业务,请联系平台客服" |

## 5. 多端适配

### 5.1 支付宝小程序
- 调用支付宝授权
- 用支付宝芝麻信用做风控
- 走直付通做支付/分账(后续)

### 5.2 微信小程序
- 调用微信授权
- 用第三方风控
- 走微信支付(V1 不优先)

### 5.3 H5
- 微信内打开:调微信 JSSDK
- 浏览器打开:走通联或直接支付宝/微信 H5 支付

## 6. 客户引导

- 首次进入显示"3 分钟办完租赁,扫脸 → 签合同 → 完成"提示
- 进度条(顶部)显示"1/8 订单确认 → 2/8 实名 → ..."

## 7. 数据模型

订单 draft 创建:
- `order.status = pending_info`
- `order.created_at`
- 关联订单参数(商品、门店、价格方案等)
