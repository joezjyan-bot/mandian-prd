# 在租期间(账单支付 / 申请购买 / 归还 / 续租)

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、申请购买、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> **来源**: P1-5 / 合规审查 C3
> **核心**: 普通渠道 C 端统一展示为“申请购买”;到期前归还设备 / 继续使用 / 申请购买三选一同等展示,不默认引导购买。

## 1. 在租期间的客户操作

订单状态为"出租中"时,客户可在订单详情进行:
- 查看账单 / 主动账单支付
- 申请购买
- 申请归还(到期前)
- 申请续租(到期前)

## 2. 账单

### 2.1 账单列表
- 按期数排序
- 每期账单:期数 / 账单日 / 应还金额 / 实还金额 / 状态(待还 / 已还 / 逾期)
- 已还期数(灰色),待还期数(高亮)
- 当前期(红框)

### 2.2 账单日提醒
- 账单日前 3 天:推送提醒
- 账单日当天:推送
- 逾期 1 天:推送 + 短信
- 逾期 3 天:推送 + 短信 + 客服跟进

### 2.3 主动账单支付
客户可主动点击"立即账单支付":
- 自动调起代扣卡支付
- 也可选其他支付方式
- 支付成功 → 入账 → 分账

### 2.4 部分支付
- 客户可账单支付一部分(支持)
- 系统按原比例拆账(P0-3)

## 3. 到期三选一同等展示

到期前,订单详情页和到期提醒页必须同时、同等展示三个选项:

| 选项 | 客户理解 | 后续流程 |
|---|---|---|
| 归还设备 | 到期后把设备交回门店/平台验收 | 进入归还预约、寄回/到店、验收、保证金结算 |
| 继续使用 | 继续履约一段时间 | 进入续租方案选择和新一轮确认/签约 |
| 申请购买 | 客户主动申请取得设备 | 进入购买申请金额确认和支付 |

展示要求:

1. 三个选项同一层级、同等视觉权重,不得默认勾选“申请购买”。
2. 不得出现“租期满自动转移所有权”“到期自动购买”“默认购买”等逻辑或文案。
3. 客户必须主动选择,系统记录选择结果和页面快照。
4. 若客户未选择,订单继续按到期提醒、归还/续租/申请购买待办处理,不得自动变更为购买。

### 3.1 选择记录 `lease_end_choice_log`

| 字段 | 说明 |
|---|---|
| `choice_log_id` | 主键 |
| `order_id` / `customer_id` | 订单和客户 |
| `presented_options` | 本次展示选项,固定包含 return/renew/purchase_request |
| `selected_option` | return/renew/purchase_request |
| `selected_at` | 客户选择时间 |
| `page_version` | 页面版本 |
| `snapshot_id` / `snapshot_hash` | 页面快照 |
| `ip` / `device_info` | 操作 IP 和设备 |

## 4. 申请购买

### 4.1 触发节点
任何时候(在租期间),客户可主动发起购买申请。

### 4.2 申请购买金额计算
- 申请购买金额 = 价格方案中的购买申请金额/内部 retain 计算结果
- **保证金可抵扣**申请购买金额
- 客户实付 = 申请购买金额 - 保证金
- 还有未到期的期数:可选"还清剩余后申请购买"或"一次性付清申请购买金额"

### 4.3 流程
```
1. 点"申请购买"
2. 系统计算:申请购买金额 / 保证金抵扣 / 实付金额
3. 二次确认弹窗:"申请购买完成后设备归您所有,所有账单结清"
4. 调起支付
5. 支付成功 → 订单状态:已完成
6. 设备状态:已下架(下架原因: purchase_request_completed = 已由客户申请购买)
7. 解锁监管锁(如有)
```

### 4.4 部分场景需提示
- 客户实付 ≤ 0(保证金 > 申请购买金额):系统自动退差额
- 保证金 < 申请购买金额:补差额

## 5. 申请归还

到期前客户可主动申请:
- 走门店归还流程
- 设备验收
- 保证金按验收结果退还

### 5.1 申请提前归还

客户在履约中订单详情页可主动点击“申请提前归还”。该入口用于客户提前结束租赁并交回设备,不得被设计成强制结清全部未到期费用。

提前归还页必须展示预结算信息:

| 展示项 | 说明 |
|---|---|
| 已使用时间 | 从起租/签收确认时间到申请时间的实际使用时长 |
| 已发生费用 | 已出账、已到期、已产生但未支付的费用 |
| 预计检测费 | 归还验收可能产生的检测费用,按配置或人工评估展示 |
| 预计物流费 | 客户寄回、上门取回或门店收车产生的预估费用 |
| 预计折旧补偿 | 因提前归还、设备使用痕迹、损坏等可能产生的补偿预估 |
| 保证金预计退还 | 保证金 - 已确定扣款 - 预计扣款,仅作为预估 |

流程要求:

1. 客户提交提前归还申请后,订单进入提前归还处理中,原账单按规则暂停或继续出账需以运营配置为准。
2. 客户寄回/到店归还后,由门店或平台完成收货、检测、验收。
3. 最终结算以实际检测结果、物流凭证、费用减免和保证金扣退结果为准。
4. 系统不得在客户提交提前归还时默认要求支付全部未到期费用;如需收取折旧补偿或实际损失,必须展示明细并留存确认快照。

## 6. 申请续租

到期前客户可申请:
- 续租期数(可配)
- 续租价格(按平台政策)
- 续租走新一轮合同流程(简化)

## 7. 逾期处理

- 计算逾期逾期费用:0.05%/日,最低 10 元/日(P0-3 规则)
- 全部归平台,不参与分账
- 客服跟进 + 推送提醒
- 长期逾期 → 监管锁触发 → 法务跟进

## 8. 数据模型

- `bill` 账单表
- `order.purchase_applied_at`(购买申请时间,内部字段名可保留)
- `order.return_request_id`(归还申请)
- `order.renewal_request_id`(续租申请)
- `overdue_penalty` 逾期逾期费用记录
- `lease_end_choice_log` 到期三选一选择记录
