# 满点租赁系统重构 V0.2.1 开发冻结版总 PRD

> **版本**: V0.2.1
> **冻结日期**: 2026-05-27
> **定位**: 开发唯一总入口
> **范围**: V1 安心用长租 + 体验租短租 + IM 客服基础能力

---

## 1. 冻结结论

V0.2.1 冻结安心用长租、体验租短租和 IM 客服基础能力,目标是让开发团队可以按一套稳定口径实现系统一期。

体验租/短租进入 V1,但必须作为独立业务线开发:入口、商品适配、库存车辆、订单状态、保证金、取还车核销、续租和控车异常均与长租隔离。长租仍按安心用链路开发,不得因为体验租引入短租库存依赖。

IM 客服进入 V1 基础能力,定位为统一客服中心 + 会话上下文 + 工单转化 + 业务跳转。IM 内不直接执行退款、关单、锁车、扣款、修改账户等高危动作;高危动作必须跳转到原业务模块,按权限、二次确认和日志执行。

## 2. V1 业务范围

### 2.1 V1 必做

- C 端双业务入口:安心用(长租/以租代用)与体验租(短租/门店取还)分开展示。
- 商家入驻、店铺审核、商家资料和员工权限。
- 商品管理、规格矩阵、指导价、长租套餐配置。
- 体验租商品适配:可选择适配短租;短租适配商品必须使用统一不可随意修改的标准商品名称。
- 办单助手:商家订单、联营订单、平台订单三类长租订单。
- 体验租:按城市/品类/门店/车辆选择,支持小时租、日租、周租、月租等可配置周期。
- 体验租车辆库存:车辆档案、门店归属、可租状态、预约保留、取车核销、还车验收、续租、逾期费用、控车日志。
- C 端扫码下单、订单确认、资料补充、实名认证、信用/信息授权、合同签署、公证、保证金/首期支付、代扣绑卡。
- C 端体验租下单、骑行人信息、短租保证金/免押、支付、商家接单、核销码、扫码绑车、用车中、续租、还车。
- 运营端订单审核、内部资金来源分配、合同模板选择、发货履约、签收确认、账单、退款、撤单、留购/提前结清。
- 运营端体验租管理:体验租工作台、门店/车辆/订单、控车异常、保证金扣款、续租链、短租数据看板。
- 商家端订单履约视图、我的钱包、提现、员工权限。
- 商家端体验租模块:订单接单、拒单、车辆管理、取车核销、还车验收、续租处理、异常上报。
- 门店手机端体验租模块:待接单、待取车、已取车、待还车、还车验收、扫码核销、联系客户/平台客服。
- 财务管理:平台代收、订单结算款、商家结算账户、内部资金台账、退款冲正、提现审核、对账。
- IM 客服:统一客服入口、会话分配、跨端会话归档、订单上下文卡片、快捷回复、转工单、敏感操作跳转。
- 发货签收:V1 默认电子确认单;AI 问答确认保留配置和状态设计,可作为后续增强。
- 中控台对接、监管锁能力、操作日志、权限矩阵、数据脱敏。

### 2.2 V1 不做

- 体验租与长租合并成同一套订单状态机。
- 在长租办单助手入口展示短租内容。
- 长租订单提前占用短租车辆库存。
- 平台自营短租的复杂调拨、仓店联动和跨城调度。
- IM 内直接执行退款、关单、锁车、扣款、改价、修改结算账户等高危动作。
- AI 机器人自动处理客诉、自动承诺补偿或自动审批扣款。
- 独立资金方端。
- 资方在 C 端展示。
- C 端展示合作模式、资金来源、风控结论、平台抽佣、分账。

## 3. 核心业务定义

### 3.1 三类长租订单

| 类型 | 底层枚举 | 中文展示 | V1 定义 |
|---|---|---|---|
| 商家订单 | `self_operate` | 商家订单 | 商家自营,商家负责审核和履约,平台提供系统、支付、合同、账单和结算工具 |
| 联营订单 | `joint_venture` | 联营订单 | 商家与平台/内部资金来源按规则共同参与,运营端统一审核和资金控制 |
| 平台订单 | `receivables_assignment` | 平台订单 | 平台主导审核、分配履约门店和内部资金规则,商家仅承担履约视图内动作 |

### 3.2 合同主线

默认合同结构为三方租赁合同:

- 甲方:商家/出租方
- 乙方:客户
- 丙方:平台

资方不出现在客户合同和 C 端。资方、资金来源、放款条件、资方台账仅作为运营端和财务端内部配置。

如后续存在银行赊销类资金来源,只通过 `funding_source + terminology_template_id` 控制内部流程和客户侧按钮话术,不得让同一客户订单同时呈现"租赁"和"借款"双重语义。

### 3.3 资金主线

平台承担代收、记账、结算和工具服务角色,不对客户履约兜底。

1. 客户支付进入平台对公/通道账户。
2. 平台按规则扣平台综合服务费。
3. 属于商家的资金穿透至通联备付金子台账,形成门店结算账户余额。
4. 联营/平台订单涉及内部资金来源的部分进入内部资金台账,不在 C 端展示。
5. 商家通过我的钱包申请提现。

### 3.4 提现规则

- 普通提现单笔范围:500-20000 元。
- 每日最多 3 次。
- 超过 20000 元或命中异常规则时进入人工审核,不直接提交代付通道。
- 手续费:千分之 2 + 0.5 元,后台可配置。
- 商家财务人员可以被授权查看钱包和发起提现,但不能修改结算账户。
- 修改结算账户仅商家老板可提交,必须经平台审核、二次确认和完整日志。

## 5. C 端展示原则

C 端只展示客户能理解和能操作的内容:

- 商品、套餐、保证金、每期应付、合同、收货、履约中、账单支付、留购、归还、客服。
- 不展示资方、资金来源、合作模式、风控结论、分账、平台抽佣、资方专属模板。
- 审核类统一展示为"审核中、平台处理中、待补充资料、待签约"等中性状态。
- 信用/信息授权页面只说明授权用途和隐私保护,不展示内部评分和资方规则。

## 6. 长租主链路

### 6.1 商家办单

1. 商家进入办单助手。
2. 选择商品、规格、套餐、订单周期、设备价、保证金、首期应付、增值服务。
3. 商家订单可由商家自主审核;联营/平台订单由运营端主控审核。
4. 联营收益、配资额度和内部资金字段不展示给客户。
5. 客户扫码进入 C 端订单确认页。

### 6.2 客户下单

1. 订单确认。
2. 资料补充。
3. 实名认证和人脸。
4. 信用/信息授权。
5. 业务审核。
6. 合同签署,可按后台模板选择默认合同或指定合同模板。
7. 公证,按配置决定是否需要。
8. 保证金/首期支付。
9. 代扣绑卡。
10. 发货或门店交付。
11. 客户确认收货。
12. 进入履约中。

### 6.3 发货和签收

发货时必须选择发货方式:

- 门店自提
- 顺丰物流
- 其他物流
- 平台配置的其他方式

发货时填写设备识别码,例如 IMEI / SN / VIN / 车架号。长租不提前占用库存。

客户确认收货支持两种方式:

1. 电子确认单:V1 默认方案。
2. AI 问答确认:保留配置和状态设计,后续增强。

## 7. 设备边界

### 7.1 长租设备档案

长租不走库存状态机。长租只在发货/交付时填写设备识别码,客户确认收货后生成设备档案。

长租设备档案用于:

- 订单绑定
- 设备识别码留存
- 发货/签收证据
- 监管锁状态展示
- 归还、留购、售后记录

### 7.2 短租车辆库存

短租车辆库存只服务体验租业务线,用于真实车辆的可租、预约、取车、在用、还车、维保、下架等状态流转。

V1 长租开发不得依赖短租库存。短租车辆库存进入 V1,但必须以独立表、独立状态机、独立验收路径实现。

## 8. 体验租短租主链路

### 8.1 C 端入口

C 端首页拆为两个清晰板块:

- 安心用:长租主营链路,按商品、规格、套餐、合同、账单和履约周期下单。
- 体验租:短租链路,按城市、品类、门店、车辆、租期方案和取还车规则下单。

体验租入口支持:

1. 选择城市和定位。
2. 选择品类,例如电动车,后续可扩展手机、相机、工具设备等。
3. 选择租期类型,例如小时租、日租、周租、月租。
4. 展示附近可服务门店和可租车辆。
5. 客户确认订单、骑行人信息、保证金/免押、协议、支付。

### 8.2 体验租订单流

1. 客户选择门店和车辆/车型。
2. 客户选择租期方案和取车时间。
3. 客户填写或选择骑行人。
4. 系统完成实名、人脸、年龄、证件、骑行资格等校验。
5. 客户确认协议、保证金/免押和支付。
6. 订单进入商家接单倒计时。
7. 商家接单后,客户到店取车。
8. 门店核销取车码或扫码绑车,车辆进入出租中。
9. 客户用车中可查看剩余时间、续租、导航到店、联系客服。
10. 客户到店还车,门店验车并确认费用。
11. 系统结算逾期费用、扣款、保证金释放或退款。

### 8.3 短租经营模式

| 模式 | 定义 | V1 要求 |
|---|---|---|
| 商家自营体验租 | 车辆归商家,门店自己经营短租 | 商家看车辆、订单、收入和保证金扣款 |
| 平台自营体验租 | 车辆归平台,平台或直营门店经营 | V1 支持基础订单和车辆归属,复杂调拨后置 |
| 平台资产商家履约 | 平台提供车辆,商家门店履约取还 | V1 可预留字段,正式结算细则按后续配置 |

## 9. IM 客服使用逻辑

### 9.1 入口原则

IM 客服不是孤立聊天插件,而是所有业务节点的统一服务入口。

C 端入口:

- 底部 Tab:客服。
- 订单详情:联系客服。
- 安心用流程异常页:联系客服。
- 体验租待接单、待取车、用车中、还车、扣款争议页:联系客服。
- 支付失败、合同失败、认证失败、控车异常页:联系客服。

商家端/门店端入口:

- 联系平台客服。
- 订单异常上报。
- 体验租控车异常上报。
- 财务/提现/结算问题咨询。

运营端入口:

- 客服中心。
- 订单详情内会话记录。
- 客诉、工单、退款、扣款、控车异常页面内关联会话。

### 9.2 会话上下文

客户进入 IM 时,系统必须带入上下文:

- 客户 ID、脱敏姓名、手机号、实名状态。
- 来源端:支付宝、微信、H5、App 预留。
- 业务线:安心用或体验租。
- 订单号、订单状态、商品/车辆、门店、商家。
- 当前页面和触发动作,例如支付失败、待取车、保证金扣款争议。
- 最近操作日志和关键待办。

客服工作台应优先展示上下文卡片,避免客服反复问客户订单号。

### 9.3 分流和处理

| 场景 | 默认队列 | 处理方式 |
|---|---|---|
| 售前咨询 | 综合客服 | 快捷回复、商品/门店链接 |
| 安心用订单 | 长租订单客服 | 查看订单卡片并跳转订单模块处理 |
| 体验租订单 | 体验租客服 | 优先处理接单、取车、控车、还车、逾期问题 |
| 财务/保证金/退款 | 财务客服 | 转工单或跳转财务模块 |
| 客诉/争议 | 客诉队列 | 转工单,记录证据和处理 SLA |
| 黑名单/高风险客户 | 主管队列 | 提醒客服谨慎沟通,禁止口头承诺 |

### 9.4 高危动作边界

IM 内可以发送订单卡片、门店信息、协议链接、客服说明和工单进度,但不得直接执行:

- 退款
- 关单
- 改价
- 保证金扣款
- 锁车/解锁/关机/开机
- 修改结算账户
- 修改合同模板
- 客户加黑名单

以上动作必须跳转到对应业务页面,由具备权限的人员完成二次确认并写入操作日志。

### 9.5 IM 与工单关系

IM 负责实时沟通,工单负责需要处理、追踪、审批、留证的事项。

触发转工单的典型场景:

- 体验租车辆无法取车、无法还车、控车失败。
- 保证金扣款争议。
- 退款/撤单/关单争议。
- 合同/公证/签收异常。
- 客户投诉商家或门店。
- 客服无法一次性解决的问题。

## 10. 权限原则

### 8.1 商家侧

| 操作 | 商家老板 | 管理员 | 财务人员 | 店长 | 普通员工 |
|---|---|---|---|---|---|
| 办单助手 | 是 | 可授权 | 否 | 可授权 | 可授权 |
| 商家订单审核 | 是 | 可授权 | 否 | 可授权 | 否 |
| 查看我的钱包 | 是 | 可授权 | 可授权 | 否/可选 | 否 |
| 发起提现 | 是 | 可授权 | 可授权 | 否 | 否 |
| 修改结算账户 | 仅老板 | 否 | 否 | 否 | 否 |
| 修改员工权限 | 是 | 可授权 | 否 | 否 | 否 |

### 8.2 运营侧

高危操作必须二次确认和完整日志:

- 提现审核和代付
- 退款执行
- 后台余额调整
- 修改结算账户审核
- 合同模板启停
- 抽佣/服务费率调整
- 关闭订单
- 修改订单金额
- 客户加黑名单
- 监管锁锁机/解锁

## 12. 开发冻结验收标准

- C 端正式页面不暴露资方、合作模式、资金来源、风控字眼。
- 三类长租订单的下单、审核、合同、付款、发货、签收、履约、账单、留购/提前结清链路闭环。
- 体验租短租的门店选择、车辆选择、骑行人、保证金/免押、接单、取车、用车中、续租、还车、扣款和控车异常链路闭环。
- IM 客服可从 C 端、商家端、门店端、运营端进入,并能携带订单/业务线/页面上下文转会话和工单。
- 资金链路从客户支付到商家钱包、内部资金台账、提现、退款冲正有完整记录。
