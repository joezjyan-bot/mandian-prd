# 计算器字段与账单公式表

> 所属端：门店手机端 / 办单助手
> 目标：把惠讯租 `phone-rent` 计算器逻辑转成新系统可配置字段。具体公式后续以 `phone-rent` 代码为准，本文件先固定字段、快照和数据流。

---

## 1. 核心口径

1. 办单助手 UI 和计算逻辑参考 `joezjyan-bot/calculator` 里的 `phone-rent`。
2. 新系统后台只负责提供商品、规格、租期、增值服务、费率和规则。
3. 门店订单使用商家端配置。
4. 分红订单、平台订单使用运营端配置。
5. 生成二维码时必须保存价格快照，客户扫码后订单沿用该快照。
6. 客服后续仍可改价，但改价必须生成新价格快照和操作日志。

| 入口 | 商品来源 | 费率来源 | 增值服务来源 | 审核主体 |

| 分红订单 | 平台/商家同步商品池 | 运营端配置 | 运营端配置 | 运营端 |
| 平台订单 | 平台商品池 | 运营端配置 | 运营端配置 | 运营端 |

## 3. 输入字段

### 3.1 商品字段

| 字段 | 说明 |

| 是否同步办单助手 | 控制是否展示 |

### 3.2 租赁字段

| 字段 | 说明 |
|---|---|
| 租赁模式 | 长租 / 短租 |

| 后续账单规则 | 每期租金、期数、账单日 |

### 3.3 金额字段

| 字段 | 说明 |
|---|---|
| 押金 | 固定金额、比例、免押或风控后调整 |

| 后期月付/期付 | 每期账单金额 |
| 设备管理费 | 可按商品或订单类型配置 |
| 公证费 | 可必选或可选 |
| 服务费 | 可计入首期或账单 |
| 运费 | 默认 0，可配置 |
| 留购价 | 到期购买价 |
| 留购总价 | 租金、押金、留购等口径合计 |

### 3.4 分红字段

分红订单增加：

| 字段 | 说明 |
|---|---|
| 设备价 | 门店填写或从商品指导价带入 |
| 需求配资比例 | 下拉选择 20% - 80% |
| 门店出资比例 | `100% - 需求配资比例` |
| 资方出资金额 | 设备价 * 需求配资比例 |
| 门店出资金额 | 设备价 - 资方出资金额 |
| 分账比例 | 按出资比例锁定 |

## 5. 账单生成规则

```text
办单助手价格快照
  -> 客户扫码确认
  -> 订单创建
  -> 生成首期支付单
  -> 审核通过后生成账单计划
  -> 客户支付或代扣
  -> 账单结清后触发分账
```

要求：

1. 首期实付和后续账单必须分开存。
2. 增值服务要标记计入首期、计入账单或单独收取。
3. 公证费、服务费、设备管理费必须在后台订单详情展示。
4. 客服改价后要保留原价、改后价、改价原因和操作人。
5. 客户已支付后再改价，必须走补差价、退款或补充合同流程。

## 6. 二维码和订单快照

二维码内容不直接暴露价格公式，只包含：

- 报价编号
- 订单类型
- 商家 / 门店
- 渠道归属
- 有效期
- 签名

客户扫码后根据报价编号读取服务端价格快照。

快照失效规则：

- 超过有效期。
- 商品下架。
- 规格停用。
- 配置版本被强制禁用。
- 门店账号被停用。

## 9. 常规方案公式

输入：

- 车辆售价：`price`
- 首付金额：`down`
- 期数：`term`
- 费率：`rate`

公式：

```text
手续费 = 车辆售价 × 费率 × 期数
预计总支出 = 车辆售价 + 手续费
剩余应付期数 = 期数 - 1
后期月付 = (预计总支出 - 首付金额) ÷ 剩余应付期数
```

代码口径：

```text
rate = rates[term] / 100
total = price + price * rate * term
monthly = (total - down) / (term - 1)
```

新系统落地：

- `total` 写入报价快照的预计总支出。
- `monthly` 写入账单计划的后续每期金额。
- `down` 写入首期实付。
- `term - 1` 是后续账单数量，不含首期。

---

## 10. 剩余应付系数方案公式

该模式用于低首付类方案，代码里 `mode = remainingMultiplier`。

输入：

- 车辆售价：`price`
- 首付金额：`down`
- 系数：`multiplier`
- 期数：`term`

公式：

```text
剩余应付总额 = (车辆售价 - 首付金额) × 系数
剩余应付期数 = 期数 - 1
后期月付 = 剩余应付总额 ÷ 剩余应付期数
预计总支出 = 首付金额 + 后期月付 × 剩余应付期数
```

代码口径：

```text
remainingTotal = (price - down) * multiplier
monthly = roundMoney(remainingTotal / (term - 1))
total = roundMoney(down + monthly * (term - 1))
```

新系统落地：

- `multiplier` 作为方案配置字段。
- 该模式不使用常规费率。
- 月付先按分四舍五入，再反算预计总支出。
- 这类方案可配置车辆售价上限和默认首付。

## 11. 当前方案配置抽象

| 配置字段 | 含义 | 新系统位置 |
|---|---|---|
| `rates` | 各期费率或系数 | 运营/商家费率方案 |
| `mode` | 计算模式 | 费率方案类型 |
| `maxPrice` | 车辆售价上限 | 商品/方案校验 |
| `defaultDown` | 默认首付 | 办单助手默认值 |
| `defaultNotary` | 默认勾选公证 | 增值服务默认值 |
| `brand` | 报价品牌 | 报价单展示配置 |
| `blue` | 报价单主题 | UI 展示配置 |

当前可抽象成两种方案类型：

| 方案类型 | 公式 |
|---|---|
| 常规费率方案 | `售价 + 售价 × 费率 × 期数` |
| 剩余应付系数方案 | `(售价 - 首付) × 系数` |

## 12. 当前期数与配置示例

当前静态计算器中，方案可配置不同期数：

- 常规方案常见：4 / 6 / 9 期。
- 部分地区方案：只开放 6 / 9 期或只开放 6 期。
- 剩余应付系数方案：7 / 10 期。

新系统要求：

1. 期数不能写死，必须由方案配置控制。
2. 未配置的期数不展示。
3. 每个订单保存当时使用的方案版本。
4. 期数展示可以仍按“全部可用期数一次展示”，方便门店给客户比较。

---

## 13. 公证费口径

当前计算器：

- 赋强公证固定 80 元。
- 勾选后单独展示。
- 不计入预计总支出。

新系统建议：

1. 公证费不写死，放到增值服务配置。
2. 可配置是否必选。
3. 可配置是否计入首期实付、单独支付或并入账单。
4. 报价快照必须保留客户看到的公证费展示口径。
