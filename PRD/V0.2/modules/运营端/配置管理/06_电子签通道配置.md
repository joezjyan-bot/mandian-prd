# 电子签通道配置

> **来源**: P1-4

## 1. 页面定位

运营端 · 配置管理 · 电子签通道配置。配置 e签宝 / 法大大等电子签通道的接入与切换。

## 2. 支持通道

V1 支持的通道:
- e签宝(e_sign_bao)
- 法大大(fa_da_da)
- (未来扩展)契约锁、上上签等

## 3. 主通道配置

平台配置当前主通道:
- 通道类型(e_sign_bao / fa_da_da / ...)
- 通道 API 配置(URL / API Key / Secret 等,走 `.env`)
- 通道是否启用(active / paused)
- 备用通道(可选,主通道失败时切换)

## 4. 通道接入设置

各通道独立配置:
- 应用 ID / 应用 Secret
- 回调地址(平台接收签署完成回调)
- 公司账号信息(用于签署平台一方)
- 签署模板配置(部分通道支持)
- 签署超时(默认 24 小时,可配)
- 签署顺序(平台 → 门店 → 客户 / 客户 → 门店 → 平台 等可配)

## 5. 通道切换

### 5.1 平台级切换
切换主通道:
- 仅运营管理员可操作
- 二次确认 + 原因说明
- 切换后**已发起未签的合同保持原通道**完成
- 新合同走新通道

### 5.2 异常切换
主通道暂时不可用时:
- 系统自动尝试备用通道
- 告警 + 通知运维

## 6. 通道经过中控台

通道调用都通过中控台对接(P0-4):
- 调用前查中控台本地缓存
- 调用后异步推送日志
- 中控台异步计费

详见 `modules/全局/09_中控台对接.md`

## 7. 监控指标

- 各通道当日成功调用数
- 各通道当日失败率
- 各通道平均响应时间
- 当前主通道 + 备用通道

## 8. 操作权限

- 运营管理员 + CTO/技术负责人:可配置通道
- 普通客服:仅查看通道状态

## 9. 操作日志

通道配置每次变更:留完整日志(谁改 / 时间 / 旧值 / 新值)

## 10. 配置存储

通道 API 凭证(敏感):`.env` 文件
通道选择(主通道、备用、超时):`platform_config` 表
