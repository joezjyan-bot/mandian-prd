# 全局措辞规范

> **适用范围**: 所有 PRD 文档、合同模板、C 端文案、商家端文案、运营端文案、数据库字段命名、接口/流水/通知/短信/邮件等所有外露内容
> **强制级别**: 必守

---

## 1. 强制替换的措辞

### 1.1 罚息 → 逾期违约金

| 维度 | 内容 |
|---|---|
| ❌ 禁止 | 罚息 |
| ✅ 使用 | 逾期违约金 |
| 原因 | "罚息"具有金融业务属性,可能被定性为金融业务;"逾期违约金"为民事违约赔偿,合规边界清晰 |

**数据库字段建议**:
- ❌ `penalty_interest`
- ✅ `overdue_penalty` / `late_payment_penalty`

**流水备注**:
- ❌ `订单 XX 第 XX 期 罚息`
- ✅ `订单 XX 第 XX 期 逾期违约金`

---

### 1.2 买断 → 留购

| 维度 | 内容 |
|---|---|
| ❌ 禁止 | 买断 |
| ✅ 使用 | 留购 |
| 原因 | "买断"暗示销售/购买,可能被定性为融资租赁或分期销售;"留购"明确为"承租人选择留下租赁物"的合规表述 |

**数据库字段建议**:
- ❌ `buyout_price` / `buyout_amount` / `buyout_at`
- ✅ `purchase_price` / `purchase_at`

**C 端按钮**:
- ❌ "申请买断" / "立即买断"
- ✅ "申请留购" / "立即留购"

**合同条款**:
- ❌ "买断价格" / "买断条款"
- ✅ "留购价格" / "留购条款"

## 2. 执行规范

### 2.1 新建文档
直接使用 ✅ 列措辞。

### 2.2 修改现有文档
```bash
# 批量替换(确认无误后执行,先 git diff 检查)
find . -name "*.md" -exec sed -i 's/罚息/逾期违约金/g' {} +
find . -name "*.md" -exec sed -i 's/买断/留购/g' {} +
```

### 2.3 代码层
- 数据库迁移走 migration 框架重命名旧字段
- API 接口字段命名按 ✅ 列
- 新代码不得出现 ❌ 列词汇(包括变量名、注释、日志)

### 2.4 客服话术
- 平台客服与客户沟通使用 ✅ 列
- 客服培训材料统一更新

## 3. 维护规则

新增措辞规范时:
1. 在本文件添加新章节
2. 更新 `00_核心决策表.md` 顶部措辞规范表
3. 更新 `changelog.md`
4. 安排专人批量替换

本文件优先级**高于**任何具体业务文档。任何文档(包括 V0.1、第三方参考、临时讨论稿、测试用例)中出现 ❌ 列措辞,以本文件为准强制修订。
