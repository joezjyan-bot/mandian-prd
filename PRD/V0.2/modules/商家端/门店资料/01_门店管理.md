# 门店管理

> **来源**: P0-2

## 1. 页面定位

商家 PC · 门店资料 · 门店管理。商家老板 / 管理员管理本商家下的所有门店。

## 2. 业务背景

商家 1:N 门店(受控字典)。一张营业执照 = 一个商家主账号,商家下可创建多个门店。

门店在数据库是独立实体,为 V2 门店级钱包 / 资质留预留。

## 3. 列表

### 3.1 字段
- 门店 ID
- 门店名称
- 所在城市 + 详细地址
- 经营状态(营业中 / 暂停 / 关闭)
- 店长(姓名 + 手机)
- 员工数
- 在租订单数 / 累计订单数
- 入驻时间
- 操作:详情 / 编辑 / 停用 / 启用

### 3.2 顶部统计
- 门店总数
- 营业中 / 暂停 / 关闭数
- 累计订单 / 累计 GMV

## 4. 新增门店

商家老板 / 管理员可新增:
- 门店名称
- 详细地址(城市 + 区 + 详细)
- 联系电话
- 店长(从员工中选,可后续指定)
- 经营状态(默 营业中)

入驻后:
- 自动分配门店 ID
- 商家钱包共享(V1 不做门店级钱包)
- 留操作日志

## 5. 编辑

可编辑字段:
- 门店名称
- 地址 / 联系电话
- 店长(可调)
- 经营状态

不可编辑:
- 门店 ID
- 入驻时间

## 6. 停用 / 关闭

### 6.1 停用(暂停)
- 暂停接新单(不影响在租订单)
- 员工可见但无法办新单
- 可恢复

### 6.2 关闭(终态)
- 在租订单全部到期清算后才可关闭
- 关闭后不可恢复(实际为 status=archived)
- 历史数据保留

## 7. 门店与员工

- 门店有多个员工
- 员工可调动门店(详见 02_员工管理)
- 调动时:
  - 不影响员工历史订单上的门店快照
  - 仅影响后续新办单的归属

## 8. 订单归属冻结

订单创建时**冻结门店归属快照**:
- `order.store_id_at_order`:订单创建时的门店 ID
- `order.staff_id_at_order`:订单创建时的办单员工 ID
- `order.staff_role_at_order`:订单创建时的员工角色

后续员工调店 / 门店关闭都不影响这些历史快照。

## 9. 权限

- 商家老板(merchant_owner):全部能力
- 商家管理员(merchant_admin):新增 / 编辑 / 停用,不可关闭
- 店长(store_lead):查看本店,不可改门店本身
- 普通员工:查看本店,不可改
