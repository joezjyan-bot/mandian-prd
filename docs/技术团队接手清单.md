# 技术团队接手清单 · 满点租赁系统

> 本清单配合 `system/`(Laravel 脚手架)与 `docs/满点租赁系统_完整产品文档_V0.2.2.html` 使用。
> 用途:让技术团队明确"接手前要确认什么、要建哪些模块、每块开工前要问产品什么"。

---

## 一、动手前必须先确认(架构选型)

1. **PHP 框架:已定 Laravel(11)。** 脚手架按此搭建。
2. **代码仓库:** 现阶段脚手架在 `mandian-prd/system/`。团队若有独立生产仓库,需决定迁移方式(旧 `mandian-java-rebuild` Java 仓已废弃,主系统 PHP、`yzz` 前缀)。
3. **中控台(Java)对接:** HTTP API + Webhook,无 SDK,PHP 用 Guzzle 调。需确认接口契约谁定、放哪、鉴权方式。
4. **数据库:** 选型 + 金额一律用整数"分"(脚手架已遵循,所有金额字段 bigint)。
5. **模拟模式:** 脚手架已实现 `EXTERNAL_MODE=mock|real` 开关(`config/external.php` + `ExternalServiceProvider`)。演示先全 mock,真实对接时逐个填 `app/Services/External/Real/*`。

## 二、该建哪些模块(按优先级)

**第一优先(脚手架已起骨架,团队补全):**
- 用户与多端认证(C端/商家/门店/运营)— *脚手架未含 auth,需团队加 sanctum/jwt + 权限中间件*
- 商品与设备价配置
- 办单助手(算价 + 报价快照 + 二维码)— 蓝本 `joezjyan-bot/calculator` 的 `phone-rent/`
- 订单 + 状态机 — *脚手架已起:Order/Bill/OrderService*
- 合同签署【模拟已通】、支付【模拟已通】、交付签收
- 账单与履约、到期三选一 — *脚手架已起 expiryChoice*

**第二优先(资金 + 后台,脚手架已起四账骨架):**
- 四账分离 — *脚手架已建表 + FinancePostingService 起步;分账比例、对账、退款冲正需团队精化*
- 提现、退款、对账中心
- 运营后台、三类合作模式差异

**第三优先:** 权限审计、风控、监管锁【模拟已通】、IM 客服、设备资产、短租取还车、法诉证据包。

## 三、每个模块开工前要问产品的事
- 状态机有哪些状态、怎么流转?(`PRD 全局/02 状态字典`)
- 哪些字段绝不能给 C 端?(C 端红线,见 `Order::toCustomerArray()`)
- 哪些是高危操作、要审批留痕?(`PRD 权限矩阵`)
- 钱怎么算怎么分?(`PRD 财务流水模型 06`、`合作模式与收益分配 12`)

## 四、关键提醒(替团队踩坑)
- **申请购买价 A 口径**(剩余租金+保证金)合规敏感、待法务确认;脚手架做成可切换策略(`BuyoutPriceCalculator`,`config('business.buyout_formula')`),二期切折旧余值只改这一处。
- 所有资金回调**必须幂等**(脚手架用 `callback_event_id` 唯一约束示范)。
- 模拟/真实在**同一接口层**切换,业务代码不感知 —— 这是"后期直接换对接"的关键。
- 全仓搜 `TODO[团队]` 可列出所有待补点。

## 五、脚手架怎么跑(团队验证)
```bash
cd system && composer install && cp .env.example .env
php artisan key:generate
# 配 .env 数据库,EXTERNAL_MODE=mock
php artisan migrate --seed && php artisan serve
# GET /api/health → {"status":"ok","external_mode":"mock"}
```

⚠️ 脚手架可演示、结构可接手,但碰钱/碰合规/碰真实通道的部分必须团队联调+测试+灰度后才能上线;A 口径以法务终审为准。本脚手架不构成法律意见。
