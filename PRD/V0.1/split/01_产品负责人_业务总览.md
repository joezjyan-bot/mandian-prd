## 1. 文档说明

### 1.1 文档目标

本 PRD 用于指导技术团队完成现有"满点 / 悦租租"系统的重构。重构目标：

1. **业务清晰**：解决现有系统中订单类型、资金流、角色关系混乱的问题
2. **可配置化**：所有业务规则由后台配置驱动
3. **多客户部署**：一套代码部署到 10+ 客户环境
4. **第三方中控**：第三方服务统一接入中控平台
5. **自研 IM**：替代"拉微信群审核"模式
6. **数据隔离**：客户之间数据完全隔离

### 1.2 文档边界

**V0.1 包含**：三端模块划分、订单模型、资金流、计费、状态机、IM、中控、配置中心、多租户、数据模型概览、短租架构预留。

**V0.1 不包含**：短租业务详细规则（V0.2 补充）、完整接口文档、完整 UI 原型、测试用例、运维手册。

## 2. 商业模式概述

### 2.1 业务本质

**"以租代售"** 模式的耐用品分期租赁：客户分期支付租金，租期内拥有设备使用权，租期结束可选归还/续租/留购。

### 2.2 资金真相

```
门店采购手机 → 客户向门店发起租赁
            → 平台/资方"从门店购买"这台手机（采购合同生效）
            → 平台支付采购款到门店采购账户
            → 平台与客户签租赁合同，客户分期付款给平台
            → 月付分账给资方/门店/平台
```

### 2.3 三类订单的本质区别

| 维度 | 门店订单 | 分红订单 | 平台订单 |
|---|---|---|---|
| **出资方** | 门店 100% | 门店出货+资方配资 | 门店纯送单 |
| **审核方** | 门店自审 | 平台审核 | 平台审核 |
| **签约方** | 客户+门店+平台 | 客户+门店+平台 | 客户+平台+执行商家 |
| **客户违约损失承担** | 门店 | 资方/平台 | 平台 |

**重构设计原则**：订单类型不写死，用"出资比例"字段驱动：
- 100% → 门店订单
- 1-99% → 分红订单
- 0% → 平台订单

### 2.4 业务链上的 6 类角色

| 角色 | 端 |
|---|---|
| 平台运营 | 运营端（PC 后台）|
| 资方 | 资方端（H5 只读）|
| 商家 | 商家端（PC 后台）|
| 门店 | 门店端（H5 移动）|
| 渠道 | 渠道端（H5 报表）|
| C 端客户 | 小程序/App |

商家与门店是 **1:N 关系**（也支持 1:1）。

### 2.5 平台收益来源

1. **会员费**：每笔分红/平台订单 99 元（**配置化**）
2. **加价部分**：客户总租金 - 设备价，与资方按约定分润
3. **手续费**：门店订单按总租金扣 X%（**配置化**）
4. **第三方接口差价**

### 2.6 商业模式扩展性

预留：短租、维修保养、配件商城、质保服务。

## 4. 核心业务模型

### 4.1 订单类型驱动模型

#### 4.1.1 设计思想

**不要把订单类型写死成枚举**，而是用多个字段组合驱动：

| 字段 | 含义 |
|---|---|
| `order_source` | 订单来源（C 端 / 门店扫码 / 渠道 / 商品码）|
| `funding_ratio` | 门店出资比例（0% / 1-99% / 100%）|
| `audit_owner` | 审核归属（STORE / PLATFORM）|
| `signing_subjects` | 签约主体集合 |
| `rental_term_type` | 租期类型（LONG_RENT / SHORT_RENT_*）|
| `goods_type` | 商品类型 |
| `risk_strategy_id` | 风控策略 ID |

#### 4.1.2 三类订单的映射

```
门店订单 = funding_ratio: 100% + audit_owner: STORE
分红订单 = funding_ratio: 1-99% + audit_owner: PLATFORM
平台订单 = funding_ratio: 0% + audit_owner: PLATFORM
```

### 4.2 订单状态机

#### 4.2.1 长租订单完整状态流转

```
DRAFT → PENDING_AUDIT → PENDING_TELE_AUDIT(可选) → APPROVED
      → PENDING_SIGN → PENDING_PAY → PENDING_SHIP → PENDING_RECEIVE
      → RENTING ⇄ OVERDUE → PENDING_RETURN → PENDING_SETTLE → COMPLETED
任何状态 → CLOSED (取消/超时/拒签)
OVERDUE → LITIGATION (严重逾期转起诉) → COMPLETED
```

#### 4.2.2 状态枚举详表

| 状态 | 中文名 | 业务含义 |
|---|---|---|
| `DRAFT` | 草稿 | 门店开单未提交 / 客户填资料中 |
| `PENDING_AUDIT` | 待审核 | 资料齐全等待审核 |
| `PENDING_TELE_AUDIT` | 待电审 | 进入电话审核环节 |
| `APPROVED` | 审核通过 | 准备签约 |
| `PENDING_SIGN` | 待签约 | 三方电签中 |
| `PENDING_PAY` | 待支付 | 等首付支付 |
| `PENDING_SHIP` | 待发货 | 首付到账，等门店发货 |
| `PENDING_RECEIVE` | 待收货 | 已发货 |
| `RENTING` | 租赁中 | 分期还款中 |
| `OVERDUE` | 逾期 | 当前期账单超期未还 |
| `PENDING_RETURN` | 待归还 | 租期结束 |
| `PENDING_SETTLE` | 待结算 | 等财务结算 |
| `COMPLETED` | 已完成 | 全部结算完成 |
| `REJECTED` | 已驳回 | 审核未通过 |
| `CLOSED` | 已关闭 | 取消/超时/拒签 |
| `LITIGATION` | 起诉中 | 严重逾期已诉 |

### 4.3 资金流转模型

#### 4.3.1 资金链路全景

客户 → 平台账户 → 按订单类型分账给资方/门店/渠道/平台收益。

#### 4.3.2 三种订单的资金流差异

**门店订单**：客户付款 → 平台扣手续费 → 剩余给门店软钱包 → 门店提现。客户违约由门店承担。

**分红订单**：订单成立时资方资金池扣 [设备价×配资比例] 给门店；客户还款扣会员费+加价后按比例分门店和资方。客户违约由资方/平台承担。

**平台订单**：平台从资方资金池支出全部采购款给执行商家；客户还款扣会员费+加价后门店得佣金。

#### 4.3.3 软钱包账户体系

| 账户类型 | 用途 | 可提现 |
|---|---|---|
| 可用余额 | 已结算资金 | ✅ |
| 结算中余额 | 客户已还款未到结算日 | ❌ |
| 冻结金额 | 提现中、争议中 | ❌ |

#### 4.3.4 软钱包允许负数

退款后门店软钱包可能变负，禁止新订单分账入账，先用新订单收入抵扣负数。

### 4.4 计费规则（搬运惠讯租办单助手）

#### 4.4.1 公式

```
首付金额 = price × ratio
未付金额 = price - 首付金额
后续应还总额 = 未付金额 × 费率（查表）
后期月付 = 后续应还总额 ÷ (periods - 1)
押金 = 首付金额 - 10
留购总价 = 首付金额 + 后续应还总额 + 设备管理费
```

#### 4.4.2 费率表

| 期数 | 首付 30% | 首付 40% | 首付 50% | 首付 60% |
|---|---|---|---|---|
| 6 期 | 1.26 | 1.20 | 1.20 | 1.15 |
| 9 期 | 1.30 | 1.28 | 1.26 | 1.21 |
| 12 期 | 1.37 | 1.30 | 1.30 | 1.28 |

#### 4.4.5 留购价（按期递减）

最后一期：仅押金（押金抵扣后实付 = 0），仍走留购合同电签，押金不退还。

#### 4.4.6 配置化层级

```
全局 → 客户 → 门店 → 商品
```

下层覆盖上层。

### 4.5 退款 / 退单流程

#### 4.5.1 退单时机

| 时机 | 资金处理 |
|---|---|
| 签约前 | 关闭 |
| 签约后未支付 | 撤销签约 |
| 支付首付未发货 | 首付原路退回 |
| 已收货 | 扣违约金，原路退回 |
| 租赁中 | 已付租金扣违约金退回 + 押金扣损耗 |

#### 4.5.3 退款审核

- ≤500 元：自动通过
- >500 元：运营审核（金额阈值**配置化**）

#### 4.5.4 违约金规则

| 时机 | 违约金 |
|---|---|
| <7 天 | 5% |
| 7-30 天 | 10% |
| >30 天 | 15% |
| 提前退租 | 剩余未付租金×30% |

### 4.6 续租流程

- 到期前 7 天通知客户
- 续租租金按当前价 + 费率表重算（不沿用原费率）
- 押金沿用、不重新收
- 签独立续租合同（关联原订单）
- 不重走采购款（设备权属沿用）

### 4.7 账单日修改

| 状态 | 改首期 | 改后续 |
|---|---|---|
| 7 天内 | ✅ | - |
| >7 天，未付首期 | ❌ | - |
| 已付首期 | ❌ | ✅ |
| 当前逾期 | ❌ | ❌ |
| 逾期结清后 | ❌ | ✅ |

- 必须由客服在 IM 工单发起
- 系统自动重算每期账单（总费用不变）
- 超 X 天加收延期费
