# 接口权限矩阵

> 所属：开发设计
> 目标：把页面权限矩阵继续落到接口层，明确每类角色能调用什么接口、看什么数据、哪些动作需要二次验证或审批。

---

## 1. 角色编码

| 角色编码 | 说明 |
|---|---|
| `ops_admin` | 平台管理员 |
| `ops_manager` | 运营主管 |
| `review_staff` | 审核客服 |
| `finance_staff` | 财务 |
| `finance_manager` | 财务主管 |
| `post_rent_staff` | 租后客服 |
| `complaint_staff` | 客诉客服 |
| `merchant_owner` | 商家老板 |
| `merchant_staff` | 商家员工/店员 |
| `channel_user` | 渠道账号 |
| `customer_user` | 客户 |
| `system_worker` | 系统任务和回调 worker |

---

## 2. 权限标记

| 标记 | 说明 |
|---|---|
| R | 可读 |
| W | 可写 |
| A | 需要审批 |
| 2FA | 需要二次验证 |
| LOG | 必须写操作日志 |
| MASK | 默认脱敏 |
| OWNER | 只能看自己主体数据 |
| SCOPE | 按运营分配的数据范围 |

---

## 11. 后端拦截要求

1. 所有接口都必须经过认证、角色权限、数据范围、按钮动作四层校验。
2. 字段脱敏不能只靠前端，后端响应前必须根据角色处理。
3. 高风险动作即使前端隐藏按钮，后端也必须拒绝越权请求。
4. 导出任务必须复用同一套字段权限和数据范围。
5. 系统 worker 只能调用内部授权接口，不允许复用人工用户权限。
