## 2. 核心口径

商品不是库存,商品只是展示和价格配置。设备库存主要服务短租商品;长租无需提前建立库存,只在发货/交付后沉淀到 `01_长租设备档案.md`。

| 层级 | 说明 |
|---|---|
| 商品 | 展示给客户或门店办单助手看的租赁品类 |
| 规格 | 商品下的价格、成色、属性、长租/短租适配 |
| 设备库存 | 短租实际租出去的一台实物,必须有唯一设备码 |
| 仓库 | 设备当前所在位置,可以是平台仓、商家仓、门店仓 |

短租必须绑定设备库存。长租不走库存锁定和库存状态,只在发货或门店交付节点填写设备识别码,例如 IMEI、SN、VIN。

## 4. 设备状态(9 个,以 P1-1 决策表为准)

| # | 编码 | 文案 | 是否可出租 | 说明 |
|---|---|---|---|---|
| 1 | `DEVICE_PENDING_IN` | 待入库 | 否 | 已采购/已配货,未完成入库登记 |
| 2 | `DEVICE_AVAILABLE` | 在库可租 | 是 | 入库完成,可对外出租 |
| 3 | **`DEVICE_PENDING_RESERVATION`** | **预约保留** | 否 | **(新增 P1-1)** 客户已扫码或办单中,设备暂时保留 |
| 4 | `DEVICE_LOCKED` | 已锁定 | 否 | 已支付,等待发货/客户取走 |
| 5 | `DEVICE_RENTING` | 出租中 | 否 | 已发货/已取走,在客户手中 |
| 6 | `DEVICE_RETURN_PENDING` | 归还待验 | 否 | 客户已归还,等待门店验收 |
| 7 | `DEVICE_REPAIRING` | 维修中 | 否 | 验收发现问题,送修 |
| 8 | **`DEVICE_DISPUTE`** | **争议中** | 否 | **(P1-1 强调)** 门店与客户对设备状态有分歧 |
| 9 | `DEVICE_RETIRED` | 已下架 | 否 | 报废/出售二手/退还供应商/丢失/留购完成等终态 |

**列表展示规则**:默认筛选器**排除 `DEVICE_RETIRED`**(已下架),需要查看时切到"已下架设备"专属视图。

## 5. 设备状态流转

```mermaid
flowchart TD
    A["待入库"] --> B["在库可租"]
    B --> RES["预约保留"]
    RES -->|取消预约| B
    RES -->|客户支付| C["已锁定"]
    B --> C
    C --> D["出租中"]
    C -->|订单取消| B
    D --> E["归还待验"]
    E -->|验收通过| B
    E -->|损坏/缺件| F["维修中"]
    E -->|双方分歧| DIS["争议中"]
    F -->|维修完成| B
    F -->|不值得修| G["已下架"]
    DIS -->|协商一致| E
    DIS -->|协商下架| G
    B --> G
```

规则：

1. 只有 `DEVICE_AVAILABLE` 在库可租可以被短租订单选择。
2. **客户下单或门店生成短租二维码后,进入 `DEVICE_PENDING_RESERVATION` 预约保留**;支付成功转 `DEVICE_LOCKED` 已锁定。
3. 预约保留超时未支付,设备自动释放回 `DEVICE_AVAILABLE`。
4. 归还后必须验收,验收通过才回到可租库存。
5. 验收发现问题且双方有分歧 → `DEVICE_DISPUTE` 争议中,客服介入。
6. 维修中、已下架、争议中设备不能出租。
7. 所有状态变化必须写操作日志。
8. 长租订单不进入本状态流转;长租发货记录只保存设备识别码和交付证据。
9. 监管锁不是设备库存固定编号字段,锁机、解锁、支付模式等能力在订单锁控台和监管锁接口配置中处理。

## 7. 操作按钮

| 操作 | 使用场景 | 结果 |
|---|---|---|
| 新增设备 | 单台入库 | 创建设备档案 |
| 批量导入 | 大量设备入库 | 按模板导入设备码和归属 |
| 入库 | 设备到仓 | 状态变为在库可租 |
| **预约保留** | **客户扫码/办单中** | **状态变为预约保留(可超时自动释放)** |
| 锁定 | 短租客户支付完成 | 状态变为已锁定 |
| 出库/交付 | 客户取货或门店交付 | 状态变为出租中 |
| 归还 | 客户归还 | 状态变为归还待验 |
| 验收通过 | 归还无异常 | 状态变为在库可租 |
| 报修 | 损坏或异常 | 状态变为维修中 |
| **标记争议** | **门店与客户分歧** | **状态变为争议中,客服介入** |
| 下架 | 不再出租 | 状态变为已下架,必填 downgrade_reason |

## 8. 与订单的关系

| 场景 | 规则 |
|---|---|
| 长租订单 | 不走库存;发货或交付时填写 IMEI/SN/VIN 等设备识别码 |
| 短租订单 | 必须绑定具体设备 |
| 商家订单 | 绑定门店或商家自己的设备 |
| 联营订单 | 按平台/商家配置决定设备归属和出资逻辑 |
| 平台订单 | 由平台或指定执行方提供设备 |
| 归还订单 | 归还验收结果影响设备状态和保证金/赔付 |
| **留购订单** | **设备状态变为 `DEVICE_RETIRED`,`downgrade_reason = liugou`** |

## 9. 数据互通

| 模块 | 互通内容 |
|---|---|
| 商品管理 | 商品、规格、是否适配短租、是否需要设备绑定 |
| 办单助手 | 当前长租不读取库存;短租后续需求包接入可用设备库存 |
| 订单管理 | 长租保存发货设备识别码;短租保存绑定设备、交付、归还、异常 |
| 仓库管理 | 设备所在仓库、出入库记录 |
| 监管锁 | 订单维度锁机/解锁、支付模式、状态回调 |
| 财务 | 保证金、损坏赔付、超时费用、短租账单 |
| 租后管理 | 逾期未还、归还异常、维修处置 |
| 操作日志 | 设备创建、入库、锁定、出库、归还、维修、下架、**预约保留、争议** |
