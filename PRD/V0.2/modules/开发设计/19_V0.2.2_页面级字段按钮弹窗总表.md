# V0.2.2 页面级字段按钮弹窗总表

> **封版声明**:本文是 V0.2.2 页面级开发契约。所有页面开发必须同时满足本文、状态字典、权限矩阵、OpenAPI 契约和全局措辞规范。若页面原型与本文冲突,以本文为准后再由产品更新原型。

## 一、全局页面规则

1. 展示层统一使用 **商家订单 / 联营订单 / 平台订单**;内部枚举可保留 `merchant / joint / platform` 等兼容值,但不得直接展示。
2. C 端不得展示资方、资金来源、合作模式、风控结论、内部审核备注、服务费拆分、商家结算价、平台利润、返点规则、黑灰名单结论。
3. 所有金额字段必须展示单位、精度和口径:订单总额、首期应付、每期应付、保证金、服务费、逾期费用、退款金额、结算金额。
4. 所有高危操作按钮必须具备禁用原因、二次确认、权限校验、审计日志和幂等键。
5. 状态页必须展示当前状态、下一步动作、失败原因、最后更新时间、责任方和客服入口。
6. IM 客服内不得直接执行退款、扣款、锁车、关单、改价、强制状态变更等高危动作;只能跳转到对应业务页面并留痕。
7. 长租未满足“客户签收确认 + 监管锁已上锁 + 激活锁已激活”三条件时,财务结算按钮必须禁用或进入人工复核,并展示未满足项。
8. 账单生成前允许有权限客服改价;账单生成后任何人不得直接修改账单,只能通过冲正、补差、退款或重新生成规则处理。

## 六、空状态、错误状态、失败状态

| 场景 | 页面表现 | 必备动作 | 日志要求 |
|---|---|---|---|
| 列表无数据 | 展示当前筛选条件和清空筛选按钮 | 清空筛选、刷新 | 记录查询条件 |
| 接口超时 | 展示重试按钮和客服入口 | 重试、复制错误码 | 记录接口调用失败 |
| 支付成功未入账 | 展示“支付处理中”而非失败 | 刷新、联系客服 | 支付回调幂等和补偿任务 |
| 签约失败 | 展示失败原因和重发入口 | 重新发起、联系客服 | 电子签回调日志 |
| 监管锁失败 | 展示人工复核入口 | 查询锁状态、提交复核 | 控车调用日志 |
| 退款失败 | 展示失败原因和处理状态 | 重新发起、人工处理 | 退款流水和冲正日志 |
| 证据包导出失败 | 展示失败附件和重试入口 | 重试、补附件 | 导出任务日志 |

## 七、页面验收标准

1. 每个页面必须能从本文找到字段、按钮、禁用规则、弹窗和权限要求。
2. 每个高危按钮必须能找到权限点、二次确认、审计日志和失败回滚规则。
3. 每个 C 端页面必须通过敏感字段隔离扫描。
4. 每个资金页面必须能落到四账分离模型。
5. 每个证据页面必须能落到法诉证据包目录。
6. 每个接口错误必须在页面展示可理解的失败原因和下一步动作。
