# 头部平台 C 端视频拆解与前端改造建议

> [历史参考 / V0.2.2 口径覆盖] 本文保留 C 端竞品拆解和体验租入口参考,其中早期“分红 / 资方 / 风控”等内部字眼不得作为 C 端展示口径;以 `docs/V0.2.2_P0_FIXUP_TASK.md` 和开发冻结总 PRD 顶部 V0.2.2 补丁为准。

> 来源：2026-05-25 用户提供的两个头部平台前端录屏。
> 用途：沉淀 C 端前台页面结构、套餐选择、告知授权、安心用长租与体验租短租的统一入口设计。

---

## 1. 拆解结论

两个录屏共同证明：C 端不能只做扫码下单页。客户需要先看到一个完整的前台入口，再分别进入长租和短租两条链路。

建议满点 C 端底部导航统一为：

| 导航 | 定位 | 说明 |
|---|---|---|
| 首页 | 综合入口 | 展示搜索、城市、活动、品类、安心用与体验租入口 |
| 安心用 | 长租以租代购 | 手机、电动车等长期租赁，走办单助手/商品详情/审核/合同/支付/发货 |
| 体验租 | 短租自营/门店经营 | 按城市、门店、车型、库存、小时/天/周/月方案下单 |
| 订单 | 统一订单中心 | 安心用订单和体验租订单分业务线筛选，不混状态机 |
| 我的 | 账户中心 | 实名、授权、地址、协议、客服、优惠券、售后入口 |

核心原则：

1. 安心用和体验租在 C 端同一个 App/小程序内展示，但业务链路、商品配置、库存模型、订单状态机分开。
2. 首页可以做综合导流，真正下单时必须进入对应业务线。
3. 体验租必须以门店和设备库存为核心，不能复用长租办单助手的锁价二维码逻辑。
4. 安心用客户侧不得出现分红、资方、分账、收益等内部概念。

## 2. 视频一页面拆解

|---|---|---|

| 综合首页 | 搜索、定位、分类、活动 banner、推荐商品、底部导航 | 满点首页作为业务入口，展示安心用与体验租，不直接承载复杂下单 |
| 分类页 | 横向品类、图标宫格、商品列表、筛选 | 安心用按品类、品牌、成色、租期、价格区间筛选 |
| 商品列表 | 商品图、标题、标签、日租价格、服务标签 | 长租商品列表展示最低月供/首期，不用日租价误导客户 |
| 商品详情 | 店铺信息、保障标签、反诈提醒、价格、规格、租赁须知、参数 | 满点商品详情必须展示门店/平台主体、服务保障、费用说明、反诈提醒、合同签署说明 |
| 规格/套餐弹层 | 规格、租期、数量、服务选项、费用快照 | 满点长租套餐弹层必须同步办单助手价格快照，客户只选择允许改的项 |
| 订单确认 | 商品、规格、费用、地址、服务、协议、合计、提交 | 满点订单确认页要展示首期、每期、押金、服务费、公证费、留购价、协议勾选 |
| 协议列表 | 租赁服务协议、隐私协议、个人授权、征信授权、数字证书协议 | 满点协议中心统一按链路配置展示必签/选签协议 |
| 人脸认证 | 亮度、正脸、本人操作提示，开始验证 | 满点实名/签收/风控人脸页必须先提示用途和授权范围 |
| 我的/订单 | 订单状态、空状态、常用工具、客服 | 满点订单中心按安心用/体验租分组，客户能直接看到待办 |

## 3. 视频二页面拆解

| 页面/环节 | 观察到的功能 | 满点借鉴方式 |
|---|---|---|
| 万物租赁首页 | 大促 banner、搜索、分类、活动、商品流 | 满点首页可有运营位，但不得做成纯营销页；必须能进入业务办理 |
| 车辆短租入口 | 城市定位、门店、地图/列表、车型卡片 | 体验租首页按城市和门店库存展示，不从长租商品详情硬跳 |
| 门店选择 | 距离、地址、营业时间、联系商家、导航 | 体验租必须有门店卡、营业状态、库存摘要和导航 |
| 车辆选择 | 车型、卖点、价格、立即租 | 车辆列表展示车型，不暴露具体 VIN，具体设备在取车核销时绑定 |
| 短租方案 | 1 小时、2 小时、4 小时、日租等，合计金额实时变化 | 体验租价格方案后台配置，支持小时、天、周、月，前端只展示该门店该车型可售方案 |
| 骑行人资料 | 证件类型、证件号、姓名、手机号、紧急联系人 | 体验租独立维护骑行人，不能默认等同下单人；可复用实名结果 |
| 人脸验证 | 协议勾选、人脸识别、验证通过 | 体验租下单前完成实名/骑行人/人脸校验 |
| 支付/免押 | 全额支付、押金/免押、授权、支付结果 | 体验租支付拆租金、保障包、押金/预授权，押金规则和安心用分开 |
| 订单状态 | 待确认、待取车、已取车、倒计时、核销码、控车、续租、还车 | 体验租订单状态独立，重点是接单、取车、用车、续租、还车、押金处理 |
| 地址/区域选择 | 省市区选择、地址维护 | 安心用发货地址和体验租取还门店地址分开维护 |

## 4. 满点 C 端推荐页面架构

### 4.1 首页

首页展示：

- 当前城市/定位授权状态。
- 搜索框：支持商品、门店、订单号、品牌关键词。
- 业务入口：安心用、体验租。
- 品类导航：手机、电动车、电脑平板、数码设备、智能穿戴、出行设备等。
- 活动 banner：可跳安心用商品、体验租门店、指定品类。
- 推荐商品区：按业务线打标，例如“安心用”“体验租”。
- 风险提示：反诈提醒、平台官方客服提醒、合同签署提醒。
- 悬浮客服：进入客服时带页面来源。

### 4.2 安心用

安心用是长租以租代购业务。页面链路：

```text
安心用首页
  -> 分类/搜索列表
  -> 商品详情
  -> 选择规格/长租套餐
  -> 订单确认
  -> 登录/授权
  -> 实名认证
  -> 资料补充
  -> 风控授权
  -> 合同/公证
  -> 押金/首期/代扣
  -> 待发货/待收货
  -> 在租/续租/留购/归还
```

安心用商品详情必须展示：

- 商品主体：商品名、主图、规格、设备价、成色、参考价。
- 门店/商家/平台主体：客户知道由谁提供服务。
- 价格快照：首期、每期、期数、押金、服务费、公证费、留购价、总金额。
- 服务保障：正品保障、合同保障、客服协助、发货签收说明。
- 租赁须知：逾期违约金、归还要求、留购规则、续租规则、损坏赔付入口。
- 反诈提醒：禁止向非平台账户转账，禁止相信非官方回收/退款电话。

### 4.3 体验租

体验租是短期租赁业务。页面链路：

```text
体验租首页
  -> 选择城市/定位
  -> 选择品类
  -> 选择门店
  -> 选择车型/商品
  -> 选择短租方案
  -> 填写骑行人/使用人
  -> 实名/人脸
  -> 支付租金/押金或免押授权
  -> 商家接单
  -> 到店取车/核销/绑车
  -> 使用中/控车/倒计时
  -> 续租或还车
  -> 门店验收
  -> 押金处理/完成
```

体验租必须单独展示：

- 城市、门店、距离、营业时间、联系商家、导航到店。
- 品类：电动车、手机、数码设备等，第一版可以先重点电动车。
- 租赁周期：时租、日租、周租、月租，是否展示由后台配置。
- 可租库存：只展示车型库存摘要，不在下单前暴露唯一设备码。
- 取还方式：第一版默认到店取还，后续扩展异地还车。
- 核销码：支付后生成，门店确认时绑定唯一设备码。
- 控车能力：开机、关机、闪灯鸣笛、定位等必须走权限和日志。

## 5. 套餐选择规则

### 5.1 安心用长租套餐

长租套餐来源于办单助手或商品价格配置。客户侧可见字段：

| 字段 | 是否可选 | 说明 |
|---|---|---|
| 商品规格 | 可选/锁定 | 颜色、容量、成色等，按商品配置展示 |
| 租期 | 可选/锁定 | 月为主，也可按配置支持天；当前不展示小时租 |
| 首期金额 | 只读 | 来自价格快照 |
| 每期金额 | 只读 | 来自价格快照 |
| 押金 | 只读/信用减免 | 展示免押或需付押金 |
| 服务费/公证费 | 只读 | 明示收费原因 |
| 增值服务 | 可选/客服可调整 | 保障服务、配送服务等 |
| 留购价 | 只读 | 统一叫留购价，不使用历史旧称 |

长租客户侧不展示：

- 分红、分账、收益、资方出资、配资比例。
- 平台抽佣、通道成本、内部审核策略。

### 5.2 体验租短租套餐

短租套餐来源于体验租价格方案。客户侧可见字段：

| 字段 | 是否可选 | 说明 |
|---|---|---|
| 城市/门店 | 可选 | 决定库存和价格 |
| 品类/车型 | 可选 | 展示车型，不展示 VIN |
| 租赁周期 | 可选 | 时租、日租、周租、月租 |
| 取车时间 | 可选 | 默认立即取车，也可预约 |
| 还车时间 | 系统计算/可选 | 按套餐计算，日租/周租/月租可选择日期 |
| 保障包 | 可选 | 保险/意外/损坏保障 |
| 押金/免押 | 系统判断 | 根据门店、车型、客户信用和规则 |
| 合计金额 | 实时计算 | 选择变化时必须刷新 |

## 7. 与悦租汇后台办单流程对齐

前端下单必须向后台传递并冻结以下快照：

- `business_line`：`assurance_rent` 安心用 / `experience_rent` 体验租。
- `order_type`：门店订单 / 分红订单 / 平台订单；体验租使用独立短租订单类型。
- `merchant_id`、`store_id`、`channel_id`、`clerk_id`。
- `product_id`、`sku_id`、`price_plan_id`。
- 长租：设备价、首期、每期、期数、押金、服务费、公证费、留购价。
- 短租：城市、门店、车型、租赁周期、取车时间、还车时间、保障包、押金规则。
- 合同模板：默认模板或运营/客服选择后的模板快照。
- 协议确认记录：协议版本、勾选时间、IP、客户端。
- 授权记录：实名、人脸、风控、代扣、签章、公证。

后台审核、合同、公证、支付、发货、核销、财务、客服都必须消费同一份订单快照，不能前台和后台各算一套价格。

## 8. UI 出图优先级

第一批 C 端出图建议：

1. 首页：底部导航含首页、安心用、体验租、订单、我的。
2. 安心用商品详情：费用、保障、须知、反诈提醒、套餐入口。
3. 安心用订单确认：协议、费用快照、授权入口。
4. 体验租首页：城市、品类、门店、周期、立即选车、扫码下单。
5. 体验租门店/车辆列表：门店卡 + 车型卡。
6. 体验租订单确认：短租方案、取还信息、骑行人、协议、合计金额。
7. 体验租订单详情：待确认、待取车、已取车、还车/续租状态。
8. 我的订单：安心用/体验租分业务线筛选。

第二批再补：

- 协议中心。
- 实名/人脸流程。
- 支付/免押授权。
- 地址/取还门店维护。
- 客服 IM 与工单。
