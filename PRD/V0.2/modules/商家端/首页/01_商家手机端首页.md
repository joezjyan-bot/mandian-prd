# 商家手机端首页

> 页面级 PRD。
> 目标：让商家老板、管理员、店长、员工在手机端快速进入办单、订单履约、交付待办、商品、钱包和员工管理，避免把运营后台能力直接下放到商家端。

> **V0.2 Peer Review 修订(2026-05-27)v1.0**:
> - 手机端首页以“办单助手”为第一入口。
> - 联营/平台订单只进入履约视图，不开放审核、改价、资金主控。
> - 钱包、提现、员工、审批配置按商家角色和老板授权展示。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家手机端首页 |
| 所属端 | 商家手机端 / 商家小程序 / H5 |
| 入口路径 | 商家端登录后默认首页 |
| 使用角色 | 商家老板、商家管理员、门店店长、普通员工 |
| 核心目标 | 聚合商家常用操作：办单、待办、订单、交付、商品、钱包、员工、门店码 |

商家手机端首页不是 PC 工作台的缩小版。它优先解决门店现场办单和履约处理，不承载复杂报表、平台配置、内部审核策略。

---

## 2. 信息架构

```text
商家手机端首页
├─ 顶部商家信息：商家名称、当前门店、角色、切换门店
├─ 经营摘要：今日办单、待审核、待交付、履约中
├─ 待办进度：开户/授权/商品/订单/交付/归还
├─ Banner：平台公告、规则提示、活动入口
├─ 九宫格入口
│  ├─ 办单助手
│  ├─ 商品管理
│  ├─ 订单管理
│  ├─ 交付待办
│  ├─ 我的钱包
│  ├─ 收支记录
│  ├─ 门店二维码
│  ├─ 客诉协助
│  └─ 员工管理
└─ 最近订单：最近 5 条订单履约状态
```

---

## 3. 顶部商家信息

| 字段 | 说明 |
|---|---|
| 商家名称 | 展示当前登录商家名称 |
| 当前门店 | 多门店时展示门店名称并允许切换 |
| 当前角色 | 老板 / 管理员 / 店长 / 员工 |
| 授权状态 | 收款账户、签章授权、合同授权、门店资料是否完整 |
| 消息入口 | 平台通知、客诉协助、订单待办 |

员工账号只显示自己被授权的门店，不显示商家总数据。

---

## 4. 经营摘要

| 指标 | 老板/管理员 | 店长 | 员工 |
|---|---|---|---|
| 今日办单 | 全商家或授权门店 | 本门店 | 本人 |
| 待审核 | 门店订单待商家自审 | 本门店 | 本人提交 |
| 待交付 | 需要门店交付/发货 | 本门店 | 本人待办 |
| 履约中 | 当前正常履约订单 | 本门店 | 本人相关 |
| 可提现 | 老板/授权管理员可见 | 不默认展示 | 不展示 |

摘要卡片点击后进入对应订单列表，自动带筛选条件。

---

## 5. 待办进度

| 待办 | 触发来源 | 点击后 |
|---|---|---|
| 门店订单待审核 | 客户扫码或办单助手生成商家订单 | 门店订单审核列表 |
| 待客户补资料 | 商家或平台要求客户补充资料 | 订单履约详情 |
| 待交付/发货 | 门店作为交付主体 | 交付待办列表 |
| 待客户签收 | 已提交交付但客户未确认 | 签收跟进列表 |
| 待归还验收 | 客户发起归还或到店归还 | 归还验收列表 |
| 商品待处理 | 商品草稿、待审核、驳回、复制待确认 | 商品管理 |
| 授权未完成 | 收款账户、签章、合同授权未完成 | 门店资料/授权状态 |
| 客诉待协助 | 平台客诉要求商家补材料 | 客诉协助 |

联营/平台订单待办只展示“履约动作”，不展示审核、资金来源、资方、合作模式。

## 6. 九宫格入口

### 6.1 默认入口顺序

| 顺序 | 入口 | 说明 |
|---|---|---|
| 1 | 办单助手 | 第一入口，生成门店订单、平台订单或联营订单对应办单码 |
| 2 | 商品管理 | 查看商品、编辑草稿、提交审核、查看驳回 |
| 3 | 订单管理 | 进入门店订单列表和联营/平台履约视图 |
| 4 | 交付待办 | 发货、门店自提、设备码、人车/人机合照、签收跟进 |
| 5 | 我的钱包 | 可提现、冻结、提现中、接口扣费，仅授权角色可见 |
| 6 | 收支记录 | 钱包流水、订单入账、接口扣费、提现记录 |
| 7 | 门店二维码 | 店铺码、员工办单码、渠道来源码 |
| 8 | 客诉协助 | 客诉材料、沟通记录、处理进度 |
| 9 | 员工管理 | 员工账号、角色、权限、禁用启用 |

### 6.2 角色可见性

| 入口 | 老板 | 管理员 | 店长 | 员工 |
|---|---|---|---|---|
| 办单助手 | 可见 | 可见 | 可见 | 可见 |
| 商品管理 | 可见 | 按授权 | 按授权 | 不默认 |
| 订单管理 | 可见 | 可见 | 本店 | 本人/授权 |
| 交付待办 | 可见 | 可见 | 本店 | 本人/授权 |
| 我的钱包 | 可见 | 按授权 | 不默认 | 不可见 |
| 收支记录 | 可见 | 按授权 | 不默认 | 不可见 |
| 门店二维码 | 可见 | 可见 | 本店 | 本人码 |
| 客诉协助 | 可见 | 可见 | 本店 | 本人/授权 |
| 员工管理 | 可见 | 按授权 | 本店员工 | 不可见 |

财务人员可以被老板授予“提现申请/查看流水”权限，但不能修改提现账户；提现账户修改仅老板或平台授权流程可操作。

## 7. 办单助手入口

办单助手是商家手机端第一入口，点击后进入订单类型选择：

| 办单类型 | 说明 |
|---|---|
| 门店订单 | 商家自营，商家自审，可使用商家自动审批配置 |
| 平台订单 | 平台主控审核，商家端仅生成办单来源和查看履约视图 |
| 分红订单 | 平台主控，商家端只展示履约和本商家收益结果 |

办单助手不在首页展示短租入口；短租“体验租”后续按独立模块接入。

## 8. 最近订单

| 字段 | 说明 |
|---|---|
| 订单号 | 脱敏展示，可复制 |
| 客户 | 姓名/手机号脱敏 |
| 商品 | 商品名、规格、订单周期 |
| 当前状态 | 待审核、待签约、待交付、履约中、待归还、已完成 |
| 下一步 | 当前角色需要执行的动作 |
| 操作 | 查看详情、联系客户、交付/归还待办 |

联营/平台订单点击进入履约视图，门店订单点击进入完整商家订单详情。

## 9. 数据互通

| 来源模块 | 首页展示 |
|---|---|
| 订单管理 | 经营摘要、待办进度、最近订单 |
| 办单助手 | 今日办单、待处理二维码 |
| 商品管理 | 商品待审核、驳回、草稿、复制待确认 |
| 交付模块 | 待发货、待签收、待归还验收 |
| 钱包模块 | 可提现、冻结、提现中、流水 |
| 员工权限 | 角色可见性、本人待办范围 |
| 客诉模块 | 客诉协助待办 |
| 授权模块 | 收款账户、合同签章、门店资料状态 |

首页不允许使用独立统计口径。所有数字必须来自对应业务模块，点击卡片进入列表后的数量应一致。

---

## 10. 审计与权限

1. 首页入口渲染由 RBAC 决定，前端隐藏不等于后端放行。
2. 钱包、提现、收款账户、员工权限属于敏感能力，必须后端二次校验。
3. 商家员工可被授权提现申请，但不能提现账户修改。
4. 联营/平台订单只允许履约视图和授权待办操作。
5. 首页待办点击、钱包访问、提现申请、员工权限变更都写操作日志。
