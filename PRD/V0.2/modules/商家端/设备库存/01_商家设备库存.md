# 商家设备库存

> 页面级 PRD 草案。
> 目标：商家 PC 端管理自己名下的真实设备，支撑短租、车辆类设备、门店发货、归还验收和员工扫码操作。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家设备库存 |
| 所属端 | 商家 PC 端 |
| 入口路径 | 商家端 > 设备库存 |
| 使用角色 | 商家老板、店长、仓库人员、售后；员工按权限只扫码处理 |
| 核心目标 | 商家维护自己的设备码、库存状态、仓库、出入库、归还验收和维修 |
| 运营联动 | 运营端库存设备管理、订单详情、门店手机端扫码出入库 |

---

## 2. 核心口径

1. 商家只能管理自己主体下的设备，不能看到其他商家的设备。
2. 商品用于展示和价格配置，设备用于真实出租和库存管理。
3. 短租订单必须绑定具体设备码。
4. 当前长租不走设备库存，只在发货/交付节点填写设备识别码，车辆类可配置为必填。
5. 员工扫码操作不能查看钱包、分账、成本等财务信息。
6. 设备状态必须与订单、交付材料、归还验收、监管锁联动。

## 4. 新增与入库

| 字段 | 类型 | 说明 |
|---|---|---|
| 商品 | 选择 | 必填 |
| 规格 | 选择 | 必填 |
| 设备码 | 生成/录入 | 唯一 |
| SN/IMEI/VIN | 文本 | 按类目 |
| 所属门店 | 选择 | 多门店必填 |
| 入库仓库 | 选择 | 必填 |
| 成色 | 下拉 | 与商品规格联动 |
| 设备照片 | 图片 | 可配置必填 |
| 配件清单 | 多选 | 电池、充电器、钥匙等 |
| 备注 | 文本 | 可选 |

规则：

1. 设备码不能重复。
2. 入库设备默认进入可租或待完善状态。
3. 已关联出租中订单的设备不允许删除。
4. 批量导入需要预览校验结果。

---

## 5. 调拨

| 场景 | 说明 |
|---|---|
| 商家仓到门店仓 | 门店补货 |
| 门店仓到商家仓 | 回收库存 |
| 门店之间调拨 | 多门店共享库存 |
| 维修仓调拨 | 设备维修 |

调拨必须记录调出方、调入方、操作人、交接照片和确认状态。调拨中设备不能被下单锁定。

---

## 6. 出库与交付

商家端出库服务于门店订单、分红订单、平台订单，但默认发货方仍按订单链路配置。

| 字段 | 说明 |
|---|---|
| 订单号 | 关联订单 |
| 设备码 | 扫码或选择 |
| 出库仓库 | 当前仓库 |
| 交付方式 | 当面交付、快递、配送 |
| 物流信息 | 快递时填写 |
| 设备照片 | 出库前照片 |
| 人机/人车合照 | 车辆类建议必填 |
| 配件清单 | 交付配件 |

规则：

1. 设备出库后状态变为出租中或待签收。
2. 交付照片同步订单详情和附件中心。
3. 设备码不一致时不允许完成出库。
4. 平台订单如运营配置为平台/资方发货，商家端只能查看进度或协助上传材料。

---

## 7. 归还验收

| 字段 | 说明 |
|---|---|
| 归还订单 | 自动匹配 |
| 设备码 | 扫码确认 |
| 外观照片 | 必填 |
| 配件检查 | 勾选 |
| 验收结果 | 正常、轻微损坏、严重损坏、缺件、设备不一致 |
| 赔付金额 | 异常时可填，进入订单售后 |
| 入库去向 | 可租、维修、争议、下架 |

验收异常时：

1. 订单进入争议或售后处理。
2. 设备不能直接回到可租。
3. 客户赔付、押金、退款和分账需要财务联动。

## 9. 数据联动

| 模块 | 联动 |
|---|---|
| 商品管理 | 设备必须关联商品和规格 |
| 办单助手 | 短租指定设备或生成设备二维码 |
| 订单详情 | 展示设备码、交付、归还、验收 |
| 门店手机端 | 扫码出库、归还验收、盘点 |
| 监管锁 | 锁控状态和设备绑定 |
| 租后管理 | 逾期未还、追回、损坏赔付 |
