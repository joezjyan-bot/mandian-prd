# 员工管理

> **来源**: P0-2

## 1. 页面定位

商家 PC · 门店资料 · 员工管理。商家老板 / 管理员 / 店长管理员工账号、角色、门店归属。

## 2. 角色四级

| 角色 | 权限大致 |
|---|---|
| 商家老板(merchant_owner) | 全部 + 财务 + 关键配置 |
| 商家管理员(merchant_admin) | 全部 - 财务关键操作 |
| 店长(store_lead) | 本店所有数据 + 本店员工管理 |
| 普通员工(staff) | 仅办单 + 查本店订单 |

详见 02_权限矩阵 / 05_账号模型与登录安全

## 3. 员工列表

### 3.1 字段
- 员工 ID
- 姓名 + 手机号(部分脱敏)
- 角色
- 所属门店
- 状态(active / paused / disabled)
- 入职日期
- 累计办单数
- 操作:详情 / 编辑 / 启用-禁用 / 调店

### 3.2 筛选
- 角色
- 所属门店
- 状态
- 姓名 / 手机搜索

## 4. 新增员工

### 4.1 字段
- 姓名
- 手机号(用作登录账号)
- 身份证号(实名,可选 V1)
- 角色(下拉)
- 所属门店(下拉,可空)
- 入职日期
- 备注

### 4.2 提交后
- 创建员工记录
- 自动发短信通知员工首次登录
- 员工首次登录设置密码

## 5. 编辑

可编辑:
- 角色
- 所属门店
- 备注

不可编辑:
- 员工 ID
- 手机号(改手机号需走特殊流程,涉及登录账号变更)
- 身份证号

## 6. 员工调店

调店操作:
- 商家老板 / 管理员可调店
- 不影响员工历史订单上的门店快照
- 仅影响后续新办单的归属

## 7. 停用 / 禁用

- 停用(active → paused):账号不可登录,但数据保留
- 禁用(active → disabled):账号不可登录 + 标记为已离职
- 启用(paused/disabled → active):恢复

操作需二次确认 + 留日志。

## 8. 权限

| 操作 | merchant_owner | merchant_admin | store_lead | staff |
|---|---|---|---|---|
| 新增员工 | ✅ | ✅ | ✅(仅本店) | ❌ |
| 编辑员工 | ✅ | ✅ | ✅(仅本店) | ❌ |
| 调店 | ✅ | ✅ | ❌ | ❌ |
| 启用 / 禁用 | ✅ | ✅ | ✅(本店员工) | ❌ |
| 查看员工 | ✅ | ✅ | ✅(本店) | ✅(本店,无敏感数据) |
| 角色调整 | ✅ | ❌ | ❌ | ❌ |

## 9. 操作日志

每次员工创建 / 编辑 / 调店 / 停用 / 启用 / 角色变更:留完整日志。
