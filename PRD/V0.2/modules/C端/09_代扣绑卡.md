# 代扣绑卡

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> **来源**: P1-5

## 1. 业务背景

订单生效后,后续期数自动代扣。客户需先绑定代扣卡(支付宝预授权 / 银行卡协议代扣)。

## 2. 触发节点

订单生效后,系统**后台发起**绑卡请求,推送 C 端通知客户去绑。

## 3. 绑卡流程

```
1. 客户在订单详情看"绑定代扣卡"按钮
2. 点击 → 跳转通道方代扣绑定页面
3. 客户填写:
   - 银行卡号(本人卡)
   - 持卡人姓名(必须与客户实名一致)
   - 身份证号(必须与实名一致)
   - 预留手机号(银行预留)
4. 通道方下发短信验证码
5. 客户输入 → 绑卡成功
6. 系统记录代扣协议号
```

## 4. 通道

- 通联代扣
- 支付宝预授权代扣
- 直付通分账代扣(V2)

## 5. 后续期数代扣

- 每月还款日,系统自动调用代扣
- 代扣成功 → 入账 → 分账
- 代扣失败:
  - 推送客户主动还款提醒
  - 第二日重试 1 次
  - 仍失败 → 计逾期 → 产生逾期费用
  - 多次失败 → 客服介入

## 6. 失败处理

| 场景 | 处理 |
|---|---|
| 绑卡失败(姓名不符) | 提示客户用本人卡 |
| 绑卡失败(银行限制) | 换银行卡 |
| 代扣失败(余额不足) | 客户主动充值并还款 |
| 代扣失败(账户冻结) | 客服介入 |

## 7. 解绑 / 换卡

客户可在订单详情解绑当前代扣卡 + 重新绑新卡

## 8. 数据模型

- `order.deduction_card_id`(代扣卡 ID,加密)
- `order.deduction_agreement_no`(代扣协议号)
- `deduction_record` 代扣流水
