# 运营端订单详情

> 页面级 PRD 草案。
> 参考：`01_订单列表.md`、`03_商品办单助手订单核心链路.md`、无界租订单列表高密度展示方式。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 运营端订单详情 |
| 所属端 | 运营端 |
| 入口路径 | 订单管理 > 门店订单 / 分红订单 / 平台订单 > 订单详情 |
| 使用角色 | 平台管理员、审核客服、订单客服、财务、风控、租后客服、运营主管 |
| 核心目标 | 承载订单完整资料、审核、合同、支付、发货、账单、分账、日志和回调状态 |

订单详情是订单数据的完整事实页。订单列表负责让客服快速判断和批量处理，订单详情负责展示更深层资料和执行高风险动作。

---

## 2. 核心口径

1. 详情页必须支持三类订单：`门店订单 / 分红订单 / 平台订单`。
2. 门店订单由商家或门店自审，运营端默认只做查看、监管、抽佣、客诉、租后和异常介入。
3. 分红订单和平台订单由运营端主控审核、资方分配、合同、公证、支付、发货链路。
4. 详情页所有人工操作、系统回调、接口调用、状态变更都必须进入操作日志或回调日志。
5. 订单详情展示的是下单时价格快照，商品、费率、增值服务后续改动不能反向污染历史订单。
6. 敏感字段按权限脱敏展示；查看明文必须记录查看人、时间、原因和来源 IP。
7. 客服改套餐、改价、发起补充合同、关闭订单、退款、变更资方、变更发货主体均属于高风险动作，需要权限和二次确认。

## 3. 页面结构

```
┌──────────────────────────────────────────────────────────────────────┐
│ 顶部：订单号 / 订单类型 / 当前状态 / 下一步待办 / 风险标签 / 操作按钮 │
├──────────────────────────────────────────────────────────────────────┤
│ 概览卡：商家门店 / 渠道来源 / 客户 / 商品 / 金额 / 资方 / 发货主体    │
├──────────────────────────────────────────────────────────────────────┤
│ 标签页：基础信息 商品价格 客户资料 风控审核 资方分配 合同授权          │
│       支付账单 发货签收 分账钱包 租后客诉 日志回调                    │
├──────────────────────────────────────────────────────────────────────┤
│ 当前标签页详情内容                                                    │
└──────────────────────────────────────────────────────────────────────┘
```

页面支持两种形态：

| 形态 | 使用场景 | 要求 |
|---|---|---|
| 独立详情页 | 深度审核、财务核对、租后处理 | 支持完整标签页、附件预览、日志展开 |
| 右侧抽屉 | 列表快速查看 | 展示概览、关键资料、最近日志和跳转详情 |

## 4. 顶部订单头

| 字段 | 说明 |
|---|---|
| 订单号 | 系统订单编号，支持复制 |
| 订单类型 | 门店订单 / 分红订单 / 平台订单 |
| 订单状态 | 待付款、待审核、待分配资方、待签约、待支付、待发货、待归还、已逾期、已完成、已关闭等 |
| 下一步待办 | 根据 `next_action` 计算，例如待客户授权签署、待发起合同、待客户办理公证 |
| 商家/门店 | 商家名称、门店名称、负责人、办单员工 |
| 渠道来源 | 渠道、推广码、渠道人员；门店自营订单可不计渠道佣金 |
| 资方 | 分红订单、平台订单展示资方名称和分配状态 |
| 发货主体 | 默认门店，可配置为商家、平台、资方、快递接口 |
| 风险标签 | 疑似黑名单、资料缺失、定位异常、历史逾期、回调异常等 |

顶部操作按钮按权限和订单类型动态展示。

## 5. 标签页与字段

### 5.1 基础信息

| 区域 | 字段 |
|---|---|
| 订单来源 | 下单渠道、扫码来源、办单助手入口、订单创建人、创建时间 |
| 主体信息 | 商家、门店、门店老板、办单员工、渠道来源、推广码 |
| 客户摘要 | 客户姓名脱敏、手机号脱敏、实名状态、年龄、性别、定位状态 |
| 状态信息 | 当前订单状态、下一步待办、审核主体、发货主体、租后状态 |
| 备注信息 | 商户备注、平台备注、客户备注、最近一条系统回调备注 |

### 5.2 商品与价格方案

| 字段 | 说明 |
|---|---|
| 商品信息 | 商品名称、商品图、类目、品牌、型号、规格、全新/二手、数量 |
| 租赁模式 | 长租 / 短租 |
| 计费单位 | 小时 / 天 / 周 / 月 |
| 租期 | 租期数量、起租时间、到期时间、总期数、账单周期 |
| 费用快照 | 首期实付、后期应付、押金、补充押金、服务费、公证费、设备管理费、运费 |
| 增值服务 | 服务名称、金额、是否可退、是否参与分账 |
| 留购信息 | 是否可留购、留购总价、到期留购价、修改记录 |
| 价格方案来源 | 办单助手生成、客服改价、商品套餐、手工录入 |

价格方案必须展示下单时的完整快照，不只展示商品当前配置。客服改套餐后，原价格方案和新价格方案都要可追溯。

### 5.3 客户资料与审核资料

| 区域 | 字段 |
|---|---|
| 实名资料 | 姓名、身份证号脱敏、实名状态、认证手机号 |
| 身份证附件 | 正反面、识别结果、识别失败原因 |
| 联系方式 | 手机号、紧急联系人、收货地址 |
| 定位信息 | 下单定位、授权定位、收货地址定位差异 |
| 银行卡 | 绑卡数量、代扣卡状态、银行卡脱敏 |
| 补充资料 | 客服要求补充的图片、视频、证明文件 |

查看身份证、银行卡、完整手机号等明文字段需要二次确认并进入敏感查看日志。

### 5.4 风控与黑名单

| 字段 | 说明 |
|---|---|
| 风控授权状态 | 未发起、待客户授权、报告生成中、已完成、失败 |
| 风控报告 | 外部报告摘要、分数、建议、报告编号 |
| 黑名单匹配 | 是否命中、命中来源、匹配字段脱敏、处理建议 |
| 历史订单 | 历史订单数量、逾期次数、关闭次数、买断记录 |
| 设备风险 | 监管锁接口状态、设备识别码历史、短租库存异常 |
| 审核建议 | 系统建议、人工结论、主管复核意见 |

命中疑似黑名单时，详情页必须弹出高风险提示，并要求审核人员填写处理意见。

### 5.5 审核与资方分配

| 字段 | 门店订单 | 分红订单 | 平台订单 |
|---|---|---|---|
| 审核主体 | 门店/商家 | 运营平台 | 运营平台 |
| 资方分配 | 不需要 | 需要 | 需要 |
| 改套餐权限 | 商家主控，平台异常介入 | 平台主控 | 平台主控 |
| 审核动作 | 查看、异常介入 | 通过、拒绝、补资料、分配资方 | 通过、拒绝、补资料、分配资方 |

资方分配信息包括资方、出资金额、出资比例、分账规则版本、分账账户、分配人、分配时间和变更记录。

### 5.6 合同、授权与公证

| 链路 | 字段 |
|---|---|
| e 签宝合同 | 合同模板、合同编号、发起时间、签署状态、签署完成时间 |
| 补充合同 | 发起原因、补充合同编号、客户签署状态 |
| 征信/风控授权 | 授权协议、发起时间、客户操作状态、回调结果 |
| 公证 | 公证服务商、公证费用、发起时间、客户办理状态、完成时间 |
| 平台签章授权 | 商家入驻时的授权状态；订单页只展示是否可用 |

发起到客户侧的合同、授权、公证动作，都要同步到客户小程序订单待办中。

### 5.7 支付、账单与代扣

| 区域 | 字段 |
|---|---|
| 支付概览 | 已付金额、待付金额、总租金、押金、首期实付 |
| 账单明细 | 期数、应付时间、应付金额、已付金额、逾期金额、状态 |
| 部分支付 | 收款码通道、支付金额、子账单、回调结果、核销状态 |
| 代扣签约 | 通道、签约状态、协议号、下一次扣款时间 |
| 退款 | 退款申请、退款金额、退款原因、退款状态 |

线下收款或收款码支付不能只改订单状态，必须生成财务流水和回调/核销记录。

### 5.8 发货、交付与签收

| 场景 | 字段 |
|---|---|
| 门店当面交付 | 设备识别码、人机合照、人车合照、门头/店内照片、交付时间、上传人 |
| 快递发货 | 快递公司、运单号、发货接口返回、签收回调 |
| 平台/资方发货 | 发货主体、物流/交接方式、设备识别码、出库时间 |
| 客户签收 | 签收照片、签收时间、签收异常说明 |
| 归还验收 | 归还照片、设备状态、验收结论、争议记录 |

交付照片证据链详见 `02_交付签收照片证据.md`，详情页只作为汇总展示和入口。

### 5.9 分账、钱包与佣金

| 字段 | 说明 |
|---|---|
| 平台抽佣 | 默认 2%，可按商家/订单配置调整 |
| 门店分账 | 门店订单收益、分红订单门店占比收益 |
| 资方分账 | 分红订单资方占比收益、平台订单资方收益 |
| 渠道佣金 | 分红订单、平台订单可计入渠道佣金；门店订单默认不计 |
| 钱包入账 | 待结算、可提现、冻结、提现中、已打款 |
| 对账状态 | 未对账、已对账、异常、冲正 |

分红订单抽佣按后者口径执行：先按比例分给门店和资方，再分别从各自份额扣平台抽佣。

### 5.10 租后、客诉与归还

| 区域 | 字段 |
|---|---|
| 租后状态 | 未进入、已逾期、跟进中、外部催收中、已结清 |
| 客诉 | 支付宝投诉编号、投诉状态、处理人、处理记录 |
| 归还 | 归还申请、归还物流、验收结果、赔付或扣费 |
| 买断/留购 | 申请时间、留购价、支付状态、完成状态 |

租后模块前期可以轻量，但订单详情必须预留入口和状态字段。

### 5.11 操作日志、回调日志与交易快照

| 日志 | 内容 |
|---|---|
| 操作日志 | 人工操作、操作前后值、原因、操作人、时间、IP |
| 系统日志 | 定时任务、状态流转、自动分账、自动关闭 |
| 回调日志 | 支付、合同、公证、风控、物流、分账等第三方回调 |
| 交易快照 | 下单时商品、价格、服务、合同、客户确认信息 |
| 敏感查看日志 | 查看明文字段、下载附件、导出资料 |

日志默认按时间倒序展示，支持按操作类型、操作人、模块、成功/失败筛选。

## 6. 操作按钮

| 按钮 | 显示条件 | 结果 |
|---|---|---|
| 审核通过 | 分红/平台订单待审核，或门店订单异常介入 | 进入下一步待办 |
| 审核不通过 | 待审核 | 填写原因，通知商家/客户，按规则关闭或退回 |
| 要求补资料 | 资料不完整 | 生成客户或商家待办 |
| 分配资方 | 分红/平台订单待分配资方 | 选择资方并生成分账规则草案 |
| 更换资方 | 合同/支付前且有权限 | 记录变更原因，更新资方和分账规则 |
| 修改套餐/改价 | 有改价权限 | 生成新价格方案，保留旧方案 |
| 发起授权 | 风控/征信未授权 | 调用授权链路并同步客户待办 |
| 发起合同 | 待签合同 | 调用 e 签宝并同步客户待办 |
| 发起公证 | 配置需要公证 | 同步客户小程序办理入口 |
| 生成收款码 | 部分支付或补缴 | 调用通联/信联收款码 |
| 发货/确认交付 | 满足发货条件 | 进入发货或照片上传流程 |
| 关闭订单 | 有关闭权限 | 二次确认并填写原因 |
| 加入黑名单 | 有风控权限 | 写入黑名单库并记录原因 |

## 7. 状态与下一步待办

详情页必须根据状态机展示明确下一步，不能只展示一个大状态。

| next_action | 触发来源 | 下一步 |
|---|---|---|
| 待客户付款 | 客户未完成首期/费用支付 | 客户支付或客服生成收款码 |
| 待审核资料 | 资料已提交但未审核 | 审核客服审核 |
| 待发起授权 | 需风控或征信授权 | 客服发起授权 |
| 待客户授权签署 | 授权已发起 | 客户在小程序完成 |
| 待查看风控报告 | 报告已生成 | 审核客服查看 |
| 待分配资方 | 分红/平台订单未分配资方 | 审核客服分配 |
| 待发起合同 | 审核通过但合同未发起 | 客服发起合同 |
| 待客户签合同 | 合同已发起 | 客户签署 |
| 待发起公证 | 配置要求公证 | 客服发起公证 |
| 待客户办理公证 | 公证已发起 | 客户办理 |
| 待绑定银行卡 | 代扣未签约 | 客户绑卡 |
| 待发货 | 审核、合同、支付满足发货条件 | 门店/平台/资方发货 |
| 待客户确认收货 | 已发货未签收 | 客户确认 |
| 待还款 | 租期中 | 客户按账单还款 |

## 8. 异常规则

| 异常 | 处理 |
|---|---|
| 第三方回调失败 | 进入回调异常队列，可手动重试，不能直接改最终状态 |
| 回调状态和人工状态冲突 | 展示冲突提示，主管复核后处理 |
| 支付金额不匹配 | 财务异常标记，禁止自动分账 |
| 合同签署失败 | 可重新发起或改为补充合同，记录失败原因 |
| 资方缺失 | 分红/平台订单不能审核通过到发货 |
| 疑似黑名单 | 审核页和详情页高亮提示，必须填写处理意见 |
| 客服改价后客户已扫码 | 需重新生成价格确认或补充确认，旧二维码失效 |
| 部分支付 | 必须保留原账单、子账单、收款码、核销记录 |

## 9. 与订单列表边界

| 能力 | 订单列表 | 订单详情 |
|---|---|---|
| 快速筛选 | 主承载 | 辅助 |

## 10. 待确认

1. 运营端是否允许对门店订单做强制关闭、强制退款、强制改价；如果允许，需要单独权限和主管审批。
2. 资方变更的截止节点建议定为合同签署前；合同签署后变更需要补充合同或冲正流程。
3. 公证是否按订单类型强制配置，还是按商品/商家/资方链路配置决定。
