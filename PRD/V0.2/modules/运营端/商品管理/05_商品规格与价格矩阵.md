> **V0.2 Peer Review 修订(2026-05-27)v1.0**:
> - 本文落实同行小程序能力吸收任务 6。

## 2. 核心口径

1. 商品是标准展示对象,规格组合才是可定价 SKU。
2. 规格组和规格值由运营或商家手动维护,系统不写死。
3. 一个商品可以有多个规格组,例如“设备成色”“颜色”“容量”“是否含电池”。
4. 系统根据规格组和值自动生成 SKU 笛卡尔积,商家可关闭不经营的 SKU。
5. 每个 SKU 必须填写设备价;设备价读取指导价区间进行校验。
6. 长租价格可使用简单模式自动生成,也可进入高级模式逐期填写。
7. 体验租适配商品必须绑定统一标准商品名,商品标准名不可随意修改。

## 3. 添加商品三步法

| 步骤 | 内容 |
|---|---|
| 1/3 基础信息 | 商品封面、轮播图、品牌、型号、商品标准名、展示标题、类目、商品介绍 |
| 2/3 规格配置 | 自定义规格组、规格值、生成 SKU、填写设备价、读取指导价区间 |
| 3/3 租赁方案 | 选择安心用长租可用期数、首期、每期应付、保证金、留购规则、增值服务、是否同步办单助手 |

体验租不混入安心用长租办单助手;体验租价格、车辆库存和门店可租时段在短租业务模块维护。

## 4. 规格配置

示例:

```text
商品:九号 Qz1 MIX
├─ 规格组 1:设备成色
│  ├─ 全新
│  └─ 二手
├─ 规格组 2:颜色
│  ├─ 白色
│  └─ 粉色
└─ 规格组 3:电池
   ├─ 无电池
   └─ 含电池
```

SKU 自动生成:

| SKU | 规格组合 | 是否启用 | 是否同步办单助手 |
|---|---|---|---|
| SKU1 | 全新 + 白色 + 无电池 | 是 | 是 |
| SKU2 | 全新 + 白色 + 含电池 | 是 | 是 |
| SKU3 | 二手 + 粉色 + 无电池 | 否 | 否 |

## 5. 价格矩阵

### 5.1 简单模式

商家只填写:
- 设备价
- 首期应付
- 可用期数
- 保证金
- 留购价/剩余应付规则

系统按办单助手计算逻辑生成每期应付。

### 5.2 高级模式

商家或运营点击“自定义逐期价”后,按期填写:

| 期数 | 每期应付 | 到期日规则 | 是否可编辑 |
|---|---|---|---|
| 第 1 期 | 首期应付 | 下单支付 | 是 |
| 第 2 期 | 每期应付 | 下单后第 30 天 | 是 |
| 第 N 期 | 每期应付 | 按订单周期生成 | 是 |

规则:
1. 逐期金额之和必须与订单总额校验一致。
2. 修改设备价后,系统提示是否重新生成逐期金额。
3. 已生成订单的价格快照不受后续 SKU 改价影响。

---

## 6. 指导价联动

每个 SKU 读取 `04_商品指导价管理.md` 的设备指导价区间:

| 场景 | 处理 |
|---|---|
| 设备价低于指导价下限 | 允许提交,记录日志 |
| 商家订单设备价高于指导价上限 | 弹窗警告 + 二次确认后允许 |
| 联营订单 / 平台订单设备价高于指导价上限 | 拦截,不可提交 |

## 7. SKU 显示与同步

| 开关 | 说明 |
|---|---|
| 是否启用 | 停用后该 SKU 不可下单 |
| 是否 C 端展示 | 商品详情中是否展示该规格 |
| 是否同步商家订单办单助手 | 商家订单是否可选 |
| 是否同步联营订单办单助手 | 联营订单是否可选 |
| 是否同步平台订单办单助手 | 平台订单是否可选 |

同步前校验:
1. SKU 启用。
2. 设备价完整。
3. 指导价校验通过。
4. 租期、首期、每期应付、保证金、留购规则完整。
5. 增值服务配置没有冲突。

## 9. 权限与日志

| 操作 | 权限 | 日志 |
|---|---|---|
| 新增规格组 | 商品运营 / 商家授权管理员 | 记录规格组和值 |
| 删除未使用规格值 | 商品运营 / 商家授权管理员 | 记录删除前内容 |
| 修改设备价 | 有价格权限角色 | 记录指导价校验结果 |
| 自定义逐期价 | 老板/管理员/运营主管 | 记录每期前后值 |
| 同步办单助手 | 商品运营/商家老板/授权管理员 | 记录同步入口和 SKU |

已被历史订单引用的规格组合不可物理删除,只能停用。
