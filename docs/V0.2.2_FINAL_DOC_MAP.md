## 四、读错文档的处理

1. 任何 `archive/` 文档不得覆盖 V0.2.2 当前主线。
2. 任一文档若仍出现旧词,必须先判断是否为“旧词到新词”的解释性出现;非解释性出现按 V0.2.2 改为商家订单、联营订单、平台订单等新口径。
3. 任一 C 端文档若要求展示内部资金来源、合作模式、风控结论、商家结算价、平台利润、返点规则,按错误口径处理。
4. 任一财务文档若把订单业务账、支付流水账、钱包账、总账分录混作一张账,按错误口径处理。
