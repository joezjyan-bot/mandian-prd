# 商家订单列表与详情

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。
> 目标：让商家 PC 端承接商家订单自审，并以履约视图查看联营订单、平台订单中与本商家相关的交付、签收、归还和售后协助事项，不越权操作平台主控流程。

> **⚠️ V0.2 Peer Review 修订(2026-05-27)v1.1**:
> - 商家订单合同签署前允许商家修改关键字段。
> - 联营订单 / 平台订单不开放审核、改价和资金风控主控操作。
> - 联营订单 / 平台订单在商家端仅开放履约视图；不展示内部资金来源、合作模式、资金来源和平台内部审核规则。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家订单列表与详情 |
| 所属端 | 商家 PC 端 |
| 入口路径 | 商家中心 > 订单管理 |
| 使用角色 | 商家老板、商家管理员、门店店长、授权员工 |
| 核心目标 | 查看本商家订单，处理门店自营订单，在履约视图中跟进联营/平台订单交付、签收、归还和售后协助 |

---

## 2. 订单范围

| 订单类型 | 商家端定位 | 操作边界 |
|---|---|---|
| 商家订单 | 商家自营订单 | 商家自审、改价、发货、售后、钱包收益 |
| 联营订单 | 商家参与履约或收益分配 | 仅履约视图：查看与本店相关的交付、签收、归还、售后协助和收益入账结果 |
| 平台订单 | 平台主控、商家协助履约 | 仅履约视图：查看与本店相关的待办、联系客服、补资料、交付/归还协助 |

---

## 3. 列表结构

```
订单管理
├─ 商家订单
├─ 联营订单
├─ 平台订单
└─ 全部订单
```

商家端默认按三类订单拆分，不建议一个列表混在一起。

## 5. 列表字段

| 区域 | 字段 |
|---|---|
| 订单摘要 | 订单号、订单类型、下单时间、门店、办单员工 |
| 客户 | 姓名脱敏、手机号脱敏、地址摘要、资料状态 |
| 商品 | 商品、规格、租赁模式、租期、设备码 |
| 费用 | 首期、后期账单、保证金、增值服务、留购价 |
| 进度 | 当前状态、下一步待办、履约节点 |
| 财务 | 商家订单收益；联营/平台订单仅展示本商家已确认收益或结算状态，不展示内部资金来源 |
| 操作 | 查看详情、审核、发货、补资料、联系客服、售后 |

员工账号只展示本人或授权范围订单，不展示钱包和分账汇总。

## 6. 商家订单详情

商家可主控商家订单：

| 能力 | 说明 |
|---|---|
| 审核资料 | 商家自行判断是否通过 |
| 改价/改套餐 | 按商家权限生成新价格方案 |
| 发起合同 | 可用商家模板或平台提供模板 |
| 发货/交付 | 默认门店发货，上传交付材料 |
| 售后/退款 | 商家处理，平台保留监管和财务总账 |
| 钱包入账 | 展示平台抽佣、可提现、冻结、流水 |

商家订单如果发生客诉、违规、黑名单命中或财务异常，运营端可介入。

### 6.1 合同签署前字段修改

商家订单满足以下条件时,商家端显示“修改订单”入口:

| 条件 | 规则 |
|---|---|
| 订单类型 | 商家订单 |
| 状态 | 待审核 / 待付款 |
| 合同 | 客户合同未签署 |
| 角色 | 商家老板或被授权管理员 |

可修改:商品/SKU、期数、首期应付、保证金、增值服务、设备价、订单总额。修改后系统重新生成订单确认快照,客户需重新确认后才能继续支付或签约。

## 7. 联营订单履约视图

| 区域 | 展示 |
|---|---|
| 订单进度 | 待补资料、待签约、待支付、待交付、待签收、履约中、待归还、已完成 |
| 合同/授权 | 客户是否完成必要签约、授权、支付节点；不展示合同模板策略和内部规则 |
| 交付待办 | 是否需要门店发货、门店自提、上传人车/人机合照、设备码 |
| 签收/归还 | 客户签收状态、归还申请、归还验收待办 |
| 售后协助 | 客诉、退款、维修、补材料等需要商家协助的事项 |
| 财务结果 | 商家可见的已入账、待结算、可提现结果 |

商家不能自行审核、改价、切换合作模式、查看资金来源、查看内部资金来源信息或绕过平台主控流程。C 端同样不暴露“内部资金来源、合作模式、资金来源、风控”等字眼。

---

## 8. 平台订单履约视图

| 区域 | 展示 |
|---|---|
| 客服 IM | 联系平台客服，自动携带订单摘要 |
| 履约进度 | 待补资料、待签约、待支付、待交付、待签收、履约中、待归还、已完成 |
| 合同/支付 | 客户签署、支付、绑卡是否完成；不展示资金通道和内部审核规则 |
| 发货主体 | 门店、平台或快递；仅展示本商家需要执行的部分 |
| 协作待办 | 补资料、上传交付材料、确认发货、归还验收、售后协助 |
| 协作收益 | 如配置门店服务费或渠道佣金，仅展示结算进度和入账结果 |

平台订单的审核、改价、合同策略、资金主控、内部配置均在运营端。商家端只显示与本商家履约动作相关的信息。

## 9. 商家端操作

| 操作 | 商家订单 | 联营订单 | 平台订单 |
|---|---|---|---|
| 查看详情 | 是 | 履约视图 | 履约视图 |
| 审核通过/拒绝 | 是 | 否 | 否 |
| 改价/改套餐 | 是 | 否 | 否 |
| 补资料 | 是 | 协助 | 协助 |
| 发货/交付 | 按配置 | 按配置 | 按配置 |
| 联系客服 | 可选 | 是 | 是 |
| 售后/退款 | 是 | 申请/协助 | 申请/协助 |
| 查看钱包收益 | 老板/管理员 | 仅收益入账结果 | 仅协作收益结果 |

---

## 10. 权限与日志

1. 老板可看全部商家订单和钱包。
2. 管理员权限由老板配置。
3. 店员只看本人或授权商家订单。
4. 员工不能查看钱包、提现、收款账户。
5. 审核、改价、发货、售后、补资料、联系客服都记录日志。
6. 联营/平台订单的履约视图访问、交付操作、归还验收、售后协助均记录日志；审核、改价、资金相关入口不渲染。
