# 商家端员工账号

> 页面级 PRD 草案。
> 目标：解决门店老板不在店里时，店员可用账号办单、处理待办，但不能查看钱包和敏感经营数据。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家端员工账号 |
| 所属端 | 商家 PC 端、门店手机端 |
| 入口路径 | 商家 PC 端 > 员工管理；门店手机端 > 我的 > 员工账号 |
| 使用角色 | 商家老板、商家管理员、门店店长、店员 |
| 核心目标 | 由商家或门店老板创建员工账号，分配办单和待办权限，限制钱包、财务、配置、敏感资料访问 |

---

## 2. 核心口径

1. 员工账号由商家老板或被授权管理员创建，不由运营平台日常代建。
2. 员工账号使用手机号 + 密码登录，可后续扩展短信验证码。
3. 员工账号只能在被授权门店、被授权办单入口内工作。
4. 店员默认只能使用办单助手、生成二维码、查看自己相关待办、上传交付材料。
5. 店员默认不能查看钱包、提现、分账、渠道佣金、收款账户、门店完整经营数据。
6. 员工账号登录态建议短于老板账号，例如 7 天；老板/商家账号可按门店端 30 天免登录策略配置。
7. 员工所有办单、改价、上传照片、查看订单动作都要写入操作日志。

---

## 3. 角色模型

| 角色 | 定位 | 默认权限 |
|---|---|---|
| 商家老板 | 入驻主体主账号 | 全部商家端权限、钱包、提现、员工管理 |
| 商家管理员 | 老板授权的管理账号 | 商品、订单、员工、配置，是否可看钱包由老板配置 |
| 门店店长 | 单门店管理 | 本门店订单、办单、员工、交付材料 |
| 店员 | 现场办单人员 | 办单助手、扫码下单、待办、交付材料 |
| 只读员工 | 运营查看 | 只看本门店订单进度，不可操作 |

V1 最少要支持：老板、管理员、店员三类。

## 5. 新增/编辑员工

| 字段 | 类型 | 说明 |
|---|---|---|
| 员工姓名 | 文本 | 必填 |
| 手机号 | 文本 | 必填，作为登录账号 |
| 初始密码 | 密码 | 新增时必填或系统生成 |
| 所属门店 | 多选 | 限制员工可操作门店 |
| 角色 | 下拉 | 管理员、店长、店员、只读 |
| 办单助手入口 | 复选 | 门店订单、分红订单、平台订单 |
| 可调整价格 | 开关 | 默认关闭，开启需老板确认 |
| 可上传交付材料 | 开关 | 默认开启 |
| 可查看订单 | 单选 | 仅本人订单 / 本门店订单 / 授权门店订单 |
| 可查看客户资料 | 单选 | 脱敏摘要 / 部分明文 / 不可查看 |
| 状态 | 开关 | 启用/禁用 |

手机号唯一性按平台账号体系校验。同一手机号如果已是其他商家的员工，需要按平台账号策略处理，不能静默覆盖。

## 7. 登录与安全

| 能力 | 规则 |
|---|---|
| 登录方式 | 手机号 + 密码，后续可加验证码 |
| 免登录时长 | 员工账号建议 7 天，老板/主账号可配置 30 天 |
| 设备限制 | 可记录登录设备，异常设备提醒老板 |
| 密码重置 | 老板/管理员可重置，员工首次登录需修改 |
| 禁用账号 | 禁用后立即失效，踢出登录态 |
| 锁定账号 | 连续密码错误后锁定，老板或管理员解锁 |

员工账号不能复用老板的登录态，也不能通过浏览器缓存绕过权限。

## 8. 门店手机端员工体验

员工登录后首页只展示和自己权限相关的入口：

```
门店手机端
├─ 办单助手
│  ├─ 门店订单
│  ├─ 分红订单
│  └─ 平台订单
├─ 待办任务
│  ├─ 待上传交付材料
│  ├─ 待客户补资料
│  ├─ 待发货/签收
│  └─ 待归还验收
└─ 我的
   ├─ 账号信息
   └─ 退出登录
```

员工首页不展示：

1. 钱包余额。
2. 提现。
3. 分账明细。
4. 商家收款账户。
5. 渠道佣金。
6. 平台配置。
7. 全量经营数据看板。

## 9. 办单归属

| 字段 | 说明 |
|---|---|
| operator_id | 实际办单员工 |
| operator_role | 办单时角色 |
| merchant_id | 所属商家 |
| store_id | 所属门店 |
| assistant_entry | 门店订单 / 分红订单 / 平台订单 |
| price_permission_snapshot | 办单时是否有改价权限 |
| device_id | 如涉及短租设备，记录设备码 |

订单详情、订单列表、财务流水、交付照片都要保留实际办单员工，不能只记录商家主账号。

## 10. 操作日志

| 操作 | 日志内容 |
|---|---|
| 新增员工 | 创建人、员工手机号脱敏、角色、门店、权限 |
| 编辑权限 | 变更前后权限、操作人、原因 |

| 禁用/启用 | 原因、操作人、时间 |
| 员工登录 | 登录时间、设备、IP、结果 |
| 员工办单 | 订单号、办单入口、价格方案、二维码 |
| 上传照片 | 订单号、照片类型、上传人、时间 |
| 查看敏感资料 | 查看人、字段、原因、时间 |

## 11. 异常规则

| 异常 | 处理 |
|---|---|
| 员工离职 | 老板禁用账号，保留历史订单归属 |
| 员工误操作 | 按日志追责，不删除历史操作 |
| 员工无权限访问钱包 | 前端隐藏入口，后端接口也必须拒绝 |
| 员工跨门店访问 | 按授权门店校验 |
| 员工生成二维码后被禁用 | 已生成订单保留原办单人，未下单二维码可按配置失效 |
| 手机号被多个主体使用 | 走账号绑定/邀请确认，不允许静默加入 |

| 模块 | 联动内容 |

| 办单助手 | 三入口权限、改价权限、价格方案归属 |
| 订单管理 | 办单员工筛选、订单归属、待办任务 |
| 交付材料 | 员工上传人机/人车合照、签收照片 |
| 财务钱包 | 员工不可见钱包和提现 |
| 权限管理 | 角色、菜单、按钮、数据权限 |
| 操作日志 | 员工所有动作可追溯 |

## 13. 待确认

1. 员工账号是否允许一个手机号绑定多个商家；建议允许，但登录后选择主体。
2. 店员是否允许审核门店订单；建议默认不允许，由老板给店长或管理员开启。
3. 员工改价是否需要单笔上限；建议按角色配置最大改价幅度。
