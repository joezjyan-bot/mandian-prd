# 运营端店铺审核

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 页面级 PRD 草案。  
> 目标：运营端集中处理商家/门店入驻申请，审核营业执照、法人身份证、门店真实性、收款账户、e 签宝授权和渠道来源，审核通过后开通商家 PC 端与门店手机端。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 店铺审核 |
| 所属端 | 运营端 |
| 入口路径 | 商家管理 > 店铺审核 |
| 使用角色 | 平台管理员、店铺审核人员、财务审核人员、运营负责人 |
| 核心目标 | 审核入驻资料，决定通过、驳回、暂停或要求补充资料 |
| 审核结果 | 通过后生成商家主体、门店、商家老板账号、钱包账户和默认链路配置 |

## 3. 筛选与页签

### 3.1 筛选字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 店铺名称 | 文本 | 模糊搜索 |
| 主体名称 | 文本 | 营业执照主体名称 |
| 联系手机号 | 文本 | 商家主账号手机号 |
| 统一社会信用代码 | 文本 | 精确搜索，输入后脱敏显示结果 |
| 法人姓名 | 文本 | 审核人员搜索 |
| 渠道来源 | 下拉/搜索 | 绑定渠道推广码时展示 |
| 邀请码 | 文本 | 邀请码搜索 |
| 审核状态 | 下拉 | 待提交、入驻中、待审核、待补资料、通过、驳回、暂停、注销/终止 |
| OCR 状态 | 下拉 | 成功、失败、人工修改 |
| e 签宝授权状态 | 下拉 | 未发起、授权中、已完成、失败 |
| 收款账户状态 | 下拉 | 待审核、实名一致、需授权、授权已提交 |
| 提交时间 | 日期范围 | 入驻申请提交时间 |

### 3.2 状态页签

| 页签 | 说明 |
|---|---|
| 全部 | 所有入驻申请 |
| 待提交 | 商家已注册但资料未提交完整 |
| 入驻中 | 资料填写、OCR、收款账户或 e 签宝授权进行中 |
| 待审核 | 用户已提交，等待运营审核 |
| 待补资料 | 已退回，等待商家修改 |
| 已通过 | 已开通商家主体 |
| 已驳回 | 审核拒绝，保留记录 |
| 已暂停 | 后续运营暂停合作或冻结 |

说明：以上状态是当前建议口径，最终状态名称和是否需要 `已注销/终止` 页签需要业务确认。

## 5. 审核详情

详情页建议按页签组织，避免一个超长表单。

### 5.1 基础资料

| 字段 | 说明 |
|---|---|
| 店铺名称 | 用户填写的门店名称 |
| 主体类型 | 企业/个体户 |
| 经营类目 | 手机、电动车等 |
| 联系人 | 联系人姓名 |
| 联系手机号 | 脱敏展示，授权角色可查看完整 |
| 渠道来源 | 渠道名称、推广码、邀请人 |
| 提交时间 | 最近提交时间 |

### 5.2 营业执照

| 字段 | 说明 |
|---|---|
| 营业执照照片 | 支持放大查看 |
| 统一社会信用代码 | 脱敏或按权限查看 |
| 主体名称 | OCR 识别值和用户确认值 |
| 法定代表人 | OCR 识别值和用户确认值 |
| 注册地址 | OCR 识别值和用户确认值 |
| 营业期限 | 判断是否过期 |
| OCR 状态 | 成功、失败、人工修改 |

### 5.3 法人身份证

| 字段 | 说明 |
|---|---|
| 身份证人像面 | 支持放大查看 |
| 身份证国徽面 | 支持放大查看 |
| 法人姓名 | OCR 识别值 |
| 身份证号 | 默认脱敏，授权后可查看 |
| 有效期 | 过期高亮 |
| 与营业执照法人是否一致 | 是/否 |

### 5.4 门店实景和地址

| 字段 | 说明 |
|---|---|
| 门头照片 | 支持放大查看 |
| 店内照片 | 如有则展示 |
| 文本地址 | 用户填写地址 |
| 地图定位 | 展示经纬度和地图点 |
| 地址一致性提示 | 文本地址和执照地址、地图定位差异提示 |

### 5.5 收款账户

| 字段 | 说明 |
|---|---|
| 收款方式 | 支付宝账号/银行卡 |
| 账户名 | 收款实名 |
| 账号 | 脱敏展示 |
| 开户行 | 银行卡时展示 |
| 是否法人本人账户 | 是/否 |
| 授权材料 | 非法人账户时必看 |
| 财务审核状态 | 待审核、通过、驳回 |

规则：

1. 收款账户审核可由财务角色单独处理。
2. 收款账户非本人不直接拒绝，但必须有授权材料。
3. 授权材料必须进入操作日志和资料归档。

### 5.6 e 签宝授权

| 字段 | 说明 |
|---|---|
| 授权状态 | 未发起、授权中、已完成、失败、过期 |
| 授权主体 | 商户主体或法人 |
| 授权用途 | 平台后续合同签署 |
| 授权文件 | 完成后可查看 |
| 回调时间 | e 签宝回调时间 |
| 回调摘要 | 不展示敏感明文 |

规则：

1. e 签宝授权完成是推荐通过条件。
2. 如业务允许先通过后补授权，系统必须标记 `授权待补`，限制合同相关能力。
3. 授权状态必须可从 e 签宝回调同步。

### 5.7 应用权限和默认配置

审核通过前可预览开通内容：

| 配置 | 默认值 |
|---|---|
| 商家 PC 端 | 开通 |
| 门店手机端 | 开通 |
| 办单助手 | 开通三入口，但按链路配置控制可用 |
| 商品管理 | 可添加商品，提交运营审核 |
| 员工账号 | 老板可创建，员工最小权限 |
| 钱包 | 生成商家钱包 |
| 商家订单 | 默认开通，自审 |
| 联营订单 | 默认可发起，运营审核 |
| 平台订单 | 默认可发起，运营审核 |
| 链路配置 | 继承平台默认，可由运营端做商户覆盖 |

## 6. 操作按钮

| 操作 | 使用场景 | 结果 |
|---|---|---|
| 审核通过 | 资料无问题 | 开通商家主体、门店、老板账号、钱包和默认配置 |
| 驳回 | 不符合入驻要求 | 填写驳回原因，商家可查看 |
| 要求补资料 | 资料缺失或需修正 | 选择需补字段，退回商家修改 |
| 修改审核备注 | 内部沟通 | 记录备注，不改变状态 |
| 查看授权文件 | e 签宝完成后 | 打开授权凭证 |
| 查看操作日志 | 任意状态 | 展示提交、修改、审核、回调记录 |
| 暂停店铺 | 已通过后风险处理 | 暂停商家登录或限制办单 |

敏感操作必须二次确认：

- 审核通过
- 驳回
- 暂停店铺
- 查看完整敏感信息
- 修改收款账户审核结论

## 7. 审核通过后的系统动作

审核通过后自动执行：

1. 创建或激活商家主体。
2. 创建门店档案。
3. 激活商家老板账号。
4. 生成商家 PC 端和门店手机端登录权限。
5. 生成商家钱包和提现账户草案。
6. 绑定渠道来源，如有推广码；扫码入驻时必须记录被扫码的渠道人员或推广人员。
7. 继承平台默认链路配置。
8. 初始化商家订单、联营订单、平台订单可用权限。
9. 写入审核通过日志。
10. 通知商家审核通过。

失败处理：

| 场景 | 处理 |
|---|---|
| 钱包创建失败 | 审核状态进入通过待初始化，通知财务/技术处理 |
| 账号激活失败 | 回滚开通动作或进入异常队列 |
| 渠道绑定失败 | 不阻断审核通过，但进入异常队列；申请记录保留原始推广码和扫码人员信息 |
| e 签宝状态异常 | 若强制要求授权完成，则阻止通过 |

## 8. 驳回与补资料

驳回/补资料必须选择原因：

| 类型 | 示例 |
|---|---|
| 营业执照问题 | 图片不清晰、主体名称不一致、证照过期 |
| 法人身份证问题 | 图片不清晰、证件过期、法人不一致 |
| 门店真实性问题 | 地址不完整、门头照片不清晰、定位异常 |
| 收款账户问题 | 账户名不一致、授权材料缺失、账号无法校验 |
| e 签宝问题 | 授权未完成、授权失败、主体不一致 |
| 其他 | 自定义说明 |

规则：

1. `要求补资料` 可以选择具体字段，商家端只开放这些字段修改。
2. `驳回` 需要明确是否允许重新申请。
3. 所有原因同步给商家端，但内部备注不一定展示给商家。

## 9. 权限控制

| 角色 | 权限 |
|---|---|
| 平台管理员 | 查看、审核、驳回、暂停、查看完整资料、导出 |
| 店铺审核人员 | 查看资料、审核、驳回、补资料、备注 |
| 财务审核人员 | 查看收款账户、审核收款账户、查看授权材料 |
| 运营负责人 | 复核高风险申请、暂停店铺、调整商户覆盖配置 |
| 普通客服 | 只读查看基础信息，不看完整敏感资料 |

敏感资料查看要求：

1. 身份证号、银行卡号、支付宝账号默认脱敏。
2. 查看完整信息需要权限和二次验证。
3. 查看动作必须写日志。

## 11. 失败态与异常

| 场景 | 页面处理 |
|---|---|
| 入驻资料不完整 | 列表标签提示，详情页定位缺失字段 |
| OCR 失败 | 标记人工填写，审核人员重点复核 |
| 法人不一致 | 风险标签高亮 |
| 收款账户非本人 | 要求查看授权材料 |
| e 签宝未完成 | 标记待授权，不建议通过 |
| 审核通过初始化失败 | 进入异常队列，不重复创建主体 |
| 重复提交 | 合并到原申请或提示已有申请 |
| 渠道码无效 | 不阻断申请，标记无有效渠道 |

## 12. 操作日志

必须记录：

1. 商家注册。
2. 资料保存和提交。
3. OCR 识别结果和人工修改。
4. 图片上传。
5. 收款账户新增和修改。
6. e 签宝授权发起、回调、失败。
7. 审核通过、驳回、补资料。
8. 查看完整敏感信息。
9. 开通商家主体、钱包、账号。
10. 暂停/恢复店铺。

日志字段：

| 字段 | 说明 |
|---|---|
| 操作时间 | 精确到秒 |
| 操作人 | 商家/运营/系统 |
| 操作端 | H5、运营端、系统回调 |
| 操作类型 | 提交、审核、授权、回调、查看、暂停 |
| 操作前 | 状态或字段摘要 |
| 操作后 | 状态或字段摘要 |
| 关联对象 | 申请、商家、门店、账号、钱包 |
| IP/设备 | 可用时记录 |
