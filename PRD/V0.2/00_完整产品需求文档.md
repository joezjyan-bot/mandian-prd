# 满点租赁系统重构完整产品需求文档 V0.2

> PRD 是 Product Requirements Document 的缩写，中文是产品需求文档。  
> 本文是 V0.2 的总版 PRD，供业务方、产品、UI、后端、前端、测试、Claude/Codex 从头阅读。  
> 页面级细节、接口草案、数据库草案、操作手册模板仍以 `modules/` 下对应文档为准。

---

## 1. 文档目标

### 1.1 目标

重构一套可长期运营的租赁业务系统，覆盖：

- 商家/门店入驻。
- 商品、规格、增值服务、办单助手。
- 三类订单：门店订单、分红订单、平台订单。
- 客户扫码下单、资料填写、授权、签约、支付、发货、签收。
- 订单审核、资方分配、改价、补资料、客服 IM。
- 账单、还款、分账、钱包、提现、对账。
- 渠道推广、佣金、渠道 H5。
- 租后、客诉、黑名单、监管锁、日志、回调。
- 后续可扩展短租，但当前版本不展示、不细化短租业务。

### 1.2 当前版本边界

当前 V0.2 重点落地长租主链路。

短租口径：

- 当前三个办单助手不展示短租入口。
- 当前 UI 不展示小时租、天租、周租。
- 当前 PRD 不继续细化短租业务需求。
- 后续由公司内部人员完成完整短租需求包后，再嫁接到系统。
- 后端只保留可接入边界：商品/规格扩展、设备库存扩展、订单子类型扩展、账单计费扩展、履约证据扩展。

## 2. 系统总体定位

### 2.1 租赁系统是核心业务系统

租赁系统负责完整租赁业务闭环：

- 入驻认证。
- 商品管理。
- 办单助手。
- 客户下单。
- 订单审核。
- 合同、公证、授权、支付。
- 发货、交付、签收、归还。
- 账单、分账、提现、对账。
- 渠道、租后、客诉、权限、日志。

### 2.2 中控台不是租赁系统

中控台是租赁系统外部能力系统，不并入租赁系统。

中控台负责：

- 第三方服务 token。
- 密钥凭证。
- 调用额度。
- 套餐购买。
- 调用计费。
- 调用记录。
- 客户级启停。

租赁系统负责：

- 订单业务。
- 业务状态。
- 合同模板。
- 账单和分账。
- 第三方回调处理。
- 业务日志。

两套系统后续通过 API 打通。

## 3. 端和角色

### 3.1 运营端

运营端是总后台，负责全平台管理：

- 商家和门店管理。
- 商品和商品审核。
- 全平台订单。
- 分红订单和平台订单审核。
- 资方管理和资方分配。
- 财务、分账、提现、对账。
- 渠道、佣金、推广码。
- 租后、客诉、黑名单、监管锁。
- 链路配置、通道配置、权限、日志。

### 3.2 商家 PC 端

商家 PC 端和门店手机端属于同一个商家主体。

商家 PC 端偏管理：

- 商品管理。
- 门店订单配置。
- 三类订单查看。
- 门店订单自审。
- 员工账号管理。
- 钱包、提现、流水、对账。
- 设备库存。
- 门店资料和授权状态。

### 3.3 门店手机端

门店手机端偏现场操作：

- 门店登录和入驻。
- 办单助手三入口。
- 客户扫码下单。
- 待办任务。
- 交付照片上传。
- 设备扫码。
- 订单进度查看。

店员账号只允许办单和处理待办，不允许查看钱包、分账、提现、全量财务数据。

### 3.4 C 端客户

C 端客户负责：

- 扫码确认订单。
- 填写资料。
- 实名、人脸、授权。
- 支付首期或账单。
- 签署合同。
- 办理公证。
- 查看订单、账单、待办。
- 签收、归还、留购、续租。

客户侧不展示内部字段：

- 资方分配。
- 平台抽佣。
- 分红比例。
- 分账比例。
- 门店出资额。
- 资方出资额。
- 预计收益。

### 3.5 渠道端

渠道端是 H5 数据看板，负责：

- 渠道登录。
- 推广码查看。
- 入驻商家统计。
- 分红订单统计。
- 平台订单统计。
- 在租、逾期、完成等状态统计。
- 佣金明细。
- 提现申请。

门店自营订单不计入渠道佣金统计。

## 4. 核心业务对象

### 4.1 商家和门店

商家/门店通过入驻流程成为系统主体。

入驻资料包括：

- 手机号和登录密码。
- 营业执照。
- 法人身份证正反面。
- 门店真实地址。
- 地图定位。
- 门头照片。
- 收款支付宝账号或银行卡号。
- 如收款账户非法人实名，需要授权材料。
- e 签宝授权签章。

入驻后进入运营端店铺审核。审核通过后，商家可使用商家 PC 端和门店手机端。

### 4.2 商品和规格

商品是展示和价格配置对象。

规格支持：

- 全新。
- 二手。
- 容量。
- 颜色。
- 电池。
- 其他业务规格。

全新和二手不应拆成两个商品，应该作为同一个商品下的规格。

商品可配置是否同步到办单助手。

### 4.3 增值服务

增值服务可由运营端或商家端维护。

- 门店订单使用商家端增值服务配置。
- 分红订单、平台订单使用运营端增值服务配置。

增值服务示例：

- 公证费。
- 服务费。
- 设备管理费。
- 监管锁服务费。
- 延保。
- 意外保障。
- 运费。

### 4.4 设备库存

设备是实际履约对象。

长租当前支持在审核、发货、交付或签收节点绑定唯一设备。

设备字段包括：

- 设备码。
- IMEI / SN / VIN。
- 所属商品和规格。
- 所属商家/门店/仓库。
- 当前状态。
- 监管锁编号。
- 最近绑定订单。

设备状态包括：

- 在库可用。
- 已锁定。
- 出租中。
- 归还待验。
- 维修中。
- 已下架。

## 5. 三类订单统一口径

系统只保留三类订单。

| 订单类型 | 定义 | 审核归属 | 财务归属 |
|---|---|---|---|
| 门店订单 | 门店自有订单，门店自己做 | 门店/商家自审 | 门店钱包，平台可抽佣 |
| 分红订单 | 门店出设备，资方补资金，收益按比例分 | 运营端审核 | 门店和资方按比例分账 |
| 平台订单 | 门店送单给平台，平台匹配资方或执行方 | 运营端审核 | 平台/资方主控，可给门店佣金 |

历史旧名称不再作为正式业务名，统一改为 `平台订单`。

### 5.1 门店订单

门店订单特点：

- 门店使用自己的商品。
- 门店使用自己的费率、套餐、增值服务。
- 门店自己审核是否通过。
- 不需要平台审核。
- 平台可设置抽佣比例，默认 2%。
- 抽佣后剩余进入门店钱包。

### 5.2 分红订单

分红订单特点：

- 门店出设备或等效出资。
- 门店选择需求配资比例。
- 资方补足资金。
- 运营端审核。
- 运营端分配资方。
- 客户每笔还款按出资比例拆分。
- 平台默认 2% 抽佣，按各方份额分别扣除。

示例：

- 设备价：5000。
- 需求配资比例：80%。
- 资方出资：4000。
- 门店等效出资：1000。
- 门店占比：20%。
- 资方占比：80%。

客户侧不展示这些分红和分账字段。

### 5.3 平台订单

平台订单特点：

- 门店手机端可以发起。
- 商家 PC 端可以查看进度。
- 审核、资方、财务主控在运营端。
- 待审核列表可点击联系客服，推送订单信息到 IM。
- 客服可根据客户资质审核、改价、改套餐、补资料。
- 可产生门店协作佣金和渠道佣金。

## 6. 办单助手

### 6.1 办单助手定位

办单助手用于门店快速生成价格方案和二维码。

办单助手逻辑参考 `joezjyan-bot/calculator` 仓库中的 `phone-rent` 惠讯租计算器逻辑和 UI。

### 6.2 三入口

门店手机端办单助手拆成三个入口：

1. 门店订单。
2. 分红订单。
3. 平台订单。

三个入口字段基本一致：

- 商品。
- 规格。
- 长租租期/套餐。
- 设备价。
- 首期。
- 后期账单。
- 押金。
- 服务费。
- 公证费。
- 增值服务。
- 留购价。
- 账单预览。
- 二维码。

分红订单只额外增加：

- 配资比例。
- 门店出资额。
- 资方出资额。
- 内部分账/收益预览。

分红内部测算只给商家老板、商家 PC 端和运营后台看，不展示给客户。

### 6.3 二维码和价格快照

办单助手生成二维码时必须保存价格快照。

价格快照包括：

- 商品快照。
- 规格快照。
- 设备价。
- 租期。
- 押金。
- 首期。
- 后期账单。
- 服务费。
- 公证费。
- 增值服务。
- 留购价。
- 价格版本。
- 操作人。
- 有效期。

客户扫码后，订单读取服务端价格快照，不允许客户侧改公式。

客服后续可改价，但必须生成新价格快照，并写操作日志。

## 7. 客户下单主链路

客户扫码后进入 C 端下单流程。

主流程：

```text
门店办单助手生成二维码
  -> 客户扫码
  -> 读取价格快照
  -> 确认商品和账单
  -> 填写资料
  -> 实名/人脸/授权
  -> 提交订单
  -> 进入待审核或待付款
  -> 后台审核
  -> 后台发起合同/公证/支付/授权
  -> 客户回到小程序订单页继续操作
  -> 审核通过
  -> 发货或门店交付
  -> 签收/起租
  -> 生成账单计划
  -> 还款/代扣/部分支付
  -> 到期归还、续租或留购
```

后台动作和客户动作通过状态和回调打通。

## 8. 订单审核

### 8.1 门店订单审核

门店订单由商家/门店自审。

可操作：

- 查看客户资料。
- 查看授权状态。
- 查看报价快照。
- 审核通过。
- 审核不通过。
- 改价。
- 补资料。
- 发起合同或门店自有合同流程。
- 发货/交付。

### 8.2 分红订单审核

分红订单由运营端审核。

审核重点：

- 客户资料。
- 实名和人脸。
- 风控授权。
- 订单价格。
- 设备价。
- 配资比例。
- 资方分配。
- 合同、公证、支付链路。

### 8.3 平台订单审核

平台订单由运营端审核。

审核重点：

- 客户资质。
- 是否分配资方。
- 是否改价或改套餐。
- 是否需要补资料。
- 是否需要联系客服。
- 是否进入合同、公证、支付、发货流程。

## 9. 合同、公证、授权和支付

### 9.1 合同

合同能力包括：

- 主合同。
- 补充合同。
- e 签宝签署。
- 商家签章授权。
- 合同模板。
- 合同变量。
- 合同回调。
- 合同附件归档。

### 9.2 公证

公证不是所有订单默认启用，应由链路配置控制。

公证状态：

- 未发起。
- 已发起。
- 办理中。
- 完成。
- 失败。

公证操作由后台发起，客户在小程序订单详情中继续办理。

### 9.3 授权

授权包括：

- 风控授权。
- 征信授权。
- 人脸核验。
- 代扣绑卡。
- 商家 e 签宝授权。

授权状态必须同步回订单详情和待审核列表。

### 9.4 支付

支付能力包括：

- 首期支付。
- 后续账单支付。
- 代扣签约。
- 部分支付收款码。
- 退款。
- 冲正。
- 通道流水。
- 回调日志。

部分支付收款码后续对接通联或信联。

## 10. 发货、交付、签收和设备

系统默认经营模式是门店发货或门店交付，但保留线上物流接口。

发货主体：

- 门店。
- 商家。
- 平台。
- 资方。

发货方式：

- 门店自提。
- 门店配送。
- 快递发货。
- 顺丰线上发货。

交付证据：

- 人机合照。
- 人车合照。
- 门店交付照片。
- 快递单号。
- 签收照片。
- 归还照片。

所有证据进入附件中心，并与订单、设备、操作日志关联。

## 11. 财务、分账、钱包和对账

### 11.1 账单

账单来源于办单助手价格快照和订单审核结果。

账单包括：

- 首期账单。
- 后续账单。
- 服务费。
- 公证费。
- 设备管理费。
- 运费。
- 留购价。
- 补差价。
- 退款。
- 部分支付。

### 11.2 分账

门店订单：

- 客户还款。
- 平台按配置抽佣，默认 2%。
- 剩余进入门店钱包。

分红订单：

- 客户还款。
- 按出资比例拆分门店份额和资方份额。
- 平台 2% 抽佣分别从双方份额扣除。
- 剩余进入门店钱包和资方账户。

平台订单：

- 平台/资方获得主要回款。
- 门店可获得协作佣金。
- 渠道可获得渠道佣金。

### 11.3 钱包和提现

钱包支持：

- 可用余额。
- 冻结余额。
- 待结算余额。
- 分账流水。
- 提现申请。
- 打款状态。
- 对账状态。

店员账号不可查看钱包。

### 11.4 对账

对账需要覆盖：

- 订单。
- 账单。
- 支付流水。
- 分账流水。
- 钱包流水。
- 提现流水。
- 通道流水。
- 回调日志。

财务异常进入异常队列，支持人工复核。

## 12. 渠道分销

渠道管理包括：

- 新增渠道。
- 渠道推广码。
- 扫码跳转门店入驻。
- 渠道 H5 登录。
- 入驻商家统计。
- 分红订单统计。
- 平台订单统计。
- 佣金明细。
- 提现申请。

佣金规则支持：

- 固定金额：每单 XX 元。
- 比例佣金：订单流水 XX%。
- 一二级关系。
- 佣金归属变更。

门店自营订单不计渠道佣金。

## 13. 租后、客诉、黑名单和监管锁

### 13.1 租后

租后管理先保留基础模块，后续可对接外部催收系统。

支持：

- 逾期订单。
- 临近到期。
- 催收跟进。
- 回款记录。
- 外部催收导入。
- 外部催收回调。

### 13.2 客诉

客诉管理保留，重点对接支付宝投诉接口。

支持：

- 投诉同步。
- 投诉列表。
- 订单上下文。
- 回复处理。
- 处理记录。
- 客诉日志。

### 13.3 黑名单

黑名单库支持：

- 姓名脱敏。
- 身份证脱敏。
- 手机号脱敏。
- 外部黑名单预留。
- 下单/审核时匹配。
- 疑似命中弹窗。
- 加黑日志。

### 13.4 监管锁

监管锁管理支持：

- 监管锁设备配置。
- 订单绑定。
- 锁机。
- 解锁。
- 接口回调。
- 操作日志。

## 14. 权限和日志

### 14.1 权限原则

权限必须覆盖：

- 菜单权限。
- 页面权限。
- 按钮权限。
- 字段权限。
- 数据范围。
- 二次验证。
- 审批流。

高风险动作必须二次确认：

- 审核通过。
- 审核拒绝。
- 改价。
- 关闭订单。
- 退款。
- 冲正。
- 打款。
- 加黑名单。
- 锁机/解锁。

### 14.2 操作日志

所有人工操作、系统操作、回调动作都要留痕。

日志至少包含：

- 操作人。
- 操作角色。
- 操作端。
- 操作对象。
- 操作前数据。
- 操作后数据。

## 16. 数据互通要求

系统不能徒有其表。以下数据必须互通：

- 商品同步到办单助手。
- 办单助手价格快照同步到订单。
- 订单同步到审核、合同、公证、支付、发货。
- 支付同步到账单。
- 账单同步到分账。
- 分账同步到钱包。
- 钱包同步到提现和对账。
- 渠道来源同步到佣金。
- 设备绑定同步到订单、发货、归还。
- 客诉、租后、黑名单同步到订单详情。
- 回调同步到状态和日志。

## 18. 非功能要求

### 18.1 安全

- 敏感信息脱敏展示。
- 敏感文件权限控制。
- 密钥不入库明文。
- `.env` 和本地配置不提交仓库。
- 高风险动作二次确认。
- 操作日志不可删除。

### 18.2 稳定性

- 回调必须幂等。
- 支付、合同、公证、发货回调失败进入异常队列。
- 支持人工补偿。
- 支持重试。
- 支持状态追踪。

### 18.3 可追溯

- 价格快照可追溯。
- 审核记录可追溯。
- 改价记录可追溯。
- 合同版本可追溯。
- 分账流水可追溯。
- 回调日志可追溯。

## 19. 验收主场景

### 19.1 入驻场景

商家注册、上传资质、填写收款账户、完成 e 签宝授权，运营审核通过后可登录商家 PC 端和门店手机端。

### 19.2 门店订单场景

门店使用门店订单办单助手生成二维码，客户扫码下单，门店自审通过，门店交付，客户还款，平台抽佣后进入门店钱包。

### 19.3 分红订单场景

门店使用分红订单办单助手选择配资比例，客户扫码下单，运营审核并分配资方，客户签约支付，后续还款按比例分账。

### 19.4 平台订单场景

门店发起平台订单，客户扫码下单，运营审核和分配资方，平台主控后续流程，门店查看进度并可获得配置的协作佣金。

### 19.5 改价场景

客服改价后生成新价格快照，账单、合同变量、客户确认、操作日志同步更新。

### 19.6 权限场景

店员账号可以办单和处理待办，但不能查看钱包、提现、分账、全量财务数据。

## 21. 下一步建议

你先按本文从头到尾审一遍，重点看这些问题：

1. 三类订单定义是否完全符合你的经营模式。
2. 门店订单、分红订单、平台订单的审核和财务归属是否正确。
3. 办单助手字段是否够用，客户侧展示是否会造成歧义。
4. 商家、门店、店员、渠道、资方权限是否符合真实业务。
5. 财务分账、钱包、提现、对账是否符合实际结算方式。
6. 哪些模块需要提前做，哪些可以后置。

确认总版 PRD 后，再进入开发落地层：

- 数据库字段最终版。
- 接口清单最终版。
- 状态机最终版。
- 权限矩阵最终版。
- UI 页面图。
