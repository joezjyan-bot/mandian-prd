# 修改还款日

> **来源**: P1-3
> **核心**: 仅运营客服可操作,7日内可修改一次,后延无需客户确认

## 1. 页面定位

运营端 · 订单管理 · 订单详情 · 修改还款日入口。

## 2. 适用条件

- 订单已生效(已签合同 + 已分账)
- 订单生效日期 ≤ 7 日
- 修改次数 = 0(仅允许 1 次)

不满足任一条件,功能禁用并提示原因。

## 3. 操作端

**仅运营客服在运营端**(客户/门店都无此功能)
- C 端:不显示
- 门店端/商家 PC:不显示
- 客户/门店需修改:联系平台客服转达,客服代操作

## 4. 操作流程

```
1. 客服在订单详情点击"修改还款日"按钮
2. 系统校验:
   - 订单是否已生效
   - 是否在 7 日内
   - 是否未修改过
   - 任一不通过:按钮置灰 + 提示原因
3. 弹窗:
   - 显示原还款日 + 客户要求
   - 选新还款日(每月几号,1-28 / 30 / 31)
   - 必填修改原因(下拉):客户收入日变化 / 客户主动要求 / 其他
4. 二次确认
5. 提交后实时:
   - 更新 order.repayment_day
   - 记 repayment_day_change_count = 1
   - 作废原账单(未到期部分)
   - 按新还款日重新生成所有未到期账单
   - 自动发 C 端推送通知
   - 留操作日志
```

## 5. 还款日修改的影响

- 修改方向必须是**后延**(对客户有利)
  - 校验:if 新还款日 > 原还款日 then 直接生效
  - if 新还款日 < 原还款日 then 加一个月:即 5月15→次月12,实际下次还款延后

- 包括已生成第 1 期账单一起改
- 后续所有期数按新还款日生成
- 不产生逾期违约金

## 6. 客户感知

- 不需客户确认
- 系统自动小程序推送:
  `您的订单 XX 还款日已由 X 号调整为 X 号,已生效。`
- 不发短信(避免打扰)

## 7. 限制规则

- 1 个订单仅允许修改 1 次
- 修改时间窗:订单生效后 7 日
- 7 日后:按钮置灰,提示"已超过修改窗口"
- 已修改过:按钮置灰,提示"该订单已修改过"

## 8. 修改记录展示

订单详情显示:
- 原还款日 / 当前还款日
- 修改时间 / 修改人 / 修改原因
- 可点击查看完整日志

## 9. 权限 & 日志

- 仅客服 + 客服主管可操作
- 二次确认 + 完整日志
- 数据库 audit:operator_id / from_day / to_day / 受影响账单列表 / changed_at

## 10. 数据模型

- `order.repayment_day`
- `order.original_repayment_day`
- `order.repayment_day_change_count`
- `order.repayment_day_change_deadline`
- `repayment_day_change_log`(完整日志表)
