# 权限矩阵

> 页面级 PRD 草案。
> 目标：把各端页面、按钮、数据范围、敏感字段和高风险动作落成可配置矩阵，避免只靠菜单权限导致越权操作。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 权限矩阵 |
| 所属端 | 运营端 |
| 入口路径 | 权限管理 > 权限矩阵 |
| 使用角色 | 平台管理员、权限管理员、审计人员 |
| 核心目标 | 统一配置菜单、按钮、字段、数据范围、二次验证和审批要求 |

---

## 2. 权限层级

| 层级 | 示例 |
|---|---|
| 系统权限 | 运营端、商家端、渠道端 |
| 菜单权限 | 订单管理、财务管理、配置管理 |
| 页面权限 | 订单列表、订单详情、提现审核 |
| 按钮权限 | 审核通过、退款、打款、改价、发起合同 |
| 字段权限 | 手机号、身份证、银行卡、风控报告 |
| 数据范围 | 全平台、指定商家、指定门店、本人待办 |
| 安全策略 | 二次验证、主管审批、操作日志 |

---

## 3. 角色模板

| 角色 | 典型权限 |
|---|---|
| 平台管理员 | 全量配置和管理 |
| 审核客服 | 待审核、补资料、发起合同、公证、风控 |
| 财务 | 钱包、提现、退款、对账 |
| 财务主管 | 打款、冲正、通道切换、异常处理 |
| 租后客服 | 逾期、跟进、部分支付、法务材料 |
| 客诉客服 | 投诉处理、订单上下文、回复 |
| 商家老板 | 自己商家订单、商品、员工、钱包 |
| 商家员工 | 办单、交付、归还、待办 |
| 渠道账号 | 推广数据、佣金、提现 |

模板可复制后自定义。

---

## 4. 按钮权限

| 按钮 | 建议权限 |
|---|---|
| 审核通过 | 审核客服，部分场景主管复核 |
| 审核拒绝 | 审核客服，需填写原因 |
| 改价/改套餐 | 审核客服，超过阈值主管复核 |
| 发起补资料 | 审核客服 |
| 发起合同/公证 | 审核客服或系统 |
| 关闭订单 | 主管或按订单状态授权 |
| 退款审核 | 财务 |
| 冲正 | 财务主管 |
| 提现审核 | 财务 |
| 打款确认 | 财务主管 |
| 查看敏感字段 | 高权限，记录日志 |
| 暂停店铺 | 管理员/运营主管 |

---

## 5. 字段权限

| 字段 | 默认策略 |
|---|---|
| 客户姓名 | 脱敏 |
| 手机号 | 脱敏，搜索可完整输入 |
| 身份证号 | 默认不展示完整 |
| 银行卡/收款账号 | 默认脱敏 |
| 详细地址 | 按角色脱敏 |
| 风控报告 | 普通角色只看摘要 |
| 合同文件 | 按订单和角色查看 |
| 财务金额 | 商家员工不可见 |

查看完整敏感字段必须二次验证并写入敏感查看日志。

---

## 6. 数据范围

| 范围 | 说明 |
|---|---|
| 全平台 | 平台管理员 |
| 指定业务线 | 多业务线预留 |
| 指定商家 | 商家客服或运营人员 |
| 指定门店 | 门店员工 |
| 本人待办 | 审核、租后、客诉人员 |
| 推广范围 | 渠道账号 |

数据范围与按钮权限同时生效，不能只有按钮权限没有数据限制。

---

## 7. 审计

权限变更必须记录：操作人、被操作角色、变更前后、原因、生效时间。高风险权限变更支持审批后生效。
