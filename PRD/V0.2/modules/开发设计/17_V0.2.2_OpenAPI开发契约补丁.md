# V0.2.2 OpenAPI 开发契约补丁

> 本文是把 PRD 从 91 分推进到 95 分的接口契约补丁。`PRD/V0.2/openapi/core-api.v0.2.yaml` 已同步新增体验租短租、IM 客服、法诉证据包和结算校验核心路径；本文用于研发评审时核对接口边界、DTO 白名单、幂等键和错误码。

## 1. 契约状态

| 项 | 当前口径 |
|---|---|
| OpenAPI 版本 | `0.2.2` |
| 技术内部枚举 | 可保留 `merchant / joint / platform` |
| 展示层订单名 | 商家订单 / 联营订单 / 平台订单 |
| C 端禁用字段 | 资方、资金来源、合作模式、风控结论、内部审核备注、服务费拆分、商家结算价、平台利润、返点规则、黑灰名单结论 |
| 必带请求头 | `X-Request-Id`、写接口 `X-Idempotency-Key`、端类型 `X-Client-Type` |
| 金额单位 | 接口金额默认用“分”,前端展示负责格式化 |

## 2. P0 接口域

| 接口域 | 必须覆盖 | 关键接口 |
|---|---|---|
| 安心用长租 | 下单、资料、审核、签约、交付、签收、结算校验、退款 | `/customer/orders`、`/orders/{order_id}/review/approve`、`/orders/{order_id}/settlement-check`、`/orders/{order_id}/refunds` |
| 体验租短租 | 下单、接单/拒单、取车核销、还车验收、异常工单 | `/experience-rent/orders`、`/experience-rent/orders/{short_order_id}/acceptance`、`/pickup`、`/return-inspection`、`/exceptions` |
| 财务四账 | 支付、退款、冲正、提现、对账、结算三条件 | `/wallets/{wallet_id}/withdrawals`、`/finance/reversals`、`/finance/reconciliation-batches` |
| IM 客服 | 创建会话、发消息、上下文卡片、合规命中 | `/im/sessions`、`/im/sessions/{session_id}/messages` |
| 法诉证据 | 证据包生成、导出留痕 | `/legal/evidence-packages` |
| 第三方回调 | 支付、合同、公证、监管锁、IM/短信回调 | `/callbacks/{channel}/{event_type}` |

## 3. DTO 白名单

### 3.1 C 端订单 DTO

C 端只允许返回:

- `order_id / short_order_id`
- `order_no`
- `product_line`: `long_rent / experience_rent`
- `status / status_label`
- 商品/车辆展示信息
- 门店展示信息
- 客户待办
- 应付金额、已付金额、保证金状态、账单状态
- 合同/公证/签收/取还车待办入口
- 客服入口和上下文 ID

C 端禁止返回:

- `funding_source`
- `funder_id`
- `cooperation_mode`
- `risk_result`
- `risk_score`
- `internal_review_note`
- `merchant_settlement_amount`
- `platform_profit`
- `rebate_rule`
- `blacklist_status`
- `graylist_status`

### 3.2 商家端 DTO

商家端允许看到自身履约、订单、结算、钱包、体验租车辆、退款责任和客服工单信息；联营订单/平台订单仅展示履约视图，不展示平台利润、内部资金来源和后台审核细节。

### 3.3 运营端 DTO

运营端可按权限查看完整业务对象。任何涉及完整身份证、完整银行卡、内部资金来源、退款审批、打款、坏账核销、法诉导出字段，接口必须返回 `permission_scope` 和 `audit_required=true`。

## 4. 幂等规则

| 场景 | 幂等键建议 | 重复请求处理 |
|---|---|---|
| 客户下单 | `customer_id + quote_snapshot_id + client_order_token` | 返回首次订单 |
| 支付回调 | `channel + channel_trade_no + event_type` | 已处理则返回 accepted，不重复入账 |
| 退款 | `order_id + payment_id + refund_amount + refund_reason_hash` | 返回首次退款单 |
| 结算打款 | `order_id + settlement_batch_id + receiver_id` | 不重复生成结算流水 |
| 提现 | `wallet_id + amount + receive_account_id + client_submit_token` | 不重复冻结余额 |
| 返点入账 | `order_id + promoter_id + rule_version` | 不重复入账 |
| 体验租取车 | `short_order_id + pickup_code + vehicle_id` | 不重复绑定车辆 |
| 体验租还车 | `short_order_id + return_inspection_token` | 不重复释放车辆和保证金 |
| IM 发消息 | `session_id + client_message_id` | 不重复落消息 |
| 法诉导出 | `order_id + export_reason_hash + include_sections_hash` | 可复用已有 package 或生成新版本 |

## 5. 核心错误码

| 错误码 | 触发场景 | 处理建议 |
|---|---|---|
| `C_SIDE_FORBIDDEN_FIELD` | C 端 DTO 包含禁用字段 | 阻断发布,接口回归失败 |
| `SETTLEMENT_CONDITION_NOT_MET` | 签收/监管锁/激活锁未满足 | 禁止打款,返回 blockers |
| `ALTERNATIVE_ACCEPTANCE_REQUIRED` | 无监管锁品类缺少替代验收材料 | 进入人工复核 |
| `SHORT_RENT_STOCK_UNAVAILABLE` | 体验租门店无可租车辆 | 提示换车/换门店/换时间 |
| `SHORT_RENT_VEHICLE_MISMATCH` | 核销车辆与订单车型不一致 | 禁止取车 |
| `DEPOSIT_DEDUCTION_REVIEW_REQUIRED` | 保证金扣款需审批 | 生成扣款审批/异常工单 |
| `IM_FORBIDDEN_ACTION` | IM 内尝试直接执行高危动作 | 返回业务页面跳转，不执行动作 |
| `EVIDENCE_PACKAGE_INCOMPLETE` | 法诉证据包缺关键证据 | 标记缺失项，允许补证 |
| `IDEMPOTENCY_REPLAY` | 重复幂等请求 | 返回首次处理结果 |
| `LEDGER_NOT_BALANCED` | 四账任一账不平 | 阻断资金类验收 |

## 6. 研发评审门槛

1. YAML 必须可被 OpenAPI 解析器读取。
2. 每个 P0 写接口必须定义幂等键、权限、状态冲突错误和审计字段。
3. C 端 DTO 必须做字段白名单测试。
4. 财务接口必须写明四账影响:订单业务账、支付流水账、钱包账、总账分录。
5. 短租接口不得复用长租订单状态机。
6. IM 接口只能创建会话、消息、工单和跳转上下文,不能直接执行退款、扣款、锁车、关单等高危动作。
