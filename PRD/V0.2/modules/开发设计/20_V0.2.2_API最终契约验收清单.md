# V0.2.2 API 最终契约验收清单

> **契约声明**:本文是 OpenAPI YAML 和接口开发评审之间的验收桥。技术可以调整字段实现细节,但不得改变 V0.2.2 业务口径、状态机、四账分离、权限审计和 C 端敏感字段隔离规则。

## 一、API 契约总原则

1. 所有接口必须有 `request_id` 或等价请求追踪号。
2. 所有会产生副作用的接口必须有 `idempotency_key`。
3. 所有资金、合同、签收、控车、退款、打款、返点、提现、法诉导出接口必须写入操作日志和审计日志。
4. 所有状态变更必须落 `order_status_log` 或对应状态流转表,不得只更新主表状态。
5. 所有第三方接口调用必须落 `third_party_call_log`,失败进入 `compensation_task`。
6. C 端 DTO 必须采用白名单返回,不得把运营端 DTO 过滤后复用。
7. API 错误必须包含稳定错误码、用户可读提示、内部排查码、是否可重试。
8. 金额字段统一以“分”为内部单位或明确 decimal 精度;对外展示由前端格式化,不得混用元/分。
9. 技术内部枚举可以保留兼容值,但返回给展示层必须包含 `display_name` 或映射到商家订单 / 联营订单 / 平台订单。

## 二、接口通用字段要求

| 字段 | 适用接口 | 要求 |
|---|---|---|
| `request_id` | 全部 | 必填或由网关生成;贯穿日志、回调、补偿任务 |
| `idempotency_key` | 创建 / 支付 / 退款 / 打款 / 返点 / 提现 / 控车 / 导出 | 必填;同一业务动作重复请求返回同一结果 |
| `operator_id` | 运营端 / 商家端 | 必填;C 端用客户身份 token 解析 |
| `permission_code` | 运营端 / 商家端高危动作 | 必填;后端不得仅靠前端隐藏按钮 |
| `business_no` | 资金 / 订单 / 合同 / 证据 | 必填;用于业务追踪 |
| `audit_required` | 高危动作 | 标记是否需要二次确认或二人复核 |
| `tenant_id` | 多租户部署 | 必须存在于服务端上下文或表字段,不得由前端任意传入 |
| `source_channel` | 下单 / 推广 / 支付 | 记录来源渠道、推广人、入口 |
| `callback_no` | 第三方回调 | 必填;用于回调幂等 |

## 三、C 端 DTO 禁返字段

C 端接口禁止返回:

- 资方信息
- 资金来源
- 合作模式
- 风控结论
- 内部审核备注
- 服务费拆分明细
- 商家结算价
- 平台利润
- 返点规则
- 内部标签
- 黑名单 / 灰名单结论
- 商家钱包余额
- 监管锁后台指令明细
- 法诉内部处理备注

若页面确实需要解释费用,只能返回客户侧费用字段:订单总额、首期应付、每期应付、保证金、逾期费用、退款金额、优惠金额、应付合计。

## 四、P0 接口验收表

| 接口域 | 关键接口 | 请求必备 | 响应必备 | 幂等键 | 权限 / 审计 | 验收标准 |
|---|---|---|---|---|---|---|
| C 端长租 | 创建订单 | 商品、规格、套餐、客户、渠道、协议确认 | 订单号、状态、下一步动作、客户侧费用 | `client_order_submit:{client_token}:{sku}:{timestamp_bucket}` | 客户操作日志 | 不返回内部资金来源和风控结论 |
| C 端资料 | 提交资料 | 订单号、资料字段、附件、授权确认 | 资料版本、状态、待办 | `profile_submit:{order_id}:{version}` | 资料变更日志 | 支持补资料和版本追踪 |
| C 端签约 | 发起 / 查询签约 | 订单号、合同模板、客户身份 | 签署链接、签署状态、失败原因 | `contract_launch:{order_id}:{template_id}` | 电子签记录 | 失败可重发,合同版本可追踪 |
| C 端支付 | 发起首期 / 账单支付 | 订单号、账单号、金额、支付方式 | 支付单号、支付状态、支付参数 | `payment:{bill_id}:{amount}` | 支付流水 | 重复点击不重复生成支付单 |
| C 端签收 | 电子确认单 / AI 问答 | 订单号、签收方式、确认内容、媒体凭证 | 签收状态、凭证 ID、下一步状态 | `acceptance:{order_id}:{mode}` | 签收证据 | 未签收不能结算 |
| 运营审核 | 审核通过 / 驳回 / 补资料 | 订单号、审核动作、原因、附件 | 新状态、客户待办、操作日志 | `audit:{order_id}:{action}:{version}` | 审核日志 | C 端只展示客户可读原因 |
| 运营改价 | 账单生成前改价 | 订单号、调整项、原因、审批 | 新价格版本、重算预览 | `price_adjust:{order_id}:{version}` | 高危审计 | 账单生成后禁止直接改账单 |
| 监管锁 | 查询 / 上锁 / 激活校验 | 订单号、设备码、供应商、指令 | 锁状态、激活状态、失败原因 | `lock_cmd:{order_id}:{device_id}:{cmd}` | 控车审计 | 未上锁或未激活禁止结算 |
| 财务结算 | 结算校验 / 发起打款 | 订单号、三条件校验、结算对象、金额 | 可结算结果、未满足项、打款状态 | `settlement:{order_id}:{amount}` | 二人复核 | 重复打款拦截 |
| 退款 | 发起 / 审核 / 回调 | 订单号、责任方、金额、原支付流水、凭证 | 退款单、冲正对象、状态 | `refund:{order_id}:{pay_id}:{amount}` | 资金审计 | 有责任方和冲正对象才允许提交 |
| 钱包提现 | 发起 / 审核 / 回调 | 商家、金额、账户、审批 | 提现单、冻结金额、状态 | `withdraw:{merchant_id}:{amount}:{account_id}` | 财务审计 | 失败回滚可提现余额 |
| 返点 / 推广 | 生成 / 入账 / 回滚 | 订单号、推广关系、规则版本、金额 | 返点单、钱包流水、状态 | `rebate:{order_id}:{promoter_id}:{rule_version}` | 资金审计 | 退款关闭按比例回滚 |
| 短租下单 | 创建体验租订单 | 门店、车型、租期、骑行人、保证金 | 短租订单号、接单状态、核销码 | `short_order:{client}:{store}:{vehicle_model}:{time}` | 客户日志 | 库存锁定失败必须返回可读原因 |
| 短租取还 | 接单 / 核销 / 取车 / 还车验收 | 短租订单、车辆码、照片、验收项 | 状态、费用、扣款建议 | `short_acceptance:{short_order}:{step}` | 商家/运营审计 | 保证金扣款必须有证据 |
| IM | 建会话 / 发消息 / 关联订单 | 用户、订单、消息、附件 | 会话 ID、消息 ID、归档状态 | `im_msg:{session_id}:{client_msg_id}` | IM 日志 | IM 不直接执行高危业务动作 |
| 法诉证据包 | 生成 / 导出 | 订单号、导出用途、附件范围 | 证据包 ID、清单、下载状态 | `legal_export:{order_id}:{export_scope}` | 法务审计 | 导出留痕且按订单归档 |

## 五、错误码验收规则

| 错误码族 | 适用场景 | 示例 | 页面处理 |
|---|---|---|---|
| `ORDER_STATE_CONFLICT` | 状态不允许当前动作 | 已生成账单后改账单 | 展示状态冲突和下一步 |
| `IDEMPOTENT_REPLAY` | 重复提交 | 重复支付、重复打款 | 返回原结果,不得二次执行 |
| `PERMISSION_DENIED` | 权限不足 | 无财务权限发起退款 | 按权限隐藏或提示申请权限 |
| `SETTLEMENT_CONDITION_UNMET` | 结算三条件不满足 | 未签收 / 未上锁 / 未激活 | 展示未满足项 |
| `SENSITIVE_FIELD_BLOCKED` | DTO 字段越界 | C 端请求内部字段 | 接口拒绝并报警 |
| `THIRD_PARTY_TIMEOUT` | 供应商超时 | 电子签 / 支付 / 锁控 | 进入补偿任务 |
| `AMOUNT_MISMATCH` | 金额不一致 | 回调金额与支付单不一致 | 冻结状态人工复核 |
| `EVIDENCE_REQUIRED` | 缺少凭证 | 保证金扣款无照片 | 禁止提交 |

## 六、回调接口验收

| 回调类型 | 幂等字段 | 必须校验 | 成功后动作 | 失败后动作 |
|---|---|---|---|---|
| 支付回调 | 第三方交易号 + 支付单号 | 金额、订单、账单、状态 | 写支付流水、更新账单、触发四账分录 | 进入支付异常池 |
| 退款回调 | 退款单号 + 第三方退款号 | 退款金额、原支付流水、责任方 | 写退款流水、冲正四账、回滚返点 | 进入退款异常池 |
| 电子签回调 | 合同号 + 签署事件 | 合同模板、签署人、订单 | 写电子签记录、推进状态 | 允许重发合同 |
| 监管锁回调 | 设备码 + 指令号 | 订单、设备、锁状态 | 更新锁状态、触发结算校验 | 人工复核 |
| IM 回调 | 会话 ID + 消息 ID | 发送人、附件、订单关联 | 写消息归档 | 补偿拉取 |
| 保险回调 | 保单号 + 订单号 | 保单状态、保障范围 | 写保单记录 | 人工补录 |

## 七、最终 API 评审通过条件

1. `core-api.v0.2.yaml` 可解析。
2. 每个 P0/P1 接口都有请求字段、响应字段、错误码、幂等键、权限点、日志表、补偿策略。
3. C 端接口通过禁返字段扫描。
4. 资金接口能对应四账分离模型。
5. 状态变更接口能对应状态机事件。
6. 第三方接口能对应供应商映射表和回调日志。
7. 测试用例能按接口域写断言。
