## 2. 核心口径

1. 短租必须绑定唯一车辆/设备,不能只扣商品库存数量。
2. 每台车必须有设备码,可按类目扩展 VIN、车架号、车牌号、监管锁信息。
3. 客户下单时锁定可用车辆或可用库存池,支付/保证金完成后进入待接单/待取车。
4. 归还后必须验收,验收通过才回到可租库存。
5. 车辆库存归属到门店或平台仓,支持平台自营短租和门店自营短租。
6. 短租订单、保证金、逾期费用、控车日志、续租链必须和车辆库存联动。

## 4. 状态边界

短租车辆库存状态沿用全局 9 状态:

```text
待入库 → 在库可租 → 预约保留 → 已锁定 → 出租中
出租中 → 归还待验 → 在库可租
归还待验 → 维修中 / 争议中 / 已下架
```

状态定义以 `全局/02_状态字典与订单状态机.md` 和 `短租业务/05_体验租数据模型与状态机.md` 为准。

## 5. 门店操作

| 操作 | 说明 |
|---|---|
| 接单 | 短租订单进入待商家接单后,门店确认是否有车可交付 |
| 拒单 | 无车或异常时拒单,写明原因 |
| 取车核销 | 客户到店出示核销码,门店绑定车辆并交付 |
| 还车验收 | 门店检查车辆、配件、照片、异常 |
| 维修登记 | 异常车辆进入维修中 |
| 争议登记 | 客户与门店对损坏/费用有分歧时进入争议中 |

## 6. 平台自营短租

平台自营体验租也使用同一套短租车辆库存:

| 场景 | 处理 |
|---|---|
| 平台仓自营 | 所属主体 = 平台,当前仓库 = 平台仓 |
| 平台委托门店交付 | 所属主体 = 平台,当前门店 = 履约门店 |
| 门店自营短租 | 所属主体 = 商家/门店 |

C 端只展示取还门店、车辆、价格和订单状态,不展示内部库存归属。

---

## 7. 与长租设备档案的区别

1. 短租库存下单前必须可用,长租不需要。
2. 短租车辆可多次出租,长租设备跟随单个长租订单履约。
3. 短租有取还车时段和超时费用,长租按订单周期和账单履约。
4. 短租需要门店接单/核销/验收,长租主要是发货/收货确认。
