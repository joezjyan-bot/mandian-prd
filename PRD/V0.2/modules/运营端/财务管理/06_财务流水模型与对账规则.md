# 财务流水模型与对账规则

> 所属端：运营端 / 财务管理
> 目标：把支付、账单、分账、钱包、提现、退款、冲正、对账拆成可开发的数据对象，确保钱流可追溯、可校验、可回滚。

---

## 1. 财务对象

| 对象 | 作用 |
|---|---|
| 账单 | 客户应付计划和实付状态 |
| 支付流水 | 客户通过通道支付或代扣的结果 |
| 分账流水 | 把实收金额拆给商家、资方、渠道、平台 |
| 钱包流水 | 各主体账户余额变化 |
| 提现单 | 主体从钱包提现到收款账户 |
| 打款流水 | 打款通道或线下打款结果 |
| 退款单 | 客户退款申请和财务处理 |
| 冲正单 | 已入账金额的反向调整 |
| 对账单 | 通道、业务、钱包三边核对结果 |

---

## 2. 账单模型

字段：

- `bill_id`
- `order_id`
- `period_no`
- `bill_type`：首期 / 租金 / 服务费 / 公证费 / 留购 / 补差价
- `amount_due`
- `amount_paid`
- `amount_refunded`
- `due_time`
- `paid_time`
- `status`
- `price_snapshot_id`
- `config_version_id`

规则：

- 首期实付和后续账单分开。
- 部分支付时账单状态为部分支付。
- 账单结清后才进入可分账流程。
- 退款和冲正不直接删除账单，必须写反向流水。

---

## 3. 支付流水模型

字段：

- `payment_id`
- `bill_id`
- `order_id`
- `channel`
- `channel_trade_no`
- `amount`
- `fee`
- `status`
- `callback_event_id`
- `paid_time`
- `raw_status`

规则：

- 支付成功必须关联账单。
- 一个账单可有多笔支付流水。
- 支付通道回调重复时不重复累加实收。
- 支付异常进入对账中心。

---

## 4. 分账流水模型

字段：

- `allocation_id`
- `payment_id`
- `bill_id`
- `order_id`
- `order_type`
- `beneficiary_type`：merchant / funder / channel / platform
- `beneficiary_id`
- `gross_amount`
- `commission_amount`
- `net_amount`
- `rule_version_id`
- `status`

规则：

- 门店订单：客户实收扣平台抽佣后进入商家钱包。
- 分红订单：按出资比例拆分后，门店和资方各自份额分别扣平台抽佣。
- 平台订单：按资方/平台配置生成收益，渠道佣金按规则生成。
- 分账失败不改钱包余额。
- 分账成功后才生成钱包流水。

---

## 5. 钱包流水模型

字段：

- `wallet_entry_id`
- `wallet_id`
- `owner_type`
- `owner_id`
- `direction`：in / out / freeze / unfreeze
- `amount`
- `balance_before`
- `balance_after`
- `source_type`
- `source_id`
- `status`

来源类型：

- 分账入账
- 平台抽佣
- 渠道佣金
- 接口计费
- 提现冻结
- 提现成功
- 退款扣回
- 冲正调整
- 人工调账

规则：

- 钱包余额只由钱包流水计算，不允许直接手改。
- 冻结、解冻、提现中都必须有独立流水。
- 人工调账必须有审批和操作日志。

---

## 6. 提现与打款

提现单字段：

- `withdraw_id`
- `wallet_id`
- `owner_type`
- `owner_id`
- `amount`
- `receive_account_snapshot`
- `status`
- `reviewer_id`
- `review_note`

打款流水字段：

- `transfer_id`
- `withdraw_id`
- `channel`
- `amount`
- `channel_trade_no`
- `status`
- `callback_event_id`

规则：

- 发起提现时冻结可提现余额。
- 审核驳回时解冻。
- 打款成功后扣减提现中余额，生成已提现流水。
- 打款失败时退回可提现余额。

---

## 7. 退款和冲正

退款适用：

- 客户取消。
- 审核拒绝后退款。
- 售后退款。
- 重复支付。

冲正适用：

- 分账错误。
- 钱包入账错误。
- 通道对账差异。
- 人工复核后反向调整。

要求：

- 退款影响支付流水、账单、订单状态。
- 冲正影响分账流水和钱包流水。
- 已提现金额不能直接扣回，需生成负向余额或后续抵扣规则。
- 退款和冲正必须二次确认和权限控制。

---

## 8. 三边对账

每日日终对账：

| 对账边 | 内容 |
|---|---|
| 通道边 | 支付、退款、打款通道流水 |
| 业务边 | 订单、账单、分账单、提现单 |
| 钱包边 | 钱包流水和余额 |

平账条件：

- 通道实收 = 支付流水成功金额。
- 支付流水成功金额 = 账单已付金额。
- 可分账金额 = 分账流水总金额。
- 分账成功金额 = 钱包入账金额 + 平台收入。
- 提现成功金额 = 钱包出账金额 = 打款成功金额。

---

## 9. 异常处理

| 异常 | 处理 |
|---|---|
| 通道有款，系统无支付流水 | 人工关联或补单 |
| 系统支付成功，通道无款 | 冻结订单并禁止分账 |
| 分账不平 | 进入分账异常队列 |
| 钱包余额不平 | 禁止提现，财务复核 |
| 打款失败 | 解冻提现金额，记录失败 |
| 退款成功但账单未更新 | 重放回调或人工补偿 |

---

## 10. 验收标准

1. 任一金额变动都有流水。
2. 任一流水能追到来源业务对象。
3. 退款和冲正不会删除原始流水。
4. 日终对账能定位差异来源。
5. 钱包余额可由流水重算。
