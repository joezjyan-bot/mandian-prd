# 满点租赁系统重构 V0.2.1 开发冻结版总 PRD

> **版本**: V0.2.1
> **冻结日期**: 2026-05-27
> **定位**: 开发唯一总入口
> **范围**: V1 长租主营业务

---

## 1. 冻结结论

V0.2.1 只冻结长租主营链路,目标是让开发团队可以按一套稳定口径实现系统一期。

体验租/短租作为 V2 独立业务线保留文档和架构预留,但不进入 V1 开发范围、接口范围、表结构强约束和验收矩阵。任何体验租页面、短租库存、短租保证金、短租取还车流程均不得阻塞 V1 长租开发。

---

## 2. V1 业务范围

### 2.1 V1 必做

- 商家入驻、店铺审核、商家资料和员工权限。
- 商品管理、规格矩阵、指导价、长租套餐配置。
- 办单助手:商家订单、联营订单、平台订单三类长租订单。
- C 端扫码下单、订单确认、资料补充、实名认证、信用/信息授权、合同签署、公证、保证金/首期支付、代扣绑卡。
- 运营端订单审核、资方分配、合同模板选择、发货履约、签收确认、账单、退款、撤单、留购/提前结清。
- 商家端订单履约视图、我的钱包、提现、员工权限。
- 财务管理:平台代收、订单结算款、门店结算账户、资方内部台账、退款冲正、提现审核、对账。
- 发货签收:V1 默认电子确认单;AI 问答确认保留配置和状态设计,可作为后续增强。
- 中控台对接、监管锁能力、操作日志、权限矩阵、数据脱敏。

### 2.2 V1 不做

- 体验租/短租正式上线。
- 短租车辆库存正式调度。
- 短租取车、还车、续租、超时计费、短租保证金扣款。
- 独立资方端。
- 资方在 C 端展示。
- C 端展示合作模式、资金来源、风控结论、平台抽佣、分账。

## 3. 核心业务定义

### 3.1 三类长租订单

| 类型 | 底层枚举 | 中文展示 | V1 定义 |
|---|---|---|---|
| 商家订单 | `self_operate` | 商家订单 | 商家自营,商家负责审核和履约,平台提供系统、支付、合同、账单和结算工具 |
| 联营订单 | `joint_venture` | 联营订单 | 商家与平台/资方按规则共同参与,运营端统一审核和资金控制 |
| 平台订单 | `receivables_assignment` | 平台订单 | 平台主导审核、分配履约门店和资金规则,商家仅承担履约视图内动作 |

### 3.2 合同主线

默认合同结构为三方租赁合同:

- 甲方:商家/出租方
- 乙方:客户
- 丙方:平台

资方不出现在客户合同和 C 端。资方、资金来源、放款条件、资方台账仅作为运营端和财务端内部配置。

如后续存在银行赊销类资金来源,只通过 `funding_source + terminology_template_id` 控制内部流程和客户侧按钮话术,不得让同一客户订单同时呈现"租赁"和"借款"双重语义。

### 3.3 资金主线

平台承担代收、记账、结算和工具服务角色,不对客户履约兜底。

1. 客户支付进入平台对公/通道账户。
2. 平台按规则扣平台综合服务费。
3. 属于商家的资金穿透至通联备付金子台账,形成门店结算账户余额。
4. 联营/平台订单涉及资方的部分进入内部资方台账,不在 C 端展示。
5. 商家通过我的钱包申请提现。

### 3.4 提现规则

- 普通提现单笔范围:500-20000 元。
- 每日最多 3 次。
- 超过 20000 元或命中异常规则时进入人工审核,不直接提交代付通道。
- 手续费:千分之 2 + 0.5 元,后台可配置。
- 商家财务人员可以被授权查看钱包和发起提现,但不能修改结算账户。
- 修改结算账户仅商家老板可提交,必须经平台审核、二次确认和完整日志。

## 5. C 端展示原则

C 端只展示客户能理解和能操作的内容:

- 商品、套餐、保证金、每期应付、合同、收货、履约中、账单支付、留购、归还、客服。
- 不展示资方、资金来源、合作模式、风控结论、分账、平台抽佣、资方专属模板。
- 审核类统一展示为"审核中、平台处理中、待补充资料、待签约"等中性状态。
- 信用/信息授权页面只说明授权用途和隐私保护,不展示内部评分和资方规则。

## 6. 长租主链路

### 6.1 商家办单

1. 商家进入办单助手。
2. 选择商品、规格、套餐、订单周期、设备价、保证金、首期应付、增值服务。
3. 商家订单可由商家自主审核;联营/平台订单由运营端主控审核。
4. 分红/资方收益类字段不展示给客户。
5. 客户扫码进入 C 端订单确认页。

### 6.2 客户下单

1. 订单确认。
2. 资料补充。
3. 实名认证和人脸。
4. 信用/信息授权。
5. 业务审核。
6. 合同签署,可按后台模板选择默认合同或指定合同模板。
7. 公证,按配置决定是否需要。
8. 保证金/首期支付。
9. 代扣绑卡。
10. 发货或门店交付。
11. 客户确认收货。
12. 进入履约中。

### 6.3 发货和签收

发货时必须选择发货方式:

- 门店自提
- 顺丰物流
- 其他物流
- 平台配置的其他方式

发货时填写设备识别码,例如 IMEI / SN / VIN / 车架号。长租不提前占用库存。

客户确认收货支持两种方式:

1. 电子确认单:V1 默认方案。
2. AI 问答确认:保留配置和状态设计,后续增强。

## 7. 设备边界

### 7.1 长租设备档案

长租不走库存状态机。长租只在发货/交付时填写设备识别码,客户确认收货后生成设备档案。

长租设备档案用于:

- 订单绑定
- 设备识别码留存
- 发货/签收证据
- 监管锁状态展示
- 归还、留购、赎回、售后记录

### 7.2 短租车辆库存

短租车辆库存只服务 V2 体验租业务线,用于真实车辆的可租、预约、取车、在用、还车、维保、下架等状态流转。

V1 长租开发不得依赖短租库存。

## 8. 权限原则

### 8.1 商家侧

| 操作 | 商家老板 | 管理员 | 财务人员 | 店长 | 普通员工 |
|---|---|---|---|---|---|
| 办单助手 | 是 | 可授权 | 否 | 可授权 | 可授权 |
| 商家订单审核 | 是 | 可授权 | 否 | 可授权 | 否 |
| 查看我的钱包 | 是 | 可授权 | 可授权 | 否/可选 | 否 |
| 发起提现 | 是 | 可授权 | 可授权 | 否 | 否 |
| 修改结算账户 | 仅老板 | 否 | 否 | 否 | 否 |
| 修改员工权限 | 是 | 可授权 | 否 | 否 | 否 |

### 8.2 运营侧

高危操作必须二次确认和完整日志:

- 提现审核和代付
- 退款执行
- 后台余额调整
- 修改结算账户审核
- 合同模板启停
- 抽佣/服务费率调整
- 关闭订单
- 修改订单金额
- 客户加黑名单
- 监管锁锁机/解锁

## 10. 开发冻结验收标准

- C 端正式页面不暴露资方、合作模式、资金来源、风控字眼。
- 三类长租订单的下单、审核、合同、付款、发货、签收、履约、账单、留购/提前结清链路闭环。
- 资金链路从客户支付到商家钱包、资方内部台账、提现、退款冲正有完整记录。
- 长租不依赖短租库存状态。
- 商家财务权限和结算账户修改权限分离。
- README 指向本冻结版作为第一开发入口。
