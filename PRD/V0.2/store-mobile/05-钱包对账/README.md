# 门店移动端：钱包与对账

> 重构决策：商家/门店不作为资方，先取消门店手机端 `配资充值`。旧系统实测到的配资额度、在线充值和配资明细只作为旧逻辑记录，新系统门店钱包以订单收益、分账收入、佣金/奖励、提现、对账和变动明细为主。

## 入口与路由

| 页面 | 入口 | 路由 |
|---|---|---|
| 我的钱包 | 工作台 `我的钱包` | `/pages/shopManage/wallet` |
| 余额/佣金/变动明细 | 钱包内明细入口 | `/pages/shopManage/walletDetail` |
| 提现明细 | 钱包 `提现明细` | `/pages/shopManage/withdrawList` |
| 对账单 | 工作台 `对账单` | `/pages/shopManage/reconciliation` |

我的钱包

## 点击反馈

| 控件 | 点击结果 | 本次动作 |
|---|---|---|
| 分成余额提现 | 余额为 0 时 Toast：当前暂无可提现余额 | 未进入提现提交 |
| 佣金余额提现 | 余额为 0 时 Toast：当前暂无可提现余额 | 未进入提现提交 |
| 在线充值 | 旧系统打开底部弹窗 `配资余额充值` | 新系统取消该入口 |
| 余额明细 | 进入明细页，Tab 默认或切到对应账户 | 只读查看 |
| 佣金明细 | 进入明细页，展示佣金相关流水 | 只读查看 |
| 提现明细 | 进入提现记录页 | 只读查看 |
| 变动明细 | 进入钱包变动流水 | 只读查看 |

## 旧系统充值弹窗

```text
配资余额充值
├─ 当前配比：充值金额将按比例放大入账
├─ 充值金额输入框
└─ 确认充值
```

新系统不保留普通门店配资充值。若未来允许商户作为资方，应放到资方管理或出资商户身份，不放在普通门店钱包。

### 对账状态

| Tab | 实测结果 |
|---|---|
| 全部 | 展示收入类记录 |
| 收入 | 展示收入类记录 |
| 支出 | 空状态 `暂无对账记录` |

## 资金权限

| 角色 | 可看余额 | 可提现 | 可充值 | 可看流水 |
|---|---|---|---|---|
| 门店老板 | 是 | 是 | 否 | 是 |
| 店长 | 可配置 | 否 | 否 | 可配置 |
| 办单员 | 否 | 否 | 否 | 否 |

## 风险控制

1. 提现必须校验收款账号实名、店铺状态、可提现余额、冻结金额和风控限制。
2. 提现、分账入账、平台抽佣扣除都要有资金流水和审计日志。
3. 员工账号默认看不到钱包入口。
4. 对账单导出如后续增加，应进入统一导出中心并脱敏。
