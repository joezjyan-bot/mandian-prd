# 订单关闭与系统费用

> **⚠️ V0.2 Stage 6 同步修订(2026-05-27)v1.1**:
> - 本文档仅保留历史参考；订单关闭、退款和售后闭环以 `05_订单关闭退款与售后.md` v1.2 为准。
> - 本次仅同步展示用语:商家订单 / 联营订单 / 平台订单 / 履约中 / 逾期费用;底层字段、接口、枚举不变。

> **来源**: P0-1 / P1-3(修订)
> **核心**: 以"合同是否签署"为分界

## 1. 页面定位

运营端 · 订单管理 · 订单关闭。订单关闭后的资金记账、平台费用产生、各主体责任划分。

## 2. 关闭规则总表

| 阶段 | 商家可关 | 费用 |
|---|---|---|
| 未签合同前 | ✅ | 免费 |
| 已签合同 + 7天内 | ✅ | 订单总租金 × 1.5%(从门店结算账户扣) |
| 订单生成 > 7天 | ❌ | 仅客服可关,按实际情况判定 |

## 3. 关闭发起人

### 门店关闭(门店手机端 / 商家 PC)
仅商家订单可门店关,联营/平台订单不允许门店直接关

### 客户关闭(C 端)
仅未签合同前,在订单详情"取消订单"

### 客服关闭(运营端)
任何阶段都可关;需责任划分 + 安排退款单

## 4. 关闭流程

### 4.1 门店关闭(已签合同 7天内)

```
1. 门店在订单详情点击"关闭订单"
2. 弹窗:"本订单已签合同,关闭需从您的结算账户扣减 ¥XXX(订单总租金 × 1.5%),是否确认?"
3. 二次确认
4. 提交后:
   - 如门店结算账户余额不足:提示补足,不进关单流程
   - 如足:扣减 → 合同作废(调 e签宝) → 设备释放 → 状态变 closed
   - 同时发起退款单:退还客户已付金额(抽佣部分全退)
```

### 4.2 未签合同关闭
免费,直接关闭:
- 会取消订单中间所有临时资源(预保留设备释放、临时锁的资金回回客户)
- 不产生任何费用

### 4.3 客服关闭
客服点击"关闭订单":
- 选择关闭原因(下拉)
- 责任方(门店/平台/资方/客户/混合)
- 是否需退款:选项
- 提交后创建退款单(走退款单与责任划分流程)

## 5. 1.5% 费用计算

费用 = 订单总租金 × fee_rate_close_order(默 1.5%)
- 订单总租金 = 首期 + 后续期 × 期数
- 不含押金
- 不含留购价格

扣减流水备注:`订单 XX 提前关闭费用`

## 6. 子关联动作

关闭后自动:
- 合同作废(发 e签宝/法大大作废申请)
- 设备释放(锁定状态 → 在库可租)
- 监管锁解除(如已下)
- 账单作废(后续未生成账单取消)
- 产生退款单(如需)
- C 端推送通知

## 7. 状态流转

任何状态 → closed
- closed 后退款单状态控制是否可计入"完结"
- 退款未完业务锁仍锁定

## 8. 权限 & 日志

- 任何关闭:二次确认 + 完整日志
- 关闭人/原因/费用/退款单快照都记

## 9. 数据模型

- `order.status` 增加 closed 状态
- `order.closed_at` / `closed_by` / `close_reason` / `close_fee_amount`
- 关联 `refund_request`(如需)
