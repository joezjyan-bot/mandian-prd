# V0.2.2 法务文本与签收话术定稿清单

> **资金/法务高风险**:本文用于法务、产品、客服、研发共同核对客户侧文本。法务终审可以调整表达,但不得改变 V0.2.2 已确认的业务责任、状态机、数据字段、证据链和 C 端敏感字段隔离规则。

## 一、文本总原则

1. 客户侧、合同正文、客服话术不得出现资方、资金来源、合作模式、风控结论、内部审核备注、商家结算价、平台利润、返点规则。
2. 统一使用:订单总额、首期应付、每期应付、保证金、账单支付、履约中、留购、逾期费用。
3. 三类订单展示统一为:商家订单 / 联营订单 / 平台订单。
4. 监管锁、设备识别码、交付照片、签收确认、AI 问答、IM 聊天记录、支付记录均属于证据链。
5. 所有客户确认文本必须记录模板版本、展示时间、确认时间、客户身份、IP/设备信息、签署或勾选证据。

## 二、合同与协议文本清单

| 文本 | 使用场景 | 必含内容 | 禁止内容 | 证据归档 |
|---|---|---|---|---|
| 三方租赁合同 | 长租签约 | 甲方商家、乙方客户、丙方平台;设备、订单周期、费用、保证金、违约责任、留购规则 | 资方名称、资金来源、合作模式、借款表述 | 合同 PDF、签署流水、模板版本 |
| 平台服务协议 | 客户下单 / 商家入驻 | 平台代收、信息服务、客服、证据保全、数据授权 | 平台兜底承诺 | 协议版本、勾选记录 |
| 信用 / 信息授权书 | 资料授权 | 授权范围、用途、有效期、撤回方式 | “风控通过/拒绝”结论承诺 | 授权流水 |
| 电子签署授权 | 发起合同 | 电子签效力、签署流程、失败重发 | 不得承诺必然通过审核 | 电子签日志 |
| 公证服务说明 | 可选公证 | 公证用途、费用、材料范围、失败处理 | 不得夸大诉讼必胜 | 公证记录 |
| 体验租服务协议 | 短租下单 | 取还车、保证金、超时费、事故/违章/损坏/丢车责任、保险 | 不得隐藏扣款规则 | 协议确认记录 |
| IM 客服告知 | IM 会话开始 | 会话归档、服务用途、敏感信息提示 | 不得在聊天内承诺资金处理结果 | IM 会话归档 |

## 三、电子确认单默认文本

### 3.1 长租交付签收确认单

默认文案:

> 本人确认已收到本订单对应设备,设备型号、外观、主要配件及交付信息与订单页面展示一致。本人知悉订单周期、首期应付、每期应付、保证金、到期日、账单支付方式、逾期费用及留购规则。本人确认本次签收为本人真实意思表示,签收后订单进入履约中,后续按合同和账单进行履约。

必须展示字段:
- 订单号
- 商品 / 设备名称
- 规格 / 配置
- 设备识别码脱敏
- 订单周期
- 首期应付
- 每期应付
- 保证金
- 到期日
- 商家名称
- 签收方式
- 签收时间

### 3.2 短租取车确认单

默认文案:

> 本人确认已从指定门店取得本体验租订单车辆,并已核对车辆外观、车辆编号、租期、取还车时间、费用、保证金、超时费、保险/保障范围及还车要求。本人承诺按约使用并按时归还,如发生事故、违章、损坏、丢车或超时,将按协议及证据承担相应责任。

必须展示字段:
- 短租订单号
- 门店
- 车型
- 车辆识别码 / 车牌脱敏
- 租期
- 取车时间
- 预计还车时间
- 保证金
- 保障方案
- 验车照片入口

## 四、AI 问答确认默认问题库

> AI 问答 V1 是可配置确认方式。后台可配置默认使用电子确认单或 AI 问答,另一种可由客服手动发起。

| 问题编号 | 问题 | 合格答案 | 异常处理 |
|---|---|---|---|
| Q1 | 请确认是否本人办理本订单? | 是 / 本人办理 | 否或沉默进入人工复核 |
| Q2 | 请说出您本次办理的设备名称或车型。 | 与订单设备匹配 | 不匹配进入人工复核 |
| Q3 | 请确认您是否清楚订单周期和每期应付金额。 | 清楚 | 不清楚进入客服解释 |
| Q4 | 请确认您是否已收到设备并检查外观。 | 已收到 / 已检查 | 否进入交付异常 |
| Q5 | 请确认您是否知悉逾期会产生逾期费用。 | 知悉 | 不知悉进入客服解释 |
| Q6 | 短租订单请确认是否知悉按时还车和超时费规则。 | 知悉 | 不知悉进入客服解释 |

验收:
- 视频总时长建议 20-30 秒。
- 必须保存问题版本、客户回答、AI 识别文本、视频文件、异常标签。
- 异常回答不得自动通过签收。

## 五、保证金扣款通知

默认文案:

> 因本订单出现【事件类型】,系统根据订单协议、验收记录和相关证据生成保证金扣款建议。扣款金额为【金额】,责任方为【责任方】,证据包括【证据清单】。如您有异议,可在【申诉期限】内联系客服补充材料。

必须字段:
- 订单号
- 事件类型:超时 / 事故 / 违章 / 损坏 / 丢车 / 其他
- 扣款金额
- 责任方
- 证据清单
- 审批人
- 申诉入口

## 六、退款通知

默认文案:

> 您的退款申请已提交。退款金额、责任方和处理方式将根据订单类型、协议约定、支付记录和实际证据确认。商家订单由商家承担对应退款责任;联营订单 / 平台订单按线下协商方案执行。退款成功后,系统将同步更新订单、账单、钱包和相关流水。

必须字段:
- 退款单号
- 订单号
- 退款金额
- 退款原因
- 责任方
- 审批状态
- 原支付流水
- 预计到账说明

## 七、留购 / 提前结清话术模板

| 业务模式 | 按钮建议 | 弹窗主文案 | 备注 |
---|---|---|---|
| 租赁留购 | 留购 | 支付剩余留购金额后,设备所有权按合同约定转移。请确认剩余应付、保证金抵扣和实际支付金额。 | 不使用“留购价” |
| 银行赊销 / 内部资金来源模式 | 提前结清 | 支付剩余应付后,本订单将提前完成。请确认剩余应付、已付金额、保证金抵扣和实际支付金额。 | C 端不展示资金来源 |

## 八、法诉证据包文本

法诉导出确认文案:

> 本次导出将生成订单法诉证据包,包含订单基础信息、客户实名资料、合同/电子签记录、签收确认、AI 视频确认、交付照片、设备型号与编码、价值证明、监管锁记录、账单明细、支付记录、逾期记录、催收记录、IM 聊天记录、退款/冲正记录、操作日志和审计日志。导出将写入审计日志。

导出必须记录:
- 导出人
- 导出时间
- 导出用途
- 订单号
- 证据版本
- 文件 hash
- 下载次数
- 后续补证据记录

## 九、法务终审验收标准

1. 客户侧文本无资方、资金来源、合作模式、风控结论等敏感词。
2. 合同、确认单、AI 问答、保证金扣款、退款通知均有模板版本。
3. 费用口径使用订单总额、首期应付、每期应付、保证金、逾期费用、留购。
4. 证据链字段能与法诉证据包一一对应。
5. 法务修改只调整文本表达,不得跳过产品和技术直接改变业务状态、费用算法或责任规则。
