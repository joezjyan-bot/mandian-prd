## 6. 审核队列(v2.0 重构)

| 队列 | 进入条件 | 主操作 |
|---|---|---|
| 待接单 | 订单提交后未分配审核人员 | 接单(系统自动标记做单客服) |
| 待资料审核 | 客户资料已提交 | 查看资料、调整套餐、补资料、拒绝 |
| **待风控** ⭐ | 联营/平台订单完成资料审核 | 调用风控接口、查看报告、写审核意见 |
| **待分配资方** ⭐ v2.0 | 风控通过的联营/平台订单 | 系统推荐资方 + 运营审核确认 |
| **待分配门店** | 平台订单资方分配后未分配履约门店 | 搜索匹配门店、工号二次确认 |
| 待补资料 | 客户或商家需补资料 | 查看补充状态、催办、重新审核 |
| 待签约支付 | 已"确认订单进入支付"但合同/支付未完成 | 系统自动串行;客服可触发银行卡代扣 |
| **退款工单**(财务)⭐ v2.0 | 撤单已发起,等门店线下转账 | 确认转账 / 触发客户退款 / 触发资方退款 / 完成工单 |
| 异常队列 | 黑名单、回调失败、支付不匹配、资料冲突、长时间不签合同(7 天)、**资方放款失败** | 主管复核、重试回调、转人工 |

## 8. 审核流程(v2.0 重写,加入资方分配环节)⭐

```
订单提交
  ↓
进入待审核队列
  ↓
接单(系统自动标记"做单客服 XXX")
  ↓
查看客户资料 + 商品价格方案
  ↓
[阶段 1] 资料审核
  ├─ 通过 → 进入风控(联营/平台订单)
  └─ 不通过 → 驳回 / 要求补资料
  ↓
[阶段 2] 风控审批(联营/平台订单必经)⭐ v2.0 新增
  ├─ 调用风控接口 / 查看报告
  ├─ 判断:客户信用 + 设备价 + 历史订单
  └─ 风控通过 → 进入资方分配
  ↓
[阶段 3] 资方分配(联营/平台订单必经)⭐ v2.0 核心
  ├─ 商家订单:无需分配,跳过
  ├─ 联营订单:
  │   - 系统默认推荐"平台自营资金"(MVP 阶段)
  │   - 检查平台自营授信额度 → 充足 → 锁定
  │   - 加载平台自营对应的合同模板 + 话术模板
  └─ 平台订单(MVP 含蓝海银行 + 平台自营):
      ↓
      系统按风控规则推荐资方(详见 `资方管理/03`):
        ├─ 优质订单 + 蓝海银行授信余额够 → 推荐蓝海银行
        ├─ 不符合银行标准 + 平台愿接 → 推荐平台自营
        └─ 都不符合 → 提示驳回订单
      ↓
      运营审核:
        ├─ 接受系统推荐 → 锁定
        ├─ 调整推荐(主管权限)→ 手动选择资方
        └─ 驳回订单
      ↓
      资方锁定后系统自动:
        ├─ order.funding_source = funder_blue_ocean / platform_self
        ├─ order.funder_id = 资方 ID
        ├─ order.contract_template_id = 资方对应合同模板
        ├─ order.terminology_template_id = 资方对应话术模板
        └─ 资方授信额度预占用(分配时不实际扣减,监管锁回调时扣减)
  ↓
[阶段 4] 判断:是否需要调整套餐?
  ├─ 商家订单 → 不开放;**仅可驳回**
  └─ 联营/平台订单 → 客服可点 [调整套餐]
       ↓
       弹出"类办单助手"界面:
         - 平台订单必须分配履约门店
         - 设备价 / 首付 / 期数 / 月供 / 留购价 / 保证金
         - 增值服务多选
         - 出资比例(联营订单)
         - 首期一笔支付汇总
         - 各期账单 + 留购价(预计算)
       ↓
       [生成办单图片]
         - 系统生成订单图
         - 客服下载图片
         - 通过任意渠道(微信/钉钉)发给商家
         - 商家与客户确认费用
       ↓
       [提交]
         - 订单信息写库
         - 状态仍为 PENDING_REVIEW
         - 等待客户在 IM 回复"确认"
  ↓
[阶段 5] 客户在 IM 回复"确认" → 客服肉眼判定
  ↓
[确认订单进入支付]
  ↓
订单状态:PENDING_REVIEW → PENDING_PAYMENT
后续:系统自动串行(支付 → 合同(按资方模板)→ 代扣 → 待发货 → 待收货 → 监管锁回调 → 订单结算款穿透)
```

**关键约束**:
- 联营/平台订单**未通过风控**,不可进入资方分配
- 联营/平台订单**未分配资方**,不可点"确认订单进入支付"
- 平台订单**未分配门店**,不可点"确认订单进入支付"
- 客户**未在 IM 确认**,不可点"确认订单进入支付"

完整 6 阶段流程详见 `08_长租订单全生命周期与客服操作.md` v1.4。

### 9.3 平台订单的资方分配(系统推荐 + 运营确认)⭐ v2.0 核心

详细推荐算法见 `资方管理/03_风控规则与资方分配.md`。本节摘要业务流程:

#### 9.3.1 系统按风控规则推荐资方

```python
def recommend_funder(order):
    """系统资方推荐算法"""
    
    # 风控分数
    customer_score = order.customer.credit_score
    
    # 优先级 1:蓝海银行(若符合赊销标准)
    if (customer_score >= 750 
        and order.device_price <= ¥20,000
        and Funder.blue_ocean.credit_limit_left >= order.device_price):
        return Funder.blue_ocean
    
    # 优先级 2:其他银行/信托资方(未来扩展)
    
    # 优先级 3:平台自营(降级方案)
    if Funder.platform_self.credit_limit_left >= order.device_price:
        return Funder.platform_self
    
    # 都不符合 → 驳回
    return None
```

```
│       话术模板:标准租赁话术                                   │
│       平台收益:应收账款受让溢价                              │
│                                                              │
│ 操作人:运营客服 ID=123                                       │
│   [驳回订单]      [按推荐分配]      [手动选择资方]            │
└──────────────────────────────────────────────────────────┘
```

#### 9.3.3 资方分配后的系统动作

```
分配确认后,系统自动:
  ① 锁定 order.funding_source / order.funder_id
  ② 加载 order.contract_template_id(继承资方)
  ③ 加载 order.terminology_template_id(继承资方)
  ④ 资方授信额度"预占用"(实际扣减发生在监管锁回调时)
  ⑤ 在订单详情顶部信息条展示资方分配结果
  ⑥ 进入下一步:分配履约门店(若平台订单)
```

### 9.4 资方授信额度管理(v2.0 新增)⭐

详见 `资方管理/01_资方主体与授信管理.md`。本节摘要:

```
资方授信额度时序:

T0 订单分配资方 → 授信额度【预占用】
   (实际未扣减,只是标记预期占用)

T1 客户签合同 / 付首期 → 仍处于"预占用"状态

T2 监管锁回调 → 授信额度【实际扣减】
   funder.credit_limit_used += amount
   funder.credit_limit_left -= amount
   同步更新 funder_receivable_ledger

T3 客户每月账单支付 → 资方应收余额【部分回收】
   funder_receivable_ledger.collected_amount += amount

T4 订单完成(全部还清) → 授信额度【释放】
   funder.credit_limit_used -= original_amount
   funder.credit_limit_left += original_amount
   funder_receivable_ledger.outstanding_balance = 0

T5 订单撤单(已分账) → 走退款工单(详见 10 §3)
   工单完成后:授信额度【按 refunded_amount 还原】
```

### 9.5 资方变更

| 节点 | 是否允许变更 | 规则 |
|---|---|---|
| 合同发起前 | 允许 | 客服主管 / 运营主管权限可操作 |
| 合同已发起未签 | 谨慎允许 | 需要主管批准 + 重新发起合同(按新资方合同模板)|
| 合同已签署 | 原则不允许 | 如必须变更,需要主管审批 + **发起补充合同**(详见 10 文档) |
| 已支付/已分账 | 不允许普通变更 | 走冲正、退款工单或财务调整流程 |

**关键**:更换资方会**同时切换合同模板 / 话术模板**,客户需重新签署或确认。

### 9.6 资方推荐规则维护

平台运营在 `资方管理/03` 中维护风控规则:

```sql
risk_rule
- rule_id
- rule_name              -- 规则名称
- funder_id              -- 关联资方
- priority               -- 优先级(数字越小越优先)
- min_credit_score       -- 客户最低信用分
- max_device_price       -- 最高设备价
- min_funder_credit_left -- 资方最低剩余额度
- applicable_modes       -- 适用合作模式
- status
```

规则匹配按优先级顺序,首个满足条件的资方即为推荐结果。

## 13. 异常处理(v2.0 扩充)

| 异常 | 处理 |
|---|---|
| 资方授信额度不足 | 系统阻止分配,提示更换资方;主管可调整授信或驳回订单 |
| 资方被停用 | 已分配未签约订单进入异常队列 |
| 平台订单未分配门店 | 阻止"确认订单进入支付"按钮 |
| 门店收款账户被冻结 | 系统阻止该门店出现在候选列表 |
| 风控报告未返回 | 可重试或人工暂挂 |
| 客户补资料超时 | 提醒商家/客户,超时后可关闭 |
| 客户长时间不签合同 | 24h 催办 + 7 天进异常队列,主管处理 |
| 支付已完成但合同未签 | 禁止发货,提示补签或退款处理 |
| 合同已签但支付失败 | 保留合同状态,订单待支付 |
| 黑名单命中 | 必须主管复核或强制填写审核意见 |
| 客服调整套餐后客户未确认 | 禁止"确认订单进入支付" |
| 代扣签约失败 | 默认切换到银行卡代扣 → 主管可批准跳过 |
| 设备编码填错 | 不可直接修改,必须发起补充合同 |
| 收货验收 二选一 | 必须二选一通过,长时间僵持由主管标记或撤单 |
| 商家订单异常介入 | 只能驳回,不可改价 |
| **资方放款失败(蓝海银行)** ⭐ v2.0 | 财务异常队列 + 客服联系银行 + 可能切换为平台自营 |
| **资方授信达到临界值** ⭐ v2.0 | 主管告警,建议运营调整资方授信或暂停接单 |
