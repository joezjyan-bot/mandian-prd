# 体验租管理 · 02 订单管理

> 运营端体验租订单的统一查询、详情查看、代操作、异常处置入口。
> 与 `modules/短租业务/` 系列文档的关系:本文档定义**运营端的页面 UI 与操作动作**;数据模型、状态机、业务规则在短租业务目录定义。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 体验租订单管理 |
| 入口路径 | 运营端 > 体验租管理 > 02 订单管理 |
| 使用角色 | 体验租运营、体验租运营主管、客服(只读) |
| 核心目标 | 一站式查询所有短租订单;查看完整生命周期(订单 + 资金 + 控车 + 续租链);代客户/代商家执行特殊动作 |

### 2.2 筛选字段

| 字段 | 类型 | 备注 |
|---|---|---|
| 订单号 | 模糊搜索 | 支持完整 / 后 6 位 |
| 客户手机号 | 精确匹配 | 加密查询 |
| 骑行人证件号 | 精确匹配 | 加密查询(需权限) |
| 状态 | 多选 | 短租状态枚举(SR_*)|
| 业务模式 | 单选 | 3 种经营模式 |
| 城市 | 多选 | 已开通城市 |
| 商家 | 搜索 + 多选 | - |
| 门店 | 搜索 + 多选 | 受商家筛选影响 |
| 车型(标准名)| 搜索 + 多选 | - |
| 创建时间 | 日期段 | - |
| 取车时间 | 日期段 | rent_start_at |
| 还车时间 | 日期段 | return_at |
| 逾期标记 | 单选 | 全部 / 已逾期 / 长时间逾期(>24h)|
| 争议标记 | 单选 | 全部 / 有争议 / 无争议 |
| 续租链 | 单选 | 全部 / 首租 / 续租 / 链中非活跃 |

### 2.3 列表字段

| 列 | 说明 |
|---|---|
| 选择 | 复选框,支持批量操作 |
| 订单号 | 点击进详情;链中非活跃单显示链中位置(如"第 2 期")|
| 客户 | 手机号脱敏(138\*\*\*\*7755)|
| 骑行人 | 姓名脱敏(若与客户不同)|
| 车型 | 商品标准名 + 车型编号 |
| 门店 | 履约门店 + 城市 |
| 业务模式 | 标签:门店自营 / 平台自营 / 平台货源-门店履约 |
| 租期 | 套餐 + 应还时间(逾期红字)|
| 金额 | 租金合计 |
| 押金状态 | 实付/免押/预授权 + 当前金额 |
| 状态 | 短租主状态(带颜色)|
| 创建时间 | - |
| 操作 | 详情 / 代操作菜单(权限控制)|

### 2.4 排序

默认按创建时间倒序;可点列头切换:取车时间 / 应还时间 / 金额。

### 2.5 批量操作

| 动作 | 权限 |
|---|---|
| 批量导出 | 全部能看到的运营 |
| 批量标记关注 | 体验租运营 |
| 批量发推送通知客户 | 体验租运营主管 |

## 4. 代操作菜单

订单详情右上角"代操作 ▼"菜单(按权限显示):

| 动作 | 权限 | 二次确认 | 触发条件 |
|---|---|---|---|
| 代接单 | 体验租运营 | ✅ | SLA 超时门店未接 / 客户主动联系客服 |
| 代取车核销 | 体验租运营主管 | ✅ + 上传客户身份证明 | 门店核销码异常 / 取车流程异常 |
| 代还车确认 | 体验租运营主管 | ✅ + 上传还车凭证 | 客户已离开门店但未完成还车系统操作 |
| 代押金扣款发起 | 体验租运营主管 | ✅ + 完整证据 | 商家未及时处理 / 商家退出业务 |
| 强制关闭订单 | 平台主管 | ✅✅ + 退款方案 | 极端异常场景 |
| 强制锁车 | 体验租运营主管 | ✅ + 必填原因 | 见短租 08 §4 |
| 强制解锁 | 体验租运营主管 | ✅ + 必填原因 | 锁车后客户已处理 |
| 违约金免除 | 体验租运营主管 | ✅ + 必填原因 | 见短租 10 §2.5 |
| 客户黑名单加入 | 风控 | ✅✅ + 等级选择 | 严重违规 |

所有代操作**强制要求**:
- 必填操作原因
- 进入操作日志(`operation_log`)
- 客户侧推送通知(IM + 短信)
- 抄送对应门店(若有)

## 5. 异常订单处置

### 5.1 异常订单的来源

| 来源 | 触发 |
|---|---|
| 自动识别 | 系统定时扫描:SLA 超时 / 长时间未支付 / 押金状态异常 / 控车失败累计 / 续租链状态不一致 |
| 客服转入 | 客诉管理转交体验租 |
| 商家上报 | 商家在商家 PC 端上报 |
| 运营主动 | 工作台预警卡片处理 |

### 5.2 异常订单的处置入口

订单列表筛选"异常标记"为"是",或工作台直接跳转。

### 5.3 处置动作

详见 §4 代操作菜单。

## 7. 数据接口

| 接口 | 用途 |
|---|---|
| `GET /api/v1/admin/exp-rent/orders` | 列表(分页 + 筛选)|
| `GET /api/v1/admin/exp-rent/orders/:id` | 详情(基础信息 + 概览)|
| `GET /api/v1/admin/exp-rent/orders/:id/fund-flow` | 资金流 |
| `GET /api/v1/admin/exp-rent/orders/:id/control-log` | 控车日志(本订单)|
| `GET /api/v1/admin/exp-rent/orders/:id/chain` | 续租链 |
| `GET /api/v1/admin/exp-rent/orders/:id/audit-log` | 操作日志 |
| `GET /api/v1/admin/exp-rent/orders/:id/evidence` | 证据材料 |
| `POST /api/v1/admin/exp-rent/orders/:id/proxy-action` | 代操作(各种 action)|
| `POST /api/v1/admin/exp-rent/orders/export` | 导出 |
