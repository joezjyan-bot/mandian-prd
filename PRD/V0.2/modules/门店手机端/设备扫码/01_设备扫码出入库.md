# 设备扫码出入库

> 页面级 PRD 草案。
> 目标：门店手机端提供轻量扫码能力，让店员在办单、交付、归还、盘点时处理设备码，不接触钱包和后台复杂配置。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 设备扫码出入库 |
| 所属端 | 门店手机端 |
| 入口路径 | 首页 > 设备扫码 / 待办任务 > 发货/归还/盘点 |
| 使用角色 | 商家老板、店长、门店员工 |
| 核心目标 | 扫设备码完成出库、交付照片、归还验收、盘点和异常上报 |
| 关联模块 | 办单助手、订单待办、商家设备库存、运营端库存、附件中心 |

---

## 2. 核心口径

1. 门店手机端只做现场操作，不做复杂库存配置。
2. 员工账号可以扫码办单、出库、上传材料、验收，但不能看钱包和敏感财务。
3. 扫码必须绑定当前门店权限，不能扫其他门店设备完成操作。
4. 短租订单必须扫具体设备码；长租是否必扫按类目和链路配置。
5. 所有照片、扫码、验收结果进入订单详情和附件中心。

---

## 3. 首页入口

| 入口 | 说明 |
|---|---|
| 扫码出库 | 处理待交付订单 |
| 扫码归还 | 客户现场归还或快递到店验收 |
| 设备查询 | 扫设备码查看状态和最近订单 |
| 库存盘点 | 按任务扫描门店在库设备 |
| 异常上报 | 损坏、缺件、设备不一致、锁异常 |

员工看到的入口由商家老板配置。

---

## 4. 扫码出库

### 4.1 进入方式

| 方式 | 说明 |
|---|---|
| 待办进入 | 从订单待发货/待交付任务进入 |
| 直接扫码 | 扫设备码后选择可处理订单 |
| 办单助手进入 | 生成订单后直接进入交付 |

### 4.2 出库字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 订单号 | 只读 | 当前待交付订单 |
| 客户 | 脱敏展示 |
| 商品/规格 | 只读 |
| 设备码 | 扫码 | 必填 |
| 设备状态 | 自动展示 | 必须可出库 |
| 交付方式 | 选择 | 当面交付、配送、快递 |
| 设备照片 | 拍照 | 必填或按类目 |
| 人机/人车合照 | 拍照 | 车辆类建议必填 |
| 配件清单 | 勾选 | 电池、充电器、钥匙等 |
| 备注 | 文本 | 可选 |

### 4.3 校验

1. 设备码必须属于当前门店或订单指定发货方。
2. 设备状态必须为可租、已锁定给当前订单或待出库。
3. 商品和规格必须与订单一致，允许同类替换时需要权限。
4. 监管锁异常时阻断或提示复核，按类目配置。

---

## 5. 扫码归还

| 字段 | 类型 | 说明 |
|---|---|---|
| 设备码 | 扫码 | 必填 |
| 匹配订单 | 自动 | 当前出租订单 |
| 客户 | 脱敏展示 |
| 归还方式 | 选择 | 到店、快递、租后追回 |
| 外观照片 | 拍照 | 必填 |
| 配件检查 | 勾选 | 按出库清单 |
| 验收结果 | 选择 | 正常、损坏、缺件、设备不一致 |
| 损坏说明 | 文本 | 异常时必填 |
| 入库去向 | 选择 | 可租、维修、争议、下架 |

归还成功后：

1. 正常归还进入订单完成或待财务结算。
2. 异常归还进入售后/争议。
3. 设备按验收结果回到可租、维修或争议状态。

---

## 6. 设备查询

扫设备码可查看：

| 信息 | 说明 |
|---|---|
| 设备状态 | 在库、锁定、出租中、归还待验、维修中 |
| 所属门店 | 当前归属 |
| 关联商品 | 商品和规格 |
| 当前订单 | 如有，脱敏展示 |
| 最近操作 | 入库、出库、归还、维修 |
| 监管锁状态 | 如绑定 |

员工只能看操作所需信息，不展示采购价、收益、分账、客户完整信息。

---

## 7. 库存盘点

| 功能 | 说明 |
|---|---|
| 创建盘点任务 | 商家 PC 或运营端创建 |
| 扫码盘点 | 门店员工逐台扫描 |
| 盘点结果 | 正常、盘盈、盘亏、状态不一致 |
| 异常上报 | 备注和照片 |
| 盘点完成 | 提交商家老板或运营复核 |

盘点不直接改库存状态，必须由有权限角色确认盘点结果。

---

## 8. 异常上报

| 异常类型 | 说明 |
|---|---|
| 设备损坏 | 上传照片和说明 |
| 缺配件 | 勾选缺失配件 |
| 设备不一致 | 扫码设备和订单不匹配 |
| 设备找不到 | 盘点盘亏 |
| 监管锁异常 | 离线、无法锁机、无法解锁 |
| 客户拒签/拒还 | 关联订单备注 |

异常上报后进入商家端或运营端待办，不能只停留在手机端。

---

## 9. 操作日志

每次扫码必须记录：

| 字段 | 说明 |
|---|---|
| 操作人 | 老板、店长、员工 |
| 操作端 | 门店手机端 |
| 设备码 | 被操作设备 |
| 订单号 | 如有关联 |
| 操作类型 | 出库、归还、查询、盘点、异常 |
| 定位/门店 | 当前门店 |
| 照片附件 | 如有 |
| 结果 | 成功、失败、异常 |
