## 三、V0.2.2 关键收口项

1. 一期范围统一:安心用长租 + 体验租短租 V1 + IM 客服基础能力。
2. 三类订单展示统一:商家订单 / 联营订单 / 平台订单。
3. 内部枚举分层:技术可保留 legacy 或内部枚举,展示、客服、合同、财务报表必须用 V0.2.2 展示名。
4. 结算三条件:客户签收确认 + 监管锁显示已上锁 + 激活锁状态已激活。
5. 无监管锁品类替代验收:签收确认 + 设备识别码归档 + 交付证据完整;高价值订单叠加 AI 视频或人工复核。
6. 退款责任:商家订单商家承担;联营订单 / 平台订单线下协商;系统支持指定金额退款并记录责任方、审批人、凭证、冲正对象和流水。
7. 四账分离:订单业务账、支付流水账、钱包 / 商家账户账、财务总账 / 分录账。
8. 服务费:默认 2%,可配置;C 端不展示平台利润和商家结算价。
9. 内部资金来源:不得在 C 端、商家端、合同正文、客服话术中暴露。
10. 权限:高危资金动作、账单动作、监管锁动作、法诉导出必须拆权限、留痕、必要时二次确认或二人复核。
11. 页面级契约:字段、按钮、弹窗、禁用态、空状态、权限态已统一。
12. 供应商适配:支付、退款、电子签、公证、监管锁、保险、IM、通知、OCR/人脸、物流、三方催收 API 均有统一字段映射。
