# 订单列表与详情

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> **来源**: P1-5

## 1. 设计原则

- **列表简洁**:订单 + 商品 + 状态 + 金额 + 关键待办高亮
- **详情详尽**:按区块组织,客户自助操作充分

## 2. 订单列表

### 2.1 字段(每张订单卡片)
- 订单号(可点击)
- 商品图 + 商品名
- 当前状态标签(待付款 / 待审核 / 在租 / 已完成等)
- 总金额 + 已付摘要
- **关键待办高亮**(若有):红点 + 按钮"去签合同" / "去支付" / "确认收货" 等

### 2.2 列表筛选
- 状态:全部 / 进行中 / 已完成 / 异常
- 时间范围

### 2.3 不显示在列表
- 完整合同 / 完整账单 / 公证状态 / 操作日志(进详情看)

## 3. 订单详情

### 3.1 按区块组织
1. 订单基础信息(订单号 / 下单时间 / 状态 / 门店)
2. 商品信息(图 / 规格 / 套餐 / 租期)
3. 价格信息(保证金 / 首期 / 每期 / 留购价 / 总金额 / 已付/待付)
4. 客户资料(脱敏)
5. 实名认证状态
6. 风控授权 / 报告状态
7. 合同状态(签署进度 / 副本查看)
8. 公证状态(可选)
9. 代扣卡状态
10. 发货 / 收货信息
11. 账单与还款记录
12. 操作记录(客户视角简化)
13. 联系门店 / 联系平台客服

### 3.2 待办按钮(按状态动态)

| 状态 | 显示按钮 |
|---|---|
| 待付款 | 立即支付 / 申请关闭订单 |
| 待审核 | (无操作,显示"审核中") |
| 待补充资料 | 补充资料 / 联系客服 |
| 待实名 | 完成实名认证 |
| 待风控授权 | 完成风控授权签署 |
| 待合同签署 | 完成合同签署 |
| 待公证 | 完成公证签署 |
| 待绑卡 | 绑定代扣卡 |
| 待发货 | (等待门店发货) |
| 待收货 | 确认收货 |
| 在租中 | 查看账单 / 主动还款 / **申请留购** / 申请归还 / 申请续租 |
| 已逾期 | 立即还款 / 联系客服 |
| 已完成 | 查看完整记录 |
| 异常 | 联系客服 |

**全文统一**:"申请留购" → "申请留购"

### 3.3 通用按钮(全程可见)
- 联系门店(电话 + 地址)
- 联系平台客服(在线 + 电话)
- 查看合同副本(签署后)
- 查看操作记录

## 4. 数据模型

订单 + 子表(客户资料 / 合同 / 账单 / 代扣 / 操作日志等)
