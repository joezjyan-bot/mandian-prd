### 任务 P1:09 文档"1 小时自动取消"的边界场景补充

**修订文档**:`PRD/V0.2/modules/运营端/订单管理/09_C端订单状态与账单支付.md` v1.4

**修订位置**:§2.3 待签约详情页

**问题**:
当前的"1 小时自动取消"机制对**正在签约过程中**的客户不够友好。

**场景举例**:
- 客户点了 [签约] 按钮跳到 e 签宝,正在阅读 13 页合同
- 1 小时倒计时归零,订单自动取消
- 客户从 e 签宝回来后,发现订单不在了
- 客户重新下单 → 体验差 + 流程冗长

**修订要求**:

#### P1.1 倒计时暂停机制

| 场景 | 倒计时行为 |
|---|---|
| 客户在订单详情页停留 | 正常倒计时 |
| 客户已点 [签约] 跳到 e 签宝 | **暂停倒计时**,改为 e 签宝独立 SLA(默认 30 分钟) |
| 客户已点 [代扣签约] 跳到银行/支付宝 | **暂停倒计时**,改为代扣独立 SLA(默认 15 分钟) |
| 客户已点 [立即支付] 跳到支付通道 | **暂停倒计时**,改为支付独立 SLA(默认 10 分钟) |
| 客户从三方页面回到订单详情 | 倒计时恢复(基于原剩余时间) |

#### P1.2 客户主动延长机制

**设计**:
```
订单详情页倒计时显示:
   将在 23:45 后自动取消,请尽快完成签约
   [立即签约]  [延长 30 分钟]

客户点 [延长 30 分钟] → 倒计时 +30 分钟
   - 最多可延长 2 次(总共 +60 分钟)
   - 延长动作写日志
   - 延长后倒计时归零 → 订单自动取消(不可再延)
```

#### P1.3 接近超时的提醒

```
倒计时剩余 10 分钟时:
   - 弹窗提示"订单将在 10 分钟后自动取消,请尽快完成签约/支付"
   - 同时发送一条小程序订阅消息或短信(若客户授权)

倒计时剩余 3 分钟时:
   - 顶部红色横幅提示"订单即将取消"
   - 强烈建议客户立即操作
```

#### P1.4 倒计时归零后的边界情况

| 场景 | 处理 |
|---|---|
| 客户在 e 签宝刚签完合同但还没回调到平台 | 等待回调最多 5 分钟,期间订单不取消;5 分钟内回调成功 → 正常进入下一步;5 分钟内未回调 → 异常队列,客服处理 |
| 客户在支付通道刚支付成功但还没回调 | 同上 |
| 客户在 e 签宝点了 [拒签] | 订单进入 CLOSED,可重新下单 |
| 客户多端登录(支付宝 + 微信同时打开同一订单)| 两端共享倒计时,任一端操作另一端同步 |

#### P1.5 配置项

| 配置 | 默认值 | 可配置 |
|---|---|---|
| 待签约自动取消时长 | 60 分钟 | 是 |
| e 签宝独立 SLA | 30 分钟 | 是 |
| 代扣独立 SLA | 15 分钟 | 是 |
| 支付独立 SLA | 10 分钟 | 是 |
| 允许客户延长次数 | 2 次 | 是 |
| 单次延长时长 | 30 分钟 | 是 |
| 第一次提醒时间 | 剩余 10 分钟 | 是 |
| 第二次提醒时间 | 剩余 3 分钟 | 是 |
| 回调容错窗口 | 5 分钟 | 是 |

#### P1.6 数据模型

```sql
order_countdown
- countdown_id          bigint PK
- order_id              bigint FK
- countdown_type        enum -- pending_sign / pending_pay / pending_auto_debit
- started_at            datetime
- expires_at            datetime
- paused_at             datetime -- 跳到三方页面时记录
- pause_reason          enum -- contract_signing / payment_processing / auto_debit_signing
- resumed_at            datetime
- extended_times        int -- 客户延长次数
- extended_total_minutes int
- final_status          enum -- completed / expired / extended / abandoned
- created_at / updated_at
```

### 任务 P2:08 文档"商家改单"的客户重新确认时效

**修订文档**:`PRD/V0.2/modules/运营端/订单管理/08_长租订单全生命周期与客服操作.md` v1.7

**修订位置**:§3.1 商家订单合同签署前字段修改

**问题**:
当前 §3.1 说"商家改了订单后客户要重新确认才能继续",但**没说时效** — 如果客户多久不确认就作废?

**修订要求**:

#### P2.1 客户重新确认的时效

```
商家修改订单后
   ↓
系统状态:waiting_customer_reconfirm
   ↓
推送给客户:小程序订阅消息 / 短信
   ↓
客户确认时效:
   ├─ ≤ 24 小时:正常确认
   ├─ 24-72 小时:订单变"过期",但可恢复(客户重新点确认后,商家需再次重新确认订单)
   └─ > 72 小时:订单自动关闭(CLOSED),需重新下单
```

#### P2.2 客户看到的差异展示

当客户重新打开订单详情页时,顶部展示"订单已变更"横幅:

```
┌─────────────────────────────────────────────┐
│ ⚠️ 订单内容已由商家更新                       │
│ 请确认更新后的内容并选择继续办理             │
│                                              │
│ [查看变更详情]  [确认继续]  [取消订单]        │
└─────────────────────────────────────────────┘
```

点 [查看变更详情] 显示对比:

```
变更明细:
┌────────────────────────────────────────┐
│ 商品                                    │
│   原:iPhone 15 Pro 256G                │
│   新:iPhone 15 Pro Max 256G            │
├────────────────────────────────────────┤
│ 期数                                    │
│   原:12 期                             │
│   新:24 期                             │
├────────────────────────────────────────┤
│ 订单总额                                │
│   原:¥6,000                            │
│   新:¥9,000                            │
├────────────────────────────────────────┤
│ 每期应付                                │
│   原:¥500/月                           │
│   新:¥375/月                           │
└────────────────────────────────────────┘

变更说明:商家更换了商品规格,期数延长,月供下降

[取消订单]    [确认继续]
```

#### P2.3 客户不在线时的处理

| 场景 | 处理 |
|---|---|
| 客户已退出小程序 | 通过订阅消息(微信)/ 小程序通知(支付宝)/ 短信(若授权)推送 |
| 客户没有授权订阅消息 | 短信通知(必选) |
| 客户已删除小程序 | 短信通知 |
| 客户未读消息超 48 小时 | 客服外呼电话(可选) |

#### P2.4 商家撤回修改

如果客户长时间未确认,商家也可主动撤回修改:

```
商家在订单详情页:
   - 看到 "客户未确认变更,已等待 XX 小时"
   - 可点 [撤回修改] → 订单恢复到修改前状态
   - 写日志(撤回原因可选填)
```

#### P2.5 数据模型扩展

```sql
-- 扩展 order_change_log
order_change_log
- log_id          bigint PK
- order_id        bigint FK
- ...(同 v1.6)
- customer_confirm_status enum -- pending / confirmed / expired / abandoned / withdrew_by_merchant
- customer_confirm_at    datetime
- expires_at      datetime -- 默认 +72 小时
- withdrew_at     datetime -- 商家撤回时间
- withdrew_reason text
```

#### P2.6 在权限矩阵 §11.1 新增

```
| 商家撤回未确认的修改 | ✅ | 可配置 | ❌ | ❌ |
```
