# 操作手册总模板

> 所属：培训交付
> 目标：后续产品文档定稿后，按无界租操作文档风格输出可培训、可截图、可交付的操作手册。

---

## 1. 输出原则

1. 每个手册必须有完整页面截图，不只写文字。
2. 每一步操作要有箭头、红框、按钮名称和路径。
3. 敏感生产数据必须打码。
4. 高风险动作只说明流程，不使用真实订单点确认。
5. 手册同时服务内部客服培训和外部商户培训。

---

## 2. 单篇手册结构

每篇操作手册统一结构：

1. 适用对象
2. 操作入口
3. 前置条件
4. 正常流程
5. 页面截图
6. 字段说明
7. 异常处理
8. 常见问题
9. 注意事项
10. 关联后台数据

---

## 3. 截图标注规范

截图要求：

- 截整屏，不只截局部。
- 页面较长时分段截图，并标明“上半部分 / 下半部分”。
- 点击路径使用箭头。
- 关键按钮使用红框。
- 状态字段使用蓝框。
- 财务金额使用黄色框。
- 客户隐私全部打码。

---

## 4. 高风险动作说明

以下动作在手册采集时只进入查看页，不点最终确认：

- 审核通过
- 审核拒绝
- 关闭订单
- 退款
- 冲正
- 提现审核
- 打款
- 分账重试
- 锁机 / 解锁
- 删除 / 停用关键配置

---

## 5. 首批手册目录

| 手册 | 读者 |
|---|---|
| 商家入驻操作手册 | 新商家、运营审核人员 |
| 门店办单操作手册 | 门店老板、店员 |
| 客户扫码下单操作手册 | 客户、门店培训 |
| 订单审核操作手册 | 运营客服、商家老板 |
| 合同公证操作手册 | 运营客服、客户培训 |
| 发货签收操作手册 | 门店、商家、运营 |
| 财务提现对账操作手册 | 商家老板、财务 |
| 渠道推广操作手册 | 渠道人员、运营 |
| 租后催收操作手册 | 租后客服、外部催收 |

---

## 6. 数据联动要求

手册不能只教“点哪里”，还要说明动作后数据去哪：

- 入驻通过后生成商家、门店、老板账号、钱包、默认链路配置。
- 办单助手生成报价后进入报价快照。
- 客户扫码后生成订单、客户资料、渠道归属。
- 审核通过后触发合同、支付、发货、账单。
- 发货签收后触发起租和设备状态变化。
- 支付成功后触发账单、分账、钱包、对账。

---

## 7. 交付格式

建议同时输出：

- Markdown 文档，方便进知识库。
- PDF，方便外发。
- 图片包，方便微信群培训。
- 视频脚本，方便后续录屏。
