# 商家订单列表与详情

> 页面级 PRD 草案。
> 目标：让商家 PC 端承接门店订单自审、分红订单进度查看、平台订单进度查看和联系客服，不越权操作平台主控流程。

> **⚠️ V0.2 Peer Review 修订(2026-05-27)v1.1**:
> - 商家订单合同签署前允许商家修改关键字段。
> - 联营订单 / 平台订单不开放审核、改价和资金风控主控操作。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商家订单列表与详情 |
| 所属端 | 商家 PC 端 |
| 入口路径 | 商家中心 > 订单管理 |
| 使用角色 | 商家老板、商家管理员、门店店长、授权员工 |
| 核心目标 | 查看本商家订单，处理门店自营订单，跟进分红/平台订单审核、资方、合同、发货和财务进度 |

---

## 2. 订单范围

| 订单类型 | 商家端定位 | 操作边界 |
|---|---|---|
| 门店订单 | 商家自营订单 | 商家自审、改价、发货、售后、钱包收益 |
| 分红订单 | 商家参与出资或出设备 | 查看进度、补资料、查看资方和分账，不主控审核 |
| 平台订单 | 商家送单给平台 | 查看进度、联系客服、补资料，不主控审核和资方 |

---

## 3. 列表结构

```
订单管理
├─ 门店订单
├─ 分红订单
├─ 平台订单
└─ 全部订单
```

商家端默认按三类订单拆分，不建议一个列表混在一起。

---

## 4. 筛选条件

| 字段 | 类型 | 说明 |
|---|---|---|
| 订单号 | 文本 | 精确查询 |
| 客户姓名 | 文本 | 权限内查询，列表脱敏 |
| 手机号 | 文本 | 支持后四位 |
| 商品名称 | 文本 | 模糊搜索 |
| 门店 | 下拉 | 多门店商家可筛选 |
| 办单员工 | 下拉 | 按员工账号筛选 |
| 订单状态 | 下拉 | 待付款、待审核、待签约、待发货、待归还、已逾期、已完成 |
| 下单时间 | 日期区间 | 默认最近 30 天 |

## 5. 列表字段

| 区域 | 字段 |
|---|---|
| 订单摘要 | 订单号、订单类型、下单时间、门店、办单员工 |
| 客户 | 姓名脱敏、手机号脱敏、地址摘要、资料状态 |
| 商品 | 商品、规格、租赁模式、租期、设备码 |
| 费用 | 首期、后期账单、押金、增值服务、留购价 |
| 进度 | 当前状态、下一步待办、审核主体 |
| 财务 | 门店订单收益、分红订单预计分账、平台订单协作收益 |
| 操作 | 查看详情、审核、发货、补资料、联系客服、售后 |

员工账号只展示本人或授权范围订单，不展示钱包和分账汇总。

---

## 6. 门店订单详情

商家可主控门店订单：

| 能力 | 说明 |
|---|---|
| 审核资料 | 商家自行判断是否通过 |
| 改价/改套餐 | 按商家权限生成新价格方案 |
| 发起合同 | 可用商家模板或平台提供模板 |
| 发货/交付 | 默认门店发货，上传交付材料 |
| 售后/退款 | 商家处理，平台保留监管和财务总账 |
| 钱包入账 | 展示平台抽佣、可提现、冻结、流水 |

门店订单如果发生客诉、违规、黑名单命中或财务异常，运营端可介入。

### 6.1 合同签署前字段修改

商家订单满足以下条件时,商家端显示“修改订单”入口:

| 条件 | 规则 |
|---|---|
| 订单类型 | 商家订单 |
| 状态 | 待审核 / 待付款 |
| 合同 | 客户合同未签署 |
| 角色 | 商家老板或被授权管理员 |

可修改:商品/SKU、期数、首期应付、保证金、增值服务、设备价、订单总额。修改后系统重新生成订单确认快照,客户需重新确认后才能继续支付或签约。

## 7. 分红订单详情

| 区域 | 展示 |
|---|---|
| 审核进度 | 待审核、补资料、已通过、已拒绝 |
| 资方分配 | 资方是否已分配、出资比例、出资金额 |
| 分账预览 | 门店占比、资方占比、平台抽佣 |
| 合同/授权 | 客户是否完成授权、合同、公证、绑卡 |
| 发货待办 | 是否需要门店发货或上传材料 |
| 财务 | 门店已入账、待结算、可提现 |

商家不能自行更换资方或绕过平台审核。

---

## 8. 平台订单详情

| 区域 | 展示 |
|---|---|
| 平台审核进度 | 待审核、补资料、审核通过、拒绝 |
| 客服 IM | 联系平台客服，自动携带订单摘要 |
| 资方/执行方 | 只展示必要进度，不展示敏感内部规则 |
| 合同/支付 | 客户签署、支付、绑卡状态 |
| 发货主体 | 门店、平台、资方或快递 |
| 协作收益 | 如配置门店服务费或渠道佣金，展示结算进度 |

平台订单的审核、资方、财务主控在运营端。

---

## 9. 商家端操作

| 操作 | 门店订单 | 分红订单 | 平台订单 |
|---|---|---|---|
| 查看详情 | 是 | 是 | 是 |
| 审核通过/拒绝 | 是 | 否 | 否 |
| 改价/改套餐 | 是 | 否 | 否 |
| 补资料 | 是 | 协助 | 协助 |
| 发货/交付 | 按配置 | 按配置 | 按配置 |
| 联系客服 | 可选 | 是 | 是 |
| 售后/退款 | 是 | 申请/协助 | 申请/协助 |
| 查看钱包收益 | 老板/管理员 | 老板/管理员 | 老板/管理员 |

---

## 10. 权限与日志

1. 老板可看全部门店订单和钱包。
2. 管理员权限由老板配置。
3. 店员只看本人或授权门店订单。
4. 员工不能查看钱包、提现、收款账户。
5. 审核、改价、发货、售后、补资料、联系客服都记录日志。
