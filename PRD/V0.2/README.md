## 二、V0.2.2 必守口径

| 主题 | 当前口径 |
|---|---|
| 一期范围 | 安心用长租 + 体验租短租 V1 + IM 客服基础能力 |
| 订单展示名 | 商家订单 / 联营订单 / 平台订单 |
| 内部枚举 | 可保留 `merchant` / `joint` / `platform` 等兼容字段,但展示层必须映射为三类中文名 |
| 长租结算 | 客户签收确认 + 监管锁已上锁 + 激活锁已激活 |
| 无监管锁品类 | 无监管锁是正常业务形态;仅苹果手机等配置为需监管锁回调的品类走锁状态结算,其它品类按签收确认 + 设备识别码归档 + 交付证据完整 + 前置检测完成结算 |
| 退款责任 | 商家订单商家承担;联营订单/平台订单线下协商;系统支持指定金额退款 |
| 财务原则 | 订单业务账 / 支付流水账 / 钱包或商家账户账 / 财务总账分录四账分离 |
| 服务费 | 当前 PRD 口径为 2%,可配置 |
| C 端隔离 | 不展示资方、资金来源、合作模式、风控结论、内部审核备注、服务费拆分、商家结算价、平台利润、返点规则、内部标签、黑灰名单结论 |
| 权限 | 高危动作拆细权限点,写操作日志和审计日志,必要时二次确认或二人复核 |
