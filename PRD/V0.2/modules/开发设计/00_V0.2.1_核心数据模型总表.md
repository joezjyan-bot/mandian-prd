# V0.2.1 核心数据模型总表

> **V0.2.1 开发冻结同步(2026-05-27)**: 本文档是 V1 开发的数据模型阅读入口。只汇总长租主营链路必需模型和 V2 预留边界，不改数据库字段名、API 路径和英文枚举。若页面级 PRD 与本文冲突，开发先按 `00_V0.2.1_开发冻结版总PRD.md`、`00_核心决策表.md` 和本文执行，并在验收报告中记录差异。

---

## 1. V1 模型边界

| 范围 | 冻结口径 |
|---|---|
| V1 主链路 | 商家订单、联营订单、平台订单三类长租主营订单 |
| V1 不进入 | 体验租/短租下单、车辆库存状态机、取还车核销、小时/日/周短租结算 |
| 长租设备 | 发货/交付节点绑定 `IMEI/SN/VIN` 等设备识别码档案，不占用短租库存 |
| 短租设备 | 仅作为 V2 预留模型，不进入 V1 migration 和验收矩阵 |
| C 端展示 | 不展示资方、资金来源、合作模式、风控字眼；内部字段可保留兼容 |

## 2. 主体与权限模型

| 模型 | 用途 | 关键字段/关系 | V1 要求 |
|---|---|---|---|
| `tenants` | 多租户部署主体 | `id`、`name`、`status`、`config_version` | 每套部署独立数据域 |
| `merchants` | 商家主体 | `id`、`tenant_id`、`status`、`source_channel_id` | 商家入驻、审核、结算归属 |
| `stores` | 门店/经营网点 | `id`、`merchant_id`、`status`、`address` | 办单、发货、交付、门店员工归属 |
| `merchant_staff` | 商家员工账号 | `id`、`merchant_id`、`store_id`、`role`、`status` | 支持老板、管理员、财务、店长、普通员工 |
| `merchant_finance_permissions` | 商家财务授权 | `staff_id`、`can_view_wallet`、`can_withdraw`、`can_edit_settlement_account=false` | 财务可授权看钱包/发起提现，但不能改结算账户 |
| `permission_matrix` | 权限矩阵配置 | `role`、`resource`、`action`、`need_2fa`、`need_log` | 资金、账户、合同、退款、关单类高危动作必须有日志 |
| `operation_logs` | 操作日志 | `actor_type`、`actor_id`、`target_type`、`target_id`、`before_snapshot`、`after_snapshot` | 高危动作记录脱敏摘要和二次确认结果 |

## 3. 商品、办单与订单模型

| 模型 | 用途 | 关键字段/关系 | V1 要求 |
|---|---|---|---|
| `products` | 商品主档 | `id`、`category_id`、`name`、`sale_mode`、`status` | 支持长租主营；短租适配仅预留标识 |
| `product_specs` | 商品自定义规格 | `product_id`、`spec_name`、`spec_value` | 规格名称和值由后台自定义，不写死颜色/成色 |
| `product_skus` | 规格价格组合 | `product_id`、`spec_snapshot`、`guide_price`、`device_price` | 办单计算必须有设备价/指导价依据 |
| `order_application_drafts` | 办单助手草稿 | `order_type`、`store_id`、`product_id`、`device_price`、`funding_amount` | 三个办单入口字段基本一致，联营多配资额度 |
| `orders` | 订单主表 | `order_no`、`order_type`、`merchant_id`、`store_id`、`customer_id`、`status`、`funding_source` | `funding_source` 为内部字段，不在 C 端展示 |
| `order_items` | 订单商品明细 | `order_id`、`product_id`、`sku_id`、`device_price`、`snapshot` | 保存下单时商品、规格、价格快照 |
| `order_status_logs` | 订单状态流转 | `order_id`、`from_status`、`to_status`、`event` | 所有状态变化可追溯 |
| `order_c_progress` | C 端断点续传 | `order_id`、`current_step`、`completed_steps` | 支撑资料、实名、授权、合同、支付、绑卡等断点恢复 |

## 4. 客户、授权、合同与签收模型

| 模型 | 用途 | 关键字段/关系 | V1 要求 |
|---|---|---|---|
| `customers` | 客户主体 | `id`、`mobile_hash`、`real_name_masked`、`cert_no_masked` | 客户数据默认脱敏展示 |
| `customer_external_accounts` | 微信/支付宝/H5 账号映射 | `customer_id`、`channel`、`openid/unionid` | 三端打通预留 |
| `customer_auth_records` | 客户授权记录 | `customer_id`、`auth_type`、`status`、`signed_at` | C 端文案为信用/信息授权，不写风控 |
| `contract_templates` | 合同模板 | `template_type`、`version`、`is_default`、`status` | 运营审核可选择默认或业务模板 |
| `order_contracts` | 订单合同 | `order_id`、`template_id`、`sign_status`、`pdf_url` | 默认三方租赁合同，资方不进入客户合同 |
| `notary_records` | 公证记录 | `order_id`、`status`、`provider`、`callback_payload` | 公证可选，发起到客户订单页操作 |
| `delivery_orders` | 发货/交付单 | `order_id`、`delivery_type`、`carrier_code`、`tracking_no`、`delivery_status` | 支持门店自提、物流、平台仓、内部代发 |
| `delivery_evidence` | 交付证据 | `delivery_id`、`evidence_type`、`file_url`、`uploaded_by` | 支持设备照、配件照、人机/人车合照 |
| `long_term_device_archives` | 长租设备识别码档案 | `order_id`、`identifier_type`、`identifier_value`、`bound_at` | 长租只在发货时绑定识别码，不走库存状态 |
| `receipt_confirmation_configs` | 签收确认配置 | `scope_type`、`method`、`template_id`、`block_billing_on_exception` | 默认电子确认单，AI 问答配置预留 |
| `receipt_confirmations` | 签收确认记录 | `order_id`、`method`、`status`、`confirmed_at`、`exception_reason` | 签收通过后可起租并生成账单计划 |
| `ai_receipt_questions` | AI 问答问题配置 | `config_id`、`question_text`、`answer_type`、`is_required` | V1 预留配置，不要求真实 AI 接入 |
| `ai_receipt_records` | AI 问答记录 | `order_id`、`video_url`、`transcript`、`result`、`manual_review_status` | V1 预留记录和异常状态 |

## 5. 账单、资金与结算模型

| 模型 | 用途 | 关键字段/关系 | V1 要求 |
|---|---|---|---|
| `order_bills` | 账单计划 | `order_id`、`period_no`、`due_date`、`amount_due`、`status` | C 端展示账单支付/每期应付，不写还款 |
| `payment_records` | 支付流水 | `order_id`、`bill_id`、`channel`、`amount`、`status` | 平台代收，支持首期、账单、保证金、留购/提前结清 |
| `guarantee_deposit_records` | 保证金记录 | `order_id`、`amount`、`status`、`deducted_amount` | 全文使用保证金，不用押金 |
| `settlement_entries` | 订单结算款入账 | `order_id`、`responsible_party`、`amount`、`entry_type` | 结算归属到商家/平台/内部台账 |
| `merchant_wallets` | 我的钱包 | `merchant_id`、`available_balance`、`frozen_balance` | 展示为我的钱包 |
| `wallet_ledgers` | 钱包流水 | `wallet_id`、`amount`、`direction`、`biz_type`、`ref_id` | 所有入账、冻结、冲正、提现可追溯 |
| `withdrawal_requests` | 提现申请 | `wallet_id`、`amount`、`status`、`review_status` | 单笔 500-20000，每日最多 3 次；超 20000 人工审核 |
| `merchant_settlement_accounts` | 商家结算账户 | `merchant_id`、`account_type`、`masked_account_no`、`status` | 修改账户必须高权限 + 二次确认 + 日志 |
| `refund_requests` | 退款申请 | `order_id`、`amount`、`responsibility_type`、`status` | 出退款必须入分账/钱包冲正闭环 |
| `refund_workflow_locks` | 退款流程锁 | `order_id`、`locked_steps`、`unlock_condition` | 退款处理中锁定相关业务动作 |
| `pending_fee_queues` | 待补扣费用队列 | `merchant_id`、`amount`、`reason`、`status` | 钱包余额不足时挂账，不强行负数处理 |
| `funder_assignments` | 内部资金来源分配 | `order_id`、`funder_id`、`funding_source`、`assigned_at` | 内部后台使用，C 端、合同、流水备注不展示 |
| `funder_receivable_ledgers` | 内部资金方/应收台账 | `order_id`、`amount`、`status`、`settled_at` | 只做内部核算和对账，不改变客户侧三方合同链路 |

## 6. 中控台与外部接口模型

| 模型 | 用途 | 关键字段/关系 | V1 要求 |
|---|---|---|---|
| `tenant_config_cache` | 租户配置缓存 | `tenant_id`、`config_key`、`config_value`、`synced_at` | 本地可运行，中控台异步同步 |
| `usage_log_queue` | 计费/接口调用队列 | `tenant_id`、`biz_type`、`payload`、`push_status` | 调用日志异步推送，失败可重试 |
| `zhongkong_reconciliations` | 中控台月度对账 | `tenant_id`、`period`、`local_amount`、`remote_amount`、`status` | 支撑接口计费和差异处理 |
| `external_api_call_logs` | 外部接口调用日志 | `provider`、`request_summary`、`response_summary`、`status` | 支付、合同、公证、物流、监管锁调用留痕 |

## 7. V2 后置模型

以下模型只作为体验租/短租预留，不进入 V1 接口、migration、验收矩阵：

| 模型 | V2 用途 |
|---|---|
| `short_rent_products` | 体验租商品展示和短租套餐 |
| `short_rent_vehicles` | 单车/设备库存档案 |
| `short_rent_vehicle_status_logs` | 车辆状态流转：在库、预约、出租中、待归还、维修等 |
| `short_rent_inventory_logs` | 库存出入库、调拨、核销 |
| `short_rent_orders` | 短租小时/日/周/月订单 |

| `short_rent_return_records` | 还车核销、验车、超时费用 |
| `short_rent_renewal_records` | 短租续租记录 |

## 8. 开发验收规则

1. 新增 V1 migration 时，不得把短租库存状态字段强行写入长租订单主链路。
2. C 端接口返回字段不得包含可直接展示的资方、资金来源、合作模式、风控文案。
3. 涉及英文枚举和历史字段兼容时，可以保留底层字段，但前端展示必须走 `modules/全局/10_全局措辞规范.md`。
4. 资金类表必须支持幂等、冲正、对账和操作日志。
5. 合同、签收、发货、账单、分账之间必须通过订单事件和状态流转闭环，不能只靠页面按钮跳转。
