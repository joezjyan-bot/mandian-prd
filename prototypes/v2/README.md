## 守的红线(已在原型中落实)

1. **C 端红线**:C 端各页完全不出现合作模式(商家/联营/平台)、资方/资金来源、分账、平台服务费拆分、商家结算价、渠道佣金、风控字眼。客户只看到"租门店一台设备 + 每期应付 + 保证金 + 到期可还/可续/可买"。
2. **合同三方制**:签约线框中合同三方仅为门店(甲)、客户(乙)、平台(丙),不出现资方,避免融资租赁定性。
3. **到期三选一同等展示**:归还 / 继续使用 / 申请购买三个选项同一视觉权重,明确标注不默认勾选、无"到期自动购买 / 自动转移所有权"。
4. **运营后台**订单详情抽屉里内置一条 C 端红线提示,提醒内部字段不可下发 C 端。
5. **门店端人脸隐私红线**:交付材料里"人机合照"标注"仅存比对结果",不暗示建人脸库。
6. **商家可见边界**:`merchant-pc.html` 标注商家不可见平台利润 / 服务费拆分 / 其他商家;财务人员可提现但不可改结算账户;店员不可编辑商品、不可见成本价。
7. **必须本人**:实名人脸、收货确认、合同签署、首期支付、代扣绑卡均标注须客户本人,不可门店/他人代替。
8. **隐私脱敏**:身份证号、银行卡号、手机号等在 C 端均脱敏展示;平台不留存完整卡号。
