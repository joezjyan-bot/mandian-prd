# 核心链路验收清单

> 所属：测试验收 / 开发交付
> 目标：让产品、开发、测试、Claude、Codex 用同一套业务链路验收，避免页面有了但数据不通、状态不走、财务不落账。

---

## 1. 验收原则

1. 每条链路必须从入口走到数据结果，不能只看页面是否展示。
2. 每个关键动作必须产生状态、日志、待办或流水。
3. 金额相关必须能追到订单、账单、支付流水、分账流水、钱包流水和对账记录。
4. 第三方能力必须验收发起、回调、失败重试和人工补偿。
5. 所有敏感资料必须验证权限、脱敏和敏感查看日志。

---

## 2. 商家入驻验收

流程：

```text
门店注册
  -> 填资料
  -> 上传营业执照/法人身份证/门头照
  -> 地图选择真实地址
  -> 绑定实名收款账户或上传授权
  -> 完成 e 签宝授权
  -> 运营端审核
  -> 开通商家 PC 和门店手机端
```

验收点：

- 营业执照识别结果能回填名称、统一社会信用代码、地址。
- 身份证识别结果能回填姓名、身份证号、有效期。
- 收款账户实名不一致时必须进入授权资料流程。
- e 签宝授权状态能同步到运营端店铺审核页。
- 审核通过后生成商家主体、门店主体、老板账号、默认链路配置、钱包账户。
- 审核拒绝必须写原因，商家端能查看并重新提交。

---

## 3. 商品到办单助手验收

流程：

```text
运营/商家添加商品
  -> 配置全新/二手规格
  -> 配置长租/短租计费单位
  -> 配置增值服务
  -> 设置是否同步办单助手
  -> 门店办单助手选择商品
  -> 生成报价和二维码
```

验收点：

- 同一商品能同时存在全新、二手规格。
- 未配置的规格不出现在办单助手。
- 商品可选择长租、短租或同时支持。
- 短租商品必须能绑定设备库存。
- 运营平台商品可复制给指定商家。
- 商家可决定是否同步到自己的门店办单助手。
- 办单助手生成的保证金、租金、服务费、公证费、留购价必须进入订单快照。

---

## 4. 三类订单验收

### 4.1 商家订单

- 门店办单助手生成商家订单二维码。
- 客户扫码提交后进入商家端待审核。
- 商家自己审核通过或拒绝。
- 运营端可只读查看，不参与主控审核。
- 财务按商家订单抽佣规则扣平台抽佣，剩余进入门店钱包。

### 4.2 联营订单

- 门店办单助手选择联营订单。
- 门店填写设备价和需求配资比例。
- 系统计算门店出资额、资方出资额、分账比例。
- 客户扫码提交后进入运营端待审核。
- 审核客服分配资方。
- 客户每笔账单支付按比例分给门店和资方，再分别扣平台抽佣。

### 4.3 平台订单

- 门店办单助手选择平台订单。
- 客户扫码提交后进入运营端待审核。
- 平台匹配资方或平台自有资金。
- 门店只查看进度，不参与审核和财务主控。
- 渠道佣金只统计联营订单和平台订单。

---

## 5. 客户下单验收

流程：

```text
扫码
  -> 查看报价
  -> 实名认证
  -> 身份证/联系人/地址/定位
  -> 人脸
  -> 风控/征信授权
  -> 代扣签约或支付
  -> 合同签署
  -> 公证办理
  -> 等待审核/发货
```

验收点：

- 客户每一步完成后订单详情和运营端列表同步状态。
- 客户未完成资料时，运营端可发起补资料。
- 补资料必须回到客户小程序订单页完成。
- 合同、公证、授权、支付失败时有可重发入口。
- 所有外部回调进入回调日志，并能重放或人工补偿。

---

## 6. 发货和设备验收

- 默认门店发货。
- 配置后可走平台、商家、资方或物流接口发货。
- 发货前短租设备必须选定唯一设备码。
- 门店手机端可上传人机/人车合照、设备照片、配件清单。
- 客户签收后订单状态、设备状态、账单起算状态同步变化。
- 归还后设备进入待验收、可入库、维修、争议或下架。

---

## 7. 财务验收

必须能完整追踪：

```text
订单金额快照
  -> 账单
  -> 支付流水
  -> 分账流水
  -> 钱包流水
  -> 提现单
  -> 打款流水
  -> 对账记录
```

验收点：

- 商家订单、联营订单、平台订单三种分账规则正确。
- 平台默认 2% 抽佣可按订单类型、商家、资方调整。
- 联营订单按比例分账后分别扣佣。
- 退款、冲正、关闭订单会冻结或回滚相关钱包余额。
- 部分支付收款码能生成子账单并同步回主账单。
- 通联/信联等收款码通道回调进入对账中心。

---

## 8. 权限和日志验收

- 店员账号只能办单、查看待办、扫码交付，不可看钱包。
- 商家老板可分配员工账号。
- 审核、改价、退款、冲正、打款、敏感查看必须有按钮权限。
- 人工操作、系统操作、第三方回调全部写日志。
- 日志能按订单、客户、商家、资方、渠道、操作者查询。

---

## 9. 验收通过标准

1. 三类订单各跑通 1 条完整链路。
2. 每条链路能在运营端订单列表直接看到关键状态。
3. 点进详情能看到完整资料、账单、合同、公证、支付、发货、日志。
4. 钱包和对账数据能追到来源单据。
5. 失败场景至少覆盖支付失败、合同回调失败、补资料、退款冲正。
6. 没有数据孤岛：同一订单在运营端、商家端、门店端、C 端状态一致。
