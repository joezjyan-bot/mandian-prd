## 四、评审操作指南

### 4.1 在飞书文档中评论

1. **选中文字** → 右键 → **"评论"**
2. 评论框里按这个格式写：
   ```
   【类型】修改 / 新增 / 删除 / 质疑 / 疑问
   【内容】具体建议
   【原因】（可选，但推荐写）
   ```
3. 重要的评论请 **@ 项目负责人**

### 4.2 反馈示例

**示例 1 - 修改某规则**：
```
【类型】修改
【内容】"最后一期留购价 = 仅押金" 应改为 "押金 + 100 元"
【原因】完全为零客户会觉得是漏洞
```

**示例 2 - 质疑设计**：
```
【类型】质疑
【内容】"默认不拉客户进会话" 这个设计反了
【原因】实际审核常需核对客户信息，没客户在会拖很久
```

**示例 3 - 提疑问**：
```
【类型】疑问
【内容】"分红订单门店不出钱" 这句具体怎么实现？
【请@项目负责人解答】
```

### 4.3 重要！避免这些做法

❌ **不要** 在飞书群里@大家说"我觉得这里不对"  
✅ **请** 直接到文档对应位置评论，留下精确位置

❌ **不要** 评论说"这块不行得重做"（太模糊）  
✅ **请** 具体说哪里不行，建议怎么改

❌ **不要** 把所有问题攒一周一次性发  
✅ **请** 读到哪里有问题就在哪里评论

## 五、反馈汇总（用飞书多维表格）

项目负责人会建一个【**PRD 评审反馈汇总表**】多维表格，所有评论会被汇总进去。结构如下：

| 字段 | 类型 | 说明 |
|---|---|---|
| 反馈编号 | 自动编号 | F-001, F-002... |
| 提出人 | 成员 | 谁提的 |
| 提出时间 | 日期 | 何时提的 |
| 章节 | 文本 | §4.4.5 等 |
| 文档 | 单选 | 哪份子文档 |
| 类型 | 单选 | 修改/新增/删除/质疑/疑问 |
| 内容 | 多行文本 | 具体反馈 |
| 原因 | 多行文本 | 提议人写的原因 |
| 紧急程度 | 单选 | 高/中/低 |
| 处理状态 | 单选 | 待处理/讨论中/已采纳/已驳回/已并入下版 |
| 决策结果 | 多行文本 | 产品决策结果 |
| 处理人 | 成员 | 谁决策 |
| 处理时间 | 日期 | 何时决策 |

## 六、常见问题 FAQ

**Q1：我看到 mermaid 流程图变成代码块怎么办？**  
A：先忽略代码块本身，文字内容看完整即可。完整图片版会单独提供。

**Q2：我对其他部门的章节也有意见，能不能看？**  
A：可以申请阅读权限，但评论请只针对自己的章节。其他部门章节的问题请通过你的 leader 转给对应部门 leader。

**Q3：我看完发现一个大问题想推翻整章怎么办？**  
A：不要在文档里直接修改。请用评论 @ 项目负责人，约个会单独讨论。

**Q4：评审周期太紧来不及怎么办？**  
A：优先看自己最熟悉的部分（重点章节标出来了）。读到哪儿提到哪儿。

**Q5：评审之外的功能建议怎么提？**  
A：欢迎在评论里提"未来期望"，但请标明"V2.0+"或"远期"，不影响当前版本决策。
