# 商品列表与新增编辑

> 页面级 PRD 草案。
> 目的：把商品从“只做商品审核”升级为完整商品管理，支撑运营商品、商家商品、规格、安心用长租、增值服务和办单助手同步；体验租配置由独立短租模块承载。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 商品列表与新增编辑 |
| 所属端 | 运营端 |
| 入口路径 | 商品管理 > 商品列表 |
| 使用角色 | 平台管理员、商品运营、审核客服、商家运营、技术支持 |
| 核心目标 | 统一管理平台商品和商家商品，并控制商品规格、价格、租赁模式、同步办单助手 |

---

## 2. 核心口径

1. 商品管理不等于商品审核，运营端必须有新增、编辑、复制、审核、上下架、同步办单助手等完整权限。
2. 规格组和规格值必须支持自定义，不写死。`全新`、`二手`、`99新`、`95新` 只是“设备成色”规格组的示例值，不应要求重复创建多个商品。
3. 商品可以来自平台，也可以来自商家。
4. 平台商品可复制给一个或多个商家，商家形成自己的商品副本。
5. 商品和规格都要有 `是否同步办单助手`，规格无价格或无可用租期时不能同步。
6. 当前商品表单只落地安心用长租配置；体验租商品、设备库存和唯一设备码在 `modules/短租业务/*` 中独立维护。
7. 增值服务可以在商品编辑中绑定，后续同步给对应办单助手和订单改套餐。
8. 适配体验租的商品必须维护统一的商品标准名；一旦开启体验租并绑定价格方案、车辆库存或订单，商品标准名、车型编号、品牌、型号、核心类目不得直接修改。

## 3. 页面结构

```text
商品管理 / 商品列表
├─ 顶部筛选：商品名称 品牌 类目 商家 商品来源 审核状态 上下架 同步状态

├─ 商品表格
│  ├─ 商品图 / 商品名称 / 品牌 / 类目
│  ├─ 商品来源：平台商品 / 商家商品

│  ├─ 规格数：设备成色、颜色、容量等自定义规格组
│  ├─ 租赁模式：当前固定长租

│  ├─ 审核状态 / 上下架状态

└─ 新增/编辑抽屉
```

## 4. 筛选字段

| 字段 | 类型 | 说明 |
|---|---|---|
| 商品名称 | 文本 | 支持模糊搜索 |
| 品牌 | 下拉/搜索 | 苹果、华为、小米、九号、极核、小牛等 |
| 类目 | 级联 | 手机、电动车、平板、手表等 |
| 商品来源 | 下拉 | 平台商品、商家商品 |
| 所属商家 | 下拉/搜索 | 商品来源为商家商品时可筛选 |
| 审核状态 | 下拉 | 草稿、待审核、已通过、已驳回 |
| 上下架状态 | 下拉 | 上架、下架、冻结 |
| 租赁模式 | 下拉 | 当前固定长租；短租字段不在本表单展示 |
| 是否同步办单助手 | 下拉 | 未同步、已同步、部分规格同步 |
| 助手入口 | 多选 | 门店订单、分红订单、平台订单 |
| 规格组/规格值 | 多选 | 自定义规格组和规格值，如设备成色、颜色、容量 |

## 5. 商品基础字段

| 字段 | 类型 | 规则 |
|---|---|---|
| 商品名称 | 文本 | 必填 |
| 商品标准名 | 文本 | 适配体验租时必填；启用体验租并产生配置/库存/订单后锁定 |
| 展示标题 | 文本 | C 端展示用，可按权限修改，不影响库存和价格方案绑定 |
| 品牌 | 选择器 | 必填，可维护品牌字典 |
| 类目 | 级联 | 必填，类目决定可用规格字段 |
| 商品来源 | 只读/下拉 | 平台商品或商家商品 |
| 所属商家 | 选择器 | 商家商品必填，平台商品为空 |
| 商品主图 | 图片上传 | 必填 |
| 轮播图 | 图片上传 | 可多张 |
| 详情图/详情描述 | 富文本/图片 | 可选 |
| 商品卖点 | 文本 | 可选，用于 C 端展示 |
| 适用租赁模式 | 只读/配置 | 当前表单固定安心用长租；体验租进入独立配置页签/模块 |
| 是否可复制给商家 | 开关 | 平台商品可配置 |
| 商品状态 | 状态 | 草稿、待审核、已通过、已驳回、上架、下架、冻结 |

体验租命名规则：

1. `商品标准名` 是后台主数据字段，必须统一，例如“九号 Qz1 MIX”“九号 C80c”。
2. `展示标题` 是前台营销字段，例如“九号 Qz1 MIX 基础款，短途出行更轻便”，可在活动或商品卡片中调整。
3. 商品开启体验租后，如果已被短租价格方案、车辆库存、订单快照引用，标准名不可直接编辑。
4. 商家新增体验租商品时优先选择平台标准商品库；没有匹配商品时提交新增标准商品申请，由运营审核。
5. 如标准名确实错误，应通过“新建标准商品 + 迁移配置/库存”的方式处理，不允许直接覆盖历史主数据。

## 6. 规格字段

规格是商品的核心。一个商品下可以自定义多个规格组，每个规格组下维护多个规格值。系统根据规格值组合生成可定价的 SKU。

示例：

- 规格组：设备成色；规格值：全新、二手、99 新、95 新。
- 规格组：颜色；规格值：白色、黑色、黄色。
- 规格组：容量；规格值：128G、256G、512G。
- 规格组：电池；规格值：有电池、无电池。

| 字段 | 类型 | 说明 |
|---|---|---|
| 规格组名称 | 文本 | 例如设备成色、颜色、容量、电池 |
| 规格值 | 文本/批量添加 | 例如全新、二手、白色、黑色、256G |
| SKU 组合名称 | 自动生成/可编辑 | 例如 `全新 256G 黑色`、`二手 95新 含电池` |
| 官方售价/指导价 | 金额 | 办单助手计算基础 |
| 成本价 | 金额 | 权限控制，普通运营不可见 |
| 库存数量 | 不展示 | 当前长租不依赖库存；体验租库存由体验租车辆/设备库存模块处理 |
| 可用租赁模式 | 只读/配置 | 基础商品可支持安心用/体验租；具体价格分业务线配置 |
| 可用计费单位 | 配置 | 当前按长租期/月配置 |
| 可用租期 | 配置 | 例如 4/6/9/12 期 |
| 押金规则 | 配置 | 固定金额、比例、免押、补充押金 |
| 留购规则 | 配置 | 到期留购、提前留购、不可留购 |
| 是否启用 | 开关 | 停用后不能下单 |
| 是否同步办单助手 | 开关 | 规格维度控制 |

规则：

1. 商品可没有全新规格，也可没有二手规格；规格值由运营或商家自行维护。
2. 办单助手只展示已启用、已同步、价格完整的规格组合。
3. 同一规格可以适配安心用长租；体验租适配和设备库存进入短租业务模块。
4. 规格价格不完整、租期配置不完整、增值服务配置冲突时，不允许发布同步。
5. 规格变更影响已生成二维码时，旧二维码继续按锁价方案执行，新二维码使用新配置。

## 7. 价格和租期配置

商品编辑页不直接写死计算公式，价格应从配置和办单助手计算逻辑读取。

| 配置 | 说明 |
|---|---|
| 长租租期 | 按期/月配置，如 4/6/9/12 期 |
| 体验租周期 | 不进入安心用长租表单；在体验租配置页签单独定义 |
| 首期实付 | 可由计算器逻辑生成，也允许权限内手动覆盖 |
| 后续账单 | 按租期生成 |
| 服务费/设备管理费 | 可绑定增值服务 |
| 公证费 | 可作为增值服务或链路费用 |
| 留购价 | 由商品价格、已付租金、规则计算 |
| 押金/补充押金 | 按规格规则生成 |

与惠讯 `phone-rent` 计算器的关系：

1. 办单助手最终展示和生成账单时，以惠讯计算器逻辑为准。
2. 商品管理只维护计算所需的商品、规格、租期、价格、增值服务和同步开关。
3. 客服审核改套餐时，可以在订单详情基于生成结果再调整。

## 8. 增值服务绑定

| 字段 | 说明 |
|---|---|
| 增值服务名称 | 从增值服务库选择 |
| 适用订单类型 | 门店订单、分红订单、平台订单 |
| 适用规格 | 可按规格绑定 |
| 是否必选 | 必选时办单助手默认勾选且不可取消 |
| 是否可改价 | 审核客服是否可调整金额 |
| 是否计入首期 | 是/否 |
| 是否计入账单 | 是/否 |
| 是否可退 | 是/否 |

规则：

1. 运营端配置的增值服务默认用于分红订单、平台订单。
2. 商家端配置的增值服务默认用于门店订单。
3. 商品绑定增值服务后，办单助手和订单详情改套餐都要可读取。

## 9. 操作按钮

| 操作 | 权限 | 说明 |
|---|---|---|
| 新增平台商品 | 商品运营 | 创建平台商品 |
| 编辑 | 商品运营/商家运营 | 已上架商品编辑需记录影响范围 |
| 提交审核 | 商家/运营 | 商家商品必须审核后可用 |
| 审核 | 商品审核角色 | 审核通过或驳回 |
| 上架/下架 | 商品运营 | 下架不影响历史订单 |
| 复制给商家 | 平台商品权限 | 复制平台商品给商家 |
| 同步办单助手 | 商品运营 | 按商品/规格/订单入口同步 |
| 查看日志 | 有权限角色 | 查看商品变更和同步记录 |

---

## 10. 权限和日志

1. 成本价、资方相关价格、利润字段按角色控制。
2. 商家只能编辑自己的商品，不能看到其他商家商品。
3. 运营可筛选全部商品，但默认应按商品来源和商家筛选，避免列表混乱。
4. 商品、规格、价格、增值服务、同步状态、上下架、审核结果都必须记录操作日志。
5. 同步办单助手必须记录同步范围、同步入口、同步前后版本。
