# 扫码进入与路由

> **来源**: P1-5
> **核心**: 三种二维码不同入口

## 1. V1 客户端

- 支付宝小程序
- 微信小程序
- H5(默认浏览器)
- 后期出独立 App

## 2. 三种二维码

### 2.1 办单二维码
- 生成场景:门店在办单助手设置好价格方案后生成
- 扫码后跳转:**订单确认页**(锁定方案,直接下单)
- 路径:`/order/confirm?order_token=XXX`
- 有效期:24 小时(可配)
- 多端兼容:同一二维码可被支付宝/微信/H5 识别

### 2.2 商品二维码
- 生成场景:门店在商品列表生成,贴在样机上
- 扫码后跳转:**商品详情页**(可自由选套餐规格)
- 路径:`/product/detail?product_id=XXX&store_id=YYY`
- 永久有效(直到商品下架)

### 2.3 小程序入口二维码
- 生成场景:平台 / 品牌印刷物料(海报 / 名片 / 包装)
- 扫码后跳转:**小程序主页**(品牌入口)
- 路径:`/`(首页)
- 永久有效

## 3. 二维码识别

二维码内容编码:
```json
{
  "type": "order_token / product / homepage",
  "data": {
    "order_token": "...",    // 办单码用
    "product_id": "...",     // 商品码用
    "store_id": "...",       // 关联门店
    "channel_id": "..."      // 关联渠道(可选)
  },
  "timestamp": 1716643200,
  "signature": "..."         // 防伪签名
}
```

## 4. 二维码失效处理

| 类型 | 失效原因 | 提示 |
|---|---|---|
| 办单码 | 订单关闭 / 方案过期 | "此办单链接已失效,请联系门店" |
| 商品码 | 商品下架 | "该商品已下架" |
| 小程序入口码 | 永久有效 | - |

失效页面提供:
- 友好提示
- 联系门店按钮(若可定位)
- 返回首页按钮

## 5. 跨端识别

| 扫码端 | 二维码内容 | 跳转结果 |
|---|---|---|
| 支付宝扫 | 平台二维码 | 跳支付宝小程序 |
| 微信扫 | 平台二维码 | 跳微信小程序 |
| 浏览器扫 | 平台二维码 | 跳 H5 |

二维码 URL 智能跳转:
- 检测扫码端
- 跳对应端的入口
- 保留参数

## 6. 渠道追踪

部分二维码携带渠道 ID:
- 客户扫渠道方的二维码下单 → 订单关联该渠道
- 渠道方获得佣金(详见 02_渠道佣金结算)

## 7. 二维码生成

各端可生成:
- 办单助手 → 生成办单码
- 商品管理 → 生成商品码
- 平台 / 商家 → 生成入口码

二维码图片:
- 标准 QR Code(支持各种扫码)
- 可自定义 Logo(品牌)
- 可包含装饰元素

## 8. 安全防伪

- 二维码内容签名
- 一旦订单确认 / 商品访问,可记录扫码次数
- 异常高频扫码触发告警(防恶意爬)
