## 三、执行规范

### 3.1 命名规则(必读)

**数据库字段名 / API 路径 / 枚举值** 一律**不改**(保留底层兼容性):
- `cooperation_mode = self_operate / joint_venture / receivables_assignment`
- `order_penalty` 表
- `/api/v1/admin/penalty-rules`

**只改 UI / 文档展示用语**(对齐 Stage 6):
- 商家端 UI / C 端 UI / 文档正文 / 字段中文名

### 3.2 修订规范

每份文档修订时:
1. 在文档顶部新增 V0.2 修订注释(v1.x 版本号 + 日期 + 修订点)
2. 文档末尾的"修订记录"表加一条记录
3. 关联文档章节同步更新引用

### 3.3 commit 规范

```
refactor(V0.2/<模块>): <文件> v<x.x> — <修订摘要>
```

例如:
```
refactor(V0.2/订单管理): 02 交付签收照片证据 v1.1 — 三模式命名统一商家/联营/平台 + 履约中字眼同步
```

建议按以下顺序推:

1. **第一批(P0)**:任务 A(状态字典)— 必须先做,其他文档依赖
