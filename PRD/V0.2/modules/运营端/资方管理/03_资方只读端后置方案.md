# 资方只读端后置方案

> [V0.2.2 同步声明] 本目录历史命名“资方管理”保留为内部配置入口兼容。当前业务语义统一为**内部资金来源管理**;相关信息仅运营端、财务端、风控端可见,不得进入 C 端、商家端、客服话术或合同正文。

> 所属端：运营端 / 资方能力边界
> 目标：V1 先不做复杂资方独立后台，避免把核心租赁系统开发拖重；但数据模型必须预留资方只读、对账、提现和通知边界。

---

## 1. 当前确认口径

1. 联营订单和平台订单的资方主控在运营端。
2. 资方配置、额度、分配、出资占用、回款、提现和对账先由运营端管理。
3. V1 不做资方自主审核、不做资方自主改价、不做资方自主控制订单。
4. 后续如开放资方端，优先开放只读看板和对账下载，不开放高风险操作。

---

## 2. 为什么后置

资方端如果一开始做成完整后台，会新增账号、权限、审核、财务、合同、消息、导出、敏感字段等整套复杂度。

当前核心目标是把商家订单、联营订单、平台订单跑通：

- 门店能办单。
- 客户能扫码下单。
- 运营能审核和分配资方。
- 财务能按比例分账、提现、对账。
- 订单能完成合同、支付、授权、发货、归还。

所以资方端先作为数据边界预留，不抢主线。

---

## 3. V1 需要先落的数据

| 数据 | 来源 | 用途 |
|---|---|---|
| 资方主体 | 运营端资方管理 | 审核台分配资方、财务分账 |
| 授信额度 | 运营端录入或导入 | 限制可分配订单金额 |
| 出资占用 | 联营订单/平台订单审核通过 | 计算剩余额度 |
| 回款流水 | 客户账单支付和分账结果 | 生成资方收益 |
| 提现记录 | 运营端发起或审核 | 后续开放资方提现 |
| 对账单 | 财务对账中心 | 后续资方下载 |
| 操作日志 | 运营端人工/系统动作 | 追溯额度、分账、提现变更 |

## 4. V1 运营端能力

### 4.1 资方列表

字段：

- 资方名称
- 资方类型：个人 / 企业 / 平台自有 / 其他
- 状态：启用 / 停用
- 授信额度
- 已占用额度
- 可用额度
- 在租订单数
- 待回款金额
- 可提现余额
- 最近对账时间

操作：

- 新增资方
- 编辑资方资料
- 启用 / 停用
- 调整授信额度
- 查看订单
- 查看账单
- 查看操作日志

### 4.2 资方订单

字段：

- 订单编号
- 订单类型：联营订单 / 平台订单
- 商家 / 门店
- 客户脱敏信息
- 商品
- 出资金额
- 出资比例
- 已回款
- 未回款
- 当前订单状态
- 当前账单状态

约束：

- 不允许资方在 V1 直接审核订单。
- 不允许资方直接关闭订单、退款、改价、改合同。
- 所有动作由运营端有权限人员处理，并写操作日志。

### 4.3 资方账单

字段：

- 账单期数
- 应收金额
- 实收金额
- 资方应分金额
- 平台抽佣
- 资方实入账金额
- 分账状态
- 对账状态
- 关联支付流水

分账口径：

联营订单按门店和资方出资比例先分份额，再从各自份额扣平台抽佣。

---

## 5. 后续只读端范围

如果 V1.5/V2 开放资方只读端，只开放：

- 登录
- 首页数据概览
- 资方订单列表
- 资方账单列表
- 回款流水
- 提现申请或提现进度
- 对账单下载
- 通知消息

不开放：

- 订单审核
- 客户敏感资料明文查看
- 改价
- 合同发起
- 退款冲正
- 手动分账
- 通道配置
- 链路配置

## 6. 开发预留

| 对象 | 预留字段 |
|---|---|
| 资方主体 | `funder_id`、`funder_type`、`status`、`credit_limit` |
| 订单 | `funder_id`、`funding_amount`、`funding_ratio`、`funding_status` |
| 分账流水 | `beneficiary_type=funder`、`beneficiary_id`、`commission_amount` |
| 提现单 | `owner_type=funder`、`owner_id`、`withdraw_status` |
| 对账单 | `scope_type=funder`、`scope_id`、`period_start`、`period_end` |
| 日志 | `actor_type=operator`、`target_type=funder` |

---

## 7. 验收标准

1. 运营端能新增资方并在审核台分配。
2. 联营订单能按比例生成资方分账流水。
3. 平台订单能把收益完全归到资方或平台指定账户，再扣平台抽佣。
4. 财务对账能按资方筛选订单、账单、分账、提现。
5. 关闭、退款、冲正后资方分账和可提现余额能同步回滚或冻结。
6. 资方端不开通时，不影响主流程下单、审核、分账和提现。
