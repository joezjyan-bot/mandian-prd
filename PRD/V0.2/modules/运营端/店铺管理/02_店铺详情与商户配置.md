## 2. 核心口径(v1.2 更新)

1. 门店入驻审核通过后,入驻主体即成为商家主体,可同时使用商家 PC 端和门店手机端。
2. 店铺详情不重复做审核页的大表单,而是保留资料快照、配置入口、授权状态和变更记录。
3. 商家可自行完成部分授权,运营端必须能同步查看状态、回调时间和失败原因。
4. 订单能力按合作模式开关:**自营订单 / 联营订单 / 平台订单**(后台用语;商家端 UI 沿用通俗叫法)。
5. 链路配置默认继承平台配置,必要时运营端可给单个商家做覆盖。
6. 渠道来源、推广码、渠道人员必须沉淀到店铺详情,后续渠道统计和佣金依赖这里。
7. 店铺详情内所有配置变更必须保留操作日志和配置版本。
8. **门店结算账户**:门店入驻审核通过后,商家可在商家端主动申请开通门店结算账户(对应"我的钱包"),用于接收所有归属门店的资金(订单结算款 / 月度分账 / 平台奖励 / 返利 等)。

## 7. 订单能力配置(v1.2 更新)

| 能力 | 配置项 | 说明 |
|---|---|---|
| **自营订单**(原商家订单) | 开启/关闭、自审权限、综合服务费率、办单助手 | 门店自有订单由商家审核,平台收综合服务费 |
| **联营订单**(原联营订单) | 开启/关闭、可选出资比例、平台审核 | 双方按比例共同出资,共担风险 |
| **平台订单**(原平台订单) | 开启/关闭、平台审核 | 平台支付应收账款转让对价,门店做履约 |
| 商品能力 | 自建商品、平台商品复制、商品审核要求、同步办单助手 | 商品必须进入审核或配置范围 |
| 发货能力 | 门店发货、平台发货、物流接口 | 默认门店发货 |
| 员工办单 | 是否允许创建员工、员工可用入口 | 员工不可看钱包和敏感财务 |

能力变更规则:

1. 关闭某类订单能力后,不影响历史订单继续履约。
2. 新订单创建时读取当前生效配置版本。
3. 员工账号能力不能高于店铺主能力。
4. 暂停店铺时默认禁止新增订单和提现,历史订单仍可处理还款、归还和售后。
5. **门店结算账户**:入驻审核通过后,商家可在商家端主动申请开通(不限订单类型)。

## 9. 财务与接口计费(v1.2 重写)⭐

| 模块 | 内容 |
|---|---|
| **门店结算账户摘要**(我的钱包)⭐ | **开通状态、账户编号、可提现余额、在途余额、提现中、累计入账、累计提现、累计手续费、收款账户** |
| 收款账户 | 实名状态、授权状态、账户变更记录 |
| 接口计费 | 风控、大数据、合同、公证、短信等调用次数和费用 |
| 欠费限制 | 欠费时限制新增调用或限制特定订单能力 |
| 提现限制 | 正常、风控冻结、资料异常、收款账户异常 |

接口计费不直接在店铺详情里扣改金额,只展示摘要并跳转财务明细。
