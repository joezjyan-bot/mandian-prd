## 四、仍需注意的上线前条件

1. 逾期费用费率和归属必须由法务书面盖章后才能上线真实收取。
2. 无监管锁品类的高价值阈值 `high_value_threshold_amount` 需要在配置评审中填具体金额。
3. 34 个核心接口已经有逐项 DTO 勾选清单,但研发开工前仍需逐接口完成字段级契约评审。
4. 所有资金、退款、打款、返点、提现、法诉导出接口仍需按权限矩阵完成二次确认、二人复核和审计日志校验。

## 五、结论

第三方复核剩余的 1 个 P0 和 5 个 P1 已按要求逐项闭环并单独提交。当前文档口径已经从“继续补内容”推进到“可进入研发评审基线”状态,但上线前仍必须完成法务盖章、金额阈值配置评审、逐接口 DTO 评审和测试验收。
