# 对账与汇总

> **来源**: P0-3 / P0-4 / P2-1
> **包含**: 多场景对账 + 跨租户报表

## 1. 页面定位

运营端 · 财务管理 · 对账与汇总。财务/运营对以下场景对账:
- 中控台接口调用对账(月度)
- 通联/直付通资金对账(每日)
- 跨租户汇总报表(平台老板)

## 2. 中控台接口对账

### 入口
运营端 · 财务管理 · 中控台对账

### 列表字段
对账周期(YYYY-MM) / 能力(e签宝/人脸/OCR/风控/流水/大数据) / 租赁系统侧计数 / 中控台侧计数 / 差异 / 状态 / 处理人

### 差异处理
- 详情看明细调用记录
- 判断原因(网络丢包/降级/bug)
- 填处理结果 → 状态已处理

## 3. 资金流水对账

### 入口
运营端 · 财务管理 · 资金对账

### 每日对账
- 从通联后台导出对账单 CSV
- 上传后系统自动匹配本地订单流水
- 差异结果列表(未匹配/金额不一致/状态不一致)
- 人工处理

### 直付通分账对账
- 直付通提供分账记录
- 与系统抽佣记录匹配
- 重点关注抽佣金额、代收金额

## 4. 跨租户汇总报表

### 限制
- 仅平台老板 + 高级运营可访问
- 从集中汇总库查,不直接访问租户业务库

### 报表类型
- 总营收(按租户/按月)
- 订单量(按租户/按月/按类型)
- 商户/门店活跃度
- 逾期率/退款率
- 中控台调用汇总

### 报表导出
- 支持导出 Excel
- 主要报表可保存模板

## 5. 同步机制

- 每日凌晨各租户推送关键指标到集中汇总库
- 工具:DataX / 自研 ETL
- 失败重试 + 告警

## 6. 数据模型

- `zhongkong_reconciliation`
- 资金对账记录表后续补
- 汇总库 schema 在《开发设计》中补
