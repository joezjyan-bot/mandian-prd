# 商家入驻手册模板

> 适用对象：新商家 / 门店老板 / 运营审核人员

---

## 1. 操作入口

```text
门店登录页
  -> 立即入驻
  -> 门店入驻
```

---

## 2. 前置条件

商家需要准备：

- 手机号
- 登录密码
- 营业执照
- 法人身份证正反面
- 门头照片
- 门店真实地址
- 收款账户
- 如收款账户非法人本人，准备授权资料

---

## 3. 正常流程

```text
注册账号
  -> 填写基础信息
  -> 上传营业执照
  -> 上传法人身份证
  -> 地图选择门店地址
  -> 上传门头照片
  -> 绑定收款账户
  -> 完成 e 签宝授权
  -> 提交审核
  -> 等待运营审核
```

---

## 4. 页面截图占位

后续截图：

1. 门店登录页，标注 `立即入驻`。
2. 注册页，标注手机号、密码、验证码、协议勾选。
3. 资料填写页，标注营业执照、身份证、地址、门头照片。
4. 收款账户页，标注实名校验和授权入口。
5. e 签宝授权页，标注授权状态。
6. 运营端店铺审核页，标注审核通过、驳回、资料查看。

---

## 5. 字段说明

| 字段 | 说明 |
|---|---|
| 商家名称 | 从营业执照识别，可人工修正 |
| 统一社会信用代码 | 从营业执照识别 |
| 法人姓名 | 从身份证识别 |
| 法人身份证号 | 脱敏存储，明文按权限查看 |
| 门店地址 | 支持地图选择和手动补充 |
| 门头照片 | 用于线下门店真实性审核 |
| 收款账户 | 默认要求与法人实名一致 |
| e 签宝授权 | 用于后续合同签署 |

---

## 6. 异常处理

| 异常 | 处理 |
|---|---|
| 营业执照识别失败 | 允许手动录入，标记人工审核 |
| 身份证识别失败 | 重新上传或人工录入 |
| 收款账户不一致 | 进入授权资料流程 |
| e 签宝授权失败 | 重发授权，运营端可查看失败原因 |
| 审核驳回 | 商家查看驳回原因后重新提交 |

---

## 7. 数据联动

审核通过后系统生成：

- 商家主体
- 门店主体
- 老板账号
- 默认员工权限
- 商家钱包
- 默认链路配置
- 默认门店订单配置
- 渠道归属快照
