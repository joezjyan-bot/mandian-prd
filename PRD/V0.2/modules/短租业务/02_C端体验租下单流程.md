# C 端体验租下单流程

> **Stage 6 术语同步(2026-05-27)**: 本文档已按 Stage 6 统一为商家、联营、平台订单、订单结算款、我的钱包、履约中、逾期费用、留购、保证金等展示术语；数据库字段、API 路径、英文枚举保持不变。

> 参考头部短租平台客户端路径:先选城市/门店,进入门店详情后可切长租/短租 Tab,选短租方案,补骑行人和实名信息,支付后进入待确认、待取车、已取车、还车/续租。

---

## 1. 入口

| 入口 | 说明 |
|---|---|
| 体验租 Tab | 主入口,展示城市、品类、附近门店(默认列表 + 地图模式切换)、租赁周期 |
| 扫码下单 | 扫门店体验租二维码,自动带入门店、品类和可租方案 |
| 门店详情 | 从附近门店列表进入,展示该门店可租车型 |
| 活动 banner | 跳转体验租品类或指定门店 |

体验租入口不得跳入安心用长租办单助手。

## 2. 体验租首页

页面字段:

- 顶部:城市切换(必选)+ 搜索框
- Banner 轮播位
- 品类标签栏(V1 仅电动车,主打标识 ⭐;V2 扩展手机/相机/户外等)
- 视图模式切换按钮:[列表 默认] / [地图]
- 取车地点或附近门店列表
- 距最近门店距离
- 租车周期标签(时租/日租/周租/月租,可在前台配置中心开关)
- 立即选车
- 扫码下单
- 活动 banner
- 门店营业状态
- 官方反诈和用车安全提示

筛选逻辑:

1. 客户授权定位后,按 GCJ-02 经纬度查询附近门店。
2. 默认半径可配置,建议 20 km。
3. 只展示**已开通体验租**的门店(店铺管理 §5 决定)+ 当前品类有可租库存或可预约方案的门店。
4. 按距离、库存、营业状态、价格可排序。

**视图模式**(P0 决策 2026-05-25):
- V1 默认列表模式
- 顶部突出"地图模式"按钮,客户可一键切换
- 地图模式:Pin 显示门店位置,点击 Pin 展示门店信息卡片
- 装修配置在 `modules/运营端/前台配置中心/03_装修管理_体验租.md` §4.2-4.3

## 3.5 门店详情(含长租/短租双 Tab) ⭐ V0.2 关键设计

**P0 决策(2026-05-25)**:客户进入门店之后,可以切换"长租 / 短租"两个 Tab 看该门店在两条业务线的内容。

### 3.5.1 双 Tab 显隐规则

| 门店开通业务线 | C 端门店详情 |
|---|---|
| 仅开通短租 | 只显示短租板块,**不显示 Tab** |
| 仅开通长租 | 只显示长租板块,**不显示 Tab** |
| 双开通(短租+长租) | **显示长租/短租双 Tab** |
| 都未开通 | 不应进入此页(异常,降级到城市未开通) |

门店业务线开通状态由 `modules/运营端/店铺管理/04_门店开通与城市管理.md` 决定。

### 3.5.2 默认 Tab(根据入口决定)

| 进入入口 | URL 参数 | 默认 Tab |
|---|---|---|
| 从体验租首页门店列表进入 | `from=experience_rent` | 短租 |
| 从安心用首页门店列表进入 | `from=assurance_rent` | 长租 |
| 从扫码进入(门店体验租二维码) | `from=qrcode_short` | 短租 |
| 从扫码进入(门店安心用二维码) | `from=qrcode_long` | 长租 |
| 客户主动切换 Tab | - | 以当前 Tab 为准 |

### 3.5.3 短租 Tab(默认状态)


短租 Tab 内容:

- 取车时间选择(默认今日,可改预约)
- 租车周期选择(时租/日租/周租/月租,该门店开通的周期)
- 车型卡列表(车型图+卖点+起租价+立即租)

### 3.5.4 长租 Tab

```
┌─────────────────────────────────┐
│ 门店头图 + 信息(同短租)            │
├─────────────────────────────────┤
│ ┌─────────┬─────────┐           │
│ │ 短租 ○  │ 长租 ●  │ ← Tab       │
│ └─────────┴─────────┘           │
├─────────────────────────────────┤
│ 长租筛选:[品类 ▼] [品牌 ▼] [期数▼]│
├─────────────────────────────────┤
│ [商品卡 1] iPhone 15              │
│  6 期免押 / 12 期分期              │
│  月供 ¥299 起     [立即租]        │
├─────────────────────────────────┤
│ [商品卡 2] ...                    │
└─────────────────────────────────┘
```

长租 Tab 内容:

- 筛选器(品类/品牌/期数,根据该门店的长租商品)
- 商品卡列表(商品图+名称+期数标签+月供起价+立即租)
- 点击商品进入安心用长租商品详情页(标准长租链路)

### 3.5.5 Tab 切换后的订单归属

**核心原则**:订单 `business_line` 字段以**客户实际下单时所在的 Tab** 为准。

| 场景 | 订单归属 |
|---|---|
| 从体验租入口进门店 → 默认短租 Tab → 选车下单 | `business_line=experience_rent`,走短租链路 |
| 从体验租入口进门店 → 切到长租 Tab → 选商品下单 | `business_line=assurance_rent`,走长租链路 |
| 从安心用入口进门店 → 默认长租 Tab → 选商品下单 | `business_line=assurance_rent` |
| 从安心用入口进门店 → 切到短租 Tab → 选车下单 | `business_line=experience_rent` |

**业务意义**:门店流量复用 — 客户进入电动车店发现既能短租试用又能长租买车。从安心用进的客户看到该店也支持长租电动车;从体验租进的客户看到该店也可以买车长租。

### 3.5.6 异常态

| 异常 | 展示 |
|---|---|
| 该门店未开通短租 | 短租 Tab 灰显,点击提示"该门店暂未开通短租业务" |
| 该门店未开通长租 | 长租 Tab 灰显,点击提示"该门店暂未开通长租业务" |
| 切换 Tab 后无可租商品 | 空状态:"暂无可租商品,可换个门店看看"+ 推荐附近门店按钮 |

## 4. 选择车辆(短租 Tab 下)

车辆列表展示的是**车型**,不直接暴露具体 VIN。

字段:

- 车型图片
- 车型名称(来自商品标准名 / 展示标题,详见商品管理 §5.1)
- 简短卖点
- 当前周期最低价
- 可租库存状态
- 立即租
- 保障标签
- 可选短租周期标签

选中车型后进入订单确认页,系统锁定门店、车型、价格方案快照。

## 5. 订单确认(短租)

订单确认页包含:

- 车型卡片
- 选择租车方案(短租/长租 Tab 见下文 §5.1)
- 取还信息
- 门店信息
- 联系商家
- 用户协议勾选
- 合计金额
- 确认订单
- 保障包/服务包
- 保证金/免押说明
- 到期和逾期说明
- 取车后开始计时提示

### 5.1 租车方案

订单确认页的方案选择区**也支持长租/短租切换**(同 §3.5 的双 Tab 设计):

- 该门店双业务线均开通时,**方案区显示双 Tab**:[选择短租方案] / [选择长租方案]
- 客户从短租 Tab 进入此页,默认短租
- 客户可切到长租 Tab,选择长租套餐(6/12/24 期)
- 切换 Tab 后,**重新进入对应业务线的下单流程**(短租走骑行人/人脸/保证金,长租走实名/风控/合同)

第一版支持的短租周期:

| 周期 | 示例 |
|---|---|
| 1 小时 | 固定价格 |
| 2 小时 | 固定价格 |
| 4 小时 | 固定价格 |
| 当日还 | 指定当日最晚还车时间 |
| 日租 | 按天 |
| 周租 | 后续可开关 |
| 月租 | 后续可开关 |

超过 8 小时是否按日租封顶由后台配置;提前还车默认不退租金。

### 5.2 协议校验

客户未勾选协议时,点击确认订单提示"请先阅读并同意相关协议"。

协议包括:

- 体验租服务协议
- 人脸识别服务协议
- 保证金/免押授权协议
- 控车和定位授权说明
- 隐私政策
- 用车安全和损坏赔付说明

### 5.3 费用明细

费用展示必须拆开:

| 费用 | 说明 |
|---|---|
| 租金 | 按所选短租方案计算 |
| 保障包 | 可选,未选择时不得默认收费 |
| 保证金/预授权 | 按门店、车型、客户信用和规则判断 |
| 优惠 | 优惠券、活动减免 |
| 合计应付 | 当前需支付金额 |

**C 端合规红线**:短租客户侧**不展示**门店分账、平台抽佣、渠道佣金、经营模式(自营/非自营)、资产归属方。

## 6. 骑行人资料与认证

如果当前客户未维护骑行人信息,进入骑行人信息页。

字段:

- 证件类型
- 证件号码
- 姓名
- 手机号
- 紧急联系人,短租第一版可选

提交后进入人脸认证:

1. 勾选人脸识别服务协议。
2. 调用实名/人脸服务。
3. 认证成功后回到订单支付。
4. 认证失败时保留订单草稿,不生成待接单订单。

---

## 7. 支付和保证金

支付顺序:

1. 支付租金和可选保障包。
2. 判断是否支持信用免押。
3. 免押成功则生成保证金授权记录。
4. 免押失败或未开启免押时支付保证金。
5. 支付完成后生成体验租订单和核销码。

支付方式展示:

- 全额支付
- 保证金支付/预授权
- 信用免押

## 8. 订单状态和客户动作

| 状态 | 客户可见文案 | 客户动作 |
|---|---|---|
| 待支付 | 待支付 | 继续支付、取消 |
| 保证金待支付 | 保证金待支付 | 支付保证金/发起免押 |
| 待确认 | 商家接单中 | 联系商家、取消 |
| 待取车 | 待取车 | 联系商家、导航到店、展示核销码/扫码绑车 |
| 已取车 | 使用中 | 开机、关机、定位、续租、去还车 |
| 待还车确认 | 还车确认中 | 查看还车进度、联系客服 |
| 已完成 | 已完成 | 查看保证金退款、再次租用 |
| 已取消 | 已取消 | 查看退款状态 |

门店 1 小时内未确认接单,第一版可自动取消或转人工处理,规则由后台配置。

订单详情顶部必须展示:

- 门店名称和首租/续租标签。
- 车型名称和图片。
- 租赁方案。
- 取车时间、还车时间、租期倒计时。
- 核销码(待取车展示)。
- 联系商家、导航到店。
- 当前可操作按钮。

## 9. 取车

取车方式:

1. 到店出示核销码,由门店核销并绑定车辆。
2. 扫码绑车,客户扫码车辆码后门店或系统确认。

取车成功后:

- 写入实际取车时间。
- 计算租赁结束时间。
- 绑定车辆 VIN/SN/设备号。
- 调用中控开机或允许客户开机。
- 订单进入已取车。

---

## 10. 用车和续租

已取车状态展示:

- 租期剩余倒计时
- 到期时间
- 开机
- 关机
- 闪灯鸣笛
- 查看定位
- 更多控车能力
- 去还车
- 去续租

逾期后:

- 客户端提示已逾期或即将逾期。
- 可隐藏部分控车按钮,要求联系商家或运营端处理。
- 逾期费第一版可只做后台手动扣保证金,不在 C 端展示复杂自动金额。

续租规则:

1. 只允许已取车订单续租。
2. 当前订单存在未结费用时不能续租。
3. 续租生成新短租订单。
4. 首租保证金可复用到续租订单。
5. 续租开始时间等于上一单结束时间。

---

## 11. 还车

第一版只支持到店还车。

客户点击"去还车":

1. 展示导航到店。
2. 到店后门店确认还车。
3. 门店检查车辆状态。
4. 系统计算是否有扣费。
5. 保证金全部退还或部分退还。
6. 订单完成。

后续再扩展异地还车、调度费、自动还车点选择。

## 12. C 端底部导航关系(V0.2 调整)

**C 端底部 4 Tab 固定**(详见 `modules/C端/01_C端视觉规范.md` §6.1):

```text
[安心用]   [体验租]   [客服]   [我的]
shield     sparkles   headph   user

```

**订单中心放在"我的"里**(不是独立 Tab),长租短租订单合并展示,顶部 Tab 筛选业务线。

```text

  -> 城市/门店/车型/方案

  -> 切换 Tab 后下单走对应业务线链路

  -> 订单中心(长租/体验租合并,Tab 筛选)
  -> 骑行人、实名、人脸、协议、设置

  -> IM 会话、工单、常见问题
```

**核心约束**:

- 体验租入口不进入安心用长租办单助手
- 长租办单助手不展示体验租入口
- 门店详情双 Tab 是**前台展示层**,后端订单类型严格隔离

## 13. 门店端对接点

C 端体验租订单会给门店端生成待办:

| C 端状态 | 门店端动作 |
|---|---|
| 待确认 | 接单/拒单 |
| 待取车 | 核销码核销、选择并绑定唯一设备码 |
| 已取车 | 查看用车中订单、必要时协助控车 |
| 待还车确认 | 车辆验收、损坏扣款、保证金处理 |
| 已完成 | 查看历史和再次出租 |

门店员工权限需细分:

- 可接单。
- 可核销取车。
- 可确认还车。
- 可发起保证金扣款申请。
- 不可修改收款账户。

## 15. 告知内容

体验租 C 端必须明确告知:

1. 取车后开始计算订单时长。
2. 到期前需要还车或续租。
3. 逾期可能影响控车能力和保证金处理。
4. 车辆损坏、丢失、违规使用可能产生扣款。
5. 控车、定位、人脸识别均需客户授权。
6. 门店和平台不会要求客户向私人账户转账。
