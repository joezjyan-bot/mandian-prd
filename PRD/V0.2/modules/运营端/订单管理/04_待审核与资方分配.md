# 运营端待审核与资方分配

> 页面级 PRD 草案。
> 参考：`01_订单列表.md`、`03_订单详情.md`、`03_商品办单助手订单核心链路.md`。

---

## 1. 页面说明

| 项 | 内容 |
|---|---|
| 页面名称 | 运营端待审核与资方分配 |
| 所属端 | 运营端 |
| 入口路径 | 订单管理 > 待审核 / 待分配资方 |
| 使用角色 | 审核客服、风控客服、运营主管、财务、平台管理员 |
| 核心目标 | 集中处理分红订单、平台订单的资料审核、风控判断、资方分配、补资料、改套餐和下一步链路发起 |

该页面是分红订单和平台订单的主工作台。门店订单默认不进入平台审核队列，除非发生客诉、异常介入、黑名单命中、平台抽佣异常或平台管理员手动转入。

---

## 2. 订单范围

| 订单类型 | 是否进入待审核 | 说明 |
|---|---|---|
| 门店订单 | 默认不进入 | 商家/门店自审；运营端只监管和异常介入 |
| 分红订单 | 进入 | 门店出设备或部分资金，平台分配资方补足资金并审核 |
| 平台订单 | 进入 | 门店把订单推给平台，平台匹配资方并主控审核财务 |

平台订单是原「低费率订单」的新命名。新系统页面、接口和文案均使用「平台订单」。

## 3. 页面结构

```
┌──────────────────────────────────────────────────────────────────────┐
│ 顶部指标：待审核 待分配资方 待补资料 待客户授权 待合同 待支付 异常     │
├──────────────────────────────────────────────────────────────────────┤
│ 筛选：订单号 / 订单类型 / 商家门店 / 资方状态 / 客户 / 商品 / 时间     │
│      风控状态 / 资料状态 / 合同状态 / 支付状态 / 审核人 / 渠道         │
├──────────────────────────────────────────────────────────────────────┤
│ 左侧队列：全部 待接单 待资料审核 待分配资方 待风控 待补资料 异常       │
├──────────────────────────────────────────────────────────────────────┤
│ 订单卡片：订单摘要 + 风控摘要 + 价格方案 + 资方状态 + 当前待办 + 操作   │

```

页面支持两种工作方式：

| 方式 | 场景 | 要求 |
|---|---|---|
| 队列模式 | 审核客服日常处理 | 按下一步待办分队列，避免漏单 |
| 搜索模式 | 主管、财务、运营追单 | 按多条件搜索，快速定位订单 |

## 4. 顶部指标

| 指标 | 说明 |
|---|---|
| 待审核 | 已提交但未给出审核结论 |
| 待分配资方 | 分红/平台订单尚未绑定资方 |
| 待补资料 | 客户或商家需补充资料 |
| 待客户授权 | 风控/征信/代扣/合同已发起但客户未完成 |
| 待合同 | 审核通过但合同未发起或未签署 |
| 待支付 | 客户未完成首期、服务费、公证费或补缴 |
| 异常 | 回调失败、黑名单命中、支付不匹配、资料异常 |

指标必须可点击跳转到对应队列，且数量按当前用户权限实时统计。

## 6. 审核队列

| 队列 | 进入条件 | 主操作 |
|---|---|---|
| 待接单 | 订单提交后未分配审核人员 | 接单、分配审核人员 |
| 待资料审核 | 客户资料已提交 | 查看资料、通过、补资料、拒绝 |
| 待分配资方 | 分红/平台订单未绑定资方 | 选择资方、生成分账规则 |
| 待风控 | 需要发起或查看风控 | 发起授权、查看报告、写审核意见 |
| 待补资料 | 客户或商家需补资料 | 查看补充状态、催办、重新审核 |
| 待签约支付 | 审核通过后合同/支付未完成 | 发起合同、公证、收款码、代扣签约 |
| 异常队列 | 黑名单、回调失败、支付不匹配、资料冲突 | 主管复核、重试回调、转人工 |

队列不是最终订单状态，而是运营工作台视角的待办归类。

## 7. 订单卡片字段

| 区域 | 展示字段 |
|---|---|
| 头部 | 下单时间、订单号、订单类型、商家、门店、渠道、当前状态、下一步待办 |
| 客户 | 姓名脱敏、手机号脱敏、实名状态、资料状态、定位摘要、历史订单 |
| 商品 | 商品名称、规格、全新/二手、租赁模式、计费单位、租期、设备价 |
| 价格 | 首期实付、后期账单、押金、服务费、公证费、留购价 |
| 风控 | 风控状态、黑名单命中、报告摘要、审核建议 |
| 资方 | 资方状态、资方名称、出资比例、出资金额、分账规则版本 |
| 链路 | 授权、合同、公证、支付、代扣、发货前置条件 |
| 日志 | 最近操作、最近回调、备注、超时提醒 |

订单卡片必须能直接看清楚是否缺资料、缺资方、缺合同、缺支付，避免客服必须点进详情才知道下一步。

## 8. 审核流程

```
订单提交
  ↓
进入待审核队列
  ↓
接单或分配审核人员
  ↓
查看客户资料 + 商品价格方案 + 风控摘要
  ↓
是否需要补资料 ── 是 ──> 发起补资料待办 ──> 客户/商家补充 ──┐
  ↓ 否                                                       │
是否需要改套餐/改价 ── 是 ──> 生成新价格方案并保留旧快照      │
  ↓ 否                                                       │
分红/平台订单分配资方                                       │
  ↓
发起授权/合同/公证/支付链路
  ↓
审核通过 / 审核拒绝 / 转异常复核
```

审核通过不一定直接进入待发货。系统要根据缺失节点计算下一步：合同未签则待签约，支付未完成则待支付，代扣未签约则待绑卡。

## 9. 资方分配规则

### 9.1 资方前置

资方必须先在 `资方管理` 中配置完成，至少包括：

| 字段 | 说明 |
|---|---|
| 资方名称 | 展示和筛选使用 |
| 资金账户 | 分账和提现账户 |
| 支持订单类型 | 分红订单、平台订单或两者 |
| 支持类目 | 可投放的商品类目 |
| 可用额度 | 当前可分配额度 |
| 分账规则 | 默认分账账户和结算周期 |
| 状态 | 启用、停用、冻结 |

### 9.2 分红订单分配

分红订单由门店填写设备价和需求配资比例。例如设备价 5000，需求配资比例 80%，则资方出资 4000，门店等效出资 1000，后续客户每一笔还款按 80/20 先分账，再分别扣平台抽佣。

| 字段 | 说明 |
|---|---|
| 设备价 | 办单助手传入，可审核时调整 |
| 资方出资比例 | 20% 到 80% 下拉选择，默认使用办单助手提交值 |
| 门店占比 | 100% 减资方出资比例 |
| 资方出资金额 | 设备价乘资方出资比例 |
| 门店等效出资 | 设备价减资方出资金额 |
| 平台抽佣 | 默认 2%，可按商家/资方/订单配置 |

### 9.3 平台订单分配

平台订单由平台或资方全额承接资金和审核主控。门店只作为订单来源、发货主体或协作门店，不参与配资充值。

| 字段 | 说明 |
|---|---|
| 资方 | 可由审核客服选择 |
| 出资比例 | 默认 100% 资方/平台资金 |
| 门店收益 | 默认无资金收益，可按渠道或服务规则给佣金 |
| 渠道佣金 | 如果来自渠道推广，按渠道规则结算 |
| 发货主体 | 默认门店，可配置平台/资方发货 |

### 9.4 资方变更

| 节点 | 是否允许变更 | 规则 |
|---|---|---|
| 合同发起前 | 允许 | 普通资方分配权限可操作 |
| 合同已发起未签 | 谨慎允许 | 需要重新发起合同或补充确认 |
| 合同已签署 | 原则不允许 | 如必须变更，需要主管审批和补充合同 |
| 已支付/已分账 | 不允许普通变更 | 走冲正、退款或财务调整流程 |

## 10. 客服 IM 联动

商家 PC 端可以查看自己发起的平台订单进度，并在待审核或补资料节点点击「联系客服」。

| 场景 | 规则 |
|---|---|
| 商家联系平台客服 | 调起 IM，并自动推送订单摘要 |
| 审核客服联系商家 | 从审核台发起会话，带订单号、缺失资料、审核意见 |
| 客服联系客户 | 通过客户侧小程序/短信/IM 触达，记录触达日志 |

推送给 IM 的订单摘要应包括订单号、订单类型、商家门店、客户脱敏信息、商品、价格方案、当前待办、需要处理的问题，不推送身份证号、银行卡号等敏感明文。

## 11. 审核结果

| 结果 | 必填 | 后续动作 |
|---|---|---|
| 审核通过 | 审核意见 | 根据链路缺失计算下一步待办 |
| 审核不通过 | 拒绝原因、通知对象 | 通知客户/商家，按规则关闭或退回 |
| 要求补资料 | 补充资料项、截止时间 | 生成客户或商家待办 |
| 转主管复核 | 复核原因 | 进入异常队列 |
| 暂挂 | 暂挂原因、预计处理时间 | 保持待审核但标记超时规则 |

拒绝原因要结构化，便于后续统计，例如资料不符、风控拒绝、黑名单命中、资方拒绝、客户放弃、商家撤回、重复下单。

所有操作日志都要保留操作前值、操作后值、原因、操作人、时间、来源 IP。

## 13. 异常处理

| 异常 | 处理 |
|---|---|
| 资方额度不足 | 禁止分配，提示更换资方或调整额度 |
| 资方被停用 | 已分配未签约订单进入异常队列 |
| 风控报告未返回 | 可重试或人工暂挂 |
| 客户补资料超时 | 提醒商家/客户，超时后可关闭 |
| 支付已完成但合同未签 | 禁止发货，提示补签或退款处理 |
| 合同已签但支付失败 | 保留合同状态，订单待支付 |
| 黑名单命中 | 必须主管复核或强制填写审核意见 |
| 客服改价后客户未确认 | 禁止审核通过到签约/支付 |

## 15. 待确认

1. 资方是否需要审批确认后才算分配成功，还是运营分配即生效。
2. 分红订单资方比例是否固定 20% 到 80%，是否允许特殊商家配置超出范围。
3. 平台订单是否允许给门店服务佣金；如果允许，应在渠道/佣金模块单独配置。
